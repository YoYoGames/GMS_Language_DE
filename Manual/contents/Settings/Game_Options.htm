Spiel-Optionen

Die Spieloptionen sind in zwei Abschnitte unterteilt, die Hauptoptionen und die Plattformeinstellungen. Die Hauptoptionen beginnen mit den Allgemeinen Optionen, in denen Sie bestimmte Dinge einstellen können, die sich auf das Spiel auswirken, unabhängig von der Plattform, für die es für die Ausführung auf der Plattform ausgelegt ist.

Das Fenster hat die folgenden Bereiche:

  • DnD™ als Standard-Skripttyp verwenden: Wenn diese Option aktiviert ist, werden alle neuen Skripte und Objekt-Events mit dem DnD™-Editorund nicht mit dem Code-Editor geöffnet.
  • Spiel-Frames pro Sekunde: Dieser Wert legt die Grenze für die Anzahl der Frames pro Sekunde fest, die GameMaker Studio 2 versuchen soll, bei der Ausführung Ihrer Spiele beizubehalten. Dieser Wert repräsentiert die Anzahl der Durchläufe der Spielschleife in einer Sekunde durchläuft (eine einzelne Spielschleife wird als ein einziger "Frame" betrachtet) und ist auf einen Standardwert von 60 eingestellt, obwohl häufig auch 30 verwendet wird. Es gibt viele Argumente darüber, welche Raumgeschwindigkeit die ideale ist, aber die tatsächliche Antwort hängt davon ab, welche Art des Spiels, das Sie machen, der Größe des Raums, der Anzahl der Instanzen und wie intensiv der Code ist. Wenn Sie z. B. einen großen Raum und eine Raumgeschwindigkeit von 30 haben, kann das Spiel "abgehackt" wirken, denn je größer der Bereich ist, den Sie betrachten, Je größer der Bereich ist, auf den Sie blicken, desto einfacher ist es für das Auge, niedrige Raumgeschwindigkeiten wahrzunehmen. Wenn das Spiel jedoch sehr intensiv ist, kann die Einstellung der Raumgeschwindigkeit auf 120 dazu führen, dass das Spiel "hinkt", da die fps aufgrund der Verarbeitungsmenge sinken. Seien Sie also vorsichtig mit diesem Wert und versuchen Sie, ihn immer so einzustellen, wie es Ihr Spiel benötigt.
  • Standard-Zeichenfarbe: Die Standardfarbe zum Zeichnen von Text oder Foren (und anderen Dingen), wenn keine Farbe per Code eingestellt wurde.
  • Standard-Zeichenfarbe Außerhalb des Raumbereichs: Die Standardfarbe, die verwendet wird, um den Hintergrund eines Bereichs außerhalb des Spielraums zu löschen.
  • Projektbilder generieren: Wenn Sie hierauf klicken, wird das Werkzeug "Projektbild-Generator" geöffnet.
  • Projektstartdatum: Das Datum, an dem das Projekt gestartet wurde.
  • Projektverwendungszeit: Die Zeitspanne, die vergangen ist, seit Sie das Projekt initiiert haben.
  • Projekt-GUID: Dies ist der eindeutige Bezeichner, der für jedes Projekt in GameMaker Studio 2 erstellt wird. Wenn Sie diese ändern möchten, können Sie auf die Schaltfläche Neue GUID erstellen klicken, um eine neue zu erzeugen.
  • Mipmaps für separate Texturseiten generieren: Mit dieser Option lässt GameMaker Studio 2 Mipmaps für die Sprites erzeugen, die in den Textureinstellungen des Sprite-Editors die Option "Separate Texturseite" markiert haben. Aktivieren von Mipmapping bedeutet, dass die ursprüngliche Texturseite für das Sprite skaliert und in mehrere Auflösungen innerhalb der Texturdatei gefiltert werden soll. Normalerweise ist dies deaktiviert, aber wenn Sie Ansichten verwenden, um große Bereiche des Spielraums zu vergrößern oder zu verkleinern, oder wenn Sie mit einem 3D-Spiel arbeiten, können Sie Wenn Sie jedoch mit Ansichten arbeiten, um große Bereiche des Spielraums zu vergrößern oder zu verkleinern, oder wenn Sie mit einem 3D-Spiel arbeiten, sollten Sie dies aktivieren und dann die Mipmapping-Funktionen verwenden, um einzustellen, wie das Spiel mit den verschiedenen Mipmap-Ebenen umgeht. Beachten Sie, dass Sie, wenn Sie Sprites nicht als individuelle Texturseiten kennzeichnen, trotzdem Mipmapping auf Basis der einzelnen Texturgruppen im Texturgruppen-Manager aktivieren. Diese Option ist standardmäßig ausgeschaltet.
  • Steam ID: Wenn Sie ein Spiel erstellen, das über Steam vertrieben werden soll, müssen Sie hier die Steam App ID angeben. Stellen Sie sicher, dass Sie den Pfad zum Steamworks-SDKin den Plattform-Voreinstellungen hinzugefügt haben, wenn Sie diese Option verwenden, und stellen Sie sicher, dass Sie im Steamworks-Dashboardalles richtig eingestellt haben.
  • Aktivieren Sie die Versionskontrolle: Wenn Sie möchten, dass das Projekt unter SCM (Source Control Management) steht, müssen Sie diese Option aktivieren. Standardmäßig ist es ausgeschaltet (siehe hier für weitere Informationen zur Versionskontrolle).
  • Autor: Der Name des Spieleautors (oder Herausgebers oder Studios).
  • Project Last Changed: Dieser Bereich zeigt einfach den Datums-/Zeitstempel für die letzte Änderung des Projekts an (dies ist nicht editierbar).
  • Spine-Lizenz akzeptieren: Hier können Sie das Kontrollkästchen aktivieren, um die Spine-Laufzeitlizenz anzuzeigen und zu akzeptieren. Dies erlaubt Ihnen, Spine-Sprites in Ihrem Spiel zu verwenden (diese Lizenz wird auch angezeigt, wenn Sie zum ersten Mal versuchen, ein Spine-Sprite zu Ihrem Projekt hinzuzufügen und die Lizenz hier nicht vorher angekreuzt haben).

Im Asset-Browser stehen Ihnen je nach den für Ihre Lizenz verfügbaren Zielplattformen auch andere Spieloptionen zur Verfügung, die möglicherweise nicht alle verfügbar oder sichtbar sind.

Es gibt die folgenden plattformspezifischen Spieloptionen:

  • macOS
  • Ubuntu
  • Windows UWP
  • HTML5
  • Android
  • iOS
  • tvOS

HINWEIS: Die verschiedenen Optionen für Konsolenspiele sind hier nicht aufgelistet, da es verschiedene rechtliche Einschränkungen gibt, die von den verschiedenen Firmen eingehalten werden. Wenn Sie eine Konsolenlizenz haben, dann können Sie diese Informationen vom YoYo Games Helpdesk erhalten, aber Sie müssen zuerst YoYo Games mit Ihrer Konsolen-ID per E-Mail kontaktieren und die Erlaubnis zur Ansicht der verschiedenen Abschnitte anfordern.

Dies sind Legacy-Optionen, da neuere Lizenzen die Fire-Plattform direkt mit den Android Spiel-Optionen.
