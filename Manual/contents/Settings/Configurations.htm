Konfigurationen

Aufgrund der Anforderungen der plattformübergreifenden Entwicklung und der Notwendigkeit, dass ein Produkt für verschiedene Clients bzw. Geräte oder Plattformen verfügbar sein und angepasst werden muss, enthält GameMaker Studio 2 Konfigurationen, die die Arbeit vereinfachen. Mit diesem System können Sie schnell zwischen verschiedenen Splash-Screens, Icons und eingebundenen Dateien umschalten und jederzeit nur die für die Zielplattform oder den Client relevanten testen oder exportieren - und das alles aus demselben Basiscode.

Um den unterschiedlichen Anforderungen von Clients und Plattformen gerecht zu werden, können Sie den Konfigurationseditor verwenden, der über das Menü Extras im Asset-Browser zugänglich ist:

Alle in GameMaker Studio 2 erstellten Spiele beginnen mit einer "Standard"-Konfiguration, die beim ersten Öffnen im Editor aufgelistet ist wird:

Um Ihre Konfigurationen einzurichten, wird empfohlen, dass Sie zunächst diese "Standard"-Konfiguration vorbereiten, indem Sie alle konfigurierbaren Assets für das Projekt aufnehmen. Gehen Sie dazu einfach die folgende Liste von Optionen durch und setzen Sie sie auf das, was standardmäßig erforderlich ist:

  • Erweiterungsoptionen: Setzen Sie die verschiedenen "Kopieren nach..."-Felder auf die für die Konfiguration erforderlichen Plattformen. Alle hier erstellten "Proxy"-Dateien haben ihre "Ziel"-Plattform auch mit der Konfiguration verknüpft.
  • Sprite-Editor: Legen Sie die Texturgruppe fest, zu der die Sprites hinzugefügt werden sollen.
  • Tileset-Editor: Legen Sie die Texturgruppe fest, zu der die Tilesets hinzugefügt werden sollen.
  • Schriftarten-Editor: Legen Sie die Texturgruppe fest, zu der die Schriftarten hinzugefügt werden sollen.
  • Texturgruppen-Manager: Hier können Sie alle Eigenschaften pro Konfiguration einstellen, einschließlich der übergeordneten Gruppe, zu der eine Texturgruppe gehört.
  • Audiogruppen-Manager: Die Sounds, die den verschiedenen Audiogruppen zugeordnet sind, sind hier konfigurierbar.
  • Enthaltene Dateien: Setzen Sie das Feld "Kopieren nach..." auf die gewünschten Plattformen für die Konfiguration.

Nachdem Sie die Standardkonfiguration eingerichtet haben, können Sie im Konfigurations-Editor eine neue Konfiguration hinzufügen, indem Sie mit der rechten Maustaste auf den Ordner "configurations" klicken und die Option " Add Config" (Konfiguration hinzufügen) wählen oder auf die Schaltfläche "Create" (Erstellen) am unteren Rand des Fensters klicken. Die neue Konfiguration wird als "Kind" der Standardkonfiguration angezeigt - alle benutzerdefinierten Konfigurationen sind Kinder der Standardkonfiguration - und Sie können ihr einen passenden Namen geben.

Um die Konfiguration zu bearbeiten, müssen Sie entweder auf doppelklicken oder die rechte Maustaste Menüoption " SetConfig" verwenden, wodurch die Konfiguration im Editor ausgewählt wird. Dies geschieht, indem Sie einfach die oben aufgeführten Optionen durchgehen und auf die gewünschten Einstellungen für die neu erstellte Konfiguration. GameMaker Studio 2 speichert diese Werte automatisch für die Config, und Sie können jederzeit zwischen den Configs wechseln, indem Sie einfach die Menüoption RMB verwenden oder auf doppelklicken.

Weitere Konfigurationen können entweder als Kinder der Standardkonfiguration oder als Kinder der von Ihnen erstellten benutzerdefinierten Konfigurationen hinzugefügt werden, je nachdem, welche Konfiguration beim Erstellen im Editor hervorgehoben wurde. In dem Beispielbild unten sehen Sie, dass wir drei benutzerdefinierte Konfigurationen definiert haben, von denen zwei Kinder der Standardkonfiguration sind und eine weitere, die das Kind einer benutzerdefinierten Konfiguration ist:

Es ist erwähnenswert, dass Sie jede beliebige Konfiguration mit jeder beliebigen Zielplattform beim Kompilieren oder Testen Ihrer Projekte verwenden können. Dazu wählen Sie im Abschnitt " Config" des Target Manager die gewünschte Konfiguration aus:

Sie können den Konfigurationseditor auch jederzeit vom Target Manager aus öffnen, indem Sie auf die Menüschaltfläche oben rechts klicken (wie in der obigen Abbildung hervorgehoben).

Sobald Sie eine Konfiguration auf diese Weise eingestellt haben, wird jedes Mal, wenn Sie zu einem Ziel kompilieren, entweder zum Testen oder zum Erstellen eines endgültigen Anlagenpakets, die gewählte Konfiguration verwendet. Weitere Informationen über das Fenster "Ziele" finden Sie im Abschnitt über Kompilieren.
