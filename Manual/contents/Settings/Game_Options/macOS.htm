macOS Spieloptionen

In diesem Abschnitt werden die verschiedenen Optionen beschrieben, die Ihnen zur Verfügung stehen und die steuern, wie Ihre macOS-Spielprojekte kompiliert werden. Die verschiedenen Abschnitte sind:

Allgemein

Geben Sie im Abschnitt " Product Information" als erstes den Anzeigenamen Ihres Spiels ein dann geben Sie die Versionsnummer, die App-ID, die App-Ausgabedatei und die Copyright-Angaben ein. Die App-Ausgabe ist der Speicherort, an den die Installationsdateien Ihres Spiels auf Ihrem Mac-Entwicklungsrechner Rechner. Wenn Sie Ihre .app.zip erstellen, kann diese Datei überall hingehen (der Speicherort, den Sie beim Speicherdialog angeben, ist nicht die einzige Kopie, die von dieser .app.zip erstellt wird), aber die .pkg und andere Dateien werden in diesem Ordner belassen, so dass Sie lokale Installationen über Terminal testen können, um den Kauf im Mac Store zu simulieren usw..

Hier müssen Sie auch die Team-Kennung angeben, die Sie zum Signieren der endgültigen Anwendung, die GameMaker Studio 2 für Sie erstellt, verwenden möchten. Die Einstellung hier überschreibt die Team-Kennung, die Sie in den macOS-Einstellungen angegeben haben. Nach der Team-Kennung haben Sie auch die Möglichkeit, die Signier-Kennung anzugeben, die von Apple für alle Nicht-Appstore-Anwendungen (wie z. B. Steam-Spiele ) benötigt wird.

Die nächste Option ist für die Aktivierung von Dampf in Ihrem Spiel. Wenn Sie diese Option aktivieren, stellen Sie sicher, dass Sie das Steamworks-SDKinstalliert haben (siehe hier), und dass Sie die Steam-App-ID in den allgemeinen Optionen für das Spiel hinzugefügt haben.

Schließlich haben Sie die Möglichkeit, die Dateisystem-Sandbox zu deaktivieren. Wenn Sie diese Option aktivieren, wird die GameMaker Studio 2-Sandbox für das Speichern und Laden von Dateien deaktiviert, so dass Ihre Spiele von überall auf dem System, auf dem das Spiel läuft, auf Dateien zugreifen können. Diese Option ist standardmäßig nicht aktiviert, da wir empfehlen, die GameMaker Studio 2-Sandbox für Dateien auf allen Systemen beizubehalten, und wenn sie aktiviert ist, sind einige Speicher-/Ladebereiche je nach den Berechtigungen auf Betriebssystemebene möglicherweise immer noch nicht verfügbar. Für weitere Informationen zur Sandbox finden Sie auf den Seiten über das GameMaker Studio 2-Dateisystem.

Grafiken

Die Grafikoptionen sind diejenigen, die Sie konfigurieren sollten, um festzulegen, wie Ihr Spiel die Grafikkarte Ihres Ziel-Macs verwendet. Die folgenden Optionen sind enthalten, die Sie ändern können:

  • Menü und Dock im Vollbildmodus zulassen: Wenn diese Option aktiviert ist, werden das OS-Menü und das Dock angezeigt, wenn sich das Spiel im Vollbildmodus befindet. Diese Option ist standardmäßig ausgeschaltet.
  • Cursor anzeigen: Wenn diese Option aktiviert ist, wird der normale macOS-Cursor angezeigt. Wenn Sie die Option deaktivieren, wird kein Cursor angezeigt, es sei denn, Sie haben in Ihrem Spielcode einen erstellt. Diese Option ist standardmäßig ausgeschaltet.
  • Vollbild starten: Wenn dies aktiviert ist, wird das Spiel im Vollbildmodus gestartet, andernfalls wird es im Fenstermodus gestartet. Dies ist standardmäßig ausgeschaltet.
  • Vollbildumschaltung zulassen: Wenn diese Option aktiviert ist, kann der Benutzer mit den Standard-MacOS-Tastenkombinationen von Vollbild auf Fenster umschalten und wieder zurück. Dies ist standardmäßig ausgeschaltet.
  • Interpoliert Farben zwischen Pixeln: Schaltet die lineare Interpolation ein, die im Grunde die Pixel "glättet". Für scharfe Pixelgrafiken sollte sie ausgeschaltet sein, aber wenn Sie schöne Alpha-Mischungen und geglättete Kanten haben, ist es besser, sie eingeschaltet zu lassen. Standardmäßig ist dies ausgeschaltet.
  • Synchronisation verwenden, um Tearing zu vermeiden: Hiermit schalten Sie v-sync ein oder aus. Beachten Sie, dass wenn Sie ein Spiel mit einer Raumgeschwindigkeit von 120 haben und der Spieler einen Monitor mit einer Bildwiederholrate von 60 hat, wird durch das Einschalten dieser Option die Spielgeschwindigkeit ebenfalls auf 60 festgelegt. Diese Option ist standardmäßig ausgeschaltet.
  • Größenänderung des Fensters zulassen: Wenn diese Option aktiviert ist, kann der Benutzer die Größe des Spielfensters ändern (die Option "Randloses Fenster" muss deaktiviert sein, damit dies funktioniert). Diese Option ist standardmäßig ausgeschaltet.
  • Enable Retina (Retina aktivieren): Wenn Sie diese Option aktivieren, wird der hintere Puffer auf die höhere (tatsächliche) Auflösung eingestellt, wenn das Spiel auf einem Retina-fähigen Monitor ausgeführt wird, während die Deaktivierung den hinteren Puffer auf die scheinbare (niedrigere) Auflösung einstellt. Das bedeutet, dass das anfängliche Spielfenster auf die doppelte Breite und Höhe des Raums/Ansichtsport eingestellt wird, auf den der erste Raum eingestellt ist. Dadurch wird die Anwendungsoberfläche jedoch NICHT entsprechend skaliert, und wenn Sie also möchten, dass Ihr Spiel das Wenn Sie also möchten, dass Ihr Spiel das Retina-Display optimal nutzt, sollten Sie die Anwendungsoberfläche so einstellen, dass sie der Fenstergröße entspricht (beachten Sie, dass dadurch die Pixel entlang der Breite und Höhe verdoppelt werden, aktivieren Sie dies also nur und skalieren Sie die Anwendungsoberfläche, wenn Ihr Spiel von der erhöhten Auflösung profitieren wird). von der erhöhten Auflösung profitiert).

    Diese Option ist standardmäßig ausgeschaltet, und wenn Sie sie aktivieren, wird es keinen Unterschied machen, wenn Sie das Spiel mit VM ausführen oder debuggen, da der Runner in diesem Fall bereits vordefiniert ist. Sie funktioniert nur, wenn Sie eine ausführbare Datei über VM erstellen oder YYC zum Testen oder das Spiel erstellen, da dies den Runner mit Ihren angepassten Spieloptionen neu erstellt.
  • Skalieren: Hier können Sie wählen, ob das Seitenverhältnis beibehalten werden soll (so dass ein 4:3-Raum auf einem 16:9-Bildschirm "im Letterbox-Format" dargestellt wird) oder ob das Bild vollständig skaliert werden soll (so dass es auf den gesamten Bildschirm passt).

    WARNUNG! WennSiedie Anwendungsoberfläche ausschalten, werden alle in den Windows-Spieloptionen eingestellten Skalierungsoptionen deaktiviert, bis sie wieder eingeschaltet wird. Siehe Die Anwendungsoberfläche für weitere Details.

Schließlich gibt es noch die Möglichkeit, die Größe der Texturseite einzustellen. Die Standardgröße (und die kompatibelste) ist 2048x2048, aber Sie können zwischen 256x256 bis zu 8192x8192 wählen. Es gibt es auch eine Schaltfläche mit der Bezeichnung Vorschau die die Texturseiten für diese Plattform generiert und dann ein Fenster öffnet, in dem Sie sehen können, wie sie aussehen. Dies kann sehr nützlich sein, wenn Sie sehen wollen, wie die Texturseiten aufgebaut sind strukturiert sind, und um zu verhindern, dass die Texturseiten größer (oder kleiner) als nötig sind. Weitere Informationen zu Texturseiten finden Sie hier.

HINWEIS: Beachten Sie, dass Ihr Spiel auf PCs mit niedrigeren Spezifikationen umso weniger kompatibel ist, je größer die Texturseite ist.

Bilder

Im Abschnitt "Bilder" stellen Sie die Bilder bereit, die Ihr Spiel benötigt. Für Mac müssen Sie eine Icon-Datei (im Format .png und 1024x1024px) und einen Splash Screen (im Format .png, .bmp, .jpg oder .gif ). Der Splash-Screen wird angezeigt, während das Spiel geladen wird.

Für DMG-Pakete können Sie ein Installer-Hintergrundbild bereitstellen, das als Hintergrund des DMG-Installationsprogramms angezeigt wird.

Es ist erwähnenswert, dass GameMaker Studio 2 über ein Projektbild-Generator-Tool verfügt, das verwendet werden kann, um automatisch alle Bilder zu erstellen, die für die verschiedenen Zielplattformen benötigt werden, für die Ihr Spiel kompiliert wird. Wenn Sie dieses Tool verwenden, sollten Sie die erstellten Bilder überarbeiten, um sicherzustellen, dass sie Ihren Anforderungen entsprechen.

Verpackung

Wenn Sie möchten, dass das fertige Spiel App Store Ready ist, müssen Sie die Option Build for Mac App Store aktivieren, aber beachten Sie, dass dies nur funktioniert, wenn Sie ein registrierter Entwickler sind und die notwendigen Zertifikate besitzen. Darunter befinden sich die App-Berechtigungen , die Sie nur aktivieren sollten, wenn sie zutreffend sind, da Apple Ihre App ablehnen kann, wenn sie Apple kann Ihre App ablehnen, wenn diese Optionen aktiviert sind, obwohl sie nicht benötigt werden, oder andersherum. Diese Optionen erlauben Ihrem Spiel lediglich, die Funktionen http_ und url_ zu nutzen. Es ist erwähnenswert, dass, wenn Sie GamePads in Ihrem Spiel unterstützen möchten Wenn Sie GamePads in Ihrem Spiel unterstützen möchten, sollte die Option zum Erstellen eines App Store-kompatiblen Pakets deaktiviert sein.

Schließlich müssen Sie die App-Kategorie auswählen (weitere Informationen finden Sie hier).

Sozial

Diese Option ist für die Verwendung der Apple Sign In Extension auf dem Mac. Sie können diese Erweiterung von der YoYo Spiele Marktplatz, und die Erweiterung selbst enthält eine vollständige Anleitung zur Verwendung. Wenn Sie diese Erweiterung nicht verwenden, sollten Sie diese Option nicht ankreuzen.
