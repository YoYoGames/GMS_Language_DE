Windows UWP-Spieloptionen

In diesem Abschnitt werden die verschiedenen Optionen beschrieben, die Ihnen zur Verfügung stehen und die steuern, wie Ihre UWP (Universal Windows Platform)-Spielprojekte kompiliert werden. Die verschiedenen Abschnitte sind:

Allgemein

Im allgemeinen Bereich der UWP-Optionen können Sie die Produktinformation einstellen:

  • Anzeigename: Gibt den freundlichen Namen für die App an, der den Benutzern angezeigt wird (diese Zeichenfolge ist lokalisierbar).
  • Paketname: Dies ist der Name des Pakets, das Sie erstellt haben, und sollte mit dem Namen des Entwicklerschlüssels übereinstimmen, den Sie für das Spiel erstellt haben (siehe den Abschnitt über die Paketierung unten).
  • Paket-Anzeigename: Dies ist der "Kurzname" des Pakets und sollte idealerweise mit dem Anzeigenamen übereinstimmen
  • Herausgeber-Anzeigename: Dies ist der Name Ihres Unternehmens oder der Person, die Sie als Herausgeber des Spiels zuordnen möchten.
  • Version: Die Versionsnummer Ihres Spiels.

Nachdem Sie das eingerichtet haben, können Sie die verschiedenen Ausrichtungen auswählen, in denen Ihr Spiel ausgeführt werden kann, um die beste Benutzererfahrung zu erzielen. Die verfügbaren Optionen sind:

  • Landschaft
  • Porträt
  • Landschaftlich geflippt
  • Hochformatig gespiegelt

Standardmäßig sind alle diese Optionen ausgewählt, und Sie sollten die Option deaktivieren, die Sie nicht benötigen.

Nachdem Sie die Ausrichtungsoptionen eingestellt haben, können Sie aus den folgenden Optionen die verschiedenen Berechtigungen signalisieren, die Ihre UWP-App benötigen wird:

  • Internet-Zugang aktivieren: Kennzeichnet das Spiel als mit Internetzugang ausgestattet. Dies ist standardmäßig ausgeschaltet.
  • Mikrofonberechtigung aktivieren: Markiert, dass das Spiel Zugriff auf das Mikrofon des Geräts hat. Dies ist standardmäßig ausgeschaltet.
  • IAP-Sandbox aktivieren: Kennzeichnet das Spiel als "Sandboxed In App Purchases" verwendend. Dies ist standardmäßig ausgeschaltet.

Geräte-Familien

Hier können Sie auswählen, auf welche Gerätefamilien der Windows UWP-Export bei der Erstellung Ihrer ausführbaren Datei abzielen soll. Unter Standardmäßig ist die Option "Support Universal device family" aktiviert, was bedeutet, dass die App alle Geräte und Familien unterstützt. Um die App darauf zu beschränken, nur bestimmte Gerätefamilien zu unterstützen, sollte die Option "Universal" nicht markiert sein und die entsprechenden Gerätefamilien markiert werden. Es ist auch möglich, die Universal-Gerätefamilie zu unterstützen, aber andere Gerätefamilien auf bestimmte Plattformversionen zu beschränken - in diesem Fall würden Sie die Universal-Gerätefamilie aktiviert lassen, aber auch In diesem Fall lassen Sie die Gerätefamilie "Universal" markiert, markieren aber auch die andere Gerätefamilie, für die Sie eine andere Plattformversion unterstützen möchten, und geben die Versionsangaben für diese Familie in die entsprechenden Textfelder ein. Wie diese Logik funktioniert, erfahren Sie auf der folgenden Microsoft-Supportseite:

  • Zielgerätefamilie: Beispiele

Im Allgemeinen möchten Sie die Ziel- und Mindestplattformversionen nicht verändern und sollten sie auf den Standardwerten belassen, aber wenn Sie Probleme mit der Erstellung haben - insbesondere, wenn Sie das standardmäßig eingestellte Windows-SDK nicht installieren können -, dann müssen Sie müssen Sie diese Werte auf die von Ihnen installierten Werte ändern. Wenn Sie nicht sicher sind, welche Werte Sie einstellen müssen, erstellen Sie ein neues UWP-Projekt in Visual Studio und sehen Sie sich an, welche Werte eingestellt sind.

Weitere Informationen zu Gerätefamilien finden Sie im folgenden Microsoft-Dokument:

  • Übersicht der Gerätefamilien

Grafiken

Die Grafikoptionen sind diejenigen, die Sie konfigurieren sollten, um festzulegen, wie Ihr Spiel die Grafikkarte Ihres UWP-Zielgeräts verwendet. Die folgenden Optionen sind enthalten, die Sie ändern können:

  • Interpoliert Farben zwischen Pixeln: Schaltet die lineare Interpolation ein, die im Grunde genommen Pixel "glättet". Für scharfe Pixelgrafiken sollte sie ausgeschaltet sein. Für scharfe Pixelgrafiken sollte diese Option ausgeschaltet sein, aber wenn Sie schöne Alpha-Mischungen und geglättete Kanten haben, ist es besser, sie eingeschaltet zu lassen.
  • Cursor anzeigen: Wenn diese Option aktiviert ist, wird der Standard-Betriebssystem-Cursor in Ihrem Spiel angezeigt. Wenn diese Option deaktiviert ist, wird kein Cursor angezeigt, während sich die Maus über dem Spielfenster befindet, es sei denn, Sie haben einen Cursor als Teil des Spielprojekts erstellt. des Spielprojekts erstellt haben (diese Option ist standardmäßig aktiviert).
  • Vollbild starten: Wenn dies aktiviert ist, wird das Spiel im Vollbildmodus gestartet, andernfalls wird es im Fenstermodus gestartet. Dies ist standardmäßig ausgeschaltet.
  • Allow Fullscreen Switching (Vollbildumschaltung zulassen): Wenn diese Option aktiviert ist, kann der Benutzer mit den Standard-Ubuntu-Betriebssystemkurzbefehlen vom Vollbild zum Fenster und wieder zurück wechseln. Dies ist standardmäßig ausgeschaltet.
  • Skalierung: Hier können Sie wählen, ob das Seitenverhältnis beibehalten werden soll (so dass ein 4:3-Raum auf einem 16:9-Bildschirm "im Letterbox-Format" dargestellt wird) oder ob er vollständig skaliert werden soll (das Bild wird gestreckt, damit es auf den gesamten Bildschirm passt).

    WARNUNG! Wenn Siedie Anwendungsoberfläche ausschalten, werden alle in den Windows-Spieloptionen eingestellten Skalierungsoptionen deaktiviert, bis sie wieder eingeschaltet wird. Siehe Die Anwendungsoberfläche für weitere Details.

Schließlich gibt es noch die Möglichkeit, die Größe der Texturseite einzustellen. Die Standardgröße (und die kompatibelste) ist 2048x2048, aber Sie können zwischen 256x256 bis zu 8192x8192 wählen. Es gibt es auch eine Schaltfläche mit der Bezeichnung Vorschau die die Texturseiten für diese Plattform generiert und dann ein Fenster öffnet, in dem Sie sehen können, wie sie aussehen. Dies kann sehr nützlich sein, wenn Sie sehen wollen, wie die Texturseiten aufgebaut sind strukturiert sind und um zu verhindern, dass die Texturseiten größer (oder kleiner) als nötig sind. Weitere Informationen zu Texturseiten finden Sie hier.

HINWEIS: Beachten Sie, dass Ihr Spiel auf PCs mit niedrigeren Spezifikationen umso weniger kompatibel ist, je größer die Texturseite ist.

Bilder

Im ersten Teil des Abschnitts Bilder haben Sie die Möglichkeit, die verschiedenen Kacheln festzulegen, die Ihr Spiel auf den verschiedenen Geräten verwendet, auf denen UWP laufen kann. Sie können die folgenden einstellen (und sie sollten alle Dateien im Format *.png sein):

  • Das kleine Kachelbild ist das Logobild, das neben dem App-Namen in der Suchleiste und anderen Bereichen der Benutzeroberfläche erscheint. Es sollte 44 x 44 Pixel groß sein.
  • Das mittlere Kachelbild sollte ein 71x71 Pixel großes Bild sein.
  • Das nächste Kachelbild ist die Store-Logo-Kachel und sollte 50 x 50 Pixel groß sein.
  • Die breiteBildkachel ist diejenige, die angezeigt wird, wenn die Kachel in ihrem breiten Format ist und sollte 310 x 150 Pixel groß sein. Wenn dieses Bild nicht bereitgestellt wird, kann die Kachel nur im quadratischen Format angezeigt werden und kann keine Benachrichtigungen annehmen die auf breiten Vorlagentypen basieren. Da der Benutzer entscheiden kann, welches Format die Kachel verwendet, wird empfohlen, eine breite Bildkachel einzubinden. Beachten Sie auch, dass die Kachel, wenn eine breite Bildkachel bereitgestellt wird, anfangs möglicherweise in ihrem breiten Format angezeigt wird.
  • Die große Bildkachel ist das große Live-Kachelformat und sollte 310 x 310 Pixel groß sein. Beachten Sie, dass dies nur auf Desktop-OS verwendet wird.

Der nächste Abschnitt dient zum Definieren des Spiellogos. Dies ist im Wesentlichen das Symbol, das Ihr Spiel innerhalb des UWP-Ökosystems verwenden wird, und es ist erwähnenswert, dass diese Bilder transparente .png Dateien sein können. In diesem Fall werden sie über die in diesem Abschnitt angegebene Hintergrundfarbe gezeichnet. Diese Hintergrundfarbe wird auch auf die Schaltflächenfarbe in allen App-eigenen Dialogfeldern und auf der App-Beschreibungsseite im Store auf Geräten angewendet.

Nachdem Sie das Logo eingerichtet haben, können Sie den Splash Screen für das Spiel einstellen. Dies ist der Bildschirm, der angezeigt wird, wenn Er darf nicht größer als 620 x 300 Pixel sein und kann wie das Logo eine transparente .png Datei sein, in diesem Fall wird die von Ihnen gewählte Hintergrundfarbe verwendet.

Es ist erwähnenswert, dass GameMaker Studio 2 über ein Projektbild-Generator-Tool verfügt, das verwendet werden kann, um automatisch alle Bilder zu erstellen, die für die verschiedenen Zielplattformen benötigt werden, für die Ihr Spiel kompiliert wird. Wenn Sie dieses Tool verwenden, sollten Sie die erstellten Bilder überarbeiten, um sicherzustellen, dass sie Ihren Anforderungen entsprechen.

Verpackung

GameMaker Studio 2 liefert einen Standard-Entwicklerschlüssel für die Windows UWP-Entwicklung, aber es wird empfohlen, dass Sie einen eigenen zu generieren und auf diesen hier zu verweisen. Erstellen Sie dazu ein leeres Projekt in Visual Studio und speichern Sie dann an einem sicheren Ort eine Kopie der automatisch generierten Datei *.pfx, die in das leere Projekt eingebunden wird (dies ist der Entwicklerschlüssel). Diese *.pfx Datei sollten Sie dann hier verlinken.

Nach dem Verknüpfen mit der Datei sollten Sie auf die zugehörige Schaltfläche Installieren klicken und dann den Anweisungen in der sich öffnenden Eingabeaufforderung folgen. Wenn Sie dies nicht tun, schlägt die Verwendung der Schaltflächen Ausführen/Debuggen (oder Drücken von F5) fehl und Ihr Spiel kann nicht auf der UWP-Plattform getestet werden (wenn Sie jedoch Ausführbare Datei erstellen verwenden, wird der Zertifizierungsinstallationsprozess automatisch gestartet, wenn die *.pfx noch nicht registriert wurde).

HINWEIS: Wenn es darum geht, Ihre App in den Store hochzuladen, müssen Sie die richtige *.pfx für Ihren Titel erhalten. Dies geschieht über das Verknüpfen Ihrer App mit dem Windows Store in Visual Studio mit dem Windows Store assoziiert, was Visual Studio veranlasst, eine neue *.pfx speziell für die App zu generieren. Es ist auch erwähnenswert, dass Sie jede App mit dem Windows Store in Visual Studio verknüpfen können, Das heißt, Sie können ein leeres Projekt mit der Windows Store-App verknüpfen und die generierte *.pfx einfach an einen sicheren Ort kopieren, der dann in den UWP-Spieleoptionen verwendet werden kann. Sie finden eine Anleitung wie das geht, finden Sie in der YoYo Games Knowledge Base.

In diesem Abschnitt müssen Sie auch den Namen des Publishers angeben, der zur Erzeugung des Signierzertifikats verwendet wurde. Der Eingabe sollte immer ein "CN=" vorangestellt werden, z. B.: CN=YoYoGames.

Xbox Live

Im Abschnitt Xbox Live der UWP-Spieleoptionen können Sie Ihr UWP-Projekt für die Verwendung auf der Xbox One-Konsole aktivieren. Das Aktivieren des Projekts für die Xbox erlaubt Ihnen hier Folgendes, wenn das Projekt auf einer Xbox One ausgeführt wird:

  • Anmeldung bei Xbox Live aus der App heraus
  • Verwenden Sie die Xbox Live-Benutzeridentität in irgendeiner Weise im Projekt (z. B. durch Anzeigen des Gamertags oder des Spielerbilds)

Bevor Sie jedoch diese Dinge tun können, müssen Sie sicherstellen, dass Sie auf das Kontrollkästchen Xbox Live aktivieren geklickt haben, und wenn Sie Teil des Microsoft Creators Program sind, sollten Sie auch die Option Creators Program App aktivieren. Wenn Sie Teil des Creators Program sind, müssen Sie anschließend die Titel-ID und die Dienstkonfigurations-ID eingeben (weitere Informationen dazu finden Sie finden Sie bei Microsoft).

Beachten Sie, dass Sie derzeit, um die Xbox Live-Funktionalität in Ihrem Projekt zu testen, eher Ausführbare Datei erstellen als Ausführen verwenden müssen, da die UWP-Xbox-Ausgabe das Windows Store-Signierzertifikat verwenden muss, das Sie als Teil der Verpackungs Info (siehe oben) angeben, und nicht das temporäre, das mit GameMaker Studio 2 geliefert wird, das verwendet wird, wenn Sie Ausführen drücken.

Sobald Sie XBox Live aktiviert haben, können Sie die XBox Live-Funktionen in Ihrem Projekt verwenden. Beachten Sie jedoch, dass Sie für die grundlegenden Funktionen Teil des Creator-Programms sein müssen und für die erweiterten Funktionen ein ID@XBox-Konto benötigen.
