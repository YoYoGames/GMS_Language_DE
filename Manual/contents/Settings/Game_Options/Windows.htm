Windows-Spieloptionen

In diesem Abschnitt werden die verschiedenen Optionen beschrieben, die Ihnen zur Verfügung stehen und die steuern, wie Ihre Windows-Spielprojekte kompiliert werden. Die verschiedenen Abschnitte sind:

Allgemein

Auf der Seite "Allgemein" für Windows-Spiele sollten Sie die notwendigen Informationen angeben, die Windows anzeigen soll wenn Ihr Spiel installiert wurde und läuft. Diese Informationen gehören in den Abschnitt " Product Information" und sollten vollständig ausgefüllt werden.

Im Abschnitt " Optionen" dieses Fensters können Sie wählen, wohin das Spielpaket installiert werden soll, entweder in die Ordner "%localappdata%" oder "%appdata%", sowie ob Sie die Option Copy exe to output folder und ob Sie " Copy exe to output folder" einstellen möchten. Im Allgemeinen sollten Sie diese Option nie benötigen, aber sie wird für Benutzer bereitgestellt, die verschiedene DLLs haben, die miteinander und mit Ihrem Spiel kommunizieren, und wenn sie aktiviert ist, wird Wenn sie aktiviert ist, wird die Spiel-EXE beim Testen im Ausgabeordner beibehalten, anstatt sie im Temp-Ordner abzulegen.

Als Nächstes können Sie den Windows-Ruheabstand einstellen. Diese Option bezieht sich auf die Reduzierung von Stottern, wenn Ihr Spiel auf bestimmten Systemen läuft. Grundsätzlich gilt: Wenn Ihr Spiel schneller läuft als Ihre Raumgeschwindigkeit, wird GameMaker Studio 2 für die verbleibende Zeit "schlafen". für die verbleibende Zeit "schlafen", aber dieser Schlaf kann ziemlich ungenau sein, und es kommt oft vor, dass der Schlaf länger dauert als nötig, wodurch die Frames länger brauchen, was zu Stottern führt. Um dies zu umgehen, können wir für eine kürzere Zeit schlafen und dann in einer engen Schleife für den Rest der Zeit, um die Genauigkeit zu erhöhen - obwohl das Problem beim Sitzen in einer Schleife darin besteht, dass die CPU-Auslastung steigt, was wiederum dazu führen kann, dass die CPU-Temperaturen steigen und der Lüfter schneller läuft. Standardmäßig ist dieser Wert standardmäßig auf 10 gesetzt, und in 99,99% der Fälle werden Sie damit gut zurechtkommen, aber für Low-End-Rechner oder für Rechner, auf denen viele Hintergrundprozesse laufen, ist dies möglicherweise keine ideale Lösung (oder auch für sehr High-End-Rechner) und ein Wert von 1 oder 15 oder etwas Ähnliches erforderlich sein kann. Beachten Sie jedoch, dass es sich hierbei um eine sehr systemspezifische Konfiguration handelt, und dass das, was auf Ihrem Rechner funktioniert, für einen anderen Benutzer möglicherweise nicht geeignet ist; lassen Sie daher im Zweifelsfall den Wert 10.

HINWEIS: Zusätzlich zum Ruhebereich können Sie die Leistung und den Stromverbrauch unter Windows weiter kontrollieren, indem Sie die Auflösung des Thread-Schedulers zur Laufzeit anpassen .

Die nächste Option ist, Steam für Ihr Spiel zu aktivieren. Wenn Sie diese Option aktivieren, stellen Sie sicher, dass Sie das Steamworks-SDKinstalliert haben (siehe hier) und dass Sie die Steam App ID in den allgemeinen Optionen für das Spiel hinzugefügt haben. Beachten Sie, dass Sie, wenn Sie GameMaker Studio 2 unter Windows und von der Steam-Version der IDE ausführen und Spiele für Steam entwickeln für Steam entwickeln, dann müssen Sie möglicherweise auch die Option " Alternativen Launcher für Steam-Anwendung verwenden" aktivieren. Diese Option bewirkt, dass Ihr Spiel beim Testen mit einem alternativen Startprogramm gestartet wird, um zu verhindern, dass es von Steammarkiert wird mit der Anwendungs-ID von GameMaker Studio 2 anstelle der eigenen Anwendungs-ID Ihres Spiels. Sie sollten diese Option nur aktivieren, wenn Sie für Steam entwickeln und wenn Sie Probleme beim Starten Ihres Spiels haben.

Die vorletzte Option, die Sie haben, ist die Option " Dateisystem-Sandbox deaktivieren". Wenn Sie diese Option aktivieren, wird die GameMaker Studio 2-Sandbox für das Speichern und Laden von Dateien deaktiviert, so dass Ihre Spiele von überall auf dem System auf Dateien zugreifen können auf dem das Spiel läuft. Diese Option ist standardmäßig nicht aktiviert, da wir empfehlen, die GameMaker Studio 2-Sandbox für Dateien auf allen Systemen beizubehalten, und wenn sie aktiviert ist, sind einige Speicher-/Ladebereiche je nach den Berechtigungen auf Betriebssystemebene möglicherweise immer noch nicht verfügbar. Weitere Informationen zur Sandbox finden Sie auf den Seiten über das GameMaker Studio 2-Dateisystem.

Schließlich haben Sie die Option, dass GameMaker Studio 2 eine ausführbare x64-Datei für Ihre Windows-Projekte erstellt. Wenn diese Option aktiviert ist, wird beim Kompilieren ein 64-Bit-Executable für Ihr Spiel erstellt, und wir empfehlen dringend, dass alle neuen Projekte diese Option verwenden. Dies wird auf lange Sicht das Standardformat (und das einzige) für ausführbare Windows-Dateien sein, da der 32bit-Runner allmählich ausläuft. Beachten Sie, dass, wenn diese Option aktiviert ist, alle Erweiterungs-Dlls, die vom Projekt verwendet werden, ebenfalls 64bit sein müssen (siehe hier für weitere Informationen).

Grafiken

Die Grafikoptionen sind diejenigen, die Sie konfigurieren sollten, um festzulegen, wie Ihr Spiel die Grafikkarte Ihres Ziel-Windows-PCs verwendet. Die folgenden Optionen sind enthalten, die Sie ändern können:

  • Cursor anzeigen: Wenn diese Option aktiviert ist, wird der normale Windows-Cursor angezeigt. Wenn Sie die Option deaktivieren, wird kein Cursor angezeigt, es sei denn, Sie haben in Ihrem Spielcode einen erstellt. Diese Option ist standardmäßig aktiviert.
  • Vollbild starten: Wenn dies aktiviert ist, wird das Spiel im Vollbildmodus gestartet, andernfalls wird es im Fenstermodus gestartet. Dies ist standardmäßig ausgeschaltet.
  • Allow Fullscreen Switching (Vollbildumschaltung zulassen): Wenn diese Option aktiviert ist, kann der Benutzer mit den Standard-Windows-Tastenkombinationen von Vollbild auf Fenster umschalten und wieder zurück. Dies ist standardmäßig ausgeschaltet.
  • Interpoliert Farben zwischen Pixeln: Schaltet die lineare Interpolation ein, die im Grunde die Pixel "glättet". Für gestochen scharfe Pixelgrafiken sollte sie ausgeschaltet sein, aber Wenn Sie aber schöne Alpha-Mischungen und geglättete Kanten haben, ist es besser, sie eingeschaltet zu lassen.
  • Synchronisation verwenden, um Tearing zu vermeiden: Hiermit schalten Sie v-sync ein oder aus. Beachten Sie, dass wenn Sie ein Spiel mit einer Raumgeschwindigkeit von 120 haben und der Spieler einen Monitor mit einer Bildwiederholrate von 60 hat, wird durch das Einschalten dieser Option die Spielgeschwindigkeit ebenfalls auf 60 festgelegt. Diese Option ist standardmäßig ausgeschaltet.
  • Größenänderung des Fensters zulassen: Wenn diese Option aktiviert ist, kann der Benutzer die Größe des Spielfensters ändern (die Option "Randloses Fenster" muss deaktiviert sein, damit dies funktioniert). Diese Option ist standardmäßig ausgeschaltet.
  • Randloses Fenster: Wenn Sie diese Option aktivieren, wird Ihr Spiel mit einem randlosen Fenster ausgeführt, wobei die normalen Schaltflächen zum Minimieren, Maximieren und Schließen sowie der Spieltitel entfernt werden. Dies ist standardmäßig deaktiviert.
  • Skalierung: Hier können Sie wählen, ob das Seitenverhältnis beibehalten werden soll (so dass ein 4:3-Raum auf einem 16:9-Bildschirm "im Letterbox-Format" dargestellt wird) oder ob er vollständig skaliert werden soll (das Bild wird gestreckt, damit es auf den gesamten Bildschirm passt).

    WARNUNG! Wenn Siedie Anwendungsoberfläche ausschalten, werden alle in den Windows-Spieleoptionen eingestellten Skalierungsoptionen deaktiviert, bis sie wieder eingeschaltet wird. Siehe Die Anwendungsoberfläche für weitere Details.

Schließlich gibt es noch die Möglichkeit, die Größe der Texturseite einzustellen. Die Standardgröße (und die kompatibelste) ist 2048x2048, aber Sie können zwischen 256x256 bis zu 8192x8192 wählen. Es gibt es auch eine Schaltfläche mit der Bezeichnung Vorschau die die Texturseiten für diese Plattform generiert und dann ein Fenster öffnet, in dem Sie sehen können, wie sie aussehen. Dies kann sehr nützlich sein, wenn Sie sehen wollen, wie die Texturseiten aufgebaut sind strukturiert sind und um zu verhindern, dass die Texturseiten größer (oder kleiner) als nötig sind. Weitere Informationen zu Texturseiten finden Sie hier.

HINWEIS: Beachten Sie, dass Ihr Spiel auf PCs mit niedrigeren Spezifikationen umso weniger kompatibel ist, je größer die Texturseite ist.

Bilder

Im Abschnitt "Bilder" stellen Sie die Bilder bereit, die Ihr Spiel benötigt. Für Windows müssen Sie eine Symboldatei (im Format .ico ) und einen Begrüßungsbildschirm (im Format .png, .bmp, .jpg oder .gif ) bereitstellen. Der Begrüßungsbildschirm wird beim Laden des Spiels angezeigt und kann durch Aktivieren der Option " Begrüßungsbildschirm verwenden " aktiviert oder deaktiviert werden (standardmäßig ist diese Option deaktiviert).

Es ist erwähnenswert, dass GameMaker Studio 2 über ein Projektbild-Generator-Tool verfügt, das verwendet werden kann, um automatisch alle Bilder zu erstellen, die für die verschiedenen Zielplattformen benötigt werden, für die Ihr Spiel kompiliert wird. Wenn Sie dieses Tool verwenden, sollten Sie die erstellten Bilder überarbeiten, um sicherzustellen, dass sie Ihren Anforderungen entsprechen.

Installateur

Windows-Spiele können mit einem Installer erstellt werden und hier können Sie die verschiedenen Einstellungen und Grafiken festlegen festlegen, die GameMaker Studio 2 bei der Erstellung verwenden wird. Als erstes haben wir die grafischen Optionen, die das Aussehen des Installers selbst bestimmen werden:

  • Beendet: Die Grafik, die auf der "Finished"-Seite des Installationsprogramms erscheint. Muss im Format *.bmp vorliegen und 164x314px groß sein.
  • Kopfzeile: Die Grafik, die erscheint, während das Installationsprogramm ausgeführt wird. Muss im Format *.bmp vorliegen und 150x57px groß sein.

Sie können auch auf eine bestimmte Lizenzvertragsdatei (ein Beispiel wird mit GameMaker Studio 2 mitgeliefert) und eine eigene NSI-Scriptdatei (Details hier) verweisen, falls Sie eine benötigen oder klicken Sie auf die entsprechenden Schaltflächen, um zu den mit GameMaker Studio 2 gelieferten Standarddateien zurückzukehren.
