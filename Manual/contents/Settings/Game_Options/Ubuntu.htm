Ubuntu-Spieloptionen

In diesem Abschnitt werden die verschiedenen Optionen beschrieben, die Ihnen zur Verfügung stehen und die steuern, wie Ihre Ubuntu(Linux)-Spielprojekte kompiliert werden. Die verschiedenen Abschnitte sind:

Allgemein

Auf dieser Registerkarte legen Sie den Namen Ihres Spiels fest, wie er während der Ausführung angezeigt werden soll, und können auch eine Kontakt-E-Mail und eine Beschreibung (kurz und lang), sowie eine URL für die Homepage des Spiels oder des Entwicklers angeben.

Hier haben Sie auch die Möglichkeit, die Option Steam in Ihrem Spiel. Wenn Sie diese Option aktivieren, stellen Sie sicher, dass Sie das SteamworksSDK installiert haben (siehe hier), und dass Sie die Steam-App-ID in den Allgemeinen Optionen für das Spiel hinzugefügt haben.

Schließlich haben Sie die Möglichkeit, die Dateisystem-Sandbox zu deaktivieren. Wenn Sie diese Option aktivieren, wird die GameMaker Studio 2-Sandbox für das Speichern und Laden von Dateien deaktiviert, so dass Ihre Spiele von überall auf dem System, auf dem das Spiel läuft, auf Dateien zugreifen können. Diese Option ist standardmäßig nicht aktiviert, da wir empfehlen, die GameMaker Studio 2-Sandbox für Dateien auf allen Systemen beizubehalten, und wenn sie aktiviert ist, sind einige Speicher-/Ladebereiche je nach den Berechtigungen auf Betriebssystemebene möglicherweise immer noch nicht verfügbar. Für weitere Informationen zur Sandbox finden Sie auf den Seiten über das GameMaker Studio 2-Dateisystem.

Grafiken

Die Grafikoptionen sind diejenigen, die Sie konfigurieren sollten, um festzulegen, wie Ihr Spiel die Grafikkarte Ihres Ubuntu-Zielrechners nutzen soll. Die folgenden Optionen sind enthalten, die Sie ändern können:

  • Vollbild starten: Wenn dies aktiviert ist, wird das Spiel im Vollbildmodus gestartet, andernfalls wird es im Fenstermodus gestartet. Dies ist standardmäßig ausgeschaltet.
  • Allow Fullscreen Switching (Vollbildumschaltung zulassen): Wenn diese Option aktiviert ist, kann der Benutzer mit den Standard-Ubuntu-Betriebssystemkurzbefehlen vom Vollbild zum Fenster und wieder zurück wechseln. Dies ist standardmäßig ausgeschaltet.
  • Interpoliert Farben zwischen Pixeln: Schaltet die lineare Interpolation ein, die im Grunde die Pixel "glättet". Für gestochen scharfe Pixelgrafiken sollte sie ausgeschaltet sein, aber Wenn Sie aber schöne Alpha-Blends und geglättete Kanten haben, ist es besser, sie eingeschaltet zu lassen.
  • Cursor anzeigen: Wenn diese Option aktiviert ist, wird der Standard-Betriebssystem-Cursor in Ihrem Spiel angezeigt. Wenn diese Option deaktiviert ist, wird kein Cursor angezeigt, während sich die Maus über dem Spielfenster befindet, es sei denn, Sie haben einen Cursor als Teil des Spielprojekts erstellt. des Spielprojekts erstellt haben (diese Option ist standardmäßig aktiviert).
  • Synchronisation verwenden, um Tearing zu vermeiden: Hiermit schalten Sie v-sync ein oder aus. Beachten Sie, dass wenn Sie ein Spiel mit einer Raumgeschwindigkeit von 120 haben und der Spieler einen Monitor mit einer Bildwiederholrate von 60 hat, wird durch das Einschalten dieser Option die Spielgeschwindigkeit ebenfalls auf 60 festgelegt. Diese Option ist standardmäßig ausgeschaltet.
  • Größenänderung des Fensters zulassen: Wenn diese Option aktiviert ist, kann der Benutzer die Größe des Spielfensters ändern (die Option "Randloses Fenster" muss deaktiviert sein, damit dies funktioniert). Diese Option ist standardmäßig ausgeschaltet.
  • Skalierung: Hier können Sie wählen, ob das Seitenverhältnis beibehalten werden soll (so dass ein 4:3-Raum auf einem 16:9-Bildschirm "im Letterbox-Format" dargestellt wird) oder ob er vollständig skaliert werden soll (das Bild wird gestreckt, damit es auf den gesamten Bildschirm passt).

    WARNUNG! Wenn Siedie Anwendungsoberfläche ausschalten, werden alle in den Windows-Spieloptionen eingestellten Skalierungsoptionen deaktiviert, bis sie wieder eingeschaltet wird. Siehe Die Anwendungsoberfläche für weitere Details.

Schließlich gibt es noch die Möglichkeit, die Größe der Texturseite einzustellen. Die Standardgröße (und die kompatibelste) ist 2048x2048, aber Sie können zwischen 256x256 bis zu 8192x8192 wählen. Es gibt es auch eine Schaltfläche mit der Bezeichnung Vorschau die die Texturseiten für diese Plattform generiert und dann ein Fenster öffnet, in dem Sie sehen können, wie sie aussehen. Dies kann sehr nützlich sein, wenn Sie sehen wollen, wie die Texturseiten aufgebaut sind strukturiert sind und um zu verhindern, dass die Texturseiten größer (oder kleiner) als nötig sind. Weitere Informationen zu Texturseiten finden Sie hier.

HINWEIS: Beachten Sie, dass Ihr Spiel auf PCs mit niedrigeren Spezifikationen umso weniger kompatibel ist, je größer die Texturseite ist.

Bilder

Hier können Sie einen Splash-Screen für Ihr Spiel hinzufügen und das Spielsymbol festlegen, das verwendet wird. Das Symbol muss ein 64x64 Pixel großes Bild im Format .png sein, und der Startbildschirm sollte mindestens so groß sein wie der erste Raum in Ihrem Spiel (oder dessen View Port) und ebenfalls das Format .png haben. Der Begrüßungsbildschirm wird beim Laden des Spiels angezeigt und kann durch Aktivieren der Option " Use Splash Screen" (Splash-Bildschirm verwenden ) aktiviert oder deaktiviert werden (standardmäßig ist diese Option deaktiviert).

Es ist erwähnenswert, dass GameMaker Studio 2 über ein Projektbild-Generator-Tool verfügt, das verwendet werden kann, um automatisch alle Bilder zu erstellen, die für die verschiedenen Zielplattformen benötigt werden, für die Ihr Spiel kompiliert wird. Wenn Sie dieses Tool verwenden, sollten Sie die erstellten Bilder überarbeiten, um sicherzustellen, dass sie Ihren Anforderungen entsprechen.
