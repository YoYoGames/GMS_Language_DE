HTML5-Spieloptionen

In diesem Abschnitt werden die verschiedenen Optionen beschrieben, die Ihnen zur Verfügung stehen und die steuern, wie Ihre HTML5-Spielprojekte kompiliert werden. Die verschiedenen Abschnitte sind:

Allgemein

In diesem Bereich stehen Ihnen Felder zur Verfügung, in die Sie die folgenden Informationen über Ihr Spiel eingeben können:

  • Browser-Titel: Hier können Sie einen Text eingeben, der für die Titelleiste der Registerkarte für Ihr Spiel im Browser verwendet werden soll.
  • Version: Die Versionsnummer für Ihr Projekt.
  • Name des Ordners: Der Name für den Ordner, in dem Ihre HTML5-Spieldateien gespeichert werden (Standard ist "html5game").
  • Ausgabe-Name: Dies ist der Name der HTML-Ausgabedatei und kann auf etwas anderes als den Standard "index.html" geändert werden.

Unterhalb der Spielinformationsfelder haben Sie außerdem die folgenden Optionen:

  • Debug-Ausgabe an Konsole: Damit werden alle Debug-Ausgaben an die JS-Konsole Ihres Browsers gesendet. Dies ist standardmäßig ausgeschaltet.
  • Cursor anzeigen: Damit wird der Mauszeiger deaktiviert, wenn er sich über der Spielfläche befindet. Dies ist standardmäßig eingeschaltet.
  • Warnung "Außerhalb des Servers ausgeführt" anzeigen: Wenn Sie ein Spiel lokal testen (d. h., Sie klicken auf die Indexdatei von Ihrem Computer aus und nicht von einem Server aus - nicht empfohlen), dann erhalten Sie normalerweise eine Warnung, die Sie darüber informiert, dass die Dinge nicht so funktioniert, wie Sie es aufgrund der Browser-Sicherheit für lokale Dateien erwarten. Sie können diese Warnung unterdrücken, indem Sie das Häkchen bei dieser Option entfernen. Sie ist standardmäßig aktiviert.

Unterhalb der allgemeinen Einstellungen haben Sie die erweiterten Optionen, die da wären:

  • Eingeschlossene Datei als index.html: Hier können Sie eine eigene "index.html"-Datei hinzufügen, die anstelle der vordefinierten Datei, die GameMaker Studio 2 standardmäßig generiert, verwendet wird. Dies kann sehr nützlich sein, wenn Sie ein Website-Design implementiert haben oder bestimmte Funktionen in alle Ihre HTML5-Builds aufnehmen möchten. Die zu verwendende Datei muss als eingeschlossene Datei in das Spielprojekt aufgenommen werden.
  • Ladebalken-Erweiterung: Damit wird die Standard-Ladeleiste durch eine ersetzt, die Ihrem GameMaker Studio 2-Projekt als Erweiterung hinzugefügt wurde.
  • Prepend output .js: Hier können Sie der Ausgabe *.js ein beliebiges zusätzliches JavaScript voranstellen, das für Ihr Spiel erforderlich ist. Für eine Chrome Store-App kann es beispielsweise erforderlich sein, dass Sie window.localStorage = undefined; zur Ausgabe-JS hinzufügen, wofür Sie diese Option verwenden würden.

Analytik

In diesem Abschnitt können Sie Ihrem Spiel Analysen hinzufügen, die es Ihnen ermöglichen, bestimmte Produkte von Drittanbietern zu verwenden, um Ihr Spiel und dessen Verkäufe oder Spiele. Sie können angeben, ob Sie Flurry Analytics oder Google Analytics verwenden möchten, um Ihr Spiel zu verfolgen. Aufgrund der Natur dieser Funktionen Sobald Sie Flurry Analytics aktivieren oder Google Analytics aktivieren markiert haben, sollten Sie auch die eindeutige ID, die Ihnen zugewiesen wurde, in den entsprechenden Bereich eingeben.

Informationen zu den Funktionen, die Sie zum Senden von Analysedaten verwenden können, finden Sie hier.

Grafiken

Hier können Sie die folgenden Details für die Anzeige Ihres Spiels ändern (beachten Sie, dass Ihr Spiel nicht in allen Browsern gleich angezeigt wird) nicht alle Browser Ihr Spiel auf die gleiche Weise anzeigen. Testen Sie daher diese Optionen in so vielen Browsern wie möglich, um sicherzustellen, dass sie die gewünschte Wirkung haben):

  • Umschalten auf Vollbildmodus zulassen: Wenn diese Option aktiviert ist, kann der Benutzer das Spiel mit / + "F" auf Mac und F10 auf Windows vom Fenster- zum Vollbildmodus umschalten. Dies ist standardmäßig aktiviert.
  • Interpoliert Farben zwischen Pixeln: Schaltet die lineare Interpolation ein, die im Grunde genommen Pixel "glättet". Für scharfe Pixelgrafiken sollte diese Option ausgeschaltet sein, aber wenn Sie schöne Alpha-Mischungen und geglättete Kanten haben, sollten Sie es eingeschaltet lassen. Dies ist standardmäßig eingeschaltet.
  • Zentriert das Spiel im Browser: Wenn Sie diese Option aktivieren, wird der Spielbildschirm automatisch im Browser zentriert, in dem das Spiel ausgeführt wird. Dies ist standardmäßig ausgeschaltet.
  • Eingebaute Partikel verwenden: Diese Option exportiert die Sprites, die für das eingebaute Partikelsystem benötigt werden, zusammen mit Ihrem Spiel. Diese Sprites werden in einzelnen Texturseiten gespeichert, was bedeutet, dass sie nicht die effizienteste Methode zur Erzeugung von Partikeln für diese Plattform sind. Daher wird empfohlen, dass Sie (sofern Sie nicht WebGL verwenden) diese Option deaktivieren und Ihre eigenen Partikel-Sprites verwenden, die auf den Haupttextur-Seiten für Ihr Spiel gespeichert werden. Dies ist standardmäßig standardmäßig aktiviert.
  • Eingebaute Schriftart verwenden: Mit dieser Option wird die in GameMaker Studio 2 eingebaute Schriftart zusammen mit den restlichen Spieldateien exportiert. Dadurch wird die Größe der Texturseiten leicht erhöht und es wird etwas mehr Speicherplatz benötigt, Wenn Ihr Spiel also nicht die eingebaute Schriftart Arial verwendet, sollte diese Option deaktiviert werden. Sie ist standardmäßig aktiviert.
  • WebGL: Wählen Sie, ob Sie die WebGL-Unterstützung ausschalten, optional machen (automatische Erkennung) oder zur Voraussetzung für Ihr Spiel machen wollen. Um die Leistung unter allen Umständen zu testen, sollten Sie dies als deaktiviert festlegen und dann normalerweise für die Veröffentlichung auf "auto-detect" setzen. Wenn Sie es zur Bedingung machen, kann dies die Kompatibilität Ihres Spiels mit verschiedenen Browsertypen erheblich verringern. Dies ist standardmäßig auf automatische Erkennung eingestellt.
  • Skalieren: Ihr Spiel kann so konfiguriert werden, dass die Zeichenfläche automatisch skaliert wird, um das Seitenverhältnis innerhalb des Browsers beizubehalten , oder Sie können auswählen, dass es im vollen Maßstab ausgeführt wird. Die Option "Voller Maßstab wird das Spiel im Browser nicht voll skaliert, sondern das Gezeichnete wird so gestreckt, dass es in die Leinwandgröße passt, die durch den ersten Raum des Spiels definiert ist. Standardmäßig ist dies so eingestellt, dass das Seitenverhältnis beibehalten wird.

Schließlich gibt es noch die Möglichkeit, die Größe der Texturseite einzustellen. Die Standardgröße (und die kompatibelste) ist 2048x2048, aber Sie können zwischen 256x256 bis zu 8192x8192 wählen. Es gibt es auch eine Schaltfläche mit der Bezeichnung Vorschau die die Texturseiten für diese Plattform generiert und dann ein Fenster öffnet, in dem Sie sehen können, wie sie aussehen. Dies kann sehr nützlich sein, wenn Sie sehen wollen, wie die Texturseiten aufgebaut sind strukturiert sind und um zu verhindern, dass die Texturseiten größer (oder kleiner) als nötig sind. Weitere Informationen zu Texturseiten finden Sie hier.

HINWEIS: Beachten Sie, dass Ihr Spiel mit verschiedenen Browsern und Geräten weniger kompatibel ist, je größer die Texturseite ist.

Bilder

In diesem Abschnitt können Sie das Favicon und den Splash Screen definieren. Das Icon sollte als eine *.ico Datei erstellt werden und kann 16x16, 32x32, 48x48 oder 64x64 Pixel groß sein und eine Farbtiefe von 8-Bit, 24-Bit oder 32-Bit haben. Der Begrüßungsbildschirm sollte die Größe der Leinwand haben, auf der Ihr Spiel angezeigt wird, und überschreibt alle Ladebalken, wenn Sie die Option "Splash-Screen verwenden" aktivieren (d. h. Sie können einen Ladebalken ODER einen Splash-Screen haben, aber nicht beides). Kleinere oder größere Bilder sind erlaubt, werden aber skaliert, um zu passen. Der Startbildschirm sollte ein *.png Bild sein.

Es ist erwähnenswert, dass GameMaker Studio 2 über ein Projektbild-Generator-Tool verfügt, das verwendet werden kann, um automatisch alle Bilder zu erstellen, die für die verschiedenen Zielplattformen benötigt werden, für die Ihr Spiel kompiliert wird. Wenn Sie dieses Tool verwenden, sollten Sie die erstellten Bilder überarbeiten, um sicherzustellen, dass sie Ihren Anforderungen entsprechen.

Sozial

Die Registerkarte "Soziales" dient zum Einrichten der Facebook-Kommunikation in Ihrem Spiel. Wenn Sie die Facebook-Funktionen benötigen, müssen Sie müssen Sie das Kontrollkästchen "Use Facebook" (Facebook verwenden) anklicken und dann die Facebook-App-ID und den Facebook-App-Anzeigenamen angeben . Beachten Sie, dass der Zugriff auf alle Facebook-Funktionen über eine Erweiterung erfolgt. Wenn Sie das Kontrollkästchen " Facebook verwenden" zum ersten Mal aktivieren, werden Sie aufgefordert, die Facebook-Erweiterung von der Website "Facebook" herunterzuladen und zu installieren. Marktplatz (möglicherweise müssen Sie sich in Ihr YoYo-Konto einloggen, bevor sie heruntergeladen werden kann).
