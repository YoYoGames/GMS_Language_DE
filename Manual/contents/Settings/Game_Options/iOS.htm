iOS-Spieloptionen

In diesem Abschnitt werden die verschiedenen Optionen beschrieben, die Ihnen zur Verfügung stehen und die steuern, wie Ihre iOS-Spielprojekte kompiliert werden. Die verschiedenen Abschnitte sind:

Allgemein

Der erste Teil des Abschnitts "Allgemein" umfasst die Build-Einstellungen für Ihr Spiel. Zu Beginn stellen Sie die Ausrichtungen ein, in denen Ihr Spiel ausgeführt werden kann, um eine optimale Benutzererfahrung zu gewährleisten. Die verfügbaren Optionen sind:

  • Porträt
  • Umgekehrtes Porträt
  • Landschaft
  • Umgekehrte Landschaft

Sie können dann zum Abschnitt " Device Support" (Geräteunterstützung ) gehen, in dem die folgenden Optionen zum Einstellen des Gerätetyps zur Verfügung stehen:

  • iPhone oder iPod Touch: iPhone 6 und höher oder iPod Touch-Geräte
  • iPad: iPad-Geräte
  • Beide: Sowohl iPhone/iPod als auch iPad

Als Zweites müssen Sie die Produktinformationen zu Ihrem Spiel ausfüllen, einschließlich des Anzeigenamens, der Bundle-ID und der Versionsinformationen. Beachten Sie, dass die Bundle-ID im korrekten umgekehrten URL-Format sein muss, damit der endgültige Build des Spiels korrekt funktioniert, z. B.: com.[Company].[GameName]. Hier können Sie auch den Team Identifier angeben, den Sie zum Signieren der endgültigen Anwendung verwenden möchten, die GameMaker Studio 2 für Sie erstellt. Wenn Sie ihn hier einstellen, wird die Team-ID überschrieben, die Sie in den iOS-Voreinstellungen angegeben haben.

Als Nächstes können Sie die Option " Defer Home Indicator Output" aktivieren oder deaktivieren. Wenn diese Option aktiviert ist, wird die Home-Leiste immer bei niedrigem a alpha angezeigt, und der erste Wisch weckt die Leiste auf und der zweite Wisch bringt Sie nach Hause. Wenn die Option nicht aktiviert ist, wird die Leiste im Spiel nicht angezeigt, aber der erste Wisch über die Kante bringt Sie nach Hause. Standardmäßig ist sie nicht aktiviert.

Schließlich haben Sie die Möglichkeit, eine Poddatei und eine Podfile.lock zu erzeugen. Diese Dateien sind erforderlich, wenn Sie CocoaPods mit dem Erweiterungseditor zu einer iOS-Erweiterung hinzufügen. Die Dateien enthalten spezifische Makros, die von GameMaker Studio 2 benötigt werden, um alle aufgelisteten CocoaPodsbeim Erstellen Ihres Spiels einzurichten. Wenn Sie auf die Schaltflächen klicken, werden die Dateien mit die Makros vorgeneriert und öffnet außerdem einen Datei-Explorer am Speicherort der Dateien, falls Sie sie manuell bearbeiten möchten. Es gibt auch eine Schaltfläche, die Sie zur folgenden Anleitung führt:

  • iOS und tvOS: Verwendung von Cocoa Pods

Grafiken

Hier können Sie die folgenden Details in Bezug auf die Anzeige Ihres Spiels ändern, wobei die folgenden Optionen zur Verfügung stehen:

  • Interpoliert Farben zwischen Pixeln: Schaltet die lineare Interpolation ein, die im Grunde genommen Pixel "glättet". Für scharfe Pixelgrafiken sollte diese Option ausgeschaltet sein, aber wenn Sie schöne Alpha-Mischungen und geglättete Kanten haben, sollten Sie es eingeschaltet lassen. Standardmäßig ist dies ausgeschaltet.
  • Halbe iPad 1-Texturen: Das iPad 1 hat viel weniger Texturspeicher als neuere Geräte, daher können Sie diese Option aktivieren, um die Größe der Texturen um die Hälfte zu reduzieren, wenn Sie auf dieses Gerät abzielen oder maximale Kompatibilität erhalten möchten.
  • Skalieren: Hier können Sie wählen, ob Sie die Zeichenfläche unter Beibehaltung des Seitenverhältnisses innerhalb des Gerätebildschirms skalieren möchten - indem Sie "Polster" an den Rändern hinzufügen, um sie an den Bildschirm anzupassen - oder ob Sie die Zeichenfläche strecken möchten, um sie an den Bildschirm anzupassen. den Gerätebildschirm anzupassen.

Schließlich gibt es noch die Möglichkeit, die Größe der Texturseite einzustellen. Die Standardgröße (und die kompatibelste) ist 2048x2048, aber Sie können eine Größe zwischen 256x256 und 8192x8192 wählen. Es Es gibt auch eine Schaltfläche mit der Bezeichnung Vorschau , die die Texturseiten für diese Plattform generiert und dann ein Fenster öffnet, in dem Sie sehen können, wie sie aussehen. Dies kann sehr nützlich sein, wenn Sie sehen möchten, wie die Texturseiten aufgebaut sind und um zu verhindern, dass die Texturseiten größer (oder kleiner) als nötig sind. Weitere Informationen zu Texturseiten finden Sie hier.

HINWEIS: Beachten Sie, dass Ihr Spiel mit verschiedenen Browsern und Geräten weniger kompatibel ist, je größer die Texturseite ist.

Icons

In diesem Abschnitt können Sie die verschiedenen Symbole hinzufügen, die für die verschiedenen iPhone- und iPad-Geräte und die verschiedenen Teile des iOS App Store. Diese Icons sollten als 24-Bit-Bilder *.png in der jeweils angegebenen Größe erstellt werden.

Es ist erwähnenswert, dass GameMaker Studio 2 über ein Projektbild-Generator-Tool verfügt, das verwendet werden kann, um automatisch alle Bilder zu erstellen, die für die verschiedenen Zielplattformen benötigt werden, für die Ihr Spiel kompiliert wird. Wenn Sie dieses Tool verwenden, sollten Sie die erstellten Bilder überarbeiten, um sicherzustellen, dass sie Ihren Anforderungen entsprechen. Das Tool wird sowohl Icons als auch Bilder generieren (siehe den Abschnitt unten).

HINWEIS: Beim Importieren eines älteren Projekts kann die folgende Warnung angezeigt werden:Dies wird angezeigt, weil die aktuelle iOS-Ziel Version das Format der erforderlichen Symbole geändert hat, seit das Projekt zuvor geladen wurde. Sie müssen alle Symbole neu erstellen, da sie durch die generischen GameMaker Studio 2-Symbole ersetzt wurden (Sie können das oben erwähnte Projektbild-Generator verwenden, um dies schnell zu erledigen).

Bilder

In diesem Abschnitt können Sie für jede Geräteausrichtung eine eigene Grafik als Startbildschirm (das Bild, das kurz angezeigt wird, während (das Bild, das kurz angezeigt wird, während Ihre App geladen wird) für jede der Geräteausrichtungen sowie die Randfarbe, die als Hintergrund für Startbildschirme verwendet wird, die nicht den gesamten Bildschirm ausfüllen sollen. Diese Bilder müssen als einzelne 24-Bit-Bilder erstellt werden *.png Bilder ohne Transparenzen und in einer geeigneten Größe erstellt werden. Beachten Sie, dass es sich hierbei nicht um einen Splash-Screen handelt und Apple ziemlich strenge Richtlinien hat, wie dieser eingerichtet und verwendet werden sollte. Weitere Informationen finden Sie auf der Entwicklerseite zu Human Interface Guidelines.

Sie können auch einstellen, wie der Startbildschirm angezeigt werden soll, und zwar mit den folgenden Optionen:

  • Seitenverhältnis beibehalten: Damit wird das Seitenverhältnis des Bildes im Vergleich zum Gerätebildschirm beibehalten, wobei an den Rändern bei Bedarf "Auffüllungen" hinzugefügt werden.
  • Beschneiden auf Anpassen: Damit wird das Bild 1:1 mit dem Gerätebildschirm beibehalten und die Ränder werden nach Bedarf zugeschnitten (oder aufgefüllt).
  • Strecken: Damit wird das Bild gestreckt, um den Gerätebildschirm auszufüllen (und kann das angezeigte Bild verzerren).

Sozial

In diesem Abschnitt können Sie verschiedene soziale Funktionen in Ihrem iOS-Spiel aktivieren. Die folgenden Optionen können gewählt werden:

  • Facebook: Wenn Sie die Facebook-Funktionen benötigen, müssen Sie dieses Kontrollkästchen aktivieren und dann die Facebook-App-ID und den Facebook-App-Namen angeben. Beachten Sie, dass der Zugriff auf alle Facebook-Funktionen über eine Wenn Sie zum ersten Mal das Kontrollkästchen "Facebook verwenden" aktivieren, werden Sie aufgefordert, die Facebook-Erweiterung von der Website "Facebook" herunterzuladen und zu installieren. Marktplatz.
  • Aktivieren Sie Push-Benachrichtigungen: Damit Ihr Spiel Push-Benachrichtigungen verwenden kann, müssen sie hier zuerst aktiviert werden, sonst funktionieren sie nicht.
  • Aktivieren Sie Anmelden mit Apple: Diese Option ist für die Verwendung der Apple Sign In Extension auf iOS. Sie können diese Erweiterung über die YoYo Spiele Marktplatz, und die Erweiterung selbst enthält eine vollständige Anleitung zur Verwendung. Wenn Sie diese Erweiterung nicht verwenden, sollten Sie diese Option nicht ankreuzen.
