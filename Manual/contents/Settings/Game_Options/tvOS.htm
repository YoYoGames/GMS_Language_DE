tvOS-Spieloptionen

In diesem Abschnitt werden die verschiedenen Optionen beschrieben, die Ihnen zur Verfügung stehen und die steuern, wie Ihre tvOS-Spielprojekte kompiliert werden. Die verschiedenen Abschnitte sind:

Allgemein

Hier können Sie die Produktinformationen zu Ihrem Spiel eingeben, einschließlich des Anzeigenamens, der Bundle-ID und der Versionsinformationen . Beachten Sie, dass die Bundle-ID im korrekten Reverse-URL-Format sein muss, damit der endgültige Build des Spiels korrekt funktioniert, z. B.: com.[Company].[GameName]. Sie können auch den Speicherort für die App-Ausgabe auf dem Build-Mac festlegen und Sie können die Team-Kennung angeben, die Sie für das Signieren der endgültigen Anwendung verwenden möchten, die GameMaker Studio 2 für Sie erstellt. Wenn Sie ihn hier einstellen, wird die Team-ID, die Sie in den tvOS-Voreinstellungen angegeben haben, außer Kraft gesetzt.

Hier haben Sie auch die Möglichkeit, ein Podfile und ein Podfile.lock zu erzeugen. Diese Dateien werden benötigt, wenn Sie CocoaPods mit dem Extension Editor zu einer tvOS-Erweiterung hinzufügen. Die Dateien enthalten spezifische Makros, die von GameMaker Studio 2 benötigt werden, um alle aufgelisteten CocaoPodsbeim Erstellen Ihres Spiels einzurichten. Wenn Sie auf die Schaltflächen klicken, werden die Dateien mit die Makros vorgeneriert und öffnet außerdem einen Datei-Explorer am Speicherort der Dateien, falls Sie sie manuell bearbeiten möchten. Es gibt auch eine Schaltfläche, die Sie zur folgenden Anleitung führt:

  • iOS und tvOS: Verwendung von Cocoa Pods

Grafiken

Hier können Sie die folgenden Details in Bezug auf die Anzeige Ihres Spiels ändern, wobei die folgenden Optionen zur Verfügung stehen:

  • Interpoliert Farben zwischen Pixeln: Schaltet die lineare Interpolation ein, die im Grunde genommen Pixel "glättet". Für scharfe Pixelgrafiken sollte diese Option ausgeschaltet sein, aber wenn Sie schöne Alpha-Mischungen und geglättete Kanten haben, sollten Sie es eingeschaltet lassen. Standardmäßig ist dies ausgeschaltet.
  • Skalieren: Hier können Sie wählen, ob Sie die Zeichenfläche unter Beibehaltung des Seitenverhältnisses innerhalb des Gerätebildschirms skalieren möchten - indem Sie "Polster" an den Rändern hinzufügen, um sie an den Bildschirm anzupassen - oder ob Sie die Zeichenfläche strecken möchten, um sie an den Bildschirm anzupassen. den Gerätebildschirm anzupassen.

Als nächstes gibt es die Möglichkeit, die Größe der Texturseite einzustellen. Die Standardgröße (und die kompatibelste) ist 2048x2048, aber Sie können eine Größe zwischen 256x256 und 8192x8192 wählen. Es Es gibt auch eine Schaltfläche mit der Bezeichnung Vorschau , die die Texturseiten für diese Plattform generiert und dann ein Fenster öffnet, in dem Sie sehen können, wie sie aussehen. Dies kann sehr nützlich sein, wenn Sie sehen möchten, wie die Texturseiten aufgebaut sind und um zu verhindern, dass die Texturseiten größer (oder kleiner) als nötig sind. Weitere Informationen zu Texturseiten finden Sie hier.

HINWEIS: Beachten Sie, dass Ihr Spiel mit verschiedenen Browsern und Geräten weniger kompatibel ist, je größer die Texturseite ist.

Schließlich wählen Sie, ob ein Cursor angezeigt werden soll oder nicht. Wenn das Kontrollkästchen aktiviert ist, zeigt das Betriebssystem einen Cursor an, wenn die Maus bewegt wird (an der Mausposition), und der Cursor wird ausgeblendet und verschwindet, wenn die Maus eine Zeit lang nicht bewegt wird. Wenn das Häkchen nicht gesetzt ist, wird kein Cursor angezeigt.

Icons

In diesem Abschnitt können Sie die verschiedenen Icons hinzufügen, die für die verschiedenen Geräte und die verschiedenen Teile des App Store. Diese Icons sollten als 24-Bit-Bilder *.png in der jeweils angegebenen Größe verfasst werden.

Es ist erwähnenswert, dass GameMaker Studio 2 über ein Projektbild-Generator-Tool verfügt, das verwendet werden kann, um automatisch alle Bilder zu erstellen, die für die verschiedenen Zielplattformen benötigt werden, für die Ihr Spiel kompiliert wird. Wenn Sie dieses Tool verwenden, sollten Sie die erstellten Bilder überarbeiten, um sicherzustellen, dass sie Ihren Anforderungen entsprechen, insbesondere für diese Plattform, da die benötigten Icons rechteckig und nicht quadratisch sind. Dieses Tool generiert neben Icons auch Bilder (siehe unten) als auch Icons.

Bilder

In diesem Abschnitt können Sie separate Grafiken als "Top Shelf"-Bilder und Splash-Screens für jede der verschiedenen tvOS-Zielgrößen. Diese Bilder müssen als einzelne 24-Bit-Bilder *.png ohne Transparenzen und in der richtigen Größe verfasst sein. Sie können auch eine Mindestzeit für die Anzeige der Splash-Screens im Dropdown-Menü am am unteren Rand einstellen (von 0 bis 10 Sekunden).

Wie im Abschnitt über Symbole erwähnt, können Sie den Projektbildgenerator verwenden, um diese Bilder zu erstellen.

Sozial

In diesem Abschnitt können Sie verschiedene soziale Funktionen in Ihrem tvOS-Spiel aktivieren. Die folgenden Optionen können gewählt werden:

  • Aktivieren Sie Push-Benachrichtigungen: Damit Ihr Spiel Push-Benachrichtigungen verwenden kann, müssen sie hier zuerst aktiviert werden, sonst funktionieren sie nicht.
  • Aktivieren Sie Anmelden mit Apple: Diese Option ist für die Verwendung der Apple Sign In Extension auf iOS. Sie können diese Erweiterung über die YoYo Spiele Marktplatz, und die Erweiterung selbst enthält eine vollständige Anleitung zur Verwendung. Wenn Sie diese Erweiterung nicht verwenden, sollten Sie diese Option nicht ankreuzen.
