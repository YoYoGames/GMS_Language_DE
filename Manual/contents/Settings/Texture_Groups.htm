Textur-Gruppen

Der Texturgruppen-Manager ist über das Menü "Werkzeuge" in der IDE verfügbar. Hier können SieTexturgruppen hinzufügen, löschen und umbenennen . GameMaker Studio 2 erlaubt es Ihnen, jedes der Grafik-Assets (Sprites, Tilesets und Schriften) verschiedenen Texturgruppen zuzuordnen, um die Anzahl der von Texturwechseln zu optimieren, die die Hardware zu einem bestimmten Zeitpunkt in Ihrem Spiel durchführt. Damit das funktioniert, müssen Sie die erforderlichen Texturgruppen in diesem Fenster definieren und dann durch die entsprechenden Asset-Typen gehen und die Gruppe, zu der sie gehören sollen, aus dem Dropdown-Menü auswählen. aus dem Dropdown-Menü auswählen. Alternativ können Sie Assets über das Menü der rechten Maustaste im Asset-Browser zu einer Texturgruppe hinzufügen. Sie können ein oder mehrere Assets (oder einen ganzen Asset-Gruppen-Ordner) auswählen und dann das RMB-Menü verwenden, um die Assets zu einer beliebigen Gruppe hinzuzufügen, wie unten für Tilesets gezeigt:

ist es wichtig zu wissen, dass eine einzelne Texturgruppe mehrere Texturseiten haben kann und diese nicht gleichwertig sind. Wenn Ihr Spiel viele einzigartige Grafiken für jede Ebene hat, dann können Sie eine einzelne Texturgruppe für jede der Ebenen definieren, aber die Grafiken erfordern zwei oder mehr Texturseiten pro Gruppe.

Standardmäßig werden alle Bilder in Ihrem Spiel der Texturgruppe "Standard" hinzugefügt, die immer vorhanden ist. Sie können jedoch auf die Schaltfläche " Neu hinzufügen" klicken, um eine neue Texturgruppe zu erstellen, und wenn Sie den Namen der Gruppe wählen, können Sie ändern Sie ihn etwas Passendes. Zu Beginn zeigt die Gruppefür jeden Abschnitt der Gruppe (Sprites, Tilesets und Fonts) "DieseGruppe ist leer" an, und so müssen Sie dann Ihre Grafik-Assets durchgehen und sie hinzufügen. Sie können eine Ressource über die Schaltfläche " Add Resource" (Ressource hinzufügen ) am unteren Rand oder durch Auswahl der Texturgruppe im Ressourceneditor für das Bild, das Kachelset oder die Schriftart. Andere Texturgruppen als die "Standard"-Gruppe können auch jederzeit gelöscht werden. Dadurch wird ihr Inhalt in die Gruppe "Standard" verschoben (beachten Sie, dass Sie, wenn Sie eine Gruppe versehentlich löschen, / + "Z" verwenden können, um die Aktion rückgängig zu machen).

Sobald Sie Texturgruppen definiert - und ihnen Grafik-Assets zugewiesen - haben, sehen Sie sie links im Hauptfenster des Texturgruppen-Editors aufgelistet, und wenn Sie auf eines der Sprites in dieser Liste klicken, werden sie im Bildvorschau-Fenster auf der auf der rechten Seite. Dies ist besonders hilfreich, wenn Sie an großen Projekten arbeiten und sehen müssen, welche Bilder welcher Gruppe zugewiesen wurden. Beachten Sie, dass, wenn die der Gruppe zugewiesenen Bilder mehrere Unterbilder haben (d. h., sie sind animiert), dann nur das erste Teilbild im Vorschaufenster angezeigt wird. Sie können auch mit der rechten Maustaste auf eine Ressource klicken, um ein kleines Menü zu öffnen, das Ihnen die Möglichkeit gibt um sie in eine andere Texturgruppe zu verschieben oder den Editor für diese Ressource zu öffnen.

Es ist erwähnenswert, dass, auch wenn die Assets in der Liste als getrennt angezeigt werden, alle Sprites, Tilesets und Fonts zur gleichen Texturseite (oder Seiten) hinzugefügt werden und nicht für jeden Ressourcentyp eine eigene Seite erhalten. In diesem Fall erhält es eine eigene Textur-Seite (siehe den Abschnitt "Textur-Einstellungen" im Sprite-Editor für weitere Details).

Mit den Gruppeneinstellungen im Texturgruppen-Editor wird genau festgelegt, wie die Texturseiten für eine bestimmte Gruppe erstellt werden sollen, wobei folgende Optionen zur Verfügung stehen:

  • Skalierung zulassen - Wenn Sie diese Option deaktivieren, wird GameMaker Studio 2 gezwungen, die Texturgruppe unter keinen Umständen zu skalieren. Dies kann vor allem bei Universal-Apps nützlich sein, da es Ihnen erlaubt, bestimmte Ressourcen wie Schriftarten in eine eigene Gruppe zu legen, die verwendet wird, wenn die App auf einem bestimmten Gerät läuft. Zum Beispiel können Schriften auf einem Retina-Display großartig aussehen, aber wenn sie herunterskaliert werden, um auf einem normalen Display zu zeichnen, können sie schrecklich aussehen, also können Sie sie einer Wenn Sie für das iPhone entwickeln, sollten Sie diese Option aktivieren, da Apps abgelehnt wurden, weil die Bilder durch die Texturskalierung unscharf wurden. Diese Option ist standardmäßig aktiviert .
  • Automatisch beschneiden - Wenn Sie Bilder in GameMaker Studio 2 hinzufügen, werden diese automatisch beschnitten, wenn sie einen "Raum" um sie herum haben (d. h. 100 % Transparenz), und "Automatisch Zuschneiden" ist markiert. Wenn Sie das Bild jedoch z. B. als nicht-wiederholende Textur für Primitives oder ähnliches verwenden, bedeutet dies, dass die UV-Koordinaten für das Texturmapping nicht korrekt dargestellt werden. In diesen Fällen können Sie diese Option deaktivieren und das Sprite wird der Texturseite "so wie es ist" hinzugefügt, wobei alle Transparenzen an den Rändern erhalten bleiben. Diese Option ist standardmäßig aktiviert .
  • Mipmaps generieren - Wenn Sie diese Option aktivieren (standardmäßig ist sie ausgeschaltet), teilt GameMaker Studio 2 mit, dass die ursprüngliche Texturseite skaliert und in mehrere Auflösungen innerhalb der Texturdatei gefiltert werden soll. Normalerweise ist diese Option ausgeschaltet, aber wenn Sie Kameras verwenden, um große Bereiche des Spielraums zu vergrößern oder zu verkleinern, oder wenn Sie mit einem 3D-Spiel arbeiten, dann sollten Sie dies aktivieren und dann die Mipmapping-Funktionen verwenden, um einstellen, wie das Spiel mit den verschiedenen Mipmap-Ebenen umgeht. Wenn Sie dieses Kontrollkästchen aktivieren, wird der Wert für "Randgröße" auf 8 gesetzt (wenn er ursprünglich auf weniger als 8 eingestellt war). Das liegt daran, dass Sie bei der Verwendung von Mipmaps größere Ränder zwischen Bildern auf den Textur-Seiten größere Ränder benötigen, da sie sonst bei höheren Mip-Levels zu verschmelzen beginnen. Der Standardwert von 8 erlaubt es, das Bild auf etwa ein Achtel seiner ursprünglichen Größe zu verkleinern, bevor die Teilbilder anfangen, zusammenzubluten, aber Sie können Sie können diesen Wert jedoch auf jeden beliebigen Wert ändern, der für Ihre Bedürfnisse erforderlich ist. Beachten Sie, dass Sie, wenn Sie die Option "SeparateTexture Page" für Sprites verwenden, hier nicht durch jede Textur gehen und sie markieren müssen, um Mipmaps zu erzeugen, Stattdessen können Sie in den allgemeinen Spieloptionen die Option "Mipmaps für separate Texturseiten generieren" aktivieren.
  • Randgröße - Die Randgröße ändert die Überlappung (oder "Pixelverdopplung") an den Rändern aller Assets auf den für die Texturgruppe generierten Seiten, so dass bei der Skalierung der Bilder keine Lücken (oder Nähte) zwischen ihnen sichtbar sind. Der Standardwert ist 2 Pixel, aber in extremen Fällen, in denen Sie die Ansicht stark vergrößern oder verkleinern, müssen Sie diesen Wert eventuell erhöhen. Beachten Sie, dass der Kachelsatz-Editor eine eine Option zum Einstellen des Ausgaberahmens für Kacheln hat (und auch eine Erklärung liefert, warum dies notwendig ist), und dies ist unabhängig von der Rahmeneinstellung hier - im Grunde werden die Kacheln mit einem Ausgaberahmen für jede Kachel auf einer temporären Textur erzeugt, und diese wird dann der Texturseite mit dem hier angegebenen Rand um die Außenkante hinzugefügt. Beachten Sie auch, dass beim Aktivieren von Mipmaps die Standard-Rahmengröße für die Texturseite automatisch auf 8 gesetzt wird, um ein "Ausbluten" zwischen Texturen bei höheren Mip-Levels zu verhindern, aber dies kann geändert werden.
  • Übergeordnete Texturgruppe - Hier können Sie eine beliebige der vorhandenen Gruppen als "übergeordnete" Gruppe für die aktuell ausgewählte Gruppe auswählen. "Untergeordnete" Gruppen erben alle Änderungen, die an der übergeordneten Texturgruppe vorgenommen werden, und dienen hauptsächlich für die Ressourcenverwaltung bei der Arbeit mit plattformübergreifenden Spielen. Eine Texturgruppe mit einer übergeordneten Gruppe ist immer noch eine eigenständige Texturgruppe und erzeugt Texturseiten wie jede andere Gruppe, aber das Zuweisen einer übergeordneten Gruppe ermöglicht es Ihnen, große Teile für verschiedene Plattformen auszuschließen. große Abschnitte für verschiedene Plattformen. Sie können z. B. eine ganze Reihe von Android-Spieler-Sprites zu einer "player"-Gruppe und Android-Feind-Sprites zu einer "enemy"-Gruppe hinzufügen und dann dafür sorgen, dass beide eine "android"-Elterngruppe verwenden. Wenn Sie dann die übergeordnete Gruppe "android" aus dem Build für Windows oder eine andere Plattform ausschließen, werden auch die untergeordneten Gruppen ausgeschlossen.

Abgesehen von diesen Einstellungen können Sie auch wählen, ob eine bestimmte Texturgruppe für eine bestimmte Plattform exportiert werden soll oder nicht, indem Sie einfach die entsprechenden Kästchen im Abschnitt " Gruppe verwenden auf..." des Fensters aktivieren oder deaktivieren. Beachten Sie jedoch, dass die Texturgruppe <Default> immer auf jede Plattform exportiert wird und Sie keine der Texturgruppen aktivieren/deaktivieren können (nur benutzerdefinierte Texturgruppen erlauben dies).

Texturgruppen sind auch mit den Einstellungen für Konfigurationen verknüpft. Das bedeutet, dass Sie Texturgruppen für jede einzelne Konfiguration Ihres Spiels definieren können und dann die Sprite-, Kachel-Set- oder Font-Ressourcen Zuordnungen zu diesen verschiedenen Gruppen festlegen und es wird "gemerkt", wenn Sie zu einer anderen Konfiguration wechseln. Dasselbe geschieht mit allen Exportoptionen, die Sie im Texturgruppen-Fenster gewählt haben, so dass Sie die Texturgruppen so einstellen können für den Export auf bestimmte Plattformen auf Basis der einzelnen Konfigurationen einstellen.

Eine Anwendung dafür ist z. B., dass Sie eine iPad-Konfiguration erstellen könnten und dafür 2048x2048px Textur-Seiten erstellen (dies über die Spieloptionen einrichten). Sie würden dann alle Grafik-Assets der entsprechenden Texturgruppe zuweisen. Texturgruppe zu, und wenn Sie fertig sind, wechseln Sie zu einer anderen Konfiguration (z. B. für HTML5) und erstellen im Fenster "Texturgruppe" neue Texturgruppen mit 512x512px, denen Sie die Assets zuweisen. Es gibt auch eine eine Schaltfläche am unteren Rand der Seite, mit der Sie die Texturgruppeneinstellungen von einer Konfiguration in eine andere kopieren können.

Schließlich sind Texturgruppen nur für Bitmap-Bilder relevant, und Nicht-Bitmap-Sprites werden keinen Texturgruppen zugewiesen.
