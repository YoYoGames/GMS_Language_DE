Enthaltene Dateien

GameMaker Studio 2 ist nicht nur auf die Assets beschränkt, die Sie innerhalb des Programms erstellen, denn Sie können dem Projekt auch Dateien als eingebundene Dateien hinzufügen, auf die dann während der Ausführung des Spiels zugegriffen werden kann. Welche Dateien Sie hinzufügen, hängt Welche Dateien Sie hinzufügen, hängt davon ab, was Sie tun möchten, aber Dinge wie Standarddateien *.ini, Sprites und Hintergrundbilder oder Sprachtextdateien werden häufig als eingeschlossene Dateien verwendet.

Um diese Dinge zu Ihrem Projekt hinzuzufügen, gibt es ein Editorfenster, das über das Menü Extras des Asset-Browsers geöffnet werden kann:

Wenn Sie dies auswählen, wird der Editor für eingeschlossene Dateien geöffnet:

Um eine Datei zu Ihren Ressourcen hinzuzufügen, verwenden Sie einfach die Schaltfläche Im Explorer öffnen, die einen Datei-Explorer im Ordner datafiles für Ihr Projekt öffnet. Sie können dann Verzeichnisse und Dateien nach Bedarf hinzufügen und GameMaker Studio 2 aktualisiert den Editor für eingefügte Dateien automatisch mit diesen.

Beachten Sie, dass alle Bearbeitungen (Umbenennen, Organisieren, etc...) vom Standard-Dateiexplorer aus erfolgen sollten.

Wenn Sie dem Editor Dateien hinzugefügt haben, können Sie auf doppelklicken, um die Dateieigenschaften zu öffnen:

Dieses Fenster zeigt den Namen der Datei, ihre Größe auf der Festplatte und auch die Plattformen, auf die sie exportiert werden soll, wenn Sie Ihr Spiel kompilieren/testen Ihr Spiel kompilieren/testen. Sie haben Schaltflächen, um "Alle" oder "Keine" auszuwählen, oder Sie können in der Liste nach unten gehen und die Tragets, in die die Datei exportiert werden soll, je nach Bedarf an- oder abwählen.

Weitere Informationen darüber, wie enthaltene Dateien mit Ihrem Spiel gespeichert werden und wie auf sie zugegriffen wird, finden Sie im Abschnitt des Handbuchs über das Dateisystem.

WICHTIG! Beim Einbinden von Sounddateien sollten Sie darauf achten, dass die Dateien keine Namen tragen, die mit Streaming-Sound-Assets kollidieren. Ein Sound-Asset namens "main_music" und eine eingebundene Datei namens "main_music.ogg" zum Beispiel wird beim Ausführen des Projekts undefiniert sein. Dies liegt daran, dass gestreamte Dateien auch außerhalb des Hauptprojekt-Bundles gespeichert werden, daher muss in diesen Situationen Vorsicht geboten sein.

Es ist erwähnenswert, dass Sie DLL (unter Windows) oder DyLib (unter macOS) Dateien als eingeschlossene Dateien einbinden und dann Ihre eigenen benutzerdefinierten Funktionen erstellen können, um mit ihnen umzugehen, indem Sie external_define(). Diese können dann verwendet werden, indem Sie die external_call() Funktion.

WARNUNG! Die Verwendung von eingebundenen Dateien, um auf diese Weise zusätzliche Funktionalität zu erzeugen, ist insofern inkompatibel mit dem Erweiterungssystem, als dass Sie nicht die extern_*() Funktionen mit Dateien, die in einer Erweiterung hinzugefügt wurden, nicht verwenden können, da Erweiterungen bereits eine Methode zum Hinzufügen von Konstanten und Funktionen haben.
