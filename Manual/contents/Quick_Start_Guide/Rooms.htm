Räume

Alle Spiele, die Sie in GameMaker Studio 2 erstellen, benötigen mindestens einen Raum (es können aber auch viele, viele mehr sein), und ein Raum ist einfach ein Bereich, in dem Sie Instanzen der Objekte platzieren, aus denen Ihr Spiel besteht, sowie Kacheln und andere grafische Ressourcen. Wenn Sie zum ersten Mal einen Raum im Asset-Browser erstellen, wird Ihnen ein neuer Raum-Editor Arbeitsbereich, in dem Sie seine Eigenschaften bearbeiten können, etwa so:

Standardmäßig werden die Raumwerkzeuge auf der linken Seite platziert, mit drei Hauptabschnitten:

  • Ebenen-Editor: Alles in einem Raum wird auf Ebenen platziert, und es gibt verschiedene Ebenentypen, aus denen Sie wählen können (mehr dazu später). Die Ebenen sind nach Tiefe geordnet, und diese Tiefe bestimmt die Reihenfolge, in der die Ebeneninhalte auf dem Bildschirm gerendert werden, wenn das Spiel läuft. Die Reihenfolge der Tiefe ist von der höchsten zur niedrigsten, d. h. je niedriger die Tiefe, desto näher an der "Kamera" und je höher die Tiefe, desto weiter weg. Zum Beispiel wird eine Ebene mit einer Tiefe von -300 über einer Ebene mit einer Tiefe von -100 gerendert, und eine Ebene mit einer Tiefe von 1000 wird unter allem gerendert, was eine geringere Tiefe als diese hat. Beachten Sie, dass GameMaker Studio 2 standardmäßig die Ebenentiefen für Sie entsprechend der Position Sie können die Ebenen mit der Maus verschieben, um sie neu anzuordnen, und GameMaker Studio 2 ordnet die Tiefen entsprechend neu an.
  • Ebeneneigenschaften: Jede Schicht, die Sie dem Raum hinzufügen, hat ihre eigenen Eigenschaften, und diese Eigenschaften ändern sich je nach Schichttyp. In diesem Fenster können Sie diese Eigenschaften ändern und festlegen, wie die Ebene gerendert werden soll. gerendert wird.
  • Raumeigenschaften: Auch Räume haben eine Reihe von Eigenschaften, die hier eingestellt werden können. Dinge wie die Raumgröße, die aktiven Kameraansichts-Ports und ein paar andere Dinge können hier eingestellt werden.

Wir haben bereits erwähnt, dass es verschiedene Ebenentypen gibt, also lassen Sie uns kurz durchgehen, was sie sind und wie sie beim Aufbau Ihrer Spiele verwendet werden können:

Hintergrund-Ebene Die Hintergrundebene ist eine Ebene, die mit einer einzelnen Farbe oder einem einzelnen Bild gefüllt werden kann. Sie kann innerhalb des Raums verschoben und positioniert werden, und Sie können mehrere Hintergrundebenen haben. Im Allgemeinen wird sie, wie der Name schon sagt, verwendet, um einen konstanten Hintergrund für alle anderen Ebenen im Raum zu erzeugen. Standardmäßig enthält ein neuer Raum immer eine Hintergrundebene, aber Sie können sie entfernen, wenn Sie sie nicht benötigen.
Instanzschicht Auf der Instanzebene platzieren Sie alle Instanzen der Objekte, die Sie für das Spiel benötigen. Um eine Instanz zu einer Ebene hinzuzufügen, klicken Sie einfach auf auf das Objekt im Asset-Browser und ziehen Sie es dann in den Arbeitsbereich des Raumeditors. Wenn Sie die Maustaste loslassen, wird dem Raum eine Instanz dieses Objekts hinzugefügt. Standardmäßig enthält ein neuer Raum immer eine Instanz-Ebene, aber Sie können sie entfernen wenn Sie sie nicht benötigen.
Kachel Kartenebene Nachdem Sie ein Kachelset erstellt haben, müssen Sie die Kacheln zu Ihrem Raum hinzufügen, was durch Erstellen einer Kachelzuordnungsebene geschieht. Eine Kachelplan-Ebene ist eine Ebene, die es Ihnen ermöglicht, Kacheln aus den von Ihnen erstellten Kachelsatz-Assets hinzuzufügen, und die automatisch so eingerichtet wird, dass sie ein Raster in der Größe der Zellen des Kachelsatzes verwendet. Sie können jeder Rasterzelle nur eine einzige Kachel hinzufügen. Wenn Sie also mehrere Kacheln benötigen, die denselben Platz einnehmen, sollten Sie mehrere Kachelzuordnungsebenen verwenden.
Pfad Schicht Die Pfadebene ist die einzige Ebene, die nicht gerendert wird, wenn Sie Ihr Spiel ausführen. Diese Ebene ist eher eine "Komfort"-Ebene, die Ihnen beim Erstellen oder Bearbeiten von Pfad-Assets hilft, da sie es Ihnen ermöglicht, Pfade innerhalb des eigentlichen Raums hinzuzufügen oder zu bearbeiten.
Asset-Ebene Manchmal wollen Sie einen schönen grafischen Effekt, aber Sie wollen nicht den Overhead einer Instanz oder brauchen sie nicht, um etwas anderes zu tun, als sich selbst zu zeichnen. Sie könnten eine Kachelkarte verwenden, aber da Kachelkarten auf die Verwendung eines Rasters und nur eines Bildes pro ein einzelnes Bild pro Rasterzelle zu verwenden, können sie etwas restriktiv sein. In diesem Fall würden Sie die Asset-Ebene verwenden. Die Asset-Ebene nimmt einfach eine Sprite-Ressource und zeichnet sie mit den Parametern, die Sie beim Hinzufügen zum Raum festlegen. Sprites und Sequenzen können zu dieser Ebene auf die gleiche Weise hinzugefügt werden wie Instanzen von Objekten zur Instanz-Ebene, d. h. Sie klicken auf auf das Sprite im Asset-Browser und ziehen es dann ziehen Sie es in den Arbeitsbereich des Raumeditors.

Um dem Raum-Editor eine Instanz hinzuzufügen, müssen Sie zunächst eine Instanz-Ebene aus dem Ebenen-Editor auf der linken Seite auswählen (oder eine erstellen, wenn keine vorhanden ist). Sie können dann zum Asset-Browser gehen und einfach auf klicken und ein gewünschtes Objekt von dort in den Raum an der gewünschten Position ziehen, wodurch eine einzelne Instanz dieses Objekts zu Ihrem Spiel hinzugefügt wird:

Wenn Sie mehrere Instanzen im Raum platzieren möchten, dann können Sie ein Objekt aus dem Ressourcenbaum auswählen und dann drücken und halten, um eine Vorschau der Ressource zu erhalten, und wenn Sie zusätzlich mit der linken Maustaste klicken, können Sie es dann in die Raumebene "malen".

Beachten Sie, dass den Assets eine Position zugewiesen wird, wenn sie in den Raumeditor eingefügt werden, basierend auf ihrer X- und Y-Achsenposition relativ zum Raumursprung. Der Raumursprung ist die Position (0, 0), die in GameMaker Studio 2 die obere linke Ecke ist, und nach rechts ist +X und nach unten ist +Y, z. B:

Im obigen Beispiel befindet sich die Instanz in der Mitte des Raums an der Position (X:512, Y:384). Es ist auch möglich, sich außerhalb der Raumgrenzen zu gehen und Positionen zu haben, die größer als die Raumachsenlängen oder sogar kleiner als der Raumursprung sind - in diesem Fall sind die X/Y-Koordinaten negative Werte.
