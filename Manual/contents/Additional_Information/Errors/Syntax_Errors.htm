Syntax-Fehler

Die erste Ebene der Fehlermeldung bei der Programmierung Ihrer Spiele in GameMaker Studio 2 befindet sich im Code- oder DnD™-Editor, und zwar die Syntaxprüfung. Während Sie Ihren Code korrigieren - oder Ihre Aktionen hinzufügen - GameMaker prüft, ob er der etablierten Syntax für die Sprache folgt, und markiert alle gefundenen Probleme, damit Sie sie beheben können, bevor das Spiel ausgeführt oder eine ausführbare Datei kompiliert wird.

Diese Fehler fallen in zwei Hauptkategorien:

  • Warnungen: Diese werden für Dinge angezeigt, die ein Fehler sein könnten, es aber nicht sind, und die wahrscheinlich nicht das das Spiel nicht am Kompilieren hindern, aber Laufzeitfehler provozieren können. Die häufigsten Ursachen für diese Warnungen sind, wenn eine Variable in einem Ausdruck verwendet wird und sie noch nirgendwo anders definiert wurde, oder wenn eine Variable definiert wurde, aber nicht verwendet wurde.
  • Fehler: Dies sind Probleme, die es nicht erlauben, das Spiel zu kompilieren, geschweige denn zu starten. Diese sollten behoben werden, bevor Sie mit dem Code weiterarbeiten. Die Tabelle unten zeigt die häufigsten Ursachen dafür.

Syntaxfehler werden aktualisiert, während Sie Ihren Code schreiben (es gibt eine kurze Pause zwischen der Eingabe von etwas und dem Erscheinen von Fehlern in diesem Fenster, um zu verhindern, dass Fehler für unfertigen Code gemeldet werden) und folgen dem Format:

[Objekt] - [Ereignis] - [Zeilennummer] - [Position in Zeile]: [Fehlerstring]

Oder, wenn der Fehler in einem Skript liegt, dann schon:

[Skript] - [Zeilennummer] - [Position in der Zeile]: [Fehlerstring]

Durch einen Doppelklick auf auf einen der in der Ausgabekonsole angezeigten Fehler gelangen Sie zu der Zeile im Objektereignis oder Skript, die den Fehler aufweist, so dass Sie bearbeiten können.

Es ist zu beachten, dass einige dieser Fehler an bestimmten Stellen in Ihrem Code angezeigt werden, der eigentliche Fehler aber an anderer Stelle liegen kann, oder ein einziges Problem kann mehrere Fehlermeldungen hervorrufen:

In der folgenden Tabelle sind die verschiedenen Syntaxfehler und ihre Hauptursachen aufgeführt:

Fehler Typ Beschreibung
Deformierte Referenz [val] Dies bedeutet, dass ein interner AST-Validierungsfehler vorliegt (Sie sollten diesen Fehler nie sehen, aber falls doch, ziehen Sie bitte in Erwägung, einen Fehlerbericht über das Hilfemenü einzureichen)
Fehlerhafte Escape-Sequenz für hexadezimale Zeichen Dies kann passieren, wenn Sie den hexadezimalen Wert für ein Escape-Zeichen in einer Zeichenkette verwendet haben und diese das falsche Format hat.
Unerwarteter Knoten [val] Dies bedeutet, dass ein interner AST-Validierungsfehler vorliegt (Sie sollten diesen Fehler nie sehen, aber falls doch, ziehen Sie bitte in Erwägung, einen Fehlerbericht über das Hilfemenü einzureichen)
Exception beim Parsen von [val] Dies bedeutet, dass ein interner AST-Validierungsfehler vorliegt (Sie sollten diesen Fehler nie sehen, aber falls doch, ziehen Sie bitte in Erwägung, einen Fehlerbericht über das Hilfemenü einzureichen)
Fehlerhafte id-Referenz [val]
Unnötiger Ausdruck [val] als Anweisung verwendet Dieser Fehler tritt auf, wenn die Syntaxprüfung einen Ausdruck (dargestellt in [val]) erkennt, wo sie eine Anweisung erwartet hat. Zum Beispiel wird der folgende Code den Fehler auslösen, da er den ternären Operator ? erwartet, aber auf 32 stößt:
temp_x = x < (room_width / 2)  32: room_width - 32;
Fehlerhafte Array-Referenz [val] Dies bedeutet, dass Sie versucht haben, ein Array auf eine falsche Art und Weise zu referenzieren, und [val] wird die Referenz anzeigen.
Unerwarteter Binäroperator [val] Dieser Fehler tritt auf, wenn Sie einen der bitweisen Operatoren an einer Stelle verwenden, an der er nicht verwendet werden sollte, wobei [val] zeigt, wo.
String-Literal ohne Ende Dieser Fehler tritt auf, wenn Sie eine Zeichenkette mit @“ oder “ öffnen, sie aber nicht vor dem Ende des Codes schließen.
Einfache Anführungszeichen für String nicht mehr erlaubt Dieser Fehler tritt auf, wenn Sie versuchen, Zeichenketten zu verwenden, die in einfache Anführungszeichen eingeschlossen sind, z. B.: "Hello World", anstelle von doppelten Anführungszeichen, z. B.: "Hello World".
Unerwarteter Syntaxfehler Dieser Fehler kennzeichnet einen Syntaxfehler, den der Parser nicht näher identifizieren kann.
Erhalten "[val1]", erwartet "[val2]" Dies geschieht, wenn der Syntax-Checker einen Wert oder ein Symbol erwartet, aber einen anderen erhält, wobei [val1] das aktuelle Symbol oder der aktuelle Wert und [val2] das erwartete Symbol oder der erwartete Wert ist. Zum Beispiel:
Got "{", expected "("
Erhalten "[val1]", erwartet "[val2]" oder "[val3]" Wie oben, aber für den Fall, dass es mehrere mögliche erwartete Symbole gibt.
Erwarteter Ausdruck Dieser Fehler tritt auf, wenn die Syntaxprüfung einen Ausdruck erwartet, dieser aber nicht gegeben ist.
Leere [val]-Anweisung Dies geschieht, wenn Sie eine Anweisung beenden und sie leer lassen. Der folgende Code gibt zum Beispiel den Fehler (beachten Sie das abschließende Semikolon):
if (x < 0);
Verschachtelte Funktionsaufrufe sind nicht erlaubt Dies kann passieren, wenn Sie versuchen, Funktionsaufrufe innerhalb einer Anweisung zu verschachteln, z. B:
ds_list_find_value(list, 0)).sprite_index
Die Verwendung von [val] ist in der GameMaker Studio 2 Testversion nicht erlaubt Diese Meldung bedeutet, dass die für [val] gezeigte Funktion oder das Symbol nicht für die Verwendung unter der Testlizenz für GameMaker Studio 2 verfügbar ist.
Anzahl der Argumente für Funktion [val1] erwartet [val2] bekam val[3] Dieser Fehler sagt Ihnen, dass Sie der für [val1] gezeigten Funktion nicht die richtige Anzahl von Argumenten übergeben haben, die die in [val2] gezeigte Anzahl erwartet, aber die in [val3] gezeigten bekommen hat.
Anzahl der Argumente für Funktion [val1] erwartet [val2] - [val3] erhalten [val4] Wie oben, nur für einen Bereich von Argumenten.
Duplizierter Enum-Eintrag gefunden Dieser Fehler sagt Ihnen, dass Sie einen Eintragsnamen einer Enum-Variablen dupliziert haben
Dupliziertes Enum gefunden Dieser Fehler sagt Ihnen, dass Sie zwei oder mehr Enums mit demselben Namen definiert haben.
Fehlender Variablenname in globalvar Dies sagt Ihnen, dass Sie die Deklaration globalvar verwendet haben, aber keinen Variablennamen angegeben haben.
Fehlende Variablennamen in var Dies sagt Ihnen, dass Sie die Deklaration var verwendet haben, aber keinen Variablennamen angegeben haben.
Keine passende #region für #endregion gefunden Dieser Fehler bedeutet, dass Sie in Ihrem Code einen Endbereich deklariert haben, ohne einen Startbereich zu definieren.
Nicht geschlossener #Bereich am Ende des Skripts gefunden Dieser Fehler bedeutet, dass Sie irgendwo im Code einen Bereich deklariert haben, aber nirgends einen Endbereich dafür definiert haben.
Unerwarteter Terminal-Operator [val] Dies bedeutet, dass ein interner AST-Validierungsfehler vorliegt (Sie sollten diesen Fehler nie sehen, aber falls doch, ziehen Sie bitte in Erwägung, einen Fehlerbericht über das Hilfemenü einzureichen)
Unerwarteter unärer Operator [val] Dies geschieht, wenn ein unärer Operator (wie +, -, =, etc...) in Ihrem Code an einer Stelle gefunden wurde, an der er nicht erwartet wird, wobei [val] Ihnen den Operator anzeigt.
Unerwarteter ternärer Operator [val] Dieser Fehler bedeutet, dass Sie einen der Teile des ternären Operators irgendwo in Ihrem Code falsch verwendet haben, wobei [val] den Teil anzeigt, der im Fehler ist.
Try muss catch oder finally Klausel haben Dieser Fehler bedeutet, dass Sie die Funktion try aufgerufen haben, aber es versäumt haben, eine catch oder finally Klausel einzuschließen.
Makro [val] existiert bereits Sie können diesen Fehler erhalten, wenn Sie versuchen, ein Makro [val] mit demselben Namen zu definieren wie ein zuvor definiertes.
Fehlerhafte Variablenreferenz, got [val] Dies bedeutet, dass ein interner AST-Validierungsfehler vorliegt (Sie sollten diesen Fehler nie sehen, aber falls doch, erwägen Sie bitte, einen Fehlerbericht über das Hilfemenü zu erstellen)
Zuweisung zu einem Ausdruck mit mehrfacher relationaler Gleichheit - GML unterstützt keine Mehrfachzuweisungen in einem Ausdruck Dieser Fehler teilt Ihnen mit, dass es ein Problem mit einer Mehrfach-Operator-Zuweisung (auch zusammengesetzte Zuweisung genannt) gibt, die von der GameMaker-Sprache nicht unterstützt wird. Dazu gehören Dinge wie "*=" oder "/=", usw...
Makro [val] ist unbenutzt Dies geschieht, wenn das Makro [val] irgendwo in Ihrem Code unbenutzt ist.
Variable [val] nur einmal referenziert Das bedeutet, dass die angegebene Variable [val] nur einmal in Ihrem Code referenziert wird.
Nicht zugewiesene Variable [val] referenziert Das bedeutet, dass die Variable [val] in Ihrem Code referenziert wurde, aber noch nirgends ein Wert zugewiesen wurde.
Nur Funktionen, die als Konstruktoren deklariert sind, können die Vererbung nutzen Dieser Fehler sagt Ihnen, dass Sie versucht haben, die Vererbung auf eine Funktion anzuwenden, die nicht als Konstruktor deklariert wurde.
Unbekanntes Funktionsattribut [val] Dieser Fehler bedeutet, dass Sie versucht haben, bei der Deklaration einer in-kine-Funktion einen ungültigen oder nicht identifizierten Funktionsmodifikator ([val]) zu verwenden (ein Funktionsmodifikator ist so etwas wie das Schlüsselwort constructor, das unterstützt wird).
Geerbtes Argument \"[val]\" ist nicht in Funktionsargumenten enthalten Dieser Fehler tritt auf, wenn Sie versuchen, in einer geerbten Funktion auf ein Argument zu verweisen, das in der übergeordneten Funktionsdefinition nicht deklariert wurde.
val] Erstellungscode Das passiert, wenn Sie irgendwo einen Fehler [val] im Raumerstellungscode haben.
