Kompilier-Fehler

Die zweite Art von Fehler, die für Ihr Spiel gemeldet werden kann, ist der Compiler-Fehler. Ein Compiler-Fehler tritt auf, wenn Ihr Spiel auf eine Art von Fehler stößt, den der Syntax-Checker möglicherweise nicht erkennen konnte - es ist sehr einfach, z. B. zu vergessen, eine Variable zu deklarieren oder eine Zeichenkette fälschlicherweise als Real einzugeben. Es ist sehr einfach, zu vergessen, eine Variable zu deklarieren oder eine Zeichenkette fälschlicherweise als Real einzugeben - oder wenn ein Fehler damit zusammenhängt, wie Sie die Kompilierwerkzeuge in den Plattformeinstellungen für die Zielplattform eingerichtet haben. Diese Fehler werden von GameMaker Studio 2 abgefangen, während das Spiel kompiliert wird, und diese Informationen werden auch in der Compiler-Fehlerausgabe angezeigt, wie unten dargestellt:

HINWEIS: Bei diesen Fehlern wird in der Regel kein Fenster angezeigt, sondern lediglich "Build Failed" in der Compiler-Ausgabe. Wenn ein Fenster erscheint, handelt es sich normalerweise um einen Runner-Fehler, der hier erklärt wird.

Die Compiler-Fehlermeldungen folgen alle dem gleichen Format:

[Objekt] - [Ereignis] - [Zeilennummer]: [Fehlerstring]

Wenn der Fehler in einem Skript gefunden wird, dann wird es einfach sein:

[Skript] - [Zeilennummer]: [Fehlerstring]

Sie können dann auf auf einen der Compiler-Fehlereinträge doppelklicken, um das betreffende Asset an der Stelle zu öffnen, an der der Fehler aufgetreten ist, und Sie können dann Sie können dann die in der Meldung enthaltenen Informationen verwenden, um genau festzustellen, wo im Objekt oder Skript der Fehler aufgetreten ist. Manchmal können diese Fehler jedoch etwas kryptisch erscheinen, daher finden Sie unten eine vollständige Liste aller Fehler und eine kurze Erklärung ihrer Bedeutung:
