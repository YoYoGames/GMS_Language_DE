Läufer-Fehler

Selbst nach der Syntaxprüfung im Code-Editor und der anschließenden Überprüfung Ihres Codes durch den Compiler auf Compiler-Fehler kann es immer noch vorkommen, dass etwas schief geht. In den meisten Fällen wird dann ein Virtual Machine (VM)- Runner-Fehler (auch Laufzeitausnahme genannt) ausgelöst, der wie folgt aussieht:

Runner-Fehler sind im Allgemeinen schwerwiegender als Kompilierfehler, da sie bedeuten, dass mit Ihrem Code etwas ernsthaft falsch ist, das weder der Code-Editor noch der Compiler erkennen konnten. Ihrem Code etwas nicht stimmt, was weder der Code-Editor noch der Compiler erkennen konnten, und deshalb sollten Sie auf solche Fehler achten. Wenn ein Fehler auftritt, können Sie mit der Schaltfläche Kopieren im Popup den Fehler in die Zwischenablage kopieren in die Zwischenablage kopieren, die Sie dann in eine Textdatei (oder wo auch immer) zur späteren Verwendung einfügen können.

Der Aufbau dieses Fehlers ist wie folgt:


  • Hier wird das Ereignis angezeigt, in dem der Fehler aufgetreten ist, sowie das Objekt (falls zutreffend).

  • Hier wird der tatsächliche Fehler angezeigt. Es zeigt also den Umfang des Fehlers und den Fehlertyp, um den es sich handelt. Der erste Wert in den Klammern ist die Instanz-ID der Instanz, die den Code ausgeführt hat - oder es könnte einer der speziellen Werte sein, die in der Tabelle unten aufgeführt sind - und der zweite ist ein interner Wert, den YoYo Games zur Identifizierung von Bugs verwendet und ignoriert werden kann. Im Fall des obigen Bildes heißt es, dass die Variable _image nicht auf das Objekt obj_EA_DEMO_Control gesetzt wurde (d.h.: existiert nicht, bevor darauf zugegriffen wird).

  • Hier wird die Zeilennummer angezeigt, in der der Fehler auftritt, sowie ein Codeschnipsel, der die Code, der die Position anzeigt. Beachten Sie, dass dies aufgrund der Art und Weise, wie das Spiel kompiliert wird, nicht immer zu 100 % genau ist, aber es sollte ausreichen, um genau herauszufinden, wo das Problem liegt.

Wie oben erwähnt, werden bestimmte Fehlermeldungen nicht durch einen Instanz-ID-Wert, sondern durch einen negativen Wert identifiziert. Diese Werte können verwendet werden, um die genaue Art des Fehlers und worauf er sich bezieht, mit den folgenden Werten festzulegen möglich:

Präfix Umfang
-1 Selbst
-2 Andere
-3 Alle
-4 Keiner
-5 Global
-6 Nicht spezifiziert
-7 Lokal

Die möglichen Fehler vom VM-Runner sind wie folgt:

Fehler Nachricht Betrieb Beschreibung
DoSet Ungültiger Vergleichstyp Daten-Typen Dies zeigt an, dass der Läufer versucht hat, zwei inkompatible Datentypen zu vergleichen, z. B. eine reelle Zahl mit einer Zeichenkette.
DoConv Ausführungsfehler Daten-Typen Dies bezeichnet einen Fehler bei der Umwandlung eines Datentyps in einen anderen.
DoAdd Ausführungsfehler Mathe Bei der Verwendung des Additionsausdrucks (+) ist etwas schief gelaufen.
DoMul Ausführungsfehler Mathe Bei der Verwendung des Multiplikationsausdrucks (*) ist etwas schief gelaufen.
DoSub Ausführungsfehler Mathe Bei der Verwendung des Subtraktionsausdrucks (-) ist etwas schief gelaufen.
DoSub Execution Engine - Kann nicht auf String-Typ operieren Mathe Sie versuchen, den falschen Typ von Variablen zu subtrahieren (z. B. eine Zeichenkette von einer reellen Zahl zu subtrahieren).
DoDiv Ausführungsfehler Mathe Bei der Verwendung des Divisionsausdrucks (/ oder div) ist etwas schief gelaufen.
DoDiv Execution Engine - Kann nicht auf String-Typ operieren Mathe Sie versuchen, den falschen Typ von Variablen zu dividieren (z. B. eine Zeichenkette durch eine reelle Zahl zu dividieren).
DoDiv Dividieren durch Null Mathe Sie versuchen, durch 0 zu dividieren (beachten Sie, dass dies nur bei der Verwendung von Ganzzahldivisionen passiert, die Division eines reellen Wertes (ungleich Null) durch 0 ergibt Unendlich als Antwort, die Division von Null durch Null ergibt NaN als Antwort). Sie können diese Werte überprüfen mit (is_infinity) und (is_nan)
DoMod Ausführungsfehler Mathe Bei der Verwendung des Modulo-Ausdrucks (mod) ist etwas schief gelaufen.
DoMod Execution Engine - Kann nicht auf String-Typ operieren Mathe Sie versuchen, modulo (mod) auf den falschen Typ von Variablen anzuwenden (z. B. mod eine Zeichenkette durch einen Real).
DoAnd Ausführungsfehler Bitweise Bei der Verwendung des bitweisen "und"-Ausdrucks (&) ist etwas schief gelaufen.
DoAnd Execution Engine - Kann nicht auf String-Typ operieren Bitweise Sie versuchen, das bitweise "und" (&) auf den falschen Typ von Variablen anzuwenden (z. B. versuchen Sie, eine Zeichenkette mit einem Real zu "und").
DoOr Ausführungsfehler Bitweise Bei der Verwendung des bitweisen "oder"-Ausdrucks (|) ist etwas schief gelaufen.
DoOr Execution Engine - Kann nicht auf String-Typ operieren Bitweise Sie versuchen, "oder" (|) auf den falschen Variablentyp anzuwenden (z. B. versuchen Sie, eine Zeichenkette mit einem Real zu "oder").
DoXor Ausführungsfehler Bitweise Bei der Verwendung des bitweisen "xor"-Ausdrucks (^) ist etwas schief gelaufen.
DoXor Execution Engine - Kann nicht auf String-Typ operieren Bitweise Sie versuchen, "xor" (^) auf den falschen Typ von Variablen anzuwenden (z. B. versuchen Sie, eine Zeichenkette mit einem Real zu "xorieren").
DoShl Ausführungsfehler Bitweise Beim Bitshifting nach links (<<) eines Wertes ist etwas schief gelaufen.
DoShl Execution Engine - Kann nicht auf String-Typ operieren Bitweise Sie versuchen, den falschen Typ von Variablen nach links zu schieben (<<) (z. B. versuchen Sie, eine Zeichenkette nach links zu schieben).
DoShr Ausführungsfehler Bitweise Beim Bitshifting nach rechts (>>) eines Wertes ist etwas schief gelaufen.
DoShr Execution Engine - Kann nicht auf String-Typ operieren Bitweise Sie versuchen, den falschen Variablentyp nach rechts zu schieben (>>) (z. B. versuchen Sie, einen String nach rechts zu schieben).
DoNeg Ausführungsfehler Negieren Sie versuchen, einen Variablentyp in ein Negativ zu verwandeln, obwohl dieser Typ eine solche Operation nicht zulässt.
DoNot Ausführungsfehler Negieren Sie versuchen, einen Variablentyp zu "nicht", wenn dieser Typ eine solche Operation nicht zulässt.
Push Ausführungsfehler - Variablenindex außerhalb des Bereichs (var) Stapel Die Variable, auf die zugegriffen wird, liegt außerhalb des festgelegten Bereichs für den Läufer.
Push Variable holen (var) Stapel Die angegebene Variable wurde nicht definiert oder ist unbekannt.
Pop Variable Index außerhalb des Bereichs (var) Stapel Die Variable, auf die zugegriffen wird, liegt außerhalb des festgelegten Bereichs für den Läufer.
Pop Variable holen (var) Stapel Die angegebene Variable wurde nicht definiert oder ist unbekannt.
With Kann nicht global in mit-Anweisung verwenden Mit Sie haben versucht, "global" als Variable innerhalb einer "with" Anweisung zu verwenden, d.h.:
with (global)
   {
   //do something;
   }
With Kann nicht lokal in mit-Anweisung verwenden Mit Sie haben versucht, "local" als Variable innerhalb einer "with" Anweisung zu verwenden, d.h.:
with (local)
   {
   //do something;
   }
DoCall Ausführungsengine Typfehler Motor Dies ist ein undefinierter Fehler innerhalb der Virtuellen Maschine. Sie sollten einen Fehlerbericht einreichen, wenn dies passiert (siehe: Das Hilfemenü für Details, wie Sie dies tun können.
Stack Overflow - Motor Ein Stapelüberlauf tritt auf, wenn zu viel Speicher auf dem Aufrufstapel verwendet wird und wenn Ihr Spiel versucht, mehr Platz zu verwenden, als auf dem Aufrufstapel verfügbar ist (d. h. wenn es versucht, auf Speicher außerhalb der Grenzen des Aufrufstapels zuzugreifen, was im Wesentlichen ein Pufferüberlauf), läuft der Stack über, was zu einem Programmabsturz führt. Starten Sie Ihren Computer und GameMaker neu: Studio und wenn der Fehler weiterhin besteht, wenden Sie sich bitte an den Support und/oder melden Sie einen Fehler (wie oben beschrieben).
