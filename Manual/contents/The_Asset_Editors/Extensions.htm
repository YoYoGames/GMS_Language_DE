Der Erweiterungseditor

Ein Erweiterungspaket kann verwendet werden, um der GameMaker Language(GML) eine Reihe zusätzlicher Funktionen und Konstantenhinzuzufügen, oder als eine Möglichkeit, um Spiel-Assets oder sogar ganze Projekte einfach zwischen Personen zu übertragen. Wenn die Erweiterung zusätzliche Funktionalität hinzufügen soll, dann können die verwendeten Funktionen Ihr Spiel mit externen SDKsoder Quellcode, der in anderen Sprachen (wie z. B. Java für Android) geschrieben wurde, sowie mit nativem GML verbinden.

Normalerweise werden 3rd-Party-Erweiterungen aus dem GameMaker Marktplatz und diejenigen, die Sie abonniert haben werden dann im Bibliotheksfenster des Marketplace aufgelistet und können von dort aus installiert werden. Wenn Sie Ihr Erweiterungspaket nicht aus dem Marketplace bezogen haben, können Sie es trotzdem zu einem Projekt hinzufügen, indem Sie erneut Sie können es trotzdem zu einem Projekt hinzufügen, indem Sie erneut mit der rechten Maustaste auf irgendwo im Asset-Browser klicken und dann Vorhandenes hinzufügen auswählen und dann zum Speicherort der Erweiterung navigieren, um sie auszuwählen. der Erweiterung navigieren, um sie auszuwählen.

Wenn Sie eine *.GMEZ mit einer früheren Version von GameMaker Studio erstellt haben, können Sie diese über die rechte Maustaste Menüoption Erweiterung importieren hinzufügen.

HINWEIS: Erweiterungen im Legacy-Format *.GEX werden nicht mehr unterstützt und können nicht in GameMaker Studio 2-Projekte importiert werden.

Nachdem Sie eine Erweiterung zum Asset-Browser hinzugefügt haben, sollten Sie ihr einen Namen und eine Versionsnummer geben und außerdem festlegen, auf welche Plattformen sie exportiert werden soll. Indem Sie das Häkchen bei ausgewählten Plattformen entfernen, können Sie GameMaker Studio 2 anweisen, die Erweiterung nicht als Teil der ausführbaren Datei für diese Plattformen zu exportieren. Sie sollten auch sicherstellen, dass die Eigenschaften der Erweiterung mit der/den Zielplattform(n) übereinstimmen, und die Exportziele für die Erweiterungsdateien auf der Grundlage der folgenden Angaben bearbeiten:

  • Alle Ziele: GML-Erweiterungen. Dies sind Erweiterungen, die mit der GameMaker Language(GML) erstellt wurden und mit allen Zielplattformen kompatibel sind.
  • Windows(Standard, UWP, XBoxOne): Sie können Windows DLL Dateien (Dynamic Link Libraries) verwenden. Bitte beachten Sie, dass, wenn Sie in den Windows-Spieleoptionen das Häkchen bei "x64 Windows Runtime verwenden " gesetzt haben , dann müssen alle DLLs für das Projekt 64bit-kompatibel sein. 32bit-DLLs werden nicht mit dem 64bit-Executable funktionieren.
  • macOS: Das macOS-Ziel kann die Standarddateien DyLib für Erweiterungen verwenden.
  • Ubuntu (Linux): Für das Ubuntu-Ziel müssen Sie sicherstellen, dass Ihre Erweiterung SO Dateien verwendet.
  • HTML5: Diese Plattformen können JavaScript Erweiterungen verwenden.
  • Android: Diese Plattform erfordert eine Erweiterung, die in Java oder einem Android Framework SDK geschrieben werden muss.
  • iOS / tvOS: Erweiterungen für diese Plattform sind in Objective C geschrieben und erfordern eine *.mm und eine *.h Datei, oder wenn Sie irgendwelche Frameworks verwenden, eine vorkompilierte *.a native Datei.
  • PlayStation: Für PlayStation 3 oder 4 sollten Ihre Erweiterungen PRX Dateien sein, wobei jedes Ziel eine andere PRX benötigt, um der Spezifikation des Ziels (entweder PS3 oder PS4) zu entsprechen. Wenn Sie auf einer PSVita arbeiten, sollte die Erweiterung stattdessen SUPRX lauten.

HINWEIS: Einige Erweiterungen müssen bestimmten Namenskonventionen folgen, damit GameMaker die Plattform und CPU-Architektur, für die sie ausgelegt sind, korrekt identifizieren kann. Weitere Informationen finden Sie im Abschnitt über das Erstellen von Erweiterungen.

Da eine einzelne Erweiterung verschiedene zugehörige Dateien haben kann, ist es möglich, eine einzelne Erweiterung für mehrere Plattformen zu erstellen. In diesem Fall müssen Sie alle kompatiblen Plattformen im Haupteditor der Erweiterung angeben (d. h.: das Kontrollkästchen neben ihnen) und dann in den individuellen Dateieigenschaften für jeden Teil der Erweiterung die Plattform festlegen, auf die diese Dateien exportiert werden sollen.

Um die Zielplattform für eine bestimmte Erweiterungsdatei oder einen Platzhalter zu ändern, müssen Sie zunächst die Eigenschaften der Erweiterung öffnen (doppelklicken Sie auf die Erweiterung im Asset-Browser oder verwenden Sie und wählen Sie Eigenschaften öffnen), und wählen Sie dann in den Erweiterungseigenschaften rechts die Ressourcendatei aus, die Sie bearbeiten möchten:

HINWEIS: Wenn Sie auf das Ziel macOS oder Ubuntu exportieren, können Sie die Option "Als Zip-Datei entpacken" einstellen, um die Dateiberechtigungen für die Erweiterung zu erhalten.

Sobald Sie Ihre Erweiterung installiert haben, können Sie die Funktionen und Konstanten dafür genauso verwenden wie die eingebauten Funktionen und Konstanten in der GameMaker-Sprache. Wenn Ihr Spiel plattformübergreifend ist (z. B. wenn es sowohl auf Windows und Android laufen soll), dann sollten Sie die Betriebssystemkonstanten und -funktionen verwenden, um die Plattform zu prüfen, bevor Sie erweiterungsspezifische Funktionen aufrufen, um Probleme zu vermeiden, zum Beispiel:

switch(os_type)
{
case os_android:
call_android_extension_function(x, y, z);
break;
case os_windows:
call_windows_extension_function(x, y, z);
break;
}

Beachten Sie, dass Ihre Erweiterungen für verschiedene Plattformen Funktionsnamen gemeinsam nutzen können. Wenn Sie also eine Erweiterung mit unterschiedlichen Dateien haben, um (z. B.) eine Meldung für Windows und für Android einzublenden, können beide Erweiterungsdateien denselben Funktionsnamen nutzen, solange solange sie beide so eingestellt sind, dass sie nur für die jeweilige Plattform exportiert werden.

YoYo Games 3rd Party Erweiterungen

Ältere Versionen von GameMaker hatten viele Funktionen von Drittanbietern eingebaut, aber das war ineffizient und schwierig zu warten, da, wenn eine bestimmte Funktionalität von einem Drittanbieter geändert wurde, die gesamte GameMaker IDE und Laufzeit aktualisiert werden musste. Daher haben aktuelle Versionen die Verwendung von Erweiterungspaketen übernommen, um die Unterstützung von Drittanbietern zu Ihren Spielen hinzuzufügen. Das heißt, Sie können Das bedeutet, dass Sie nur die zusätzlichen Funktionen hinzufügen können, die Sie benötigen, und dass Sie diese sogar an Ihre speziellen Bedürfnisse anpassen und bearbeiten können.

Die meisten dieser Erweiterungen werden für Sie heruntergeladen und installiert, wenn Sie eine bestimmte Spieloption auswählen (wie z.B. Facebook oder Google Mobile Ads), aber Sie können weitere Informationen über sie finden unter dem entsprechenden Abschnitt für jede Plattform in der YoYo Games Knowledge Base:

  • GameMaker Studio 2 Plattformen und Einrichtung

Die Erweiterungen selbst finden Sie auf der folgenden Marketplace-Seite:

  • YoYo Games Marktplatz Assets
