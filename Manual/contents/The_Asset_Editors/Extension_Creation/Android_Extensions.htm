Android-Erweiterungen

Um eine Erweiterung für Android zu erstellen, müssen Sie es in zwei Teilen tun. Der erste Teil wäre das Hinzufügen der Erweiterung selbst, zusammen mit den erforderlichen Dateien usw... und der zweite Teil ist die Erstellung der Funktionen und Makros/Konstanten, die die Erweiterung benötigt. Die Funktionen und Konstanten werden mit Hilfe von Platzhalterdateien hinzugefügt, um sie zu gruppieren. Sie würden also einen Platzhalter hinzufügen und dann die Funktionen und Makros wie im obigen Abschnitt beschrieben definieren. Um den Rest der Dateien hinzuzufügen, müssen Sie jedoch müssen Sie zuerst das Kontrollkästchen Android Kontrollkästchen im Abschnitt " Zusätzliche Funktionen" des Editors aktivieren:

Hier machen Sie die folgenden Angaben:

  • Klassenname: Ihre Erweiterung kann mehrere Klassen haben, wobei jede Klasse ihre eigenen Funktionen und Konstanten hat, daher sollten Sie ihr einen Namen geben, der ihren Zweck widerspiegelt.
  • Android-Berechtigungen: Hier können Sie alle zusätzlichen Berechtigungen hinzufügen, die Ihre Erweiterung benötigt. Welche Berechtigungen dies sind, hängt ganz von der Verwendung der Erweiterung ab. Daher sollten Sie die von Google bereitgestellte Dokumentation für die Android-Plattform prüfen oder, wenn Sie ein SDK eines Drittanbieters verwenden, die mit dem SDK gelieferte Dokumentation. Um eine neue Berechtigung hinzuzufügen, müssen Sie auf die Schaltfläche klicken, um eine Platzhalterberechtigung hinzuzufügen, und dann mit einem langsamen Klick auf die Berechtigung so bearbeiten, dass sie den Anforderungen entspricht. Sie können Berechtigungen mit der Schaltfläche entfernen.
  • In die Gradle-Abhängigkeiten injizieren: Hier können Sie jeglichen zusätzlichen Code hinzufügen, der in die Gradle-Build-Abhängigkeiten injiziert (hinzugefügt) werden soll.
  • In das AndroidManifest.xml-Manifest injizieren: Hier legen Sie jeglichen zusätzlichen Code fest, der in die Android-Manifest-XML-Datei injiziert (hinzugefügt) werden soll, wenn Ihr Spiel zum Testen oder für die endgültige Veröffentlichung gebaut wird. Stellen Sie sicher, dass Sie dies (und Ihre Berechtigungen) sorgfältig überarbeiten bevor Sie Spiele an den Store übermitteln, da falsche Einstellungen dazu führen, dass Ihr Spiel nicht übermittelt werden kann.
  • In die AndroidManifest.xml-Anwendung injizieren: Hier legen Sie zusätzlichen Code fest, der in die Android-Manifest-XML-Datei unter der Überschrift "Application" (Anwendung) injiziert (hinzugefügt) werden soll, wenn Ihr Spiel zum Testen oder zur endgültigen Veröffentlichung erstellt wird. Stellen Sie sicher, dass Sie (und Ihre Berechtigungen) sorgfältig, bevor Sie Spiele an den Store übermitteln, da falsche Einstellungen dazu führen, dass Ihr Spiel nicht übermittelt werden kann.
  • Inject to AndroidManifest.xml RunnerActivity: Hier legen Sie zusätzlichen Code fest, der in die Android-Manifest-XML-Datei unter der Überschrift "RunnerActivity" injiziert (hinzugefügt) werden soll, wenn Ihr Spiel zum Testen oder für die endgültige Veröffentlichung erstellt wird. Stellen Sie sicher, dass Sie Sie dies (und Ihre Berechtigungen) sorgfältig, bevor Sie Spiele an den Store übermitteln, da falsche Einstellungen dazu führen, dass Ihr Spiel nicht übermittelt werden kann.

Sobald Sie dies korrekt eingerichtet haben, müssen Sie die erforderlichen Dateien hinzufügen, damit Ihr Erweiterungspaket funktioniert. Klicken Sie dazu auf die Schaltflächen am unteren Rand, entweder Add SDK oder Add Source, und suchen Sie dann die Dateien, die Sie die Sie hinzufügen möchten. Die hinzugefügten Dateien werden zusammen mit Ihrer Erweiterung im Verzeichnis AndroidSource gespeichert. Sie können diesen Speicherort jederzeit öffnen, indem Sie mit der rechten Maustaste auf auf die Erweiterung klicken und Erweiterungsverzeichnis öffnen auswählen.

Weitere Informationen zur Erstellung von Android-Erweiterungen finden Sie in der YoYo Games Knowledge Base.
