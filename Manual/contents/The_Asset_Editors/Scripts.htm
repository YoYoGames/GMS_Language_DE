Der Skript-Editor

Dieser Abschnitt befasst sich mit dem Skript-Editor (auch Text-Editor genannt), in den Sie den gesamten Code eingeben, den Ihr Spiel benötigt, um zu laufen. Dieser Editor wird zum Erstellen von benutzerdefinierten Skript-Assets sowie zum Codieren von Objektereignissen, zum Hinzufügen von Code für die Raumerstellung und für viele andere Dinge verwendet, und für viele andere Dinge. Daher wird in diesem Abschnitt nur beschrieben, wie der Skript-Editor funktioniert, nicht aber, wie Sie Skript-Assets erstellen oder wie Sie Skripts per Drag and Drop™erstellen. Dazu lesen Sie bitte die folgenden Abschnitte:

  • GML-Skripte
  • DnD™Skripte

Im Script-Editor schreiben Sie Ihren GameMaker Language-Code, die eingebaute Programmiersprache, die GameMaker Studio 2 verwendet (weitere Einzelheiten finden Sie im Abschnitt GML-Referenz ). Sobald Sie mit GameMaker Studio 2 besser vertraut sind und es in vollem Umfang nutzen möchten, ist es ratsam, mit dem Erlernen dieser Sprache zu beginnen, da sie Ihre Möglichkeiten beim Erstellen von Spielen erheblich erweitert.

Ein Skript-Asset kann benannt werden, indem Sie im Asset-Browser mit der rechten Maustaste auf klicken und Umbenennen wählen, Beachten Sie jedoch, dass der Skriptname den Skriptregeln für Funktionen entsprechen muss, d. h. er darf nur alphanumerisch sein, muss mit einem Buchstaben beginnen und darf als einziges Symbol das "_"-Unterstrich-Symbol enthalten. Der Skript-Editor an anderen Stellen kann nicht umbenannt werden und hat einen Namen, der für das zu bearbeitende Objekt spezifisch ist, wie z. B. ein Objekt-Ereignis.

Registerkarte Skript / Ereignis

Der Skript-Editor wird in einem Fenster mit Registerkarten am oberen Rand geöffnet, damit Sie mehrere Skripte oder Ereignisse in einem Fenster haben können (obwohl dieses Verhalten in den Voreinstellungen geändert werden kann, um jedem Skript oder Ereignis ein neues Fenster zu geben). Sie können auf auf eine Registerkarte klicken und sie dann ziehen, um die Reihenfolge der Registerkarten zu ändern, oder - wenn Sie es vorziehen - können Sie aus dem aktuellen Fenster herausziehen und auf der Arbeitsfläche platzieren, um ein neues Fenster für dieses Skript zu erstellen (oder es zu einem anderen Fenster hinzuzufügen), und Sie können auch den Skript-Editor maximieren, um ebenfalls einen neuen Arbeitsbereich zu erstellen. Wenn Sie eine Skript-Registerkarte aus dem IDE-Fenster ziehen, wird eine neue IDE erzeugt, die diese Skript-Ressource enthält und die Sie wie das Hauptfenster verwenden können.

Beachten Sie, dass, wenn Sie Code aus einem Objektereignis im Skript-Editor bearbeiten und Sie den Skript-Editor maximiert haben oder ihn in einem separaten Fenster oder Arbeitsbereich haben, das Rechtsklick-Menü Editor einige zusätzliche Optionen enthält:

  • In Arbeitsbereich wiederherstellen: Dies nimmt den Skript-Editor aus einem maximierten/Arbeitsbereich-Zustand heraus und kettet ihn wieder an das Objekt im Arbeitsbereich.
  • Gehen Sie zu Objekt: Damit wechseln Sie zu dem Arbeitsbereich, auf dem sich das Objekt mit dem Code befindet, und fokussieren auf das Objekt.
  • Ereignis hinzufügen/öffnen: Damit können Sie dem Objekt, zu dem das aktuelle Skript gehört, ein neues Ereignis hinzufügen und eine neue Code-Registerkarte im Skript-Editor für das hinzugefügte Ereignis öffnen. Wenn das ausgewählte Ereignis bereits Code enthält, wird dieser wird in einer neuen Registerkarte geöffnet.

Informationen zu den übrigen Optionen des Rechtsklick-Menüs finden Sie im Abschnitt " Bearbeiten" weiter unten.

Haupt-Code-Editor

Der Hauptcode-Editor ist der Ort, an dem Sie Ihren gesamten Code schreiben, um das Skript-Asset zu erstellen oder ein Objekt-Event zu füllen. Die eigentliche Bearbeitung Ihres Codes wird im Abschnitt " Bearbeiten" weiter unten auf dieser Seite behandelt.

Informationen

Die Informationsleiste am unteren Rand der IDE zeigt Ihnen die aktuelle Zeilennummer, in der Sie sich befinden, und die Position in der Zeile an. Hier sehen Sie auch die Codehilfe, eine Textzeile, die die Funktion, die Sie gerade bearbeiten, zusammen mit mit den Argumenten, die sie benötigt. Wenn Sie die Funktion im Editor ausfüllen, werden die Argumente hervorgehoben, um Ihnen zu zeigen, welches Argument Sie gerade bearbeiten.

Beachten Sie, dass alle optionalen Argumente in Ihren Skriptfunktionen mit [] -Klammern um sie herum angezeigt werden. Wenn Sie die JSDoc-Skriptkommentare innerhalb eines Skript-Assets verwendet haben, werden die Informationen, die Sie bereitgestellt haben, auch hier angezeigt.
