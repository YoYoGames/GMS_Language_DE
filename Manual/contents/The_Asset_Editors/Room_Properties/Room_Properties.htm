Raum-Eigenschaften

Im Abschnitt "Raumeigenschaften" können Sie die Vererbung der Einstellungen umschalten, den Raum benennen und festlegen, ob er persistent ist oder nicht. Wenn Sie einen Raum verlassen und später in denselben Raum zurückkehren, wird dieser Raum normalerweise auf seine ursprünglichen Einstellungen zurückgesetzt. Das ist für die meisten Spiele in Ordnung, aber es ist vielleicht nicht das, was Sie wollen, z. B. in einem RPG oder einem RPG oder einem nicht-linearen Spiel, in dem Sie zwischen Räumen hin- und herwechseln und sie so vorfinden möchten, wie Sie sie beim letzten Mal verlassen haben. Wenn Sie das Kontrollkästchen " Persistent" aktivieren, wird genau das erreicht. Der Zustand des Raums wird gespeichert und wenn Sie später in den Raum Wenn Sie später in den Raum zurückkehren, ist er genau so, wie Sie ihn verlassen haben, und wird nur beim Neustart des Spiels auf den Anfangszustand zurückgesetzt. Beachten Sie, dass es hier eine Ausnahme gibt - wenn Sie bestimmte Objekte als persistent markiert haben, bleiben Instanzen dieser Objekts nicht in dem Raum bleiben, sondern in den nächsten Raum wechseln.

Danach haben Sie die Möglichkeit, den Anzeigepuffer zu löschen. Wenn diese Option aktiviert ist, füllt sich der Anzeigepuffer mit einer Farbe. Wenn Sie wissen, dass die Ansichten Wenn Sie wissen, dass die Ansichten immer den gesamten Bildschirm bedecken oder dass ein bildschirmfüllender Hintergrund gezeichnet wird, können Sie diese Option deaktivieren (was ein erneutes Zeichnen erspart und somit zur Optimierung Ihres Spiels beiträgt), aber wenn Sie überlappende Ansichten haben, die, wenn sie auf dem Bildschirm sind, offene Bereiche lassen, Wenn Sie jedoch überlappende Ansichten haben, die auf dem Bildschirm offene Bereiche hinterlassen, oder wenn Ihr Hintergrund transparent ist, sollten Sie diese Option aktivieren, damit die Bereiche, in denen sich keine Ansicht befindet, mit der von Ihnen gewählten Farbe gefüllt werden. Zurzeit kann diese Farbe nur über Code eingestellt werden, indem Sie window_set_colour().

HINWEIS: Wenn Sie die automatische Seitenverhältniskorrektur verwenden (wie in den Spieloptionen eingestellt), dann sollten Sie dies immer aktiviert haben, da Sie sonst seltsame Effekte über die "Letterbox", in der Ihr Spiel gezeichnet wird. Wenn Sie diese Option nicht verwenden, können Sie das Häkchen entfernen und so einen kleinen Leistungsschub für Ihr Spiel erhalten (besonders spürbar auf Android und anderen mobilen Plattformen).

Ein Raum muss eine Größe haben, und diese wird durch die Werte definiert, die Sie für seine Breite und Höhe in Pixeln eingeben, und sobald das festgelegt wurde, können Sie dann fortfahren, Erstellungscode hinzuzufügen, falls Sie ihn benötigen. In diesem Editor können Sie Funktionen/Aktionen und Code eingeben, die beim Start des Raums ausgeführt werden, und zwar nach dem Erstellungsereignis aller Instanzen, aber vor ihrem Raumstart-Ereignis (weitere Informationen zur Ereignisreihenfolge finden Sie hier). Kameras und Sichtfenster

Die nächste Gruppe von Eigenschaften für den Raum, die Sie definieren können, sind die, die sich auf die Raumkamera und die Ansichtsfenster beziehen. Kameraansichten bieten einen Mechanismus zum Zeichnen verschiedener Teile Ihres Raums an verschiedenen Stellen auf dem Bildschirm, oder um nur einen Teil des Raums zu zeichnen, der den gesamten Bildschirm abdeckt. In den meisten Jump'n'Run-Spielen folgt die Kameraansicht beispielsweise der Hauptfigur, denn wenn man den gesamten Level auf dem Bildschirm sehen könnte, wäre die Figur zu klein, um sie zu sehen und es gäbe würde es keine Überraschungen für den Spieler geben. Kameraansichten können auch in Multiplayer- oder Koop-Spielen verwendet werden, da sie es ermöglichen, einen geteilten Bildschirm zu erstellen, in dem Sie in einem Teil des Bildschirms einen Spieler und in einem anderen Teil den anderen Spieler sehen Spieler sehen. Dies alles lässt sich in GameMaker Studio 2 mit Hilfe von Kameraansichten leicht realisieren.

Oben in den Eigenschaften der Ansichten können Sie die Ansichtsvererbung ein- oder ausschalten, dann gibt es ein Kästchen mit der Aufschrift " Ansichtsfenster aktivieren". Dieses Kästchen muss markiert sein, bevor eine der Kameraansichten in Ihrem Spiel aktiv sein kann.

HINWEIS: Sie können die Vererbung für die drei Hauptansichtsport-Einstellungen ein- oder ausschalten und dann die Vererbung für jede einzelne Kameraansicht ein- oder ausschalten.

Das nächste Kontrollkästchen heißt "Clear Viewport Background" (Hintergrund des Ansichtsfensters löschen) und kann deaktiviert werden, wenn Sie wissen, dass Sie die gesamte Oberfläche mit der Fensterfarbe Sie zeichnen, um die gesamte Oberfläche abzudecken.

Eine Kameraansicht wird durch zwei verschiedene Sätze von Werten definiert, die Kameraansicht selbst und den Anschluss auf dem Bildschirm, an dem diese Ansicht gezeichnet werden soll. Dies kann manchmal zu Verwirrung führen, also lassen Sie uns dies ein wenig erklären, bevor wir erläutern, wie wir die beiden definieren:

  • Die Kamera: Ein Punkt innerhalb des Raums, mit dem festgelegt wird, wie der Raum auf dem Bildschirm angezeigt wird (dies ist ein abstrakter Punkt im Raumeditor und seine Position wird automatisch auf der Grundlage der Einstellungen für die Ansicht und den Ansichtsanschluss festgelegt)
  • Die Ansicht: Was die Kamera sieht, basierend auf der Position, Projektion und Drehung der Kamera
  • Der Ansichtsanschluss: Der Bereich des physischen Bildschirms, in dem die Kameraansicht angezeigt wird

Das bedeutet zum Beispiel, dass Sie eine 640x480-Kameraansicht in Ihrem Raum haben können und dann den Port auf 320x240 einstellen, wodurch die Ansicht Sie können aber auch die Ansicht auf einen kleineren Wert und den Port auf einen größeren Wert einstellen, so dass das Bild auf die Größe des Ports skaliert und auf dem Bildschirm größer angezeigt wird als es ist. Auf diese Weise können Sie eine Bildschirmgröße (Port) beibehalten, während Sie die Kameraansicht ändern, um mehr oder weniger des Raums im gleichen Bereich des Bildschirms anzuzeigen.

Die Kameraansicht wird immer als rechteckiger Bereich im Raum definiert, wobei Sie die Position der linken oberen Ecke, die Breite und die Höhe dieses Bereichs festlegen. Dann müssen Sie festlegen, wo dieser Bereich im Fenster auf dem Bildschirm angezeigt wird, indem Sie Hier geben Sie wiederum die Position der linken oberen Ecke und die Größe an (beachten Sie, dass alles andere als 0,0 für die linke obere Ecke zu seltsamen Ergebnissen führen kann). Sie können mehr als einen Anschluss haben und sie können sich überlappen. werden sie in der angegebenen Reihenfolge übereinander gezeichnet.

Bitte beachten Sie, dass der gesamte Bildschirmbereich immer als rechteckiger Bereich definiert ist, so dass Ihre Ports, auch wenn sie versetzt sind, ein Rechteck bilden, wobei alle leeren Bereiche durch die Fensterfarbe des Anzeigepuffers ausgefüllt werden, was bedeutet, dass Sie unter diesen Umständen immer die Option " Clear Display Buffer" (Anzeigepuffer löschen) aktiviert haben sollten, da Sie sonst seltsame Artefakte in den Bereichen zwischen den Ports erhalten. Kameras verfügen auch über die Option Objektverfolgung . Diese Option ist dafür gedacht, dass die Kamera einem bestimmten Objekt "folgt" (d. h.: die (d. h. die Ansicht auf ein bestimmtes Objekt fokussieren). Klicken Sie dazu auf auf das Menüsymbol und wählen Sie ein Objekt aus der Liste, die sich öffnet (wenn es mehrere Instanzen dieses Objekts im Raum gibt, wird nur eine von der Kamera verfolgt). Wenn es mehrere Instanzen dieses Objekts im Raum gibt, wird nur eine von ihnen von der Kamera verfolgt).

Das normale Verhalten einer Kamera besteht darin, sich nur zu bewegen, wenn die verfolgte Instanz zu nahe an eine "Pufferzone" kommt, die eine unsichtbare Grenze um den Rand der Ansicht bildet. Diese Zone können Sie mit den Werten Horizontal Border und Wenn Sie diese Werte beispielsweise auf 64 einstellen, bedeutet dies, dass die Ansicht erst dann beginnt, sich zu bewegen und der Figur zu folgen, wenn sie 64 Pixel vom Rand der Ansicht entfernt ist.

Schließlich können Sie die horizontale und vertikale Geschwindigkeit angeben, mit der sich die Kamera bewegt, wenn der Charakter die Pufferzone erreicht hat, und dies hat einen Standardwert von -1. Dieser Standardwert ist im Grunde "augenblicklich" und bedeutet, dass in dem Moment, in dem sich die Verfolgungsinstanz außerhalb der Pufferzone für den horizontalen oder vertikalen Rand befindet, die Ansicht zur aktuellen Position springt. Dies ist nicht immer das, was Sie wollen, und deshalb können Sie die vertikale und horizontale Scrollgeschwindigkeit Geschwindigkeit für die Kamera einstellen, indem Sie die Werte auf etwas anderes als -1 setzen. Beachten Sie, dass ein Wert von 0 dazu führt, dass sich die Ansicht überhaupt nicht bewegt, und dass jeder andere positive Wert angibt, um wie viele Pixel sie sich in jedem Bild bewegt, so dass die Einstellung der horizontalen Geschwindigkeit auf 5 dazu führt, dass Wenn Sie also die horizontale Geschwindigkeit auf 5 einstellen, folgt die Ansicht dem Objekt mit 5 Pixeln pro Bild in horizontaler Richtung.

Raum-Physik

Bevor Sie die eingebauten Physikfunktionen in Ihren Spielen verwenden können, müssen Sie GameMaker Studio 2 mitteilen, dass der Raum ein Physikraum ist. Dazu müssen Sie die Option " Physik aktivieren " oben im Abschnitt " Raumphysik " ankreuzen (Sie können die Vererbung dieses Abschnitts auch unabhängig von den übrigen Raumeinstellungen umschalten). Wenn Sie das getan haben, können Sie mit dem Einrichten der Physikwelt-Eigenschaften fortfahren, d. h. mit einigen grundlegenden Eigenschaften, die Sie vordefiniert haben müssen bevor Ihr Raum Physikinstanzen als solche arbeiten lässt. Für eine genauere Kontrolle über die Welt können Sie Code verwenden (siehe die Physikfunktionen für weitere Informationen).

Als nächstes müssen Sie die Schwerkraft der Welt einstellen. Deren Stärke und Richtung wird berechnet als Vektor der x/y-Position, die Sie um einen (0, 0)-Punkt setzen. Bei einem x-Wert von 0 und einem y-Wert von 1 wird die Schwerkraft also mit einer Kraft von 1 Meter pro Sekunde nach unten gerichtet sein (für eine genauere Erklärung siehe eine detailliertere Erklärung finden Sie unter - Die Welt der Physik).

Schließlich müssen Sie das Verhältnis von Pixeln zu Metern einstellen, das GameMaker Studio 2 als Basis für alle seine Physikberechnungen verwenden soll. Das liegt daran, dass die Physikfunktionen mit realen Maßen arbeiten, weshalb wir diesen Wert einstellen müssen, und Sie werden diese Einstellung so lange anpassen wollen, bis die durchschnittliche Pixelgröße der von Ihnen verwendeten Objekte in etwa in simulierte Physikobjekte einer angemessenen Größe übersetzt wird.

Es ist zu beachten, dass das Aktivieren der Physik in einem Raum bedeutet, dass alle Instanzen im Raum die Physik-Funktionen und -Variablen verwenden müssen, um sich zu bewegen. Grundsätzlich gibt es die "traditionelle" Bewegung, bei der Sie die X/Y-Position einer Instanz festlegen können oder ihre Geschwindigkeit und Richtung einstellen können, und dann haben Sie die "Physik"-Bewegung, die physikalische Kräfte und Impulse benötigt, um sich zu bewegen. Diese Systeme schließen sich gegenseitig aus, und Sie können eine Physik-Instanz nicht mit Nicht-Physik Funktionen bewegen und Sie können eine Nicht-Physik-Instanz nicht mit den Physik-Funktionen bewegen.
