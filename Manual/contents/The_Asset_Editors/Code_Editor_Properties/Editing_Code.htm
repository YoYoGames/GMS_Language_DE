Code editieren

Wenn Sie den Skript-Editor verwenden, geben Sie einfach den gewünschten Code für das Skript-Asset oder für das Ereignis ein und folgen dabei der korrekten GML-Syntax . Während Sie dies tun erhalten Sie Hilfe durch das Autovervollständigungsfenster:

Dieses Fenster erscheint, während Sie Ihren Code schreiben (normalerweise nach einer kleinen Pause, aber das kann in den Voreinstellungen eingestellt werden) und gibt Ihnen eine Auswahl von "Best-Fit"-Antworten in alphabetischer Reihenfolge.

Während Sie den Code-Editor verwenden, können Sie auf klicken und die Maus ziehen, um eine Zeile des Codes auszuwählen, oder Sie halten gedrückt und klicken dann auf und ziehen, um Spalten des Codes auszuwählen. Alles, was auf diese Weise ausgewählt wird, kann dann kopiert, ausgeschnitten und eingefügt werden , indem Sie die Standard-Tastaturkürzel / + " C", / + "X" und / + "V".

Es ist erwähnenswert, dass Sie Ihren Code auf verschiedene Arten kommentieren können. Das Kommentieren Ihres Codes ist sehr wichtig für die Aufrechterhaltung der Produktivität, da es die Dinge klar hält und Codeabschnitte so trennt, dass es offensichtlich ist, was jeder Teil tut und warum er da ist. Es bedeutet auch, dass Sie, wenn Sie in einem Team arbeiten, Notizen für die anderen Mitglieder hinterlassen können und dass diese genau wissen, was Sie getan haben oder wofür ein bestimmter Abschnitt des Codes da ist. Auch wenn Sie ein Solo-Entwickler sind, ist es wichtig, Kommentare in Ihrem Code zu hinterlassen, besonders wenn Sie das Projekt verlassen und zu einem späteren Zeitpunkt darauf zurückkommen müssen.

Das Auskommentieren Ihres Codes ist einfach, denn Sie müssen nur "//" vor jeder Textzeile verwenden oder Sie können ganze Absätze auskommentieren, indem Sie "/* text here */" verwenden. Es gibt sogar zwei Tastatur Tastenkombinationen, um ausgewählte Codezeilen zu kommentieren bzw. unkommentiert zu lassen: / + "K" zum Kommentieren und / + + "K" zum Dekommentieren. In Skripten können auch bestimmte JSDoc-ähnliche Kommentare hinzugefügt werden, um die Eigenschaften der darin enthaltenen Funktionen zu definieren, so dass sie bei der Verwendung von die Funktionen in Ihrem Code verwenden, werden sie in der Autovervollständigung zusammen mit ihren Argumenten und anderen Details angezeigt. Weitere Informationen hierzu finden Sie im Abschnitt JSDoc-Skriptkommentare für weitere Informationen.

Standardmäßig verfügt jeder geschriebene Code über Codefaltungssymbole neben jedem Satz passender geschweifter Klammern {}. Diese haben die Form von kleinen Plus- + oder Minus-Symbolen -, und wenn Sie darauf klicken, wird der Code innerhalb der passenden Klammern "gefaltet", so dass er weniger visuellen Platz einnimmt.

Sie können auch manuell Code-Faltbereiche setzen, indem Sie die speziellen #region -Tags verwenden, um einen Codeabschnitt zu markieren. Dies funktioniert so, dass Sie ein #region -Tag (zusammen mit beliebigem Kommentartext) an den Anfang eines eines Codeabschnitts ein #endregion -Tag (zusammen mit Kommentartext) am Anfang des Abschnitts einfügen und dann ein entsprechendes -Tag am Ende des Abschnitts hinzufügen. zu einem bestimmten Zeitpunkt:

Der aktuelle Code sieht in etwa so aus:

#region Dies ist eine Beispielregion

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Hier ggf. weiterer Kommentar

Wie Sie vielleicht schon bemerkt haben, sind Teile des Skripttextes farbig gekennzeichnet. Der Skript-Editor "weiß" über vorhandene Objekte, eingebaute Variablen und Funktionen etc... Bescheid und kodiert sie entsprechend farblich. Die Farbcodierung hilft sehr bei der Vermeidung von Fehlern da Sie sofort sehen können, ob Sie einen Namen falsch geschrieben oder ein Schlüsselwort als Variable verwendet haben oder ob ein Syntaxfehler vorliegt. Wenn Ihnen die Farbcodierung jedoch nicht gefällt, können Sie sie in den Voreinstellungen ein- und ausschalten (Sie können auch F10 verwenden, um sie im Editor ein- oder auszuschalten) sowie die verwendeten Farben anpassen.

Sie können auch mit der rechten Maustaste auf an beliebiger Stelle im Editor klicken, um das folgende Menü zu öffnen:

Neben den üblichen Funktionen zum Ausschneiden, Kopieren und Einfügen von Codezeilen haben Sie auch die Möglichkeit, markierten Text zu kommentieren oder unkommentiert zu lassen. Sie haben auch die Möglichkeit, markierten Text zu kommentieren oder unkommentiert zu lassen, Einzüge hinzuzufügen oder zu entfernen sowie das Codefenster in 1, 2 oder 3 separate Bereiche aufzuteilen (beachten Sie, dass die Optionen in Bezug auf #regions nicht verfügbar sind, wenn Sie im bearbeiteten Code keine Einzüge definiert haben).

Die letzte Option für die Aufteilung in Bereiche teilt das eigentliche Fenster in verschiedene Teile auf, in die Sie dann ein Skript oder Event ziehen können, um es neben den anderen zum Vergleichen und Bearbeiten anzuzeigen.

Dies ist besonders nützlich, um zwei Skripte zu vergleichen oder um verbundene Skripte gleichzeitig zu betrachten, ohne ein weiteres Fenster zu benötigen oder ständig zwischen Registerkarten wechseln zu müssen.

Die anderen Optionen in diesem Menü sind das Umschalten eines Haltepunkts für die Verwendung mit dem Debugger und die Konvertierung des GML-Codes in DnD™oder umgekehrt. Beachten Sie, dass bei der Konvertierung von Code in DnD™ keine einzelne Aktionen für jede Codezeile erstellt werden, sondern eine einzige Codeaktion verwendet wird, die alles enthält, was zuvor im Ereignis oder Skript enthalten war.
