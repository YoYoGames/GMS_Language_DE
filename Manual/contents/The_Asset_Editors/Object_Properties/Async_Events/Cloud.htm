Wolke

Das Cloud-Ereignis ist ein Ereignis, das durch den Rückruf von einer der cloud_*() Funktionen, wie cloud_synchronise(). Es erzeugt tatsächlich eine DS-Map, die exklusiv für dieses Ereignis ist und in der speziellen Variable async_load gespeichert wird (siehe die einzelnen Funktionen für Code-Beispiele, die die Verwendung dieses Ereignisses näher erläutern). Diese DS-Map hat den folgenden Aufbau:

  • "status": Hier steht der Statuscode, wobei ein negativer Wert einen Fehler bedeutet, dessen Beschreibung im "errorString" enthalten ist. Ein Wert von 0 (oder ein positiver Wert) zeigt einen Erfolg an (siehe unten für genaue Werte siehe unten), und der "resultString" enthält die zurückgegebenen Daten oder eine Erfolgsmeldung.
  • "id": Die ID, die von der aufgerufenen Funktion zurückgegeben wurde. Wenn Sie eine Reihe von "cloud_"-Anfragen abfeuern, müssen Sie wissen, auf welche Sie eine Antwort erhalten, und daher würden Sie diesen Wert zum Vergleich mit dem Wert, den Sie beim ursprünglichen Senden der Anfrage gespeichert haben, um die richtige Anfrage zu finden.
  • "description": Die Beschreibung der zuletzt hochgeladenen Datei.
  • "resultString": Dies enthält eine Zeichenkette, die der von der Cloud zurückgegebene Datenblob ist.
  • "errorString": gibt eine Fehlerzeichenfolge für einen beliebigen Fehler zurück.

Die genaue Bedeutung des zurückgegebenen Karteneintrags "Status" wird in der folgenden Tabelle erläutert:

Status Wert errorString / resultString Beschreibung
-1 errorString = "Nicht eingeloggt bei <SERVICE>" Sie haben sich nicht erfolgreich bei dem angegebenen Cloud-Dienst angemeldet
0 resultString = wiederhergestellte Daten Neue Spieldaten werden aus der Cloud heruntergeladen (nach einem Aufruf von cloud_synchronise() )
1 resultString = "BereitsSynchronisiert" Keine neuen Daten seit dem letzten Aufruf von cloud_synchronise
2 resultString = "ConflictDeferral" Ein Konflikt ist aufgetreten, aber der Spieler hat sich entschieden, ihn zu ignorieren
3 resultString = "GameUploadSuccess" Daten von cloud_string_save() oder cloud_file_save() wurden erfolgreich in die Cloud hochgeladen
-n errorString = Beschreibung des Fehlers Jede andere negative Zahl bedeutet einen Synchronisationsfehler
