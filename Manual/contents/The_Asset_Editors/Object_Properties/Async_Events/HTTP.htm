HTTP

Das HTTP-Ereignis ist eines, das durch den Rückruf von einer der Funktionenhttp_*() ausgelöst wird, wie http_post_string(). Es erzeugt tatsächlich eine DS-Map, die exklusiv für dieses Ereignis ist und in der speziellen Variable async_load gespeichert wird (siehe die einzelnen Funktionen für Code-Beispiele, die die Verwendung dieses Ereignisses näher erläutern). Diese DS-Map hat den folgenden allgemeinen Aufbau:

  • "id": Die ID, die vom Befehl zurückgegeben wurde. Wenn Sie eine Reihe von http_ -Anfragen abfeuern, müssen Sie wissen, auf welche Sie eine Antwort erhalten, und deshalb würden Sie diesen Wert mit dem Wert vergleichen, den Sie beim ursprünglichen Senden der Anfrage gespeichert haben, um die richtige Anfrage zu finden.
  • "status": Gibt einen Wert kleiner 0 für einen Fehler, 0 für Erfolg und 1, wenn der Inhalt heruntergeladen wird, zurück.
  • "result": Die empfangenen Daten (nur String), oder der Pfad zur heruntergeladenen Datei, wenn Sie http_get_file().
  • "url": Die vollständige URL, die Sie angefordert haben.
  • "http_status": Der rohe http-Statuscode (falls verfügbar). Dies gibt den Standard-Web-Statuscode für die meisten Browser zurück, z. B.: 304 für "Not Modified" oder 204 für "No Content", usw...

Die obige Abbildung zeigt, was Sie erhalten, wenn Sie die http_post_string() Funktion, aber jede der http_ Funktionen kann eine leicht abweichende Abbildung zurückgeben, daher lesen Sie bitte den Handbucheintrag für jede Funktion, um die genauen Daten herauszufinden, die für sie zurückgegeben werden.

HINWEIS: Da async_load eine DS-Map erzeugt, sind diese Funktionen besonders nützlich in Verbindung mit den Funktionen json_encode() und json_decode() Funktionen.

Es können auch zusätzliche Daten von dieser Map geliefert werden, wenn Sie Dateien zum Herunterladen angefordert haben. In diesem Fall wird der "Status" den Wert 1 haben und die DS-Map wird diese zusätzlichen Schlüssel enthalten:

  • "contentLength": Dies ist die Größe der Datei, die Sie nach Angaben des Webservers erwarten sollten (kann -1 sein, wenn der Server diese Daten nicht zurückgibt).
  • "sizeDownloaded": Die Größe der Daten, die bereits heruntergeladen wurden.

Beachten Sie, dass das Ereignis nicht für jedes einzelne Datenpaket, das empfangen wird, ausgelöst wird, sondern zu jedem Zeitpunkt während des Downloads innerhalb der Hauptspielschleife aktualisiert wird. Beachten Sie auch, dass diese Funktionalität derzeit nur für reguläre Windows Zielplattformen verfügbar ist.
