Bild geladen

Dieses Ereignis wird ausgelöst, wenn Sie ein Bild in GameMaker Studio 2 laden, entweder über HTML5 oder über einen URL. Nehmen wir an, Sie möchten ein Sprite-Bild laden und das aktuelle Sprite für die Instanz erst dann in das neue ändern, wenn es geladen wurde. Im Create-Ereignis (oder jedem anderen Ereignis) würden Sie so etwas haben:

var _url = "http://www.angusgames.com/game/background1.png";
spr = sprite_add(_url, 0, false, false, 0, 0);

Dieser beginnt nun, das Bild in das Gerät oder den Browser zu laden, blockiert aber nicht GameMaker Studio 2, während es darauf wartet, dass die Datei geladen wird. Stattdessen läuft GameMaker Studio 2 normal weiter, bis das Bild geladen ist und der Callback das Ereignis "Bild geladen" auslöst, bei dem eine DS-Map erstellt und in der speziellen Variablen async_load. Die Map enthält die folgenden Informationen:

  • "filename": Der vollständige Pfad zu der angeforderten Datei.
  • "id": Die ID der Ressource, die Sie geladen haben. Dies ist dieselbe wie die Variable, der Sie die Ressource zugewiesen haben.
  • "status": Gibt bei einem Fehler einen Wert kleiner als 0 zurück.

In diesem Ereignis würden Sie dann das neu geladene Bild einem Sprite zuweisen. Das folgende Codebeispiel zeigt, wie die zurückgegebenen Informationen verwendet werden würden:

if ds_map_find_value(async_load, "id") == spr
{
if ds_map_find_value(async_load, "status") >= 0
{
sprite_index = spr;
}
}

Der obige Code prüft zuerst die id der erstellten DS-Map und dann die status des Callbacks. Wenn der Wert größer oder gleich 0 ist (was Erfolg signalisiert), wird das Ergebnis von des Callbacks dann dazu verwendet, den Sprite-Index auf das neu geladene Bild zu setzen.
