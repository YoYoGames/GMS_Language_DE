Sozial

Dieses Ereignis kann nur durch die Callbacksvon verschiedenen spezifischen Social Functions und UWP Xbox Live Functions ausgelöst werden und gibt eine DS Map zurück, die in der Variablen async_load, gespeichert ist, die je nach Callback der Funktion, die das Ereignis ausgelöst hat, unterschiedliche Schlüssel/Wertpaare enthält.

Die DS-Map enthält eine Reihe von Schlüsseln, von denen der wichtigste der Schlüssel "id" ist. Dieser gibt eine Konstante zurück, die dann im Code überprüft werden kann, um festzustellen, welchen der vielen Rückrufe das Ereignis erhalten hat. Nachdem Sie diesen Schlüssel geparst und den zurückgegebenen Wert mit den verfügbaren Konstanten verglichen haben, können Sie damit fortfahren, den Rest der Informationen aus der Map zu extrahieren.

Die Funktionsseiten zu Errungenschaften und Bestenlisten im Abschnitt "Referenz" beschreiben die Funktionen und Callbacks sowie die Konstanten, die zum Auslösen dieses Ereignisses verwendet werden, aber es gibt auch bestimmte Umstände, unter denen es ohne einen Funktionsaufruf des laufenden Spiels ausgelöst wird:

  • wenn Ihr Spiel über das OS-Dashboard gestartet wird
  • wenn Sie eine Herausforderung abgeschlossen haben
  • wenn ein anderer Spieler eine Herausforderung abschließt
  • wenn das Spiel über das OS-Dashboard durch Auswahl einer Herausforderung gestartet wird

Jeder der oben genannten Rückrufe löst das soziale Ereignis aus, und es wird eine async_load -Map mit den folgenden Details erzeugt (beachten Sie, dass es einen unterschiedlichen "id" Schlüsselwert gibt, um jeden der verschiedenen Gründe, aus denen das Ereignis aufgerufen wurde, aber alle anderen Inhalte der Karte sind gleich):

  • "id" - Der Wert dieses Schlüssels hängt vom Typ des Rückrufs ab, der das Ereignis auslöst. er kann eine dieser Konstanten sein:
Konstant Beschreibung
achievement_challenge_received Es ist eine Herausforderung eingegangen
achievement_challenge_completed Eine Herausforderung wurde abgeschlossen.
achievement_challenge_completed_by_remote Eine Herausforderung wurde vom anderen Spieler abgeschlossen.
achievement_challenge_launched Das Spiel wurde über das OS-Dashboard mit der vorgegebenen Herausforderung gestartet.

  • "playerid" - Die Spieler-ID für die Herausforderung.
  • "issuerid" - Die ID der Person, die die Herausforderung ausgestellt hat.
  • "state" - Der Status der Challenge, der einen Wert von 0 - 3 (als String) für ungültig, ausstehend, abgeschlossen oder abgelehnt hat.
  • "message" - Die Textnachricht für die Herausforderung.
  • "issueddate" - Das Ausgabedatum für die Challenge
  • "completeddate" - Das Abschlussdatum für die Challenge.
  • "type" - Der Typ der gegebenen Herausforderung. Kann eine von zwei Konstanten sein:
Konstant Beschreibung
achievement_type_score_challenge Eine Herausforderung auf der Grundlage des Punktwerts.
achievement_type_achievement_challenge Eine Herausforderung, die auf einer Leistung basiert.

  • "identifier" - Die identifizierende Zeichenfolge für die Challenge.
  • "score" - Der Spielstand ist mit der Herausforderung verbunden.

Nachfolgend finden Sie einen kleinen Codeschnipsel als Beispiel für die Verwendung:

var ident = ds_map_find_value(async_load, "id" );
if ident == achievement_challenge_completed;
{
player_id = ds_map_find_value(async_load, "playerid");
issuer_id = ds_map_find_value(async_load, "issuerid");
state = ds_map_find_value(async_load, "state");
date_completed = ds_map_find_value(async_load, "completeddate");
date_issued = ds_map_find_value(async_load, "issueddate");
ach_type = ds_map_find_value(async_load, "type");
ach_ident = ds_map_find_value(async_load, "identifier");
ach_score = ds_map_find_value(async_load, "score");
}
