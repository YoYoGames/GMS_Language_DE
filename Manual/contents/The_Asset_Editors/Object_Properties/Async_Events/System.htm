System

Dieses Ereignis kann nur durch einen Rückruf von einem Ereignis auf Systemebene ausgelöst werden (z. B. die Erkennung eines Gamepads oder die automatische Anmeldung bei XBox Live) ausgelöst werden und gibt eine DS-Map zurück, die in der Variablen async_load, gespeichert ist, die verschiedene Schlüssel/Wertpaare enthält, abhängig von dem Ereignis auf Systemebene, das den Rückruf ausgelöst hat. Diese Ereignisse sind:

Spiel-Pads

Wenn dieses Ereignis für ein Gamepad ausgelöst wird, das angeschlossen oder getrennt wird, gibt es eines der folgenden Schlüssel/Wertpaare in der Karte async_load zurück:

  • "event_type" - der Typ des empfangenen Systemereignisses, der eine der folgenden Zeichenketten sein wird:
    • "gamepad discovered" - tritt auf, wenn das System meldet, dass ein neues Gamepad angeschlossen wurde.
    • "gamepad lost" - geschieht, wenn das System die Verbindung zu einem Gamepad verloren hat
  • "pad_index" - der Index des Pads, das hinzugefügt oder entfernt wurde

Mit diesem Ereignis können Sie nun Ihre gesamte Gamepad-Prüflogik aus dem Schritt-Ereignis oder einem Alarm-Ereignis in das System-Ereignis verschieben und sie nur dann ausführen, wenn sie tatsächlich benötigt wird.

WICHTIG! Dieses Ereignis wird NICHT ausgelöst, es sei denn, Sie haben mindestens eine gamepad_* Funktion oder DnD™-Aktion in Ihrem Spielcode. Der Runner initialisiert das Gamepad-Subsystem nur, wenn die Funktionen im Projekt verwendet werden. Wenn sie also nicht vorhanden sind, wird das Hinzufügen/Entfernen von Gamepads das Systemereignis nicht auslösen.

Virtuelle Tastaturen

Wenn dieses Ereignis für eine virtuelle Tastatur ausgelöst wird, die geöffnet oder geschlossen wird, werden die folgenden Schlüssel/Wertpaare in der Map async_load zurückgegeben:

  • "event_type" - der Typ des empfangenen Systemereignisses, der für virtuelle Tastaturen "virtual keyboard status" sein wird.
  • "screen_height" - die Höhe der virtuellen Tastatur (in Pixel). Dieser Wert ist 0, wenn die Tastatur unsichtbar ist.
  • "keyboard_status" - der aktuelle Status der Tastatur, zurückgegeben als eine der folgenden Zeichenketten:
    • "versteckt"
    • "versteckt"
    • "zeigend"
    • "sichtbar"

Siehe hier für weitere Informationen zur virtuellen Tastatur.

XBox Live

Das asynchrone Systemereignis kann ausgelöst werden, wenn Sie die XBox One über den UWP-Export anvisieren und die Option "XBox Live aktivieren" in den UWP-Spieloptionen aktivieren. Wenn Sie ein GameMaker Studio 2 UWP-Projekt, das Xbox Live aktiviert hat, wird das Projekt automatisch versuchen, sich stillschweigend bei Xbox Live anzumelden. Die Ergebnisse dieses Anmeldeversuchs werden als eines der folgenden Schlüssel/Wertpaare in der async_load -Map zurückgegeben:

  • "event_type" - der Typ des empfangenen Systemereignisses, der eine der folgenden Zeichenketten sein wird:
  • "user signed in" - die stille Benutzeranmeldung wurde erfolgreich abgeschlossen
  • "user sign in failed" - die stille Benutzeranmeldung ist fehlgeschlagen (wenn dies passiert, können Sie die Funktion xboxlive_show_account_picker() verwenden, um den Benutzer dazu bringen, ein Konto zur Anmeldung zu wählen)
  • "user signed out" - der Benutzer hat sich abgemeldet

Andere Funktionen für XBox Live können dieses Ereignis ebenfalls auslösen, aber die verschiedenen Rückrufe für diese Funktionen sind auf den entsprechenden Funktionsseiten detailliert beschrieben. Weitere Informationen zu den spezifischen Funktionen, die für XBox Live verfügbar sind, finden Sie hier.

Status des Audiosystems

Dieser Ereignistyp ist speziell für die Arbeit mit dem HTML5-Ziel gedacht, da er jedes Mal ausgelöst wird, wenn sich der Web Audio-Kontextstatus ändert. Das bedeutet, dass Sie, wenn Sie z. B. ein Stück Hintergrundmusik in Schleife haben, dieses anhalten können oder stoppen und neu starten, basierend auf dieser Ereignisauslösung. Dies kann überprüft werden, indem Sie nach dem folgenden Schlüssel/Wert-Paar in der async_load -Map suchen:

  • "event_type" - der Typ des empfangenen Systemereignisses, der die Zeichenkette "audio_system_status" ist, wenn das Audiosystem initialisiert oder der Kontext geändert wurde.

Wenn dieser Ereignistyp ausgelöst wird, dann gibt es einen zusätzlichen Schlüssel in der Karte async_load:

  • "status" - Der Status des Audiosystems, der eine der beiden folgenden Zeichenketten sein wird
  • "available" - Das Audiosystem ist initialisiert und zur Wiedergabe von Tönen verfügbar
  • "unavailable" - Das Audiosystem ist nicht initialisiert oder der Kontext wird derzeit nicht ausgeführt und kann daher keine Sounds abspielen (alle Funktionen zum Abspielen von Sounds geben -1 zurück)

Weitere Informationen zu den spezifischen Funktionen, die für Audio verfügbar sind, finden Sie hier.

HINWEIS: Dieses Ereignis ist zwar für die Verwendung mit dem HTML5-Ziel vorgesehen, wird aber auch auf allen anderen Plattformen ausgelöst, allerdings wird es auf allen (außer HTML5) nur einmal beim Spielstart ausgelöst, wenn die Audio-Engine zum ersten Mal initialisiert wird.
