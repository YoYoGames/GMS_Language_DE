Dialog

Das asynchrone Ereignis Dialog wird nur ausgelöst, wenn es einen Rückruf von einer der speziellen asynchronen Dialogfunktionen erhält, wie get_login_async() (bitte siehe diese Funktion für ein erweitertes Codebeispiel, wie dieses Ereignis verwendet wird).

Diese Ereignisse sind diejenigen, die nach irgendeiner Art von Benutzereingabe fragen, die ein Name, Anmeldedaten, eine Zahl oder eine Farbe usw. sein kann. Da die meisten Geräte es nicht mögen, in einer Schleife zu sitzen und auf eine Antwort zu warten, müssen sie asynchron sein und GameMaker Studio 2 wird läuft im Hintergrund weiter, während diese Funktionen einen Dialog geöffnet haben und bis sie die erforderliche Benutzereingabe erhalten, die dieses Ereignis auslöst. Das heißt, wenn der Dialog geöffnet ist, während ein Spiel läuft, sollten Sie Maßnahmen ergreifen, um das Spiel zu pausieren Spiel zu unterbrechen, bis der Dialog-Callback empfangen wurde.

Auch hier wird eine DS-Map zurückgegeben, deren ID in der speziellen Variable async_load. Die Werte, die in dieser Map gehalten werden, hängen von der verwendeten Funktion ab, und Sie sollten die einzelnen Einträge für jede Funktion in diesem Handbuch für weitere Details konsultieren.

HINWEIS: Die asynchronen Dialogfunktionen sind nur für Debugging- und Testzweckegedacht und sollten nicht in veröffentlichten Spielen verwendet werden. Zu diesem Zweck sollten Sie eine benutzerdefinierte Benutzeroberfläche erstellen, um Eingaben von Spielern zu empfangen, damit Sie die vollständige Kontrolle über das Aussehen und Verhalten der Dialoge haben.
