Die Gestenereignisse

Ereignisse in der Kategorie "Gesten-Ereignis" werden immer dann ausgelöst, wenn GameMaker Studio 2 eine "Geste" der Maus oder ein Touchscreen-Ereignis erkennt (obwohl diese Gesten-Ereignisse speziell für den mobilen Einsatz konzipiert sind, können sie auch auf anderen Targets zur Erkennung der Maus verwendet werden, obwohl sie in diesem Fall keine Mehrfachberührungen erkennen). Das Ziel des Gestensystems ist es, zu versuchen, Eingaben auf einer höheren Ebene als die direkten Maus\Berührungslesefunktionen zu erkennen, und ist so konzipiert, dass es die Implementierung von häufig verwendeten Eingaben auf berührungsbasierten Geräten zu vereinfachen.

HINWEIS: Diese Ereignisse werden auf der HTML5-Plattform nicht ausgelöst, da Multi-Touch mit diesen Ereignissen auf dieser Plattform nicht unterstützt wird. Wenn Sie nach Gesten auf diesem Ziel suchen, sollten Sie die Gerätefunktionen verwenden.

Sie können wählen, ob Sie entweder Instanzgesten oder globale Gesten erkennen möchten, wobei Instanzgestenereignisse nur ausgelöst werden, wenn die erste Berührung/Klick auf eine Instanz innerhalb des Raums erfolgt. Beachten Sie, dass die Instanz eine gültige Kollisionsmaske haben muss (weitere Informationen finden Sie in den Abschnitten Der Sprite-Editor - Kollisionsmaske und Der Objekt-Editor - Kollisionsmaske ), damit dieses Ereignis ausgelöst wird. Globale Ereignisse hingegen werden jedoch durch Berühren/Klicken an beliebiger Stelle im Spielraum ausgelöst, und zwar für alle Instanzen, die das Ereignis haben.

Wenn eine Geste erkannt wird, löst sie eines oder mehrere der verfügbaren Ereignisse aus, und das ausgelöste Ereignis hängt von der Art der erkannten Geste ab. In jedem Fall wird jedoch eine DS-Map generiert und in der eingebauten Variable event_data gespeichert. Die verfügbaren Tasten hängen von der Art des Ereignisses ab und werden in jedem der folgenden Unterabschnitte gezeigt.

HINWEIS: Die Variable event_data ist nur in diesen Ereignissen gültig, da die DS-Map, auf die sie zeigt, zu Beginn des Ereignisses automatisch erstellt und am Ende wieder zerstört wird, wobei diese Variable zu allen anderen Zeiten auf den Wert -1 zurückgesetzt wird.

Es ist erwähnenswert, dass, wenn Sie mehrere Instanzen unter der Position haben, die berührt wird, und diese alle ein Gestenereignis haben, dann werden alle von ihnen ausgelöst, nicht nur die "oberste" Instanz. Beachten Sie auch, dass bei der Verwendung mehrerer Kameraansichten und dem Ziehen einer Instanz, die zurückgegebenen Werte auf der Ansicht basieren, in der Sie sich befanden, als die erste Berührung/Klick empfangen wurde - dies gilt für alle nachfolgenden Ereignisse in dieser Geste für diese Instanz. Wenn Sie also eine Instanz in einer Ansicht berühren und ziehen Wenn Sie also eine Instanz in einer Ansicht berühren und ziehen und dann die Berührung in einer anderen Ansicht loslassen, werden Werte relativ zu der ursprünglichen Ansicht zurückgegeben, in der die Geste zuerst erkannt wurde.

Das Ereignis "Antippen" wird ausgelöst, wenn eine Instanz berührt oder angeklickt wurde oder - wenn es ein globales Ereignis ist - wenn das Spiel eine Berührung oder einen Klick irgendwo im Raum registriert. Ein Antippen wird als schnelles Berühren und Loslassen betrachtet, und wenn die Berührung zu lange dauert Wenn die Berührung zu lange dauert, wird sie als Ziehen betrachtet (und löst das Ereignis "Ziehen" anstelle des Ereignisses "Tippen" aus). Dieses Ereignis erzeugt eine event_data DS-Map, die Sie dann verwenden können, um Informationen über das Ereignis zu erhalten. Zum Beispiel:

Ereignis erstellen

x_goto = x;
y_goto = y;

Tap-Ereignis

x_goto = event_data[? "posX"];
y_goto = event_data[? "posY"];

Schritt Ereignis

var _pd = point_distance(x, y, x_goto, y_goto);
move_towards_point(x_goto, y_goto, clamp(_pd, 0, 5);

Der obige Code erkennt ein Antippen des Bildschirms und ermittelt dann die Position des Antippens, um die Instanz an diese Position zu bewegen. Beachten Sie, dass Sie, wenn Sie eine längere oder kürzere Antipp-Erkennungszeit haben möchten, diese mit der Funktion gesture_drag_time(). Damit wird die Zeit zwischen der ersten Erkennung und dem Antippen als Ziehen eingestellt. Stellen Sie also einen höheren Wert ein, um die Antipperkennung länger zu machen, oder einen niedrigeren Wert, um sie kürzer zu machen (der Wert ist in Sekunden angegeben und standardmäßig 0,16).

Doppeltippen

Das Ereignis "Doppeltippen" wird ausgelöst, wenn eine Instanz zweimal kurz hintereinander berührt oder angeklickt wurde (oder - wenn es sich um ein globales Ereignis handelt - wenn das Spiel zwei schnelle Berührungen oder Klicks irgendwo im Raum registriert). Ein Doppeltap wird als zwei Wenn jedoch eine der Berührungen zu lange dauert, wird sie als Ziehen betrachtet (und löst das Ereignis "Ziehen" anstelle des Ereignisses "Doppeltippen" aus). Dieses Ereignis erzeugt eine event_data DS Karte, die Sie dann verwenden können, um Informationen über das Ereignis zu erhalten. Zum Beispiel:

Ereignis erstellen

x_goto = x;
y_goto = y;

Doppeltipp-Ereignis

instance_destroy();

Der obige Code erkennt einfach einen Doppeltap und zerstört dann die Instanz. Beachten Sie, dass Sie die Zeit zwischen den Antippen einstellen können, um einen Doppeltap auszulösen, indem Sie die Funktion gesture_double_tap_time() (die hat einen Standardwert - in Sekunden - von 0,16) und Sie können auch den Abstand für die Erkennung zwischen den Antippern mit der Funktion gesture_double_tap_distance() (wenn ein zweites Antippen außerhalb dieses Abstands erkannt wird, wird es als reguläres Antippereignis betrachtet).

Start ziehen

Das Ereignis "Start des Ziehens" wird ausgelöst, wenn der Benutzer eine Berührung oder einen Klick beibehält, ohne ihn loszulassen. Es wird einmal ausgelöst, wenn nach der ersten Berührung eine bestimmte Zeit verstrichen ist, die standardmäßig 0,16 Sekunden beträgt (Sie können diesen Wert jedoch mit der Funktion anderen Wert in Sekunden mit der Funktion gesture_drag_time()). Nachdem dieses Ereignis ausgelöst wurde, und solange der Benutzer eine Berührung/Klick gedrückt hält, wird das Ereignis "Ziehen" bei jedem Schritt ausgelöst, bis die Berührung/Klick losgelassen wird. Dieses Ereignis generiert eine event_data DS-Map, die Sie dann verwenden können, um Informationen über das Ereignis zu erhalten. Zum Beispiel:

Ereignis erstellen

drag_offsetX = 0;
drag_offsetY = 0;

Start-Ereignis ziehen

drag_offsetX = x - event_data[? "posX"];
drag_offsetY = y - event_data[? "posY"];

Der obige Code verwendet das Ereignis "Drag Start", um die Position der Berührung/des Klicks zu ermitteln und damit einen Offset-Wert für die x- und y-Achse zu setzen. Dieser kann dann beim Ziehen der Instanz verwendet werden, um sicherzustellen, dass sie nicht an die Position "springt". an der die Berührung/der Klick erkannt wurde (siehe das Ereignis "Ziehen" unten für eine Fortsetzung dieses Beispiels).

Ziehen

Das Ereignis "Ziehen" wird nach dem Ereignis "Ziehen Start" ausgelöst und wird für jeden Schritt ausgelöst, bei dem der Benutzer die Berührung/Klick auf die Instanz (oder den Bildschirm, wenn es sich um ein globales Ereignis handelt) beibehält und sich mehr als der definierte Schwellenwert für das Ziehen bewegt. Dieser Abstand beträgt standardmäßig 0,1 Zoll, kann aber mit der Funktion gesture_drag_distance(). Wenn es keine Bewegung stattfindet oder die Bewegung unter dem definierten Schwellenwert liegt, wird das Ereignis nicht ausgelöst. Dieses Ereignis erzeugt eine event_data DS-Karte, die Sie dann verwenden können, um Informationen über das Ereignis zu erhalten. Zum Beispiel:

Ereignis erstellen

drag_offsetX = 0;
drag_offsetY = 0;

Start-Ereignis ziehen

drag_offsetX = x - event_data[? "posX"];
drag_offsetY = y - event_data[? "posY"];

Ereignis "Ziehen

x = event_data[? "posX"] + drag_offsetX;
y = event_data[? "posY"] + drag_offsetY;

Der obige Beispielcode verwendet die im Ereignis "Ziehen Start" festgelegten Offset-Variablen, um die Instanz zu verschieben, wenn das Ereignis "Ziehen" ausgelöst wird.

Schleppende

Das Ereignis "Drag End" wird ausgelöst, wenn der Benutzer die Berührung/Klick auf die Instanz (oder den Bildschirm, wenn das Ereignis global ist) loslässt. Dieses Ereignis erzeugt eine event_data DS-Map, die Sie dann verwenden können, um Informationen über das Ereignis zu erhalten, aber in diesem Ereignis wird die Karte einen zusätzlichen Schlüssel haben: "isflick". Flick wird als die Distanz pro Sekunde berechnet, über die das Ziehen stattgefunden hat, und der Wert für den "isflick" ist wahr, wenn er größer ist als der definierte Wert für den Abstand pro Sekunde, andernfalls ist er falsch. Beachten Sie, dass der Standardwert 2 Zoll pro Sekunde ist, aber Sie können ihn mit der Funktion "Flick" auf einen anderen Wert setzen gesture_flick_speed(). Beachten Sie auch, dass es ein eigenes Flick-Ereignis gibt, das ebenfalls ausgelöst wird, wenn die Variable "isflick" wahr ist. Ein Beispiel für die Verwendung wäre:

Ereignis erstellen

flickVelX = 0.0;
flickVelY = 0.0;

Ziehen Ende Ereignis

isFlick = event_data[? "isflick"];
if (isFlick)
{
flickVelX = event_data[? "diffX"];
flickVelY = event_data[? "diffY"];
}
sonst
{
flickVelX = 0;
flickVelY = 0;
}

Schritt Ereignis

x += flickVelX;
y += flickVelY;
flickVelX *= 0.7;
flickVelY *= 0.7;

Der obige Code ermittelt einfach die Differenz der x- und y-Position zwischen dem letzten Ziehen-Ereignis und dem aktuellen Ziehen-Ende-Ereignis, und wenn die Bewegung größer war als der Flick-Schwellenwert, werden einige Variablen gesetzt, die für die Bewegung der Instanz verwendet werden im Schritt-Ereignis.

Flick

Das Ereignis "Flick" wird nur ausgelöst, wenn ein Touch/Klick gehalten, gezogen und dann losgelassen wurde und der Abstand zwischen der letzten Ziehposition und der Loslassposition größer als 2 Zoll pro Sekunde ist (dies ist die Standardeinstellung, die jedoch mit der Funktion kann jedoch mit der Funktion gesture_flick_speed()). Dieses Ereignis erzeugt eine event_data DS-Karte, die Sie dann verwenden können, um Informationen über das Ereignis zu erhalten. Zum Beispiel:

Ereignis erstellen

flickVelX = 0.0;
flickVelY = 0.0;

Flick-Veranstaltung

flickVelX = event_data[? "diffX"];
flickVelY = event_data[? "diffY"];

Schritt Ereignis

x += flickVelX;
y += flickVelY;
flickVelX *= 0.7;
flickVelY *= 0.7;

Der obige Code ermittelt einfach die Differenz in der x- und y-Position zwischen dem letzten Dragging-Ereignis und dem aktuellen Flick-Ereignis, und wenn die Bewegung größer als der Flick-Schwellenwert war, setzt er einige Variablen, die zum Bewegen der Instanz im dem Schritt-Ereignis.

Kneifende Ereignisse

Die "Pinch"-Ereignisse basieren darauf, dass zwei Berührungen des Gerätebildschirms gleichzeitig erkannt werden, bei denen sich eine (oder beide) um mehr als eine bestimmte Strecke bewegt haben. Der Bewegungswinkel der Berührungen zusammen mit der Bewegung jeder Berührung ist das, was Der Winkel der Bewegung der Berührungen zusammen mit der Bewegung jeder Berührung bestimmt die Erkennung eines Einklemm- oder Dreh-Ereignisses, wobei (im Fall des Einklemm-Ereignistyps):

  • Wenn sich eine der Berührungen nicht bewegt, muss sich die andere innerhalb eines Schwellenwinkels auf sie zu oder von ihr weg bewegen (der mit den Funktionen gesture_pinch_angle_towards() und - gesture_pinch_angle_away()).
  • Wenn sich beide Touches bewegen, müssen ihre Geschwindigkeiten in etwa entgegengesetzte Richtungen gehen und es wird auch die gleiche Winkelschwellenprüfung durchgeführt, um sicherzustellen, dass sich die Touches in ungefährer Ausrichtung bewegen.

Wenn zwei Berührungen und eine Bewegung mit den oben genannten Kriterien erkannt werden, wird ein Pinch-Ereignis ausgelöst, und in jedem der Ereignisse wird die event_data DS Map mit den folgenden Tasten bestückt:

Pinch Start

Das Ereignis "Pinch Start" wird ausgelöst, wenn eine Instanz (oder der Bildschirm, wenn das Ereignis global ist) von zwei "Fingern" berührt wurde (und die Berührung beibehalten wird) und dann einer oder beide der "Finger" bewegt werden. Wenn sich die Berührungen voneinander weg oder zueinander hin um mehr als den minimalen Prüfabstand (der standardmäßig 0,1 Zoll beträgt, aber mit der Funktion gesture_pinch_distance()), und der Winkel zwischen ihnen innerhalb des definierten Wertes liegt (dieser ist standardmäßig auf 45° eingestellt, kann aber mit gesture_pinch_angle_towards() und eingestellt werden. GameMaker_Language/GML_Reference/Game_Input/Gesture_Input/gesture_pinch_angle_away.htm">gesture_pinch_angle_away()), dann wird ein Pinch-Start-Ereignis ausgelöst. In diesem Ereignis können Sie Variablen setzen oder Positionsdaten zur späteren Verwendung speichern. Zum Beispiel:

Pinch-Start-Ereignis

pinch = true;
pinch_x = event_data[? "midpointX"]; pinch_y = event_data[? "midpointY"];

Der obige Code erkennt eine Einklemmung und speichert die Mittelpunktsposition für diese Einklemmung.

Einklemmen / Ausklemmen

Die Ereignisse "Pinch In" und "Pinch Out" werden bei jedem Schritt ausgelöst, bei dem sich der Abstand zwischen den beiden Berührungen, aus denen der Pinch besteht, über den Mindestschwellenwert hinaus ändert (standardmäßig auf +/- 0,1 Zoll eingestellt, aber Sie können ihn mit der Funktion gesture_pinch_distance ändern). Wenn es keine Bewegung der Pinch-Berührungen gibt, werden diese Ereignisse nicht ausgelöst. Diese Ereignisse erzeugen eine event_data DS-Map, die Sie dann verwenden können, um Informationen über das Ereignis zu erhalten. Zum Beispiel:

Globales Pinch In /Pinch Out Ereignis

var _scale = event_data[? "relativescale"];
var _w = camera_get_view_width(view_camera[0]);
var _h = camera_get_view_height(view_camera[0]);
var _x = camera_get_view_x(view_camera[0]) + (_w / 2);
var _y = camera_get_view_y(view_camera[0]) + (_h / 2);

_w *= _scale;
_h = _w * (room_height / room_width);
_x -= _w / 2;
_y -= _h / 2;

camera_set_view_pos(view_camera[0], _x, _y);
camera_set_view_size(view_camera[0], _w, _h);

Der obige Code skaliert die Ansicht basierend auf dem relativen Maßstab der Pinch-Berührungen.

Quetschende

Das Ereignis "Pinch End" wird ausgelöst, wenn der Benutzer eine (oder beide) der Berührungen vom Gerät loslässt. Dieses Ereignis erzeugt eine event_data DS-Map, die Sie dann verwenden können, um Informationen über das Ereignis zu erhalten. Für Beispiel:

Quetschende Ereignis

var _pinchx = event_data[? "midpointX"];
var _pinchy = event_data[? "midpointY"];
var _w = camera_get_view_width(view_camera[0]);
var _h = camera_get_view_height(view_camera[0]);
var _x = _pinchx - (_w / 2);
var _y = _pinchy - (_h / 2);

camera_set_view_pos(view_camera[0], _x, _y);

Der obige Code stellt die Ansichtsposition so ein, dass sie auf dem Mittelpunkt der beiden Berührungen zentriert ist, aus denen der Pinch besteht, wenn die Berührungen losgelassen werden.

Ereignisse rotieren

Die "Rotate"-Ereignisse basieren darauf, dass zwei Berührungen des Gerätebildschirms auf einmal erkannt werden und innerhalb einer bestimmten Zeit eine konsistente Winkeldrehung zwischen den beiden Berührungen stattgefunden hat. Der Winkel der Bewegung der Berührungen zusammen mit der Der Winkel der Bewegung der Berührungen zusammen mit der Bewegung jeder Berührung ist ausschlaggebend für die Erkennung eines "Pinch"- oder "Rotate"-Ereignisses, wobei (im Falle des Ereignistyps "Rotate"):

  • Zwei Berührungen müssen für eine bestimmte Mindestzeit gehalten werden (die Standardzeit ist 0,16 Sekunden, aber Sie können sie mit der Funktion gesture_rotate_time()).
  • Innerhalb dieser Mindestzeitspanne müssen sie sich in eine gleichbleibende Richtung drehen (wenn sich die Drehrichtung innerhalb dieser Zeit ändert, wird keine Drehung gestartet).
  • Der Drehbetrag muss den minimalen Schwellenwinkel überschreiten (der standardmäßig auf 5° eingestellt ist, aber mit der Funktion gesture_rotate_angle()).

Wenn zwei Berührungen und eine Bewegung mit den oben genannten Kriterien erkannt werden, wird ein Rotate-Event ausgelöst, und in jedem der Events wird die event_data DS Map mit den folgenden Tasten bestückt:

Drehen Start

Das Ereignis "Rotate Start" wird ausgelöst, wenn eine Instanz (oder der Bildschirm, wenn das Ereignis global ist) von zwei "Fingern" berührt wurde (und die Berührung beibehalten wird) und dann einer oder beide der "Finger" aus ihrer Startposition gedreht werden Position gedreht. Die Drehung der Berührungen muss innerhalb einer kurzen Zeitspanne begonnen haben (standardmäßig 0,16 Sekunden, aber sie kann mit der Funktion gesture_rotate_time()) und größer als die minimale Winkelschwelle sein (standardmäßig 5°, dies kann aber mit der Funktion gesture_rotate_angle()). Wenn diese Überprüfungen wahr sind, wird ein Rotate Start-Ereignis ausgelöst und Sie können es verwenden, um Werte zu speichern oder Variablen für die Verwendung mit den restlichen Rotate-Ereignissen zu setzen. Zum Beispiel:

Ereignis erstellen

rotating = false;
view_a = camera_get_view_angle(view_camera[0]);

Drehen Start Ereignis

rotierend = wahr;

Der obige Code richtet einfach einige Variablen für das Drehen der Fachkamera ein und setzt dann im Ereignis "Rotate Start" eine davon auf true.

Drehen

Das Ereignis "Drehen" wird bei jedem Schritt ausgelöst, bei dem sich die Berührungen auf dem Bildschirm umeinander drehen, solange die Bewegung größer als der minimale Winkelschwellenwert ist (standardmäßig 5°, aber das kann mit der Funktion gesture_rotate_angle()). Dieses Ereignis kann z. B. zum Setzen von Variablen und zur Manipulation von Instanzen verwendet werden:

Rotierendes Ereignis

var _relangle = event_data[? "relativeangle"];
var _a = camera_get_view_angle(view_camera[0]);
_a += _relangle;
camera_set_view_angle(view_camera[0], _a);

Der obige Code dreht die Kameraansicht in Abhängigkeit von der Drehbewegung der Berührungen im Event.

Drehen Ende

Das Ereignis "Rotate End" wird ausgelöst, wenn eine (oder beide) Berührungen, aus denen die Geste besteht, vom Gerätebildschirm losgelassen werden. Dieses Ereignis kann z. B. zum Setzen von Variablen und Manipulieren von Instanzen verwendet werden:

Drehen Ende Ereignis

rotierend = false;

Schritt Ereignis

if !rotating
{
var _a = camera_get_view_angle(view_camera[0]);
var _adif = angle_difference(view_a, _a);
_a += median(-5, _adif, 5);
camera_set_view_angle(view_camera[0], _a);
}

Der obige Code verwendet das Ereignis "Rotate End", um zu erkennen, wann der Benutzer die Geste beendet, und setzt dann eine Variable. Diese Variable wird dann im Schritt-Ereignis verwendet, um die Fachkamera wieder in ihre ursprüngliche Position zu drehen.
