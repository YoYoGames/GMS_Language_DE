Die asynchronen Ereignisse

Ein asynchrones Ereignis wird ausgelöst, wenn GameMaker Studio 2 einen Rückruf von einer externen Quelle empfängt, z. B. aus dem Web oder von dem Gerät, auf dem Ihr Spiel läuft. Im Grunde genommen, sagen Sie dem GameMaker Studio 2, dass es etwas tun soll - z. B. ein Bild laden - und es wird damit beginnen, dies zu tun, aber in der Zwischenzeit mit allem anderen fortfahren, was es zu tun hat. Dann, wenn die Anforderung abgeschlossen ist, wird ein Callback an GameMaker Studio 2 gesendet und alle asynchronen Ereignisse, die für diese Art von Callback definiert sind, werden ausgelöst.

Bitte beachten Sie, dass die asynchronen Ereignisse für alle Instanzen abgefeuert werden, die sie haben, ähnlich wie die Tastaturereignisse, so dass Sie - zum Beispiel - einen http_get() -Aufruf in einer Instanz durchführen können, aber das asynchrone HTTP-Ereignis in einer anderen haben, um den Rückruf zu behandeln.

HINWEIS: Sie sollten sich bewusst sein, dass aufgrund des XSS-Schutzes in Browsern Anfragen und Versuche, Ressourcen von anderen Domänen zu laden, blockiert werden und möglicherweise leere Ergebnisse zurückgeben, wenn Sie eines der folgenden Ereignisse verwenden.
