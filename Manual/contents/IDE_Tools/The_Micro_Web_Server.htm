Der Micro Web Server

Die Hauptfunktion dieses kleinen Tools besteht darin, eine Verbindung zu einigen der verfügbaren Zielplattformen für GameMaker Studio 2 (wie z. B. dem HTML5-Ziel) herzustellen und Dateien an diese zu übergeben. Wenn Sie ein Spiel auf einem Ziel ausführen, das den Micro Web Server benötigt, wird ein neues Fenster geöffnet, während das Spiel für das gewählte Gerät bereitgestellt wird:

HINWEIS: Wenn Sie zum ersten Mal ein Spiel auf einem Ziel ausführen, für das der Micro Web Server erforderlich ist, kann es sein, dass Sie eine Firewall-Warnung erhalten. Sie sollten dem Webserver erlauben, durch die Firewall zu kommunizieren, wenn dies geschieht.

Der Webserver zeigt eine bestimmte Menge an Debug-Informationen in Bezug auf die Dateien an, die serviert wurden, und verfügt über eine Schaltfläche zum Löschen dieser Informationen am unteren Rand. Neben der Schaltfläche "Clear" befindet sich der Pfad zur Root-JS-Datei für das Projekt, und darunter befindet sich die Option " Netzwerkgeschwindigkeit ". Diese ist standardmäßig auf "No Limit" eingestellt, aber wenn Sie darauf klicken, öffnet sich eine Liste mit verschiedenen Netzwerkoptionen:

Wenn Sie diese Einstellung ändern, wird auch die Option für die Paketgröße nach rechts verschoben. Sie können dies auch manuell bearbeiten, In diesem Fall wird die Netzwerkgeschwindigkeit auf "Benutzerdefiniert" gesetzt. Wenn Sie hier einen anderen Wert als den Maximalwert einstellen, wird die Netzwerkübertragungsgeschwindigkeit effektiv gedrosselt und Sie können Spiele in einer Umgebung mit geringer Bandbreite testen.

Sie können auch die Liste der erlaubten IPs zusammen mit der verwendeten Portnummer sehen, aber diese können nicht im Micro Web Server bearbeitet werden und sollten stattdessen in den Einstellungen des Gerätemanagers festgelegt werden für das jeweilige Geräteziel eingestellt werden.
