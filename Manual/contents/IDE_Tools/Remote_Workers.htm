Ferngesteuerte Arbeiter

Wenn Sie Ihr Spiel kompilieren, verwenden Sie normalerweise den "lokalen" Worker, d. h. den Computer, auf dem GameMaker Studio 2 installiert wurde. Unter bestimmten Umständen müssen Sie jedoch einen anderen Computer verwenden, um Ihre Projekte zu erstellen. Zum Beispiel, wenn Sie auf einem Mac arbeiten und ein Windows-Projekt erstellen möchten, dann würden Sie einen " Remote Worker" konfigurieren und einen Windows-PC mit der Erstellung beauftragen wollen. Wenn Sie einen Remote Worker verwenden, sendet GameMaker Studio 2 im Wesentlichen alle Wenn Sie einen Remote Worker verwenden, sendet GameMaker Studio 2 im Wesentlichen alle erforderlichen Dateien von dem (lokalen) Rechner, auf dem es läuft, an den Remote Worker-Rechner und verwendet dann diesen Rechner, um die endgültige ausführbare Datei für das Projekt zu erstellen (indem es sie entweder auf dem Remote-Rechner speichert oder sie anschließend an den lokalen Rechner zurückgibt). Bevor Sie diese Funktionalität nutzen können, müssen Sie jedoch bestimmte Tools auf dem Build-Rechner installiert und einige Einstellungen vorgenommen haben, die wir hier erläutern werden.

WICHTIG! Sie können einen Remote Worker mit einem lokalen Windows-PC und einem entfernten Windows-PC oder zwischen einem lokalen Mac-PC und einem entfernten Windows-PC einrichten, aber derzeit können Sie keinen lokalen Windows- (oder Mac-) PC und einen entfernten Mac-PC einrichten, d. h.: Sie können nicht Sie können keinen entfernten Mac-Arbeiter haben, nur einen Windows-Arbeiter.

Erstmalige Einrichtung

Um einen Remote Worker einzurichten, müssen Sie zunächst die Anwendung "Remote Worker Build Tool" auf dem PC installieren, der verwendet werden soll. Sie finden dieses Tool, indem Sie in GameMaker Studio 2 auf dem lokalen Rechner in das Build-Menü gehen und die Option " ShowRemote Worker Installer" wählen:

Alternativ können Sie auch auf das "Auge"-Symbol im Ziel-Manager klicken und so den das Remote Worker-Installationsprogramm auf diese Weise:

Das Installationsprogramm heißt in etwa "GameMakerStudio-Remote-Installer-X.X.X.XXX.exe" und Sie müssen es auf den PC kopieren, den Sie als Remote Worker verwenden möchten (Sie müssen GameMaker Studio 2 nicht auf dem Remote Worker installiert haben, nur dieses Programm).

Nachdem Sie es auf den Build-PC kopiert haben, installieren Sie es, indem Sie den Anweisungen auf dem Bildschirm folgen. Wenn das Installationsprogramm fertig ist, werden Sie möglicherweise vom Betriebssystem aufgefordert, über die Firewall auf Ihr Netzwerk zuzugreifen. Sobald Sie den Zugriff gewähren, öffnet das Installationsprogramm Sie müssen sich die IP-Adresse und die Port-Nummer, die auf dieser Seite angezeigt werden, notieren, da Sie sie für den nächsten Schritt benötigen.

Sie müssen nun zurück zu dem Rechner gehen, auf dem Sie GameMaker Studio 2 installiert haben, und klicken Sie auf die Schaltfläche Target Manager , um das Fenster Target Manager zu öffnen, und dort in den Bereich Worker gehen und auf die Schaltfläche Add Worker klicken. Dies öffnet dann das folgende Fenster:

Hier müssen Sie Ihrem Remote Worker einen Namen geben (das ist das, was in der in der Worker-Liste im Ziel-Manager angezeigt wird) und fügen Sie dann im Abschnitt Host-Name die Adresse zusammen mit dem Port hinzu, die Ihnen zuvor vom Worker-Rechner angezeigt und notiert wurde.

Wenn alles richtig gelaufen ist, sollten Sie nun eine Meldung erhalten, dass Sie sich erfolgreich mit dem Remote Worker-Rechner verbinden können. Sobald Sie Ihren Remote Worker installiert und mit Ihrem Haupt-IDE-Computer verbunden haben, müssen Sie die lokalen Build-Einstellungen für jede Plattform einrichten (siehe den Abschnitt unten für weitere Details), bevor Sie Ihr Projekt kompilieren können.

Kompilieren

Sobald Sie Ihren Remote Worker eingerichtet und die lokalen Build-Einstellungen vorgenommen haben (siehe den obigen Abschnitt), können Sie ihn zum Kompilieren Ihrer Spiele verwenden. Dazu müssen Sie lediglich den Remote Worker aus der Liste Target Manager auszuwählen, die Plattform zu wählen, für die getestet/kompiliert werden soll, sowie alle anderen Optionen (wie Kompilierungstyp oder Konfiguration), und dann auf Play oder Create Executable zu klicken.

HINWEIS: Die verfügbaren Plattformen im Ziel-Manager ändern sich je nach den Fähigkeiten des Remote Worker-Computers, und nicht alle Ziele sind verfügbar. Beachten Sie auch, dass es kein "Test"-Ziel für Remote Worker gibt.

Wenn Sie das Projekt einfach nur abspielen, wird es auf dem Remote Worker-Rechner kompiliert und ausgeführt, aber wenn Sie eine ausführbare Datei erstellen, dann kompiliert Remote Worker das Projekt und gibt dann eine Datei *.zip an den lokalen Rechner zum Speichern zurück.

Es ist zu beachten, dass weder die Schaltfläche "Debug" noch die Schaltfläche "Clean" verfügbar sind, wenn Sie einen Remote Worker verwenden. Beachten Sie auch, dass Sie derzeit nur einen Remote-Build zur gleichen Zeit durchführen können.


Sobald der Remote Worker auf dem entfernten Rechner installiert ist und läuft, erscheint er in der Taskleiste als Symbol, und Sie können mit der rechten Maustaste darauf klicken, um die folgenden Optionen aufzurufen:

  • Öffnen: Öffnen Sie die Remote Worker-Anwendung in Ihrem Standardbrowser.
  • Protokoll anzeigen: Öffnen Sie die Protokolldatei für den Remote Worker in Ihrem Browser.
  • Netzwerk-Einstellungen: Hier können Sie filtern, welche Rechner in der Lage sein sollen, Builds an den Remote Worker-Rechner zu senden. Jeder Filter kann ein CIDR-Filter oder ein Hostname einer Maschine sein, die zugelassen werden soll. Sie sollten sich bewusst sein, dass Sie den zulässigen IP-Bereich hier bearbeiten müssen, wenn Sie mehrere Maschinen mit einem Remote Worker verbinden möchten oder wenn Sie verschiedene Verbindungstypen verwenden (z. B. kabelgebunden und WLAN).
  • Einstellungen bearbeiten: Öffnen Sie das Menü für die Fernarbeitereinstellungen (erklärt im Abschnitt Einstellungen, oben).
  • Beenden: Beenden Sie die Remote-Worker-App. Dies bedeutet, dass Sie sich nicht mehr mit der App von einem lokalen Rechner aus verbinden (und auch nicht kompilieren) können, bis Sie die Remote Worker-App neu starten.
