Uhren

Die ganze Idee des Debuggers ist es, Ihnen zu erlauben, jedes Detail Ihres Spiels zu "beobachten". Dies geschieht mit Hilfe von "Überwachungsfenstern", die standardmäßig im Ausgabefenster am unteren Rand der IDE angedockt sind. am unteren Rand der IDE geöffnet sind. Sie können auf jede der verschiedenen Registerkarten klicken, um die darin enthaltenen Überwachungen zu sehen. Einige davon können in Echtzeit aktualisiert werden (wenn das Echtzeit-Debugging in der Debugger-Symbolleiste aktiviert ist), während andere erfordern, dass der Debugger das Spiel pausiert hat.

Wenn Sie Variablen in einem der Fenster beobachten, kann es sich um einen von mehreren Datentypen handeln. Aufgrund der Art und Weise, wie GameMaker Studio 2 bestimmte Dinge speichert, weiß der Debugger jedoch nicht immer genau, was die Variable enthält. Das liegt daran, dass die internen ID-Werte für Datenstrukturen oder Instanzen Integer-Werte sind und der Debugger daher nicht immer weiß, ob der Integer-Wert, der in einer Variablen gespeichert wird, für eine Datenstruktur oder eine Instanz steht oder einfach ein Integer-Wert ist. einfach ein Integer-Wert ist. Aus diesem Grund können Sie mit der rechten Maustaste auf einen beliebigen Variablenwert klicken und erhalten ein Popup-Fenster, das die verfügbaren Datentypen auflistet die diese Variable sein könnte:

Strings und Arrays erlauben diese Option nicht, und es ist erwähnenswert, dass selbst wenn eine Datenstruktur in einer Instanzvariablen gespeichert ist, ihre ID-Nummer global ist. Wenn Sie also vier Instanzen haben und jede von ihnen eine ds_list erzeugt (zum Beispiel), hat jede Liste eine ID von 0 bis 3, und sie werden nicht alle 0 sein.

Wenn Sie Ihren Datentyp ausgewählt haben, werden die Einträge in der Struktur der Variablen angezeigt, indem Sie einfach auf das kleine neben dem Namen der Variablen klicken. Dadurch wird der Inhalt der Struktur erweitert. Inhalt, und Sie können auch einige der Werte für die angezeigten Variablen ändern, indem Sie einfach auf den Wert doppelklicken und ihm einen neuen Wert geben.

HINWEIS: Sie können Arrays oder Datenstrukturen auf diese Weise nicht neu zuordnen, aber Sie können einige ihrer Inhalte ändern (Sie können weder die "Schlüssel"-Werte in Maps noch den Prioritätswert für Prioritätswarteschlangen ändern).

Einheimische

Das Fenster " Locals" zeigt die aktuellen Schritteigenschaften an, darunter den Namen des aktuellen Ereignisses (oder Skripts), das ausgeführt wird, den Offset im Code der virtuellen Maschine (VM) für das aktuelle Skript (die Abkürzung "PC" steht für "Programmzähler"), die "Selbst"-Instanz, die den den Codeblock ausführt, und die "andere" Instanz im Ereignis. Dieser letzte Teil ist immer nur dann gültig, wenn das Ereignis ein Kollisionsereignis ist oder im Code ein "with" verwendet wird. Wenn die Instanz in Kollision ist, Wenn die Instanz in Kollision ist, zeigt "other" die andere Instanz (und ihre Variablen) in der Kollision an, aber wenn es sich um eine with handelt, dann wird die "Self"-Instanz zu derjenigen, die den Code ausführt, während "Other" dann die Instanz ist ist, die tatsächlich die with enthält. Zu allen anderen Zeiten zeigt "Other" einfach die gleichen Daten wie "Self".

Beachten Sie, dass die Instanzen "Self" und "Other" durch Klicken auf erweitert werden können, um eine Liste aller lokalen und Instanzvariablen zu sehen, die aktuell für den durchlaufenen Codeblock deklariert sind. Wenn Sie mit der rechten Maustaste auf einen Wert klicken, können Sie seinen Typ festlegen, um Details zu Datenstrukturen zu sehen.
