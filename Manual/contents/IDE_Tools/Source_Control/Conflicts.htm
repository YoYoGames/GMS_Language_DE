Konflikte

In diesem Abschnitt werden wir uns das Auflösen von Konflikten zwischen einem geklonten lokalen Repository und einem Haupt-Repository ansehen. Unter der Annahme, dass Sie jetzt ein lokales Repository verwenden, müssen wir eine Datei bearbeiten und dann Öffnen Sie also das Testskript, das wir verwendet haben, und ändern Sie die "Hello World"-Meldung in etwas wie dieses:

show_debug_message("Hello World, How Are You?");

Sie sollten nun das Projekt speichern und es in das lokale Repository übertragen. Da wir nun mit einem entfernten Master-Repository verbunden sind (auch wenn es eines ist, das wir auf dieselbe Festplatte geklont haben), müssen wir einen Push durchführen, um die Änderungen in das Master-Repository zu übertragen. Gehen Sie dazu zu Versionsverwaltung > Änderungen pushen, was uns eine Statusaktualisierung in der Registerkarte Versionsverwaltung Ausgabe gibt :

Wir müssen jetzt unser Originalprojekt öffnen (nicht das geklonte), also suchen Sie nach dem Ort, an dem Sie es auf der Festplatte gespeichert haben - oder schauen Sie einfach unter " Letzte Projekte" im Menü "Datei" nach, da es direkt unter dem aktuellen Projekt stehen sollte - und laden Sie es in GameMaker Studio 2. Ein einfacher Weg, um zu überprüfen, ob Sie das richtige Projekt haben, ist, einfach das Script und wenn die Meldung "Hello World" lautet, dann haben Sie das richtige, oder Sie können das Fenster " View History " öffnen und auf diese Weise überprüfen.

Sie müssen nun das Skript erneut bearbeiten, so dass es etwa so lautet:

show_debug_message("Dies wird einen Konflikt verursachen.");

Dies kann auf der Festplatte gespeichert und die Änderungen festgeschrieben werden, aber wenn Sie versuchen, diese Änderungen zu übertragen, erhalten Sie die folgende Meldung:

Jemand hat bereits Änderungen gepusht, die Sie noch nicht haben! Das bedeutet, dass Sie einen Pull auf das Projektarchiv ausführen und sehen, was genau das Problem ist, also gehen Sie zu Versionsverwaltung > Änderungen ziehen. Dies wird zwei Fenster zeigen, das erste ist eine Warnung über geänderte Dateien (wie bei den vorherigen Aktionen, wollen Sie Reload und nicht Save wählen), und das andere ist das Fenster Conflicts :

In diesem Fenster werden die konfliktbehafteten Dateien in einer Liste auf der linken Seite angezeigt, und Sie können die Dateien in dieser Liste mit der linken Maustaste auswählen Taste (oder wählen Sie mehrere Dateien mit / oder zusammen mit ). Sie haben vier Möglichkeiten, mit den konfliktbehafteten Dateien umzugehen, die Sie über die Schaltflächen auf der rechten Seite auswählen können:

  • Use Theirs - Damit werden die von Ihnen vorgenommenen Änderungen mit denen aus dem entfernten Repository überschrieben.
  • Meine verwenden - Damit werden die Änderungen aus dem entfernten Repository mit denen überschrieben, die Sie gerade vorgenommen haben.
  • Zusammenführen - Damit wird versucht, ein externes Zusammenführungswerkzeug zu öffnen, um den Konflikt zu behandeln.
  • Wählen Sie Merge Tool - Damit können Sie das Merge Tool konfigurieren (weitere Informationen finden Sie im Abschnitt über externe Merge/Diff Tools ).

Jetzt können Sie die Schaltflächen verwenden, um den Konflikt zu beheben, aber bevor Sie das tun, lassen Sie uns einfach die konfliktbehaftete Datei öffnen, um zu sehen, was genau das Problem ist. Wenn Sie das konfliktbehaftete Skript öffnen, sieht es nun wie folgt aus:

Dies zeigt, dass sich "This Will Cause A Conflict" im HEAD befindet (das ist das, was wir in das lokale Repository geschoben haben) und dass "Hello World, How Are You?" aus master, dem entfernten Repository, gezogen wird. Sie können jetzt zurück zum Konfliktfenster gehen und auf Use Theirs klicken, um die geänderte Datei zu ziehen und die aktuelle HEAD-Datei zu überschreiben. Das Skript wird nun als geändert markiert und Sie sollten einen neuen Commit und Push durchführen, bevor Sie fortfahren.

Beachten Sie, dass wir dies tatsächlich in der GameMaker Studio 2 IDE beheben können. Wenn wir die Zeilen 1, 2, 3 und 5 entfernen, so dass nur noch show_debug_message("Hello World, How Are You?") übrig bleibt, und es speichern, wird der Konflikt behoben, da Git davon ausgeht, dass Sie wissen, was Sie mit Ihrer eigenen Datei tun. Dies ist jedoch fehleranfällig, daher wird empfohlen, ein Zusammenführungswerkzeug zu installieren, zu konfigurieren und dieses zu verwenden, anstatt manuell zu versuchen, die Zusammenführung durchzuführen.
