Repository-Optionen

Nachdem Sie die GIT-Anmeldeinformationen eingerichtet und die Versionskontrolle für Ihr Projekt aktiviert haben, müssen Sie nun die GameMaker Studio 2 IDE mit einem Repository verknüpfen, was auf eine von drei Arten geschieht:

  • Erstellen eines Projekt-Repositorys
  • Das aktuelle Projekt in ein externes Repository verschieben
  • Klonen eines vorhandenen Repositorys

Im ersten Fall, dem Anlegen eines Projekt-Repositorys, wird ein GIT-Repository direkt dort angelegt, wo sich Ihr Projekt befindet. Im GIT-Sprachgebrauch ist dies ein "Nicht-Bare-Repository", daher machen Push/Pull in diesem Fall keinen Sinn, aber Sie können trotzdem Änderungen am Projekt übertragen und nachverfolgen, sowie Änderungen und Revisionen rückgängig machen, wenn es nötig ist. Um dies einzurichten, müssen Sie nur im Hauptmenü auf Versionsverwaltung > Projekt-Repository erstellen klicken.

Möglicherweise haben Sie jedoch bereits ein bestehendes Projekt und möchten es in ein externes Repository zur Verfolgung und gemeinsamen Nutzung für Ihr Team. Dazu muss ein externes Repository bereits eingerichtet sein, und Sie müssen in Ihren Identitätseinstellungen den richtigen Benutzernamen und das richtige Passwort für den Zugriff darauf festgelegt haben. Wenn Sie dies vorbereitet haben, klicken Sie auf Versionsverwaltung > Projekt in Repository importieren und geben Sie die URL ein:

Schließlich können Sie als Komfortfunktion ein externes Repository über die IDE klonen . Auch hier benötigen Sie den Benutzernamen und das Passwort für das Repository, mit dem Sie sich verbinden wollen, in Ihren Identitätseinstellungen, und dann ist es klicken Sie einfach auf Versionsverwaltung > Repository klonen . Füllen Sie die URL aus (verwenden Sie die HTTPS-Variante statt SSH) und sagen Sie, wohin sie gehen soll:

Sobald der Klonvorgang abgeschlossen ist, wird automatisch ein Dateibrowser geöffnet, in dem Sie das Projekt öffnen können, wenn Sie dies wünschen.
