Das GIT-Plugin einrichten

Wenn wir mit der Verwendung des GIT-Plugins beginnen, müssen wir eine Identität einrichten, mit der GIT die Übergabe durchführt, d. h. wir müssen zum Abschnitt Plugins - Source Control (Git ) der Voreinstellungen gehen und Ihre Authentifizierungsdetails hinzufügen.

Diese Authentifizierungsidentität wird für alle zukünftigen Projekte verwendet, und GameMaker Studio 2 gibt Ihnen zwei verschiedene Möglichkeiten, sie einzurichten einrichten:

  • Benutzername und Kennwort: Wenn Sie auf die Schaltfläche "Neuen Benutzer/Passwort-Authentifizierung hinzufügen" klicken, wird das folgende Fenster angezeigt, in dem Sie die Repository-URL, den Benutzernamen und das Kennwort eingeben können das Passwort:
    Wenn Sie möchten, dass dies nur ein bestimmtes Repository betrifft, geben Sie die URL für das Repository in das obere Feld ein. Wenn Sie jedoch eine Wenn Sie jedoch eine domänenweite Authentifizierung wünschen (z. B. "bitbucket.org" oder "github.com"), geben Sie nur die Domäne an und sonst nichts. Wenn etwas auf Authentifizierung geprüft werden muss, wird zuerst nach einer spezifischen Repository-Übereinstimmung und dann nach einer Domänenübereinstimmung gesucht. Auf diese Weise können Sie eine Standardauthentifizierung für eine Domäne einrichten und diese später mit spezifischen Details für bestimmte Repositorys überschreiben.

HINWEIS: Wenn Sie ein externes Repository verwenden, sollten Sie den Benutzernamen und das Passwort verwenden, die mit dem für dieses Repository eingerichteten Konto verknüpft sind, aber wenn Sie ein lokales Repository verwenden möchten, können Sie einen beliebigen Benutzernamen und ein beliebiges Passwort verwenden.

  • SSH-Schlüsselpaare: Wenn Sie auf "Neue SSH-Schlüsselpaar-Authentifizierung hinzufügen" klicken, wird das folgende Fenster angezeigt, in dem Sie die Repository-URL und das Passwort eingeben können , bevor Sie die Pfade zu den erforderlichen Dateien für den öffentlichen und den privaten Schlüssel angeben:
    Wie bei der Benutzername/Passwort-Authentifizierung können Sie auch bei der Verwendung von SSH-Schlüsselpaaren domänen- und repositoryspezifische Authentifizierungen vornehmen. Wenn Sie bereits ein SSH-Schlüsselpaar generiert haben, können Sie hier die Pfade sowie das für den Zugriff erforderliche Passwort einrichten. Beachten Sie, dass, sollten Sie schnellen Zugriff auf den öffentlichen Schlüssel benötigen, die Kopierschaltfläche den Inhalt automatisch für Sie in die Zwischenablage kopiert. Wenn Sie noch kein SSH-Schlüsselpaar haben, klicken Sie auf die Schaltfläche " Create KeyPair" (Schlüsselpaar erstellen ), um ein Dateidialogfeld zu öffnen, in dem Sie den privaten Schlüssel ablegen können, und der öffentliche Schlüssel wird daneben erzeugt - die Pfade zu den öffentlichen und privaten Schlüsseln werden dabei automatisch für Sie ausgefüllt.

Diese Einstellungen werden nun für die Kommunikation mit dem Repository (IE: Git Hub, Bit Bucket, etc.) verwendet, wenn Klon-, Push- und Pull-Anfragen bearbeitet werden. Sie müssen jedoch noch die projektspezifischen Optionen einrichten, was Sie tun können, indem Sie die Hauptoptionen öffnen und auf die Option " Source Control aktivieren " klicken. Dadurch wird SCM für das aktuelle Projekt aktiviert.

HINWEIS: Wenn Sie die Authentifizierungsdetails erneut bearbeiten müssen, doppelklicken Sie einfach auf den Eintrag und das Detailfenster wird geöffnet, damit Sie es bearbeiten können, aber Sie können die URL nicht umbenennen. Wenn Sie die URL ändern müssen, müssen Sie müssen Sie die Authentifizierungsdetails löschen und neu hinzufügen. Um die Authentifizierungsdetails zu löschen, klicken Sie auf die Schaltfläche "Schließen" links neben der Zeile im Hauptfenster "Voreinstellungen".
