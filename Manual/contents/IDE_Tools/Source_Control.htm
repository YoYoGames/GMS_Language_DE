Quellensteuerung

Source Control Management (SCM) ist die Bezeichnung für die Methode, mit Unterversionen oder Backups Ihrer Projekte über ein Repository und einen lokalen Quellcode zu arbeiten. Im Grunde genommen ist eine SCM-Lösung ein unabhängiges Softwarepaket das alle Aspekte der Pflege, Änderung und des Vergleichs von Versionen Ihres Projekts steuert, während Sie daran arbeiten. Dies ist besonders nützlich für diejenigen, die in einem Team arbeiten und in der Lage sein müssen, zu kontrollieren, wer was macht und sich keine Sorgen über Datenverlust oder Änderungen, die zu einem späteren Zeitpunkt wieder rückgängig gemacht werden müssen, aber auch Einzelpersonen können von diesem leistungsstarken und flexiblen System profitieren.

Es stehen Ihnen verschiedene SCM-Tools zur Verfügung, und GameMaker Studio 2 wird mit einem Git-Plugin geliefert, das sofort verwendet werden kann, und Sie müssen keine zusätzlichen Pakete installieren, da alles mit dem Plugin selbst gebündelt ist. Unten finden Sie Links zu einem kleinen Tutorial, wie man dieses Plugin einrichtet und die SCM-Tools mit einem Projekt verwendet.

Wenn Sie die Versionskontrolle für ein beliebiges Projekt aktiviert haben (siehe den Abschnitt Einrichten des Git-Plugins für die Versionskontrolle weiter oben), zeigt das GameMaker Studio 2 das folgende Kontextmenü am im oberen Bereich der IDE:

Hier wird jede der verfügbaren Optionen umrissen (die meisten davon werden in den obigen Abschnitten der Tutorials ausführlicher erklärt):

  • Projekt-Repository erstellen: Mit dieser Option können Sie ein lokales Repository in demselben Verzeichnis wie Ihre Projektdateien erstellen.
  • Projekt in Repository importieren: Mit dieser Option können Sie ein Projekt in ein externes Repository importieren.
  • Repository klonen: Mit dieser Option können Sie ein Repository von einer Quelle zu einem neuen Ziel klonen.
  • Änderungen festschreiben: Mit dieser Option können Sie geänderte Dateien in Ihrem Projekt bereitstellen und dann an das Repository übergeben.
  • Änderungen pushen: Nach einem Commit oder Merge werden mit dieser Option die Änderungen in das Master-Repository verschoben.
  • Änderungen ziehen: Mit dieser Option können Sie das lokale Repository aktualisieren, indem Sie die geänderten Dateien aus dem Haupt-Repository ziehen.
  • Verlauf anzeigen: Dadurch wird das Verlaufsfenster geöffnet, in dem Sie den gesamten Versionsverlauf des Projekts einsehen und wählen können, ob Sie bestimmte Dateipfade oder ganze Revisionen zurücksetzen möchten.
  • Konflikte anzeigen: Dies öffnet das Fenster "Konflikte" und listet alle konfliktbehafteten Dateien auf, die möglicherweise im Projekt vorhanden sind, so dass Sie sie entweder über die IDE von GameMaker Studio 2 oder mit einem speziellen Zusammenführungswerkzeug bearbeiten können.
