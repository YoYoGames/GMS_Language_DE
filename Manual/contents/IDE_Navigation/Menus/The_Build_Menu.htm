Das Menü "Erstellen

Im Menü "Build" können Sie wählen, ob Sie Ihr Projekt zum Testen, Debuggen oder als endgültige ausführbare Datei erstellen möchten. Die verfügbaren Optionen sind:

  • Run - Führen Sie Ihr aktuelles Projekt auf der gewählten Zielplattform zum Testen aus. Die verschiedenen Kompilieroptionen, die beim Kompilieren zur Verfügung stehen, finden Sie im Abschnitt " Kompilieren". Beachten Sie, dass dieser Befehl dasselbe ist wie das Drücken der Schaltfläche Ausführen in der IDE.
  • Debuggen - Führen Sie Ihr aktuelles Projekt zum Testen mit dem Debugger aus, damit Sie detaillierte Leistungsinformationen sehen und eventuelle Fehler oder Probleme beheben können. Weitere Informationen zum Debuggen finden Sie im Abschnitt Debugging. Beachten Sie, dass dieser Befehl dasselbe ist wie das Drücken der Schaltfläche Debuggen in der IDE.
  • Clean - GameMaker Studio 2 hält einen Cache von Dateien, um die Kompilierzeit beim Testen zu beschleunigen. Manchmal kann ein "veralteter" Cache jedoch zu seltsamen und unerwarteten Fehlern in Ihrem Spiel führen. in Ihrem Spiel führen. In diesem Fall sollten Sie als erstes den Cache von hier aus leeren und erneut testen. Es ist auch eine gute Idee, den Cache zu bereinigen, bevor Sie eine endgültige ausführbare Datei für die Zielplattform erstellen. Beachten Sie, dass dieser Befehl dasselbe ist wie das Drücken von die Schaltfläche " Cache leeren " in der IDE.
  • Executable erstellen - Damit wird das aktuelle Projekt kompiliert und ein ausführbares Paket (oder Pakete) für die Zielplattform erstellt. Es öffnet zunächst den Datei-Explorer damit Sie auswählen können, wo Sie das Projekt speichern möchten, und dann das Spiel kompilieren. Weitere Informationen zum Einrichten der Kompilierziele finden Sie im Abschnitt über das Kompilieren. Beachten Sie, dass dieser Befehl dasselbe ist dasselbe ist wie das Drücken der Schaltfläche Ausführbare Datei erstellen in der IDE.
  • Show Remote Worker Installer - Dies öffnet ein Datei-Explorer-Fenster am Speicherort des Installationsprogramms für das Remote Worker-Tool (siehe hier für weitere Informationen zu Remote Worker).
