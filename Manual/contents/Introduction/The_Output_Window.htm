Das Ausgabefenster

Wenn Sie ein Projekt in GameMaker Studio 2 zum ersten Mal öffnen, wird das Ausgabefenster am unteren Rand des Bildschirms angedockt angezeigt. Dieses angedockte Fenster enthält verschiedene Registerkarten, die die verschiedenen Ausgabeinformationen für Ihr Projekt anzeigen, abhängig von bestimmten Umständen. Das Dock kann durch Klicken auf die Schaltfläche am unteren Rand der IDE geschlossen werden, und Sie können auf klicken und eine beliebige Registerkarte im angedockten Fenster in ein anderes Dock ziehen, um ihre Position zu ändern, oder Sie können auf in der Inhaltsleiste des Docks klicken und es in den Arbeitsbereich ziehen, um ein eigenständiges Fenster zu erstellen:

Sie können auch auf klicken und eine angedockte Ausgabe in eine andere ziehen, um ein Ausgabefenster mit geteilter Ansicht zu erstellen, wie im folgenden Beispiel gezeigt, in dem die beiden Fehlerausgaben in derselben Registerkarte platziert wurden (Sie können langsam auf auf eine Registerkarte klicken, um auch ihren Namen zu ändern):

Sie können den Standardzustand der IDE jederzeit wiederherstellen, indem Sie Layout zurücksetzen aus dem Fenster Layouts wählen und Sie können jede geschlossene Registerkarte über das Fenster-Menü wieder öffnen. Sie können auch die Art und Weise ändern, wie der Inhalt jedes der Ausgabefenster angezeigt wird, indem Sie die Ausgabefenster-Voreinstellungen bearbeiten.

Ausgang

Im allgemeinen Ausgabefenster werden alle Compiler-Ausgaben angezeigt, sowie alle Meldungen, die Sie Ihrem Spiel mit die Funktion show_debug_message() Funktion hinzugefügt haben. Die meisten der anfänglich angezeigten Informationen sind lediglich Debug-Informationen darüber, wie das Spiel erstellt wird, und und können daher im Allgemeinen ignoriert werden. Wenn Sie jedoch ein Problem bei der Erstellung Ihres Projekts für eine Zielplattform haben, kann dies hilfreich sein, um die Ursache zu finden und Informationen für den Support bereitzustellen, falls Sie diesen kontaktieren. Beachten Sie, dass Sie die Menge der hier angezeigten Informationen im Abschnitt " Kompilieren" in den Voreinstellungen ändern können.

Suchergebnisse

Sie können das Fenster Suchen und Ersetzen mit dem Tastaturkürzel / + + " F" öffnen, oder gehen Sie in das Menü "Bearbeiten". Nachdem Sie Ihre Suchbegriffe eingegeben haben, werden die Ergebnisse in diesem Fenster mit dem Format angezeigt:

[Objekt] - [Ereignis] - [Zeilennummer]: [Suchbegriff]

Wenn der Suchbegriff in einem Skript gefunden wird, dann wird es einfach sein:

[Skript] - [Zeilennummer]: [Suchbegriff]

Sie können dann auf auf einen der zurückgegebenen Einträge doppelklicken, um das betreffende Asset an der richtigen Stelle zur Bearbeitung zu öffnen.

Quellensteuerung

In diesem Fenster werden alle Ausgaben für Ihr SCM-Plugin angezeigt. Sie können mehr über die Einrichten der Versionskontrolle erfahren Sie hier.

Haltepunkte

Haltepunkte sind Stellen in Ihrem Spielcode oder DnD™, an denen Sie GameMaker Studio 2 angewiesen haben, die Ausführung des Projekts anzuhalten, während Sie sich im Debug-Modus. Sie können mit der Taste " F9" an beliebiger Stelle in der Spielschleife einen Haltepunkt hinzufügen, der dann in dieser Ausgaberegisterkarte erscheint. Sie können sie aktivieren und deaktivieren (ohne sie zu entfernen) aktivieren und deaktivieren (ohne sie zu entfernen), indem Sie auf das Kontrollkästchen links daneben klicken, und wenn Sie mit der rechten Maustaste auf einen klicken, erhalten Sie ein kleines Menü, das es Ihnen erlaubt, den Haltepunkt zu entfernen oder das Code/DnD™-Fenster zu öffnen, in dem sich der Haltepunkt befindet. Syntax-Fehler

Syntaxfehler sind die Fehler in Ihrem Code, die der GameMaker Studio 2-Syntax-Checker aufgespürt hat. Es gibt viele Gründe, warum Sie einen Syntaxfehler erhalten können - z. B. ein ";" in Ihrem Code, obwohl ein ":" erwartet wird - und diese sollten behoben werden, bevor Sie versuchen, Ihr Spiel zu testen oder zu kompilieren. Jeder Syntaxfehler-Eintrag im im Ausgabefenster kann mit der linken Maustaste doppelt angeklickt werden, um ein Fenster an der markierten Stelle zu öffnen, in dem der Fehler behoben werden kann.

Syntaxfehler werden aktualisiert, während Sie Ihren Code schreiben (es gibt eine kurze Pause zwischen der Eingabe von etwas und dem Erscheinen von Fehlern in diesem Fenster, um zu verhindern, dass Fehler für unfertigen Code gemeldet werden) und folgen dem Format:

[Objekt] - [Ereignis] - [Zeilennummer] - [Position in Zeile]: [Fehlerstring]

Oder, wenn der Fehler in einem Skript liegt, dann schon:

[Skript] - [Zeilennummer] - [Position in der Zeile]: [Fehlerstring]

Beachten Sie, dass Syntaxfehler in der Regel verhindern, dass das Spiel kompiliert wird. Es gibt jedoch Fehler, die markiert werden, die das Kompilieren nicht verhindern, die aber dennoch behoben werden sollten. Die fraglichen Fehler sind:

  • für deklarierte Variablen, die nirgendwo verwendet werden
  • für irgendwo verwendete Variablen, die noch nicht deklariert wurden

Im Allgemeinen deutet beides auf einen Tippfehler in einem Variablennamen oder einen anderen leicht zu machenden Fehler hin - und wird daher zur Überarbeitung markiert -, aber es kann auch Absicht sein, da Sie aber es kann auch Absicht sein, da Sie die Variable vielleicht für die zukünftige Verwendung deklariert haben, aber noch nicht dazu gekommen sind, sie tatsächlich zu verwenden. Aus diesem Grund kann Ihr Spiel mit diesen Fehlern trotzdem kompiliert werden. Eine vollständige Liste der möglichen Syntax-Fehlermeldungen finden Sie hier.

Kompilier-Fehler

Ein Compiler-Fehler tritt auf, wenn Ihr Spiel auf eine Art Fehler stößt, den der Syntax-Checker möglicherweise nicht erkennen konnte, oder wenn ein Fehler damit zusammenhängt, wie Sie die Kompilieroptionen eingestellt haben. In diesem Fall meldet Ihr Spiel einen Compiler-Fehler und hört auf zu laufen. Alle Compiler-Fehler werden auch im Fenster "Allgemeine Ausgabe" angezeigt, aber sie werden hier auch separat aufgeführt (da sie im Rest der allgemeinen Ausgabe "verloren" gehen können).

Die Compiler-Fehlermeldungen folgen alle dem gleichen Format:

[Objekt] - [Ereignis] - [Zeilennummer]: [Fehlerstring]

Wenn der Fehler in einem Skript gefunden wird, dann wird es einfach sein:

[Skript] - [Zeilennummer]: [Fehlerstring]

Sie können dann auf auf einen der Compiler-Fehlereinträge doppelklicken, um das angegebene Asset an der Position zu öffnen, die als fehlerhaft gekennzeichnet ist. Eine vollständige Liste der möglichen Fehlermeldungen finden Sie hier.
