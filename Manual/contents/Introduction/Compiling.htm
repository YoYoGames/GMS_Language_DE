Kompilieren

DasKompilieren Ihres Spiels kann eines von zwei Dingen bedeuten: Kompilieren zum Testen oder Kompilieren zum Erstellen eines ausführbares Paket oder eine bestimmte Zielplattform. Sie können Ihr Spiel zum Testen kompilieren, indem Sie einfach auf die Schaltfläche "Play" oben in der IDE klicken, wodurch das Spiel zum Testen gestartet wird unter Verwendung des angegebenen Ziels. Sie können das Spiel auch im Debug-Modus ausführen, indem Sie zum Testen die Schaltfläche Debug verwenden. Dadurch wird das Spiel ausgeführt, aber es wird auch das Debug-Fenster, in dem Sie die Leistung Ihres Spiels überwachen und eventuelle Probleme erkennen können (weitere Informationen finden Sie im Abschnitt über Debugging ).

Standardmäßig wird GameMaker Studio 2 mit der eingebauten virtuellen Maschine (VM) ausgeführt und debuggt, was mehr oder weniger dem Betrieb auf dem verwendeten Desktop-Betriebssystem entspricht. Allerdings ist GameMaker Studio 2 ist jedoch eine plattformübergreifende Engine und Sie können ausführbare Pakete Ihrer Projekte auf einer Reihe verschiedener Zielplattformen testen, debuggen und kompilieren (welche Plattformen genau zur Verfügung stehen, hängt von den Details Ihrer Lizenz ab). Um die aktuelle Zielplattform zu ändern, können Sie auf auf die Schaltfläche "Ziele" klicken, um das Fenster "Ziele" zu öffnen, das etwa so aussieht (die genauen Details hängen von Ihrem Lizenztyp ab):

Ausgang

Jede Zielplattform kann ein oder mehrere Ausgabeformate haben, die wichtigsten sind:

  • VM: Das VM (Virtual Machine)-Target verwendet einen generischen Runner für jede Plattform und interpretiert dann den Code für Ihr Spiel. Im Allgemeinen wird diese Option aufgrund der schnelleren Erstellungszeiten zum Testen verwendet, aber sie bietet nicht den aber sie bietet nicht den gleichen Leistungsschub wie die YYC-Option (falls verfügbar). Sie können dies verwenden, um ausführbare Pakete für kleinere Spiele oder Spiele zu kompilieren, bei denen die Leistung nie ein Thema sein wird.
  • YYC: Der YYC(YoYo Compiler) kompiliert die normale GameMaker Studio 2-Ausgabe in nativen Code für die Zielplattform, wobei er nicht benötigte Funktionen "entfernt" und eine Reihe anderer Optimierungsverfahren durchführt um ein kleineres und leistungsfähigeres ausführbares Paket zu erstellen. Dies kann die Leistung Ihrer Spiele um mindestens das Zwei- oder Dreifache steigern, insbesondere bei logiklastigen Spielen, und ist ideal für größere oder CPU-intensive Projekte. Die Kompilierzeiten können länger dauern und Sie sollten immer den Compiler-Cache leeren, bevor Sie ein endgültiges vollständiges Asset-Paket für eine Zielplattform erstellen. Beachten Sie, dass für das YYC-Ziel möglicherweise zusätzliche Tools für die ausgewählte Plattform installiert werden müssen, da es sonst nicht funktioniert. sonst funktioniert es nicht - weitere Informationen dazu finden Sie im YoYo Games Help Center sowie auf den einzelnen Target Preferences Seiten in diesem Handbuch.
  • JavaScript: Das JavaScript-Ziel ist nur für einige wenige Ziele verfügbar, wie das HTML5-Ziel, und legt fest, dass das Spiel in reinem JavaScript kompiliert wird.

Erstellen eines endgültigen ausführbaren Pakets

Das Kompilieren Ihres Spielprojekts für eine beliebige Zielplattform setzt voraus, dass Sie die richtigen Build-Tools (siehe hier) und auch die entsprechenden Plattform-Voreinstellungen ausgefüllt haben. Sobald Sie das getan haben, können Sie einfach auf die Schaltfläche "Ausführbare Datei erstellen" in der IDE klicken, um den Compiler-Build zu starten, oder Sie wählen " Ausführbare Datei erstellen" aus dem Build-Menü. Beide Optionen öffnen ein Datei-Explorer-Fenster, in dem Sie den endgültigen Namen für die ausführbare Datei angeben können. den Sie für die ausführbare Datei Ihres Spiels verwenden möchten, bevor Sie auf Speichern klicken, um den Kompilier- und Erstellungsprozess zu starten. Sobald Sie dies getan haben, werden die erforderlichen Dateien generiert, so dass Sie das Spiel wie gewünscht verteilen können.

WICHTIG! Wenn Sie die kostenlose 30-Tage-Testlizenz besitzen, ist das Kompilieren einer endgültigen ausführbaren Datei nicht möglich, und Sie müssen auf eine der kostenpflichtigen Lizenzen upgraden, um Ihr Spiel für den Vertrieb zu kompilieren.

HINWEIS: Bevor Sie einen endgültigen Build Ihres Projekts für die Veröffentlichung durchführen, sollten Sie immer den Asset-Compiler-Cache löschen (über das "Besen"-Symbol oben in der IDE). GameMaker Studio 2 speichert viele Ihrer Spieldateien im Cache, um die Kompilierungsgeschwindigkeit auf ein Minimum zu reduzieren.

Jede Zieloption speichert in einem plattformspezifischen Format, das unten aufgeführt ist:

  • Windows - Beim Kompilieren für das allgemeine Windows-Betriebssystem werden Sie zunächst aufgefordert, zwischen der Erstellung eines Installers und eines Zip-Pakets zu wählen, wobei der Installer eine einzelne ausführbare Datei ist, die Ihr Spiel installiert, und die Zip-Datei ist eine einzelne komprimierte Datei im Format *.zip, in der alle Spieldateien gespeichert sind (die Dateien müssen extrahiert werden, damit das Spiel ausgeführt werden kann). Wenn Sie das Kontrollkästchen " Verpackungsoption merken " aktivieren, merkt sich GameMaker Studio 2 diese Wahl für alle zukünftigen Kompilierungen (dies kann in den Windows-Voreinstellungen zurückgesetzt oder geändert werden). Weitere Informationen finden Sie im YoYo Games Help Center. Bitte beachten Sie, dass das Windows-Target 32bit- oder 64bit-Executables kompilieren kann, je nachdem, was Sie in den den Allgemeinen Spieloptionen ausgewählt haben. Wir empfehlen dringend, dass alle neuen Projekte die x64-Runtime verwenden, um 64bit-Executables zu erstellen, da die 32bit-Runtime in Zukunft veraltet sein und entfernt werden wird entfernt wird.
  • Ubuntu (Linux ) - Ubuntu(Linux)-Apps können als *.zip -Dateien kompiliert werden - die entweder eine x64- oder eine ARM-Executable enthalten - je nachdem, auf welches Gerät sie ausgerichtet sind. Weitere Informationen finden Sie im YoYo Games Help Center.
  • HTML5 - Wenn Sie sich für HTML5 entschieden haben, wird eine Datei index.html (dies ist der Standardname, aber Sie können in den Plattformeinstellungen auch einen eigenen Namen vergeben) zusammen mit einem Ordner, der Ihre Spieldateien enthält erstellt und an dem angegebenen Speicherort gespeichert. Damit Ihr Spiel funktioniert, müssen diese beiden Dateien auf einen Server hochgeladen werden. Die Seite index.html kann auch angepasst werden um Ihr Spiel mit einer anderen Hintergrundfarbe oder an einer anderen Position usw. anzuzeigen. Dafür sind jedoch HTML-Kenntnisse erforderlich, und Sie können auch Ihre eigene benutzerdefinierte Indexdatei angeben, wenn Sie das Paket erstellen (siehe die HTML5-Spieloptionen). Mehr dazu erfahren Sie im YoYo Games Help Center.
  • Android - Für Android-Geräte können Sie wählen, ob Sie eine *.apk oder eine *.aab (Android App Bundle)-Datei aus dem Fenster erstellen, das zum Speichern des Spiels angezeigt wird:

    . Die Art der Datei, die Sie wählen, hängt von dem Store ab, auf den Sie abzielen möchten, wobei die *.aab -Datei für Google Play erforderlich ist, während die *.apk -Datei in anderen Stores verwendet werden kann. Weitere Informationen finden Sie im YoYo Games Help Center.
  • iOS - Beim Kompilieren für iOS wird eine xarchive -Datei erstellt, die dann in Xtools verwendet wird, um das endgültige iOS-Paket zu erstellen. Beachten Sie, dass Sie zum Kompilieren für iOS einen Apple Mac-Computer mit OSX oder höher sowie die entsprechenden Zertifikate und Berechtigungen benötigen. Weitere Informationen finden Sie im YoYo Games Help Center.
  • macOS - Beim Kompilieren für das macOS-Ziel werden Sie zunächst aufgefordert, zwischen der Erstellung einer DMG- oder einer Zip-Datei zu wählen:
    "Package as DMG" erstellt ein .dmg-Installationspaket, das zur Installation des Spiels auf einem Mac-Computer verwendet werden kann einem Mac-Computer installiert werden kann. Beachten Sie, dass während dieses Erstellungsprozesses der Installer auf dem Mac erscheint, der nach Abschluss des Erstellungsprozesses wieder verschwindet.

    "Package as Zip" erstellt entweder eine *.app -Datei oder eine *.pkg -Datei, je nachdem, ob Sie sie später in den Mac App Store hochladen möchten oder nicht. Wie bei iOS benötigen Sie einen Apple-Mac-Computer, auf dem OSX oder höher läuft, sowie die entsprechenden Zertifikate und Berechtigungen. Weitere Informationen finden Sie im YoYo Games Help Center.
  • Windows UWP - Für Windows UWP erstellt GameMaker Studio 2 ein *.appxbundle -Paket, das dann in den Windows Store hochgeladen oder anderweitig verteilt werden kann. Wenn Sie auf die Schaltfläche Ausführbare Datei erstellen klicken, werden Sie aufgefordert, GameMaker Studio 2 mitzuteilen, welche Art von Paket Sie erstellen möchten (kann für ARM-, x64- und/oder x86-Architekturen sein), und Sie sollten die Ziele auswählen, die (Sie müssen mindestens eines auswählen, können aber auch alle drei oder eine beliebige Kombination auswählen):

    Wenn Sie das Kontrollkästchen " Package For Store Upload" (Paket für den Store-Upload) aktivieren, wird als endgültiges Paket eine *.appxupload -Datei erstellt, die laut Microsoft für das Einreichen von Apps in ihrem Store verwendet werden sollte, wie in diesem Artikel hier erklärt (alle ausgewählten Architekturen werden enthalten, genau wie bei der *.appxbundle). Weitere Informationen zum Einrichten und Kompilieren für die UWP-Plattform finden Sie im YoYo Games Help Center.
