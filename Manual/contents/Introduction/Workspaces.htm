Arbeitsbereiche

Nachdem Sie sich angemeldet und ein neues Projekt gestartet haben, bringt Sie GameMaker Studio 2 zum anfänglichen Arbeitsbereich mit einigen grundlegenden Fenstern, die an die IDE angedockt sind. Im Allgemeinen ist der Arbeitsbereich einfach ein Bereich, in dem Sie die verschiedenen Assets für Ihr Spiel organisieren können, während Sie arbeiten:

Wie Sie sehen können, befindet sich der anfängliche Arbeitsbereich auf einer Registerkarte am oberen Rand des Bildschirms (und Sie können ihn umbenennen, indem Sie auf die Registerkarte doppelklicken), aber Sie können weitere Arbeitsbereiche für das Projekt erstellen, indem Sie auf die an der Seite klicken, wodurch Sie mehrere mögliche Arbeitsbereiche für jedes einzelne Projekt erhalten. Wenn Sie zum Beispiel an Interaktionen zwischen dem Spieler und mehreren gegnerischen Objekten, so hätten Sie den Spieler in einem eigenen Arbeitsbereich und die gegnerischen Objekte in einem anderen, und vielleicht einen weiteren Arbeitsbereich nur für die Skripte, die beide verwenden.

Eine weitere wichtige Funktion von Arbeitsbereichen ist, dass Sie auf klicken und - immer noch mit gedrückter Maustaste - die Registerkarte aus dem Haupt-IDE-Fenster in ein eigenes, individuelles Fenster ziehen können, was es sehr einfach macht, Dinge zu organisieren, wenn Sie - zum Beispiel - mehrere Bildschirme verwenden. Sie können auch diese sekundären Arbeitsbereich-Fenster wieder mit dem Hauptfenster zusammen, indem Sie die Registerkarte zurück auf das erste Fenster ziehen. Beachten Sie, dass Sie zwar zwei Instanzen der IDE laufen haben, wenn Sie dies tun, aber beide sind für dasselbe Projekt und Sie können nicht Sie können nicht ein Projekt in einem und ein anderes in dem anderen haben, es sei denn, Sie öffnen speziell zwei Instanzen von GameMaker Studio 2.

Wenn Sie GameMaker Studio 2 zum ersten Mal starten, wird Ihr Arbeitsbereich bereits von einer Reihe von Fenstern bevölkert, die an die IDE "angedockt " sind. Diese werden im Folgenden kurz erklärt:

  • Ausgabe - Im Bild oben sehen Sie auch das Ausgabefenster. Es gibt eine Reihe von Unterregisterkarten in diesem Fenster, die sich auf die Versionskontrolle, das Suchen und das Debuggen beziehen, Die erste Registerkarte ist für die Konsolen-/Compiler-Ausgabe, die Ihnen zeigt, was passiert, wenn Sie ein Spiel zum Testen kompilieren oder wenn Sie ein endgültiges Paket für die Verteilung erstellen. Hier werden auch alle Debug-Meldungen angezeigt, die Sie Diese Ausgabe zeigt auch alle Debug-Meldungen an, die Sie zur Laufzeit von Ihrem Projekt senden, und kann so konfiguriert werden, dass sie verschiedene Mengen an Informationen in den allgemeinen Einstellungen anzeigt. Wenn Sie das Ausgabefenster schließen Fenster schließen und es wiederherstellen möchten, können Sie das Windows-Menü verwenden.
  • Assets - Auf der rechten Seite des Bildschirms finden Sie die Asset-Browser. Hier können Sie die verschiedenen Assets, die Ihr Spiel verwendet, erstellen und bearbeiten, aber auch Konfigurationen, Spieloptionen, Raumanordnung und andere Dinge. Assets werden erstellt, indem Sie mit der rechten Maustaste auf einen Asset-Ordner (oder irgendwo in der Hauptansicht des Asset-Browsers) klicken und Erstellen oder durch Klicken auf das Plus-Symbol oben und Auswählen eines Assets im Fenster "Asset erstellen" (beachten Sie, dass Sie mit dieser Methode mehrere Assets desselben Typs auf einmal erstellen können, indem Sie den Wert den Wert am unteren Rand):

Andocken

Wir haben oben kurz erwähnt, dass Sie viele Fenster an das aktuelle Arbeitsbereich-Fenster andocken können und dass Sie die bereits angedockten Fenster wieder abkoppeln können. Wenn Sie z. B. oben auf auf die Registerkarte Asset-Browser klicken und sie nach links aus dem aktuellen Arbeitsbereich ziehen, werden Sie sehen, dass sie zu einem frei schwebenden Fenster wird.

Wenn Sie bestimmte Fenster an die Seiten des Arbeitsbereichs (oder an den Boden) ziehen, werden sie angedockt, was bedeutet, dass es im Wesentlichen Teil des IDE-Fenster-Overlays wird und nicht mehr unabhängig ist. Wenn Sie dies tun, ziehen Sie das Fenster auf den Bereich, an den Sie andocken möchten, und es wird hervorgehoben, um anzuzeigen, dass Sie das Fenster dort andocken können:

Auf diese Weise können Sie eine personalisierte IDE-Erfahrung schaffen, die zu Ihrem Arbeitsablauf und Ihrer Art, Dinge zu tun, passt. Es ist erwähnenswert, dass alle angedockten Alle angedockten Fenster können einzeln durch Anklicken der Schaltfläche neben den angedockten Objekten (im Bild oben auf dieser Seite hervorgehoben) versteckt/aufgelöst werden, oder Sie können alle angedockten Objekte mit Hilfe der Schnellschaltfläche oben in der IDE verstecken/auflösen, und beachten Sie auch, dass die IDE sich angedockte Fenster zwischen Sitzungen merkt.

HINWEIS: Einige Fenster verschwinden aus dem Dock, wenn Sie den Editor wechseln, z. B. im Raum-Editor. Wenn Sie jedoch zu dem betreffenden Editor zurückkehren, werden sie wieder im Dock geöffnet.
