Tutorial zum Erstellen eines Marktplatzes

Das Erstellen eines Tutorials ist ähnlich wie das Erstellen eines regulären Marketplace-Asset-Pakets, nur dass Sie jetzt bestimmte enthaltene Dateien hinzufügen sowie eine erste Tutorial-Seite (und optional eine Video-Seite) einfügen müssen. Der Arbeitsbereich des Tutorials sieht in etwa so aus:Wie oben für allgemeine Marketplace-Assets erklärt, müssen Sie einen Herausgeber auswählen und dann Ihrem Tutorial einen Namen und eine eindeutige ID (umgekehrtes URL-Format) geben, und dann im Dropdown-Menü "Typ" Tutorial auswählen. Daraufhin erhalten Sie zusätzliche Optionen, um eine Basis-HTML-Datei hinzuzufügen, von der aus das Tutorial gestartet wird, sowie sowie eine (optionale) Videodatei und URL. Die Videodatei sollte - wie auch der Rest des Tutorials - die vom Marketplace gelieferte Vorlage verwenden, die Sie hier erhalten können, und wenn der Benutzer das Tutorial lädt, wird das Video in einen separaten Frame innerhalb der IDE geladen. Beachten Sie, dass die Seiten, die Sie für die Startseite und die Videoseite angeben, relative Pfade innerhalb der Enthaltenen Dateien sind, und Sie können einen Explorer öffnen, um etwas anderes als die Standardseiten, mit denen GameMaker Studio 2 die Felder auffüllt.

Die eigentliche Erstellung Ihres Tutorials erfordert das Hinzufügen von eingeschlossenen Dateien in Ihr Tutorial-Projekt, und diese müssen auf eine ganz bestimmte Art und Weise hinzugefügt werden, damit die IDE richtig erkennt, welche Dateien für das Lernprogramm verwendet werden sollen. Wir haben also zwei Hauptordner:

  • TutorialInhalt: Alle Dateien Ihres Tutorials HTML, die CSS für das Styling und die dafür benötigten Bilder (nicht die Bilder, die für die Ressourcen benötigt werden, sondern die, die im Tutorial angezeigt werden sollen innerhalb des Tutorials), sowie die Video-Vorlage, falls erforderlich, kommen hierher. Sie können auch Unterordner haben, wenn Sie das Tutorial z. B. in Kapitel aufteilen wollen, aber Sie sollten mindestens eine Basisdatei HTML haben, die als Startseite für das Lernprogramm verwendet wird.
  • TutorialResources: In diesem Ordner befinden sich zwei weitere Unterordner für Bilder und Sound-Assets . Wenn Ihr Tutorial vom Benutzer verlangt, ein Sprite hinzuzufügen (z. B.), und Sie es in das Tutorial einbinden möchten dann würden Sie die Bildressource in den Ordner "Images" legen. Diese Ordner werden in das Tutorial-Projekt importiert, und wenn der Benutzer den Sprite-Editor öffnet und ein Bild hinzufügen möchte, wird der Standardpfad für das Explorer-Fenster zu diesem Ort führen, Dadurch wird es für den Benutzer einfacher, die benötigten Assets zu finden.

Damit Ihr Tutorial in der IDE funktioniert , müssen Sie diese Ordnerstruktur sowie eine Basis-Startseite HTML mit einer CSS Datei für das Styling haben. Es gibt eine Reihe von speziellen Links die in Ihren Lernprogramm-Projekten verwendet werden können, um zu Teilen eines Videos zu springen oder das Handbuch zu einem bestimmten Abschnitt oder einer Funktion zu öffnen, und diese sind alle in den offiziellen Lernprogramm-Vorlagen vom Marktplatz enthalten.

WICHTIG! Wir empfehlen Ihnen dringend, die Tutorial-Vorlagen aus dem Marketplace herunterzuladen, bevor Sie versuchen, eigene Tutorials zu erstellen. Sie enthalten Beispiele für alle wichtigen Teile des HTML sowie CSS und Bilder, um Schaltflächen und Menüs innerhalb des Tutorials nachzubilden, so dass Sie sich um den Inhalt kümmern können und nicht darum, wie er dargestellt wird.

Sobald Sie die enthaltenen Dateien eingerichtet und die erforderlichen Informationen angegeben haben, können Sie auf "Okay" klicken, und das Tutorial wird auf den Marktplatz hochgeladen, wobei nach Fertigstellung ein Arbeitsbereich in der Asset-Liste geöffnet wird, in dem Sie es bearbeiten und Symbole hinzufügen können, Text, Screenshots, etc. hinzufügen können. Sobald das Tutorial-Asset veröffentlicht wurde, können Benutzer es im Abschnitt Tutorials auf der Startseite finden, wo es heruntergeladen und installiert werden kann.
