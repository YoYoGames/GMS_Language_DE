Erstellen eines Marktplatz-Assets

Wenn Sie die Option " Create Package" (Paket erstellen) aus dem IDE-Marktplatzmenü auswählen, gelangen Sie in den folgenden Arbeitsbereich: Hier Sie müssen zuerst die Publisher Identität auswählen, unter der Sie das Asset-Paket veröffentlichen möchten (die Sie in Ihrem Publisher Dashboard eingerichtet haben sollten), dann einen Anzeigenamen und eine Versionsnummer angeben. Dann geben Sie einen Anzeigenamen und eine Versionsnummer an. Auf der Grundlage des Anzeigenamens und des verwendeten Publisher-Kontos wird dann eine Paket-ID für Sie erstellt, die Sie aber auch nach Belieben ändern können, solange sie das umgekehrte URL-Format beibehält. Sie müssen dann wählen den Typ des zu erstellenden Asset-Pakets wählen:

  • Asset: Dies ist eine Sammlung von Assets (Sprites, Skripte, Objekte, etc...), die Sie auf dem Marketplaceverkaufen möchten
  • Demo: Dies ist eine Demo zur Demonstration eines Konzepts oder einer Spielidee
  • Tutorial: Das hochgeladene Asset ist ein Lernprogramm, das für die Ausführung in der GameMaker Studio 2 IDE konzipiert ist (weitere Informationen finden Sie auf dieser Seite ).

Wenn Sie das getan haben, können Sie im rechten Bereich auswählen, welche Assets aus dem Projekt Sie in das Asset-Paket aufnehmen möchten. Sie können alle Ihre Assets oder nur einige davon hinzufügen und auch solche entfernen, die Sie versehentlich hinzugefügt haben. Es wird vorausgesetzt, dass Sie die hochzuladenden Assets zuvor in GameMaker Studio 2 erstellt haben und dass das aktuelle Projekt alle Assets enthält, die Sie hochladen möchten.

ERINNERN SIE SICH! Um Asset-Pakete erstellen oder verwenden zu können, müssen Sie zuvor ein Benutzerprofil-Zertifikat erstellt haben. Dies können Sie in den Marktplatz-Einstellungen tun, was hier erklärt wird.

Wenn Sie mit der Auswahl der Assets zufrieden sind und Ihr Paket auf den Marktplatz hochladen möchten, klicken Sie einfach auf die Schaltfläche " Okay" am unteren Rand, und das Asset-Paket wird erstellt und auf den Marktplatz hochgeladen. Wenn der Upload abgeschlossen ist, wird ein neuer Arbeitsbereich mit einer Browser-Ansicht der Asset-Seite geöffnet, die Sie bearbeiten können. Weitere Informationen zum Einrichten dieser Seite finden Sie auf der folgenden Seite:

  • Auflisten einer Anlage

Wenn Sie ein Marketplace-Asset aktualisieren müssen, gehen Sie mehr oder weniger genauso vor wie bei der Erstellung des ursprünglichen Pakets, nur dass Sie jetzt das zu aktualisierende Paket aus einer Dropdown-Liste der verfügbaren Pakete auswählen müssen:

Beim Aktualisieren können Sie die Paket-ID nicht ändern, und Sie sollten sich bewusst sein, dass die angezeigte Versionsnummer die aktuelle Versionsnummer ist und Sie diese gegebenenfalls erhöhen sollten, da sonst die Aktualisierung fehlschlägt (da Sie nicht zwei Asset-Pakete mit derselben Versionsnummer haben können). Es Es gibt auch ein Feld, in das Sie einen Text für das Änderungsprotokoll für das Asset eingeben können, der zusammen mit dem Asset hochgeladen und auf der Marketplace-Seite angezeigt wird.

WICHTIG! Assets, die mit GameMaker Studio Version 2.3+ erstellt wurden, sind nur für Benutzer verfügbar, die Version 2.3 und höher verwenden. Wenn Sie ein Asset aus einer früheren Version aktualisieren, ist dieses Asset weiterhin für Benutzer verfügbar, die nicht aktualisiert haben. die nicht aktualisiert haben, weiterhin zur Verfügung und wird mit der aktualisierten Version 2.3+ koexistieren.

Hochladen durch die Web-Seite

Sie können Assets über die eigentliche Marketplace-Webseite und nicht über die IDE hochladen, was notwendig sein kann, wenn der Upload über die IDE aus irgendeinem Grund fehlgeschlagen ist. Zu Beginn würden Sie das Paket erstellen und versuchen, es über die IDE hochzuladen. Wenn der Upload fehlgeschlagen ist, gehen Sie zum Ausgabefenster und suchen im Protokoll nach einer Zeile, die anzeigt, wo das erstellte Paket vor dem Uploadversuch gespeichert wurde. Zum Beispiel:

Auf Windows:

Erstellte Paketdatei:
C:\Benutzer\WINDOWSUSERNAME\AppData\Roaming\GameMakerStudio2\GMS2USERNAME\Marketplace\Packages\com.publishername.assetname.yymp

Auf macOS:

Paketdatei erstellt:
/Benutzer/MACUSERNAME/.config/GameMakerStudio2/GMS2USERNAME/Marketplace/Packages/com.publishername.assetname.yymp

Sie müssen zu diesem Dateispeicherort navigieren und die Datei *.YYMP an einen sicheren Ort kopieren, da dies die Datei ist, die Sie auf den Marketplace hochladen werden.

An diesem Punkt müssen Sie den Marketplace in Ihrem Webbrowser öffnen und zur Asset-Liste für das Asset navigieren und dann zum Abschnitt Versionen gehen und dann auf die Schaltfläche Version hochladen klicken, wie im Bild gezeigt unten gezeigt:

Dies führt Sie zur Upload-Seite für die neue Version, auf der Sie bei Bedarf Informationen über die hochzuladende Version hinzufügen können und auch die Datei bereitstellen können, indem Sie auf die Schaltfläche " Choose FIle" klicken und die zuvor erstellte Datei *.YYMP auswählen:

Beim Hinzufügen der Datei werden die Versionen hochgeladen und ein "Fortschrittsbalken" angezeigt, und wenn das Hochladen abgeschlossen ist und Sie mit den Details zufrieden sind, können Sie auf die Schaltfläche "Speichern" klicken, um das Asset zu speichern. Wenn der Upload abgeschlossen ist und Sie mit den Details zufrieden sind, können Sie auf die Schaltfläche Save klicken, um das Asset in der Liste zu speichern. Beachten Sie, dass sich das Asset einige Minuten lang in der Bearbeitungsphase befindet, bevor es veröffentlicht werden kann.
