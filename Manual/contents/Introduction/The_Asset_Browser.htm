Der Asset-Browser

Der Kern Ihres Spiels wird aus Assets erstellt, die dem Asset-Browser hinzugefügt werden, der sich standardmäßig auf der rechten Seite der IDE befindet. Hier können Sie alles hinzufügen, was Ihr Spiel benötigt um zu laufen, einschließlich eines Spielraums, Sprites, Objekte, Pfade und eine Reihe anderer Dinge. Ein grundlegendes Spiel in GameMaker Studio 2 benötigt einen Raum, in dem es läuft (neue Projekte werden immer mit einer (neue Projekte werden immer mit einem bereits erstellten Raum-Asset erstellt), und normalerweise mindestens ein Objekt und ein Sprite, obwohl Sie wahrscheinlich viel mehr verwenden werden!

Im Asset-Browser können Sie die Assets für Ihr Spiel erstellen und organisieren sowie bestimmte andere Aspekte wie plattformspezifische Einstellungen und Konfigurationen ändern. Nachfolgend finden Sie Erklärungen zu den einzelnen Abschnitten.

Suchleiste

In der Suchleiste können Sie einen Text eingeben und damit filtern, was in der Haupt-Kühlstellenliste unten angezeigt wird. Wenn Sie zum Beispiel "room" eingeben, werden alle Kühlstellen angezeigt, die "room" in ihrem Namen enthalten. Zum Zurücksetzen der Kühlstellenliste auf die Standardansicht zurückzusetzen, löschen Sie einfach den Text aus der Suchleiste.

Es ist wichtig zu beachten, dass Assets, die erstellt werden, während Tags Teil des aktiven Filters sind , automatisch mit diesen Tags versehen werden. Das bedeutet, dass, wenn Sie beispielsweise alle Assets unter dem Tag "Level 1" betrachten, alle Das bedeutet, wenn Sie beispielsweise alle Assets unter dem Tag "Level 1" anzeigen, werden alle neuen Assets, die mit diesem Filter erstellt werden, bei der Erstellung ebenfalls mit dem Tag "Level 1" versehen.

Animation Kurven

Eine Animationskurve ist eine Ressource, die eine oder mehrere Kurven enthält, die darstellen, wie sich ein Wert im Laufe der Zeit ändert, und kann eine lineare Interpolation oder eine Katmull-rom-Interpolation verwenden, um zwischen den Werten in der Kurve zu wechseln. Die Werte, die Sie einstellen, können zwischen -1 und 1 auf der vertikalen Achse liegen und die Dauer entlang der horizontalen Achse ist immer von 0 bis 1 normalisiert, was es einfach macht, verschiedene Zeitbereiche mit Hilfe von Multiplikatoren in Ihrem Spielcode. Animationskurven bestehen aus "Kanälen" und jeder Kanal kann seine eigenen Kurveneinstellungen haben, was es Ihnen erlaubt, z. B. eine räumliche Position mit zwei Kanälen für die x/y-Position oder einen Farbverlauf mit 4 Kanälen zur Darstellung eines Farbformats zu beschreiben. Animationskurven-Assets können bei der Arbeit mit Sequenzen verwendet werden und sind auch über Code zugänglich, was sie zu einem mächtigen Werkzeug bei der Erstellung Ihrer Spiele macht.

Weitere Informationen zum Hinzufügen von Animationskurven finden Sie im Abschnitt über Animationskurven.

Objekte

Objekte sind eine besondere Ressource, die wir verwenden, um Aspekte eines Spiels zu steuern und bestimmte Dinge zu tun. Meistens sind ihnen Sprites zugeordnet, so dass man sie im Spielraum sehen kann. Raum zu sehen sind, aber manchmal werden sie auch als Controller "hinter den Kulissen" verwendet, um Dinge in Bezug auf den Benutzer oder für das Timing usw. zu tun. Man kann ihnen Verhaltensweisen zuweisen und sie können auf bestimmte Ereignisse sowie auf einander reagieren, und die meisten der Dinge die Sie in einem Spiel sehen, basieren auf Objekten und deren Interaktionen. Beachten Sie, dass wir "basierend auf" sagen, weil Sie eigentlich keine Objekte direkt in den Spielraum platzieren, sondern Instanzen dieser Objekte, die die im Grunde genommen Kopien (oder Klone, wenn Sie so wollen) der Objektressource sind. Dies ist eine sehr wichtige Sache, die Sie sich merken sollten, da Instanzen und Objekte nicht dasselbe sind und beide ihre eigene Reihe von Funktionen haben, die sie beeinflussen können.

Alle Objekte haben eine Reihe von Eigenschaften, die Sie im Objekteditor festlegen können, wie z. B. das Sprite, ob es Physik verwendet oder nicht, oder ob es ein "Kind"-Objekt eines anderen ist. Außerdem haben sie eine Reihe von eingebauten Variablen, die dann in Aktionen, Skripten und Code verwendet werden können. Diese eingebauten Variablen werden verwendet, um die Position der Instanz zu definieren, wenn sie im Spielraum platziert wird, die Animationsgeschwindigkeit, die Richtung der Bewegung und eine ganze Reihe anderer Dinge. Um zu steuern, wie sich das Objekt im Laufe der Zeit verhält, wird jeder Spiel-Frame in eine Reihe von Ereignissen aufgeteilt, so dass Sie Code oder Aktionen in ein bestimmtes Ereignis einfügen können, das nur ausgeführt wird, wenn das Ereignis ausgelöst wird. Ereignisse können durch Dinge wie das Drücken von Maustasten oder Kollisionen mit anderen Instanzen ausgelöst werden.

Objekte sind im wahrsten Sinne des Wortes die Bausteine Ihres Spiels und bilden zusammen mit den Räumen den Kern eines jeden Projekts, das Sie erstellen.

Das Menü RMB für Objekte enthält die folgende Option (sowie die weiter oben auf dieser Seite aufgeführten):

  • Alle Ereignisskripte öffnen - Wenn Sie diese Option wählen, werden alle zugewiesenen Ereignisse für das Objekt in einem neuen Arbeitsbereich als Skriptregisterkarten geöffnet.

Weitere Informationen zum Hinzufügen von Objekten finden Sie im Abschnitt über den Objekteditor.

Pfade

Irgendwann bei der Erstellung Ihrer Spiele benötigen Sie vielleicht eine Instanz eines Ihrer Objekte, um einem Pfad durch einen Level zu folgen. Das kann man tun, indem man einfach ein Array von Positionen im Raum erstellt und dann die Instanz zwischen diesen Positionen hin- und herbewegen lässt, aber das kann eine lästige Aufgabe sein und ist schwierig zu ändern, zu testen und für verschiedene Dinge anzupassen. In diesem Fall sollten Sie eine Pfadressource erstellen.

Die Grundidee der Pfad-Ressource ist recht einfach - Sie definieren einen Pfad, indem Sie ihn im Pfad-Editor zeichnen, dann können Sie eine Aktion (oder einen Code) in einem Ereignis eines Objekts platzieren, um das Objekt anzuweisen, diesem bestimmten Pfad im Spielraum zu folgen. Sie können Sie können die Geschwindigkeit festlegen, mit der das Objekt dem Pfad folgt, und auch eine Reihe anderer Aktionen, die sich auf die Position und die Ausrichtung des Pfades im Raum beziehen.

Weitere Informationen zum Hinzufügen von Pfaden finden Sie im Abschnitt über den Pfad-Editor.

Räume

Alle Spiele, die Sie in GameMaker Studio 2 erstellen, benötigen mindestens einen Raum (es können aber auch viele, viele mehr sein), und ein Raum ist im Grunde ein Bereich, in dem Sie Instanzen der Objekte platzieren, aus denen Ihr Spiel besteht und in dem sich die Handlung des Spiels abspielt stattfindet. Der Raum-Editor ist jedoch eines der mächtigsten Hilfsmittel, die Ihnen zur Verfügung stehen, da er Ihnen nicht nur erlaubt, Instanzen von Objekten hinzuzufügen, sondern auch den "Blick" in den Spielraum einzurichten, die Hintergründe und Kacheln hinzuzufügen, die gezeichnet werden um die Spielwelt zu erstellen, zusätzliche Spezialeffekte zu erzeugen, die unabhängig vom Objekt/Instanz-System sind, und auch Code unabhängig von einem Objekt auszuführen. Räume können auch so eingestellt werden, dass sie Eigenschaften von anderen Räumen erben , was bedeutet dass Sie z. B. einen einzelnen Raum mit einer Reihe von Kacheln erstellen und dann einen weiteren Raum erstellen können, der diese Kacheln erbt, so dass Sie sie nicht noch einmal neu erstellen müssen.

Das Menü RMB für Zimmer enthält die folgende Option (sowie die weiter oben auf dieser Seite aufgeführten Optionen):

  • Create Child Room (Untergeordneten Raum erstellen ) - Damit wird ein neuer Raum als untergeordneter Raum des ausgewählten Raums erstellt. Untergeordnete Räume sind neue Raum-Assets, die die Eigenschaften, Ebenen und Ebeneninhalte des übergeordneten Raums erben und im Raum-Manager als mit dem übergeordneten Raum verknüpft angezeigt werden. Raum im Raum-Manager angezeigt.

Weitere Informationen zum Hinzufügen von Räumen finden Sie im Abschnitt über den Raumeditor.

Skripte

Eine Skript-Ressource ist einfach ein Container für eine oder mehrere Funktionen, die Sie geschrieben haben. Nehmen wir an, Sie möchten, dass alle Ihre gegnerischen Objekte eine Reihe von verschiedenen Instanzen erzeugen, wenn sie sterben - wie etwas Gold, eine Explosion und etwas Blut sowie sowie einen Sound abspielen. Nun könnten Sie den gesamten erforderlichen Code in jedes Objekt einfügen, das ihn benötigt, aber im Allgemeinen ist es besser, eine einzelne Funktion in der Skriptressource zu erstellen und diese dann aufzurufen. Das bedeutet, dass Sie nur eine Zeile Code in die Objekte einfügen müssen Das bedeutet, dass Sie nur eine Zeile Code in die Objekte einfügen müssen, um diese neue Skriptfunktion aufzurufen, und wenn Sie etwas ändern möchten, brauchen Sie es nur einmal im Skript zu ändern, und alle Objekte führen automatisch den neuen Code aus, anstatt durch jedes Objekt zu gehen und die gleiche Sache an mehreren Stellen ändern.

Das Menü RMB für Skripte enthält die folgende Option (sowie die weiter oben auf dieser Seite aufgeführten Optionen):

  • In DnD™/ GML konvertieren - Damit wird das ausgewählte Skript in DnD™oder GML konvertiert. Wenn das Skript GML ist, wird es innerhalb einer Aktion " Code ausführen" platziert, während DnD™direkt in GML-Code übersetzt wird (wie im Abschnitt DnD™ in Code umwandeln gezeigt).

Weitere Informationen zum Hinzufügen von Skripten finden Sie im Abschnitt über den Skript-Editor.

Sequenzen

Ein Sequenz-Asset ist im Wesentlichen eine Sammlung von anderen Ressourcen, die eine dynamische Animation im Laufe der Zeit ausführen. Sie können Sprites, Instanzen, Sounds und andere Dinge enthalten, und jedes dieser Elemente kann so eingestellt werden, dass es sich bewegt, die Farbe ändert oder mit der Zeit startet/stoppt innerhalb der Sequenz selbst, indem die Assets bestimmten Spuren innerhalb der Sequenz zugewiesen werden. Auf diese Spuren können dann verschiedene Attribute angewendet werden, die statisch sein oder sich mit der Zeit ändern können. Die gesamte Bearbeitung einer Sequenz erfolgt Die gesamte Bearbeitung einer Sequenz findet im Sequenzeditor statt, der aus folgenden Elementen besteht:

  • die Leinwand - wo Sie die Assets platzieren, aus denen die Sequenz besteht
  • das Spur-Bedienfeld - hier können Sie Assets platzieren, um Asset-Spuren zu erstellen, sowie Parameter-Spuren erstellen, um die Assets im Laufe der Zeit zu manipulieren
  • Dope-Blatt - in dem Sie die Keyframes der einzelnen Spuren hinzufügen/entfernen/bearbeiten und steuern, wie sie sich im Laufe der Zeit verhalten.

Beachten Sie, dass Sie Sequenzen innerhalb von Sequenzen haben können, um komplexe Animationen und Effekte zu erstellen, und dass Sequenzen auch über Code aufgerufen und geändert werden können, um Ihnen so viel Kontrolle wie möglich darüber zu geben, wie sie in Ihrem Spiel angezeigt und verwendet werden.

Weitere Informationen zum Hinzufügen von Sequenzen finden Sie im Abschnitt über den Sequences Editor.

Shader

Shader sind ein sehr leistungsfähiges Werkzeug, mit dem die Grafiken, die Ihr Spiel auf den Bildschirm rendert, manipuliert werden können, wodurch unglaublich schnelle Effekte möglich sind, die z. B. vom Hinzufügen eines subtilen Farbtons zu einem Sprite bis hin zu bildschirmfüllenden Verzerrungseffekte. Im Grunde handelt es sich um ein zweiteiliges Programm, das direkt auf der Grafikkarte selbst läuft, was es sehr schnell macht, da die GPU die ganze Arbeit erledigt und CPU-Zyklen für Ihren Spielcode freigibt. Der vollständige Shader besteht aus einem Vertex Shader-Programm und einem Fragment-Shader-Programm (auch bekannt als Pixel-Shader). Diese beiden kleinen Programme arbeiten zusammen, um zu manipulieren, was die Grafikkarte auf den Bildschirm rendert. Dies erlaubt Ihnen dann, in Echtzeit die Position, die Farbe und die Alphawerte zu manipulieren, die tatsächlich auf dem Bildschirm dargestellt werden, Farb- und Alphawerte, die tatsächlich in den Anzeigepuffer gerendert werden, in Echtzeit zu manipulieren.

GLSL Mac und Ubuntu(Linux)
HLSL11 Windows, UWP, XboxOne
PSSL Playstation 4

  • Shader Type - Hier können Sie den Typ der Shader-Sprache einstellen, die für die Shader-Ressource verwendet wird.

Klingt

Sound ist ein wichtiger Bestandteil eines jeden Spiels, sowohl um dem Gameplay Tiefe zu verleihen und Feedback in Form von Soundeffekten zu geben, als auch um Atmosphäre in Form von Musik hinzuzufügen. GameMaker Studio 2 akzeptiert WAV, MP3 und OGG Formatdateien.

Im Allgemeinen werden WAV -Dateien für kurze Soundeffekte verwendet, da sie trotz ihrer Größe sofort abgespielt werden können, da sie nicht dekodiert werden müssen. MP3 und OGG sollten für Hintergrundmusik oder Effekte verwendet werden, die über einen längeren Zeitraum abgespielt werden oder eine große Dateigröße haben. Diese Dateien sind viel kleiner als eine WAV-Datei, haben aber einen CPU-Overhead, da sie vor dem Abspielen dekodiert werden müssen.

Das Menü RMB für Sounds enthält die folgenden Optionen (sowie die weiter oben auf dieser Seite aufgeführten):

  • Im Soundmixer öffnen - Damit werden alle ausgewählten Sounds im Soundmixer geöffnet.
  • Audiogruppe zuweisen - Wenn Sie diese Option wählen, wird eine Liste der verfügbaren Audiogruppen angezeigt, und Sie können dann eine auswählen, der Sie den Ton hinzufügen möchten, oder bei Bedarf eine neue Gruppe erstellen. Wenn mehrere Sounds ausgewählt sind im Asset-Browser ausgewählt sind - oder Sie einen Gruppenordner mit Sounds ausgewählt haben -, werden alle Sounds der gewählten Audiogruppe hinzugefügt. Weitere Informationen zu Audiogruppen finden Sie hier.

Weitere Informationen zum Hinzufügen von Sounds finden Sie im Abschnitt über den Sound-Editor.

Kacheln

Kachelsätze werden von den Sprite-Ressourcen übernommen, aber als separate Ressource klassifiziert, da GameMaker Studio 2 sie bei der Erstellung Ihres Spiels anders behandelt. Im Grunde ist ein Kachelsatz ein einzelnes Bild, das GameMaker Studio 2 in separate Abschnitte aufteilt, basierend auf den Werten, die Sie für die verschiedenen Einstellungen angeben. Diese können Sie dann im Raumeditor (oder prozedural durch Code) verwenden, um eine Kachelkarte in Ihrem Raum zu erzeugen. Sie eignen sich hervorragend für die Gestaltung aller statischen Elemente in Ihrem Raum, wie Terrain, Wände, Kulissen usw., da sie nicht den gleichen Overhead wie Objekte haben.

Oben sehen Sie ein Beispiel-Sprite, das als Kachelsatz zum Erstellen von Wänden verwendet werden würde. Wie Sie sehen können, kann es in 64x64-Pixel-Stücke "aufgeteilt" werden, die die dann in den Raum-Editor eingefügt würden.

HINWEIS: Kacheln sind immer quadratisch, wenn Sie also etwas anderes benötigen, sollten Sie eine Instanz mit einem Sprite oder eine Asset-Ebene im Raumeditor verwenden.

Wenn mehrere Tilesets im Asset-Browser ausgewählt sind - oder Sie einen Gruppenordner mit Tilesets ausgewählt haben -, werden alle diese Tilesets der gewählten Texturgruppe hinzugefügt. Zeitleisten

In den meisten Spielen werden Sie wollen, dass bestimmte Dinge zu bestimmten Zeitpunkten passieren. Nun können Sie versuchen, dies mit Hilfe der Alarm-Ereignisse in einer Instanz zu erreichen, aber wenn die Dinge zu kompliziert werden, wird dies nicht mehr funktionieren, zumal Sie auf nur zwölf Alarme beschränkt sind. Deshalb gibt es die Ressource Zeitleiste. In einer Zeitleiste legen Sie fest, welche Aktionen zu einem bestimmten Zeitpunkt in der Spielzeit passieren müssen, und Sie können alle Aktionen, die für ein Objekt verfügbar sind, in seinen verschiedenen Ereignissen als auch im Code. Sobald Sie eine Zeitleiste erstellt haben, können Sie diese dann einem Objekt zuweisen, und die Instanz dieses Objekts führt dann die Aktionen und den Code zu den angegebenen Zeitpunkten aus, wenn sie in einem Raum platziert oder erstellt wird.

Weitere Informationen zum Hinzufügen von Zeitleisten finden Sie im Abschnitt über den Zeitleisten-Editor.
