Laufzeit-Funktionen

Die allgemeine Definition einer Funktion ist etwa so:

Eine Funktion hat einen Eingang und einen Ausgang, und der Ausgang ist irgendwie mit dem Eingang verbunden.

In GameMaker Studio 2 bedeutet dies zwei verschiedene Dinge, die aber beide auf die gleiche Weise funktionieren:

  • Laufzeitfunktionen - Eine runtime Funktion ist eine Funktion, die als Teil des GameMaker-Sprache(GML). Diese sind in die Sprache eingebaut und können verwendet werden, um Dinge in Ihren Spielen zu bewirken, und es gibt eine große Anzahl von GML Laufzeitfunktionen zur Verfügung, die alle im Abschnitt " Sprachreferenz " des Handbuchs erläutert werden.
  • Skriptfunktionen und Methoden - Skript Funktionen und Methoden-Variablen sind Funktionen, die Sie selbst erstellt haben, indem Sie die Grundbausteine Bausteine des GameMaker-Sprache(GML) erstellt haben, und diese können als Teil ihres Codes Laufzeitfunktionen enthalten.

In beiden obigen Fällen hat eine Funktion die Form eines Funktionsnamens, gefolgt von den Eingangsargumenten zwischen Klammern () und getrennt durch Kommata getrennt sind (wenn die Funktion keine Eingabeargumente hat, werden nur Klammern verwendet). Diese Seite konzentriert sich auf die Runtime-Funktionen (d. h. die Funktionen, die in GameMaker eingebaut sind und die GameMaker-Sprache bilden), aber die allgemeinen Regeln gelten für alle Funktionstypen.

Hier sehen Sie einen Überblick über den Aufbau einer Funktion:

<Funktion>(<arg0>, <arg1> ,... <argN>);

Einige Funktionen können Werte zurückgeben und in Ausdrückenverwendet werden, während andere einfach nur Befehle ausführen, wie in den folgenden zwei Laufzeitfunktionen gezeigt wird Beispiele:

// Zerstört die aufrufende Instanz
// Dies erfordert keine Argumente und gibt nichts zurück
instance_destroy();

// Ermittelt den Abstand von der aktuellen Instanzposition zur Mausposition
// Dies benötigt vier Argumente und gibt einen reellen Wert zurück
dist = point_distance(x, y, mouse_x, mouse_y);

HINWEIS: Überall dort, wo im Handbuch "N/A" als Rückgabewert für eine Funktion angegeben ist, bedeutet dies, dass die Funktion keinen Wert als Teil ihrer Operation zurückgeben soll. Wenn Sie versuchen, nach einem Rückgabewert dieser Funktionen zu suchen, erhalten Sie erhalten Sie den Wert undefined.

Manchmal kann der Rückgabewert einer Funktion ein Wert sein, den Sie in einer Zuweisung verwenden möchten, aber Sie sollten beachten, dass es nicht möglich ist, jede Funktion direkt als linke Seite einer Zuweisung zu verwenden. Zum Beispiel würde der folgende Code zum Einstellen der Geschwindigkeit einer Instanz einen Fehler ergeben:

instance_nearest(x, y, obj).speed = 0;

Der Rückgabewert für den Ausdruck in diesem Codebeispiel ist eine Ganzzahl (der eindeutige ID-Wert für die nächste Instanz) und muss daher in Klammern eingeschlossen werden, um auf diese Weise verwendet zu werden und die erforderliche Instanz korrekt zu adressieren (weitere Informationen finden Sie unter Adressieren von Variablen in anderen Instanzen ). Der obige Code würde korrekt geschrieben werden als:

(instance_nearest(x, y, obj)).speed = 0;

/oder richtiger noch

var inst = instance_nearest(x, y, obj);
inst.speed = 0;

Der Abschnitt " Sprachreferenz" des Handbuchs listet alle Ihnen zur Verfügung stehenden Laufzeitfunktionen auf und gibt Beispiele für deren Verwendung sowie weitere wichtige Informationen darüber, was sie zurückgeben können oder Ereignisse, die sie auslösen können, etc. Weitere Informationen zu benutzerdefinierten Skriptfunktionen und Methoden finden Sie im Abschnitt über Skriptfunktionen und im Abschnitt über Methodenvariablen.
