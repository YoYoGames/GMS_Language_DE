Instanz-Variablen

Eine Instanzvariable wird innerhalb einer Instanz eines Objekts erstellt und gilt als eindeutig für diese Instanz - d. h.: viele Instanzen desselben Objekts können dieselbe Variable haben, aber jede Variable kann einen anderen Wert enthalten, da sie für jede Instanz eindeutig ist jede Instanz. Aber wie wird eine Instanzvariable erstellt? Nun, Sie erstellen neue Variablen, indem Sie ihnen einfach einen Wert zuweisen, wie in diesem kleinen Beispiel gezeigt (dies wird als Deklarieren der Variable bezeichnet):

Tränke = 12;
Leben = 100;
Name = "Jock MacSweeney";
Stärke = 5.5;
Rüstung = -2;

Wie Sie sehen, müssen Sie nur den Namen und dann einen Wert angeben (der Wert kann ein beliebiger Datentyp sein), um diese Variable zu setzen und sie für die Verwendung innerhalb einer Instanz des Objekts, für das Sie programmieren, bereit zu haben (beachten Sie, dass der Wert aus dem Rückgabewert einer Funktion oder dem Ergebnis einer Operation zwischen anderen Variablen usw. stammen kann...). Diese Variablen können dann auf verschiedene Weise innerhalb der Instanz verwendet und verändert werden, zum Beispiel könnte dieser Code in einem Kollisionsereignis und verwendet werden, um einen Betrag von der Variablen "life" zu nehmen:

Leben -= 5 + Panzerung;

Wenn "life" auf 100 steht, hat es jetzt einen Wert von 97 (100 - (5 + -2) = 97). Nun, das ist ein einfaches Beispiel, und Sie könnten "armour" durch den tatsächlichen Wert von -2 ersetzen, aber was passiert, wenn dieser Wert an mehreren Stellen hart kodiert ist und Sie sich dann entscheiden, ihn zu ändern? Sie müssten dann durch Ihren gesamten Code gehen und jede -2 in den neuen Wert ändern. ändern, was zeitaufwendig und sehr fehleranfällig ist! Wenn Sie aber eine Variable verwenden, müssen Sie ihr nur einen neuen Wert zuweisen, und der Code wird von da an automatisch diesen neuen Wert verwenden, was die Dinge viel flexibler und viel einfacher zu beheben macht sollte es ein Problem geben. Es sollte auch beachtet werden, dass selbst wenn sich ein Wert nicht ändert, es viel einfacher ist, sich zu merken, was eine Variable mit dem Namen "life" bedeutet, als nur auf eine Zahl zu schauen.

GameMaker Studio 2 hat auch eine Sammlung von "eingebauten" Instanzvariablen, die Sie kennen sollten, da Sie vielleicht eine Ihrer eigenen Instanzvariablen gleich benennen oder eine eigene globale Variable mit demselben Namen haben wollen und sich wundern, warum Sie Fehler erhalten. Sie sind jedoch leicht zu erkennen, da sie im Code-Editor in einer anderen Farbe angezeigt werden und auch in der Autovervollständigung auftauchen und in der Leiste am unteren Rand des Code-Editors angezeigt werden.
