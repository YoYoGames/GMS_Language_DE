steam_get_quota_total

Wenn Sie die Steam-Cloud zum Speichern und Synchronisieren von Dateien verwenden, müssen Sie das Kontingent an Speicherplatz einrichten, das Ihr Spiel benötigt. Dieses Kontingent wird für jedes Cloud-aktivierte Spiel pro Benutzer und Spiel erzwungen, d. h. wenn das Kontingent für Spiel X 1 Megabyte beträgt, kann jedes Steam-Konto, das Spiel X besitzt, maximal 1 Megabyte der mit diesem Spiel verbundenen Daten in der Cloud speichern. Alle anderen Cloud-fähigen Spiele, die der Benutzer besitzt (z. B. Spiel Y), sind nicht von den Daten betroffen, die von Spiel X gespeichert werden. Die Standardquote für neue Steamworks-Spielebeträgt ein Gigabyte, aber Sie können dies über das Steamworks-Bedienfeldfür Ihr Spiel ändern.

HINWEIS: Sobald das Kontingent erschöpft ist , schlägt das Schreiben von Dateien fehl. Wenn Sie es für möglich halten, dass das Kontingent für den Benutzer Ihres Spiels erschöpft sein könnte, sollten Sie Code erstellen, um damit umzugehen, denn wenn Sie nichts tun, lassen Sie die Benutzer in einer Situation, in der sie nicht in der Lage sind, Dinge zu reparieren, und das wird zu einer schlechten Erfahrung mit Ihrem Spiel führen.

Syntax:

steam_get_quota_total();

Rückgabe:

Real

Beispiel:

if steam_is_cloud_enabled_for_app()
{
quota = steam_get_quota_total();
}

Der obige Code prüft, ob die Steam-Cloud für das Spiel aktiviert ist, und wenn ja, ermittelt er die Größe des Speicherkontingents und speichert sie in einer Variablen.
