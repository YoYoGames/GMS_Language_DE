steam_ugc_request_item_details

Diese Funktion kann verwendet werden, um Informationen über eine gegebene Datei-ID abzurufen. Sie geben die Datei-ID an und geben ein maximales Alter für die Überprüfung an (weitere Informationen finden Sie in den Steam-Dokumenten ), dann gibt die Funktion einen eindeutigen Async-ID-Wert zurück, der verwendet werden kann, um die die Details korrekt im Steam-Async-Ereignis. Dieses Ereignis wird die async_load DS-Map, die mit den folgenden Schlüssel/Wert-Paaren bestückt wird:

  • "id" - Die asynchrone ID, die von der Funktion zurückgegeben wird, die das Ereignis ausgelöst hat
  • "result" - Das Ergebnis der Operation (ein reeller Wert). Dies wird entweder die GML-Konstante ugc_result_success oder eine andere reelle Zahl sein. Sie sollten also auf diese Konstante prüfen, um sicherzustellen, dass der Aufruf erfolgreich war, und wenn andernfalls irgendetwas nicht korrekt funktioniert hat. Der Rest der möglichen Werte, die zurückgegeben werden, wird als Ergebnis des Steam- "EResult"-Wertes angezeigt, und Sie sollten in der Datei steamclientpublic .h in den SDK-Headern alle 89 möglichen Werte finden.
  • " event_type" - Die Zeichenfolge " ugc_item_details"
  • "cached_data" - Wird true, wenn die zurückgegebenen Details aus dem lokalen Cache stammen, oder false, wenn sie vom Server genommen werden
  • "published_file_id" " -Hält die eindeutige veröffentlichte Datei-ID für das Element
  • " file_type" - Der Typ der verwendeten Datei
  • "creator_app_id" - Die Steam-ID des Artikelerstellers
  • "consumer_app_id" - Die Steam-ID des Artikelkonsumenten
  • "Titel" - Der Titel des Elements
  • "description" - Die Beschreibung des Artikels
  • "steam_id_owner" - Die Steam-ID des Eigentümers des Artikels
  • "time_created" - Die Zeit, zu der das Element erstmals erstellt wurde
  • "time_updated" - Das letzte Mal, dass das Element aktualisiert wurde
  • "time_added_to_user_list" - Die Zeit, zu der das Element abonniert wurde
  • "visibility" - Die Sichtbarkeit des Elements (siehe steam_ugc_set_item_visibility() für die zurückgegebenen Konstanten)
  • " banned" - Ob das Element gesperrt wurde (true) odernicht (false)
  • "accepted_for_use" - Ob das Element zur Verwendung akzeptiert wurde (true) odernicht (false)
  • " tags_truncated" - Kurzversion der Tags als Array
  • "Tags"- Ein Array mit den Tags für das Element
  • "handle_file" - Das eindeutige Dateihandle für das Element
  • "handle_preview_file" - Das eindeutige Handle für die Bildvorschau für den Artikel (kann als Argument mit steam_ugc_download() verwendet werden, um ein Vorschaubild herunterzuladen)
  • "Dateiname" - Der Name der Artikeldatei
  • "file_size" - Die Größe der Artikeldatei
  • "preview_file_size" - Die Größe des Vorschaubildes
  • "url"- Die vollständige URL für das Element
  • "votes_up" - Die Anzahl der erhaltenen Up-Votes
  • "votes_down" - Die Anzahl der erhaltenen Down-Votes
  • " score" - Die Gesamtbewertung des Elements
  • "account_id_owner" - Die Konto-ID des Steam-ID-Besitzers (diese kann in der Funktion steam_ugc_create_query_user_ex())

Syntax:

steam_ugc_request_item_details(published_file_id, max_age_seconds);

Argument Beschreibung
veröffentlichte_datei_id Die eindeutige Datei-ID für die zu prüfende UGC.
max_alter_sekunden Das "Alter" der zu prüfenden Daten (empfohlen 30 - 60 Sekunden).

Rückgabe:

Async-ID

Erweitertes Beispiel:

In diesem Beispiel senden wir eine Detailanforderung für ein Element ab und parsen dann die resultierende async_load DS-Map, um einige Variablen zu setzen. Zuerst senden wir die Anfrage ab:

steam_details= steam_ugc_request_item_details(global.fileID, 60);

Der obige Code fordert Details zum Element mit der in der globalen Variablen gespeicherten Datei-ID an und löst ein Steam Async-Ereignis mit den zurückgegebenen Informationen aus. In diesem Ereignis können wir dann die Karte parsen und einige der Werte in Variablen speichern, die dann verwendet werden können, um die Informationen für den Benutzer anzuzeigen:

var map_id = async_load[? "id"];
var result = async_load[? "result"];
if (map_id == steam_details) && ( result== ugc_result_success)
{
mTitle = async_load[? "title"];
mDesc = async_load[? "description"];
mTags = async_load[? "tags"];
m_hPreviewFile = async_load[? "handle_preview_file"];
m_hOwnerSteamId = async_load[? "steam_id_owner"];
mOwnerAccountId = async_load[? "account_id_owner"];
mPubFileId = async_load[? "published_file_id"];
mScore = async_load[? "score"];
}
