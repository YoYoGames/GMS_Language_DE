steam_ugc_get_item_update_info

Mit dieser Funktion können Sie Informationen über den aktuellen Download-Status für die angegebene Datei-ID abrufen. Sie geben die Element-ID an und liefern den Index einer leeren DS-Map, die dann mit folgenden Schlüssel/Wertpaaren gefüllt wird mit den folgenden Schlüssel/Wertpaaren gefüllt wird, wenn das Element existiert:

  • "needs_update" - Gibt true oder false zurück , je nachdem, ob das Element eine Aktualisierung benötigt oder nicht.
  • "is_downloading" - Gibt true oder false zurück , je nachdem, ob das Element gerade heruntergeladen wird oder nicht.
  • "bytes_downloaded" - Die Anzahl der Bytes, die heruntergeladen wurden.
  • "bytes_total" - Die Gesamtgröße (Anzahl der Bytes), die für das Element auf der Festplatte benötigt wird.

Wenn das Element existiert, gibt die Funktion true zurück und füllt die Karte auf, andernfalls gibt sie false zurück und die Karte bleibt leer.

Syntax:

steam_ugc_get_item_update_info(published_file_id, info_map);

Argument Beschreibung
veröffentlichte_datei_id Die eindeutige Datei-ID für die zu prüfende UGC.
info_map Ein (zuvor erstellter) DS-Map-Index.

Rückgabe:

Boolesche

Beispiel:

var info_map = ds_map_create();
var info = steam_ugc_get_item_update_info(global.fileID, info_map);
if info
{
draw_text(32, 15, "needs_update: " + string(info_map[? "needs_update"]));
draw_text(32, 30, "is_downloading: " + string(info_map[? "is_downloading"]));
draw_text(32, 45, "bytes_downloaded: " + string(info_map[? "bytes_downloaded"]));
draw_text(32, 60, "bytes_total: " + string(info_map[? "bytes_total"]));
}

Der obige Code fragt den Download-Status des Elements ab, das in der globalen Variablen "fileID" indiziert ist, wobei eine DS-Map zum Speichern der Informationen verwendet wird.
