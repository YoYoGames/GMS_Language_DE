steam_ugc_send_query

Diese Funktion kann verwendet werden, um eine Abfrageanfrage zu senden. Sie definieren zunächst die Abfrage mit den entsprechenden steam_ugc_create_query_* () Funktionen, die ein "Abfrage-Handle" zurückgeben. Dieses Handle wird dann zum Setzen von Filtern etc.... verwendet, bevor es in dieser Funktion verwendet wird, um die Abfrage abzuschicken. Die Funktion gibt einen eindeutigen Async-ID-Wert zurück, der verwendet werden kann, um die Details im Steam-Async-Ereignis korrekt zu überprüfen. Dieses Ereignis wird die async_load DS-Map, die mit den folgenden Schlüssel/Wert-Paaren bestückt wird:

  • "id" - Die asynchrone ID, die von der Funktion zurückgegeben wird, die das Ereignis ausgelöst hat
  • "Ergebnis" - Das Ergebnis der Operation (ein reeller Wert). Dies wird entweder die GML-Konstante ugc_result_success oder eine andere reelle Zahl sein. Sie sollten also auf diese Konstante prüfen, um sicherzustellen, dass der Aufruf erfolgreich war, und wenn andernfalls irgendetwas nicht korrekt funktioniert hat. Der Rest der möglichen Werte, die zurückgegeben werden, wird als Ergebnis des Steam- "EResult"-Wertes angezeigt, und Sie sollten in der Datei steamclientpublic .h in den SDK-Headern alle 89 möglichen Werte finden.
  • " event_type" - Die Zeichenkette " ugc_query"
  • "num_results" - Die Anzahl der zurückgegebenen Ergebnisse (max. 50)
  • "total_matching" - Die Gesamtzahl der übereinstimmenden Ergebnisse
  • " cached_data" - Zeigt an, ob diese Daten aus dem lokalen On-Disk-Cache abgerufen wurden oder nicht (wird entweder true oder false sein )
  • "results_list" - Ein DS-Listenindex, bei dem jeder Listeneintrag ein DS-Map-Index ist, der Details zu dem jeweiligen Element enthält

Wenn Sie den asynchronen Ereignistyp "ugc_query" erhalten, können Sie anschließend die Ergebnisliste parsen und die folgenden Informationen aus jeder der DS-Maps extrahieren (eine Map pro Element in der Liste):

  • "published_file_id" " -Hält die eindeutige veröffentlichte Datei-ID für das Element
  • " file_type" - Der Typ der verwendeten Datei
  • "creator_app_id" - Die Steam-ID des Artikelerstellers
  • "consumer_app_id" - Die Steam-ID des Artikelkonsumenten
  • "Titel" - Der Titel des Elements
  • "description" - Die Beschreibung des Artikels
  • "steam_id_owner" - Die Steam-ID des Eigentümers des Artikels
  • "time_created" - Die Zeit, zu der das Element erstmals erstellt wurde
  • "time_updated" - Das letzte Mal, dass das Element aktualisiert wurde
  • "time_added_to_user_list" - Die Zeit, zu der das Element abonniert wurde
  • "visibility" - Die Sichtbarkeit des Objekts (siehe steam_ugc_set_item_visibility() für die zurückgegebenen Konstanten)
  • " banned" - Ob das Element verboten wurde (true) odernicht (false)
  • "accepted_for_use" - Ob das Element zur Verwendung akzeptiert wurde (true) odernicht (false)
  • " tags_truncated" - Kurzversion der Tags als Array
  • "Tags"- Eine Zeichenkette mit den Tags für das Element, jeweils durch ein Komma getrennt
  • "handle_file" - Das eindeutige Dateihandle für das Element
  • "handle_preview_file" - Das eindeutige Handle für die Bildvorschau für den Artikel (kann als Argument verwendet werden mit steam_ugc_download() verwendet werden, um ein Vorschaubild herunterzuladen)
  • "Dateiname" - Der Name der Artikeldatei
  • "file_size" - Die Größe der Artikeldatei
  • "preview_file_size" - Die Größe des Vorschaubildes
  • "url"- Die vollständige URL für das Element
  • "votes_up" - Die Anzahl der erhaltenen Up-Votes
  • "votes_down" - Die Anzahl der erhaltenen Down-Votes
  • " score" - Die Gesamtbewertung des Elements
  • "account_id_owner" - Die Konto-ID des Steam-ID-Besitzers (diese kann in der Funktion steam_ugc_create_query_user_ex())

Syntax:

steam_ugc_send_query(ugc_query_handle);

Argument Beschreibung
ugc_query_handle Das zu sendende Abfrage-Handle.

Rückgabe:

Boolesche

Beispiel:

var query_handle = steam_ugc_create_query_all(ugc_query_RankedByTrend, ugc_match_Items, 1);
steam_ugc_query_add_required_tag(query_handle, "RPG");
steam_ugc_query_set_return_long_description(query_handle, true);
steam_ugc_query_set_allow_cached_response(query_handle, true);
query_ID = steam_ugc_send_query(query_handle);

Der obige Code erstellt eine Abfrage und speichert deren Handle in einer lokalen Variablen für die spätere Verwendung in den übrigen Funktionen, die die Abfrage vor dem Senden der Abfrage weiter definieren.
