steam_ugc_query_set_allow_cached_response

Diese Funktion kann verwendet werden, um eine beliebige UGC-Abfrage weiter zu filtern, insbesondere um anzufordern, dass die Abfrage den lokalen Cache und nicht online überprüft. Das Abfrage-Handle ist der Wert, der zurückgegeben wird, wenn Sie die Abfrage erstellen (z. B. mit, steam_ugc_create_query_user()) und das zweite Argument ist entweder true oder false. Die Funktion gibt true zurück, wenn der Abfragefilter korrekt gesetzt wurde, und andernfalls false.

Syntax:

steam_ugc_query_set_allow_cached_response(ugc_query_handle , cache);

Argument Beschreibung
ugc_query_handle Das zu verwendende Abfrage-Handle.
cache Ob die Abfrage den lokalen Cache prüfen soll oder nicht.

Rückgabe:

Boolesche

Beispiel:

var query_handle = steam_ugc_create_query_all(ugc_query_RankedByTrend, ugc_match_Items, 1);
steam_ugc_query_add_required_tag(query_handle, "RPG");
steam_ugc_query_set_return_long_description(query_handle, true);
steam_ugc_query_set_allow_cached_response(query_handle, true);
query_ID = steam_ugc_send_query(query_handle);

Der obige Code erstellt eine Abfrage und speichert deren Handle in einer lokalen Variablen für die spätere Verwendung in den übrigen Funktionen, die die Abfrage vor dem Senden der Abfrage weiter definieren.
