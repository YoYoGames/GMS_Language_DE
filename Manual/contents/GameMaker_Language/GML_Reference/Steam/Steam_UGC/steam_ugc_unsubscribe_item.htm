steam_ugc_unsubscribe_item

Diese Funktion kann verwendet werden, um ein UGC-Element abzubestellen. Sie geben die ID der veröffentlichten Datei an und dann gibt die Funktion einen eindeutigen Async-ID-Wert zurück, der verwendet werden kann, um zu prüfen, ob das Element im Steam-Async-Ereignis korrekt abgemeldet wurde. Dieses Ereignis hat die async_load DS-Map, die mit den folgenden Schlüssel/Wert-Paaren bestückt wird:

  • "id" - Die asynchrone ID, die von der Funktion zurückgegeben wird, die das Ereignis ausgelöst hat
  • "result" - Das Ergebnis der Operation (ein reeller Wert). Dies wird entweder die GML-Konstante ugc_result_success oder eine andere reelle Zahl sein. Sie sollten also auf diese Konstante prüfen, um sicherzustellen, dass der Aufruf erfolgreich war, und wenn andernfalls irgendetwas nicht korrekt funktioniert hat. Der Rest der möglichen Werte, die zurückgegeben werden, wird als Ergebnis des Steam- "EResult"-Wertes angezeigt, und Sie sollten in der Datei steamclientpublic .h in den SDK-Headern alle 89 möglichen Werte finden.
  • " event_type" - Die Zeichenfolge " ugc_unsubscribe_item".
  • "published_file_id" - Die eindeutige ID für das veröffentlichte Element.

Syntax:

steam_ugc_unsubscribe_item(published_file_id);

Argument Beschreibung
veröffentlichte_datei_id Die eindeutige Datei-ID für die UGC, von der abgemeldet werden soll.

Rückgabe:

Async-ID

Beispiel:

steam_sub= steam_ugc_subscribe_item(global.pubFileID);

Der obige Code abonniert (und lädt) das Element mit der Datei-ID, die in der globalen Variablen "pubFileID" gespeichert ist.
