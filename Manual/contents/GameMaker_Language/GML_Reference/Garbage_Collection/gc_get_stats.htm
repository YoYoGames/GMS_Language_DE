gc_get_stats

Mit dieser Funktion können Sie Informationen über den aktuellen Zustand des Garbage Collectors abrufen. Die Funktion gibt eine Struktur zurück, die die folgenden Mitgliedsvariablen enthält (beachten Sie, dass "Objekte" sich hier auf alles bezieht, was garbage collected werden kann, und nicht auf allgemeine Objektinstanzen, wie sie im Asset Browser definiert sind):

  • objects_touched - dies ist die Anzahl der aktiven Objekte, die der Garbage Collector im vorherigen Frame gefunden hat. Dies variiert je nachdem, welche Generation gesammelt wurde.
  • objects_collected - die Anzahl der Objekte, die der Garbage Collector festgestellt hat, dass sie im vorherigen Frame nicht aktiv waren und daher gelöscht werden konnten.
  • traversal_time - das ist die Zeit in Mikrosekunden (auf dem Haupt-Thread), die der Garbage Collector brauchte, um herauszufinden, welche Objekte aktiv waren.
  • collection_time - dies ist die Zeit in Mikrosekunden (auf einem separaten Thread), die der Garbage Collector benötigt hat, um die als inaktiv eingestuften Objekte aufzuräumen.
  • gc_frame - Dies ist ein Zähler, der jedes Mal erhöht wird, wenn ein Garbage Collection-Durchgang stattfindet. Wenn die Garbage Collection deaktiviert ist, wird er nicht erhöht.
  • generation_collected - dies ist der Index der Generation, die zuletzt gesammelt wurde. 0 ist die jüngste Generation und 3 ist die aktuell älteste.
  • num_generations - dies ist die Gesamtzahl der Garbage-Collection-Generationen.
  • num_objects_in_generation - dies ist ein Array (der Größe num_generations), das die Anzahl der Objekte in jeder Generation enthält.

HINWEIS: Auf der HTML5-Zielplattform wird die Garbage Collection von der JavaScript-Engine gehandhabt und daher geben die Mitgliedsvariablen in der obigen Struktur alle 0 zurück, wenn diese Funktion auf dieser Plattform verwendet wird.

Wenn Sie diese Funktion verwenden, beachten Sie bitte, dass die für die Objekte angezeigten Informationen erst dann aktualisiert werden, wenn die Verarbeitung einer vollständigen Generation abgeschlossen ist, was je nach Einstellung der Frame-Time mehrere Frames dauern kann (siehe hier für weitere Informationen zur Frame-Time).

Syntax:

gc_get_stats();

Rückgabe:

Struktur

Beispiel:

if (global.debug == true)
{
var _s = gc_get_stats();
var _t = _s.traversal_time;
var _c = _s.collection_time;
show_debug_message("Traversal time = " + string(_t))
show_debug_message("Collection time = " + string(_c))
}

Der obige Code prüft eine globale Variable und wenn sie wahr ist, holt er sich Informationen vom Garabge-Kollektor und gibt sie als Debug-Meldungen auf der Konsole aus.
