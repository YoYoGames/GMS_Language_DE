gc_target_frame_time

Mit dieser Funktion können Sie dem Garbage Collector mitteilen, dass er nicht mehr als die angegebene Zeit für die Ausführung jedes Frames benötigen soll. Die Funktion nimmt einen Zeitwert an , der in Mikrosekunden angegeben wird (wobei 1.000.000 Mikrosekunden einer Sekunde entsprechen), und die standardmäßige Ziel-Frame-Zeit ist 100 Mikrosekunden.

Bitte beachten Sie, dass es sich hierbei lediglich um einen Zielwert ist, da einige Operationen noch komplett in einem Frame ausgeführt werden müssen und manchmal länger als die Zielzeit dauern können. Beachten Sie auch, dass eineErhöhung der Zielzeit dazu führt, dass der Garbage Collector schneller auf schnelle Änderungen in der Speichernutzung reagiert und den Speicher schneller freigibt, obwohl dies in der Praxis in den meisten Fällen nicht erforderlich sein dürfte. Beachten Sie auch, dass das Setzen der Ziel-Frame-Zeit auf 0 nicht dazu führt, dass der Collector komplett stoppt - er wird weiterhin eine minimale Menge an Arbeit pro Frame verrichten. Um den Collector vollständig zu deaktivieren, verwenden Sie die Funktion gc_enable().

Syntax:

gc_target_frame_time(time);

Argument Beschreibung
Zeit Die Zielzeit - in Mikrosekunden -, die der Garbage Collector pro Frame arbeiten soll

Rückgabe:

N/A (undefiniert)

Beispiel:

if (gc_get_target_frame_time() != 50)
{
gc_target_frame_time(50);
}

Der obige Code prüft das aktuelle Frame-Zeitziel für den Garbage Collector und wenn es nicht 50 Mikrosekunden beträgt, wird es auf diesen Wert gesetzt.
