puffer_sha1

In der Kryptografie ist SHA-1 eine kryptografische Hash-Funktion, die von der United States National Security Agency entwickelt wurde und in mehreren weit verbreiteten Anwendungen und Protokollen wie dem beliebten Git verwendet, wo es zur Überprüfung von Dateiänderungen eingesetzt wird. Diese Funktion nimmt Eingabedaten aus einem Puffer entgegen und gibt einen 160-Bit-Message-Digest im ASCII-Format zurück. Auf diese Weise können Sie einen sicheren Schlüssel erzeugen, der gespeichert und verwendet werden kann, um die Integrität zu prüfen der Informationen, die z. B. an einen externen Server gesendet (oder von diesem empfangen) werden, zu überprüfen. Wenn Sie dies mit dieser Funktion auf Puffer anwenden, müssen Sie die Puffer-ID des zu verwendenden Puffers angeben, dann einen Offset-Wert (in Bytes) für die Stelle, an der begonnen werden soll, und dann eine Größe (wiederum in Bytes) für den zu hashenden Bereich.

HINWEIS: SHA-1 ist nicht vollständig sicher und kann gebrochen werden. Siehe diese Seite für weitere Informationen.

Syntax:

buffer_sha1(buffer, offset, size);

Argument Beschreibung
Puffer Der Index des zu verwendenden Puffers.
Offset Der Datenoffsetwert.
Größe Die Größe des Puffers.

Rückgabe:

Zeichenfolge

Beispiel:

check_string = buffer_sha1(buff, 0, buffer_get_size(buff));

Der obige Code erstellt einen sha1-Hash für die vollständigen Daten, die in dem durch die Variable "buff" indizierten Puffer gespeichert sind, und speichert den zurückgegebenen Hash in der Variablen "check_string".
