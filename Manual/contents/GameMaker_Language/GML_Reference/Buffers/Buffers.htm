Puffer

Ein Puffer ist ein Bereich des physischen Speichers, der verwendet wird, um Daten vorübergehend zu speichern, während sie von einem Ort zum anderen verschoben oder auf irgendeine Weise manipuliert werden, z. B. können Sie einen Puffer verwenden, um Paketdaten zu speichern, die auf die Übertragung über ein Netzwerk warten, oder um eines Pakets, das über ein Netzwerk empfangen wurde. GameMaker Studio 2 verfügt über eine Reihe spezifischer Funktionen im Zusammenhang mit dem Erstellen, Ändern und Laden/Speichern von Puffern, aber bevor Sie diese verwenden, beachten Sie bitte die folgenden Punkte:

  • Wenn ein Puffer angelegt wird, wird er automatisch gelöscht und auf 0 gefüllt.
  • Das Argument "Größe" der Pufferfunktionen bezieht sich immer auf die Größe des Puffers in Bytes.
  • Das "Alignment"-Argument bezieht sich darauf, wie die Daten in einem Puffer gespeichert werden. Wenn die Ausrichtung auf z. B. 4 eingestellt ist und Sie ein einzelnes Datenstück mit einer Größe von 1 Byte schreiben und dann einen buffer_tell ausführen, erhalten Sie einen Offset von 1, Wenn Sie jedoch ein weiteres Stück Daten schreiben, das ebenfalls 1 Byte groß ist, und dann einen buffer_tell ausführen, erhalten Sie einen Offset von 5, da die Ausrichtung die Daten auf diese Größe "aufgefüllt" hat. Die Ausrichtung wirkt sich nur darauf aus , wohin die Daten geschrieben werden, wenn Sie also Wenn Sie also einen buffer_tell ausführen, nachdem Sie etwas geschrieben haben, wird die aktuelle Schreibposition zurückgegeben, die unmittelbar auf die Daten folgt, die Sie geschrieben haben. Beachten Sie jedoch, dass, wenn Sie dann ein weiteres Stück Daten schreiben, der Puffer intern die die Schreibposition auf das nächste Vielfache der Ausrichtungsgröße verschiebt, bevor er die Daten tatsächlich schreibt.
  • Beim "Offset" handelt es sich um den Wert in Bytes , um den die Daten innerhalb des gegebenen Puffers versetzt werden sollen. Wenn Ihr Puffer also 2 Byte ausgerichtet ist und Sie die ersten 4 ausgerichteten Stellen überspringen möchten, hätten Sie einen Offset von 2 * 4 Byte.
  • Einige der Pufferfunktionen erzeugen auch einen neuen Puffer (wie z. B. buffer_load ). Denken Sie daran, dass diese Puffer auch aus dem Speicher entfernt werden müssen, wenn sie nicht verwendet werden, indem Sie die Funktion buffer_delete verwenden.
  • Der Speicher, der für die Erstellung von Puffern verwendet wird, ist Systemspeicher, so dass alle in einem Puffer gespeicherten Daten sicher sein sollten, auch wenn das Spiel nicht den Fokus hat.

WICHTIG! Wenn Sie einen Buffer erstellen, ist der Indexwert, mit dem er identifiziert wird, ein ganzzahliger Wert, der bei 0 beginnt. Diese Indizes werden vom GameMaker wiederverwendet, so dass ein zerstörter Buffer-Indexwert anschließend von einem neu erstellten verwendet werden kann, und wir empfehlen, dass Sie jede Variable, die einen Buffer-Index enthält, auf -1 setzen, nachdem Sie den Buffer zerstört haben.
