puffer_async_gruppe_option

Mit dieser Funktion können Sie einige plattformspezifische Optionen für die zu speichernde Puffergruppe einstellen. Die verfügbaren Optionen sind wie folgt:

  • " subtitle" oder" slottitle" -Der Wert für diese Option wäre eine Zeichenfolge, die dem Benutzer beim Verwalten seiner Speicherdaten im Betriebssystem angezeigt wird. Dies ist nur beim Speichern von Daten wichtig, nicht beim erneuten Laden.
  • "showdialog" - Der Wert für diese Option ist ein boolesches true oder false. Wenn sie auf true gesetzt ist, zeigt GameMaker Studio 2 beim Speichern oder Laden die System-Benutzeroberfläche an, andernfalls wird ein QuickSave/Quickload ohne Anzeige der Benutzeroberfläche. Normalerweise benötigen Sie diese Option nur, wenn Sie mehrere Speicherplätze unterstützen, damit der Benutzer einen Speicherplatz auswählen kann, aber wenn Sie nur einen Speicherplatz pro Benutzer unterstützen, setzen Sie diese Option auf false.
  • " savepadindex" - Bei dieser Option geben Sie den Pad-Index des Spielers an, der gerade speichert oder lädt, und das System schreibt Daten in den Speicherordner dieses Spielers und liest Daten daraus.
  • " saveslotsize" - Diese Option erfordert, dass Sie die tatsächliche Größe in Bytes angeben, die Sie speichern wollen (so können Sie eine buffer_seek() und buffer_tell() erhalten das zum Beispiel). Beachten Sie, dass es nicht zwingend erforderlich ist, diesen Wert anzugeben, da allen Speicherungen ein Mindestplatz zugewiesen wird, der normalerweise etwa 10 MB beträgt.

WICHTIG! Diese Funktion ist derzeit nur für die Zielmodule PSVita, PS4 und XBox One gültig. Bei allen anderen Targets bewirkt sie nichts.

Syntax:

buffer_async_group_option(option, value);

Argument Beschreibung
Option Die Option zum Einstellen.
Wert Der zu setzende Wert (kann je nach Option ein String oder Real sein).

Rückgabe:

N/A

Beispiel:

buffer_async_group_begin("save_folder_name");
buffer_async_group_option("showdialog", false);
buffer_async_group_option("slottitle", "Catch The Haggis Save");
buffer_async_group_option("subtitle", "Alle Ihre Haggis sind hier gespeichert!");
save = buffer_save_async(buff, "Player_Save.sav", 0, 16384);
buffer_async_group_end();

Der obige Code startet eine Puffergruppe und legt dann die Gruppenoptionen fest, bevor er 4 Dateien zum asynchronen Speichern einstellt. Die Gruppendefinition wird dann beendet (zu diesem Zeitpunkt beginnt das Speichern).
