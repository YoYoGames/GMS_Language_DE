buffer_async_group_begin

Diese Funktion wird aufgerufen, wenn Sie das Auslagern von mehreren Puffern in mehrere Dateien beginnen wollen. Der "Gruppenname" ist eine Zeichenkette und wird als Verzeichnisname für den Ort verwendet, an dem die Dateien gespeichert werden, und sollte als Teil des Dateipfads verwendet werden Pfad verwendet werden, wenn Sie die Dateien später wieder in die IDE laden (mit einer der buffer_load() Funktionen). Diese Funktion ist nur für die Verwendung mit der buffer_save_async() Funktion und Sie müssen auch die Speicherdefinition mit dem Aufruf der buffer_async_group_end() Funktion abschließen, sonst werden die Dateien nicht abgespeichert.

Beachten Sie, dass für die Konsolenplattformen (wie z. B. PS4) der "Gruppenname" als Beschreibung des Speicherplatzes verwendet wird, und dass Sie mit dieser Funktion vermeiden können, dass die Benutzeroberfläche für jede Datei, die gespeichert wird, angezeigt wird. Beachten Sie auch, dass Sie bei bei der Verwendung von UWP Ihre Spielstände zwischen nativen XBox- und Microsoft-Store-Fenstern umherwandern können, indem Sie eine Gruppe erstellen, die z. B. mit einer ERA-Xbox-Gruppe identisch ist:

if os_type == os_uwp
{
buffer_async_group_begin("root/yourgroupname");
}
sonst
{
buffer_async_group_begin("ihrgruppenname");
}

Syntax:

buffer_async_group_begin(groupname);

Argument Beschreibung
Gruppenname Der Name der Gruppe (als Zeichenkette).

Rückgabe:

N/A

Beispiel:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
buffer_async_group_end();

Der obige Code startet eine Puffergruppe und stellt sie dann so ein, dass 4 Dateien asynchron gespeichert werden. Die Gruppendefinition wird dann beendet (zu diesem Zeitpunkt beginnt das Speichern).
