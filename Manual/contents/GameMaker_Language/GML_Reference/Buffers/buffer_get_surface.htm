buffer_get_surface

Mit dieser Funktion können Sie die Informationen aus einer Oberfläche in einen gegebenen Puffer schreiben. Der Puffer muss zuvor erstellt worden sein und sollte ein 1Byte ausgerichteter Wachstumspuffer mit einer ungefähren Größe (in Bytes) der Oberfläche sein, die Sie schreiben wollen. Die Informationen für die Oberfläche werden immer an den Anfang des Puffers geschrieben (unabhängig von der Suchposition), aber Sie können einen Offset-Wert (in Bytes) wählen, um die Schreibposition vom Anfang des Puffers dorthin zu verschieben, wo Sie sie benötigen. Beachten Sie, dass die Funktion jedes Pixel der Oberfläche unter Verwendung einer BGRA-Formatierung auf dem Windows-Ziel in den Puffer schreibt, aber auf anderen Zielen kann dies je nach Betriebssystem oder sogar Gerät anders sein.

HINWEIS: Diese Funktion hat sich ab dem GameMaker Studio Update 2.3.1 geändert. Frühere Versionen verwendeten die Konstante buffer_surface_copy, die jetzt veraltet ist.

Syntax:

buffer_get_surface(buffer, surface, offset);

Argument Beschreibung
Puffer Der Index des zu verwendenden Puffers.
Oberfläche Der Index der zu verwendenden Oberfläche.
Offset Der Datenoffsetwert (in Bytes).

Rückgabe:

N/A

Beispiel:

buffer_get_surface(buff, application_surface, 0);

Der obige Code kopiert die in der Anwendungsoberfläche gespeicherten Daten ohne Offset in den in der Variablen "buff" indizierten Puffer.
