Streicher

Irgendwann bei der Erstellung Ihres Spiels werden Sie Text verwenden müssen. Text in Spielen wird mit Hilfe der String-Funktionen behandelt (ein String ist nur eine andere Art, eine Textzeile zu sagen), und GameMaker Studio 2 verfügt über einen kompletten Satz von Funktionen, die GameMaker Studio 2 verfügt über einen kompletten Satz von Funktionen, die es Ihnen erlauben, Strings auf vielfältige Weise zu manipulieren, einschließlich des Einfügens eines Strings in einen anderen, des Kopierens von Strings und der Möglichkeit, Strings auf die darin enthaltenen Ziffern oder Buchstaben zu analysieren. Im Allgemeinen kann eine Zeichenkette nur erstellt werden durch Hinzufügen von Text innerhalb von doppelten Anführungszeichen " " erstellt werden, und Strings in einfachen Anführungszeichen werden nicht akzeptiert. Auch können Sie den String nicht über mehrere Zeilen aufteilen und erwarten, dass GameMaker Studio 2 ihn so darstellt, als wären die Zeilenumbrüche Zeilenumbrüche (es sei denn, ein String-Literal @ Bezeichner verwendet wird, wie unten erklärt).

Es ist erwähnenswert, dass es bestimmte Konventionen gibt, die Sie beim Erstellen von Zeichenketten verwenden können, die sich hauptsächlich auf die Verwendung von Escape-Zeichen beziehen. Dies sind Zeichen, denen ein "\" Symbol vorangestellt ist. Wenn Sie also zum Beispiel Wenn Sie also beispielsweise Anführungszeichen in eine Zeichenkette einfügen möchten, würden Sie etwa so vorgehen:

str = "Hallo\"Welt\"";

GameMaker Studio 2 verfügt außerdem über eine vollständige, vier Byte breite Unicode-Zeichenunterstützung, die es Ihnen ermöglicht, Unicode-Zeichen im oberen Bereich des Standards zu dekodieren und zu kodieren (einschließlich - aber nicht beschränkt auf - Emoji). Um mit Unicode-Zeichen umzugehen, können Sie können Sie das "\" verwenden, um ein beliebiges Unicode-Literal voranzustellen - Ziffern in Hexadezimalzeichen, denen ein "u" vorangestellt ist, z. B. "\u00e2" für "á" - wobei die Ziffern die Nummer des Unicode-Zeichens sind. Wenn Sie auf diese Weise mit Unicode arbeiten, müssen Sie sich darüber im Klaren sein, dass GameMaker Studio 2 alle Ziffern nach dem "u" interpretiert, also wenn Sie z. B. "áa" schreiben wollten, sollten Sie verwenden:

"\u00e2\a"

oder

"\u00e2\u61"

oder

"\u00e2" + "a"

da die Verwendung von " \u00e2a" tatsächlich das Unicode-Zeichen " ส" ergeben würde (im Wesentlichen zu " \ue2a").

GameMaker Studio 2 kann auch jedes hexadezimale Literal verarbeiten - normalerweise geschrieben als Hexadezimalziffern nach "0x", zum Beispiel "0xff", wobei die Ziffern die Nummer des des zu verwendenden Zeichens sind. In GameMaker Studio 2 werden diese mit " \x" und dann dem Hex-Wert geschrieben. Diese und andere vordefinierte Escape-Zeichen sind in der folgenden Tabelle aufgeführt:

Konstant Beschreibung
\n Zeilenumbruch
\r Schlittenrücklauf
\b Rücktaste (0x08)
\f Form Feed (0x0c)
\t Horizontaler Tabulator (0x09)
\v Vertikaler Tabulator (0x0b)
\\ Backslash selbst (0x5c)
\a Warnung (0x07)
\u[Hex-Ziffern] Unicode-Zeichen einfügen
\x[Hex-Ziffern] Hex-Literalzeichen einfügen
Oktalziffern] Oktales Unicode-Zeichen einfügen

HINWEIS: Strings unterstützen Formfeed, vertikale Tabulatoren usw., aber das bedeutet nicht, dass das Rendering dies auch tut, und beim Zeichnen von Strings können diese Zeichen ignoriert werden.

Sie können auch wortwörtliche Zeichenfolgenliterale erstellen, indem Sie der gesamten Zeichenfolge das Zeichen @ voranstellen:

var test = @"
Zeilenumbruch
über mehrere
Zeilen
";

Der obige Code rendert die Zeichenfolge über mehrere Zeilen, als ob ein Zeilenumbruch-Escape-Zeichen enthalten wäre. Ein Verbatim-Stringliteral ähnelt den Stringliteralen früherer GameMaker-Versionen, aber sie verwenden ebenfalls doppelte oder einfache Anführungszeichen und müssen @ Sie können über mehrere Zeilen in der Codedatei umbrochen werden und sie unterstützen KEINE Escape-Zeichen, d. h. @"Hello\World" versucht nicht, das W bei World nicht zu entschlüsseln und wird wortwörtlich gespeichert. Beachten Sie jedoch, dass Sie bei der Verwendung von String-Literalen wie diesem den String umbrechen müssen, wenn Sie Anführungszeichen als Teil des Strings einschließen möchten, d. h:

var test = @"Hallo " + "\"" + @"Welt" + "\""

Zu beachten ist auch, dass das Unicode-Zeichen 9647 (▯) verwendet wird, um fehlende Glyphen zu ersetzen, die Sie möglicherweise in der gewünschten Schriftart haben, wenn Sie diese im Draw-Event rendern. Wenn also Ihre Schriftart z. B. das °-Symbol nicht hat, dann wird das Schreiben von 90° tatsächlich 90▯ erzeugen.
