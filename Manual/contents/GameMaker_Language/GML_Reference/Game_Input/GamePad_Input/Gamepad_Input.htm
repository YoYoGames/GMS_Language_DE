GamepadEingabe

GameMaker Studio 2 verfügt über eine Reihe spezieller Funktionen, mit denen Sie sowohl analoge als auch digitale Steuerelemente von mehreren angeschlossenen Gamepads erkennen können. Diese Funktionen funktionieren ähnlich wie die Geräteeingänge, insofern, als dass Sie bis zu vier verschiedene angeschlossene XInput-Gamepads (und bis zu 8 DirectInput-Gamepads) erkennen und die Eingaben von jedem einzelnen mit denselben Funktionen verarbeiten können. Beachten Sie, dass, wenn ein Gamepad an Ihr Gerät angeschlossen wird (oder es entfernt wird) dann wird ein asynchrones Systemereignis ausgelöst, bei dem Sie die Situation mit den entsprechenden Funktionen behandeln können.

Die Gamepad-"Steckplätze" sind von 0 aufwärts indiziert, und der tatsächliche Steckplatz, der einem Gamepad zugewiesen wird, hängt von einer Vielzahl von Faktoren ab, nicht zuletzt vom Betriebssystem, unter dem das Projekt läuft. Auf dem Windows Target sind die Steckplätze 0 - 3 einschließlich nur für Xinput-Gamepads, d. h. Xbox360-Controller und kompatible Geräte. Sie können jedoch auch die Slots 4 bis einschließlich 11 für DirectInput-Gamepads überprüfen, was bedeutet, dass Sie viele andere Controller-Modelle erkennen können, wenn sie über diese Slots über diese Steckplätze. Auf anderen Plattformen können Pads an jedem Steckplatz erkannt werden, den das Betriebssystem ihnen zugewiesen hat, also z. B. Steckplatz 3 oder Steckplatz 20 oder mehr. Ein Beispiel, Android Geräte speichern Bluetooth-Gamepads in einem Steckplatz und reservieren dann Sie können also nicht davon ausgehen, dass ein einzelnes angeschlossenes Gamepad mit Steckplatz 0 verbunden ist, da dies wahrscheinlich nicht der Fall sein wird.

Es ist zu beachten, dass bei der Verwendung von DirectInput-Gamepads unter Windows oder generischen Gamepads auf anderen Plattformen die unten angegebenen Konstanten möglicherweise nicht genau mit den Tasten übereinstimmen, die Sie erwarten, wenn sie gedrückt werden, da die API der Controller-Hersteller fragmentiert und nicht standardisiert ist. nicht standardisierten Art und Weise, wie die API von den Controller-Herstellern implementiert wird. Aus diesem Grund wird empfohlen, dass Sie in Ihren Spielen eine Art Gamepad-Setup-Bildschirm einrichten, in dem die Tasten des Gamepads basierend auf den Eingaben von einem angeschlossenen Gerät neu definiert werden können. Es gibt Gamepad-"Mapping"-Funktionen, die bei Windows Desktop, Ubuntu, macOS und Android-Zielen helfen können, während Sie dies bei allen anderen Geräten selbst mit Code tun müssen.

Die folgenden Gamepad-Funktionen sind ebenfalls vorhanden und werden für die Neuzuordnung der eingebauten Konstanten zu den direkten physischen Eingaben eines bestimmten Gamepads verwendet. Diese Funktionen sind nur für die Zielplattformen Windows Desktop, Ubuntu, macOS und Android und unter Windows sind sie nur für Direkteingabegeräte gültig. GameMaker Studio 2 wird zwar mit Mappings für eine Reihe verschiedener Gamepads basierend auf der SDL Gamepad Controller DB ausgeliefert, aber Aufgrund der großen Anzahl von Controller-Typen und -Marken ist es jedoch unmöglich, die GML-Konstanten auf die richtigen Eingänge für jedes Fabrikat und Modell abzubilden, so dass Sie mit diesen Funktionen die Möglichkeit haben, Ihre eigenen benutzerdefinierten Mappings zu erstellen.

  • gamepad_get_mapping
  • gamepad_test_mapping
  • gamepad_remove_mapping

Es ist erwähnenswert, dass Direct-Input-Gamepads im kooperativen Modus betrieben werden, was bedeutet, dass Ihr Spiel nur Zugriff auf sie hat, wenn es die Vordergrundanwendung ist, was wiederum dazu führt, dass Direct-Input-Controller "verloren" gehen, wenn das Das wiederum führt dazu, dass Direct Input-Controller "verloren" gehen, wenn das Spiel den Fokus verliert, und dann wieder "gefunden" werden, wenn es wieder in den Fokus kommt (dies kann im Systemereignis erkannt und behandelt werden). In ähnlicher Weise werden keine Eingaben von Gamepads erkannt, wenn das Spiel nicht im Fokus ist, und wir empfehlen, dass Sie verwenden Sie die Funktion os_is_paused() oder window_has_focus() zu verwenden, um dies zu erkennen und das Spiel zu pausieren oder etwas Wir empfehlen, die Funktion oder zu verwenden, um dies zu erkennen und das Spiel anzuhalten oder etwas Ähnliches, da jede Taste, die zu dem Zeitpunkt gedrückt wird, an dem das Spiel den Fokus verliert, den gedrückten Zustand beibehält, bis das Spiel den Fokus wiedererlangt.

Kompatibilität

Die folgende Liste zeigt die aktuelle Kompatibilität zwischen den Plattformen (beachten Sie, dass sich diese mit zukünftigen Updates ändern wird):

  • Windows wird vollständig unterstützt, wobei bis zu 12 angeschlossene Geräte gleichzeitig zulässig sind (nummeriert von 0 bis 11, wobei 0 - 3 XInput-Geräte und 4 - 11 DirectInput-Geräte sind). Die Neuzuordnung von Controller-Konstanten ist ebenfalls zulässig.
  • macOS wird unterstützt, wobei maximal 4 angeschlossene Geräte auf einmal erlaubt sind und diese Geräte NUR vom Typ Playstation3oder Xbox 360 sein können. Bitte beachten Sie, dass die Option "Build for Mac AppStore" in den Mac-Spieloptionen ausgeschaltet sein muss, damit die Pad-Unterstützung funktioniert. Das Remapping von Controller-Konstanten ist ebenfalls erlaubt.
  • Ubuntu unterstützt auch die Gamepad-Eingabe, aber Sie müssen möglicherweise zusätzliche Bibliotheken aus dem Ubuntu-Repository installieren. Sie können dies einfach tun, indem Sie ein Kommandozeilen-Terminal öffnen und Folgendes eingeben (dadurch wird die GUI-Unterstützung für den Joystick/das Gamepad sowie die Joystick/Gamepad-API selbst - ein Remapping von Controller-Konstanten ist nicht erlaubt):

    sudo apt-get install jstest-gtk
    sudo apt-get install joystick.

  • HTML5-Spiele mit Gamepads werden von den meisten gängigen Browsern unterstützt, außer von Safari. Die Neuzuordnung von Controller-Konstanten ist nicht zulässig.
  • Die Gamepad-Unterstützung erstreckt sich auch auf iOS mit dem iCade-Gehäuse. Die linke Achse entspricht dem Stick-Controller (obwohl der Eingang digital und nicht analog ist), die vier "Face"-Tasten entsprechen den Tasten an der Vorderseite des Gehäuses, und die vier Schultertasten entsprechen den Tasten auf der Rückseite des Gehäuses.
  • Android (und Amazon Feuer) Export unterstützt NYKO-Controller und generische Bluetooth-Controller, aber nur, wenn sie aktiviert sind, was bedeutet, dass Sie die iCade/Bluetooth-Option im Abschnitt "Allgemein" der Android-Spieloptionen ankreuzen müssen. Sie benötigen API-Level 12, damit sie vollständig funktionieren, und es sollte beachtet werden, dass GameMaker Studio 2 alle Bluetooth-Geräte, mit denen Ihr Gerät gekoppelt ist, als verbunden registriert, unabhängig davon, ob es tatsächlich verbunden ist oder nicht verbunden ist. Daher sollte dies bei der Zuweisung und Überprüfung von "Slots" berücksichtigt werden. Beachten Sie, dass auch das Remapping von Controller-Konstanten erlaubt ist.
  • Auf der PS4 müssen Sie, wenn Sie das Touchpad-Tracking verwenden möchten, die gerät_maus_* Tasten. Das Remapping von Controller-Konstanten ist nicht erlaubt.
  • Auf Windows UWP, XBoxOne und Nintendo Switch Targets werden Gamepads vollständig unterstützt, aber das Mapping von Controller-Konstanten ist nicht erlaubt.

Idealerweise möchten Sie auf allen Zielplattformen eine Liste der verfügbaren Gamepad-"Steckplätze" auflisten und dann prüfen, ob Geräte erkannt werden, etwa so:

var _maxpads = gamepad_get_device_count();

for (var i = 0; i < _maxpads; i++)
{
if (gamepad_is_connected(i))
{
// Sachen mit Pad "i" machen
}
}
