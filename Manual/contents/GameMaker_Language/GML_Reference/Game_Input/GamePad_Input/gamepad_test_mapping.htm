gamepad_test_mapping

Mit dieser Funktion können Sie die Gamepad-Zuordnung auf den Zielen einstellen, die dies zulassen. Sie geben den "Slot"-Index für das zu setzende Gamepad an und dann den Map-String, der im SDL-Format mit den folgenden Feldern erstellt worden sein sollte:

<guid>,<Beschreibung>,Plattform:<Plattform-Name>,<Bindings>

Die Details, die in jedem dieser Felder enthalten sein sollten, sind:

  • guid - eine Zeichenkette aus Ziffern und Buchstaben, die einen Gerätetyp eindeutig identifiziert (diese kann mit der gamepad_get_guid() Funktion)
  • description - eine Ascii-Beschreibung des Geräts (diese kann mit der gamepad_get_description() Funktion)
  • plattform-name - ist entweder "plattform:android", "plattform:windows", "linux" oder "mac" (beachten Sie, dass dies automatisch an die Zeichenkette angehängt wird , wenn Sie diese Funktion aufrufen, und Sie daher Sie müssen ihn also nicht angeben, wenn Sie Ihre eigenen Mapping-Strings erstellen)
  • Bindungen - ein Satz von Einträgen, die durch ein ',' getrennt sind, um aktuelle Eingaben an bestimmte GML-Konstanten zu binden

Die kommagetrennte Liste der Einträge folgt einem bestimmten Format für jeden Eintrag:

<Abstrakter-gp-Name>:<Wert-Bindung>

Hier kann <Wertbindung> (d. h. die direkte Gamepad-Eingabe) eine der folgenden Möglichkeiten sein:

  • a<N> - Dies repräsentiert eine Achse des Gamepads, wobei das <N> eine der Achsen von 0 bis gamepad_axis_count() -1
  • b<N> - Dies stellt eine Taste des Gamepads dar, wobei das <N> eine der Tasten von 0 bis gamepad_button_count() -1
  • h<N>.<Maske> - Dies stellt einen Hut des Gamepads dar, wobei das <N> einen Hut von 0 bis gamepad_hat_count() -1 steht, und die <Maske> entweder 1, 2, 4 oder 8 sein wird.

Jeder Eingabewert sollte an einen SDL-Namen gebunden werden, der als <>abstract-gp-name> ausgedrückt wird , der wiederum an eine GML-Konstante gebunden ist. Die folgende Tabelle zeigt die Äquivalenz zwischen dem SDL-Namen und der GameMaker Studio 2-Konstante:

Wenn Sie Ihre Mapping-Zeichenkette aufgebaut haben, sollte sie etwa so aussehen:

"050000005e040000fd020000ffff3f00,Xbox Wireless Controller,a:b0,b:b1,start:b4,dpdown:h0.4,dpleft:h0.8,dpright:h0.2,dpup:h0.1,guide:b6,leftshoulder:b9,leftstick:b7,lefttrigger:a4,leftx:a0,lefty:a1,rightshoulder:b10,rightstick:b8,righttrigger:a5,rightx:a2,righty:a3,x:b2,y:b3,platform:android"

Syntax:

gamepad_test_mapping(index, mapping_string);

Argument Beschreibung
index Welcher Gamepad-Index "Slot" eingestellt werden soll.
Farbe Die zu verwendende Kartenzeichenfolge (weitere Informationen finden Sie in der Beschreibung).

Rückgabe:

N/A

Beispiel:

var mapping = gamepad_get_guid(global.padIndex) + "," + gamepad_get_description(global.padIndex);
var len = array_length(global.PadInstances);
for( i = 0; i < len; i += 2)
{
var left = global.PadInstances[i];
var right = global.PadInstances[i+1];
mapping += "," + left.sdlLabel + ":" + right.binding;
}
gamepad_test_mapping(global.padIndex, mapping);

Der obige Code durchläuft eine Reihe von Instanzen und verwendet die Werte der verschiedenen Variablen, die sie enthalten, um eine Zuordnungszeichenfolge zu erstellen, die dann für die Verwendung auf dem Gamepad im angegebenen Steckplatzindex eingestellt wird.
