Geräteeingang

Wenn Ihr Spiel auf die Plattformen ausgerichtet ist, die Multi-Touch erlauben, sollten Sie diese Funktionen verwenden. Sie funktionieren so, wie ein PC mit mehreren angeschlossenen Mäusen funktionieren würde, und die Konstanten, auf die geprüft wird, sind die gleichen wie die für die Standard-Mausfunktionen die für die Standard-Mausfunktionen verwendet werden:

Konstant Beschreibung
mb_links Die linke Maustaste
mb_middle Die mittlere Maustaste (dies ist möglicherweise nicht für alle Zielplattformen gültig)
mb_rechts Die rechte Maustaste
mb_side1* Seitliche Maustaste 1
mb_side2* Seitliche Maustaste 2
mb_beliebig Eine der Maustasten
mb_keine Keine Maustaste


* HINWEIS: Die Schaltflächen mb_side1 und mb_side2 sind nur unter Windows, macOS, Ubuntu und HTML5 verfügbar.

Die Funktionen können alle einen "Touch"-Wert annehmen, der einer Berührung auf dem Gerätedisplay entspricht. Die maximale Anzahl der Berührungen, die erkannt werden können, hängt jedoch stark vom Gerät und dem verwendeten Betriebssystem ab. Zum Beispiel können einige Low-End-Android Geräte möglicherweise nur 2 oder 3 Berührungen gleichzeitig erkennen, während ein UWP-Gerät möglicherweise bis zu 10 Berührungen erkennen kann. Beachten Sie auch, dass auf Touchscreen-Geräten ein Doppeltipp als das Äquivalent eines Ereignisses der rechten Maustaste zurückgegeben wird. Wenn Sie also Wenn Sie also z. B. mehrere Berührungen in schneller Folge erkennen müssen, müssen Sie sowohl auf mb_left als auch auf mb_right prüfen. Diese Funktionalität kann ein- oder ausgeschaltet werden mit device_mouse_dbclick_enable().

Nachfolgend finden Sie eine Liste aller gerätespezifischen Steuerfunktionen:

  • device_mouse_check_button
  • device_mouse_check_button_pressed
  • device_mouse_check_button_released
  • device_mouse_dbclick_enable
  • gerät_maus_x
  • gerät_maus_y
  • device_mouse_raw_x
  • device_mouse_raw_y
  • gerät_maus_x_zu_gui
  • gerät_maus_y_zu_gui
  • gerät_ist_tastatur_offen
  • gerät_gekippt_x
  • gerät_gekippt_y
  • gerät_gekippt_z

Bitte beachten Sie, dass Sie auf mobilen Geräten(Android, iOS und Windows Phone) jederzeit die Zurück-Taste verwenden können, da sie der Tastaturrücktaste zugeordnet ist, die die GameMaker Studio 2-Tastaturkonstante vk_backspace ist, so dass Sie Prüfungen für diese Taste durchführen und Aktionen auf diesen Geräten ausführen können, als ob die Zurück-Taste verwendet worden wäre. Beachten Sie auch, dass das HTML5-Target nur für "Gerät 0" Eingaben korrekt akzeptiert, wenn Sie diese Funktionen verwenden.
