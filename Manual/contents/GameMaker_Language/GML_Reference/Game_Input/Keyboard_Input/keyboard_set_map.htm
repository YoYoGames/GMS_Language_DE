keyboard_set_map

Manchmal, wenn Sie ein Spiel erstellen, möchten Sie vielleicht, dass eine Taste dasselbe tut wie eine andere. Zum Beispiel benutzen viele Leute die Tasten WASD für die Bewegung, aber dann benutzen viele Leute auch die Pfeiltasten! Was ist also zu tun? Nun, man könnte das Bewegungssystem zweimal codieren, aber das ist ein bisschen kompliziert und zum Glück überflüssig, da diese Funktion es Ihnen erlaubt, eine Taste auf eine andere zu "mappen", so dass jede Eingabe von einer der beiden Tasten als dieselbe interpretiert wird. Dazu wählen Sie eine Taste, die Sie mappen wollen (Taste2 - dies wird die Taste sein, für die Sie den Code schreiben) und eine Taste, auf die sie abgebildet werden soll (Taste1). Danach werden Tastendrücke auf beide Tasten von GameMaker Studio 2 so interpretiert, als kämen sie von Taste2. Sie können diese Funktion auch verwenden, um Sie können diese Funktion auch verwenden, um ein System zu entwerfen, bei dem der Benutzer seine eigenen Tasten zum Spielen definieren kann, indem er einfach die Benutzereingabetaste der Taste zuordnet, die Sie im Spiel kodiert haben.

Die Funktion nimmt einen Keycode-Wert, wie er von der Funktion ord() zurückgegeben wird (nur Großbuchstaben von A-Z oder Zahlen von 0-9), oder eine der vk_*-Konstanten, die auf der Hauptseite Tastatureingabe aufgeführt sind.

WICHTIG! Die Taste, die Sie zuordnen (Taste1), ist nicht mehr als ihr eigentliches Tastensymbol verwendbar, sobald Sie diese Funktion verwenden. Wenn Sie z. B. den Pfeil nach oben auf "W" abbilden, können Sie das Drücken der Taste "W" nicht mehr als "W" erkennen. der Taste "W" nicht mehr als "W" erkannt werden, sondern wird immer als Aufwärtspfeil betrachtet. Um dies rückgängig zu machen, ordnen Sie entweder die Taste sich selbst zu - so wären sowohl Taste1 als auch Taste2 ord("W") - oder verwenden Sie die Funktion keyboard_unset_map().

Syntax:

keyboard_set_map(key1, key2);

Argument Beschreibung
Taste1 Dies ist der Schlüssel, auf den Taste1 abgebildet werden soll
Taste2 Dies ist die Taste, die abgebildet werden soll

Rückgabe:

Boolesche

Beispiel:

keyboard_set_map(ord("A"), vk_left);

Im obigen Beispielcode wird die Taste "A" der linken Pfeiltaste zugeordnet. Das bedeutet, dass der Spieler entweder die Taste "A" oder die linke Pfeiltaste verwenden kann und dass der gesamte Code, der für den linken Pfeil geschrieben wurde, auch auf die Taste "A" Taste reagiert.
