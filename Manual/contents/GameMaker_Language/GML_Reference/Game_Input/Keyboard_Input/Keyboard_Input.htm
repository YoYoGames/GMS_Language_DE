Tastatur-Eingabe

Für den Umgang mit der Tastatur in GameMaker Studio 2 gibt es eine Vielzahl von Funktionen, mit denen verschiedene Tastaturzustände wie gedrückt oder losgelassen erkannt werden können. Es gibt auch einige, die alle Tastendrücke als String speichern oder die Es gibt auch einige, die alle Tastendrücke als String speichern oder die Ihnen sagen, was die zuletzt gedrückte Taste war, sowie andere, mit denen Sie den Tastaturstatus komplett löschen können.

HINWEIS: Diese Funktionen sind nur für Windows/Mac/Ubuntu-Desktop-Plattformen konzipiert. Es kann sein, dass einige der eingebauten Variablen und Konstanten auf anderen Plattformen nicht gültig sind und viele der Funktionen auf Mobiltelefonen nicht funktionieren.

Jedes Eingabezeichen von einer Taste (oder mehreren Tasten) wird durch seinen UTF8-Code definiert, der ein numerischer Wert ist. Dieser Wert kann für jedes Zeichen mit der Funktion ord() Funktion abgerufen werden, aber GameMaker Studio 2 hat auch eine Reihe von Konstanten für die am häufigsten verwendeten Sondertasten der Tastatur und eine spezielle Funktion. Typischerweise würden Sie eine Kombination aus ord( ) mit den keyboard_check*() -Funktionen verwenden, etwa so:

if keyboard_check(ord("A"))
{
hspeed = -5;
}

Die obige Funktion prüft also die Taste "A" und wenn sie gedrückt wird, wird die horizontale Geschwindigkeit des Objekts auf -5 gesetzt. Beachten Sie, dass die Verwendung von ord() auf diese Weise nur dann korrekt funktioniert, wenn die Eingabezeichenfolge nur ein Zeichen lang ist lang ist und eine Zahl von 0 bis 9 oder ein großgeschriebenes römisches Zeichen von A bis Z ist. Die Funktion ord() gibt einen vollständigen UTF8-Wert zurück, aber die Funktionen keyboard_check*() erkennen nur A - Z und 0 - 9. Was aber, wenn Sie die Pfeiltasten benutzen wollen? Oder wenn Sie eine Aktion mit der "Shift"-Taste ändern wollen? Nun, dafür hat GameMaker Studio 2 eine Reihe von vk_-Konstanten(vk_ steht für virtuelle Taste), die Sie anstelle von von ord() verwenden können:

Die folgenden Konstanten können nur mit keyboard_check_direct() verwendet werden:

Konstant Beschreibung

Im Folgenden finden Sie ein kleines Beispiel für die Verwendung der vk_ Konstanten:

if keyboard_check_pressed(vk_tab)
{
instance_create_layer(x, y, "Controllers", obj_Menu);
}

Der obige Code erkennt, ob die "Tabulatortaste" gedrückt wird , und erzeugt in diesem Fall eine Instanz des Objekts "obj_Menu".

Wenn Sie auf ein Tastenzeichen prüfen müssen, das nicht 0 - 9, A - Z oder eine der VK-Konstanten ist, dann sollten Sie eine der keyboard_*-Variablen prüfen, wie z. B. keyboard_lastchar:

var _key = keyboard_lastchar;
if ord(_key) == ord("ç")
{
show_debug_message("ç-Taste gedrückt");
}
