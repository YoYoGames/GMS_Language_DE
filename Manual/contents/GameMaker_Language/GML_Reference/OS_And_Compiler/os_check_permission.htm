os_check_permission

Mit dieser Funktion können Sie prüfen, ob dem Spiel vom Benutzer eine bestimmte Berechtigung erteilt wurde. Sie geben die zu prüfende Berechtigung als String im Format " android.permission.<Permission> " an, d. h., um z. B. die RECORD_AUDIO-Berechtigung zu prüfen, würden Sie

os_check_permission("android.permission.RECORD_AUDIO");

Die Funktion gibt eine von drei Konstanten zurück - siehe unten - um dem Spiel mitzuteilen, wie der Status der Berechtigung ist. Weitere Informationen zu App-Berechtigungen finden Sie in der Android-Dokumentation.

WICHTIG! Diese Funktion ist nur für das Android nur für das Ziel.

Syntax:

os_check_permission(erlaubnis)

Argument Beschreibung
Genehmigung Die zu prüfende Berechtigung (eine Zeichenkette)

Rückgabe:

Konstant

Konstant Beschreibung
os_erlaubnis_erteilt Dies zeigt an, dass die Genehmigung erteilt wurde
os_erlaubnis_verweigert Dies zeigt an, dass die Genehmigung nicht erteilt wurde
os_erlaubnis_verweigert_keine_anforderung Dies zeigt an, dass die Berechtigung entweder durch die Telefoneinstellungen blockiert wurde oder dass der Benutzer die Anfrage zuvor abgelehnt und "Nicht mehr nachfragen" ausgewählt hat.

Beispiel:

if (os_type == os_android)
{
if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
{
os_request_permission("android.permission.INTERNET");
}
}

Der obige Code prüft den Betriebssystemtyp und wenn es sich um Android handelt, führt er eine Überprüfung der Berechtigungen durch und fordert die Berechtigung "INTERNET" an, wenn sie noch nicht erteilt wurde.
