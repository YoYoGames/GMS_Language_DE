os_get_info

Diese Funktion gibt ein DS-Map mit detaillierten Informationen über das Betriebssystem, auf dem das Spiel läuft. Die genauen Informationen, die zurückgegeben werden, hängen vom Betriebssystem und dem Gerät ab, daher wird empfohlen dass Sie diese Funktion auf allen Plattformen testen, die Sie ansprechen möchten, und die zurückgegebenen Werte mit einer Funktion wie show_debug_message() ausgeben. Beachten Sie, dass die DS-Map nicht nicht automatisch aus dem Speicher gelöscht wird und Sie sollten die Funktion ds_map_destroy() Funktion verwenden, wenn Sie die darin enthaltenen Daten nicht mehr benötigen.

Nachfolgend finden Sie einige Details zu den zusätzlichen Daten, die der DS-Karte speziell für die folgenden Plattformen zur Verfügung gestellt werden, wobei zu beachten ist, dass die Karte auf allen Plattformen außer HTML5 den Schlüssel "is64bit" haben wird, der entweder true oder false sein wird, je nachdem, ob das Betriebssystem der Plattform 64bit ist oder nicht.

Windows / UWP

Auf Windows Desktop und Windows UWP erhalten Sie einige zusätzliche Daten von DX11 mit den folgenden Tasten, die in der DS-Map verfügbar sind (nützlich für Erweiterungen):

  • video_d3d11_device - Zeiger auf das DX11-Gerät
  • video_d3d11_context - Zeiger auf den DX11-Kontext
  • video_adapter_vendorid - String mit der Herstellerkennung des Adapters
  • video_adapter_deviceid - String mit der Geräte-ID des Adapters
  • video_adapter_subsysid - String mit der Adapter-Subsystem-ID
  • video_adapter_revision - String mit der Revisionsnummer des Adapters
  • video_adapter_dedicatedvideomemory - String mit der Größe des Videospeichers
  • video_adapter_dedicatedsystemmemory - String mit der Größe des Systemspeichers (wird vom DX11-Adapter verwendet)

Android

Auf der Android Plattform erhalten Sie einige zusätzliche Daten von verschiedenen APIs mit den folgenden Schlüsseln, die in der DS-Map verfügbar sind:

  • android_tv - dies gibt true zurück, wenn das Gerät als Android-TV-Gerät erkannt wird, oder false andernfalls
  • GL_VERSION - die Version von OpenGL, wie vom Treiber gemeldet
  • GL_VENDOR - der Hersteller von OpenGL, wie vom Treiber gemeldet
  • GL_RENDERER - der Renderer von OpenGL, wie vom Treiber gemeldet
  • GL_EXTENSIONS - die Erweiterungen von OpenGL, die laut Meldung des Treibers verfügbar sind
  • GL_SHADING_LANGUAGE_VERSION - die Version der GLSL, die von diesem OpenGL-Treiber unterstützt wird
  • GL_MAX_TEXTURE_SIZE - die maximale Texturgröße, die von diesem OpenGL-Treiber unterstützt wird
  • SDK_INT - Wert von android.os.Build.VERSION.SDK_INT siehe hier für weitere Informationen.
  • RELEASE - Wert von android.os.Build.VERSION.RELEASE siehe hier für weitere Informationen.
  • MODEL - Wert von android.os.Build.MODEL siehe hier für weitere Informationen.
  • DEVICE - Wert von android.os.Build.DEVICE siehe hier für weitere Informationen.
  • MANUFACTURER - Wert von android.os.Build.MANUFACTURER siehe hier für weitere Informationen.
  • CPU_ABI - Wert von android.os.Build.CPU_ABI siehe hier für weitere Informationen.
  • CPU_ABI2 - Wert von android.os.Build.CPU_ABI2 siehe hier für weitere Informationen.
  • BOOTLOADER - Wert von android.os.Build.BOOTLOADER siehe hier für weitere Informationen.
  • BOARD - Wert von android.os.Build.BOARD siehe hier für weitere Informationen.
  • VERSION - Wert von os.version aus der Methode Android System.getProperty
  • REGION - Werte von user.region aus der Android System.getProperty Methode
  • VERSION_NAME - Wert des Versionsnamens dieses Pakets - siehe hier für weitere Informationen.
  • PHYSICAL_KEYBOARD - wenn wir denken, dass eine physische Tastatur vorhanden ist, dann die Zeichenkette "TRUE" sonst "FALSE"

Syntax:

os_get_info()

Rückgabe:

Real (ds_map index)

Beispiel:

if (os_type == os_android)
{
var _info = os_get_info();

if (_info[? "android_tv"])
{
global.android_tv = true;
}
}

Der obige Code prüft, ob das aktuelle Betriebssystem Android ist, holt sich die OS-Informationen und liest die zurückgegebene Karte, um zu prüfen, ob das Spiel auf einem Android-TV läuft. In diesem Fall setzt er eine globale Variable auf true.
