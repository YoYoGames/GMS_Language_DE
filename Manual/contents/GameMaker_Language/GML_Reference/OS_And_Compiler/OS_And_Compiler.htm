OS und Compiler

Beim Erstellen von plattformübergreifenden Spielen ist es oft von entscheidender Bedeutung, dass Sie die Details des Betriebssystems erhalten, das das Gerät, auf dem Ihr Spiel läuft, verwendet. Dinge wie die Sprache, die Version oder der Netzwerkstatus können genutzt werden, um Ihr GameMaker Studio 2 verfügt daher über eine Reihe spezieller Funktionen, mit denen Sie die erforderlichen Informationen abrufen können.

Die folgenden Funktionen und Variablen stehen Ihnen zur Verfügung, um bestimmte Details über das Betriebssystem oder den Browser zu erhalten, auf dem Ihr Spiel läuft:

  • os_browser
  • os_device
  • Os_Typ
  • os_version
  • os_ist_pausiert
  • os_is_network_connected
  • os_get_config
  • os_get_language
  • os_get_region
  • os_get_info
  • os_powersave_enable
  • os_lock_orientation
  • os_check_permission
  • os_request_permission

Bestimmte Funktionen werden vorkompiliert, wenn Sie Ihr Spiel ausführen (d. h.: sie werden verwendet, um zu definieren, wie das endgültige Spiel kompiliert wird), und einige sind auch zur Verwendung vorgesehen, wenn das Spiel kompiliert wurde, um spezifische Details über die Laufzeitumgebung zu erhalten. Hier finden Sie die vollständige Liste dieser Funktionen:

  • GM_build_date
  • GM_Version
  • GM_Laufzeit_Version
  • gml_freigabe_mode
  • gml_pragma
  • Parameter_Zahl
  • Parameter_Zeichenfolge
  • umwelt_get_variable

Unter Windows ist die Auflösung des Windows-Thread-Schedulers standardmäßig auf 1 ms eingestellt, wenn Ihr Spiel läuft. Die folgenden Funktionen sind verfügbar, um die Auflösung des Windows-Thread-Schedulers zur Laufzeit abzurufen und zu ändern:

  • scheduler_resolution_get
  • scheduler_resolution_set

Schließlich müssen Sie bei der Arbeit mit Erweiterungen möglicherweise in der Lage sein, externe Funktionen zur Laufzeit zu definieren und aufzurufen, und daher würden Sie Folgendes verwenden

HINWEIS: Dies sind Legacy-Funktionen und Sie sollten alle Erweiterungsfunktionen im Erweiterungseditor selbst definieren.

HINWEIS: Diese Funktionen sind für die Windows und macOS nur für die Zielplattformen.

  • extern_definieren
  • externer_Aufruf
  • extern_frei
