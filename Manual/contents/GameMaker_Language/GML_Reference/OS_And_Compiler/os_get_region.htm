os_get_region

Diese Funktion gibt eine Zeichenkette mit dem zwei- oder dreibuchstabigen Regionalcode für das Betriebssystem zurück, auf dem das Spiel läuft, wie im ISO3166-1-Standard festgelegt. Wenn die Information nicht verfügbar ist, enthält sie einfach eine leere Zeichenkette "".

HINWEIS: Dies ist nicht der Regionalcode des Standorts, der zurückgegeben wird, sondern der regionale Sprachcode des Betriebssystems.

Syntax:

os_get_region()

Rückgabe:

Zeichenfolge

Beispiel:

switch (os_get_language())
{
case "zh":
var region = os_get_region();
if (region == "HK" || region == "MO" || region == "TW")
{
ini_open("chinese_traditional.ini");
}
sonst
{
ini_open("chinese_simplified.ini");
}
break;
case "fr":
ini_open("french.ini");
break;
case "it":
ini_open("italienisch.ini");
break;
default:
ini_open("english.ini");
break;
}

Der obige Code prüft die Sprache des Betriebssystems und wenn es Chinesisch ist, prüft er anschließend die Betriebssystemregion und öffnet je nach den zurückgegebenen Werten eine andere *.ini-Datei.
