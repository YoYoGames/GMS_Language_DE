os_is_network_connected

Mit dieser Funktion können Sie prüfen, ob Ihr Gerät derzeit eine Internetverbindung hat. Wenn ja, gibt sie true zurück, wenn nicht, false. Abhängig vom Betriebssystem kann sie versuchen, eine Verbindung herzustellen, bevor sie einen Wert zurück.

Die Funktion hat ein optionales Argument zum Aktivieren/Deaktivieren des Verbindungsversuchs beim Aufruf, und diese Option ist derzeit nur für das Nintendo Switch-Ziel. Wenn sie auf true gesetzt ist, versucht die Funktion eine Verbindung herzustellen, und wenn sie auf false gesetzt ist, beschränkt sie sich darauf, die Verbindung zu überprüfen. Beachten Sie, dass der Versuch, eine Verbindung herzustellen, dazu führen kann, dass das Switch-Betriebssystem für ein paar Sekunden hängen bleibt.

HINWEIS: Diese Funktion prüft die interne Geräte-API, die Verbindungen steuert, und kann daher true zurückgeben, wenn eine Bluetooth-Verbindung, eine Wi-Fi-Verbindung oder auch nur eine normale Netzwerkverbindung besteht, die den Internetzugang erlaubt.

Syntax:

os_is_network_connected(attempt_connection);

Argument Beschreibung
Versuch_Verbindung OPTIONAL! Nur für Nintendo Switch Target. Setzen Sie diese Option auf true, um beim Aufruf eine Verbindung auf OS-Ebene zu versuchen.

Rückgabe:

Boolesche

Beispiel:

if os_is_network_connected()
{
global.connected = true;
}

Der obige Code prüft, ob das Gerät eine Verbindung zum Internet hat und wenn ja, setzt er eine globale Variable.
