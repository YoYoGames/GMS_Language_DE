Kollisionen

Bei der Planung von Bewegungen oder der Entscheidung für bestimmte Aktionen ist es oft wichtig zu sehen, ob es an bestimmten Stellen in der Spielwelt Kollisionen mit anderen Objekten gibt, und oft ist die Auswahl der richtigen Kollision für die Aufgabe die wichtigste Aufgabe von allen. GameMaker Studio 2 verfügt über eine Reihe von eingebauten Funktionen, die Ihnen helfen, mit Kollisionen korrekt und entsprechend den Anforderungen Ihres Projekts umzugehen.

WICHTIG! Alle Kollisionsfunktionen arbeiten mit ganzzahligen Positionen, unabhängig von den Eingabewerten. Alle nicht-ganzzahligen Werte werden auf die nächste Ganzzahl gerundet.

Die folgenden Funktionen behandeln die verschiedenen Möglichkeiten, wie Sie eine Instanz auf Kollisionen mit einer anderen Instanz prüfen lassen können:

  • platz_leer
  • platz_frei
  • ort_besprechung
  • position_leer
  • position_meeting
  • position_change
  • position_destroy
  • Instanz_Ort
  • instanz_platz_liste
  • Instanz_Position
  • instanz_position_liste

Erweiterte Kollisionsprüfung

Dazu können die folgenden Routinen verwendet werden (sowie weitere Funktionen, die in den Abschnitten zu " Moving Around" und " Instances" beschrieben sind). Diese Kollisionsfunktionen arbeiten alle etwas anders, aber sie haben drei gemeinsame Argumente die wir hier erläutern werden:

  • obj. Hier geben wir das Objekt an, mit dem die Funktion auf eine Kollision prüfen soll. Es kann eine Instanz-ID, ein Objekttyp oder das spezielle GM-Schlüsselwort all sein. Es ist auch erwähnenswert, dass, wenn der gewählte Objekttyp der designierte Elternteil anderer Objekte ist, dann werden auch alle Kinder dieses Objekts in die Prüfung einbezogen.
  • prec. Dies kann entweder true oder false sein und bezieht sich auf die Genauigkeit der Kollisionsprüfung. Bei true werden präzise Kollisionen berechnet (aber nur, wenn das zu prüfende Objekt in seinen Maskeneigenschaften "precise" für seine Kollisionserkennung in den Maskeneigenschaften markiert hat), was eine etwas komplexere Aufgabe für die CPU darstellt und daher mit Vorsicht verwendet werden sollte. Wenn false, dann werden Kollisionen auf der Grundlage des Begrenzungsrahmens des zu prüfenden Objekts berechnet für.
  • notme. Möglicherweise müssen Sie nach Kollisionen mit einem Objekt suchen, das denselben Index hat wie das Objekt, das den Code ausführt. Um zu verhindern, dass der Code eine Kollision mit sich selbst findet, können Sie dies auf true setzen. Wenn Sie speziell nach nach der Instanz des Objekts suchen, das den Code ausführt, oder es nicht wichtig ist, können Sie den Wert auf false setzen.

Die Basisfunktionen geben entweder die ID einer kollidierenden Instanz oder das spezielle Schlüsselwort noone zurück, wenn keine Kollision vorliegt, während die Listenfunktionen die Anzahl der kollidierenden Instanzen zurückgeben und eine vorgefertigte DS-Liste mit deren IDs auffüllen. eine vorgefertigte DS-Liste mit ihren IDs. Beachten Sie, dass, wenn es mehrere Kollisionen mit den von diesen Funktionen definierten Bereichen und Instanzen des angegebenen Objekts gibt, nur eine Instanz-ID zurückgegeben wird, und diese kann eine beliebige der Instanzen in der Kollision sein.

Es gibt die folgenden Funktionen, die sich mit erweiterten Kollisionen befassen.

  • kollision_kreis
  • kollision_kreis_liste
  • kollision_ellipse
  • kollision_ellipse_liste
  • kollisions_linie
  • kollisions_linien_liste
  • kollisions_punkt
  • kollisionspunkt_liste
  • Kollisions_Rechteck
  • kollision_rechteck_liste

Kollisionsprüfung ohne Maske

Alle oben genannten Funktionen beziehen sich auf Kollisionsprüfungsinstanzen und sind als solche auf die Kollisionsmaske angewiesen, die für die Instanz definiert ist. Es gibt jedoch viele Momente, in denen Sie auf "Kollisionen" mit einem Punkt oder einem Bereich zu überprüfen, insbesondere wenn Ihrer Instanz kein Sprite zugewiesen ist, oder wenn Sie mit der Maus arbeiten usw... Deshalb bietet GameMaker Studio 2 auch die folgenden Funktionen, die Ihnen in diesen Situationen helfen:

  • Punkt_im_Rechteck
  • Punkt_im_Dreieck
  • Punkt_im_Kreis
  • Rechteck_im_Rechteck
  • Rechteck_im_Dreieck
  • Rechteck_im_Kreis

Physik Kollisionen

Wenn Sie die eingebaute Physik verwenden, funktionieren die oben genannten Kollisionsfunktionen bei physikfähigen Instanzen nicht garantiert. Das liegt vor allem daran, dass diese Instanzen die meisten der regulären eingebauten Variablen nicht mehr verwenden (stattdessen haben Physik aktivierte Instanzen haben stattdessen einen eigenen Satz von Variablen) und sie verwenden auch nicht die Kollisionsmaske oder die Bounding Box, da sie stattdessen Fixtures verwenden. Es gibt jedoch eine spezielle Funktion zum Testen von Kollisionen mit aktivierter Physik, die auf überlappende Fixtures prüft:

  • physik_test_überlappung
