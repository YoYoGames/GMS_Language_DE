kollisions_punkt

Collision_point prüft den durch die Argumente x1,y1 angegebenen Punkt auf eine Kollision mit einer beliebigen Instanz des durch das Argument "obj" angegebenen Objekts. Diese Prüfung kann entweder präzise oder nicht präzise sein, aber damit präzise Kollisionen aktiviert werden, muss für das Objekt oder die Instanz, auf die geprüft wird, auch die Option "Präzise Kollisionen" für das Sprite aktiviert sein. Wenn nicht, basiert die Standardprüfung auf Bounding Boxes. Das folgende Bild veranschaulicht, wie dies funktioniert:

Denken Sie daran, dass für die Berücksichtigung präziser Kollisionen sowohl das Objekt-Sprite und die Kollisionsfunktion als präzise markiert sein müssen. Es sollte auch beachtet werden, dass der Rückgabewert der Funktion die ID einer beliebigen der Instanzen sein kann, die als kollidierend betrachtet werden, wenn sich also drei Instanzen an diesem Punkt überschneiden, kann jede Wenn sich also drei Instanzen an diesem Punkt überschneiden, könnte jede ihrer IDs der Rückgabewert der Funktion sein.

Beachten Sie, dass Sie anstelle eines Objektindexes auch eine Instanz id angeben können, um nach einer bestimmten Instanz zu suchen, oder die Instanz-Schlüsselwörter all bzw. other (je nach Ereignis und aktuellem Codeumfang).

Syntax:

collision_point(x, y, obj, prec, notme);

Argument Beschreibung
x Die x-Koordinate des zu prüfenden Punktes.
y Die y-Koordinate des zu prüfenden Punktes.
obj Das Objekt, das auf Instanzkollisionen geprüft werden soll.
prec Ob die Prüfung auf präzisen Kollisionen(true, was langsamer ist) oder auf der Bounding Box im Allgemeinen(false, schneller) basiert.
notme Ob die aufrufende Instanz, falls relevant, ausgeschlossen werden soll(true) oder nicht(false).

Rückgabe:

Instanz-ID oder keine

Beispiel:

if collision_point(x, y, obj_Cursor, false, true)
{
Spielstand += 10S;
}

Hier prüfen wir den Punkt an der Position des Objekts, das den Code für das Objekt "obj_Cursor" hat. Wenn es einen gibt, dann addieren wir 10 auf die Score-Variable.
