Rechteck_im_Dreieck

Diese Funktion prüft eine von Ihnen definierte rechteckige Fläche darauf, ob sie entweder nicht kollidiert, vollständig innerhalb der Zielgrenzen liegt oder ob sie lediglich eine definierte dreieckige Fläche berührt. Wenn sie sich überhaupt nicht berühren, wird die Funktion 0 zurück, wenn die Quelle vollständig innerhalb des Ziels liegt, wird 1 zurückgegeben, und wenn sie sich einfach nur überlappen, wird 2 zurückgegeben. Das Bild unten veranschaulicht dies:

Syntax:

rechteck_in_dreieck(sx1, sy1, sx2, sy2, dx1, dy1, dx2, dy2, dx3, dy3);

Argument Beschreibung
sx1 Die x-Koordinate der linken Seite des Quellrechtecks.
sy1 Die y-Koordinate der oberen Seite des Quellrechtecks.
sx2 Die x-Koordinate der rechten Seite des Quellrechtecks.
sy2 Die y-Koordinate der unteren Seite des Quellrechtecks.
x1 Die x-Koordinate der ersten Seite des zu prüfenden Dreiecks.
y1 Die y-Koordinate der ersten Seite des zu prüfenden Dreiecks.
x2 Die x-Koordinate der zweiten Seite des zu prüfenden Dreiecks.
y2 Die y-Koordinate der zweiten Seite des zu prüfenden Dreiecks.
x3 Die x-Koordinate der dritten Seite des zu prüfenden Dreiecks.
y3 Die y-Koordinate der dritten Seite des zu prüfenden Dreiecks.

Rückgabe:

Real

Beispiel:

var inst = instance_nearest(x, y, obj_Player);
if instance_exists(inst)
{
var x1 = x + lengthdir_x(100, image_angle - 45);
var y1 = y + lengthdir_y(100, image_angle - 45);
var x2 = x + lengthdir_x(100, image_angle + 45);
var y2 = y + lengthdir_y(100, image_angle + 45);
if rectangle_in_triangle(inst.bbox_left, inst.bbox_top, inst.bbox_right, inst.bbox_bottom, x, y, x1, y1, x2, y2) == 1
{
can_see = true;
}
}

Der obige Code verwendet die Funktion rectangle_in_triangle als "Sichtkegel", um nach einer Instanz von "obj_player" zu suchen. Wenn das vollständige Begrenzungsrechteck innerhalb des angegebenen dreieckigen Bereichs liegt, wird eine Variable auf true gesetzt.
