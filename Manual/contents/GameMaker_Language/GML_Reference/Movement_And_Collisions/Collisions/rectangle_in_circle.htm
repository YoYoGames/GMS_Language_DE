Rechteck_im_Kreis

Diese Funktion prüft einen von Ihnen definierten rechteckigen Bereich darauf, ob er entweder nicht kollidiert, vollständig innerhalb der Zielgrenzen liegt oder ob er lediglich eine definierte Kreisfläche berührt. Wenn sie sich überhaupt nicht berühren, gibt die Funktion 0 zurück, wenn die Quelle vollständig innerhalb des Ziels liegt, gibt sie 1 zurück, und wenn sie sich einfach überlappen, gibt sie 2 zurück. Das Bild unten veranschaulicht dies:

Syntax:

Rechteck_im_Kreis(sx1, sy1, sx2, sy2, x, y, rad);

Argument Beschreibung
sx1 Die x-Koordinate der linken Seite des Quellrechtecks.
sy1 Die y-Koordinate der oberen Seite des Quellrechtecks.
sx2 Die x-Koordinate der rechten Seite des Quellrechtecks.
sy2 Die y-Koordinate der unteren Seite des Quellrechtecks.
x Die x-Koordinate des Kreismittelpunkts
y Die y-Koordinate des Mittelpunkts des Kreises.
rad Der Radius um den Mittelpunkt, in dem auf eine Kollision geprüft werden soll.

Rückgabe:

Real

Beispiel:

inst = instance_nearest(x, y, obj_Bullet);
if instance_exists(inst)
{
if rectangle_in_circle(inst.x - 5, inst.y - 5, inst.x + 5, inst.y + 5, x, y - 25, 20) > 0
{
hit = true;
}
}

Der obige Code verwendet die Funktion rectangle_in_circle, um auf eine Kollision innerhalb eines kreisförmigen Bereichs und des Rechtecks um eine gefundene Instanz zu prüfen. Wenn es eine Kollision gibt (entweder eine Kantenüberschneidung oder umschlossen), wird eine Variable auf true gesetzt.
