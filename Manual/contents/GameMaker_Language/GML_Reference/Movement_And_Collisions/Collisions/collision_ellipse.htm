kollision_ellipse

Collision_ellipse verwendet die ersten vier Argumente (x1,y1,x2,y2), um die Breite und Höhe einer Ellipse innerhalb des aktuellen Raums zu definieren, und prüft dann, ob irgendein Objekt, das durch das Argument "obj" definiert ist definiert ist, in Kollision mit diesem Bereich ist. Diese Kollision kann als genau oder nicht geprüft werden, und Sie können auch wählen, ob die Instanz, die den Code ausführt, selbst geprüft werden soll oder nicht. Betrachten Sie dieses Bild:

Hier verwendet die Instanz in der Mitte eine Kollisionsellipse zur Prüfung auf Ball-Objekten. Die blauen Objekte haben keine präzise Bounding Box und wie Sie sehen können, kann die Kollision auch dann stattfinden, wenn das Sprite die Ellipse nicht berührt (selbst wenn Sie die Option precise in der Funktion auf true setzen), da das Begrenzungsrechteck dieses Sprites den durch collision_circle definierten elliptischen Bereich überlappt. Andererseits werden die grünen Kugeln nur dann als Kollisionen betrachtet, wenn das aktuelle Sprite die definierte Ellipse überlappt. Denken Sie daran, dass für präzise Kollisionen berücksichtigt werden sollen, müssen sowohl das Objekt-Sprite als auch die Kollisionsfunktion als präzise markiert sein. Es sollte auch beachtet werden, dass der Rückgabewert der Funktion die ID einer der Instanzen sein kann, die als kollidierend betrachtet werden.

Beachten Sie, dass Sie anstelle eines Objektindexes auch eine Instanz id angeben können, um nach einer bestimmten Instanz zu suchen, oder die Instanzschlüsselwörter all oder other (je nach Ereignis und aktuellem Codeumfang).

Syntax:

collision_ellipse(x1, y1, x2, y2, obj, prec, notme);

Argument Beschreibung
x1 Die x-Koordinate der linken Seite der zu prüfenden Ellipse.
y1 Die y-Koordinate der oberen Seite der zu prüfenden Ellipse.
x2 Die x-Koordinate der rechten Seite der zu prüfenden Ellipse.
y2 Die y-Koordinate der unteren Seite der zu prüfenden Ellipse.
obj Das Objekt, das auf Instanzkollisionen geprüft werden soll.
prec Ob die Prüfung auf präzisen Kollisionen(true, was langsamer ist) oder allgemein auf der Bounding Box basiert(false, schneller).
notme Ob die aufrufende Instanz, falls relevant, ausgeschlossen werden soll(true) oder nicht(false).

Rückgabe:

Instanz-ID oder keine

Beispiel:

if collision_ellipse(50, 50, 200, 100, obj_Player, false, true)
{
instance_create_layer(obj_Player.x, obj_Player.y, "Effekte", obj_Splash);
}

Es wird eine elliptische Zone innerhalb der Grenzen von 50x, 50y und 200x, 100y für das Objekt "obj_Player" geprüft. Wenn es eine Kollision mit diesem Objekt gibt, dann wird eine Instanz von "obj_Splash" an den x/y-Koordinaten von obj_Player erzeugt.
