mp_grid_create

Mit dieser Funktion können Sie ein mp_grid für die Bewegungsplanungsfunktionen erstellen. Sie gibt einen Index zurück, der in einer Variablen für die Verwendung in allen anderen MP-Grid-Funktionsaufrufen gespeichert werden sollte. Die x- und y-Koordinaten geben die Position der linken, oberen Ecke Die x- und y-Koordinaten geben die Position der linken oberen Ecke des Gitters an, hcells und vcells geben die Anzahl der horizontalen und vertikalen Zellen an, die innerhalb des Gitters erstellt werden sollen, und cell width und cell height geben die Größe der Zellen in Pixeln an. Sie können mehrere Gitterstrukturen gleichzeitig erstellen und pflegen Sie können mehrere Gitterstrukturen gleichzeitig erstellen und pflegen, wenn Sie sie für verschiedene Zwecke benötigen. Beachten Sie jedoch, dass jedes Gitter Speicherplatz benötigt, und je mehr Zellen es hat, desto mehr Speicherplatz wird benötigt. Beachten Sie auch, dass die MP-Funktionen recht prozessorlastig sind, je öfter je häufiger Sie sie aufrufen und je höher die Zellenauflösung ist, desto wahrscheinlicher ist ein Leistungsabfall.

Syntax:

mp_grid_create(left, top, hcells, vcells, cellwidth, cellheight):

Argument Beschreibung
links Start-X-Koordinate des mp_grid im Raum
oben Start-Y-Koordinate des mp_grid im Raum
hZellen Anzahl der horizontalen Zellen, die das mp_grid enthalten wird
vcells Anzahl der vertikalen Zellen, die das mp_grid enthalten wird
Zellbreite Die Breite (in Pixel) jeder einzelnen Zelle des mp_grid
Zellenhöhe Die Höhe (in Pixel) jeder einzelnen Zelle des mp_grid

Rückgabe:

Real

Beispiel:

global.grid = mp_grid_create(0, 0, room_width / 32, room_height /32, 32, 32);

Der obige Code erzeugt eine globale Variable "grid", generiert dann ein mp_grid und weist dessen Index (id) dieser Variablen zur Verwendung in allen weiteren mp_grid-Funktionsaufrufen zu. Das Raster deckt den Raum ab, mit einer Zellenauflösung von 32x32 Pixeln. Das bedeutet Das bedeutet, dass das Raster z. B. in einem 640x480 großen Raum 300 Zellen enthalten würde: 20 horizontale Zellen (640 / 32) multipliziert mit 15 vertikalen Zellen (480 /32) ergibt 300 Zellen
