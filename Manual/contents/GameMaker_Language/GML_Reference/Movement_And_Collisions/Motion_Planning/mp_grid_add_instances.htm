mp_grid_add_instances

Diese Funktion verwendet die Instanz-Kollisionsmaske (entschieden durch die Sprite-Eigenschaften oder den Maskenindex der aufrufenden Instanz), um Zellen in einem MP-Gitter als verboten oder nicht verboten zu markieren, wobei verbotene Zellen von keiner der Pathfinding-Funktionen gekreuzt werden können. Sie können in der Funktion angeben, ob präzise Kollisionen berücksichtigt werden sollen oder nicht und die markierten verbotenen Zellen ändern sich in Abhängigkeit von dieser Einstellung. Dieses Bild veranschaulicht dieses Verhalten:

Die beiden obigen Instanzen wurden mit der Funktion mp_grid_add_instances() in das MP-Gitter eingefügt, wobei precise auf true gesetzt wurde. Wie Sie sehen können, hat die grüne Instanz nur die Gitterquadrate, die sie "berührt", als verboten markiert, da ihre Sprite-Maske auch auf präzise eingestellt ist. Die zweite Instanz hat jedoch auch andere Zellen markiert, die sich nicht zu berühren scheinen. Das liegt daran, dass die Sprite-Maske nicht auf präzise eingestellt ist, d. h., auch wenn Sie die präzise Einstellung in der Funktion verwenden, wird nur die Bounding-Box des Sprites berücksichtigt. Dies würde auch passieren, wenn Sie das precise-Argument der Funktion auf false setzen - selbst Instanzen mit einer präzisen Sprite-Maske werden anhand ihrer Bounding-Boxen in das Raster eingefügt.

Syntax:

mp_grid_add_instances(id, obj, prec);

Argument Beschreibung
id Index des mp_grid, das verwendet werden soll
obj Objektindex bzw. Instanz-ID der Instanzen, die in das mp_grid eingefügt werden sollen
prec Ob die Prüfung auf pixelgenauen Kollisionen (true = langsam) oder allgemein auf der Bounding Box basiert (false = schnell).

Rückgabe:

N/A

Beispiel:

mp_grid_add_instances(grid, obj_Wall, true);

Der obige Code fügt alle Instanzen von "obj_Wall" in das mp_grid ein, das in der Variablen "grid" indiziert ist, wobei die genaue Kollisionsmaske und nicht die Bounding Box verwendet wird.
