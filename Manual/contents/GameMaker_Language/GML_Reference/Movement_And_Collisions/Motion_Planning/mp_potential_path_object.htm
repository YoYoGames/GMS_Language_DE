mp_potential_pfad_Objekt

Diese Funktion berechnet einen Pfad für die Instanz von ihrer aktuellen Position und Orientierung zu der durch die Argumente xgoal, ygoal angegebenen Position. Sie verwendet die angegebene Schrittweite und versucht dabei, Kollisionen mit Instanzen des durch das Argument "obj" angegebenen Objekts zu vermeiden. durch das Argument "obj" angegebenen Objekt zu vermeiden. Wie bei der Funktion mp_potential_step() verwendet sie dazu Potenzialfeldschritte und die Art und Weise, wie sie dies tut, kann durch die Funktion mp_potential_settings(). Der angegebene Pfad muss bereits existieren und wird durch den neuen Pfad überschrieben. Die Funktion gibt zurück, ob ein Pfad gefunden wurde (true) oder nicht (false).

Um zu vermeiden, dass die Funktion ewig weiterrechnet, müssen Sie einen Faktor größer als 1 angeben - die Funktion hält an und meldet einen Fehler, wenn sie keinen kürzeren Weg als diesen Faktor mal die Entfernung zwischen Start und Ziel finden kann. Ein Faktor von 4 ist normalerweise gut genug, aber wenn Sie lange Umwege erwarten, können Sie ihn auch größer wählen. Wenn die Funktion fehlschlägt, wird immer noch ein Pfad erstellt, der in die ungefähre Richtung des Ziels verläuft, dieses aber nicht erreichen wird.

HINWEIS: Mit dieser Funktion wird die Instanz nicht bewegt. Sie legt nur einen Pfad fest, und Sie müssen die Pfadfunktionen für die Bewegung verwenden.

Syntax:

mp_potential_pfad_objekt(pfad, xZiel, yZiel, Schrittweite, Faktor, obj)

Argument Beschreibung
Pfad Index des Pfades, der von der Funktion verwendet werden soll
xgoal Die Ziel-X-Position.
yZiel Die Ziel-Y-Position.
Schrittweite Die Geschwindigkeit, mit der sich die Instanz in Pixeln pro Schritt bewegt.
Faktor Diese Zahl wird verwendet, um eine Endlosschleife zu verhindern. Sie sollte größer als 1 sein. Siehe die Funktionsbeschreibung für weitere Informationen.
obj Das Objekt, das den Pfad sperren soll. Kann ein Objektindex, eine Instanz-ID oder das spezielle Schlüsselwort,all' sein

Rückgabe:

Boolesche

Beispiel:

path = path_add();
mp_potential_path_object(path, obj_Player.x, obj_Player.y, 3, 4, obj_Wall);
path_start(path, 3, 0, 0);

Der obige Code erzeugt eine neue Pfadressource und speichert deren Index in der Variablen "path". Er erzeugt dann mit mp_potential_path einen Pfad zwischen dem Objekt mit diesem Code und dem Objekt "obj_Player", wobei er auf Kollisionen mit dem Objekt "obj_Wall". Schließlich startet es das Objekt entlang dieses Pfades, auch wenn die Funktion keinen vollständigen Weg zum "obj_Player" zurückgegeben hat.
