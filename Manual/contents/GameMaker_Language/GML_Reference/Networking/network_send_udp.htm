netzwerk_senden_udp

Mit dieser Funktion können Sie Daten über das Netzwerk mittels UDP an einen Server senden. Die Funktion nimmt die Socket-ID, über die eine Verbindung hergestellt werden soll, die URL, zu der eine Verbindung hergestellt werden soll, und den zu verwendenden Port. Sie müssen dann die Puffer-ID angeben, die die zu sendenden Daten enthält. (weitere Informationen zu Puffern finden Sie unter Referenz - Puffer) und schließlich die Größe (in Bytes) der Daten. UDP ist "verbindungslos", d. h. Sie stellen keine Verbindung her, sondern senden nur ein Paket direkt an eine IP-Adresse, und der Server bekommt eingehende Daten von einer IP-Adresse und muss sie "so wie sie sind" verarbeiten. Die Funktion gibt die Anzahl der gesendeten Datenbytes zurück, oder eine Zahl kleiner 0, wenn das Senden fehlgeschlagen ist. Es ist erwähnenswert, dass die endgültige Größe der gesendeten Daten, die von dieser Funktion zurückgegeben wird, auch die GameMaker-Header-Informationen enthält, die zusätzlich 12 Byte betragen.

HINWEIS: Diese Funktion funktioniert nicht, wenn sie in einem Projekt verwendet wird, das auf dem HTML5-Target läuft, und HTML5-Projekte können auch kein UDP empfangen.

Syntax:

network_send_udp(socket, url, port, buffer, size);

Argument Beschreibung
Steckdose Die ID des zu verwendenden Sockets.
url Die URL oder IP, zu der eine Verbindung hergestellt werden soll (eine Zeichenfolge).
Hafen Der Anschluss, zu dem eine Verbindung hergestellt werden soll.
Puffer Die ID des Puffers, aus dem die Daten geholt werden sollen.
Größe Die Größe (in Bytes) der Daten.

Rückgabe:

Real

Beispiel:

network_send_udp(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff));

Der obige Code sendet ein UDP-Paket an den durch die URL definierten Server auf dem Port 6510. Die Daten werden dem Puffer entnommen, der in der Variablen "buff" indiziert ist.
