Vernetzung

Die Netzwerkfunktionen von GameMaker Studio 2 basieren auf Sockets, was ein Industriestandard auf allen Plattformen ist. Mit Sockets haben Sie die Möglichkeit, sowohl Clients als auch Server mit einfachem GML zu erstellen, wodurch Sie sogar ein Einzelspieler-Spiel im Client/Server-Modell erstellen können. Spieler-Spiel mit dem Client/Server-Modell zu erstellen (was später das einfache Hinzufügen von Multiplayer ermöglicht).

Grundsätzlich ist ein Socket ein Objekt, das senden, empfangen, verbinden und auf Ports im Netzwerk hören kann. Diese Funktionen befassen sich nur mit TCP/IP-Verbindungen, auf denen das Internet basiert (IPv4, um genau zu sein). Im Wesentlichen können Sie damit einen Socket mit einer IP-Adresse (z. B. 192.168.1.10) mit einem anderen Socket mit einer anderen IP-Adresse (z. B. 192.168.1.11) verbinden. Sobald die beiden Sockets verbunden sind, können sie Daten hin- und herschicken, indem sie die network_send_*-Funktionen (wie network_send_packet()) und dem dedizierten asynchronen Ereignis, das von ihnen ausgelöst wird. Dieses asynchrone Ereignis erzeugt eine DS-Map, die die empfangenen Daten sowie weitere Details enthält, und Sie finden die vollständigen Details finden Sie im Abschnitt über das asynchrone Netzwerkereignis.

Anstatt dass Programme jedes Stückchen Netzwerkverkehr, das auf einen Rechner zukommt, lesen und verarbeiten müssen, befassen sich IP-Adressen auch mit Ports, die von 0 bis 65535 durchnummeriert sind. Dies erlaubt jedem Programm, nur die Datenpakete zu erhalten, an denen es interessiert ist interessiert, und nicht alles, was von jedem Programm gesendet wurde. Dies spart auch erhebliche CPU-Zeit, da die Datenverarbeitung stark reduziert wird, so dass Sie mit GameMaker Studio 2 auch auf Ports "hören" können. Ein Server erstellt also einen Socket und bindet ihn dann bindet ihn an einen bestimmten Port zum Abhören. Er wird dann über Verbindungen, Verbindungsabbrüche und Datentransfers von diesem Socket benachrichtigt.

Die Daten-"Pakete" selbst bestehen aus Binärdaten, die einem Puffer entnommen werden. Dieser Puffer kann mit den Buffer-Funktionen des GameMaker Studio 2 erstellt und manipuliert werden.

Es ist erwähnenswert, dass Sie, wenn Sie ein Projekt für das HTML5-Ziel erstellen, die Funktionen zum Erstellen eines Servers aufgrund von Browser-Einschränkungen nicht verwenden können und einen eigenen Server für diese Plattform schreiben müssen, indem Sie etwas wie Electron oder NodeJS. Beachten Sie auch, dass, wenn Ihr Spiel plattformübergreifend ist und Sie eine Verbindung zu einer HTML5-Instanz des Spiels von einem Server aus planen, der auf einem mobilen oder Desktop-Gerät läuft, Ihr Server nur mit der HTML5-Instanz kommunizieren kann über das Web-Socket-Protokoll kommunizieren (dies geschieht über die Konstante network_socket_ws - siehe network_create_server() für weitere Informationen). Das bedeutet, dass Ihr Server möglicherweise zwei Sockets erstellen muss auf verschiedenen Ports erstellen muss, einen für die Verbindung mit Nicht-HTML5-Zielen(Android, Windows, etc...) und einen für die Verbindung mit dem HTML5-Ziel.
