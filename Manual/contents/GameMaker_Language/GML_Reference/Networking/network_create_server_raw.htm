netzwerk_erstellen_server_raw

Diese Funktion wird verwendet, um einen neuen Netzwerkserver für Ihr Spiel zu erstellen, der eines der zulässigen Verbindungsprotokolle verwendet (siehe die unten aufgeführten Konstanten). Sie geben den Servertyp an, dann einen Port, den er verwenden soll, und schließlich die Anzahl der maximalen Verbindungen die zu einem beliebigen Zeitpunkt zum Server zugelassen werden sollen (beachten Sie, dass diese Zahl Ihnen überlassen ist, aber zu viele verbundene Clients das Netzwerk sättigen oder die CPU des Geräts nicht in der Lage ist, die Verarbeitung dieser Anzahl von Spielern zu bewältigen, also verwenden Sie sie mit Vorsicht). Die Funktion gibt eine eindeutige ID zurück, die in einer Variablen gespeichert und zur Identifizierung des Servers in allen weiteren Netzwerkfunktionen verwendet werden sollte, oder einen Wert kleiner als 0, wenn die Verbindung fehlschlägt.

Konstant Beschreibung
network_socket_tcp Erstellen Sie einen Socket mit TCP.
network_socket_udp Erstellen Sie einen Socket mit UDP.
netzwerk_sockel_ws Erstellen Sie einen Websocket(nur für die Verbindung zu HTML5-Projekten), unter Verwendung von TCP.



Da dies einen "rohen" Server erzeugt, akzeptiert und verwendet er keine der eingebauten GameMaker-Datenheader für die Kommunikation, und daher sollten Sie die Funktionen network_send_raw() und verwenden. network_send_udp_raw() verwenden, um unformatierte Daten an den erzeugten Server zu senden.

Syntax:

network_create_server_raw(type, port, max_client);

Argument Beschreibung
Typ Der Typ des zu erstellenden Servers (siehe die unten aufgeführten Konstanten).
Hafen Der Port, den der Server verwenden wird.
max_Client Die maximale Anzahl von Clients, die sich gleichzeitig verbinden können.

Rückgabe:

Real

Beispiel:

var port = 6510;
server = network_create_server_raw(network_socket_tcp, port, 32);
while (server < 0 && port < 65535)
{
port++
server = network_create_server(network_socket_tcp, port, 32);
}

Der obige Code versucht, einen Server mit TCP über Port 6510 zu erstellen. Wenn dieser Port nicht verfügbar ist, wird eine Schleife durch die Ports gezogen, um einen zu finden, der verfügbar ist.
