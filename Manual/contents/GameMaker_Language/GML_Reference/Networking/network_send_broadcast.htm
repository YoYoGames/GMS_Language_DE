netzwerk_senden_broadcast

Mit dieser Funktion können Sie die Daten aus einem Puffer lokal an einen Bereich von IP-Adressen senden (für weitere Informationen zu Puffern siehe Referenz - Puffer). Der Bereich ist auf den des Geräts beschränkt, auf dem der Server läuft, Wenn also das Gerät eine IP-Adresse von 92.168.11.130 hat, werden die Daten über den Bereich 92.168.11.* gesendet. Die Funktion gibt die Anzahl der gesendeten Datenbytes zurück oder eine Zahl kleiner als 0, wenn das Senden fehlgeschlagen ist.

HINWEIS: Diese Funktion funktioniert nur, wenn sie mit UDP verwendet wird - Ihr Server muss TCP sein und Ihr Client muss einen UDP-Client-Socket erstellt haben mit network_create_socket_ext() erstellt werden, um alle vom Server gesendeten Broadcasts zu empfangen.

HINWEIS: Diese Funktion funktioniert nicht, wenn sie in einem Projekt verwendet wird, das auf dem HTML5-Target läuft.

Syntax:

network_send_broadcast(socket, port, buffer, size);

Argument Beschreibung
Steckdose Die ID des zu verwendenden Sockets.
Hafen Der Port, den der Server verwenden wird.
Puffer Die ID des Puffers, aus dem die Daten geholt werden sollen.
Größe Die Größe (in Bytes) der Daten.

Rückgabe:

Real

Beispiel:

buffer_seek(broadcast_buffer, buffer_seek_start, 0);
buffer_write(broadcast_buffer, buffer_string, global.ServerName);
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer));

Der obige Code schreibt die Namenszeichenkette des aktuellen Servers (gespeichert in "global.ServerName"), dann schreibt er sie in einen Binärpuffer mit der ID "broadcast_buffer". Diese Daten werden dann lokal an einen Bereich von IPs (die Geräte-IP ist derzeit als Broadcast-Basisbereich impliziert) an Port 6511 gesendet.
