sprite_get_speed

Mit dieser Funktion kann die Sprite-Geschwindigkeit abgerufen werden, wie sie für die Sprite-Ressource im Sprite-Editor definiert ist. Der zurückgegebene Wert kann dann z. B. verwendet werden, um zu berechnen, wie viele Frames gezeichnet werden dürfen für verschiedene image_speed. Beachten Sie, dass der Rückgabewert sehr unterschiedlich sein wird, je nachdem, welche Art von Geschwindigkeit die im Sprite-Editor angewendet wurde, entweder Frames pro Sekunde oder Frames pro Spielframe. Die folgenden zwei Beispiele verdeutlichen dies:

  • Wenn Sie ein Sprite haben, das 1 Bild pro Sekunde zeichnet, und die Bildgeschwindigkeit auf 0,5 einstellen, wird es mit 0,5 Bildern pro Sekunde gezeichnet. Wenn die Bildrate Ihres Spiels 60 Bilder pro Sekunde beträgt, zeichnet das Sprite 1 Bild alle 120 Spiel-Frames.
  • Wenn Sie ein Sprite haben, das 1 Bild pro Spielbild zeichnet, und die Bildgeschwindigkeit auf 0,5 einstellen, zeichnet es 0,5 Bilder pro Spielbild. Wenn Ihre Spielbildrate 60 Bilder pro Sekunde beträgt, zeichnet das Sprite 30 Bilder pro alle 60 Spiel-Frames.

Sie können herausfinden, welcher Typ von Animationsgeschwindigkeit bei der Definition des Sprites verwendet wurde, indem Sie die Funktion sprite_get_speed_type()und Sie können die Animationsgeschwindigkeit und den Typ mit der Funktion sprite_set_speed().

Syntax:

sprite_get_speed(index)

Argument Beschreibung
index Der Index des Sprites, um die Geschwindigkeit von

Rückgabe:

Real

Beispiel:

s_speed = sprite_get_speed(sprite_index);
s_type = sprite_get_speed_type(sprite_index);

Der obige Code ruft die Sprite-Geschwindigkeit und den Sprite-Animationstyp ab und speichert sie in Instanzvariablen für die zukünftige Verwendung.
