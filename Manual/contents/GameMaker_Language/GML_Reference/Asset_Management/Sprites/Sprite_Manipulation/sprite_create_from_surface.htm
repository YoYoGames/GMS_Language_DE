sprite_erzeugen_aus_der_Oberfläche

Mit dieser Funktion können Sie ein Sprite aus einer zuvor initialisierten Oberfläche erstellen (der Wert der Oberflächenindex-ID wird zurückgegeben, wenn Sie die Oberfläche mit surface_create()). Die x- und y-Koordinaten, die Sie dann in die Funktion eingeben, sollten sich auf die (0,0)-Position der Oberfläche (die linke obere Ecke) beziehen und nicht auf das Spielfenster (oder die Ansicht, falls Sie eine aktiv haben). Die Argumente für Breite und Höhe sind in Pixel und definieren die Breite und Höhe des zu verwendenden Teils der Oberfläche.

Wenn Sie das Argument "removeback" auf true setzen, wird eine Farbe aus dem Sprite entfernt, indem das untere linke Pixel des Sprites auf die dortige Farbe überprüft wird und diese dann als die zu entfernende Farbe verwendet wird.

Wenn Sie die Option "removeback" wählen, möchten Sie vielleicht auch, dass GameMaker Studio 2 die Kanten des Sprites glättet, indem Sie das Argument "smooth" auf true setzen. Dadurch wird lediglich ein halbtransparenter Rand um die Kanten des des Sprites, nachdem der Hintergrund entfernt wurde.

Schließlich können Sie auch den x- und y-Ursprung für das Sprite angeben. Dies ist der Punkt, an dem das Sprite auf der Instanz "fixiert" ist, die es verwendet, und wird immer relativ zur 0,0 oberen linken Ecke eines Sprite-Subbildes berechnet. Also, Ein Sprite, das 32 x 32 Pixel groß ist und bei dem diese Werte auf (16,16) gesetzt sind, hat seinen Ursprung also in der Mitte.

Standardmäßig werden die Boundingboxen aller neuen Sprites automatisch berechnet (die genaue Bbox hängt von der Größe und der Transparenz des Sprites ab), Sie können dies jedoch anpassen, in diesem Fall sollten Sie auch die Funktion sprite_collision_mask().

HINWEIS: Wenn Sie ein Sprite in GameMaker Studio 2 mit dieser Methode erstellen, müssen Sie daran denken, es wieder zu entfernen (mit sprite_delete()), wenn es nicht mehr benötigt wird, sonst besteht die Gefahr eines Speicherlecks, das verlangsamen und schließlich Ihr Spiel zum Absturz bringen.

Syntax:

sprite_create_from_surface(index, x, y, w, h, removeback, smooth, xorig, yorig);

Argument Beschreibung
index Der Index der Oberfläche, von der aus erstellt werden soll.
x Die x-Position, ab der kopiert werden soll.
y Die y-Position, ab der kopiert werden soll.
w Die Breite des zu kopierenden Bereichs (ab der x-Position).
h Die Höhe des zu kopierenden Bereichs (ab der y-Position).
zurück entfernen Gibt an, ob alle Pixel mit der Hintergrundfarbe (linkes unteres Pixel) transparent gemacht werden sollen.
glatt Gibt an, ob die Kanten geglättet werden sollen.
xorig Gibt die x-Position des Ursprungs im Sprite an.
yorig Gibt die y-Position des Ursprungs im Sprite an.

Rückgabe

Real (eindeutiger Sprite-ID-Wert)

Beispiel:

var surf;
surf = surface_create(32, 32);
surface_set_target(surf);
draw_clear_alpha(c_black, 0);
draw_sprite(spr_Body, 0, 0, 0);
draw_sprite(spr_Clothes, 0, 0, 0);
draw_sprite(spr_Hair, 0, 0, 0);
spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
surface_reset_target();
surface_free(surf);

Der obige Code erzeugt eine Oberfläche und speichert ihren Index in der lokalen Variablen "surf". Er zielt dann auf diese Oberfläche, löscht sie und zeichnet mehrere Sprites übereinander. Schließlich wird ein neues Sprite aus dem zusammengesetzten Bild auf der Oberfläche gezeichneten zusammengesetzten Bild und weist dessen Index "spr_Custom" zu, bevor er den von der Oberfläche verwendeten Speicher freigibt.
