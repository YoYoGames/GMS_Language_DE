sprite_replace

Diese Funktion funktioniert fast genau wie die Funktion sprite_add()nur, dass sie nicht den Index des zu importierenden Sprites zurückgibt, sondern einen zuvor erstellten Sprite-Index überschreibt. Wenn Sie dieser Funktion sollten Sie einen Sprite-Index verwenden, der mit anderen Funktionen wie sprite_add() oder sprite_create_from_surface(), oder sogar sprite_duplicate()erstellt und in einer Variablen gespeichert wurde, und nicht ein Sprite-Asset aus dem Ressourcenbaum. Sie können mit dieser Funktion ein Sprite aus den Spiel-Assets ersetzen, aber das bedeutet, dass dass Sie die Referenz-ID für das zu ersetzende Sprite verlieren, unabhängig davon, ob Sie game_restart() aufrufen oder nicht, und wird daher nicht empfohlen. Unabhängig davon, welches Sprite ersetzt wird, erstellt diese Funktion eine neue Texturseite für das Sprite und Daher ist bei der Verwendung dieser Funktion Vorsicht geboten, da sie sich negativ auf die Leistung auswirken kann, indem sie die Anzahl der erforderlichen Textur-Swaps für das Rendern erhöht.

Die zu ladende Bilddatei sollte immer im *.png-Format sein und alle Bilder, die in animierte Sprites umgewandelt werden sollen, sollten ein "Streifen"-Format haben (siehe das Bild unten). Sie werden in die angegebene Anzahl von Unterbildern nach der Regel sprite width = strip width / sub images aufgeteilt.

Wie Sie im obigen Bild sehen können, wurde das Sprite auf einem dunkelvioletten Hintergrund platziert, und dieser kann entfernt werden, indem das Argument "removeback" auf true gesetzt wird. Dies funktioniert, indem das untere linke Pixel des Sprites auf die dortige Farbe überprüft und diese dann als die zu entfernende Farbe verwendet. Wenn zum Beispiel im obigen Bild die Farbe des unteren linken Pixels grün wäre, würden alle grünen Teile des Sprites entfernt und der Rest des lila Hintergrunds ignoriert.

Wenn Sie die Option "removeback" wählen, möchten Sie vielleicht auch, dass GameMaker Studio 2 die Kanten des Sprites glättet, indem Sie das Argument "smooth" auf true setzen. Dies bewirkt lediglich, dass einen halbtransparenten Rand um die Kanten des Sprites, nachdem der Hintergrund entfernt wurde.

Schließlich können Sie auch den x- und y-Ursprung für das Sprite angeben. Dies ist der Punkt, an dem das Sprite auf der Instanz "fixiert" ist, die es verwendet, und wird immer relativ zur linken oberen Ecke 0,0 berechnet eines Sprite-Subbildes berechnet. So hat z. B. ein Sprite, das 32 x 32 Pixel groß ist und bei dem diese Werte auf (16,16) gesetzt sind, seinen Ursprung in der Mitte.

Standardmäßig werden die Boundingboxen aller neuen Sprites automatisch berechnet (die genaue Bbox hängt von der Größe und der Transparenz des Sprites ab), Sie können dies jedoch anpassen. sprite_collision_mask().

HINWEIS: Je nach gewählter Zielplattform sind Sie eingeschränkt, woher Sie Dateien speichern und laden können. Weitere Informationen finden Sie unter Dateihandhabung.

HINWEIS: Wenn Sie diese Funktion in Ihrem HTML5-Zielspiel verwenden, um Ressourcen von einem externen Server zu laden, sollten Sie sich bewusst sein, dass aufgrund des XSS-Schutzes in Browsern Versuche, Ressourcen von anderen Domänen zu laden, blockiert werden können und möglicherweise scheinbar leere Ergebnisse zurückgeben.

Syntax:

sprite_replace(ind, fname, imgnumb, removeback, smooth, xorig, yorig);

Argument Beschreibung
ind Der Index des Sprites, das permanent ersetzt werden soll.
fname Der Dateiname des Bildes, aus dem das neue Sprite erstellt werden soll.
imgnumb Die Anzahl der Frames, in die das Sprite horizontal zerlegt wird. 1 = ein einzelnes Bild / *.gif.
zurück entfernen Gibt an, ob alle Pixel mit der Hintergrundfarbe (linkes unteres Pixel) transparent gemacht werden sollen.
glatt Gibt an, ob die Kanten geglättet werden sollen.
xorig Die x-Koordinate des Ursprungs, bezogen auf die linke obere Ecke des Sprites.
yorig Die y-Koordinate des Ursprungs, bezogen auf die linke obere Ecke des Sprites.

Rückgabe

N/A

Beispiel:

sprite_replace(spr_banner, "gravemaker.png", 1, false, false, 0, 0);

Der obige Code ersetzt das in "spr_banner" indizierte Bild-Asset durch ein anderes, das aus einer externen Quelle geladen wurde.
