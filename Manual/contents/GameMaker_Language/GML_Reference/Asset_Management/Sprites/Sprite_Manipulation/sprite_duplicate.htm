sprite_duplicate

Diese Funktion gibt den Index eines neu erstellten Sprites zurück, das ein Duplikat (Kopie) desjenigen ist, das als Argument "index" eingegeben wurde. Wenn die Funktion fehlschlägt, wird -1 zurückgegeben. Diese Funktion muss verwendet werden, um alle Sprites aus den ursprünglichen Assets zu kopieren, bevor irgendwelche Transformationen an ihnen vorgenommen werden können. Wenn Sie z. B. den Begrenzungsrahmen für ein Sprite ändern oder sein Alpha von einem anderen Sprite aus einstellen möchten, müssen Sie es zuerst duplizieren, dann die Operation an dem duplizierten Sprite durchführen und dieses verwenden. Ein dupliziertes Sprite wird bei der Erstellung auf seiner eigenen, eindeutigen Texturseite platziert, was bedeutet, dass das Duplizieren mehrerer Sprites mehrere Texturseiten erzeugt und sich auf die Leistung auswirkt, weshalb Sie diese Funktion nur wenn nötig.

HINWEIS: Wenn Sie ein Sprite in GameMaker Studio 2 duplizieren, müssen Sie daran denken, es wieder zu entfernen (mit sprite_delete()) zu löschen, wenn es nicht mehr benötigt wird, sonst besteht die Gefahr eines Speicherlecks, das Ihr Spiel verlangsamt und schließlich zum Absturz bringt.

Syntax:

sprite_duplicate(index);

Argument Beschreibung
index Der Index des zu duplizierenden Sprites.

Rückgabe

Real (eindeutiger Sprite-ID-Wert)

Beispiel:

new_spr = sprite_duplicate(sprite_index)

Der obige Code dupliziert das aktuell verwendete Sprite als Sprite-Index der Instanz und speichert den Index für dieses neue Sprite in der Variablen "new_spr".
