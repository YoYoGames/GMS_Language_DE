sprite_add

Mit dieser Funktion können Sie ein Bild als Sprite hinzufügen, indem Sie es aus einer externen Quelle laden, wobei die zu ladende Bilddatei immer entweder im Format *.png, *.gif, *.jpg/jpeg oder *.json vorliegen sollte(*json-Dateien werden zum Laden von mit Spine erstellten Skelett-Animationssprites verwendet). Die Funktion gibt den neuen Sprite-Index zurück, der dann in jedem weiteren Code, der sich auf das Sprite bezieht, verwendet werden muss. Wenn Sie diese Funktion mit HTML5 verwenden oder ein Bild von einer URL abrufen, erzeugt diese Funktion auch ein asynchrones Ereignis "Image Loaded". Siehe diese Seite für mehr Informationen.

Alle Bilder, die in animierte Sprites umgewandelt werden sollen - außer *.gif (siehe unten) - sollten ein "Streifen"-Format haben (siehe das Bild unten) und es wird in die angegebene Anzahl von Unterbildern aufgeteilt, die der Regel sprite width = strip width / sub images folgen.

Wie Sie im obigen Bild sehen können, wurde das Sprite auf einem dunkelvioletten Hintergrund platziert, und dieser kann entfernt werden, indem das Argument "removeback" auf true gesetzt wird. Dies funktioniert, indem das untere linke Pixel des Sprites auf auf die dortige Farbe überprüft und diese dann als die zu entfernende Farbe verwendet. Wenn zum Beispiel im obigen Bild die Farbe des unteren linken Pixels grün wäre, würden alle grünen Teile des Sprites entfernt und der Rest des lila Hintergrunds ignoriert werden.

Wenn Sie die Option "removeback" wählen, möchten Sie vielleicht auch, dass GameMaker Studio 2 die Kanten des Sprites glättet, indem Sie das Argument "smooth" auf true setzen. Dadurch wird lediglich ein halbtransparenter Rand um die Kanten des des Sprites, nachdem der Hintergrund entfernt wurde.

Schließlich können Sie auch den x- und y-Ursprung für das Sprite angeben. Dies ist der Punkt, an dem das Sprite auf der Instanz "fixiert" ist, die es verwendet, und wird immer relativ zur 0,0 oberen linken Ecke eines Sprite-Subbildes berechnet. Also, Ein Sprite, das 32 x 32 Pixel groß ist und bei dem diese Werte auf (16,16) gesetzt sind, hat seinen Ursprung also in der Mitte.

Wenn Sie ein Bild im *.gif-Format hinzufügen, müssen Sie nur den Dateinamen und die Bildnummer angeben (die für das erste Bild 1 sein sollte) und können alle anderen Argumente auf 0 belassen, wobei Sie beachten, dass nur das erste Bild des GIFs geladen wird. Bei *.json-Sprites sollte die Bildnummer 1 sein, und die Einstellungen "removeback" und "smooth" werden ignoriert.

Standardmäßig werden die Bounding-Boxen aller neuen Sprites automatisch berechnet (die genaue Bounding-Box hängt von der Größe und der Transparenz des Sprites ab), Sie können dies jedoch anpassen. sprite_collision_mask().

Es ist zu beachten, dass der verwendete Speicher beim Hinzufügen eines Sprites auf diese Weise größer ist, als Sie vielleicht erwarten. Das liegt daran, dass GameMaker Studio 2 das Sprite als Texturseite speichert und es auch im GPU-Speicher abgelegt wird, so dass der Gesamtspeicher Der Gesamtspeicher ist also größer als für eine Bilddatei desselben Sprites zu erwarten wäre.

HINWEIS: Wenn Sie ein Sprite in GameMaker Studio 2 laden, müssen Sie daran denken, es wieder zu entfernen (mit sprite_delete()), wenn es nicht mehr benötigt wird, sonst besteht die Gefahr eines Speicherlecks, das Ihr Spiel verlangsamt und schließlich Ihr Spiel zum Absturz bringt.

HINWEIS: Bei iOSwenn Sie eine eingebundene Datei aus einem Ordner laden, d. h. mit einem Pfad wie "Hintergründe/background1.png", müssen Sie den Ordner NICHT als Teil des Pfades angeben und es reicht aus, den Dateinamen anzugeben. Auf allen anderen Plattformen müssen Sie den/die Ordner als Teil des Pfades zur Datei angeben.

HINWEIS: Bei HTML5 müssen Sie, wenn Sie ein Sprite von einem sicheren Server laden, möglicherweise den Cross-Origin-Typ setzen und einen relativen Pfad anstelle eines absoluten Pfads verwenden. Siehe http_set_request_crossorigin() für weitere Details.

Syntax:

sprite_add(fname, imgnumb, removeback, smooth, xorig, yorig);

Argument Beschreibung
fname Der Name (ein String-Dateipfad) der hinzuzufügenden Datei.
imgnum Geben Sie damit die Anzahl der Teilbilder an (1 für ein einzelnes Bild oder für ein *.gif).
zurücknehmen Gibt an, ob alle Pixel mit der Hintergrundfarbe (linkes unteres Pixel) transparent gemacht werden sollen.
glatt Gibt an, ob die Kanten geglättet werden sollen, wenn sie transparent sind.
xorig Geben Sie die x-Position des Ursprungs im Sprite an.
yorig Geben Sie die y-Position des Ursprungs im Sprite an.

Rückgabe:

Real (Sprite-ID-Wert)

Beispiel:

spr = sprite_add("spieler_5.png", 16, true, true, 0, 0);

Der obige Code lädt ein Sprite in das Spiel und speichert dessen Index in der Variablen "spr".
