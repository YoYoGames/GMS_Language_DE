sprite_collision_mask

Mit dieser Funktion können Sie die Eigenschaften der Kollisionsmaske einstellen, die ein Sprite haben soll. Wenn Sie als Bounding-Box-Modus entweder Automatik (0) oder Vollbild (1) wählen, können die einzelnen Bounding-Box-Werte auf 0 gesetzt werden. Bei einer benutzerdefinierten definierten Maske (2) müssen Sie diese Werte jedoch selbst festlegen. Die verschiedenen Bounding-Box-Werte beziehen sich immer auf die linke obere Ecke des Sprites (unabhängig vom x- und y-Ursprung), die als Position (0, 0) angesehen wird.

Beachten Sie jedoch, dass alles andere als eine rechteckige Maske beim Auflösen von Kollisionen mehr Rechenleistung erfordert, was zu einem Leistungsabfall führt. Im Allgemeinen sollten Sie nur andere Maskentypen als rechteckig nur verwenden, wenn es unbedingt notwendig ist.

HINWEIS: Diese Funktion lässt die Art der gedrehten Rechteck-Kollisionsmaske nicht zu.

Die Art der Maske, die eingestellt werden kann, ist eine der folgenden Konstanten:

Konstant Beschreibung
bboxkind_rectangular Ein rechteckiges (nicht rotierendes) Rechteck als Kollisionsmaske
bboxkind_ellipse Eine elliptische Form der Kollisionsmaske
bboxkind_diamond Form einer Diamant-Kollisionsmaske
bboxkind_präzise Eine präzise Kollisionsmaske, wobei sich die Maske an die nicht-transparenten Pixel des Sprites anpasst, basierend auf dem angegebenen Toleranzwert (siehe unten))


Schließlich wird mit der Toleranz festgelegt, wie genau die Maske ist (bei Verwendung einer "Vollbild"-Maske hat dies keine Auswirkungen), wobei eine Toleranz von 0 bedeutet, dass die Maske jedem einzelnen Pixel folgt, das eine Transparenz über 0 hat, während andere Werte den Umfang der Kollisionsmaske in Abhängigkeit von der Transparenz der Pixel verschieben.

HINWEIS: Diese Funktion ist nur für Bitmap-Sprites nützlich und funktioniert nicht mit SWF- oder JSON (Spine)-Sprites.

HINWEIS: Diese Funktion funktioniert nur bei hinzugefügten oder duplizierten Sprites und nicht direkt bei vorgefertigten Ressourcen. Sie können Sprites duplizieren, indem Sie die Funktion sprite_duplicate().

Syntax:

sprite_collision_mask(ind, sepmasks, bboxmode, bbleft, bbtop, bbright, bbbottom, kind, tolerance);

Argument Beschreibung
ind Der Index des Sprites, dessen Bounding Box gesetzt werden soll.
sepmasks Ob Kollisionsmasken für jedes Teilbild des Sprites(true) oder eine Maske für alle(false) erstellt werden sollen.
bboxmode Welche Art von Begrenzungsrahmen verwendet werden soll. 0 = automatisch, 1 = Vollbild, 2 = benutzerdefiniert.
bbleft Die maximale linke Position des Begrenzungsrahmens.
bbtop Die maximale obere Position des Begrenzungsrahmens.
bbright Die maximale rechte Position des Begrenzungsrahmens.
bbbottom Die maximale untere Position des Begrenzungsrahmens.
freundlich Die Art der Maske, eine Konstante (siehe die Tabelle in der Beschreibung).
Toleranz Gibt die Toleranz im Transparenzwert an (0=keine Toleranz, 255=volle Toleranz).

Rückgabe

N/A

Beispiel:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);
sprite_collision_mask(spr, true, 1, 0, 0, 0, 0, 0, 0, 0);

Der obige Code lädt ein Sprite aus einer externen Quelle und speichert den neuen Index in der Variablen "spr". Der Code stellt dann das neue Sprite so ein, dass es für jedes seiner Teilbilder eine präzise Kollisionsmaske hat.
