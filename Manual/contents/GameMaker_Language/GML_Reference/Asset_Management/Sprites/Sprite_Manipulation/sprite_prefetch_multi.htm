sprite_prefetch_multi

Diese Funktion kann verwendet werden, um eine Anzahl von Texturseiten, die die angegebenen Sprites enthalten, vorzuholen (im Texturspeicher zu platzieren). Sie liefern ein Array mit den Sprite-Indizes (wie beim Erstellen des Sprite-Assets definiert) und die Textur-Seiten, auf denen sie sich befinden, werden in den Speicher geladen. auf denen sie sich befinden, werden in den Speicher geladen. Beachten Sie, dass die Funktion -1 zurückgibt, wenn Prefetch für die gewählte Ressource nicht unterstützt wird oder die Zielplattform HTML5 ist, oder sie gibt 0 zurück, wenn alles korrekt funktioniert hat.

HINWEIS: Es gibt einen Leistungseinbruch, da die Textur auf den meisten Geräten in den Texturspeicher hochgeladen wird, daher wird empfohlen, dass Sie sprite_prefetch_multi() auf am Anfang eines Levels aufzurufen, um einen Stillstand zu vermeiden.

Syntax:

sprite_prefetch_multi

Argument Beschreibung
Array Array mit den zu holenden Sprite-Indizes

Rückgabe:

Int (-1 oder 0)

Beispiel:

spr_a[0] = spr_Player_Aura1;
spr_a[1] = spr_Player_Aura2;
spr_a[2] = spr_Player_Aura3;
spr_a[3] = spr_Player_Aura4;
sprite_prefetch_multi(spr_a);

Der obige Code erstellt ein Array, in dem jedes Element einen Sprite-Index enthält. Dieses Array wird dann verwendet, um diese Sprite-Texturen im Speicher zu platzieren.
