sprite_set_cache_size_ext

Bei der Durchführung von Bildüberblendungen kann HTML5 dies nicht dynamisch in der gleichen Weise wie eine ausführbare Datei durchführen. Daher muss GameMaker Studio 2 eine überblendete Kopie des Bildes temporär speichern und laden. Diese Funktion legt fest, wie viele überblendete Kopien des angegebenen Sprites zwischengespeichert werden können, bevor die alten überschrieben werden. Standard ist 4. Dies wird auf ein einzelnes gegebenes Teilbild des Sprites angewendet.

Syntax:

sprite_set_cache_size_ext(ind, index, max);

Argument Beschreibung
ind Der Index des Sprites, dessen Cache-Größe geändert werden soll.
index Das Unterbild von ind, für das die Cache-Größe geändert werden soll.
max Die maximale Anzahl der zwischengespeicherten Kopien des Sprites, die gespeichert werden können.

Rückgabe

N/A

Beispiel:

sprite_set_cache_size_ext(sprite0, 0, 2);

Dadurch wird der Sprite-Cache des ersten Teilbildes von sprite0 auf 2 Kopien gesetzt.
