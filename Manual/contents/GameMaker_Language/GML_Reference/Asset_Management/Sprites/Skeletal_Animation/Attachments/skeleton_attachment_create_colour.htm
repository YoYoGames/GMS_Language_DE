skeleton_attachment_create_colour

Wenn Sie Ihr Skelett-Animationssprite erstellen, können Sie Attachment-Slots und Attachments zuweisen, die in diese Slots passen. Dies sind einfach Bilder (Sprites), die von der Animation getrennt sind, sich aber, wenn sie angehängt sind, zusammen mit dem Bone bewegen, an dem sie angehängt sind. Normalerweise würden Sie diese Anhänge in Ihrem Animationsprogramm (Spine) zuweisen, aber mit dieser Funktion können Sie Ihre eigenen zur Laufzeit erstellen, indem Sie ein Sprite-Asset aus Ihrem Spiel verwenden. die Funktion erfordert, dass Sie dem Anhang einen Namen geben (als eine Zeichenkette) und setzen Sie dann die sprite_index und bild_index sowie den x- und y-Ursprung (der von dem des Sprites abweichen kann). Sie können auch alle Transformationen festlegen, die auf das Bild angewendet werden sollen, wenn es angehängt wird, sowie die Farbe, die mit dem Bild überblendet werden soll, und den Alpha-Wert (Transparenz) Wert. Wenn das Attachment erfolgreich erstellt wurde, gibt die Funktion 1 zurück, wenn nicht (Sie haben einen ungültigen Sprite-Index angegeben oder das Basis-Sprite ist kein Spine-Sprite), gibt sie -1 zurück.

Es ist erwähnenswert, dass das Instanz-Sprite eine Überblendfarbe und eine Alpha-Einstellung haben kann (image_blend und bild_alpha), haben kann, ebenso wie der Attachment-Slot (siehe die Funktion skeleton_slot_colour_set()), so dass die endgültige Farbe und Alpha, die das zugewiesene Attachment-Sprite für den Slot hat, eine Zusammensetzung aus all diesen Werten.

Syntax:

skeleton_attachment_create_colour(name, sprite, ind, xorigin, yorigin, xscale, yscale, rot, color, alpha);

Argument Beschreibung
Name Der Name (als String) des zu erstellenden Anhangs.
Sprite Der sprite_index, von dem das Anhangsbild abgerufen werden soll.
ind Der image_index, von dem das Anhangsbild abgerufen werden soll.
xOrigin Der x-Ursprung für das verwendete Bild.
yHerkunft Der y-Ursprung für das zu verwendende Bild.
x-Skala Die horizontale Skalierung des Bildes, als Multiplikator: 1 = normale Skalierung, 0,5 ist die Hälfte usw...
y-Skala Die vertikale Skalierung des Bildes, als Multiplikator: 1 = normale Skalierung, 0,5 ist die Hälfte usw...
rot Die Drehung des Bildes: 0 = normal, 90 = um 90° gegen den Uhrzeigersinn gedreht usw.
Farbe Der zu verwendende Farbwert (eine Konstante, eine Ganzzahl oder ein Hex-Wert).
Alpha Der zu verwendende Alphawert (von 0 bis 1).

Rückgabe:

1 wenn erfolgreich, -1 wenn nicht

Beispiel:

skeleton_attachment_create_colour("sword", spr_Weapons, 0, 0, 80, 1, 1, 90, c_red, 1);
skeleton_attachment_create_colour("knife", spr_Weapons, 1, 0, 45, 1, 1, 90, c_yellow, 1);
skeleton_attachment_create_colour("crossbow", spr_Weapons, 0, 10, 30, 1, 1, 0, c_white, 0.5);
skeleton_attachment_set("slot_leftHand", choose("sword", "knife", "crossbow"));

Der obige Code würde den aktuell zugewiesenen Anlagennamen für den Steckplatz mit dem Namen "slot_leftHand" prüfen und wenn eine leere Zeichenfolge zurückgegeben wird, wird ein neues Sprite angehängt.
