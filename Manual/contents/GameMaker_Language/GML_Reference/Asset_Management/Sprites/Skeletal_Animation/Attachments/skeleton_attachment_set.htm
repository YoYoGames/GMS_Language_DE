skelett_anhang_set

Einem Skelett-Animationssprite können andere Bilder als Anhänge hinzugefügt werden, wobei diese Bilder zu einem benannten Slot hinzugefügt werden (der Name wird beim Erstellen des Anhangs-Slots in Ihrem Animationsprogramm vergeben) und sie werden zusammen mit der Animation des des aktuellen Sprites gezeichnet werden. Mit dieser Funktion können Sie ein Attachment auf einen bestimmten Slot setzen, wobei Sie die Namen (als Strings) des Slots und des Attachments angeben müssen. Diese Namen werden durch das verwendete Animationsprogramm oder (im Falle des Attachments) beim Aufruf von skeleton_attachment_create().

Beachten Sie, dass Sie auch einen sprite_index als Anhang übergeben können, und dieses sprite wird verwendet, oder Sie können -1 verwenden, um den Anhang aus dem Slot zu entfernen. Wenn Sie einen Sprite-Index als Argument übergeben, wird ein einen Slot für den Anhang, wobei der Name des Sprites als Zeichenkette für den Namen des Slots verwendet wird (wenn Sie also z. B. spr_sword verwenden, wird ein Slot für den Anhang "spr_sword" erstellt), und der Slot verwendet den ersten Bildindex (0) des Sprites sowie dessen x- und y-Ursprungs-Offset, mit einer Skalierung von (1,1) und einer Drehung von 0.

Syntax:

skeleton_attachment_set(slot, attachment);

Argument Beschreibung
Steckplatz Der Name des Steckplatzes (eine Zeichenkette), von dem der Anhang abgerufen werden soll.
Anhang Der Name (als String oder als sprite_index) des Anhangsbildes.

Rückgabe:

Zeichenfolge

Beispiel:

if skeleton_attachment_get("slot_leftHand") == ""
{
skeleton_attachment_set("slot_leftHand", choose("sword", "spear", "knife"));
}

Der obige Code würde den aktuell zugewiesenen Anlagennamen für den Steckplatz mit dem Namen "slot_leftHand" prüfen und wenn eine leere Zeichenfolge zurückgegeben wird, wird ein neues Sprite angehängt.
