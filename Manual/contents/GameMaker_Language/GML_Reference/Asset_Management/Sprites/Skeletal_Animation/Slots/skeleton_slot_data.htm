skelett_schlitz_daten

Mit dieser Funktion können Sie eine (vorab erstellte) DS-Liste mit Daten für jeden der verfügbaren Attachment-Slots, die eine Animation enthält, auffüllen. Diese Daten werden im Forum einer DS-Map zurückgegeben, die die folgenden Schlüssel/Wertpaare enthält:

  • "Name": Der Name des Steckplatzes.
  • "Knochen": Der Name des Bones.
  • "Anhang": Der Name des Anhangs, wenn einer verwendet wird, oder "(none)", wenn kein Anhang zugeordnet ist.

Die Werte für jeden Schlüssel werden Strings sein und können dann in den anderen Skelett-Anhängefunktionen für diese Sprite-Typen verwendet werden. Beachten Sie, dass die erstellten DS-Maps nicht zerstört werden, so dass Sie die DS-Liste in einer Schleife durchlaufen und jede der die erstellten Maps selbst zerstören.

Syntax:

skeleton_slot_data(sprite, list);

Argument Beschreibung
Sprite Der Sprite-Index der Wirbelsäulen-Skelettanimation, von der die Daten abgerufen werden sollen.
Liste Die ID der DS-Liste, die mit den DS-Maps gefüllt werden soll.

Rückgabe:

N/A

Beispiel:

var list = ds_list_create();
var open = true;
slot_name = "";
skeleton_slot_data(sprite_index, list);
for (var i = 0; i < ds_list_size(list); i++;)
{
var map = list[| i];
if open
{
if map[? "attachment"] == "(none)"
{
open = false;
slot_name = map[? "name"];
}
}
ds_map_destroy(map);
}
ds_list_destroy(list);

Der obige Code erstellt eine DS-Liste und füllt sie dann mit den Slot-Daten für das Instanz-Sprite. Diese Daten werden dann geparst, um die einzelnen DS-Maps mit den Slot-Daten zu extrahieren. Anschließend wird geprüft, ob ein leerer Slot vorhanden ist, und wenn ja, wird der Variablen "slot_name" wird der leere Slot-Name zugewiesen, bevor die DS-Map zerstört wird. Schließlich wird die DS-Liste zerstört, da sie nicht mehr benötigt wird.
