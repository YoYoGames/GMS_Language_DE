image_index

Ein Sprite besteht aus einem oder mehreren Teilbildern, die das Sprite animiert erscheinen lassen können, wenn sie von einem zum anderen wechseln, oder zwischen denen im Code umgeschaltet werden kann, um verschiedene "Zustände" zu erzeugen, ähnlich wie eine Schaltfläche in Fenstern. Wenn das Sprite animiert ist, können Sie das aktuelle Bild der Animation abrufen, indem Sie die Variable image_index überprüfen, oder wenn Sie den Zustand eines statischen Sprites ändern möchten, können Sie ein neues Unterbild auswählen, indem Sie diese Variable auf das gewünschte Teilbild des Sprites setzen. Bitte beachten Sie, dass die Instanz entweder kein Draw-Ereignis haben sollte (so dass GameMaker Studio 2 das Sprite standardmäßig zeichnet) oder mit einer der Zeichenfunktionen gezeichnet werden sollte, damit Änderungen in dieser Variablen sichtbar werden draw_self() oder draw_sprite_ext() gezeichnet werden (indem Sie den image_index im entsprechenden Argument übergeben).

Beachten Sie, dass das direkte Setzen dieser Variablen auf einen Frame keine Broadcast-Meldungen auslöst, die möglicherweise auf diesem Frame vorhanden sind.

Beachten Sie auch, dass Sie bei der Verwendung von Skelett-Animations-Sprites weiterhin die image_index-Werte erhalten und setzen können - siehe die Funktion skeleton_animation_get_duration() für Beispiele, wie Sie dies tun können.

Syntax:

image_index;

Rückgabe:

Real (Gleitkommawert mit einfacher Genauigkeit)

Beispiel:

if (image_speed > 0)
{
if (image_index >= image_number - 1) instance_destroy();
}

Der obige Code prüft, ob das Sprite animiert ist, und wenn ja, dann prüft er, ob der aktuelle image_index am letzten Frame ist und zerstört in diesem Fall die Instanz.
