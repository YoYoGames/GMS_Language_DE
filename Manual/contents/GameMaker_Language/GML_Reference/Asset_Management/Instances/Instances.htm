Instanzen

Instanzen werden aus Objekten erstellt und können, sobald sie erstellt sind, durch GML-Code und DnD™-Aktionengeändert und manipuliert werden. Die wichtigsten Instanzfunktionen im Zusammenhang mit Instanzen, die verfügbar sind und am häufigsten verwendet werden, sind:

  • Instanz_erzeugen_Schicht
  • instance_create_depth
  • instanz_zerstören
  • instance_exists
  • Instanz_Änderung
  • Instanz_kopieren
  • instance_find
  • Instanz_am-weitesten
  • instanz_nächste
  • Instanz_Nummer
  • Instanz_Ort
  • instanz_platz_liste
  • Instanz_Position
  • instanz_position_liste

Beachten Sie, dass es zwei zusätzliche eingebaute Variablen gibt (dies sind Variablen mit globalem Geltungsbereich ) und eine Funktion, die sich auf die Verwendung von Instanzen bezieht, die für das Abrufen bestimmter Details nützlich sein können:

  • instance_id_get
  • instance_id
  • Instanz_Zahl

Es gibt eine Reihe von eingebauten Variablen, die in jeder Instanz eines Objekts zur Verfügung stehen und die verwendet werden können, um verschiedene Aspekte zu steuern, wie die Instanz angezeigt wird, wie sie andere Assets verwendet und wie sie auf Kollisionen reagiert usw., die alle aufgelistet sind hier aufgelistet sind:

  • Instanz-Variablen

Instanzen verfügen über eine Array-Variable, die zum Abrufen oder Setzen von Alarmereigniswerten verwendet werden kann, aber es gibt auch zwei Funktionen, mit denen das Gleiche erreicht werden kann:

  • alarm_get
  • alarm_setzen

Schließlich haben Sie auch die Möglichkeit, Instanzen zu deaktivieren, was bedeutet, dass sie keine ihrer Ereignisse mehr verarbeiten, obwohl sie im Spiel weiterhin existieren werden. Die diesbezüglichen Funktionen finden Sie auf der folgenden Seite:

  • Deaktivieren von Instanzen
