instanz_zerstören

Sie rufen diese Funktion immer dann auf, wenn Sie eine Instanz "zerstören" möchten, was normalerweise ein Destroy-Ereignis und auch ein Clean Up-Ereignis auslöst . Dadurch wird sie aus dem Raum entfernt, bis der Raum neu gestartet wird (sofern der Raum nicht persistent ist). Wenn Sie die Funktion ohne Argumente aufrufen, wird einfach die Instanz zerstört, die gerade im Gültigkeitsbereich ist und den Code ausführt, aber Sie können ein optionales "id"-Argument angeben und eine bestimmte Instanz anvisieren, indem Sie den Wert der Instanz id verwenden, oder Sie können alle Instanzen eines bestimmten Objekts anvisieren, indem Sie ein Objekt_Index. Zum Beispiel:

instance_destroy(other); // zerstört die Instanz "other" bei einem Kollisionsereignis
instance_destroy(obj_Bullet); // zerstört ALLE Instanzen des Objekts "obj_Bullet"

Mit dem zweiten optionalen Flag können Sie das Destroy-Event für die zu zerstörende Instanz "ausschalten". Standardmäßig wird das Zerstörungsereignis immer ausgeführt, aber wenn Sie dieses Flag auf false setzen, können Sie die Instanz selbst zerstören und die Ausführung dieses Ereignisses überspringen.

WICHTIG! Dadurch wird das Ereignis "Zerstören" übersprungen, aber das Ereignis "Aufräumen" wird trotzdem aufgerufen.

Es ist erwähnenswert, dass, wenn Sie eine Instanz zerstören, ihr Zerstörungsereignis unmittelbar nach dem Code oder der Aktion aufgerufen wird, die die Zerstörungsfunktion aufruft. Auch wenn das Zerstörungsereignis ausgeführt wird, wird die Instanz nicht sofort aus dem Spiel entfernt, Sie wird weiterhin den im aktuellen Ereignis enthaltenen Code ausführen. Erst wenn das aktuelle Ereignis vorbei ist, wird sie aus dem Spiel entfernt.

Wenn Sie also z. B. diesen Code haben:

f hp <= 0 instance_destroy();
score += 10;

Die Variable "score" wird inkrementiert, obwohl die Funktion instance_destroy aufgerufen wurde, und die Instanz wird schließlich am Ende des Ereignisses aus Ihrem Spiel entfernt. Seien Sie damit vorsichtig, denn wenn Sie (zum Beispiel) Wenn Sie (z. B.) eine dynamische Ressource für die Instanz erstellt haben, wie z. B. eine Datenstruktur, und diese Ressource dann im destroy-Ereignis zerstört haben, werden Sie, wenn es irgendwelche Referenzen darauf gibt, nachdem die destroy-Funktion oder -Aktion ausgeführt wurde, einen "unknown Ressource"-Fehler, da das Zerstörungsereignis sie aus dem Spiel entfernt hat.

Syntax:

instance_destroy([id, execute_event_flag]);

Argument Beschreibung
id Die zu zerstörende Instanz-ID oder der Objekt_Index (optional, Standard ist die aufrufende Instanz)
ereignis_ausfuehren_flag Auf true oder false setzen, um das Zerstörungsereignis auszuführen oder nicht (optional, Standard ist true)

Rückgabe:

N/A

Beispiel:

if bbox_right < 0 || bbox_left > room_width || bbox_bottom < 0 || bbox_top > room_height
{
instance_destroy(id, false);
}

Der obige Code zerstört die Instanz, die den Code ausführt, ohne das Destroy-Ereignis aufzurufen, wenn sich die Instanz außerhalb der Raumbegrenzung befindet.
