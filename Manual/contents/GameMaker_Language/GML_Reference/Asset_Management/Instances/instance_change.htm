Instanz_Änderung

Sie können diese Funktion verwenden, um eine Instanz eines Objekts in eine andere Instanz eines anderen Objekts zu verwandeln und dabei zu entscheiden, ob die Ereignisse "Zerstören" und "Aufräumen" der ursprünglichen Instanz und das Ereignis "Erzeugen" der neuen Instanz ausgeführt werden sollen. Auf diese Weise können Sie z. B. eine Bombe in eine Explosion verwandeln - in diesem Fall wäre das "perf"-Argument wahrscheinlich zutreffend, da Sie möchten, dass die Bombe ihr Destroy-Ereignis und ihr Clean-Up-Ereignis ausführt, und die Explosion ihr Create Ereignis ausführt - oder Sie könnten Ihren Spielercharakter in einen anderen verwandeln lassen - in diesem Fall wäre das Argument "perf" wahrscheinlich falsch, da Sie nicht möchten, dass die Instanzen ihre Ereignisse "Erzeugen" und "Zerstören/Aufräumen" ausführen.

Es ist erwähnenswert, dass das Ändern der Instanz bedeutet, dass Sie bis zum nächsten Schritt keine weiteren Aktionen mit dieser Instanz durchführen sollten, insbesondere den Versuch, auf Variablen usw. zuzugreifen, da dies einen Fehler verursacht. Im Grunde ändern Sie die Instanz, aber sie ist bis zum Ende des aktuellen Schritts nicht wirklich verfügbar, so dass ein direkter Zugriff auf eine der darin enthaltenen Variablen (z. B. durch den Aufruf von obj_Changed.x) nicht funktioniert.

WARNUNG! Wenn Sie eine physikalisch aktivierte Instanz ändern, werden die physikalischen Eigenschaften nicht auf die neue Instanz übertragen, in die sie geändert wird. Daher sollten Sie einen Code haben, der den physikalischen Zustand der aktuellen Instanz auf die neue Instanz "überträgt". Instanz auf die neue Instanz "übertragen" wird, oder Sie haben die physikalischen Eigenschaften der neuen Instanz in ihrem Erstellungsereignis oder im Objekteditor definiert. Aus diesem Grund wird empfohlen, dass Sie diese Funktion nicht mit physikfähigen Instanzen verwenden, sondern eine Kombination aus instance_destroy() und instance_create_layer()zu verwenden .

Syntax:

instance_change(obj, perf);

Argument Beschreibung
obj Das neue Objekt, in das sich das aufrufende Objekt verwandeln soll.
perf Ob die Ereignisse "Erzeugen" und "Zerstören" dieses neuen Objekts ausgeführt werden sollen (true) oder nicht (false).

Rückgabe:

N/A

Beispiel:

if keyboard_check(vk_enter)
{
instance_change(obj_Player_Swimming, false);
exit;
}

Der obige Code ändert die aufrufende Instanz in eine Instanz von "obj_Player_Swimming", ohne das Zerstörungsereignis für die ursprüngliche Instanz oder das Erstellungsereignis für die neue Instanz auszuführen, wenn der Benutzer die "Enter"-Taste drückt, und verlässt den Code-Block, wenn er fertig ist.
