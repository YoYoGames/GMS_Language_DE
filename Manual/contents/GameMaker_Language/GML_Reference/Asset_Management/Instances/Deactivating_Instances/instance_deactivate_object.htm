instance_deactivate_object

Mit dieser Funktion können Sie eine einzelne Instanz oder alle Instanzen eines bestimmten Objekts aus allen zuvor aktivierten deaktivieren. Beachten Sie, dass Sie, wenn Sie eine Instanz oder ein Objekt deaktiviert haben, das als persistent gekennzeichnet wurde, es müssen Sie es vor dem Raumwechsel wieder mit der Funktion instance_activate_object() reaktivieren, bevor Sie den Raum wechseln, da es sonst nicht übernommen und stattdessen verworfen wird. Beachten Sie auch, dass die Deaktivierung nicht sofort erfolgt, und eine Instanz, die auf diese Weise deaktiviert wurde, wird erst nach dem Ende des Ereignisses, in dem die Funktion aufgerufen wurde, als inaktiv betrachtet.

HINWEIS: Wenn Sie eine Instanz beim Raumstart deaktivieren (d. h.: über den Raumerstellungscode oder über ein Instanzerstellungsereignis einer Instanz innerhalb des Raums), führen alle Instanzen, die über den Raumeditor im Raum platziert werden, weiterhin ihr Erstellungsereignis aus, bevor sie deaktiviert werden.

ACHTUNG: Das Deaktivieren von Instanzen, bei denen die Physik aktiviert ist, verhindert NICHT, dass deren Fixtures innerhalb der Physiksimulation interagieren. Dafür sollten Sie ihre phy_active Variable auf true oder false setzen, wenn Sie die Instanzen aktivieren/deaktivieren.

Syntax:

instance_deactivate_object(obj);

Argument Beschreibung
obj Das Objekt oder die Instanz, die deaktiviert werden soll (das Schlüsselwort all kann ebenfalls verwendet werden).

Rückgabe:

N/A

Beispiel:

instance_deactivate_object(obj_Enemy);
var _vx = camera_get_view_x(view_camera[0]);
var _vy = camera_get_view_y(view_camera[0]);
var _vw = camera_get_view_width(view_camera[0]);
var _vh = camera_get_view_height(view_camera[0]);
instance_activate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false);

Der obige Code deaktiviert alle Instanzen des Objekts "obj_Enemy" und aktiviert dann eine Region innerhalb des Raums.
