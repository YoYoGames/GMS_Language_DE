instanz_aktivieren_region

Mit dieser Funktion können Sie einen Bereich innerhalb des Raums definieren, um Instanzen zu aktivieren, die zuvor deaktiviert wurden. Dieser Bereich kann entweder als "innen" oder "außen" gekennzeichnet werden, wie im folgenden Bild gezeigt:

Sie können im obigen Bild sehen, dass die "Apfel"-Instanz immer aktiv ist weil, auch wenn das Sprite selbst die Region nicht überlappt, die Bounding Box dies tut. Es wird also davon ausgegangen, dass sich Instanzen innerhalb der angegebenen Region befinden, wenn sich ihr Begrenzungsrahmen mit ihr überschneidet, und der Zustand der Kollisionsmaske (präzise oder nicht) wird nicht berücksichtigt. Beachten Sie, dass die Aktivierung nicht sofort erfolgt und eine Instanz, die auf diese Weise aktiviert wurde, bis zum Ende des Ereignisses, in dem die Funktion aufgerufen wurde, nicht als aktiv betrachtet wird.

Syntax:

instance_activate_region(left, top, width, height, inside);

Argument Beschreibung
links Die x-Koordinate der linken Seite des zu aktivierenden rechteckigen Bereichs.
oben Die y-Koordinate der Oberkante des zu aktivierenden rechteckigen Bereichs.
Breite Die Breite des zu aktivierenden Bereichs.
Höhe Die Höhe des zu aktivierenden Bereichs.
innerhalb Ob Instanzen auf der Innenseite der Region (true) oder auf der Außenseite (false) aktiviert werden sollen.

Rückgabe:

N/A

Beispiel:

instance_deactivate_all(true);
var _vx = camera_get_view_x(view_camera[0]);
var _vy = camera_get_view_y(view_camera[0]);
var _vw = camera_get_view_width(view_camera[0]);
var _vh = camera_get_view_height(view_camera[0]);
instance_activate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false);

Der obige Code deaktiviert alle Instanzen außer derjenigen, die den Code ausführt, und aktiviert dann eine Region innerhalb des Raums.
