instance_deactivate_all

Mit dieser Funktion werden alle Instanzen deaktiviert, die sich in dem Moment, in dem der Code ausgeführt wird, im Raum befinden. Dazu kann die Instanz gehören, die den Code ausführt, wenn das Flag "notme" auf false gesetzt ist, aber normalerweise möchten Sie, dass diese Instanz aktiv sein soll, in diesem Fall sollte das "notme"-Flag auf true gesetzt werden. Beachten Sie, dass die Deaktivierung nicht sofort erfolgt und dass eine Instanz, die auf diese Weise deaktiviert wurde, erst nach dem Ende des Ereignisses in dem die Funktion aufgerufen wurde.

HINWEIS: Wenn Sie eine Instanz beim Raumstart deaktivieren (d. h.: über den Raumerstellungscode oder über ein Instanzerstellungsereignis einer Instanz innerhalb des Raums), führen alle Instanzen, die über den Raumeditor im Raum platziert werden, weiterhin ihr Erstellungsereignis aus, bevor sie deaktiviert werden.

WARNUNG: Das Deaktivieren von Instanzen, bei denen die Physik aktiviert ist, stoppt NICHT die Interaktion ihrer Fixtures innerhalb der Physiksimulation. Dafür sollten Sie ihre phy_active Variable auf true oder false setzen, wenn Sie die Instanzen aktivieren/deaktivieren.

Syntax:

instance_deactivate_all(notme);

Argument Beschreibung
notme Ob die aufrufende Instanz aktiviert bleiben soll (true) oder nicht (false).

Rückgabe:

N/A

Beispiel:

instance_deactivate_all(true);
var _vx = camera_get_view_x(view_camera[0]);
var _vy = camera_get_view_y(view_camera[0]);
var _vw = camera_get_view_width(view_camera[0]);
var _vh = camera_get_view_height(view_camera[0]);
instance_activate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false);

Der obige Code deaktiviert alle Instanzen außer derjenigen, die den Code ausführt, und aktiviert dann eine Region innerhalb des Raums.
