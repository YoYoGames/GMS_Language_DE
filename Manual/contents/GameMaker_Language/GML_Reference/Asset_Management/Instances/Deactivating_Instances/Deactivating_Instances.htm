Deaktivieren von Instanzen

GameMaker Studio 2 bietet Ihnen die Möglichkeit, Instanzen "abzuschalten", so dass sie in keiner Weise mehr bearbeitet werden. Technisch gesehen existieren sie nicht mehr wirklich, außer als Zeiger innerhalb des Deaktivierungsprozesses selbst, was bedeutet, dass eine deaktivierte Instanz in keiner Weise manipuliert oder verändert werden kann, bis sie wieder aktiviert wird. Diese Funktionen sollten also mit großer Vorsicht verwendet werden, da sie bei unsachgemäßer Verwendung Probleme verursachen können, insbesondere bei persistenten Objekten, da ein persistentes Objekt, das deaktiviert wurde, nicht in den nächsten Raum verschoben werden kann, wenn es nicht zuerst wieder aktiviert wird (es wird also effektiv aus dem Spiel gelöscht). Beachten Sie auch, dass Aktivierung und Deaktivierung nicht sofort erfolgen, und eine Instanz, deren Zustand auf diese Weise geändert wurde, wird bis zum Ende des Ereignisses, in dem die Funktion aufgerufen wurde, nicht als aktiv/inaktiv betrachtet.

Es ist normalerweise nicht notwendig, Instanzen bei jedem Schritt Ihres Spiels zu deaktivieren, und dies kann sogar dazu führen, dass Ihr Spiel Leistungsprobleme hat. Stattdessen wird empfohlen, diese Funktionen nur alle paar Schritte auszuführen (z. B. bei einem Alarm), oder wenn die Kameraansicht ihre Position geändert hat, und es ist besonders wichtig, dass Sie diese Funktionen nicht im Draw-Ereignis verwenden, da dies zu schweren Fehlern in Ihrem Spiel führen kann.

Eine deaktivierte Instanz hört effektiv auf, im Spiel zu existieren, aber auf die Variablen der einzelnen Instanzen kann weiterhin zugegriffen werden. Sie können jedoch nicht with(object) oder with(instance) verwenden, sondern Sie müssen direkt auf die Instanz zugreifen, indem Sie ihre eindeutige ID in der folgenden Form verwenden:

val = inst.variable;

Im obigen Beispiel wäre "inst" eine Variable, die die ID der deaktivierten Instanz enthält. Dies ist die einzige Möglichkeit, auf eine deaktivierte Instanz zu verweisen, ohne dass sie zuvor aktiviert wurde.

HINWEIS: Wenn Sie eine Instanz beim Raumstart deaktivieren (d. h. über den Raumerstellungscode oder über ein Instanzerstellungsereignis einer Instanz innerhalb des Raums), führen alle Instanzen, die vom Raumeditor aus im Raum platziert werden, weiterhin ihr Erstellungsereignis, bevor sie deaktiviert werden.

ACHTUNG: Das Deaktivieren von Instanzen, bei denen die Physik aktiviert ist, stoppt NICHT die Interaktion ihrer Fixtures innerhalb der Physiksimulation. Setzen Sie deshalb die Variable phy_active auf true oder false setzen, wenn Sie die Instanzen aktivieren/deaktivieren.

Zum Deaktivieren von Instanzen gibt es die folgenden Funktionen:

  • instance_activate_all
  • instance_activate_object
  • instanz_aktivieren_region
  • Instanz_aktivieren_Schicht
  • instance_deactivate_all
  • instance_deactivate_object
  • instance_deactivate_region
  • instance_deactivate_layer
