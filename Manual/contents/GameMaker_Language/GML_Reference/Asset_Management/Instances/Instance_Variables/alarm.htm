Alarm

Dieses eindimensionale Array wird verwendet, um den aktuellen Wert für alle Alarme zu erhalten, die die Instanz möglicherweise hat, oder es kann verwendet werden, um diese Alarme zu setzen. Es gibt zwölf Alarme, die in jede Instanz eines Objekts eingebaut sind, und jeder hat sein eigenes Ereignis, das ausgelöst wird, wenn diese Variable 0 erreicht.

Alarme werden zu Beginn eines jeden Schritts heruntergezählt, nach dem Ereignis "Begin Step", aber vor dem Hauptereignis "Step" und bevor irgendetwas für den Schritt gerendert wird. Es ist zu beachten, dass der Alarm nicht beendet ist, wenn er 0 erreicht (obwohl das Ereignis ausgelöst wurde), da er im nächsten Schritt auf -1 zurückgeht. Wenn Sie also einen Alarm aus irgendeinem Grund anhalten müssen, sollten Sie seinen Array-Wert auf -1 und nicht auf 0 setzen. Die Alarmzeiten werden in Spielschritten berechnet, wobei ein Wert von 30 für 30 Schritte steht, 60 steht für 60 Schritte usw..

HINWEIS: Ein Alarm, der keine Aktionen oder keinen Code enthält, wird nicht ausgeführt. Aber auch mit nur einem Kommentar und ohne Code wird der Alarm heruntergezählt.

Sie können das Alarm-Array direkt setzen (und den aktuellen Alarmwert auch direkt über das Array abrufen), aber unter bestimmten Umständen ist dies nicht die geeignetste Methode, daher können Sie auch die Funktion alarm_set() verwenden, um einen Alarm zu setzen, und die Funktion alarm_get() um den Wert eines Alarms abzurufen.

Syntax:

alarm[val];

Rückgabe:

Integer (-1, wenn der Alarm nicht läuft)

Beispiel:

if canshoot
{
if keyboard_check_pressed(vk_space)
{
canshoot = false;
alarm[0] = room_speed;
instance_create_layer(x, y, "Bullets", obj_Bullet);
}
}

Der obige Code prüft eine Variable und dann, wenn sie wahr ist, prüft er auf einen Tastendruck und wenn dieser wahr ist, setzt er schließlich die Variable auf false, setzt einen Alarm (bei dem sie wieder auf true gesetzt wird) und erzeugt eine Instanz.
