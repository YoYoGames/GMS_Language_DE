Diese Variable kann gelesen werden, um herauszufinden, ob die Instanz als persistent gekennzeichnet ist oder nicht, oder sie kann verwendet werden, um die Persistenz für die Instanz auf true (persistent) oder false (nicht persistent) zu setzen. Eine persistente Instanz ist eine, die von Raum zu Raum "mitgenommen" wird, d. h. sie muss z. B. nur einmal zu Beginn des Spiels angelegt werden und ist dann in allen weiteren Räumen vorhanden. Mit der Persistenz sollte man vorsichtig umgehen, da man leicht den Überblick über persistente Instanzen zu verlieren, was später bei der Entwicklung des Spiels zu Problemen führen kann.

Wenn Sie eine Instanz mit aktivierter Persistenz in einem Raum erstellen, wird diese Variable auf true gesetzt und ihr wird eine Ebene oder eine Tiefe zugewiesen (je nachdem, welche Funktion Sie zum Erstellen der Instanz des Objekts verwendet haben). Wenn der Raum gewechselt wird, und wenn der folgende Raum keine Ebene mit demselben Namen oder derselben Tiefe wie die zugewiesene hat, wird eine neue Ebene für die Instanz erstellt, die über die Räume hinweg bestehen bleibt. Wenn Sie beim Anlegen der Instanz einen benannten Layer vergeben haben, Wenn Sie der Instanz eine Tiefe zugewiesen haben, erhält die neue Ebene den Namen "_layer_XXX", wobei "XXX" ein Hexadezimalwert ist, um der Ebene einen eindeutigen Namen zu geben. Namen zu geben. Diese Ebenen werden bei "Room End" aus dem Raum entfernt. Ein weiterer Hinweis zu Ebenen und persistenten Instanzen ist, dass, wenn Sie der Instanz bei der Erstellung eine benannte Ebene zugewiesen haben und es in den folgenden Räumen eine andere Ebene mit demselben Namen gibt, die persistente Instanz nicht mehr existiert. gleichen Namen gibt, dann wird die persistente Instanz der Ebene mit dem gleichen Namen zugewiesen , unabhängig von der Tiefe der Ebene. Wenn schließlich eine persistierte Instanz in einen Raum mit einer Ebene in der gleichen Tiefe wie bei der Erstellung der Instanz wechselt, wird sie wird sie nicht dieser Ebene zugewiesen, sondern es wird eine neue Ebene mit der gleichen Tiefe erstellt (gemäß der zuvor erläuterten Namenskonvention).

Bei einem persistenten Objekt werden weiterhin die Ereignisse " Spielstart", " Spielende", " Raumstart" und "Raumende" ausgelöst, aber wenn Sie das Spiel neu starten (z. B. mit der Funktion game_restart() ), werden alle persistenten Objekte entfernt und existieren nur noch, wenn sie vom Spiel neu erstellt werden. Beachten Sie auch, dass, wenn Sie ein persistentes Objekt deaktivieren, es nicht mehr von einem Raum in einen anderen übergeht, es sei denn, es wird vor dem Auslösen des Ereignisses "Raumende" wieder aktiviert.

Syntax:

hartnäckig;

Rückgabe:

Boolesche

Beispiel:

if !persistent
{
persistent = true;
}

Der obige Code prüft die Instanz, um zu sehen, ob Persistenz als falsch gekennzeichnet ist, und wenn die Instanz NICHT persistent ist, wird das Persistenz-Flag auf true gesetzt.
