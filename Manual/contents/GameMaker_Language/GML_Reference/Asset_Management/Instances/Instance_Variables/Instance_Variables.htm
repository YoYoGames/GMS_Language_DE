Instanz-Variablen

Wenn Sie ein neues Objekt erstellen, wird es mit bestimmten Variablen geliefert, die bereits mit Standardwerten initialisiert sind. Diese Variablen nennen wir Instanzvariablen, da sie für jede Instanz, die Sie von dem Objekt in Ihren Spielräumen platzieren, erstellt werden, und Sobald eine Instanz erstellt wurde, sind die Werte dieser Variablen eindeutig für diese Instanz und nur für diese Instanz. Einige dieser Variablen werden Sie häufig verwenden, wie x und y, während andere weniger benötigt werden, je nachdem, was Sie tun wollen. aber im Allgemeinen sind sie sehr nützlich und Sie sollten sie nach Möglichkeit verwenden, anstatt eigene benutzerdefinierte Variablen zu erstellen.

Die meisten Instanzvariablen können nicht nur gelesen, sondern auch gesetzt werden, so dass Sie die Eigenschaften und das Verhalten einer Instanz ändern können, indem Sie einfach den Wert einer bestimmten Variablen ändern - Sie können zum Beispiel verhindern, dass eine Instanz gezeichnet wird, indem Sie einfach die visible auf false gesetzt wird. Im Folgenden finden Sie die verschiedenen Variablen, die für alle Instanzen aller Objekte in Ihrem Spiel initialisiert werden.

WICHTIG! Wenn sich eine Instanz in einer Sequenz befindet, werden einige dieser Variablen - wie image_xscale / image_yscale / image_angle / x / y - überschrieben werden, wenn die Sequenz jeden Schritt aktualisiert, nachdem sie begonnen hat, abgespielt zu werden.

Diese Variablen befassen sich mit allgemeinen Instanzeigenschaften:

  • id
  • sichtbar
  • solide
  • beständig
  • Tiefe
  • Ebene
  • Alarm

Diese Variablen befassen sich mit der Position und Bewegung der Instanz:

  • Richtung
  • Reibung
  • Schwerkraft
  • Schwerkraft_Richtung
  • hspeed
  • vspeed
  • Geschwindigkeit
  • xstart
  • ystart
  • x
  • y
  • xprevious
  • yvorherige

Objekt-Eigenschaften

Die folgende Variable enthält die eindeutige Index-ID des Objekts, aus dem die Instanz erstellt wurde:

  • Objekt_Index

Diese Variablen beziehen sich alle auf das der Instanz zugewiesene Sprite und können verwendet werden, um zu ändern, was und wie gezeichnet wird:

  • sprite_index
  • sprite_width
  • sprite_height
  • sprite_xoffset
  • sprite_yoffset
  • image_alpha
  • Bild_Winkel
  • image_blend
  • image_index
  • Bild_Nummer
  • image_speed
  • Bild_x-Skala
  • image_yscale

Diese Variablen befassen sich mit der Kollisionsmaske:

  • maske_index
  • bbox_bottom
  • bbox_left
  • bbox_right
  • bbox_top

Diese Variablen beziehen sich auf Pfade und darauf, wie die Instanz mit einem solchen interagiert, wenn er zugewiesen ist:

  • Pfad_Index
  • weg_position
  • weg_positionvorher
  • Weg_Geschwindigkeit
  • Pfad_Skala
  • Pfad_Orientierung
  • Pfad_Aktion

Diese Variablen dienen zum Einstellen einer Instanz zur Verwendung einer Zeitleiste:

  • zeitleiste_index
  • timeline_running
  • timeline_speed
  • zeitleiste_position
  • timeline_loop

Sequenzen

Die folgenden Variablen speichern Informationen über die Sequenz, die die Instanz zu einem bestimmten Zeitpunkt steuern kann:

  • in_reihenfolge
  • Reihenfolge_Instanz

Es gibt eine große Anzahl eingebauter Variablen für die Verwendung mit den Physikfunktionen von GameMaker Studio 2. Der Übersichtlichkeit halber finden Sie diese in dem Teil des Handbuchs, der alles rund um die Physiksimulation behandelt:

  • Physik
