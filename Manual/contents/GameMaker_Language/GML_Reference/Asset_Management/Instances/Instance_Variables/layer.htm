Ebene

Diese eingebaute Variable wird für jede Instanz in einem Raum angelegt und enthält den Layer-ID-Wert des Layers, dem die Instanz zugeordnet ist. Dieser Wert kann dann in anderen Funktionen verwendet werden, wie layer_get_depth() oder verwendet werden. geändert werden, um die Instanz in eine andere Ebene zu verschieben. Beachten Sie jedoch, dass Sie eine Fehlermeldung erhalten, wenn die zugewiesene Ebene im aktuellen Raum nicht vorhanden ist, so dass Ihr Spiel geschlossen werden muss. Wenn Sie eine Ebene zuweisen, müssen Sie die eindeutige Ebenen-ID angeben, die von der Funktion layer_get_id() zurückgegeben wird (wenn Sie benannte Raumebenen verwenden), oder wie sie von der Funktion layer_create() (wenn Sie Ihre eigenen Ebenen zur Laufzeit erstellen).

WICHTIG! Wenn Sie die Instanz mit der instance_create_depth() Funktion erstellt oder die Variable Tiefe Variable manuell geändert haben, wird die der Instanz zugewiesene Ebene zu einer "verwalteten" Ebene, d. h. einer Ebene, die von GameMaker Studio 2 automatisch gesteuert und verwaltet wird. In diesen Fällen gibt die Ebenenvariable -1 zurück.

Syntax:

Schicht;

Rückgabe:

Real (Ganzzahl - Ressourcenindexwert)

Beispiel:

Ebene = Ebene_erstellen(-1000);

Der obige Code erstellt eine neue Ebene mit einer Tiefe von -1000 und setzt dann die Instanz-Ebenenvariable auf die zurückgegebene Ebenen-ID, wodurch die Instanz von der Ebene, auf der sie sich gerade befindet, auf die neu erstellte Ebene verschoben wird.
