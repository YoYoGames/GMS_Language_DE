Tiefe

Wenn Sie ein Objekt erstellen, können Sie ihm eine anfängliche Tiefe zuweisen, die definiert, wie die Instanzen dieses Objekts im Raum gezeichnet werden, wenn das Spiel gespielt wird, und diese Variable kann verwendet werden, um diesen Tiefenwert zu erhalten und zu ändern, während das Spiel läuft. Normalerweise brauchen Sie diese Variable nicht zu verwenden, da Sie die Instanzen so einstellen sollten, dass sie zu diskreten Ebenen hinzugefügt werden, die wiederum auf eine bestimmte Tiefe eingestellt sind, aber es kann sein, dass Sie die Tiefe einer Instanz mit diesem Wert ändern möchten. In diesem Fall wird eine "temporäre Ebene" speziell für die Instanz mit der angegebenen Tiefe erstellt. In diesem Fall wird eine "temporäre Ebene" speziell für die Instanz in der angegebenen Tiefe erstellt. Beachten Sie, dass diese temporäre Ebene aus dem Speicher entfernt wird, wenn sich keine Instanzen in der gleichen Tiefe befinden aus dem Speicher entfernt wird (im Gegensatz zu regulären Ebenen, die auch dann erhalten bleiben, wenn sich nichts auf ihnen befindet).

WICHTIG! Wenn Sie die Tiefenvariable ändern und GameMaker Studio 2 die Ebenen verwaltet, wird die eingebaute Ebene Variable -1 zurück und nicht die Ebenen-ID zurück, da verwaltete Ebenen nicht durch Code manipuliert werden können.

In GameMaker Studio 2 bedeutet ein niedriger Tiefenwert für eine Instanz, dass die Instanz "näher an der Kamera" gezeichnet wird, während ein höherer Tiefenwert bedeutet, dass die Instanz "weiter weg von der Kamera" gezeichnet wird, d. h: -1000 wird über -100 gezeichnet, das über 0 gezeichnet wird, das über 100 gezeichnet wird, das über 1000 gezeichnet wird.

Beachten Sie, dass Instanzen eines Objekts, die dieselbe Tiefe haben, über- oder untereinander gezeichnet werden können, auch wenn sie im Raumeditor scheinbar übereinander liegen. Wenn Sie garantieren wollen, dass etwas über oder unter alles andere gezeichnet wird, sollten Sie die Tiefe (oder Ebene) immer explizit festlegen. Beachten Sie auch, dass es ein Minimum (-16000) und einen Maximalwert (+16000) für die Tiefe gibt, außerhalb derer die Instanzen nicht gezeichnet werden, obwohl sie weiterhin existieren und Ereignisse verarbeiten.

HINWEIS: Sie können die Tiefe einer Instanz nicht in ihrem Zeichnungsereignis festlegen (alle anderen Ereignisse sind in Ordnung).

Syntax:

Tiefe;

Rückgabe:

Real (Gleitkommawert mit einfacher Genauigkeit)

Beispiel:

if y != yvorher
{
depth = -y;
}

Der obige Code prüft, ob sich die y-Position geändert hat, und wenn ja, wird auch die Tiefe entsprechend eingestellt.
