tag_get_asset_ids

Mit dieser Funktion können Sie alle Assets eines bestimmten Typs abrufen, denen die angegebenen Tags zugewiesen sind. Sie übergeben entweder ein einzelnes Tag (als Zeichenkette) oder ein Array, wobei jedes Element im Array ein Tag (als Zeichenkette) ist, sowie den Typ des zu prüfenden Assets. Der Typ des Assets sollte eine der folgenden Konstanten sein:

Konstant Beschreibung
Asset_Objekt Der angegebene Name bezieht sich auf ein Objekt.
asset_sprite Der angegebene Name bezieht sich auf ein Sprite.
asset_klang Der Vorname bezieht sich auf einen Klang.
asset_raum Der angegebene Name bezieht sich auf einen Raum.
asset_tiles Der angegebene Name bezieht sich auf einen Kachelsatz.
asset_pfad Der angegebene Name verweist auf einen Pfad.
asset_script Der angegebene Name verweist auf ein Skript.
asset_font Der angegebene Name bezieht sich auf eine Schriftart.
asset_Zeitleiste Der angegebene Name bezieht sich auf eine Zeitlinie.
asset_shader Der angegebene Name bezieht sich auf einen Shader.
asset_animationcurve Der angegebene Name bezieht sich auf eine Animationskurve.
asset_sequenz Der angegebene Name bezieht sich auf eine Sequenz.

Die Funktion gibt ein Array zurück, wobei jedes Element im Array ein einzelner Asset-Indexwert ist. Wenn es keine Assets des Typs gibt, die den/die angegebenen Tag(s) haben, wird ein leeres Array zurückgegeben.

Syntax:

tag_get_asset_ids(tags, asset_type);

Argument Beschreibung
tags Eine einzelne Asset-Tag-Zeichenkette oder ein Array mit verschiedenen Asset-Tags.
asset_type Eine Asset-Typ-Konstante (oben aufgeführt)

Rückgabe:

Array

Beispiel:

var _paths = tag_get_asset_ids("enemy", asset_path);
var _num = irandom(array_length(_paths) - 1);
path_start(_paths[_num], 1, path_action_reverse, false);

Der obige Code verwendet das Tag "enemy", um alle Pfad-Assets mit diesem Tag zu finden, bevor er eines zufällig auswählt und es der Instanz zuweist, die den Code ausführt.
