asset_hats_tags

Mit dieser Funktion können Sie prüfen, ob ein oder mehrere Tag-Strings einem beliebigen Asset aus dem Asset-Browser zugeordnet sind. Sie geben entweder den Asset-Namen (als Zeichenkette) oder seinen Asset-Index an, sowie entweder eine einzelne Tag-Zeichenkette oder ein Array, in dem jedes Element eine einzelne Tag-Zeichenkette ist. Wenn Sie einen Asset-Indexwert angeben, müssen Sie das optionale Asset-Typ-Argument (eine Konstante) angeben, da Assets unterschiedlichen Typs denselben Index haben können, auch wenn sie nicht denselben Namen haben. Die verfügbaren Kühlstellentypen sind in der folgenden Tabelle aufgeführt:

Konstant Beschreibung
Asset_Objekt Der angegebene Name bezieht sich auf ein Objekt.
asset_sprite Der angegebene Name bezieht sich auf ein Sprite.
asset_klang Der Vorname bezieht sich auf einen Klang.
asset_raum Der angegebene Name bezieht sich auf einen Raum.
asset_tiles Der angegebene Name bezieht sich auf einen Kachelsatz.
asset_pfad Der angegebene Name verweist auf einen Pfad.
asset_script Der angegebene Name verweist auf ein Skript.
asset_font Der angegebene Name bezieht sich auf eine Schriftart.
asset_Zeitleiste Der angegebene Name bezieht sich auf eine Zeitleiste.
asset_shader Der angegebene Name bezieht sich auf einen Shader.
asset_animationcurve Der angegebene Name bezieht sich auf eine Animationskurve.
asset_sequenz Der angegebene Name bezieht sich auf eine Sequenz.

Wenn die Funktion erfolgreich ist und alle angegebenen Tags für das Asset vorhanden sind, gibt sie true zurück, andernfalls wird false zurückgegeben. Wenn Sie nicht auf alle Tags, sondern auf eine Auswahl von Tags prüfen wollen, verwenden Sie die Funktion asset_hats_any_tag().

Syntax:

asset_has_tags(name_oder_index, tags, [asset_type]);

Argument Beschreibung
name_oder_index Der Name des Assets (eine Zeichenfolge) oder sein Indexwert (eine ganze Zahl).
tags Eine einzelne Asset-Tag-Zeichenkette oder ein Array mit verschiedenen Asset-Tags.
[asset_type] OPTIONAL! Der Typ des Assets, auf den die Tags geprüft werden sollen, wird nur verwendet, wenn ein Index für das erste Argument angegeben wird.

Rückgabe:

Boolesche

Beispiel:

var _a = array_create(3);
_a[0] = "enemy";
_a[1] = "level_" + string(global.Level);
_a[2] = "boss";
if asset_has_tags(object_index, _a, asset_object)
{
instance_create_layer(0, 0, "Overlay", obj_Boss_Text);
}

Der obige Code erstellt ein Array von Tags und prüft dann, ob alle von ihnen auf das angegebene Objekt angewendet werden, und wenn dies der Fall ist, wird eine Instanz eines anderen Objekts erstellt.
