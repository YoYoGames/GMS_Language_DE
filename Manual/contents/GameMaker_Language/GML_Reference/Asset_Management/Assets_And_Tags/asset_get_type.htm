asset_get_type

Mit dieser Funktion können Sie den Typ des Assets, auf das verwiesen wird, aus seinem Namen (einer Zeichenkette) ermitteln. Es wird eine der unten aufgeführten Konstanten zurückgegeben.

Syntax:

asset_get_type(name);

Argument Beschreibung
Name Der Name des Spiel-Assets, dessen Typ ermittelt werden soll (eine Zeichenkette).

Rückgabe:

Konstant

Konstant Beschreibung
Asset_Objekt Der angegebene Name bezieht sich auf ein Objekt.
asset_sprite Der angegebene Name bezieht sich auf ein Sprite.
asset_klang Der Vorname bezieht sich auf einen Klang.
asset_raum Der angegebene Name bezieht sich auf einen Raum.
asset_tiles Der angegebene Name bezieht sich auf einen Kachelsatz.
asset_pfad Der angegebene Name verweist auf einen Pfad.
asset_script Der angegebene Name verweist auf ein Skript.
asset_font Der angegebene Name bezieht sich auf eine Schriftart.
asset_Zeitleiste Der angegebene Name bezieht sich auf eine Zeitlinie.
asset_shader Der angegebene Name bezieht sich auf einen Shader.
asset_animationcurve Der angegebene Name bezieht sich auf eine Animationskurve.
asset_sequenz Der angegebene Name bezieht sich auf eine Sequenz.
asset_unbekannt Der angegebene Name bezieht sich auf ein Asset, das entweder nicht existiert oder nicht zu den oben aufgeführten gehört.

Beispiel:

if asset_get_type("pth_Path_" + string(global.Game)) == asset_unknown
{
show_debug_message("Path doesn't exist!!!");
}
sonst
{
path_index = asset_get_index("pth_Path_" + string(global.Game));
}

Der obige Code prüft einen dynamisch erstellten Asset-Namen, um zu sehen, ob das Asset vom richtigen Typ ist. Wenn dies nicht der Fall ist, wird eine Debug-Meldung angezeigt, andernfalls wird der Asset-Name verwendet, um das Asset der Instanz zuzuordnen.
