asset_get_tags

Mit dieser Funktion können Sie alle einem Asset zugeordneten Tags aus dem Asset-Browser abrufen. Sie geben entweder den Asset-Namen (als Zeichenkette) oder seinen Asset-Index an, und die Funktion gibt ein Array von Tags für dieses Asset zurück. Wenn keine Tags gefunden werden oder ein Fehler auftritt (d. h.: die angegebene Namenszeichenkette existiert nicht), ist das zurückgegebene Array leer. Wenn Sie einen Asset-Indexwert angeben, müssen Sie das optionale Asset-Typ-Argument (eine Konstante) angeben, da Kühlstellen unterschiedlichen Typs denselben Index haben können, auch wenn sie nicht denselben Namen haben können. Die verfügbaren Kühlstellentypen sind in der folgenden Tabelle aufgeführt:

Konstant Beschreibung
asset_object Der angegebene Name bezieht sich auf ein Objekt.
asset_sprite Der angegebene Name bezieht sich auf ein Sprite.
asset_klang Der Vorname bezieht sich auf einen Klang.
asset_raum Der angegebene Name bezieht sich auf einen Raum.
asset_tiles Der angegebene Name bezieht sich auf einen Kachelsatz.
asset_pfad Der angegebene Name verweist auf einen Pfad.
asset_script Der angegebene Name verweist auf ein Skript.
asset_font Der angegebene Name bezieht sich auf eine Schriftart.
asset_Zeitleiste Der angegebene Name bezieht sich auf eine Zeitlinie.
asset_shader Der angegebene Name bezieht sich auf einen Shader.
asset_animationcurve Der angegebene Name bezieht sich auf eine Animationskurve.
asset_sequenz Der angegebene Name bezieht sich auf eine Sequenz.

Syntax:

asset_get_tags(name_oder_index, [asset_type]);

Argument Beschreibung
name_oder_index Der Name des Assets (eine Zeichenfolge) oder sein Indexwert (eine ganze Zahl).
[asset_type] OPTIONAL! Der Typ des Assets, für den die Tags abgerufen werden sollen, wird nur verwendet, wenn ein Index für das erste Argument angegeben wird.

Rückgabe:

Array

Beispiel:

my_tags = asset_get_tags(object_get_name(object_index));

Der obige Code ruft alle Tags ab, die dem Objekt zugewiesen sind, von dem die Instanz, die den Code ausführt, erstellt wurde.
