object_is_ancestor

Diese Funktion kann verwendet werden, um ein Objekt zu überprüfen und festzustellen, ob es ein Vorfahre eines anderen Objekts ist oder nicht. Beide Argumente müssen also Objektindizes und nicht die Instanz-IDs sein, und das erste Argument ist immer das Objekt, das Sie herausfinden wollen, ob es ein Kind des zweiten Arguments ist, das immer das Objekt ist, das als Vorfahre (Elternteil) überprüft werden soll.

Syntax:

object_is_ancestor(obj, par);

Argument Beschreibung
obj Das Objekt, das als untergeordnetes Objekt geprüft wird.
par Das Objekt, das als Vorfahre (Parent) geprüft wird.

Rückgabe:

Boolesche

Beispiel:

if object_is_ancestor(object_index, obj_Enemy)
{
instance_destroy();
}

Der obige Code prüft, ob die Instanz, die den Code ausführt, ein Kind des Objekts "obj_Enemy" ist, und wenn ja, wird es zerstört.
