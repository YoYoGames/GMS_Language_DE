Objekte

In GameMaker Studio 2 haben Sie Objekte und Sie haben Instanzen. Objekte sind im Wesentlichen die Basisvorlage für eine Instanz und sind als solche nie direkt in einem Raum vorhanden... nur Instanzen des Objekts werden im Raum platziert. Dies bedeutet, dass Sie, wenn Sie etwas für alle Instanzen, die Sie erstellen, ändern möchten, dies tun können, indem Sie das Objekt ändern.

WICHTIG! Wenn Sie etwas an einem Objekt ändern, werden keine aktuell im Raum vorhandenen Instanzen geändert, sondern nur die, die nach der Änderung angelegt werden.

Sie können auch auf Informationen über ein Basisobjekt zugreifen und diese verwenden, um Verhaltensweisen oder Änderungen im Spiel zu diktieren. Sie können zum Beispiel nach der ID eines übergeordneten Objekts suchen, und wenn im folgenden Code eine Instanz mit diesem übergeordneten Objekt gefunden wird, können Sie dann prüfen seinen object_index prüfen, um zu entscheiden, was getan werden soll.

GameMaker Studio 2 stellt Ihnen eine Reihe von Funktionen zur Verfügung, mit denen Sie verschiedene Details über ein Objekt erhalten können. Beachten Sie, dass ein Objekt keine Instanz (eine Entität im Spiel) ist, sondern lediglich eine Ressource, aus der alle Instanzen erzeugt werden. Das bedeutet dass die Rückgabewerte dieser Funktionen von den tatsächlichen Werten abweichen können, die von Instanzen im Raum angezeigt werden, da eine einmal erstellte Instanz durch Code und DnD™-Aktionenverändert werden kann.

  • Objekt_Index
  • Objekt_existiert
  • objekt_get_name
  • objekt_get_mask
  • object_get_parent
  • objekt_get_persistent
  • object_get_solid
  • object_get_sprite
  • object_get_visible
  • objekt_get_physik
  • object_is_ancestor

Es gibt auch eine Reihe von Funktionen, mit denen Sie die Eigenschaften eines Objekts einstellen können. Es ist zu beachten, dass alle Instanzen dieses Objekts, die bereits im Raum vorhanden sind, von diesen Funktionen nicht betroffen sind, aber alle neuen Instanzen dieses Objekts, die im Raum angelegt werden, schon. Es wird daher empfohlen, die Eigenschaften eines Objekts niemals zu ändern, wenn Instanzen dieses Objekts im aktuellen Raum vorhanden sind.

  • object_set_mask
  • object_set_persistent
  • object_set_solid
  • object_set_sprite
  • object_set_visible

Sie können auch bestimmte Funktionen verwenden, um Ereignisse von Code aus einer bestimmten Instanz oder einem bestimmten Objekt zu erzeugen. Diese Funktionen können sehr nützlich sein, insbesondere beim Umgang mit Parents und Children, und werden im folgenden Abschnitt beschrieben:

  • Erzeugen von Objektereignissen
