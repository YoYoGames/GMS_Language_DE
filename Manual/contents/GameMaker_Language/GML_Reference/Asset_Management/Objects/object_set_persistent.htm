object_set_persistent

Mit dieser Funktion können Sie die Persistenz eines bestimmten Objekts einstellen. Ein persistentes Objekt wird von Raum zu Raum "mitgenommen" und behält seine Eigenschaften zu jeder Zeit bei (weitere Informationen zur Persistenz finden Sie im Abschnitt über den Objekteditor). Das Einstellen der Persistenz des Objekts bedeutet, dass alle Instanzen dieses Objekts, die nach dem Ändern der Persistenz angelegt werden, persistent angelegt werden, während Instanzen, die sich bereits im Raum befinden, möglicherweise nicht betroffen sind.

Wenn Sie eine Instanz mit aktivierter Persistenz in einem Raum erstellen, wird ihr eine Ebene oder eine Tiefe zugewiesen (je nachdem, welche Funktion Sie zum Erstellen der Instanz des Objekts verwendet haben). Wenn der Raum gewechselt wird und der folgende Raum nicht über eine Ebene mit dem gleichen Namen oder der gleichen Tiefe wie die zugewiesene Ebene hat, wird eine neue Ebene für die Instanz erstellt, die über die Räume hinweg bestehen bleibt. Wenn Sie beim Anlegen der Instanz eine benannte Schicht zugewiesen haben, wird auch die neue Schicht mit dem gleichen Namen wie die ursprüngliche Schicht benannt. Wenn Sie der Instanz eine Tiefe zugewiesen haben, erhält die neue Ebene den Namen "_layer_XXX", wobei "XXX" ein Hexadezimalwert ist, um der Ebene einen eindeutigen Namen zu geben. Diese Ebenen werden aus dem Raum entfernt am Ende des Raums entfernt. Ein weiterer Hinweis zu Ebenen und persistenten Instanzen ist, dass, wenn Sie der Instanz bei der Erstellung eine benannte Ebene zugewiesen haben und es in den folgenden Räumen eine andere Ebene mit demselben Namen gibt, die persistente Instanz der Ebene mit demselben Namen zugewiesen , unabhängig von der Tiefe der Ebene. Wenn schließlich eine persistierte Instanz in einen Raum mit einer Ebene in der gleichen Tiefe wie bei der Erstellung der Instanz wechselt, wird sie nicht dieser Ebene zugewiesen, sondern stattdessen wird eine neue Ebene in der gleichen Tiefe erstellt (gemäß der zuvor erläuterten Namenskonvention).

Bitte beachten Sie, dass es sich hierbei nicht um eine Instanzfunktion handelt! Sie können das Persistent-Flag einzelner Instanzen mit der persistent Variable setzen und so zehn Instanzen alle als persistent kennzeichnen auch wenn das betreffende Objekt das Flag in der IDE auf false gesetzt hat, und selbst wenn Sie das Persistent-Flag des Objekts mit dieser Funktion auf true ändern, bleiben alle Instanzen, die sich derzeit im Raum befinden, so wie sie waren, und nur Instanzen, die nach dem Aufruf der Funktion Instanzen, die nach dem Aufruf der Funktion erstellt werden, beginnen mit dem Flag "persistent" auf "true".

Syntax:

object_set_persistent(index, pers);

Argument Beschreibung
index Der Index des zu ändernden Objekts.
pers Ob das Objekt als persistent gekennzeichnet werden soll(true) oder nicht(false).

Rückgabe:

N/A

Beispiel:

if (room == rm_final)
{
object_set_persistent(obj_Player, false);
}
instance_create_layer(32, 32, "Instances", obj_Player);

Der obige Code prüft, in welchem Raum sich die Instanz gerade befindet, und wenn sie sich im angegebenen "rm_final" befindet, setzt er die Persistenz des Objekts "obj_Player" auf false und erzeugt dann die Instanz.
