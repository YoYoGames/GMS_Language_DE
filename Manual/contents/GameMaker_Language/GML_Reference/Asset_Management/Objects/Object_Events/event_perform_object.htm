event_perform_object

Diese Funktion funktioniert genauso wie event_perform() mit dem Unterschied, dass Sie dieses Mal Ereignisse von einem anderen Objekt angeben können. Hier gibt es viele Optionen, die eine vollständige Simulation aller möglicher Ereignisse, aber beachten Sie, dass dies buchstäblich nur den gesamten Code des Ereignisses ausführt und das Spiel nichts ändert, um es selbst manuell auszulösen, z. B. wenn Sie ein Tastaturdruck-Ereignis wählen, wird das Ereignis ausgelöst aber die entsprechende Taste wird nicht als gedrückt erkannt. Oder wenn Sie ein Alarmereignis ausführen, wird der Alarmzähler nicht auf -1 gesetzt, sondern zählt weiter herunter. Eine vollständige Liste der verfügbaren Konstanten, die diese Funktion benötigt, finden Sie auf der Seite für die Funktion ereignis_ausfuehren().

HINWEIS: Aktionen in dem mit dieser Funktion aufgerufenen Ereignis werden auf die aktuelle Instanz angewendet und nicht auf Instanzen des angegebenen Objekts.

Syntax:

event_perform_object(obj, type, numb);

Argument Beschreibung
obj Das Objekt, dessen Ereignis ausgelöst werden soll.
Typ Der Typ des auszuführenden Ereignisses. Siehe unten.
taub Das spezifische Ereignis, falls eines erforderlich ist (andernfalls verwenden Sie einfach 0).

Rückgabe:

N/A

Beispiel:

event_perform_object(obj_Player, ev_keypress, ord("W"));

Dies würde das Ereignis ausführen, das mit Tastaturprüfung gedrückt > W-Taste vom Objekt "obj_Player" in der aktuellen Instanz verbunden ist.
