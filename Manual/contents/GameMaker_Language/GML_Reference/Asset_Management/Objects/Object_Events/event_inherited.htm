event_inherited

Mit dieser Funktion wird das aktuelle Ereignis des übergeordneten Objekts der Instanz aufgerufen. Normalerweise erbt eine Instanz, wenn sie ein übergeordnetes Objekt hat, automatisch die gleichen Ereignisse wie das übergeordnete Objekt, aber wenn (z. B.) Ihr übergeordnetes Objekt ein Ereignis "create" hat und Aber wenn (z. B.) Ihr übergeordnetes Objekt ein create-Ereignis hat und Sie Ihrem untergeordneten Objekt eines hinzufügen, werden alle Instanzen des untergeordneten Objekts das neue create-Ereignis ausführen, das Sie hinzugefügt haben, und nicht das, das im übergeordneten Objekt vorhanden ist. Sollten Sie sowohl das Ereignis des übergeordneten Objekts als auch das Ereignis des untergeordneten Objekts verwenden müssen des gleichen Typs verwenden, sollten Sie diese Funktion verwenden, da sie das Ereignis des übergeordneten Objekts ausführt, bevor sie mit dem Rest des Codes oder der Aktionen fortfährt, die das Ereignis des untergeordneten Objekts enthält.

Syntax:

event_inherited();

Rückgabe:

N/A

Beispiel:

event_inherited();
switch (sprite_index)
{
case spr_Enemy_1: dmg += 2; break;
case spr_Enemy_4: dmg -= 1; break;
case spr_Enemy_10: dmg +=10; break;
}

Der obige Code ruft das geerbte übergeordnete Ereignis auf (in dem wir die Variable "dmg" initialisieren) und fährt dann fort, die Variable "dmg" zu ändern. Wenn kein übergeordnetes Ereignis für die Instanz, die diesen Code ausführt, würde ein Fehler "Unbekannte Variable" angezeigt, da dmg nicht definiert wurde.
