font_set_cache_size

Das Überblenden und Einfärben von Bildern kann in HTML5 nicht so dynamisch erfolgen, wie dies bei einer ausführbaren Datei möglich wäre. Daher muss GameMaker Studio 2 eine überblendete Kopie der Bilder vorübergehend speichern und bei Bedarf laden. laden, wenn sie benötigt werden. Diese Funktion legt fest, wie viele überblendete Kopien der angegebenen Schriftart zwischengespeichert werden können, bevor alte überschrieben werden. Der Standardwert ist 4.

HINWEIS: für Sprite-Fonts sollten Sie die entsprechende Funktion für Sprites verwenden, sprite_set_cache_size().

Syntax:

font_set_cache_size(ind, max);

Argument Beschreibung
ind Der Index der Schrift, deren Cache-Größe geändert werden soll.
max Die maximale Anzahl der zwischengespeicherten Kopien der Schrift, die gespeichert werden können.

Rückgabe:

N/A

Beispiel:

font_set_cache_size(fnt_MainMenu, 2);

Dadurch wird der Schrift-Cache der in der Variablen "fnt_MainMenu" indizierten Schrift auf 2 Kopien gesetzt.
