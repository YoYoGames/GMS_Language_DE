Schrift_hinzufügen

Mit dieser Funktion können Sie Ihrem Spiel eine Schriftart aus den Schriftarten hinzufügen, die auf dem System installiert sind, auf dem es läuft. Sie können die Größe der Schrift (in Punkten) festlegen, sowie ob die Schrift fett oder kursiv sein soll, und Sie können Sie können auch den Bereich der einzuschließenden Zeichen definieren. Die Funktion gibt einen Indexwert zurück, der in einer Variablen gespeichert werden sollte, da dieser in allen weiteren Codes, die sich auf diese Schriftart beziehen, benötigt wird, oder sie gibt -1 zurück, wenn die Funktion aus irgendeinem Grund fehlschlägt. Grund fehlschlägt.

Für das HTML5-Zielmodul kann diese Funktion nur verwendet werden, um Web Fonts hinzuzufügen, und sie funktioniert nur, wenn WebGL nicht aktiviert ist - wenn WebGL aktiviert ist, müssen Sie eine allgemeine Font-Ressource verwenden (d. h. mit draw_set_font() zusammen mit einer Schrift aus dem Asset-Browser) oder Bitmap-Schriften (siehe font_add_sprite()).

WICHTIG! Bei der Ausrichtung auf HTML5 wird kursiver und fetter Text nur unterstützt, wenn das Font-Asset oder die Webschrift selbst diese Zeichen enthält. Sie werden nicht aus der Basisschriftart synthetisiert.

Auf allen anderen (nicht Web-) Targets können Sie mit dieser Funktion eine Schrift aus einer Datei hinzufügen. Die Datei muss mit der Funktion " Enthaltene Dateien" von GameMaker®Studio 2 in das Spiel-Bundle aufgenommen werden. Studio 2 eingebunden werden und muss entweder eine Schriftartdatei im Format *.ttf oder *.otf sein, was für das Hinzufügen von Nicht-Standard-Schriftarten wie asiatisch oder arabisch nützlich ist.

WARNUNG! Wenn Sie eine solche Schriftartdatei in ein Spiel einbinden, muss sie für den Vertrieb mit dem Spiel lizenziert werden.

Wenn Sie eine Schriftart auf diese Weise aus einer Datei laden, wird die Größe der Schriftart in Pixeln angegeben und der erste und letzte Wert werden ignoriert, d. h. alle Glyphen der Schriftart werden hinzugefügt. Auf diese Weise hinzugefügte Schriften werden einer eigenen Textur zugewiesen Seite zugewiesen, daher ist bei der Verwendung dieser Funktion Vorsicht geboten, da sie beim Zeichnen die Anzahl der Texturwechsel erhöht. Es ist auch zu beachten, dass Schriften beim Zeichnen etwas größer erscheinen können, da Glyphen Teile haben können, die außerhalb des für die Schrift definierten Begrenzungsrahmens, der für die Schriftart definiert ist, gezeichnet werden. Sie sollten auch beachten, dass es eine Begrenzung von ca. 200 Glyphen gibt, die in einem einzelnen Frame für eine bestimmte Schriftart gerendert werden können.

Standardmäßig ist bei hinzugefügten Fonts Anti-Aliasing aktiviert. Sie können den Status von AA jedoch jederzeit mit der Funktion font_add_get_enable_aa() und Sie können AA mit der Funktion font_add_enable_aa().

HINWEIS: Wenn Sie eine Schriftart in GameMaker Studio 2: Studio laden, müssen Sie daran denken, sie wieder zu entfernen (mit font_delete()), wenn sie nicht mehr benötigt wird, sonst besteht die Gefahr eines Speicherlecks, das Ihr Spiel verlangsamt und schließlich zum Absturz bringt.

Syntax:

font_add(name, größe, fett, kursiv, erste, letzte);

Argument Beschreibung
Name Der Name der hinzuzufügenden Schriftart (z. B. "Arial") oder der Dateipfad, wenn die Schriftart eine enthaltene *.ttf- oder *. otf-Datei ist .
Größe Die Größe der Schrift - Punkte für Webfonts, Pixel für Dateifonts.
fett Ob die Schrift fett ist (true) oder nicht (false).
Kursivschrift Ob die Schrift kursiv ist (true) oder nicht (false).
erste Das erste einzuschließende Zeichen (wenn Sie unsicher sind, nehmen Sie 32).
zuletzt Das letzte einzuschließende Zeichen (wenn Sie unsicher sind, nehmen Sie 128).

Rückgabe:

Real

Beispiel:

newfont = font_add("Arial", 24, true, true, 32, 128);

Dadurch wird eine neue Schriftart mit einer Größe von 24pt erstellt, die "Arial" verwendet, die fett und kursiv ist, und der Index für diese neue Schriftart wird in der Variablen newfont gespeichert. Der Schriftbereich umfasst Groß- und Kleinbuchstaben, Zahlen und alle gängigen Interpunktion.
