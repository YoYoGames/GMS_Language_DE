font_get_first (veraltet)

Wenn Sie in GameMaker Studio 2 eine Schriftart definieren, können Sie einen Bereich von Zeichen festlegen, der einbezogen werden soll. Der Grund dafür ist, dass die Schriftart selbst nicht in Ihrem Spiel enthalten ist (aus rechtlichen Gründen), sondern ein Bild der der Schrift ist auf einer Texturseite enthalten, und das ist das, was Ihr Spiel verwenden wird (genau wie jedes andere Grafikobjekt). Das bedeutet, dass Sie die Anzahl der Zeichen, die Sie verwenden, auf ein Minimum beschränken und nur den Bereich der Zeichen angeben sollten, den Ihr Spiel benötigt, um den Texturspeicher so optimal wie möglich zu nutzen. Diese Funktion kann verwendet werden, um das Startzeichen (als ASCII-Wert) zu finden, das verwendet wurde, als Ihr Font-Asset zu Ihrem Spiel hinzugefügt wurde.

HINWEIS: Aufgrund von Änderungen in der Handhabung von Schriftarten gibt diese Funktion immer 32 zurück, da GameMaker Studio 2 das Leerzeichen (32) in der Schriftart erzwingt und auch keine Zeichen mit Codes kleiner als 32 zulässt. Neue Funktionen zur Erledigung ähnlicher Aufgaben werden in einem zukünftigen Update hinzugefügt.

Syntax:

font_get_first(ind);

Argument Beschreibung
ind Index der zu prüfenden Schriftart.

Rückgabe:

Real

Beispiel:

num = font_get_first(fnt_Main);

Der obige Code speichert den ASCII-Wert des ersten Buchstabens des Schriftbereichs für die in "fnt_Main" indizierte Schrift.
