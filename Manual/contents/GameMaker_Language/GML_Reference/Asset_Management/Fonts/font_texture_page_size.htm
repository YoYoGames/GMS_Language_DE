font_texture_page_size

Diese eingebaute Variable kann verwendet werden, um die Textur-Seitengröße entweder zu erhalten oder zu setzen, wenn die Funktion font_add(). Wenn eine Schriftart mit dieser Funktion hinzugefügt wird, legt GameMaker Studio 2 beim Hinzufügen einer Schrift mit dieser Funktion einen Texturseiten-Cache mit den benötigten Glyphen bis zu der durch diese Variable definierten Größe anlegen. Der Vorgang ist wie folgt:

  • Jedes Zeichen, das Sie verwenden, wird von der Schriftart in einer Texturseite zwischengespeichert
  • Wenn die Texturseite voll ist (weil Sie große Zeichen oder viele verschiedene Zeichen verwenden), entfernt GameMaker Studio 2 Zeichen aus dem Cache und ersetzt sie durch neue, die Sie benötigen
  • GameMaker Studio 2 versucht, die Textur nicht zu groß zu machen, weil nicht alle Geräte mit großen Texturen zurechtkommen, und es versucht, die Textur nicht zu klein zu machen, weil das Entfernen und Wiedereinfügen von Zeichen in die Textur CPU-Zeit kostet und dazu führt, dass Ihr Spiel langsam läuft

Wenn Sie der Meinung sind, dass Sie eine größere oder kleinere Textur-Seitengröße als die standardmäßige 1024px benötigen, die GameMaker Studio 2 verwendet, können Sie den Wert mit dieser eingebauten Variablen einstellen. Wenn Sie nicht die Funktion font_add() verwenden, verwenden, hat diese Einstellung keine Auswirkung auf die Leistung Ihres Spiels, da in der IDE hinzugefügte Schriftarten auf den regulären Textur-Seiten zwischengespeichert werden (was Sie über den Schriftarten-Editor und die die Spieloptionen für die Zielplattform).

Syntax:

font_texture_page_size;

Rückgabe:

Real (maximale Breite/Höhe in Pixel)

Beispiel:

font_texture_page_size = 512;
newfont = font_add("Arial", 24, true, true, 32, 128);

Der obige Code setzt die Schrift-Cache-Textur-Seite auf eine maximale Größe von 512px und fügt dann eine neue Schrift zum Spiel hinzu.
