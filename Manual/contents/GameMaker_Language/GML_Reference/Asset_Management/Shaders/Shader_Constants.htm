Shader-Konstanten

Neben den Shader-Funktionen und -Konstanten, die in den OpenGL ES Shading Language (GLSL ES) -Referenzseiten definiert sind, gibt es auch eine Reihe Shader-Konstanten zur Verfügung, die es nur in GameMaker Studio 2 gibt.

Die folgenden Display-Matrix-Konstanten können bei Verwendung der Shader-Array-Konstante gm_Matrices als Array-Indizes verwendet werden:

Konstant Beschreibung
MATRIX_ANSICHT Diese Array-Indexkonstante enthält den Index der aktuellen Ansichtsmatrix. Der zurückgegebene Index würde als Array-Wert beim Aufruf der Konstante gm_Matrices innerhalb des Shader-Codes verwendet werden.
MATRIX_PROJEKTION Diese Array-Indexkonstante hält den Index der aktuellen Projektionsmatrix. Der zurückgegebene Index würde als Array-Wert beim Aufruf der Konstante gm_Matrices innerhalb des Shader-Codes verwendet werden.
MATRIX_WELT Diese Array-Indexkonstante hält den Index zur aktuellen Weltmatrix. Dies kann für Dinge wie Beleuchtung verwendet werden, wenn Sie Lichtinformationen im Weltraum haben. Der zurückgegebene Index würde als Array-Wert beim Aufruf der Konstante gm_Matrices im Shader-Code verwendet.
MATRIX_WELT_ANSICHT Diese Array-Indexkonstante hält den Index für das Ergebnis der Welt- und Ansichtsmatrizen, die miteinander multipliziert werden. Dies wird oft für Dinge wie Nebel verwendet. Der zurückgegebene Index wird als Array-Wert verwendet, wenn die Konstante gm_Matrices innerhalb des Shader-Codes verwendet.
MATRIX_WELTANSICHT_PROJEKTION Diese Array-Indexkonstante enthält den Index für das Ergebnis der Welt-, Ansichts- und Projektionsmatrizen, die miteinander multipliziert werden. Dies ist die normale Transformationsmatrix, die für Scheitelpunktpositionen verwendet wird. Der zurückgegebene Index würde als Array-Wert verwendet werden, wenn Aufruf der Konstante gm_Matrices innerhalb des Shader-Codes verwendet.
MATRIX_MAX Dies ist kein Array-Index, sondern gibt die Größe des Matrix-Arrays im Vertex-Shader zurück.
MAX_VS_LICHTEN Dies ist kein Array-Index, sondern gibt die Nummer der Lichter im Vertex-Shader zurück.

Die folgende Konstante ist auch für die Beleuchtung verfügbar:

Konstant Beschreibung
MAX_VS_LICHTEN Die maximale Anzahl der im Shaderverfügbaren Punkt- und Richtungslichter

Die folgenden vordefinierten Matrix-Uniformen und Konstanten können in Ihrem Shader verwendet werden, um auf GameMaker Studio 2-spezifische Werte zuzugreifen:

Konstant Beschreibung
gm_Matrices[matrix] Diese Array-Konstante gibt eine Transformationsmatrix zurück und ist eine der verfügbaren vordefinierten Uniformen, die GameMaker Studio 2 für Sie zur Verwendung im Shader-Code-Editor erstellt. Der Array-Index wird aus einer der oben aufgeführten Konstanten gewählt.
gm_BaseTextur Dies ist eine 2D-Sampler-Konstante, die die Textur des aktuellen Objekts zurückgibt, wie sie von GameMaker Studio 2 eingestellt wurde. Es wäre also das aktuelle Sprite, die Oberfläche oder die Textur, die normalerweise beim Zeichnen verwendet würde, ohne dass der Shader aufgerufen wird.
gm_LightingEnabled Hiermit können Sie die GameMaker Studio 2-Beleuchtung bei der Verwendung von 3D abrufen oder einstellen.
gm_NebelStart Damit lässt sich der Abstand ermitteln, ab dem Polygone mit der Nebelfarbe überblendet werden.
gm_RcpFogRange Damit lässt sich die Entfernung ermitteln, bei der der Nebel maximal ist und nichts mehr zu sehen ist.
gm_PS_FogEnabled Dies gibt true oder false zurück, wenn die GPU Pixelnebel aktiviert hat oder nicht.
gm_NebelFarbe Damit können Sie die von GameMaker Studio 2 verwendete Nebelfarbe erhalten.
gm_VS_FogEnabled Dies gibt true oder false zurück, wenn die GPU Vertex-Nebel aktiviert hat oder nicht.
gm_AlphaTestEnabled Dies kann verwendet werden, um Alpha-Tests im Shader zu aktivieren. Siehe gpu_set_alphatestenable() für weitere Informationen zum Alphatest.
gm_AlphaRefValue Hiermit kann der aktuelle Alpha-Test-Referenzwert ermittelt werden. Siehe gpu_set_alphatestref() für weitere Informationen über die Alphatestreferenz.
