audio_play_sound_at

Mit dieser Funktion können Sie eine beliebige Klangressource an einer bestimmten Position innerhalb des Audio-Raums abspielen. Sie geben den Sound-Index an und weisen ihm dann eine Position innerhalb des 3D-Raums zu. Die Standardposition des Hörers ist (0, 0, 0), d. h., wenn der Hörer nicht verschoben wurde und Sie möchten, dass der Ton z. B. von links kommt, sollten Sie die x-Position auf einen negativen Wert setzen (weitere Informationen zum Einstellen der Hörerposition finden Sie unter audio_listener_position()).

Sie können auch eine Abklingdistanz einstellen (bei 0 ist der Ton stumm, Voreinstellung ist 100), die den Ton ausklingen lässt, wenn er sich weiter von der Hörerposition entfernt. Wie die Ausblendung selbst gehört wird, hängt von der Falloff-Referenz ab (das ist der Abstand (das ist die Entfernung, unter der die Lautstärke der Quelle normalerweise um die Hälfte abfällt) und dem Abklingfaktor (der den Klang nur über die Abklingreferenzdistanz hinaus beeinflusst) ab. Der voreingestellte Faktor ist normalerweise 1, und die Auswirkung der verschiedenen Falloff-Werte hängt von dem gewählten Modell ab (eine vollständige Anleitung zu den verschiedenen Falloff-Modellen und der Verwendung dieser Werte finden Sie in der Funktion audio_falloff_set_model()).

Die letzten beiden Argumente dienen zum Festlegen, ob der Sound in einer Schleife laufen soll oder nicht, und schließlich zum Zuweisen einer Priorität für den Sound. Diese Priorität wird dann verwendet, um zu bestimmen, wie Klänge behandelt werden, wenn die Anzahl der spielenden Klänge über dem von der Funktion festgelegten Limit liegt audio_channel_num(). Klänge mit niedrigerer Priorität werden zugunsten von Klängen mit höherer Priorität gestoppt, und der Prioritätswert kann eine beliebige reale Zahl sein (der tatsächliche Wert ist willkürlich und kann von 0 bis 1 oder 0 bis 100 sein, da GameMaker Studio 2 sie gleich priorisiert). Beachten Sie, dass bei der Priorität gilt: Je höher die Zahl, desto höher die Priorität, so dass ein Sound mit der Priorität 100 gegenüber einem Sound mit der Priorität 1 bevorzugt wird.

Diese Funktion gibt eine eindeutige Indexnummer für den abgespielten Sound zurück, die dann in einer Variablen gespeichert werden kann, so dass Sie ihn dann mit den entsprechenden Funktionen anhalten oder stoppen können. Das bedeutet, dass Sie, wenn Sie mehrere Instanzen desselben Klangs haben Wenn Sie also mehrere Instanzen desselben Tons gleichzeitig abspielen, können Sie nur eine Instanz dieses Tons mit den Audiofunktionen bearbeiten.

Syntax:

audio_play_sound_at(index, x, y, z, falloff_ref, falloff_max, falloff_factor, loop, priority);

Argument Beschreibung
index Der Index des abzuspielenden Tons.
x Die X-Position.
y Die y-Position.
z Die z-Position.
falloff_ref Die Abblendreferenz relativ zum Hörer (Klammer).
falloff_max Die maximale Falloff-Distanz relativ zum Hörer.
falloff_factor Der Falloff-Faktor (Standardwert 1).
Schleife Kennzeichnet den Ton als loopend oder nicht.
Priorität Stellen Sie die Kanalpriorität für den Ton ein.

Rückgabe:

Index (Sound-ID)

Beispiel:

if global.SFX
{
audio_play_sound_at(snd_Wasserfall, x, y, 0, 100, 300, 1, true, 1);
}

Der obige Code prüft die globale Variable "SFX" und wenn er true zurückgibt, wird der in der Variable "snd_Waterfall" indizierte Sound an seiner Raumposition geloopt, mit einer Falloff-Referenz von 100, einem Falloff-Abstand von 300, einem Falloff-Faktor von 1 und einer niedrigen Priorität.
