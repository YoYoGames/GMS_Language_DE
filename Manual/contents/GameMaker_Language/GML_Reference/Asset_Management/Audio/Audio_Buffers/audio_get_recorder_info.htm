audio_get_recorder_info

Diese Funktion gibt eine DS-Map mit Informationen über den angegebenen Recorder-Quellenindex zurück. Sie können herausfinden, wie viele Recorder-Quellen verfügbar sind, indem Sie die Funktion audio_get_recorder_count(), und die zurückgegebene Map enthält die folgenden Schlüssel/Wert-Paare:

  • "Name"- ein Name zur Beschreibung des Geräts
  • "index" - der Index, der für die Aufzeichnung verwendet werden soll
  • "data_format" - das Format, in dem die Daten zurückgegeben werden (derzeit ist dies immer buffer_s16 , aber andere Formate können in Zukunft unterstützt werden)
  • "sample_rate" - die Abtastrate (in Hz) der zurückgegebenen Daten (derzeit auf 16000hz geklemmt, aber das kann sich in Zukunft ändern)
  • "channels" - die Konstante audio_mono (weitere Konstanten für Stereo und 3D werden möglicherweise in Zukunft unterstützt)

Beachten Sie, dass die Funktion zwar eine DS-Map für Sie anlegt, diese aber später nicht wieder entfernt. Daher sollten Sie die Map selbst zerstören, wenn sie nicht mehr benötigt wird, um Speicherlecks zu vermeiden.

HINWEIS: Die meisten Plattformen außer HTML5 unterstützen die Aufnahme von Audio in irgendeiner Form, aber das bedeutet nicht, dass alle Geräte dies zulassen, selbst wenn die Plattform dies tut. audio_get_recorder_count() Funktion einen Wert größer als 0 zurückgibt, um zu prüfen, ob Aufnahmegeräte verfügbar sind, bevor Sie die restlichen Aufnahmefunktionen verwenden.

Syntax:

audio_get_recorder_info(recorder_index);

Argument Beschreibung
schreiber_index Der Index der Schreiberquelle, von der die Informationen abgerufen werden sollen.

Rückgabe:

Integer (DS-Karten-ID)

Beispiel:

len = async_load[? "data_len"];
audio_buff = buffer_create(len, buffer_fast, 1);
buffer_copy(async_load[? "buffer_id"], 0, len, buff, 0);
audio_queue_sound(audio_queue, audio_buff, 0, len);
audio_play_sound(audio_queue, 10, 0);

Der obige Code würde im asynchronen Audio-Aufnahme-Ereignis aufgerufen werden und weist einige aufgenommene Daten einem Puffer zu, der dann zu einer Audio-Warteschlange hinzugefügt wird. Diese wird dann abgespielt.
