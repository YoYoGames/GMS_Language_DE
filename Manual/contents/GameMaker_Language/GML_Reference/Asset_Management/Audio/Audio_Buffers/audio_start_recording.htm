audio_start_aufzeichnung

Diese Funktion startet die Audioaufnahme von der indizierten Rekorderquelle. Sie können die Anzahl der Rekorderquellen mit der Funktion audio_get_recorder_count()abrufen, und sobald Sie die Aufnahme starten, wird das Audio wird in einem temporären Puffer gespeichert und löst ein asynchrones Ereignis für die Audioaufzeichnung aus. Dieses Ereignis wird bei jedem Schritt ausgelöst, bei dem die Aufnahme stattfindet und erzeugt die spezielle DS-Map in der Variablen async_load mit den folgenden Schlüssel/Wert-Paaren:

  • "buffer_id" - die ID des temporären Puffers, den Sie zum Abrufen der Audiodaten verwenden können
  • "channel_index" - der Index des Aufnahmekanals (wie von der Funktion zurückgegeben), von dem diese Daten stammen
  • "data_len" - die Länge der empfangenen Daten (in Bytes)

Beachten Sie, dass nach dem Auslösen des asynchronen Ereignisses die Audiodaten im temporären Puffer gelöscht werden, so dass Sie die Daten in einen benutzerdefinierten Puffer kopieren sollten, den Sie zuvor erstellt haben.

HINWEIS: Die meisten Plattformen mit Ausnahme von HTML5 unterstützen die Aufnahme von Audio in irgendeiner Form, aber das bedeutet nicht, dass alle Geräte dies zulassen, selbst wenn die Plattform dies tut, daher sollten Sie immer prüfen, ob die audio_get_recorder_count() Funktion einen Wert größer als 0 zurückgibt, um zu prüfen, ob Aufnahmegeräte verfügbar sind, bevor Sie die restlichen Aufnahmefunktionen verwenden.

Syntax:

audio_start_aufzeichnung(recorder_index);

Argument Beschreibung
schreiber_index Der Index der zu verwendenden Schreiberquelle.

Rückgabe:

Real (Puffer-ID)

Beispiel:

audio_record = audio_start_recording(0);

Der obige Code startet die Aufnahme von der Rekorderquelle 0 und speichert den Kanalindex der Aufnahme in der Variablen "audio_record" zur Verwendung im asynchronen Ereignis "Audioaufnahme".
