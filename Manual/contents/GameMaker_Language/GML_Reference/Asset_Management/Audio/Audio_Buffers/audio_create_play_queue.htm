audio_erzeugen_wiedergabe_warteschlange

Diese Funktion bereitet eine Pufferwarteschlange für Audio vor. Sie übergeben das Datenformat (derzeit werden nur buffer_u8 oder buffer_s16 unterstützt), die Abtastrate (die zwischen 1000hz und 48000hz liegen kann) und die Kanäle, die das Audio benötigt, aus einer der unten aufgeführten Konstanten:

Konstant Beschreibung
audio_mono Mono (einkanalig) Audio.
audio_stereo Stereo (Zweikanal) Audio.
audio_3D 3D (5.1) Audio.


Beachten Sie, dass Sie mit dieser Funktion erstellte Audio-Warteschlangen wieder freigeben sollten, wenn sie nicht mehr benötigt werden, um Speicherlecks zu vermeiden, indem Sie die Funktion audio_free_play_queue().

Diese Funktion löst ein asynchrones Ereignis für die Audiowiedergabe aus, wenn die Warteschlange das Ende erreicht hat, und in diesem Ereignis wird eine spezielle DS-Map in der Variablen async_load mit den folgenden Schlüssel/Wert-Paaren erstellt:

  • " queue_id" - die Warteschlange, die die Datenwiedergabe beendet hat
  • "buffer_id" - der Puffer, von dem nicht mehr gespielt wird
  • " queue_shutdown" - dies wird während der normalen Wiedergabe auf 0 gesetzt und auf 1, wenn das Ereignis audio_free_play_queue() aufgerufen wurde. Wenn dieser Wert auf 1 gesetzt ist, wollen Sie keine wollen Sie keine weiteren Daten in die Warteschlange stellen.

Syntax:

audio_create_play_queue(queueFormat, queueRate, queueChannels);

Argument Beschreibung
queueFormat Das Format der zu verwendenden Pufferdaten(buffer_u8 oder buffer_s16).
queueRate Die Abtastrate der Daten in der Pufferwarteschlange.
queueChannels Die zu verwendenden Kanäle aus einer der unten aufgeführten Konstanten.

Rückgabe:

Real (Puffer-ID)

Beispiel:

audio_queue = audio_create_play_queue(buffer_s16, 11052, audio_mono);

Der obige Code erstellt eine neue Audio-Warteschlange und weist ihre ID einer Variablen für die zukünftige Verwendung zu.
