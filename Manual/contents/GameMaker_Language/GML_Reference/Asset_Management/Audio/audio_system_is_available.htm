audio_system_ist_verfügbar

Mit dieser Funktion kann geprüft werden, ob das Audiosystem initialisiert wurde oder ob der Audiokontext ausgeführt wird. Auf allen Plattformen gibt diese Funktion unmittelbar nach dem Spielstart true zurück, wenn die Audio-Engine initialisiert ist, außer auf dem HTML5-Ziel. Auf HTML5 kann sich der Status des Audiokontexts jederzeit ändern, abhängig von den Benutzereingaben, dem verwendeten Browser und dessen Version, so dass diese Funktion verwendet werden kann, um zu prüfen, ob Audio abgespielt werden kann oder nicht. Wenn die Funktion false zurückgibt (d. h.: der Audiokontextstatus ist nicht aktiv), dann können nur nicht gestreamte Töne abgespielt werden (aber das ist nicht garantiert, und Sie sollten davon ausgehen, dass kein Audio abgespielt werden kann), während, wenn sie true zurückgibt, dann werden alle Audiosignale abgespielt.

Syntax:

audio_system_is_available();

Rückgabe:

Boolesche

Beispiel:

if audio_system_is_available()
{
if audio_is_paused(global.Music)
{
audio_resume_sound(global.Musik)
}
sonst
{
if !audio_is_playing(global.Music)
{
global.Music = audio_play_sound(snd_Music, 0, true);
}
}
}
sonst
{
if audio_is_playing(global.Music)
{
audio_pause_klang(global.Musik);
}
}

Der obige Code pausiert/unpausiert eine Audiospur, je nachdem, ob das Audiosystem initialisiert und verfügbar ist oder nicht.
