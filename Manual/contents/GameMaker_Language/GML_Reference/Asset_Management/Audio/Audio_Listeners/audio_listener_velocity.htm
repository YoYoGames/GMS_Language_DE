audio_listener_velocity

Diese Funktion kann verwendet werden, um dem Hörer Doppler-Effekte zu geben und eine Audio-Bewegung basierend auf dem Vektor zu simulieren, der aus den gegebenen relativen x-, y- und z-Positionen aufgelöst wird. Wenn sich der Hörer selbst nicht bewegt oder die Bewegung Wenn Sie aber z. B. ein scrollendes Spiel machen, bei dem der Spieler eine konstante Bewegung von unten nach oben und die Feinde eine konstante Bewegung von oben nach unten haben, würden Sie die Hörer- und Emitter-Geschwindigkeiten (für Emitter würden Sie audio_emitter_velocity() verwenden) auf die entsprechenden Vektoren setzen, um den korrekten Dopplereffekt zu simulieren, wenn sie sich an der Spielerinstanz vorbei bewegen. Effekt zu simulieren, wenn sie sich an der Player-Instanz vorbeibewegen.

HINWEIS: wenn Sie mehrere Hörer haben, sollten Sie die Funktion audio_listener_set_velocity().

Das folgende Bild zeigt, wie dies für das oben angegebene Beispielspiel eingerichtet werden könnte:

Syntax:

audio_listener_velocity(vx, vy, vz);

Argument Beschreibung
vx Die x-Geschwindigkeitskomponente des Hörers (Voreinstellung 0).
vy Die y-Geschwindigkeitskomponente des Hörers (Standardwert 0).
vz Die z-Geschwindigkeitskomponente des Hörers (Voreinstellung 0).

Rückgabe:

N/A

Beispiel:

if speed > 0
{
audio_listener_velocity(abs(hspeed), abs(vspeed), 0);
}

Der obige Code prüft, ob die Geschwindigkeit der Player-Instanz über 0 liegt, und wenn dies der Fall ist, verwendet er die entsprechenden absoluten Komponenten hspeed und vspeed, um die Zuhörergeschwindigkeit einzustellen.
