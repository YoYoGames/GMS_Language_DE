audio_sync_group_debug

iese Funktion kann verwendet werden, um Debug-Informationen über eine beliebige Sync-Gruppe anzuzeigen (der Gruppenindex ist der Wert, der zurückgegeben wird, wenn die Gruppe mit der Funktion audio_create_sync_group()). Sie Sie können die Funktion mit einem Wert von -1 aufrufen, um das Overlay auszuschalten.

Wenn Sie die Audio-Sync-Gruppen debuggen, wird das Overlay angezeigt:

  • Welche Sync-Gruppe wird debuggt
  • Wie viele Kanäle sind darin enthalten
  • Wie oft die Sync-Gruppe fehlgeschlagen ist (das bedeutet, dass sie versucht hat, zu viel Arbeit zu erledigen)
  • Eine Liste der Kanäle
    • Rot bedeutet, dass dieser Kanal momentan keine Arbeit verrichtet
    • Gelb bedeutet, dass sich der Kanal im Anfangszustand befindet (noch nicht dekodiert oder abgespielt)
    • Grün bedeutet, dass die Spur gerade gesucht/dekodiert wird
    • Lila bedeutet, dass der Kanal keine Daten mehr hat
  • Ein Profiling-Balken, der anzeigt, welche Arbeit der Dekodier-Thread gerade verrichtet
    • blau bedeutet, dass eine Datei zum ersten Mal gelesen wird
    • rot bedeutet, dass der Thread eine Datei liest, um herauszufinden, von wo aus gelesen werden soll
    • grün bedeutet, dass der Thread Daten dekodiert
    • lila bedeutet, dass der Thread schläft, bis er arbeiten muss

Beachten Sie, dass Sie für das allgemeine Debugging von Audio auch die Funktion audio_debug().

HINWEIS: Diese Funktionalität ist für die HTML5-Zielplattform nicht verfügbar.

Syntax:

audio_sync_group_debug(group_index);

Argument Beschreibung
gruppen_index Der Gruppenindex zum Debuggen, oder -1, um das Debugging auszuschalten.

Rückgabe:

N/A

Beispiel:

if (debug_mode)
{
audio_sync_group_debug(sg);
}
sonst
{
audio_sync_group_debug(-1);
}

Der obige Code schaltet das Debug-Overlay für die Audio-Sync-Gruppen ein oder aus und zeigt die in der Variablen "sg" indizierte Sync-Gruppe an, je nachdem, ob das Spiel im Debug-Modus ausgeführt wird oder nicht.
