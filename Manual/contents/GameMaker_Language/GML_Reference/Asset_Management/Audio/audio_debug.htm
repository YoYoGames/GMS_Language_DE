audio_debug

Diese Funktion kann verwendet werden, um Debug-Informationen über das Audiosystem anzuzeigen, wobei true sie ein- und false sie ausschaltet. Wenn Sie Audio auf diese Weise debuggen, zeigt das Overlay die folgenden Informationen an:

  • Handle - Das OpenAL-Quellhandle (einzigartig für jeden abgespielten Sound)
  • Data Handle - Das OpenAL-Puffer-Handle (zeigt denselben Wert für mehrere Sounds mit demselben Source-Handle)
  • Verarbeitete Puffer - wie viele Puffer wurden abgespielt
  • Puffer in der Warteschlange - wie viele Puffer sich in der Warteschlange befinden (und noch auf die Wiedergabe warten)
  • Gain - Der Verstärkungswert für das Audio (0 - 65536)
  • Play Offset - die Sample-Position (aktuelle Abspielposition) im aktuellen Puffer
  • Balken für Puffer in der Warteschlange - ein kleines Kästchendiagramm der bearbeiteten/eingereihten Puffer mit dem Wiedergabeoffset (aktuelle Position) als blaue Linie gezeichnet

Für synchronisierte Audiogruppen haben Sie außerdem die Funktion audio_sync_group_debug().

HINWEIS: Diese Funktion funktioniert nicht auf der HTML5-Zielplattform.

Syntax:

audio_debug(enable);

Argument Beschreibung
aktivieren. Aktivieren (auf true gesetzt) oder deaktivieren(false) Sie das Audio-Debugging.

Rückgabe:

N/A

Beispiel:

if (debug_mode)
{
audio_debug(true);
}
sonst
{
audio_debug(false);
}

Der obige Code schaltet das Audio-Debug-Overlay ein oder aus, je nachdem, ob das Spiel im Debug-Modus läuft oder nicht.
