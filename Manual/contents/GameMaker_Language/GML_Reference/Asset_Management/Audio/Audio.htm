Audio

GameMaker Studio 2 verfügt über eine vollständige Audio-Engine, die auf den Soundformaten *.ogg, *.mp3 und *.wav basiert. Klänge dieser Typen, die der IDE hinzugefügt werden, können dann mit den unten gezeigten grundlegenden Audiofunktionen in Ihrem Spiel verwendet werden. Für Dinge, die komplexer sind als grundlegende Soundeffekte oder das Abspielen eines einzelnen Musikstücks, können Sie sich an die erweiterten Audiofunktionen wenden, mit denen Sie die Art und Weise, wie ein Sound abgespielt wird, verändern können. Es gibt auch eine Auswahl an spezielleren Funktionen für das Streaming Audio, das Positionieren von Audio - für 3D-Sound - und das Gruppieren von Audio.

WICHTIG! Wenn Sie Audio auf dem HTML5-Ziel verwenden, sollten Sie sich bewusst sein, dass nicht alle Browser Web Audio unterstützen und daher möglicherweise keinen Ton für Ihr Projekt abspielen, wenn es ausgeführt wird. Unter dem folgenden Link können Sie sich einen Überblick über die Web Audio-Unterstützung verschaffen: Kann ich Web Audio verwenden?

Die folgenden Funktionen dienen dem möglichst unkomplizierten Umgang mit Audiosignalen. Sie vermeiden die Verwendung von Emittern und ermöglichen es dem Benutzer, auf einfache Weise Klänge zu erzeugen und Musik abzuspielen, da diese Klänge immer an der Position des Hörers erzeugt werden (weitere Einzelheiten zum Hörer finden Sie im unten verlinkten Abschnitt) und daher im Allgemeinen nicht von Änderungen am Hörer beeinflusst werden:

  • audio_exists
  • audio_get_name
  • audio_get_type
  • audio_play_sound
  • audio_play_sound_at
  • audio_pause_ton
  • audio_pause_alle
  • audio_resume_sound
  • audio_resume_all
  • audio_stop_ton
  • audio_stop_all
  • audio_ist_abspielend
  • audio_ist_pausiert
  • audio_erzeugen_strom
  • audio_destroy_stream

Die folgenden Funktionen dienen dazu, mehr Kontrolle darüber zu erhalten, wie die Audio-Engine arbeitet und wie die über sie abgespielten Klänge vom Hörer "gehört" werden. Daher wird empfohlen, dass Sie über gute Kenntnisse darüber verfügen, wie der Rest der GameMaker Studio 2 Audio-Engine haben, bevor Sie eine der folgenden Funktionen verwenden:

  • audio_sound_set_track_position
  • audio_sound_get_track_position
  • audio_sound_set_listener_mask
  • audio_sound_get_listener_mask
  • audio_sound_length
  • audio_sound_pitch
  • audio_klang_get_pitch
  • audio_klang_ist_abspielbar
  • audio_falloff_set_model
  • audio_sound_gain
  • audio_sound_get_gain
  • audio_master_gain
  • audio_set_master_gain
  • audio_get_master_gain
  • audio_channel_num
  • audio_debug

Weitere erweiterte Funktionen, wie das Einstellen von Hörern, das Aufnehmen von Audio oder das zeitliche Synchronisieren mehrerer Audiospuren, finden Sie in den unten aufgeführten Unterabschnitten:

Web Audio

Beim Erstellen von Spielen für die HTML5-Zielplattform benötigt die Audio-Engine Web-Audio-Unterstützung, was wiederum bedeutet, dass Ihr Audio manchmal nicht abgespielt wird, wenn oder wie Sie es erwarten. Dies liegt daran, dass der Web Audio-Kontext möglicherweise nicht ausgeführt wird oder nicht mehr läuft, wenn Ihr Spiel abgespielt wird. Die Ursachen hierfür variieren stark zwischen den Browsern und sogar zwischen verschiedenen Versionen desselben Browsers, daher ist es sehr wichtig zu erkennen, wann sich der Status des Web-Audio-Kontexts ändert, z. B. um zu erkennen, wenn sich der Kontext ändert, und Audioschleifen wie Hintergrundmusik zu pausieren/starten.

Um bei diesem Problem zu helfen, verfügt GameMaker Studio 2 über zwei verschiedene Möglichkeiten, die Änderung des Web-Audio-Kontextstatus zu erkennen, entweder mit der folgenden Funktion:

  • audio_system_ist_verfügbar

Sie können auch das asynchrone Systemereignis verwenden, das immer dann ausgelöst wird, wenn sich der Status von Web Audio ändert. In diesem Ereignis erhalten Sie die eingebaute async_load DS-Map mit dem Schlüssel " event_type" gefüllt, der wiederum die Zeichenkette " audio_system_status" enthält, wenn es sich um ein Audio-Ereignis handelt. Wenn dieser Schlüssel vorhanden ist, gibt es auch einen weiterenSchlüssel"status", der entweder "verfügbar" oder"nicht verfügbar" ist. Beachten Sie, dass dieses Ereignis auf allen Plattformen ausgelöst wird, aber auf allen außer HTML5 wird es nur einmal beim Spielstart ausgelöst, wenn die Audio-Engine zum ersten Mal initialisiert wird.
