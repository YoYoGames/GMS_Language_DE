audio_play_sound_on

Mit dieser Funktion können Sie eine beliebige Sound-Ressource über einen Emitter abspielen, wobei jede Änderung der Emitter-Verstärkung, -Position, -Tonhöhe oder -Anschlagstärke die Art und Weise beeinflusst, wie der Benutzer den endgültig abgespielten Sound hört. Sie geben den zu verwendenden Emitter-Index, den Sound-Index des abzuspielenden Klangs an und legen dann fest, ob der Klang in einer Schleife abgespielt werden soll oder nicht. Schließlich können Sie eine Priorität zuweisen, die dann verwendet wird, um zu bestimmen, wie Klänge behandelt werden, wenn die Anzahl der abgespielten Klänge über dem von der Funktion festgelegten Limit liegt audio_channel_num(). Klänge mit niedrigerer Priorität werden zugunsten von Klängen mit höherer Priorität gestoppt, und der Prioritätswert kann eine beliebige reale Zahl sein (der tatsächliche Wert ist willkürlich und kann von 0 bis 1 oder 0 bis 100 reichen, da GameMaker Studio 2 sie gleich priorisiert). Beachten Sie, dass die Priorität umso höher ist, je höher die Zahl ist, d. h. ein Sound mit der Priorität 100 wird gegenüber einem Sound mit Priorität 1.

Diese Funktion gibt auch eine eindeutige Indexnummer für den abgespielten Ton zurück, die dann in einer Variablen gespeichert werden kann, so dass Sie ihn dann mit den entsprechenden Funktionen anhalten oder stoppen können. Das bedeutet, dass Sie, wenn Sie mehrere Instanzen desselben Wenn Sie also mehrere Instanzen desselben Tons gleichzeitig abspielen, können Sie mit den Audiofunktionen nur eine Instanz dieses Tons bearbeiten.

Syntax:

audio_play_sound_on(emitter, sound, loop, priority);

Argument Beschreibung
Emitter Der Index des zu verwendenden Emitters.
Ton Der Index des zu verwendenden Tons.
Schleife Kennzeichnet den Ton als loopend oder nicht.
Priorität Stellen Sie die Kanalpriorität für den Ton ein.

Rückgabe:

Index

Beispiel:

if global.SFX
{
audio_play_sound_on(s_emit, snd_Explode, false, 1);
}

Der obige Code prüft die globale Variable "SFX" und wenn er true zurückgibt, wird der in der Variablen "snd_Explode" indizierte Sound über den in der Variablen "s_emit" indizierten Emitter abgespielt mit niedriger Priorität und ohne Schleifenbildung abgespielt.
