audio_play_sound

Mit dieser Funktion können Sie jede Sound-Ressource in Ihrem Spiel abspielen. Sie geben den Sound-Index an und weisen ihm eine Priorität zu, die dann verwendet wird, um zu bestimmen, wie mit den Sounds umgegangen wird, wenn die Anzahl der abgespielten Sounds über dem von der Funktion festgelegten Limit liegt audio_channel_num(). Klänge mit niedrigerer Priorität werden zugunsten von Klängen mit höherer Priorität gestoppt, und der Prioritätswert kann eine beliebige reale Zahl sein (der tatsächliche Wert ist willkürlich und kann zwischen 0 und 1 oder 0 und 100 liegen, da GameMaker Studio 2 sie gleich priorisiert). Beachten Sie, dass die Priorität umso höher ist, je höher die Zahl ist, d. h. ein Sound mit der Priorität 100 wird gegenüber einem Sound mit Priorität 1. Das letzte Argument dient dazu, den Sound in einer Schleife abzuspielen. Wenn Sie es auf true setzen, wird der Sound in einer Schleife abgespielt, bis er gestoppt wird, und wenn Sie es auf false setzen, wird der Sound nur einmal abgespielt nur.

Diese Funktion gibt auch eine eindeutige Indexnummer für den abgespielten Ton zurück, die dann in einer Variablen gespeichert werden kann, so dass Sie ihn dann mit den entsprechenden Funktionen anhalten oder stoppen können. Das bedeutet, dass Sie, wenn Sie mehrere Instanzen desselben Wenn Sie also mehrere Instanzen desselben Tons gleichzeitig abspielen, können Sie mit den Audiofunktionen nur eine Instanz dieses Tons bearbeiten.

Syntax:

audio_play_sound(index, priority, loop);

Argument Beschreibung
index Der Index des abzuspielenden Tons.
Priorität Stellen Sie die Kanalpriorität für den Ton ein.
Schleife Legt fest, ob der Ton eine Schleife bilden soll oder nicht.

Rückgabe:

Index (Sound-ID)

Beispiel:

if health <= 0
{
lives -= 1;
audio_play_sound(snd_PlayerDead, 10, false);
}

Der obige Code prüft die globale Variable "health" und wenn sie kleiner oder gleich 0 ist, wird 1 aus der globalen Variable "lives" entfernt und ein Ton abgespielt.
