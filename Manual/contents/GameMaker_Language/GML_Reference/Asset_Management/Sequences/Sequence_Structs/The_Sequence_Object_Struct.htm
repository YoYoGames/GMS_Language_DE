Das Sequenz-Objekt Struktur

Ein Sequenz-Objekt ist der Name für die Struktur, die vom Sequenz-Asset abgerufen wird, wie im Asset-Browser für Ihr Spiel definiert. Diese Struktur hat die folgenden Eigenschaften, die geändert werden können:

  • name: Dies ist der Name der Sequenz als Zeichenkette und Sie können diesen Wert nach Bedarf abrufen oder setzen. Beachten Sie, dass Sequenzen, die mit der Funktion sequence_create() erstellt werden, keinen Namen haben und dieser einfach eine leere Zeichenkette "" sein wird.
  • loopmode: Dies ist der Wiedergabemodus des Sequenz-Objekts und kann ausgelesen oder gesetzt werden. Er gibt die folgenden Konstanten zurück (oder kann auf sie gesetzt werden):
Konstant Beschreibung Wert
seqplay_oneshot Die Sequenz wird einmal abgespielt und stoppt, wenn sie beendet ist. 0
seqplay_loop Die Sequenz wird in einer Schleife abgespielt, wobei der Abspielkopf an den Anfang zurückkehrt, wenn er das Ende des Wiedergabebereichs erreicht. 1
seqplay_pingpong Die Sequenz wird in einer Schleife abgespielt, wobei der Abspielkopf die Richtung umkehrt, wenn er das Ende des Wiedergabebereichs erreicht. 2

  • playbackSpeed: Gibt die Wiedergabegeschwindigkeit der Sequenz an, die je nach playbackSpeedType (siehe unten) entweder als Bilder pro Sekunde oder als Bilder pro Spiel-Frame interpretiert wird. Sie können diesen Wert abrufen oder setzen.
  • playbackSpeedType: Dies gibt an, wie die Wiedergabegeschwindigkeit interpretiert werden soll, und Sie können diesen Wert erhalten oder setzen. Sie können eine der folgenden Konstanten verwenden, um den Wert zu setzen (oder eine dieser Konstanten wird zurückgegeben, wenn die der Eigenschaft):
Konstant Beschreibung Wert
spritespeed_framespersecond Gibt an, dass playbackSpeed als Bilder pro Sekunde interpretiert werden soll 0
spritespeed_framespergameframe Gibt an, dass playbackSpeed als Frames-per-Game-Frame interpretiert werden soll. 1

  • Länge: Die Länge der Sequenz in Frames. Sie können diesen Wert abrufen oder festlegen, aber beachten Sie, dass eine Verkürzung der Sequenz zu Problemen führen kann, wenn der Abspielkopf einer Sequenzinstanz, die diese Sequenz referenziert, über die neue Länge hinaus eingestellt ist.
  • Lautstärke: Dies ist ein skalarer Wert von 0 bis 1, der verwendet wird, um die Lautstärke aller Audiospuren in der Sequenz zu skalieren. Sie können diesen Wert abrufen oder setzen und er ändert die globale Audioausgabe für alle Spuren - wenn Sie z. B. eine Audiospur mit einer Lautstärke von 0,8 haben und dann die Eigenschaft " Sequenzlautstärke" auf 0,5 setzen, hat die Audiospur eine Endlautstärke von 0,4.
  • xUrsprung: Dies ist der Ursprung der Sequenz entlang der X-Achse. Sie können diesen Wert abrufen oder setzen.
  • yUrsprung: Dies ist der Ursprung der Sequenz entlang der Y-Achse. Sie können diesen Wert abrufen oder setzen.
  • messageEventKeyframes: Ermöglicht den Zugriff auf die Message-Event-Keyframes für die Sequenz. Sie können diese Nachrichtenereignisse abrufen oder setzen, und wenn Sie diese Eigenschaft abrufen, wird ein Array von Keyframe-Strukturen zurückgegeben, und zum Setzen der Eigenschaft sollten Sie sollten Sie ein Array von Keyframe-Strukturen bereitstellen. Weitere Informationen finden Sie auf der Seite über Sequenzereignisse und Momente.
  • momentKeyframes: Dies ermöglicht den Zugriff auf die Moment-Event-Keyframes für die Sequenz. Sie können diese Moment-Ereignisse abrufen oder setzen, und wenn Sie diese Eigenschaft abrufen, wird ein Array von Keyframe-Strukturen zurückgegeben, und zum Setzen der Eigenschaft sollten Sie ein Array von Keyframe-Strukturen. Weitere Informationen finden Sie auf der Seite über Sequenzereignisse und Momente.
  • Spuren: Dies ermöglicht den Zugriff auf die Liste der Asset-Spuren auf der obersten Ebene der Sequenz. Sie können diese Eigenschaft abrufen oder setzen, und wenn Sie diese Eigenschaft abrufen, wird ein Array von Track-Strukturen zurückgegeben, und zum Setzen der Eigenschaft sollten Sie ein Array von Track-Strukturen liefern. Weitere Informationen finden Sie im Abschnitt über Spur-Strukturen.

Beachten Sie, dass Sie, wenn Sie auf die Eigenschaften einer Sequenz zugreifen wollen, die im Asset-Browser erstellt wurde, zuerst die Funktion sequence_get() für den Asset-Index aufrufen, um die Sequenzobjekt-Struktur abzurufen. Beachten Sie auch, dass alle Änderungen, die an dieser Sequenz-Struktur vorgenommen werden, bedeuten, dass alle weiteren Instanzen dieses Sequenz-Assets ebenfalls diese Änderungen haben werden, und sie werden beibehalten, solange das Spiel läuft, auch wenn Sie die Funktion game_restart() aufrufen.
