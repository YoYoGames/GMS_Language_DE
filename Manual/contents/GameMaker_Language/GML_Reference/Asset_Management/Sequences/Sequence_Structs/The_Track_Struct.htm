Die Schiene Struktur

Die Track-Struktur kann entweder fürAsset-Tracks der obersten Ebene oder fürParameter-Tracks der unteren Ebene verwendet werden, und das Verhalten des Tracks wird durch zwei Dinge definiert, seinen Namen und seinen Typ . Track-Strukturen werden mit der Funktion sequence_track_new() erstellt und können von Sequenz-Assets über die Eigenschaft activeTracks aus der Sequence Instance Struct oder die Eigenschaft tracks aus der Sequence Object Struct abgerufen werden.

Die für die Trackstruktur verfügbaren Eigenschaften sind:

  • Typ: Die folgenden Konstanten können alle verwendet werden, um den Typ der Spur zu definieren, die in der Sequenz erstellt wird (diese Konstanten werden auch beim Erzeugen von Keyframes und Keyframe-Daten verwendet, die in den folgenden Abschnitten erklärt werden):
Konstant Beschreibung Wert
seqtracktype_graphic Dies ist eine Grafik- (Sprite-) Asset-Spur. 1
seqtracktype_audio Dies ist eine Audio-Asset-Spur. 2
seqtracktype_instance Dies ist eine Instance-Asset-Spur. 14
seqtracktype_sequence Dies ist ein Sequenz-Asset-Track. 7
seqtracktype_clipmask Dies ist eine Clip-Masken-Gruppen-Asset-Spur. 8
seqtracktype_clipmask_mask Dies ist eine Clip-Masken-Sprite-Asset-Spur, die zum Erzeugen der Clip-Maske verwendet wird. 9
seqtracktype_clipmask_subject Dies ist eine Clip-Masken-Sprite-Asset-Spur, die maskiert wird. 10
seqtracktype_group Dieser Gruppenordner Asset-Track. 11
seqtracktype_colour Dies ist eine Farbdaten-Parameterspur. 4
seqtracktype_real Dies ist eine reale Zahlenwert-Parameterspur. 3
seqtracktype_message Dies ist eine Spur für Broadcast-Nachrichten. 15
seqtracktype_moment Dies ist ein Ereignis-/Moment-Track. 16
seqtracktype_bool Derzeit nicht verwendet. 5
seqtracktype_string Derzeit nicht verwendet. 6
seqtracktype_spriteframes Derzeit nicht verwendet. 13
seqtracktype_empty Derzeit nicht verwendet. 12

  • name: Wenn Sie eine Asset-Spur erstellen, kann der Name, den Sie hier angeben, eine beliebige Zeichenkette sein, die Sie zur Identifizierung der Spur benötigen. Für Parameterspuren müssen Sie jedoch bestimmte Zeichenketten angeben, um GameMaker Studio 2 mitzuteilen, welche Art von Parameterspur Sie erstellen (diese Zeichenketten gelten nur für die Parameterspur-Typen seqtracktype_real und seqtracktype_colour ):
    • "position" - wenn dies bei einer Spur vom Typ seqtracktype_real gesetzt wird, zeigt dies an, dass der Inhalt dieser Spur als Positionsdaten verwendet werden soll.
    • "scale"- wenn dies auf einer Spur vom Typ seqtracktype_real gesetzt wird, zeigt dies an, dass der Inhalt dieser Spur als Skalierungsdaten verwendet werden soll.
    • "rotation" - wenn auf einer Spur vom Typ seqtracktype_real gesetzt, zeigt dies an, dass der Inhalt dieser Spur als Rotationsdaten verwendet werden soll.
    • "blend_multiply"- wenn dies auf eine Spur vom Typ seqtracktype_colour gesetzt wird, zeigt dies an, dass der Inhalt dieser Spur als Farbmultiplikationsdaten verwendet werden soll. Die Keyframe-Daten für eine solche Spur sollten ein Array aus vier Werten mit dem Format [A, R, G, B] sein. Beachten Sie, dass die Werte für jede Komponente als Werte zwischen 0 und 1 ausgedrückt werden, wobei 0 dem HEX-Wert #00 und 1 dem HEX-Wert #FF entspricht (0 - 255 wie in der Farbauswahl für Bildüberblendungsspuren im Picker für Bildüberblendungsspuren im Sequenzeditor).
    • "image_speed"- wenn auf einer Spur vom Typ seqtracktype_real gesetzt, zeigt dies an, dass der Inhalt dieser Spur als Wert für die Bildgeschwindigkeit verwendet werden soll.
    • "image_index"- wenn auf einer Spur vom Typ seqtracktype_real gesetzt, zeigt dies an, dass der Inhalt dieser Spur als Bildindexwert verwendet werden soll.
    • "image_angle"- wenn auf einer Spur vom Typ seqtracktype_real gesetzt, zeigt dies an, dass der Inhalt dieser Spur als Bildwinkelwert verwendet werden soll.
    • "image_blend"- wenn auf einer Spur vom Typ seqtracktype_colour gesetzt, zeigt dies an, dass der Inhalt dieser Spur als Bildüberblendungswert verwendet werden soll. Die Keyframe-Daten für eine solche Spur sollten ein Array aus vier Werten mit dem Format [A, R, G, B] sein. Beachten Sie, dass die Werte für jede Komponente als Werte zwischen 0 und 1 ausgedrückt werden, wobei 0 dem HEX-Wert #00 und 1 dem HEX-Wert #FF entspricht (0 - 255 wie im Farbwähler für Farbmultiplexspuren im Sequenzeditor).
  • Spuren: Diese Eigenschaft ermöglicht den Zugriff auf die Liste der Spuren, die Kinder dieser Spur sind. Beim Abrufen dieser Eigenschaft wird ein Array von Track-Strukturen zurückgegeben, und beim Setzen dieser Eigenschaft sollte ein Array von Track-Strukturen angegeben werden.
  • sichtbar: Dies gibt an, ob diese Spur sichtbar ist (der Wert ist true) oder nicht (der Wert ist false). Sie können diesen Wert abrufen oder setzen und wenn eine Spur nicht sichtbar ist, wird auch keine ihrer untergeordneten Spuren gezeichnet.
  • Keyframes: Diese Eigenschaft ermöglicht den Zugriff auf die Liste der Keyframe-Strukturen für die Spur. Beim Abrufen dieser Eigenschaft wird ein Array von Keyframe-Strukturen zurückgegeben, und beim Setzen dieser Eigenschaft sollte ein Array von Keyframe Strukturen angegeben werden.
