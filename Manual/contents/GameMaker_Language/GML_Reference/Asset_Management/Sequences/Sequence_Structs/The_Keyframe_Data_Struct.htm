Die Keyframe-Datenstruktur

Wenn Sie eine Keyframe-Struktur für eine Spur definieren, müssen Sie auch die Daten definieren, die mit ihr verbunden werden sollen. Diese setzen sich aus verschiedenen Kanälen zusammen, und jedem Kanal kann ein bestimmter Jeder Kanal kann mit einem bestimmten Datentyp versehen werden, je nachdem, für welche Art von Spur Sie die Keyframe-Daten festlegen. Diese Eigenschaften sind:

  • Kanal: Dies ist der Kanal, auf den die Keyframe-Daten angewendet werden sollen. Es handelt sich um einen positiven ganzzahligen Wert, der bei 0 beginnt. Es ist erwähnenswert, dass Sie beim Erstellen von Parameterspuren für Keyframes "Position" oder "Skalierung", dann müssen Sie ganz bestimmte Kanalwerte verwenden. Diese sind: Kanal 0 ist die X-Position oder die X-Skala, Kanal 1 ist die Y-Position oder die Y-Skala.
  • spriteIndex: Der Index des Sprite-Assets, das für die Spur verwendet werden soll. Diese Eigenschaft ist nur für Spuren des Typs seqtracktype_graphic verfügbar und Sie können sie abrufen oder festlegen.
  • soundIndex: Der Index des Audio-Assets, das für die Spur verwendet werden soll. Diese Eigenschaft ist nur für Spuren des Typs seqtracktype_audio verfügbar und Sie können sie abrufen oder festlegen.
  • playbackMode: Der Wiedergabemodus für den Ton. Diese Eigenschaft ist nur für Tracks vom Typ seqtracktype_audio verfügbar und Sie können sie abrufen oder setzen. Die Eigenschaft sollte eine der folgenden Konstanten sein:
Konstant Beschreibung Wert
seqaudiokey_loop Der Ton wird in einer Schleife abgespielt. 0
seqaudiokey_oneshot Der Ton wird nur einmal abgespielt und stoppt dann. 1

  • Kurve: Diese Eigenschaft erfordert eine Animationskurven-Struktur (siehe hier für weitere Informationen) und ist nur für Spuren vom Typ seqtracktype_real verfügbar. Wenn keine Kurvenstruktur verwendet wird, ist der Wert für diese Eigenschaft -1.
  • Wert: Diese Eigenschaft ist einfach ein Wert, der mit dem Keyframe-Datenkanal verknüpft ist, und ist nur für Spuren des Typs seqtracktype_real verfügbar, wenn keine curve struct geliefert wird.
  • colour: Diese Eigenschaft gibt ein Array für den Farbwert des Keyframes mit dem Format [A, R, G, B] zurück (bzw. erfordert, wenn sie gesetzt ist). Dies ist nur für Spuren des Typs seqtracktype_colour verfügbar. Beachten Sie, dass die Werte für jede Komponente als zwischen 0 und 1 ausgedrückt werden sollten, wobei 0 dem HEX-Wert #00 und 1 dem HEX-Wert #FF entspricht (0 - 255, wie in der Farbauswahl für Farbspuren im Sequences Editor angezeigt).
  • Sequenz: Diese Eigenschaft gibt eine Sequenz-Objektstruktur zurück (oder erfordert, wenn sie gesetzt ist) und ist nur für Spuren vom Typ seqtracktype_sequence verfügbar.
  • objectIndex: Diese Eigenschaft liefert (bzw. erfordert, wenn sie gesetzt ist) einen Objektindex und ist nur für Spuren vom Typ seqtracktype_instance verfügbar.
  • Ereignisse: Diese Eigenschaft ermöglicht den Zugriff auf die Ereignisse und Broadcast-Nachrichten, die mit der Keyframe-Datenstruktur verbunden sind. Sie können diese Eigenschaft abrufen oder setzen, und wenn Sie sie abrufen, wird ein Array von Strings zurückgegeben, und beim Setzen sollte ein Array von Strings angegeben werden. Weitere Informationen zu Ereignissen finden Sie im Abschnitt Sequenzereignisse und Momente. Diese Eigenschaft ist nur für Tracks vom Typ seqtracktype_message verfügbar.
  • Ereignis: Diese Eigenschaft gibt die Methode zurück (oder kann auf sie gesetzt werden), die mit der Keyframe-Datenstruktur verbunden ist. Wenn keine Methode angegeben wurde oder Sie die Methode entfernen möchten, sollte die Eigenschaft -1 sein. Wenn keine Methode angegeben wurde oder Sie die Methode entfernen möchten, sollte die Eigenschaft -1 sein. Diese Eigenschaft ist nur für Spuren vom Typ seqtracktype_moment verfügbar.
