in_reihenfolge

Dies ist eine eingebaute Variable, die Teil der Instanzvariablen ist, die für jede Objektinstanz in Ihrem Spiel erstellt werden. Wenn die Instanz von einer Sequenz gesteuert wird, gibt diese Variable den Wert true zurück, andernfalls gibt sie den Wert false zurück. Dies ist eine schreibgeschützte Variable und kann nicht geändert werden.

Beachten Sie, dass diese Variable falsch wird, nachdem die steuernde Sequenz beendet wurde, auch wenn ihr Sequenzelement noch existiert, und wahr wird, wenn dieses Sequenzelement erneut abgespielt wird.

Sie können diese Variable in Ihren spieler- (oder CPU-) gesteuerten Objekten verwenden, um sicherzustellen, dass sie nur dann per Code verschoben werden, wenn sie sich nicht in einer Sequenz befinden. Dies kann sich als nützlich für Zwischensequenzen erweisen, wenn Sie die gleichen Instanzen in einer Sequenz verwenden müssen (mit die sequence_instance_override_object() Funktion) und Sie müssen sicherstellen, dass sie nicht durch ihren üblichen Code bewegt werden können, während die Cutscene aktiv ist.

Syntax:

in_reihenfolge

Rückgabe:

Boolesche

Beispiel:

if (!in_sequence)
{
x += move_x;
y += move_y;
}

Der obige Code prüft die Variable in_sequence, und wenn sie false ist (was bedeutet, dass die Instanz nicht von einer Sequenz gesteuert wird), dann fügt er move_x und move_y zur Position der Instanz Position hinzu, um sicherzustellen, dass sie sich nur bewegt, wenn sie sich nicht in einer Sequenz befindet.
