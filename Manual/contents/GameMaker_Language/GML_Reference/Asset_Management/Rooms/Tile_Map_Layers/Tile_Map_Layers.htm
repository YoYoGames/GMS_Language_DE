Kachelkartenebenen

Der GameMaker Studio 2-Raumeditor ermöglicht es Ihnen, mithilfe von Ebenen Kacheln in jeden beliebigen Raum einzufügen. Wie der Name schon sagt, ist eine Kachelebene einfach eine Sammlung von Kachelsätzen, die sich alle in der gleichen Tiefe im Raum befinden. Wie der Name schon sagt, ist eine Kachelplan-Ebene einfach eine Sammlung von Kachelsätzen, die sich alle in der gleichen Tiefe innerhalb des Raums befinden, und durch das Stapeln von Ebenen können Sie einige Dinge über oder unter andere zeichnen. Sie können auch bestimmte Aspekte der Ebenen vom Code aus steuern, indem Sie Dinge hinzufügen oder entfernen oder bestimmte Eigenschaften der Ebene oder ihrer Inhalte zur Laufzeit ändern. Beachten Sie, dass Sie im Raumeditor auf eine Kachelkarte pro Ebene beschränkt sind (die Kachel-Ebene), aber wenn Sie mit ihnen im Code können Sie einer einzelnen Ebene mehrere Kachelmaps zuweisen, wobei jede Map ihre eigene eindeutige ID und ihre eigenen Eigenschaften hat, so:

  • Das Kachelset ist die Ressource, die Sie zum Erstellen von Kachelschichten und Kachelkarten verwenden
  • Die Kachelebene ist das, was Sie im Raum-Editor erstellen, um Kachelsätze hinzuzufügen
  • Als Kachel-Map wird die Sammlung von Kacheln bezeichnet, die einer Ebene entweder im Raumeditor oder durch Code als einzelnes Element auf dieser Ebene hinzugefügt werden

Neben den oben aufgeführten Funktionen zum Setzen von Tilemaps gibt es noch eine Reihe von Funktionen, mit denen die Daten einzelner Kacheln manipuliert werden können. Ein "Blob" von Kacheldaten besteht aus dem Kachelindex und ein paar zusätzlichen Bits, die anzeigen, ob die Kachel gedreht, gespiegelt oder gewendet wurde. Sie haben auch die Möglichkeit, Ihre eigene Bitmaske zu den Kacheldaten hinzuzufügen, obwohl dies nur für fortgeschrittene Benutzer geeignet ist, da Sie damit Bits ausblenden, die für den Kachelindex reserviert sind - im Wesentlichen gibt es 19 Bits, die für Ihre Kachelindizes reserviert sind, aber wenn Sie nur 8 davon verwenden, bleiben 11 Bits übrig, die Sie zum Erstellen einer eigenen Maske verwenden können. Diese Maske kann dann im Code verwendet und überprüft werden, um benutzerdefinierte Kachelkollisionen oder ähnliches zu erstellen. Normalerweise brauchen Sie jedoch keine eigenen Kacheldaten-Blobs zu erstellen, sondern verwenden stattdessen die Funktion tilemap_get() verwenden, um die Kacheldaten zu erhalten, sie dann mit den folgenden Funktionen zu bearbeiten und dann setzen Sie sie wieder mit tilemap_set().
