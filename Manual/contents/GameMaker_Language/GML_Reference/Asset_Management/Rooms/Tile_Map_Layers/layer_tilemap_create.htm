layer_tilemap_create

Mit dieser Funktion können Sie einem Layer eine Kachel-Set-Ressource zuweisen, die als Kachel-Map in Ihrem Projekt verwendet werden soll. Sie übergeben die Layer-ID (die Sie beim Erstellen des Layers mit layer_create()) oder den Ebenennamen (als String - dies wirkt sich auf die Leistung aus) und dann eine Anfangsposition (x, y), um die Kachelübersicht dem Raum hinzuzufügen, die zu verwendende Kachelsatz-Ressource und dann die Breite und Höhe der Kachelübersicht in Zellen (z. B.: eine Breite von 20 und einer Höhe von 10 wird eine Kachelkarte mit 200 Zellen erstellt, die 20 Zellen breit und 10 Zellen hoch ist), wobei die Größe der Zellen selbst durch das gewählte Kachelset definiert wird. Es ist erwähnenswert, dass Sie keine Kacheln an negativen Positionen Alle Kacheln müssen also innerhalb der Zellbereiche 0 bis Breite - 1 und 0 bis Höhe - 1 platziert werden.

Syntax:

layer_tilemap_create(layer_id, x, y, Kachelsatz, Breite, Höhe)

Argument Beschreibung
layer_id Der eindeutige ID-Wert der anzuvisierenden Ebene (oder der Ebenenname als String)
x Die x-Position der Kachelkarte im Raum
y Die y-Position der Kachelkarte im Raum
Fliesensatz Der zu verwendende Kachelsatzindex
Breite Die Breite der Kachelkarte (in Zellen)
Höhe Die Höhe der Kachelkarte (in Zellen)

Rückgabe:

Kachelplan-Element-ID (Real)

Beispiel:

global.back_layer = layer_create(10000);
global.back_tiles = layer_tilemap_create(global.back_layer, 0, 0, tmap_Trees, 16, 32);

Der obige Code erstellt eine neue Ebene und fügt dann ein neues Kachelkartenelement hinzu, wobei die Position der Kachelkarte auf (0,0) sowie das zu verwendende Kachelset und die Breite und Höhe der Kachelkarte festgelegt werden.
