layer_tilemap_exists

Mit dieser Funktion können Sie prüfen, ob auf einer bestimmten Ebene ein Tilemap-Element existiert. Sie übergeben die ID der Ebene (die Sie beim Erstellen der Ebene mit layer_create()) oder den Ebenennamen (als String - dies hat Auswirkungen auf die Performance) und die ID des Kachelzuordnungselements (die Sie erhalten, wenn Sie ein Kachelzuordnungselement mit layer_tilemap_create() oder wenn Sie die Funktion layer_tilemap_get_id()) und die Funktion gibt entweder true zurück, wenn das Element existiert, oder false, wenn es nicht existiert.

HINWEIS: Diese Funktion arbeitet im Rahmen des aktuellen Zielraums - standardmäßig der Raum, in dem die Funktion aufgerufen wird - der mit der Funktion layer_set_target_room().

Syntax:

layer_tilemap_exists(layer_id, tilemap_element_id)

Argument Beschreibung
layer_id Der eindeutige ID-Wert der anzuvisierenden Ebene (oder der Ebenenname als String)
tilemap_element_id Der eindeutige ID-Wert des zu prüfenden Tile-Map-Elements

Rückgabe:

Boolesche

Beispiel:

var lay_id = layer_get_id("tilemap_trees");
if layer_tilemap_exists(lay_id, global.Treestilemap)
{
layer_tilemap_destroy(lay_id, global.Treestilemap);
}

Der obige Code prüft die Ebene "tilemap_trees", um zu sehen, ob das angegebene Kachelkartenelement existiert, und wenn ja, dann wird es zerstört (aber nicht die Ebene).
