Räume

Räume werden im Raumeditor von GameMaker Studio 2erstellt und sind der Ort, an dem sich alles in Ihrem Spiel abspielt. Sie müssen mindestens einen Raum in jedem Spiel haben, damit es läuft, aber Sie können Je nach Art des Spiels und der Struktur Ihres Projekts benötigen Sie jedoch möglicherweise viel mehr Räume. In diesem Abschnitt finden Sie alle allgemeinen GML-Funktionen und Variablen, die sich auf Räume beziehen und mit denen Sie Informationen über sie erhalten sowie bestimmte Eigenschaften in ihnen einstellen können ihnen.

Die folgenden globalen Variablen und Funktionen können verwendet werden, um grundlegende Informationen über einen bestimmten Raum zu erhalten:

  • Zimmer_erste
  • raum_last
  • raum_next
  • raum_vorher
  • Zimmer
  • Raum_Geschwindigkeit
  • Raum_Höhe
  • raum_breite
  • raum_persistent

Sie können auch die folgenden Funktionen verwenden, um den Namen des aktuellen Raums (wie im Asset-Browser definiert) abzurufen sowie zwischen Räumen zu wechseln:

  • raum_holen_name
  • Raum_Springen
  • raum_zur_nächsten
  • raum_zur_vorherigen
  • raum_neustart

Als Nächstes haben wir eine Reihe von Funktionen, mit denen Sie Räume und deren Inhalt ändern oder sogar neue Räume erstellen und zu ihnen hinzufügen können. Es ist zu beachten, dass diese Funktionen niemals aus dem Raum heraus ausgeführt werden sollten , den Sie ändern möchten, sondern zuerst von einer Instanz in einem anderen Raum aus ausgeführt werden. Beachten Sie auch, dass, wenn Sie auf diese Weise ein Raum-Asset ändern, die Änderung permanent ist und für die gesamte Dauer des Spiels beibehalten wird, auch wenn Sie die Funktion room_restart() aufrufen aufrufen (das Schließen und Neustarten des Spiels setzt die Raumeigenschaften jedoch zurück).

  • raum_vorhanden
  • Raum_hinzufügen
  • raum_duplizieren
  • raum_zuweisen
  • room_instance_add
  • room_instance_clear
  • raum_setzen_höhe
  • room_set_width
  • room_set_persistent
  • room_set_view_enabled
  • room_set_viewport
  • room_get_viewport
  • room_set_camera
  • room_get_camera

Es gibt auch eine Reihe von verschiedenen Funktionen, die sich auf das Erstellen und Manipulieren der verschiedenen Elementtypen beziehen, die auf Ebenen im Raum platziert werden können. Diese Funktionen sind in den entsprechenden Abschnitten für jeden unten aufgeführten Elementtyp aufgeführt:

  • Allgemeine Schichtfunktionen
  • Kachelkartenebenen
  • Hintergrund-Ebenen
  • Sprite-Ebenen
  • Sequenzschichten

Ein wichtiger Hinweis beim Erstellen von Raumebenen ist, dass Sie mehrere verschiedene Elementtypen auf einer Ebene haben können. So können Sie z. B. eine Ebene für Feinde haben und auf dieser Ebene die Instanzen der Feinde sowie bestimmte Sprite-Elemente die sich nur auf diesen Feind beziehen. Oder Sie könnten ein Hintergrundelement mit einem Kachel-Map-Element haben, um das Aussehen eines Raums zu erzeugen. Dies ist im Raum-Editor nicht erlaubt, aber beim dynamischen Erstellen von Dingen können Sie dies tun, weshalb Funktionen wie layer_get_all_elements() existieren.
