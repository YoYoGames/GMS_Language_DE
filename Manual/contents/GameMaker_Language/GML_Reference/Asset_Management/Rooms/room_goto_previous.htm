raum_zur_vorherigen

Mit dieser Funktion können Sie dafür sorgen, dass Ihr Spiel in den vorherigen Raum wechselt, der zum Zeitpunkt der Kompilierung des Spiels im Raum-Manager aufgeführt war. Wenn dieser Raum nicht existiert, wird ein Fehler ausgelöst und das Spiel wird gezwungen, sich zu schließen. Beachten Sie, dass sich der Raum erst am Ende des Ereignisses ändert, in dem die Funktion aufgerufen wurde, so dass jeglicher Code nach dem Aufruf dieser Funktion weiterhin ausgeführt wird, wenn er sich im selben Ereignis befindet. Der Nachteil dabei ist, dass jeder Code, der Instanzen erzeugt der danach aufgerufen wird, die Instanz nicht erzeugt und daher vor dem Raumwechsel aufgerufen werden sollte. Diese Funktion löst auch das Ereignis " Room End" aus.

Syntax:

room_goto_previous();

Rückgabe:

N/A

Beispiel:

if room_exists(room_previous(room))
{
room_goto_previous();
}

Der obige Code prüft, ob es einen anderen Raum vor dem aktuellen Raum gibt, und wenn ja, geht er zu diesem Raum.
