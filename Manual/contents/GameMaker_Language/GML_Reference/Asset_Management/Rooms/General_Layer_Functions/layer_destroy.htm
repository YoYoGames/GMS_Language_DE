layer_destroy

Diese Funktion zerstört die angegebene Ebene. Sie übergeben die Ebenen-ID (die Sie beim Erstellen der Ebene mit layer_create()) oder den Ebenennamen (als String - dies hat Auswirkungen auf die Leistung) und diese wird sie aus dem aktuellen Raum entfernt. Wenn es sich um eine Ebene handelt, die im Raum-Editor erstellt wurde, wird die Ebene beim nächsten Verlassen des Raums und bei der Rückkehr mit dem ursprünglichen Inhalt neu erstellt, wenn der Raum jedoch persistent ist, wird die Ebene entfernt, es sei denn, die Raumpersistenz wird wieder ausgeschaltet. Wenn Sie eine Ebene auf diese Weise zerstören, wird auch ihr gesamter Inhalt entfernt, d. h. alle Referenz-IDs für Hintergründe oder Kachelmaps usw. sind nicht mehr gültig und alle der Ebene zugewiesenen Instanzen die der Ebene zugewiesen sind, werden zerstört (wobei gleichzeitig ihr Destroy-Ereignis ausgeführt wird, sofern sie eines haben, sowie das Clean Up-Ereignis).

Syntax:

layer_destroy(layer_id)

Argument Beschreibung
layer_id Der eindeutige ID-Wert der zu zerstörenden Ebene (oder der Ebenenname als String)

Rückgabe:

N/A

Beispiel:

if !instance_exists(obj_Bullet_Parent)
{
layer_destroy(global.Bullet_Layer);
}

Der obige Code prüft, ob eine Instanz des Objekts "obj_Bullet_Parent" vorhanden ist, und wenn nicht, wird die Ebene mit der in der globalen Variablen gespeicherten ID zerstört.
