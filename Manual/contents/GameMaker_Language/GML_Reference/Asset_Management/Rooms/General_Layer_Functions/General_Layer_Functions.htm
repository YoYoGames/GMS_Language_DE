Allgemeine Schichtfunktionen

Der GameMaker Studio 2 Raum-Editor erlaubt es Ihnen, Ihrem Raum verschiedene Ebenen von Assets hinzuzufügen. Dies können Kachelmaps (unter Verwendung von Kachelsets), Hintergründe, Sprites oder Sequenzen und Instanzen sein, Beachten Sie jedoch, dass Sie im Gegensatz zum Raum-Editor nicht auf einen einzigen Asset-Typ pro Ebene beschränkt sind und mehrere Asset-Typen zur selben Ebene hinzufügen können - Sie können z. B. einen Hintergrund und eine Tile Map auf derselben Ebene haben. Beachten Sie, dass jedes Element das zu einer Ebene hinzugefügt wird, als ein Element dieser Ebene betrachtet wird.

Dieser Abschnitt beschreibt die allgemeinen Funktionen, die bei der Arbeit mit Raumebenen verwendet werden, sowie einige spezielle Funktionen für die Arbeit mit Instanz-Layer Elementen:

Wir haben auch ein paar zusätzliche Funktionen, die sich auf das Ansteuern von Ebenen in einem anderen Raum beziehen. Es kann sein, dass Sie etwas in einem Raum ändern möchten, der nicht der aktuelle Raum ist, oder vielleicht möchten Sie Räume prozedural generieren, oder eine beliebige Anzahl von Dingen. Um dies zu ermöglichen, verwenden wir die folgenden Funktionen:

  • layer_set_target_room
  • layer_get_target_room
  • layer_reset_target_room

Was Sie hier tun, ist, den Zielraum mit der entsprechenden Funktion einzustellen und dann die Ebenenfunktionen normal aufzurufen. Diese Layer-Funktionen gelten nun nur für den Zielraum und nicht für den aktuellen Raum. Wenn Sie fertig sind, rufen Sie die reset-Funktion auf, um den Codebereich wieder auf den aktuellen Raum zurückzusetzen.

WARNUNG! Während Sie einen anderen Raum anvisieren, können Sie alle regulären Layer-Funktionen verwenden , außer dass Sie keine Instanzen mit instance_create_layer() oder instance_create_depth()erstellen, noch wird die Layer-Funktion layer_add_instance() sein verfügbar.

Schließlich, und dies ist ausschließlich für die Verwendung mit Instanzen auf Ebenen (obwohl die Ebene selbst andere Elemente haben kann, werden diese von den folgenden Funktionen nicht beeinflusst), haben Sie ein paar Funktionen, die zum Deaktivieren oder Reaktivieren verwendet werden können alle Instanzen:

  • instance_deactivate_layer
  • Instanz_aktivieren_Schicht
