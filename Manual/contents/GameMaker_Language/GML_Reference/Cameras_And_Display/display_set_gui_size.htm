display_set_gui_size

Diese Funktion wirkt sich nur auf Dinge aus, die im Draw-GUI-Ereignis gezeichnet werden, und kann verwendet werden, um eine bestimmte Breite und Höhe für alle GUI-Komponenten in diesem Ereignis festzulegen, wobei es keine Rolle spielt, welche unabhängig von der Größe des Displays oder des Fensters, in dem die GUI angezeigt wird, korrekt skaliert wird. Wo die GUI positioniert wird (d. h. die (0,0)-Position), hängt davon ab, ob Sie die Funktion display_set_gui_maximise() verwendet haben oder nicht. Wenn Sie diese Funktion nicht verwenden (Standardeinstellung), wird der GUI-Layer mit seinem (0,0)-Punkt an der gleichen Position positioniert wie die Anwendungsoberfläche (der Bereich, in dem Ihr Spiel gezeichnet wird) und die Breite und Höhe wird Wenn Sie mit dieser Funktion eine GUI-Breite von 256 Pixel einstellen, während die Anwendungsoberfläche 512 Pixel groß ist, werden diese 256 Pixel gestreckt, so dass die Pixel doppelt so groß sind. Dies ist ideal für jene Spiele die eine Seitenverhältniskorrektur in den globalen Spieleinstellungen eingestellt haben.

Wenn Sie die Funktion display_set_gui_maximise() verwendet haben, wird die Position (0,0) für die GUI-Ebene standardmäßig auf die linke obere Ecke des Spielfensters oder des Displays (im Vollbildmodus oder auf Geräten) gesetzt, wobei der Skalierungsfaktor durch diese Funktion festgelegt wird, und die Breite und Höhe durch diese Funktion festgelegt werden. Sie können die GUI-Ebene jederzeit auf die Standardkonfiguration zurücksetzen, indem Sie diese Funktion mit -1 als Breite und Höhe aufrufen - dadurch wird die Position (0, 0) auf die Anwendungsoberfläche gesetzt und die Breite und Höhe werden ebenfalls auf 1:1 mit der Oberfläche zurückgesetzt.

Syntax:

display_set_gui_size(Breite, Höhe);

Argument Beschreibung
Breite Die Breite der GUI
Höhe Die Höhe der GUI

Rückgabe:

N/A

Beispiel:

display_set_gui_size(768, 1024);

Der obige Code sperrt das Draw-GUI-Ereignis auf die angegebene Breite und Höhe und skaliert alle Komponenten so, dass sie entweder auf die Anwendungsoberfläche oder auf das Display passen, wobei dieses Verhältnis verwendet wird.
