bildschirm_speichern_teil

Mit dieser Funktion können Sie einen Screenshot eines Teils des Spiels speichern, wie er gerade in der Spielanzeige erscheint. Die Auswahlkoordinaten basieren auf der absoluten Größe des Bildschirms (oder des Fensters, wenn es sich nicht um einen Vollbildschirm handelt), so dass (0,0) immer die obere linke Ecke des Bildschirms (oder des Fensters). Nachfolgend finden Sie ein Bild, das zeigt, wie diese Funktion funktioniert:

Die Datei wird im Arbeitsverzeichnis des des Spiels gespeichert. Es ist wichtig zu beachten, dass die Funktion für den Aufruf im Draw-GUI-End-Ereignis konzipiert ist, das auf allen Plattformen konsistente Ergebnisse liefern sollte. Wenn Sie die Funktion in einem anderen Ereignis verwenden, funktioniert sie möglicherweise nicht wie erwartet. Ergebnisse auf verschiedenen Zielen und sogar Geräten liefern.

HINWEIS: Diese Funktion funktioniert nicht mit dem HTML5-Ziel.

Syntax:

screen_save_part(fname,x,y,w,h);

Argument Beschreibung
fname Der Dateiname für das zu speichernde Bildschirmfoto
x Die x-Koordinate der linken oberen Ecke des zu erfassenden Bildschirmbereichs
y Die y-Koordinate der linken oberen Ecke des zu erfassenden Bildschirmbereichs
w Die Breite des Bereichs des Bildschirms, der erfasst werden soll
h Die Höhe des zu erfassenden Bereichs des Bildschirms

Rückgabe:

N/A

Beispiel:

if keyboard_check_pressed(ord("S"))
{
screen_save_part("Screens\Screen_" + string(num++) + ".png", 0, 0, 100, 100)
}

Der obige Code macht einen Screenshot von einem Teil des Spielbildschirms, der 100 Pixel im Quadrat ist, wenn der Benutzer die Taste "S" drückt. Dieser Screenshot wird in einem bestimmten Ordner gespeichert und mit einer Variablen nummeriert, die dann inkrementiert wird.
