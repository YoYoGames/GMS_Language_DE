display_get_orientation

Diese Funktion gibt eine von zwei Konstanten zurück, mit denen GameMaker Studio 2 feststellen kann, ob das Gerät, auf dem das Spiel läuft, im Quer- oder Hochformat gehalten wird (siehe Tabelle unten). Beachten Sie, dass diese Funktion die Ausrichtung des Geräts möglicherweise nicht korrekt erkennt des Geräts nicht korrekt erkennt, wenn sie im HTML5-Zielmodul verwendet wird. Dies kann jedoch durch die Verwendung des folgenden Scripts leicht nachgeahmt werden:

return (browser_width < browser_height);

Sie gibt true für Hochformat und false für Querformat zurück.

Syntax:

display_get_orientation()

Rückgabe:

Konstant

Konstant Beschreibung
display_landschaftsbild Das Gerät wird waagerecht gehalten, d. h.: Die längste Kante ist von links nach rechts und die Menütaste ist rechts.
display_landscape_flipped Wie oben, nur ist jetzt die Menütaste auf der linken Seite.
display_portrait Das Gerät wird senkrecht gehalten, d. h.: Die längste Kante ist von oben nach unten, und die Menütaste befindet sich am unteren Rand.
display_portrait_flipped Wie oben, nur ist jetzt die Menütaste oben.

Beispiel:

if display_get_orientation() == display_landscape
{
global.Config = 0;
}
sonst
{
global.Config = 1;
}

Der obige Code prüft die Ausrichtung des Geräts und setzt eine globale Variable in Abhängigkeit vom zurückgegebenen Wert.
