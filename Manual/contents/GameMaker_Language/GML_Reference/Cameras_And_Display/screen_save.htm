Bildschirm_sichern

Mit dieser Funktion können Sie einen Screenshot des Spiels speichern, wie er aktuell im Spielfenster angezeigt wird. Die Datei wird im Arbeitsverzeichnis des Spiels gespeichert, das sich unter Windows unter <Laufwerk>:/Benutzer/YURUSERNAME/AppData/Local/[Spielname]/ befindet und auf dem Mac wäre es ~/Library/Application Support/[Spielname]/. Das erfasste Bild ist das endgültige Rendering der Anwendungsoberfläche des Spiels oder (wenn die Anwendungsoberfläche deaktiviert wurde) ein Bild des gesamten Displays oder Fensters. Es ist wichtig zu beachten, dass die Funktion dafür ausgelegt ist, im Draw GUI End Event aufgerufen zu werden, was auf allen Plattformen konsistente Ergebnisse liefern sollte. Wenn Sie die Funktion in einem anderen Ereignis verwenden, kann es sein, dass sie nicht wie erwartet funktioniert und unterschiedliche Ergebnisse auf verschiedenen Zielen und sogar Geräten unterschiedliche Ergebnisse liefern. Sollten Sie mehr Kontrolle über das, was gespeichert wird, benötigen, sollten Sie die Funktion screen_save_part().

HINWEIS: Diese Funktion funktioniert nicht mit dem HTML5-Ziel.

Syntax:

screen_save(fname);

Argument Beschreibung
fname Der Dateiname für das zu speichernde Bildschirmfoto

Rückgabe:

N/A

Beispiel:

if keyboard_check_pressed(ord("S"))
{
screen_save(arbeits_verzeichnis + "Bildschirme\Bildschirm_" + string(num++) + ".png")
}

Der obige Code macht einen Screenshot des Spiels, wenn der Benutzer die Taste "S" drückt. Dieser Screenshot wird in einem bestimmten Verzeichnis gespeichert und über eine Variable nummeriert, die dann inkrementiert wird.
