window_get_visible_rects

Mit dieser Funktion können Sie den überlappenden Bereich des durch (x1,y1) bis (x2,y2) definierten Rechtecks auf jedem der angeschlossenen Displays finden. Die Funktion gibt ein Array mit 8 Werten pro Display zurück (d. h.: wenn Sie zwei Displays haben, hat das Array eine Länge von 16 Indizes), wobei die Werte [0 ... 3] den Werten overlapx1, overlapy1, overlapx2, overlapy2 entsprechen - sie definieren den Bereich der Überlappung auf diesem Display und werden auf 0,0,0,0 gesetzt, wenn keine Überlappung vorliegt - und die Werte [4 ... 7] entsprechen den Werten monitorx1, monitory1, monitorx2, monitory2 - den Koordinaten des Displays im virtuellen Anzeigeraum. Dies kann verwendet werden, um zu testen, ob eine gespeicherte Fensterposition sichtbar sein wird oder nicht (der Benutzer hat vielleicht einen externen Monitor abgezogen oder das Fenster aus dem Bildschirm verschoben, wodurch die gespeicherte Fensterposition nicht mehr gültig ist).

Syntax:

window_get_visible_rects(x1, y1, x2, y2);

Argument Beschreibung
x1 Die linke Kante des zu prüfenden Rechtecks
y1 Die obere Kante des zu prüfenden Rechtecks.
x2 Die rechte Kante des zu prüfenden Rechtecks
y2 Die untere Kante des zu prüfenden Rechtecks.

Rückgabe:

Array

Beispiel:

var wx = window_get_x();
var wy = window_get_y();
var ww = window_get_width();
var wh = window_get_height();
display_data = window_get_visible_rects(wx, wy, wx + ww, wy + wh);
display_num = array_length(display_data) / 8;

Der obige Code erzeugt ein 1D-Array in der Variablen display_data, das die Informationen über die Displays enthält, sowie die Variable display_num, die die Anzahl der gefundenen aktiven Displays enthält.
