gif_open

Mit dieser Funktion können Sie ein leeres Bild im GIF-Format erzeugen, das bereit ist, mit Daten versehen zu werden. Wenn Sie die Funktion aufrufen, müssen Sie die Breite und Höhe des GIFs angeben (in Pixel, und es gibt keine Obergrenze für die Größe, außer für den verfügbaren Speicher), und die Funktion gibt den eindeutigen ID-Wert zurück, mit dem das Gif in nachfolgenden Funktionen identifiziert wird, oder sie gibt -1 zurück, wenn das Gif nicht initialisiert werden konnte (z. B. wenn die Breite/Höhe zu groß für den verfügbaren Speicher ist). Sie können auch angeben ein optionales Argument, um die "klare Farbe" für das GIF festzulegen. Dabei handelt es sich um einen RGB-Wert (ohne Alphakomponente), der das GIF auf diese Farbe zurücksetzt, bevor ihm etwas hinzugefügt wird. Wenn Sie kein Argument für eine klare Farbe angeben, wird die Standardfarbe von Schwarz verwendet.

Beachten Sie, dass Sie bei Verwendung dieser Funktion die Funktion gif_save() aufrufen müssen, um die Erstellung des GIFs zu beenden, bevor Sie eine andere Datei zur Aufnahme öffnen (für jedes gif_open() muss also ein begleitendes gif_save() vorhanden sein).

Syntax:

gif_open(Breite, Höhe, [clear_colour]);

Argument Beschreibung
Breite Die Breite des zu erstellenden Gifs
Höhe Die Höhe des zu erstellenden Gifs
[clear_colour] [Optional] Die Farbe (RGB), auf die das Gif gelöscht werden soll

Rückgabe:

Integer

Beispiel:

if save_gif == true
{
if count == 0
{
gif_image = gif_open(room_width, room_height);
}
else if count < 30
{
gif_add_surface(gif_image, application_surface, 6/100);
}
sonst
{
gif_save(gif_image, "GameCapture.gif");
count = 0;
save_gif = false;
}
count++;
}

Der obige Code erstellt eine GIF-Bilddatei mit 30 Bildern aus der Anwendungsoberfläche und speichert sie anschließend.
