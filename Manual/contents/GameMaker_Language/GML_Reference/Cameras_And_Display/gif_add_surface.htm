gif_add_surface

Mit dieser Funktion können Sie ein Bild aus einer Oberfläche in eine GIF-Datei speichern. Sie liefern die ID der GIF-Datei (wie von der Funktion gif_open()) und eine Oberflächen-ID (zurückgegeben von der Funktion surface_create()), sowie eine Verzögerungszeit zwischen den Bildern. Die Verzögerungszeit wird in 1/100 Sekunden berechnet, aber beachten Sie, dass nach der Erstellung des GIFs, wenn Sie es z. B. für Werbezwecke verwenden, verschiedene Browser die Frame-Verzögerung etwas anders interpretieren können unterschiedlich interpretieren können - siehe hier für weitere Details. Sie können (optional) X- und Y-Offset-Werte für die Fläche angeben, die Sie zum GIF hinzufügen möchten. Sie können (optional) X- und Y-Offset-Werte für die Fläche angeben, die zum GIF hinzugefügt werden soll. In diesem Fall beginnt der Teil, der zum GIF gezeichnet wird, an der Offset-Position und nicht an der Standardposition (0,0), so dass Sie nur einen Teil der Fläche zum Hinzufügen auswählen können. Sie können auch einen Quantisierungswert angeben, der eine ganze Zahl zwischen 0 und 3, wobei 0 für volle Quantisierung und 3 für Nullquantisierung steht (niedrigere Werte führen zu einer Qualitätsverschlechterung, erzeugen aber ein kleineres GIF). Beachten Sie, dass es keinen großen Unterschied zwischen der Standardqualität von 2 und der höchsten von 3 gibt, und die Verwendung von 3 die Erstellung des GIFs erheblich verlangsamt, so dass man bei der Verwendung des Maximalwerts vorsichtig sein sollte.

Beachten Sie, dass es keine eingebauten Grenzen für die Anzahl der Frames oder für die Größe dieser Frames in Ihrem GIF gibt, aber wenn Sie zu viel Speicher verwenden, schlägt die Funktion fehl und gibt -1 zurück, andernfalls gibt sie 0 zurück.

Syntax:

gif_add_surface(gif_index, surface, delay_time, [xoffset], [yoffset]);

Argument Beschreibung
gif_index Die ID des gif, zu dem die Oberflächendaten hinzugefügt werden sollen
Oberfläche Die ID der Oberfläche, die als hinzugefügter Rahmen verwendet werden soll
delay_time Die Verzögerungszeit für den Frame (in 100stel einer Sekunde)
[xoffset] [Optional] Der Versatz entlang der X-Achse der Oberfläche, der von
[yOffset] [Optional] Der Versatz entlang der Y-Achse der Oberfläche, der von
[Quantisierung] [Optional] Der Quantisierungsbetrag von 0 bis 3 (0 niedrigste Qualität, 3 höchste Qualität, Standard ist 2)

Rückgabe:

Integer

Beispiel:

if save_gif == true
{
if count == 0
{
gif_image = gif_open(room_width, room_height);
}
else if count < 30
{
gif_add_surface(gif_image, application_surface, 6/100);
}
sonst
{
gif_save(gif_image, "GameCapture.gif");
count = 0;
save_gif = false;
}
count++;
}

Der obige Code erstellt eine GIF-Bilddatei mit 30 Bildern aus der Anwendungsoberfläche und speichert sie anschließend.
