view_set_surface_id

Mit dieser Variablen können Sie den Inhalt eines Ansichtsports zum Zeichnen auf eine Oberfläche festlegen. Wenn Sie mit Oberflächen arbeiten, ist es oft erforderlich, den gesamten sichtbaren Bereich des Bildschirms auf die Oberfläche zu übertragen, und deshalb würden Sie ihn mit dieser Funktion einem Ansichtsfenster zuweisen diese Funktion. Dies bedeutet, dass alles, was in der gewählten Port-Ansicht angezeigt wird, nun auf die zugewiesene Oberfläche gezeichnet wird und der Inhalt dieses View-Ports nicht mehr angezeigt wird, was bedeutet, dass Sie entweder:

  • Aktiviert eine neue Ansicht und zeichnet die Oberfläche nur in dieser Ansicht (mit view_current um zu prüfen, welche Ansicht gerade gezeichnet wird)
  • Zeichnen Sie die Oberfläche im Ereignis "Draw GUI" einer Instanz, da die GUI-Ebene unabhängig von Ansichten ist.

Wenn Sie diese Funktion verwenden, geben Sie den Index des View-Ports (von 0 bis 7) und einen Oberflächenindex (entweder den anwendung_oberflaeche oder den eindeutigen Indexwert, der von der Funktion oberfläche_erstellen()) oder, wenn bereits eine Oberfläche zugewiesen wurde und Sie diese entfernen wollen, den Wert -1. Für weitere Informationen zum Setzen des Ansichtsanschlusses auf eine Oberfläche siehe auch die Variable view_oberflaeche_id.

Syntax:

view_set_surface_id(view_port, surf)

Argument Beschreibung
Ansicht_Port Der anzuvisierende Ansichtsanschluss (0 - 7)

Rückgabe:

Real

Beispiel:

if view_get_surface_id(0) == -1
{
view_set_surface_id(0, global.vSurf);
}

Der obige Code prüft, ob dem View Port[0] eine Oberfläche zugewiesen wurde, und wenn nicht, wird eine zugewiesen.
