Kamera_anwenden

Diese Funktion wendet die angegebenen Kameraeinstellungen sofort auf die aktuell gerenderte Kameraansicht an. Während Sie mit den spezifischen camera_set_*-Funktionen eine beliebige Kamera-Ansicht einstellen können, werden die neuen Einstellungen erst im nächsten Spiel-Frame für das Rendering verwendet. Die neuen Einstellungen werden erst beim nächsten Spielframe für das Rendering verwendet, aber mit dieser Funktion können Sie diese Änderungen sofort vornehmen. Wenn Sie mehrere Ansichten verwenden, sollte die Funktion im Draw-Event verwendet werden und Sie können die Variable ansicht_aktuell verwenden, um die angegebene Kamera selektiv nur auf die Kameraansicht anzuwenden, die gerade gerendert wird. Beachten Sie, dass dies NICHT den Wert der Kamera-ID ändert, sondern nur die Einstellungen für diese Kamera, und dass, wenn Sie diese Funktion auf die Standardkamera anwenden (d. h.: die interne Kamera, die GameMaker Studio 2 verwendet, wenn keine Ansichtskameras aktiv sind), werden die Änderungen im nächsten Spielframe durch die internen Einstellungen für das Zeichnen des Raums überschrieben.

Syntax:

camera_apply(camera_id)

Argument Beschreibung
Kamera_id Der eindeutige Kamera-ID-Wert, der beim Erstellen der Kamera zurückgegeben wurde.

Rückgabe:

N/A

Beispiel:

if view_current == 0
{
camera_apply(cutscene_cam);
}

Der obige Code (in einem Draw-Event) prüft, welche Kameraansicht gerade gerendert wird, und wenn es Kameraansicht[0] ist, dann werden die Einstellungen für die Kamera, die durch die Variable "cutscene_cam" referenziert wird, darauf angewendet.
