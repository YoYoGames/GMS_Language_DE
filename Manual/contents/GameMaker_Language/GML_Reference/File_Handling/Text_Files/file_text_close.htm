file_text_close

Wenn Sie die Arbeit mit einer bestimmten Datei beendet haben (egal ob Sie aus ihr lesen oder in sie schreiben), müssen Sie die Datei wieder schließen, da sonst die Gefahr besteht, dass Sie die darin enthaltenen Informationen verlieren. Dies verhindert auch Speicherlecks und stellt sicher, dass Sie niemals die das Dateilimit nicht überschreiten, indem Sie mehr als 32 Dateien geöffnet haben.

Syntax:

file_text_close(fileid);

Argument Beschreibung
fileid Die ID der zu schließenden Datei.

Rückgabe:

N/A

Beispiel:

var file = file_text_open_write(working_directory + "Game_Data.txt");
while (!file_text_eof(file))
{
file_text_readln(file);
}
file_text_write_string(file, level_data);
file_text_close(file);

Der obige Code öffnet eine Datei zum Schreiben und läuft dann in einer Schleife durch die bereits in die Datei geschriebenen Textzeilen, bis er das Ende erreicht. An diesem Punkt schreibt er eine Zeichenkette und schließt dann die Datei wieder.
