file_text_write_real

Mit dieser Funktion können Sie eine Zahl in die zuvor geöffnete Textdatei schreiben. Beachten Sie, dass, da der zu schreibende Wert eine reelle Zahl sein kann, alle Nachkommastellen mit einem "."-Punkt als Trennzeichen geschrieben werden. Wenn die Datei bereits Informationen enthält, werden diese Informationen gelöscht und die Zeichenfolge wird an den Anfang der Datei geschrieben, es sei denn, Sie haben die Datei mit der Option file_text_open_append(). Sie können dies auch vermeiden, indem Sie den file_text_readln() Funktion zusammen mit der file_text_eof() Funktion den Inhalt der Datei in einer Schleife durchlaufen, bis Sie am Ende angelangt sind, und dann mit dem Schreiben beginnen.

Es ist wichtig zu beachten, dass beim Schreiben sehr großer Zahlen in eine Textdatei mit dieser Funktion der Wert möglicherweise in ein vereinfachtes Standardformat wie "6.6624e+003" übersetzt wird, das nicht korrekt in GameMaker Studio 2 zurückgelesen werden kann. Um solche Probleme zu vermeiden, sollten Sie den Wert stattdessen in einen String konvertieren und die Funktion file_text_write_string() verwenden.

Syntax:

file_text_write_real(fileid, val);

Argument Beschreibung
fileid Die ID der zu bearbeitenden Datei.
val Der tatsächliche Wert, der in die Datei geschrieben werden soll.

Rückgabe:

Real

Beispiel:

var file = file_text_open_write(working_directory + "hiscore.txt");
for (var i = 0; i < 10; ++i;)
{
file_text_write_real(file, scr[i]);
file_text_writeln(file);
file_text_write_string(datei, scr_name[i]);
file_text_writeln(datei);
}
file_text_close(file);

Der obige Code öffnet eine Datei zum Schreiben und läuft dann in einer Schleife durch zwei Arrays, wobei jeder Array-Wert in eine neue Zeile der Datei geschrieben wird. Die Datei wird dann geschlossen, wenn die Schleife beendet ist.
