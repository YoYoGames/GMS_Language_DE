Text-Dateien

Textdateien sind eine nützliche Möglichkeit, um große Datenmengen extern von Ihrem Spiel zu speichern. Im Gegensatz zu ini-Dateien können Sie bis zu 32 Textdateien gleichzeitig geöffnet haben, und Sie können auch große Datenpakete darin lesen und schreiben. Es liegt jedoch an Ihnen zu entscheiden wie diese Daten strukturiert sein sollen, da es keine feste "Abschnitts-" oder "Schlüssel"-Struktur wie bei *.ini-Dateien gibt.

Bitte beachten Sie, dass es für Spiele mit Lokalisierungen sehr wichtig ist, dass akzentuierte Buchstaben aus externen Dateien gelesen werden können. Das heißt, Sie sollten die txt-Datei zunächst im UTF8-Format erstellen und dann in GameMaker Studio 2 als eingebundene Datei hinzufügen, damit sie beim Ausführen des Spiels exportiert und anstelle der Standard-txt-Datei im ANSI-Format verwendet wird, die von den GameMaker Studio 2-Dateifunktionen Funktionen erstellt wird, wenn vorher keine Datei übergeben wurde. Auf diese Weise können Sie sie korrekt lesen und beschreiben, wobei alle Akzente und nicht-römischen Buchstaben erhalten bleiben.

Es gibt die folgenden Funktionen, die sich mit Dateien befassen:

  • file_text_open_read
  • file_text_open_write
  • file_text_open_append
  • file_text_open_from_string
  • file_text_read_real
  • file_text_read_string
  • file_text_readln
  • file_text_write_real
  • file_text_write_string
  • file_text_writeln
  • file_text_eoln
  • file_text_eof
  • file_text_close
