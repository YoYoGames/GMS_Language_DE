md5_string_utf8

In der Kryptografie ist MD5 (Message-Digest-Algorithmus 5) eine weit verbreitete kryptografische Hash-Funktion mit einem 128-Bit-Hash-Wert und wurde in einer Vielzahl von Sicherheitsanwendungen eingesetzt. Sie wird auch häufig verwendet, um die Integrität von Dateien und Zeichenketten zu prüfen. Diese Funktion nimmt eine UTF-8-Eingabezeichenfolge (mit einer variablen Anzahl von Bytes pro Zeichen) und gibt den 32-stelligen hexadezimalen MD5-Hashwert zurück, der für diese Zeichenfolge eindeutig ist. Auf diese Weise können Sie einen sicheren Schlüssel erzeugen, der gespeichert und verwendet werden kann um die Integrität der Informationen zu prüfen, die z. B. an einen externen Server gesendet (oder von diesem empfangen) werden.

HINWEIS: Es gibt zwei Formate für die MD5-Kodierung, UTF-8 und Unicode. Beide sind vorgesehen, um die Kommunikation mit verschiedenen Serverkonfigurationen zu erleichtern, aber am häufigsten wird Unicode verwendet.

HINWEIS: MD5 ist nicht vollständig sicher und kann geknackt werden. Siehe diese Seite für weitere Informationen.

Syntax:

md5_string_utf8(zeichenkette)

Argument Beschreibung
String Die zu hashende Zeichenfolge.

Rückgabe:

Zeichenfolge

Beispiel:

var hash, str;
str = json_encode(hiscore_map);
hash = md5_string_utf8(str);
ini_open("local.ini");
ini_write_string("info", "0", hash);
ini_close();
get[0] = http_post_string("http://www.MacSweeney Games.com/CatchTheHaggis?game_hiscores=" + string(global.game_id), str)

Der obige Code kodiert eine DS-Map in einen JSON-String. Anschließend wird ein MD5-Hash generiert und in einer Ini-Datei gespeichert, so dass dieser später verwendet werden kann, um die Integrität des JSON zu prüfen, falls die gleichen Informationen später vom Server empfangen werden. Das JSON wird dann gesendet.
