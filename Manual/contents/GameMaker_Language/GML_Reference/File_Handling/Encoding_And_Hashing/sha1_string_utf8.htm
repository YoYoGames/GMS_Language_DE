sha1_string_utf8

In der Kryptografie ist SHA-1 eine kryptografische Hash-Funktion, die von der United States National Security Agency entwickelt wurde und in mehreren weit verbreiteten Anwendungen und Protokollen wie dem beliebten Git verwendet, wo sie zur Überprüfung von Dateiänderungen eingesetzt wird. Diese Funktion nimmt eine UTF-8-Eingabezeichenfolge (mit einer variablen Anzahl von Bytes pro Zeichen) und gibt z. B. einen 160-Bit-Message-Digest im ASCII-Format zurück:

sha1 input string = "Der schnelle braune Fuchs springt über den faulen Hund"
sha1 output hash = "2fd4e1c6 7a2d28fc ed849ee1 bb76e739 1b93eb12"

Auf diese Weise können Sie einen sicheren Schlüssel erzeugen, der gespeichert und verwendet werden kann, um die Integrität der Informationen zu prüfen, die z. B. an einen externen Server gesendet (oder von diesem empfangen) werden.

HINWEIS: Es gibt zwei Formate für die sha1-Kodierung, UTF-8 und Unicode. Beide sind vorgesehen, um die Kommunikation mit verschiedenen Serverkonfigurationen zu erleichtern, aber am häufigsten wird Unicode verwendet.

HINWEIS: SHA-1 ist nicht vollständig sicher und kann gebrochen werden. Siehe diese Seite für weitere Informationen.

Syntax:

sha1_string_utf8(zeichenkette)

Argument Beschreibung
String Die zu hashende Zeichenfolge.

Rückgabe:

Zeichenfolge

Beispiel:

var hash, str;
str = json_encode(hiscore_map);
hash = sha1_string_utf8(str);
ini_open("local.ini");
ini_write_string("info", "0", hash);
ini_close();
get[0] = http_post_string("http://www.MacSweeney Games.com/CatchTheHaggis?game_hiscores=" + string(global.game_id), str)

Der obige Code kodiert eine DS-Map in einen JSON-String. Anschließend wird ein sha1-Hash generiert und in einer ini-Datei gespeichert, so dass dieser später verwendet werden kann, um die Integrität des JSON zu überprüfen, falls die gleichen Informationen später vom Server empfangen werden. Das JSON wird dann gesendet.
