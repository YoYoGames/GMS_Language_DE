file_find_first

Diese Funktion gibt den Namen der ersten Datei zurück, die die Maske und die Attribute erfüllt. Wenn keine solche Datei existiert, wird eine leere Zeichenkette zurückgegeben. Die Maske kann einen Pfad enthalten und kann Platzhalter enthalten, z. B. 'C:\temp\*.doc'. Die Attribute geben die zusätzlichen Dateien an, die Sie sehen wollen, so dass die normalen Dateien immer zurückgegeben werden, wenn sie die Maske erfüllen. Sie können die folgenden Konstanten addieren, um den gewünschten Dateityp zu sehen (wenn Sie keine Attribute hinzufügen möchten, verwenden Sie 0):

Konstant Beschreibung
fa_readonly Schreibgeschützte Dateien
fa_hidden Versteckte Dateien
fa_sysfile System-Dateien
fa_volumeid Volume-id-Dateien
fa_directory Verzeichnisse
fa_archive Archivierte Dateien

WARNUNG! Diese Funktion funktioniert möglicherweise nicht so, wie Sie es erwarten, da GameMaker Studio 2 sandboxed ist! Bitte lesen Sie den Abschnitt über das Dateisystem für weitere Informationen.

HINWEIS: Diese Funktion funktioniert überhaupt nicht auf dem HTML5-Ziel, und die Filterflags funktionieren nur unter Windows (verwenden Sie 0 für alle anderen Plattformen).

Syntax:

file_find_first(mask, attr);

Argument Beschreibung
Maske Die für die Suche zu verwendende Maske.
attr Das spezifische Dateiattribut, nach dem gesucht werden soll.

Rückgabe:

Zeichenfolge

Beispiel:

if directory_exists("\Benutzer-Inhalt")
{
file = file_find_first("/Benutzerinhalt/*.doc", fa_readonly);
}

Dies prüft, ob das angegebene Verzeichnis existiert, und wenn ja, geht es dorthin und gibt die erste gefundene "read only" doc-Datei zurück.
