Arbeits_Verzeichnis

Working_directory kann tatsächlich zwei verschiedene Werte zurückgeben, je nachdem, wofür Sie es verwenden. Wenn Sie eine Datei auf die Festplatte schreiben, zeigt working_directory auf den Bereich des lokalen Speichers, der für Ihr Spiel auf dem Zielgerät Gerät reserviert ist (unter Windows ist dies "%LOCALAPPDATA%/gamedir/", wobei "gamedir" das Verzeichnis mit dem Namen Ihres Spiels ist). Wenn Sie jedoch von der Festplatte lesen, kann working_directory entweder der lokale Speicher oder der Bereich sein, in dem die eingeschlossenen Dateien gespeichert sind.

HINWEIS: Die Variable working_directory gibt den Pfad einschließlich des letzten Backslashs zurück.

Wenn Sie z. B. eine standardmäßige *.txt-Datei in Ihr Spiel eingebunden haben und diese mit working_directory in den Speicher einlesen, dann wird diese Datei aus dem Bereich genommen, in dem die eingebundenen Dateien gespeichert sind. Wenn Sie dann wählen, dass Sie diese Informationen mit working_directory in eine Textdatei schreiben, wird die Datei in den lokalen Speicher der Zielplattform geschrieben, von wo sie beim nächsten Mal gelesen wird, es sei denn, Sie weisen GameMaker Studio 2 an, die Datei zu löschen, in diesem Fall verweist working_directory wieder auf den Bereich, in dem die enthaltenen Dateien gespeichert sind.

Es ist erwähnenswert, dass Sie dies im Allgemeinen nicht verwenden müssen und der einfache Aufruf des Dateinamens (plus eventuelle zusätzliche Pfadinformationen) ausreicht, z. B. dies:

file_text_open_read(arbeits_verzeichnis + "meine_datei.txt");

wäre besser als dies:

file_text_open_read("meine_datei.txt");

WARNUNG! Diese Funktion funktioniert möglicherweise nicht so, wie Sie es erwarten, da GameMaker Studio 2 sandboxed ist! Bitte lesen Sie den Abschnitt über das Dateisystem für weitere Informationen.

Syntax:

Arbeits_Verzeichnis

Rückgabe:

Zeichenfolge

Beispiel:

ini_open(arbeits_verzeichnis + "temp_ini.ini");

Dadurch wird eine Ini-Datei aus dem Arbeitsverzeichnis des Spiels geöffnet (und erstellt, falls sie noch nicht vorhanden ist). Dies kann der lokale Speicher oder der Bereich sein, in dem sich die eingebundenen Dateien befinden, je nachdem, ob die gesuchte Datei an einem der beiden Orte existiert.
