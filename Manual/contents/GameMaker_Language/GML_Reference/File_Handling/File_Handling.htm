Datei-Handhabung

Irgendwann bei der Erstellung Ihres Spiels wird es notwendig sein, Informationen extern zu speichern und in der Lage zu sein, diese Informationen abzurufen. Zum Beispiel möchten Sie vielleicht Anmeldedaten oder Highscores speichern, oder vielleicht den Fortschritt des Spielers durch Ihre Levels. Was auch immer der Grund ist, Sie müssen wissen, wie Sie die verschiedenen Dateifunktionen verwenden, die Ihnen beim Arbeiten mit GameMaker Studio 2 zur Verfügung stehen.

Es ist sehr wichtig, sich darüber im Klaren zu sein, dass GameMaker Studio 2 standardmäßig sandboxed ist, was bedeutet, dass es eine Reihe von Einschränkungen und kleineren Komplikationen bei der Verwendung der verfügbaren Dateisystemfunktionen gibt, und es ist auch erwähnenswert, dass das Dateisystem System je nach Plattform etwas anders funktioniert. Weitere Informationen hierzu finden Sie im folgenden Abschnitt:

  • Das Dateisystem

Sie können das Sandboxing auf den Desktop-Zielen(Windows, macOS und Ubuntu(Linux)) in den Spieloptionen für die gewählte Plattform ausschalten, was bedeutet, dass Sie dann von überall aus speichern und laden können, innerhalb der Grenzen der Betriebssystemberechtigungen.

Wenn Sie von Steam die Erlaubnis erhalten haben, ihre dedizierte API mit Ihrem Spiel zu verwenden, werden Sie feststellen, dass Ihnen auch ein neuer Satz von Dateifunktionen zur Verfügung steht. Diese werden im Abschnitt " Steam-API " in diesem Handbuchs erläutert.

Schließlich sollten Sie noch beachten, dass es einige deutliche Unterschiede zwischen bestimmten Zielplattformen gibt, die bei der Verwendung der Dateifunktionen zu berücksichtigen wären. Diese sind:

  • iOS: Wenn Sie unter iOS eine eingebundene Datei aus einem Ordner laden, d. h. mit einem Pfad wie "Backgrounds/background1.png", müssen Sie den Ordner NICHT als Teil des Pfades angeben, sondern es genügt, den Dateinamen anzugeben. ausreichend. Auf allen anderen Plattformen müssen Sie den/die Ordner als Teil des Pfades zur Datei angeben.
  • macOS und Ubuntu (Linux): Auf macOS und Ubuntu wird bei Dateinamen und Pfaden auf diesen Plattformen zwischen Groß- und Kleinschreibung unterschieden. Um mögliche Probleme zu vermeiden, verwendet GameMaker Studio 2 daher standardmäßig für alle Namen/Pfade automatisch Kleinschreibung.
  • HTML5: Bei HTML5 funktioniert das Speichern und Laden in und aus dem lokalen Speicher wie auf den verschiedenen Funktionsseiten beschrieben. Wenn Sie jedoch eine Datei von einem Server laden, dürfen Sie nur die Funktion buffer_load_async() verwenden, da das synchrone Laden in den meisten Browsern veraltet ist und irgendwann überflüssig wird. Beachten Sie auch, dass Sie im lokalen Speicher des Browsers keine Verzeichnisse erstellen oder zerstören können.
