Ini-Dateien

INI-Dateien sind kleine, leichtgewichtige Dateien, die mit den meisten Plattformen kompatibel sind. Sie sind ideal zum Speichern kleiner Informationen, wie z. B. Interface-Einstellungen, lokale Highscores, Level-Daten usw... und sind daher eine großartige und einfache Möglichkeit für Sie, Informationen aus Ihren Spielen zu speichern. Es ist jedoch erwähnenswert, dass sie ein Limit für die Daten haben, die sie speichern können, nämlich ein Maximum von 64 KB. Wenn Sie also Dinge wie Datenstrukturen speichern möchten, sollten Sie sich andere Dateifunktionen wie Textdateien ansehen.

WICHTIG! Alle GameMaker Studio 2-Projekte haben eine Datei namens "options.ini", Sie können also keine eine Datei mit diesem Namen benennen (Sie erhalten einen uid-Fehler, wenn Sie versuchen, Ihr Spielprojekt zu kompilieren).

Bitte beachten Sie, dass es für Spiele mit Lokalisierungen sehr wichtig ist, dass akzentuierte Buchstaben aus externen Dateien gelesen werden können. Das heißt, Sie sollten die ini-Datei zunächst im UTF8-Format erstellen und dann in GameMaker Studio 2 als eingebundene Datei hinzufügen, damit sie beim Ausführen des Spiels exportiert und anstelle der standardmäßigen Ini-Datei im ANSI-Format verwendet wird, die von den Ini-Funktionen erzeugt wird, wenn zuvor keine geliefert wurde. Auf diese Weise können Sie sie korrekt lesen und schreiben, wobei alle Akzente und nicht-römischen Buchstaben erhalten bleiben.

Für ini-Dateien gibt es die folgenden Funktionen:

  • ini_öffnen
  • ini_schliessen
  • ini_write_real
  • ini_write_string
  • ini_lesen_real
  • ini_read_string
  • ini_key_exists
  • ini_section_exists
  • ini_key_delete
  • ini_section_delete
  • ini_open_from_string
