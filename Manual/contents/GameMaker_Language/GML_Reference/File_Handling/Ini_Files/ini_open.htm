ini_öffnen

Dies öffnet eine ini_file zum Lesen und / oder Schreiben. Wenn die ini_Datei an dem Ort, den Sie prüfen, nicht existiert, kann GameMaker Studio 2 eine erstellen, aber nur, wenn Sie Daten in sie schreiben. Wenn Sie nur Informationen aus der ini-Datei gelesen haben, dann werden die Standardwerte für die Lesefunktion zurückgegeben, aber die ini-Datei wird nicht tatsächlich erstellt.

Bitte beachten Sie, dass Sie immer nur eine Ini-Datei geöffnet haben können, und denken Sie daran, die Funktion ini_close() sobald Sie mit dem Lesen/Schreiben aus der .ini-Datei fertig sind, da die Informationen erst dann tatsächlich auf der Festplatte gespeichert werden. Festplatte gespeichert werden (sie werden auch im Speicher gespeichert, bis die Datei geschlossen wird).

WARNUNG! Diese Funktion funktioniert möglicherweise nicht so, wie Sie es erwarten, da GameMaker Studio 2 sandboxed ist! Bitte lesen Sie den Abschnitt über das Dateisystem für weitere Informationen.

Syntax:

ini_open(name);

Argument Beschreibung
Name Der Dateiname für die .ini-Datei.

Rückgabe:

N/A

Beispiel:

ini_open("Settings/savedata.ini");
score = ini_read_real("save1", "score", 0);
ini_close();

Dadurch wird die "savedata.ini" geöffnet und der Wert "score" unter der Sektion "save1" mit dem Schlüssel "score" darin ausgelesen, dann wird die .ini wieder geschlossen. Sollte unter "save1", "score" kein Wert stehen oder keine "savedata.ini" Datei vorhanden sein, wird score auf 0 gesetzt (der Standardwert). Beachten Sie, dass die ini-Datei im Unterverzeichnis "Settings" abgelegt wurde, dem Ordner, der die ini-Datei in den enthaltenen Dateien des Asset Browsers enthält.
