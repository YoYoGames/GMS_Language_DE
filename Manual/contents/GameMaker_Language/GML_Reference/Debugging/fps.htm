fps

In GameMaker Studio 2 gibt es im Wesentlichen zwei Möglichkeiten, die Geschwindigkeit, mit der Ihr Spiel läuft, zu bestimmen. Die Spielgeschwindigkeit (wie in den Spieloptionen angegeben) und die fps (frames per second). Diese Werte werden oft verwechselt, aber im Grunde ist der eine die Anzahl der Spielschritte, die GameMaker Studio 2 in einer Sekunde ausführen soll, während der andere (die fps) die Anzahl der CPU-Schritte ist, die GameMaker Studio 2 tatsächlich in einer Sekunde bis zu einem Maximalwert der Raumgeschwindigkeit selbst ausführt. Um die wahren fps zu erhalten, d. h. die tatsächliche Anzahl von CPU-Schritten pro Spielschritt, verwenden Sie die fps_real Variable.

Diese schreibgeschützte Variable gibt die aktuelle fps als Integer-Wert zurück. Bitte beachten Sie, dass die Funktion nur einmal pro Spielschritt aktualisiert wird und daher von einem Wert zum anderen zu "springen" scheint, aber das ist ganz normal.

Syntax:

fps

Rückgabe:

Real

Beispiel:

if debug_mode
{
draw_text(32, 32, "FPS = " + string(fps));
}

Der obige Code prüft, ob sich das Spiel im Debug-Modus befindet, und wenn dies der Fall ist, wird die aktuelle fps auf dem Bildschirm angezeigt.
