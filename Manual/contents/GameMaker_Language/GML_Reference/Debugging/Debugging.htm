Fehlersuche

Bei der Entwicklung eines Spiels ist es oft sehr wichtig, dass Sie bestimmte Details der Spielmechanik zu bestimmten Zeitpunkten kennen, oder dass Sie bestimmte Schlüsselwerte für einige Funktionen eingeben können, um verschiedene Methoden und Ideen für Ihr Spiel zu testen. Zu diesem GameMaker Studio 2 stellt Ihnen zu diesem Zweck eine Reihe von Debug-Funktionen zur Verfügung, die im Debug-Modus (oder auf einem Windows-Target ) Meldungen anzeigen und nach Eingaben fragen, die aber beim Kompilieren für andere Target-Module ignoriert werden, da diese Funktionen das Spiel in eine geschlossene Schleife bringen, bis die Eingabe aufgelöst ist.

HINWEIS: Obwohl sie auf bestimmten Plattformen funktionieren können, sollten sie nicht für andere Zwecke als zur Fehlersuche verwendet werden.

  • Debug_mode
  • Debug_Ereignis
  • debug_get_callstack
  • ausnahme_unbehandelt_handler
  • get_integer
  • get_string
  • show_error
  • show_message
  • show_question
  • show_debug_message
  • show_debug_overlay
  • code_ist_kompiliert
  • fps
  • fps_real

Neben diesen speziellen Funktionen, die Ihnen bei der Erstellung Ihres Spiels helfen, verfügt GameMaker Studio 2 auch über eine lange Liste von Fehlermeldungen, die entweder beim Schreiben Ihres Spielcodes, beim Kompilieren Ihres Spiels oder während der Ausführung des Spiels selbst angezeigt werden können. Diese Meldungen weisen auf den jeweiligen Fehler hin und können verwendet werden, um ein Problem innerhalb eines bestimmten Objekts oder einer Scriptfunktion zu lokalisieren, was bei der Fehlersuche in Ihrem Spiel sehr hilfreich ist. Es ist jedoch nicht immer offensichtlich, was genau die Die folgenden Abschnitte listen daher alle möglichen Meldungen auf und erklären, was sie bedeuten, um Ihnen die Arbeit zu erleichtern:

  • Syntax-Fehler
  • Compiler-Fehler
  • Läufer-Fehler

Weitere Informationen zum Debuggen Ihres Spiels finden Sie auch im Abschnitt über den Debugger.
