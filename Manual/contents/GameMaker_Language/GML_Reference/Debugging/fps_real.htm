fps_real

In GameMaker Studio 2 gibt es im Wesentlichen zwei Möglichkeiten, die Geschwindigkeit, mit der Ihr Spiel läuft, zu bestimmen. Die Spielgeschwindigkeit (wie in den Spieloptionen angegeben) und die fps (frames per second). Diese Werte werden oft verwechselt, aber im Grunde ist der eine die Anzahl der Spielschritte, die GameMaker Studio 2 in einer Sekunde ausführen soll (Spielgeschwindigkeit), während der andere die Anzahl der CPU-Schritte ist, die GameMaker Studio 2 tatsächlich in einer Sekunde ausführt (die tatsächlichen fps), und dieser Wert ist im Allgemeinen viel höher als die Spielgeschwindigkeit, sinkt aber, wenn Ihr Spiel komplexer wird und mehr Rechenleistung benötigt, um die eingestellte Raumgeschwindigkeit beizubehalten.

Diese schreibgeschützte Variable gibt die aktuelle fps als Integer-Wert zurück. Bitte beachten Sie, dass die Funktion nur einmal pro Spielschritt aktualisiert wird und daher von einem Wert zum anderen zu "springen" scheint, aber das ist ganz normal.

HINWEIS: Auf dem HTML5-Ziel gibt diese Variable stattdessen ein Vielfaches der Bildwiederholfrequenz des Monitors zurück, da sich GameMaker Studio 2 beim Timing und der Versendung auf den Browser verlassen muss.

Syntax:

fps_real

Rückgabe:

Real

Beispiel:

if debug_mode
{
draw_text(32, 32, "FPS = " + string(fps_real));
}

Der obige Code prüft, ob sich das Spiel im Debug-Modus befindet, und wenn dies der Fall ist, zeigt er die aktuellen realen fps auf dem Bildschirm an.
