Web und HTML5

GameMaker Studio 2 bietet verschiedene Funktionen für die Interaktion mit dem Internet oder mit einem Server, die auf allen verfügbaren Zielplattformen verwendet werden können. Diese Funktionen sind:

  • url_open
  • url_open_ext
  • url_open_full
  • url_get_domain
  • http_Anfrage
  • http_get
  • http_get_file
  • http_post_string

Abgesehen von den oben aufgeführten allgemeinen Funktionen verfügt das HTML5-Target über einige zusätzliche Webfunktionen, die speziell für die Verwendung durch die Spiele, die darauf abzielen, vorgesehen sind:

  • browser_height
  • browser_width
  • browser_input_capture
  • webgl_enabled
  • klickbar_vorhanden
  • anklickbar_hinzufügen
  • klickbar_hinzufügen_ext
  • klickbare_Änderung
  • klickbar_ändern_ext
  • klickbar_set_style
  • anklickbar_löschen
  • analyse_ereignis
  • analyse_ereignis_ext
  • os_browser
  • http_get_request_crossorigin
  • http_set_request_crossorigin

Neben diesen Funktionen für das HTML5-Zielmodul gibt es auch eine spezielle JavaScript-Variable, die in den seltenen Fällen verwendet werden kann, wenn der Inhalt des html5game-Ordners an einem ganz anderen Ort (URL) gespeichert ist. Normalerweise ist dieser Pfad in der JavaScript-Datei fest kodiert und relativ zu dem Ort, von dem aus das JavaScript aufgerufen wird, aber Sie können die Variable g_GameMakerHTML5Dir vor dem Aufruf des Spielscripts setzen und das Spiel wird von der neu definierten Stelle aus. Zum Beispiel:

<script>var g_GameMakerHTML5Dir = "http://mygamealternatedirectory.com/";</script>

Es ist auch erwähnenswert, dass Sie Skriptfunktionen mit dem Präfix gmcallback_ kennzeichnen können, was bedeutet, dass sie nicht verschleiert werden, wenn Sie das Projekt für HTML5 kompilieren. Dies ist sehr sinnvoll, wenn Sie die oben aufgeführten clickable_* Funktionen verwenden (weitere Informationen finden Sie hier).
