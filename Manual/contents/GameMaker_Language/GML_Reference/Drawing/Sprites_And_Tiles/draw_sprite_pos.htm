sprung_pos zeichnen

Mit dieser Funktion können Sie ein Sprite verzerrt über den durch die vier Eckkoordinaten definierten Bereich zeichnen. Die ersten beiden Argumente sind das zu zeichnende Sprite und das Sub-Image des Sprites (das gleiche wie bei der Funktion draw_sprite()) aber die nächsten sind diejenigen, die die Position jeder der vier Ecken des Begrenzungsrahmens des angegebenen Sprites definieren. Diese sollten im Uhrzeigersinn angegeben werden, also ist die erste Koordinate die obere linke, dann die obere rechte, dann die untere rechts und schließlich die untere linke. Sie können auch einen Wert für das Alpha des Sprites setzen, um es mit Transparenz zu zeichnen. Das folgende Bild veranschaulicht, wie diese Funktion funktioniert:

WARNUNG! Das obige Bild dient nur zur Veranschaulichung. Wenn Sie diese Funktion auf ein Sprite anwenden, erhalten Sie andere Ergebnisse und es kann zu einer "Scherung" der Textur kommen, da ein Sprite aus einem Quad von Primitiven aufgebaut ist.

HINWEIS: Diese Funktion ist nur für Bitmap-Sprites nützlich und funktioniert nicht mit SWF- oder JSON (Spine)-Sprites.

HINWEIS: Wenn Sie diese Funktion verwenden, sollten Sie die Einstellung " Automatisch beschneiden" für Texturseiten deaktiviert haben.

Syntax:

draw_sprite_pos(sprite, subimg, x1, y1, x2, y2, x3, y3, x4, y4, alpha);

Argument Beschreibung
Sprite Der Index des zu zeichnenden Sprites.
subimg Das Subimg (Frame) des zu zeichnenden Sprites (image_index oder -1 entsprechen dem aktuellen Frame der Animation im Objekt).
x1 Die erste x-Koordinate.
y1 Die erste y-Koordinate.
x2 Die zweite x-Koordinate.
y2 Die zweite y-Koordinate.
x3 Die dritte x-Koordinate.
y3 Die dritte y-Koordinate.
x4 Die vierte x-Koordinate.
y4 Die vierte y-Koordinate.
alpha Das Alpha des Sprites (von 0 bis 1, wobei 0 transparent und 1 undurchsichtig ist).

Rückgabe:

N/A

Beispiel:

draw_sprite_pos(sprite_index, image_index, x - 100, y - 50, x - 50, y +150, x + 100, y + 200, x + 100, y, 1);

Der obige Code zeichnet das Sprite, das mit der Instanz verbunden ist, die den Code ausführt, verzerrt um die x/y-Position der Instanz und mit einem vollständig opaken Alpha.
