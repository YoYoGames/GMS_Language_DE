Sprites und Kacheln

Auch wenn Sie im Raumeditor Hintergründe einrichten und Instanzen Sprites zuweisen können, ist dies oft nicht flexibel genug für das Spiel, das Sie erstellen. Wenn Sie zum Beispiel eine Instanz haben, die ein Menü zeichnen muss, möchten Sie vielleicht verschiedene Schaltflächen-Sprites gleichzeitig anzeigen, oder wenn Sie ein Spiel mit scrollenden Parallaxen-Hintergründen haben, müssen Sie in der Lage sein, diese durch Code in einer Controller-Instanz irgendeiner Art zu setzen. In diesen Situationen sollten Sie die folgenden Funktionen verwenden, die Sprites und Hintergründe an beliebiger Stelle im Raum zeichnen, wobei einige der Funktionen auch die Möglichkeit bieten, Form, Größe, Farbe und Alpha des Sprites zu manipulieren, um spezielle Effekte zu erzielen.

Sie können einer Instanz ein Sprite zuweisen und standardmäßig wird GameMaker Studio 2 es mit den Eigenschaften zeichnen, die durch die verschiedenen Sprite-Variablen vorgegeben sind, Wenn Sie jedoch etwas in das Draw-Ereignis einfügen, "übernehmen" Sie die Kontrolle und müssen GameMaker Studio 2 genau sagen, was es zeichnen soll. Das bedeutet, dass Sie, wenn Sie die folgenden Funktionen verwenden, genau definieren müssen, was die Instanz zeichnen soll und wo ungefähr im Raum sie es zeichnen soll:

  • zeichnen_selbst
  • zeichnen_sprite
  • zeichnen_sprite_ext
  • draw_sprite_general
  • spalt_teil_zeichnen
  • zeichnung_spalte_teil_ext
  • draw_sprite_stretched
  • draw_sprite_stretched_ext
  • sprung_pos zeichnen
  • draw_sprite_tiled
  • draw_sprite_tiled_ext

Wenn Sie Sprites im SWF-Format verwenden, gibt es auch zusätzliche Optionen zur Steuerung des beim Zeichnen verwendeten Anti-Aliasings (siehe Importieren von Nicht-Bitmap-Sprites für weitere Informationen zu diesen Sprite-Typen). Beachten Sie, dass diese Funktionen auf globaler Ebene arbeiten und die AA auf alle Vektor-SWF-Sprites setzen, die nach ihrem Aufruf gezeichnet werden.

  • draw_enable_swf_aa
  • draw_set_swf_aa_level
  • draw_get_swf_aa_level

Neben den oben aufgeführten Funktionen gibt es auch zwei spezielle Funktionen, die für die Verwendung mit Skelettanimationen entwickelt wurden (siehe Importieren von Nicht-Bitmap-Sprites für Weitere Informationen zu diesen Sprite-Typen finden Sie unter Importieren von Nicht-Bitmap-Sprites und im Abschnitt Skelettanimationen für spezielle Funktionen im Zusammenhang mit diesen Sprite-Typen):

  • zeichnen_skelett
  • zeichnen_skelett_instanz
  • draw_skeleton_collision
  • zeichnen_skelett_zeit

Kacheln werden automatisch gezeichnet, basierend auf den Daten, die sie enthalten, dem verwendeten Kachelsatz und den Eigenschaften der Kachelkarte. Sie können jedoch einzelne Kacheln oder Kachelkarten mit den folgenden Funktionen zeichnen:

  • kachel_zeichnen
  • kachelplan_zeichnen
