vertex_submit

Mit dieser Funktion können Sie den Inhalt eines Scheitelpunktpuffers an die Grafikpipeline zur Verwendung mit einem Shader übergeben. Sie geben den zu verwendenden Pufferindex, den zu verwendenden Basis-Primitivtyp (siehe die Konstanten unten) und die zu verwendende Textur an. Der Basisprimitivtyp wird nur für die Zuweisung der Reihenfolge verwendet, in der die Scheitelpunkte, die Sie im Puffer gespeichert haben, gezeichnet und verbunden werden, aber die tatsächlichen Daten, die für jeden der Scheitelpunkte verwendet werden, sind die, die Sie beim Erstellen des Scheitelpunktpuffers definiert haben.

HINWEIS: Diese Funktion kann nur im Ereignis "Zeichnen" verwendet werden.

Ein visuelles Beispiel für die verschiedenen verfügbaren Basisprimitive finden Sie in der folgenden Abbildung:

Konstant Beschreibung
pr_punktliste Ein Primitiv, das aus einer Liste von Punkten besteht.
pr_linelist Ein Primitiv, das aus einzelnen Zeilen einer Liste besteht.
pr_linestrip Ein Primitiv, das aus einem aufeinanderfolgenden Streifen von Linien besteht.
pr_dreiecksliste Ein Primitiv, das aus einzelnen Dreiecken in einer Liste besteht.
pr_trianglestrip Ein Primitiv, das aus einem aufeinanderfolgenden Streifen von Dreiecken besteht.

Syntax:

vertex_submit(buffer, primitiv, textur);

Argument Beschreibung
Puffer Der zu verwendende Puffer.
primitiv Der primitive Basistyp.
Textur Die zu verwendende Textur (oder -1 für keine Textur).

Rückgabe:

N/A

Beispiel:

shader_set(shader_prim);
vertex_submit(buff, pr_trianglelist, sprite_get_texture(sprite_index));
shader_reset();

Der obige Code übergibt den in der Variablen "buff" indizierten Vertex-Puffer zum Zeichnen mit dem Shader " shader_prim ",wobei eine Dreiecksliste als Basisprimitiv und die Textur des Sprites für die Instanzverwendet wird, die den Code ausführt.
