part_system_create_layer

Diese Funktion erzeugt ein neues Partikelsystem auf einem gegebenen Layer. Sie geben die eindeutige Layer-ID, wie sie von der Funktion schicht_erzeugen() oder den Namen der der Ebene als Zeichenkette an - z. B. "instance_layer" - und kennzeichnen dann das System als persistent oder nicht. Wenn das System nicht als dauerhaft gekennzeichnet ist, wird es am Ende des Raums, in dem es erstellt wurde, automatisch zerstört (dies ist dasselbe, als hätten Sie die Funktion part_system_destroy() aufgerufen hätten, und es werden auch alle mit dem System verbundenen Emitter zerstört). Wenn es jedoch als persistent gekennzeichnet ist, wird das System Wenn der folgende Raum keine Ebene mit dem gleichen Namen oder der gleichen Tiefe wie die zugewiesene Ebene hat, wird eine neue Ebene für das System erstellt, die über die Räume hinweg bestehen bleibt. denselben Namen wie die ursprüngliche Ebene. Wenn beim Wechseln von Räumen in den folgenden Räumen eine andere Ebene mit demselben Namen vorhanden ist, wird die persistierte Instanz der Ebene mit diesem Namen zugewiesen , unabhängig von der Tiefe der Ebene. Zum Schluss, Wenn schließlich ein persistiertes System in einen Raum mit einer Ebene in der gleichen Tiefe wie die Ebene, auf der das System erstellt wurde, wechselt, wird es nicht dieser Ebene zugewiesen, sondern es wird eine neue Ebene in der gleichen Tiefe (mit dem gleichen Namen wie die ursprüngliche Ebene).

Die Funktion gibt einen eindeutigen ID-Wert für das Partikelsystem zurück, der in allen weiteren Funktionsaufrufen verwendet werden sollte, bei denen Sie eine System-ID angeben müssen.

WICHTIG! Wenn Sie das Partikelsystem als persistent kennzeichnen, müssen Sie es (und alle ihm zugewiesenen Emitter) manuell mit den entsprechenden Zerstörungsfunktionen bereinigen, wenn es nicht verwendet wird, sonst riskieren Sie ein Speicherleck, das sich negativ das endgültige Spiel auswirkt.

Syntax:

part_system_create_layer(layer_id, persistent);

Argument Beschreibung
Ebene Der Layer-ID-Wert oder String, dem das Partikelsystem zugewiesen werden soll (kann ein beliebiger Layertyp sein)
beständig Kennzeichnet das Partikelsystem als persistent (auf true gesetzt) oder nicht (auf false gesetzt)

Rückgabe:

Real (eindeutiger Partikelsystem-ID-Wert)

Beispiel:

global.p_sys = part_system_create_layer("effects_layer", true);

Der obige Code erstellt ein neues Partikelsystem auf der angegebenen Ebene und kennzeichnet es als dauerhaft für nachfolgende Räume. Die ID für das Partikelsystem wird in einer globalen Bereichsvariablen für zukünftige Referenzen gespeichert.
