teil_system_automatisch_aktualisieren

Diese Funktion steuert, ob GameMaker Studio 2 das Partikelsystem automatisch aktualisieren soll oder nicht. Normalerweise brauchen Sie diese Funktion nicht zu verwenden, da der Standardwert von true (automatische Aktualisierung ist eingeschaltet) das ist, was Sie wünschen geschieht. Für spezielle Effekte oder zum Pausieren des Spiels können Sie diese Funktion jedoch auf false setzen, und sie verhindert, dass Aktualisierungen für das angegebene Partikelsystem durchgeführt werden, es sei denn, dies wird explizit durch die Verwendung der Funktion part_system_update() oder Sie verwenden diese Funktion erneut, um die automatischen Aktualisierungen einzuschalten. Keine Aktualisierung bedeutet, dass Partikel, die in diesem System erzeugt werden, nicht mehr ihre Position, Farbe oder andere Parameter ändern und dass auch Emitter nicht mehr funktionieren, wodurch das System "eingefroren" wird genau an dem Punkt "einfriert", an dem die automatische Aktualisierung auf false gesetzt wurde.

Eine Sache, die zu beachten ist, ist, dass wenn Sie die einfachen Effekte verwenden, die durch die Funktionen effect_create_above() oder effect_create_below() erzeugen, dann können Sie mit die Werte 0 (für Effekte unterhalb) oder 1 (für Effekte oberhalb) als Partikelsystem-Index verwenden und so die automatische Aktualisierung auch für diese umschalten (dies funktioniert auch, um die Aktualisierung für die DnD™-Partikel-Effekteumzuschalten).

Syntax:

part_system_automatic_update(ind, automatic);

Argument Beschreibung
ind Der Index des zu ändernden Partikelsystems.
automatisch Ob die automatische Aktualisierung eingeschaltet ist(true) oder nicht (false).

Rückgabe:

N/A

Beispiel:

if global.Pause
{
part_system_automatic_update(global.Sname, false);
}

Der obige Code schaltet die Partikel-Updates aus, wenn die globale Variable "Paused" wahr ist.
