Partikel-Systeme

Bevor Sie Partikel erstellen können, müssen Sie ein Partikelsystem erstellen. Dies ist ein "Container", in dem Sie Ihre Partikel und Emitter (falls Sie sie benötigen) zur Verwendung bereitstellen, und Sie können so viele oder so wenige Partikel in ein System legen, wie Sie Sie können so viele oder so wenige Partikel in ein System legen, wie Sie für nötig halten, und Sie können auch so viele Systeme haben, wie Sie für nötig halten. Es ist jedoch immer besser, diese Anzahl so gering wie möglich zu halten, da jedes System, jeder Emitter und jedes Partikel Speicherplatz benötigt und zu viele davon kann Ihr Spiel verlangsamen oder Probleme verursachen. Wenn Sie z. B. einige Effekte oberhalb von Instanzen und andere Effekte unterhalb von Instanzen benötigen, würden Sie zwei Systeme erstellen und ihre Tiefen so einstellen, dass Sie den gewünschten Effekt erhalten, wobei alle Partikel die zu jedem System hinzugefügt werden, in der von Ihnen angegebenen Tiefe gezeichnet werden.

Da ein Partikelsystem eine dynamisch erstellte Ressource ist, müssen Sie es erstellen und den zurückgegebenen Index in einer Variablen speichern, um das System in allen weiteren Funktionsaufrufen zu referenzieren, und es ist sehr wichtig, dass Sie das System auch zerstören, wenn es nicht mehr benötigt wird sonst haben Sie ein Speicherleck, das Ihr Spiel verlangsamt und schließlich zum Absturz bringt. Es ist auch erwähnenswert, dass Partikelsysteme für immer weiterleben, nachdem sie erstellt wurden, auch wenn der Index nicht mehr gespeichert ist. Selbst wenn Sie also den Raum wechseln oder das Spiel neu starten, bleiben die Systeme und die Partikel in allen weiteren Räumen erhalten und sind sichtbar.

Im Folgenden sind die grundlegenden Funktionen zum Einrichten eines Partikelsystems aufgeführt:

  • teil_system_existiert
  • teil_system_erstellen
  • part_system_create_layer
  • part_system_get_layer
  • part_system_layer
  • teil_system_tiefe
  • teil_system_position
  • teil_system_klar
  • teil_system_zerstören
  • teil_partikel_klar
  • part_particles_count

Sobald Partikel zu einem Partikelsystem hinzugefügt und dann in den Raum geplatzt oder gestreamt wurden, werden sie normalerweise bei jedem Schritt automatisch aktualisiert und basierend auf den Parametern gezeichnet, die Sie zu ihrer Definition verwendet haben. Es kann jedoch manchmal notwendig sein, zu steuern wann und wie das System aktualisiert wird und wie das System gezeichnet werden soll, und dafür bietet GameMaker Studio 2 die folgenden Funktionen:

  • teil_system_automatisch_aktualisieren
  • teil_system_automatisch_ziehen
  • teil_system_aktualisierung
  • teil_system_drawit
  • part_system_draw_order

Sie können Partikelemitter verwenden, um Partikel aus einem Bereich auszubrechen oder zu strömen, aber in vielen Fällen sind diese nicht notwendig und es ist tatsächlich besser, die Partikel einfach zu erzeugen direkt mit den folgenden Funktionen zu erzeugen:

  • teil_partikel_erzeugen
  • teil_partikel_erzeugen_farbig
