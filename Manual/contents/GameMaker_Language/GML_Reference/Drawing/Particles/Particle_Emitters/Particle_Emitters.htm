Partikel-Emitter

Partikel-Emitter werden von GameMaker Studio 2 verwendet, um Partikel über einen Bereich des Bildschirms zu emittieren, die unterschiedliche Formen und Verteilungen haben können. Außerdem können sie entweder einen kontinuierlichen Strom von Partikeln erzeugen oder sie können eine Anzahl von Partikeln auf einmal ausstoßen, je nachdem, wie die Funktionen verwendet werden.

Da ein Partikelemitter eine dynamisch erstellte Ressource ist, müssen Sie ihn erstellen und den zurückgegebenen Index in einer Variablen speichern, um den Emitter in allen weiteren Funktionsaufrufen referenzieren zu können, und es ist sehr wichtig, dass Sie den Emitter auch zerstören, wenn er nicht mehr benötigt wird Sie müssen den Emitter unbedingt zerstören, wenn er nicht mehr benötigt wird, da sonst ein Speicherleck entsteht, das Ihr Spiel verlangsamt und schließlich zum Absturz bringt. Es ist auch erwähnenswert, dass Partikelemitter für immer weiterleben, nachdem sie erstellt wurden, auch wenn der Index nicht mehr gespeichert ist. Selbst wenn Sie also den Raum wechseln oder das Spiel neu starten, bleiben die Systeme und die Partikel in allen weiteren Räumen sichtbar, so dass Sie sicherstellen sollten, dass Sie sie zerstören, wenn Sie sie nicht mehr benötigen.

Die folgenden Funktionen stehen zur Verfügung, um die Emitter zu setzen und um sie Partikel erzeugen zu lassen. Beachten Sie, dass jede von ihnen den Index des Partikelsystems, zu dem sie gehört, als erstes Argument erhält.

  • part_emitter_exists
  • part_emitter_create
  • part_emitter_clear
  • part_emitter_region
  • part_emitter_burst
  • part_emitter_stream
  • part_emitter_destroy
  • part_emitter_destroy_all
