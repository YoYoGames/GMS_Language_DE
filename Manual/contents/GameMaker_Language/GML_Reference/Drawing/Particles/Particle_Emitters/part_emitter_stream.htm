part_emitter_stream

Diese Funktion ermöglicht es Ihnen, einen Emitter so einzustellen, dass er einen bestimmten Partikeltyp streamt, und wird typischerweise für das Ereignis "Create" verwendet, da es sich um einen einmaligen Code handelt, der den Emitter so einstellt, dass er die angegebene Anzahl von Partikeln in jedem weiteren Schritt des Spiels erzeugt. Die Partikel werden entsprechend der Verteilung, Form und Position emittiert, die mit der Funktion part_emitter_region(). Sie können die Anzahl der gestreamten Partikel auf Null setzen und diese Funktion erneut aufrufen aufrufen, um das Partikel-Streaming "auszuschalten". Die Funktion akzeptiert auch negative Zahlen für die Menge, in diesem Fall produziert der Emitter Partikel nach dem Zufallsprinzip. Wenn Sie zum Beispiel die Partikelanzahl auf -5 gesetzt haben, besteht besteht eine 1:5-Chance, dass bei jedem Schritt ein Partikel produziert wird. Wenn Sie möchten, dass die Partikel auf einmal und nicht bei jedem Schritt erscheinen, sollten Sie die Funktion part_emitter_burst().

Syntax:

part_emitter_stream(ps, ind, parttype, number);

Argument Beschreibung
ps Das Partikelsystem, in dem sich der Emitter befindet.
ind Der Index des Emitters, von dem der Burst ausgehen soll.
parttype Der Index (Typ) der zu erstellenden Partikel.
Nummer Die Anzahl der zu erstellenden Partikel pro Schritt.

Rückgabe:

N/A

Beispiel:

part_emitter_stream(global. Sname, p_emit1, p1, 1);

Der obige Code streamt bei jedem Schritt des Spiels 1 Partikel, bis der Emitter zerstört oder der Stream auf 0 gesetzt wird.
