teil_art_ausrichtung

Diese Funktion wird verwendet, um die Ausrichtung des Partikel-Sprites zu bestimmen, wenn es erstellt wird, und kann auch verwendet werden, um die Ausrichtung des Partikels während seiner Lebensdauer zu erhöhen oder zu verringern. Die Werte für die minimale und maximale Ausrichtung sind standardmäßig auf 0 gesetzt, aber diese Werte können jedoch geändert werden, um die Ausrichtung nach dem Zufallsprinzip entsprechend der Standardausrichtung in GameMaker Studio 2 zu ändern: 0 Grad ist rechts, 90 Grad ist oben, 180 Grad ist links und 270 Grad ist unten. Wenn Sie sie auf denselben Wert setzen, werden die Partikel alle mit der gleichen Ausrichtung erstellt werden.

Sie können auch einen Inkrementwert einstellen, der über die Lebensdauer der Ausrichtung einen Gradbetrag addiert (bei einer positiven Zahl) oder subtrahiert (bei einer negativen Zahl). Dieser Wert kann ein Minimum von (+/-) 0,01 sein.

Sie können auch den "Wiggle"-Faktor einstellen, wie bei anderen Partikelfunktionen auch. Dies ist ein Wert, der bei jedem Schritt der Partikellebensdauer zufällig zur Ausrichtung addiert oder subtrahiert wird. Offensichtlich sind größere Werte stärker ausgeprägt als kleinere und dieser Wert kann sogar negativ sein, wobei der maximale Bereich zwischen -20 und 20 liegt.

Schließlich können Sie wählen, ob die Ausrichtung relativ sein soll oder nicht, was bedeutet, dass das Partikelsprite um diesen Vektor herum ausgerichtet wird, während das Partikel eine Richtung (und Geschwindigkeit) hat. Beachten Sie, dass, wenn Sie die Partikelgeschwindigkeit auf eingestellt haben und die Geschwindigkeit 0 erreicht, setzt der Mangel an Geschwindigkeit die Richtung auf den Standardwert von 0°, so dass eine relative Ausrichtung dazu führt, dass das Partikel-Sprite in einen anderen Winkel "springt".

Syntax:

part_type_orientation( ind, ang_min, ang_max, ang_incr, ang_wiggle, ang_relative );

Argument Beschreibung
ind Der Index des zu ändernden Partikeltyps.
ang_min Der minimale Startwinkel der Partikel.
winkel_max Der maximale Startwinkel der Partikel.
ang_incr Die Zunahme (oder Abnahme, wenn negativ eingestellt) des Winkels in Grad pro Schritt.
ang_wiggle Der Betrag, um den der Winkel pro Schritt im Uhrzeigersinn/gegen den Uhrzeigersinn wackelt.
winkel_relativ Ob sein Winkel relativ zur Bewegungsrichtung des Partikels gesetzt werden soll (true) oder nicht (false).

Rückgabe:

N/A

Beispiel:

part_type_shape(particle2, pt_shape_spark);
part_type_size(particle2, 0.10, 0.50, 0.01, 0);
part_type_scale(particle2, 0.30, 0.30);
part_type_colour1(particle2, 8454143);
part_type_alpha1(particle2, 0.50);
part_type_speed(particle2, 4, 4, -0.07, 1);
part_type_direction(particle2, 0, 359, 0, 20);
part_type_orientation(particle2, 0, 359, 0, 20, 1);
part_type_blend(particle2, 1);
part_type_life(particle2, 1, 5);

Der obige Code setzt verschiedene Partikelwerte, einschließlich der Ausrichtung, die ein Zufallswert zwischen 0º und 359º ist. Außerdem wird bei jedem Schritt ein Zufallswert zwischen 0 und 20 hinzugefügt, und die Ausrichtung ist relativ zur Richtung der Bewegung.
