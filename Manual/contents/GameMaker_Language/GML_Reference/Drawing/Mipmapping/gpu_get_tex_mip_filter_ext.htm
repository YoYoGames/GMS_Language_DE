gpu_get_tex_mip_filter_ext

Mit dieser Funktion können Sie den aktuellen Mip-Filtermodus für einen bestimmten Shader-Sampler ermitteln. Sie liefern den Indexwert für den Shader-Sampler (wie er von der Funktion shader_get_sampler_index(), zurückgegeben wird) und die Funktion gibt eine der unten aufgeführten Moduswert-Konstanten zurück.

Syntax:

gpu_get_tex_mip_filter_ext(sampler_index);

Argument Beschreibung
sampler_index Der Index des zu erhaltenden Shader-Samplers

Rückgabe:

Konstante (unten aufgeführt):

Konstant Beschreibung
tf_punkt Dies bedeutet, dass das Überblenden zwischen den Mipmap-Ebenen deaktiviert ist, was zu sichtbaren Texturübergängen führen kann, aber die beste Leistung bietet.
tf_linear Dies bedeutet, dass das Überblenden zwischen den Mipmap-Ebenen aktiviert ist (dies wird auch als trilineare Filterung bezeichnet), wodurch die Texturübergänge geglättet werden, was jedoch zu einer geringfügigen Leistungseinbuße führt.
tf_anisotrop Dies bedeutet, dass die anisotrope Filterung aktiviert ist, die die Qualität der Texturübergänge stark verbessert und die bei anderen Filterungsmodi sichtbare Unschärfe reduzieren kann, aber den größten Einfluss auf die Leistung hat.

Beispiel:

var _sampleIndex = shader_get_sampler_index(shd_Glass, "s_Background");
var _spriteTex = sprite_get_texture(sprite_index, 0);
shader_set (shd_Glass);
if gpu_get_tex_mip_filter_ext(_sampleIndex) != tf_point
{
gpu_set_tex_mip_filter_ext(_sampleIndex, tf_point);
}
texture_set_stage(_sampleIndex , _spriteTex);
shader_reset();

Der obige Code setzt den Mip-Filtermodus für den angegebenen Shader-Textursampler auf tf_point (Deaktivierung von Mipmapping), wenn er nicht bereits eingestellt ist.
