gpu_set_blendmode_ext

Wenn GameMaker Studio 2 ein Pixel zeichnen will, gibt es eine Quellfarbe (die Farbe des Pixels, das wir zeichnen wollen) und eine Zielfarbe (die Farbe, die bereits in dem Pixel enthalten ist, auf das wir zeichnen wollen), so dass bei der Bestimmung der neuen Farbe von Bei der Bestimmung der neuen Farbe des Pixels werden die Quell- und Zielfarben entsprechend dem gewählten Mischmodus berechnet. Jede Komponente der Farben wird als Fließkommawert zwischen 0 und 1 gespeichert, und die neue Farbe wird durch Multiplikation jeder Komponente der Die neue Farbe wird berechnet, indem jede Komponente der Quellfarbe mit einem Faktor und jede Komponente der Zielfarbe mit einem anderen Faktor multipliziert wird und die Ergebnisse dann Komponente für Komponente addiert werden.

Mit dieser Funktion können Sie die verschiedenen Komponenten einstellen, die zusammengerechnet werden sollen, um einen benutzerdefinierten Mischmodus zu erstellen. Die Quelle und das Ziel haben jeweils eine Rot-, Grün-, Blau- und Alphakomponente, und im folgenden Diagramm werden die RGBA der Quelle der Quelle als (Rs, Gs, Bs, As) und die des Ziels als (Rd, Gd, Bd, Ad). Die elf Konstanten, die zur Verwendung zur Verfügung stehen, können entweder Quelle oder Ziel (oder beides) sein, wenn sie in dieser Funktion verwendet werden.

Konstant Überblendfaktor (Rot, Grün, Blau, Alpha)
bm_null (0, 0, 0, 0)
bm_eins (1, 1, 1, 1)
bm_src_colour (Rs, Gs, Bs, As)
bm_inv_src_colour (1-Rs, 1-Gs, 1-Bs, 1-As)
bm_src_alpha (As, As, As, As)
bm_inv_src_alpha (1-As, 1-As, 1-As, 1-As)
bm_dest_alpha (Anzeige, Anzeige, Anzeige, Anzeige)
bm_inv_dest_alpha (1-Ad, 1-Ad, 1-Ad, 1-Ad)
bm_dest_colour (Rd, Gd, Bd, Ad)
bm_inv_dest_colour (1-Rd, 1-Gd, 1-Bd, 1-Ad)
bm_src_alpha_sat (f, f, f, 1) mit f = min(As, 1-Ad)

WICHTIG! HTML5 ohne aktiviertes WebGL wird die folgenden Modi nicht korrekt anzeigen können:

  • bm_src_colour
  • bm_inv_src_colour
  • bm_dest_colour
  • bm_inv_dest_colour
  • bm_src_alpha_sat

Beachten Sie, dass Sie entweder zwei einzelne Argumente an diese Funktion übergeben können oder ein Array von Argumenten (wie von der Funktion gpu_get_blendmode_ext() zum Beispiel). Wenn Sie ein Array übergeben übergeben, sollte es die folgenden zwei Elemente enthalten:

  • 0] = Quell-Mischmodus (Standard ist bm_src_alpha)
  • 1] = Ziel-Mischmodus (Standard ist bm_inv_src_alpha)

Damit Sie die Mischmodi optimal nutzen können und um zu verstehen, wie sie funktionieren und wie sie das endgültige Bild auf dem Bildschirm beeinflussen, empfehlen wir Ihnen, den folgenden Leitfaden zu lesen:

  • Anleitung zur Verwendung von Blendmodes

Syntax:

gpu_set_blendmode(src, dest);

Argument Beschreibung
src Faktor des Quellüberblendungsmodus (siehe Konstanten oben).
dest Ziel-Mischmodus-Faktor (siehe Konstanten oben)

Rückgabe:

N/A

Beispiel:

gpu_set_blendmode_ext(bm_src_alpha, bm_one);
draw_circle_colour(100, 100, 50, c_white, c_black, 0);
gpu_set_blendmode(bm_normal);

Dadurch wird das Schwarz transparent und es entsteht ein "Glüheffekt", da das Weiß an der Außenseite stark ist und von der Kreismitte aus allmählich schwächer wird. Der Mischmodus wird nach dem Zeichnen des Kreises zurückgesetzt, um sicherzustellen, dass die additive Überblendung nicht auch auf alles andere nach ihm angewendet wird.
