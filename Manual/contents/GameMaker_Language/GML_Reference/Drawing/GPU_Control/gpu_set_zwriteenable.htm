gpu_set_zwriteenable

Mit dieser Funktion können Sie die Tiefenordnung über den z-Puffer ein-(true) und ausschalten(false). Wenn eine Instanz in GameMaker Studio 2 gerendert wird, wird die Tiefe eines erzeugten Pixels (seine z-Koordinate) im z-Puffer gespeichert, Wenn dann eine andere Instanz der Szene an demselben Pixel gerendert werden muss, werden die beiden Tiefen verglichen (z-Test - siehe gpu_set_ztestenable()) und diejenige, die dem Betrachter "näher" ist, wird dann in den z-Buffer gespeichert und ersetzt die alte. Dies ist jedoch nicht immer das, was Sie wünschen, und es gibt Momente, in denen Sie möchten, dass etwas über alles andere gezeichnet wird, egal, welchen z-Wert es hat. In diesen Fällen können Sie mit dieser Funktion verwenden, um das Schreiben in den z-Puffer abzuschalten und so zu erzwingen, dass das, was gezeichnet wird, über alles andere gezeichnet wird. Beachten Sie, dass die hier gegebene Beschreibung für den standardmäßigen z-Test-Vergleichsmodus gilt, der aber geändert werden kann

Der Standardwert ist, dass die z-Pufferung aktiviert ist(true).

Syntax:

gpu_set_zwriteenable(enable);

Argument Beschreibung
aktivieren. Der zu verwendende Vergleichsmodus (siehe Liste oben)

Rückgabe:

N/A

Beispiel:

gpu_set_zwriteenable(false);
with (obj_Menu_Containers)
{
draw_self();
}
gpu_set_zwriteenable(true);

Der obige Code schaltet das Schreiben des Z-Puffers aus, zeichnet eine Reihe von Objekten und schaltet ihn dann wieder ein, um das Zeichnen fortzusetzen.
