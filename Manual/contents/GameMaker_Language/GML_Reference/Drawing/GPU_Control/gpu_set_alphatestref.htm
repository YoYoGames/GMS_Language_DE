gpu_set_alphatestref

Diese Funktion setzt den Referenzwert für den Alphatest (wenn dieser mit der Funktion gpu_set_alphatestenable()). Dies ist der "Cut-off"-Schwellenwert, bei dem Pixel mit Alpha nicht gezeichnet werden.

Wenn dieser Wert auf 0 gesetzt ist und Sie die lineare Interpolation eingeschaltet haben (entweder in den Spieloptionen oder mit gpu_set_texfilter()), können Sie immer noch visuelle Fehler an den Rändern von überlappenden Objekten. Das liegt daran, dass Sie fast vollständig transparente Bereiche haben können, die alles blockieren, was dahinter gezeichnet wird. Wenn Sie diesen Referenzwert jedoch z. B. auf 254 (d. h. einen Wert unter dem Maximalwert) setzen, garantiert dies, dass nur 100 % opake Pixel gezeichnet werden. gezeichnet werden. Sie können die Texturinterpolation auch ausschalten, wodurch Sie normalerweise die Standardwerte ohne Probleme verwenden können und diese Funktion überhaupt nicht benötigen, allerdings hängt viel von der Ausgangsgrafik ab.

HINWEIS: Dies ist eine erweiterte Zeichenfunktion. Wenn Sie also nicht sicher sind, ob Sie sie benötigen oder nicht, oder nicht genau wissen, was sie bewirkt, sollten Sie sie besser nicht verwenden, da sie die Zeichnung Ihres Spiels ernsthaft beeinträchtigen kann.

Syntax:

gpu_set_alphatestref(val);

Argument Beschreibung
val Legt den Referenzwert für den Alphatest von 0 bis 255 fest (Standardwert ist 0)

Rückgabe:

N/A

Beispiel:

if !gpu_get_alphatestenable()
{
gpu_set_alphatestenable(true);
gpu_set_alphatestref(128);
}

Der obige Code prüft, ob der Alphatest aktiviert ist, und wenn nicht, schaltet er den Alphatest ein und setzt den Testschwellenwert auf 128 (nur Pixel mit einem Alpha-Wert über 0,5 werden gezeichnet).
