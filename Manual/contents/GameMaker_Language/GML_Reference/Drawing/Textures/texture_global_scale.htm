textur_global_skala

Mit dieser Funktion können Sie die Skalierung der Textur-Seiten beim Laden aus der WAD-Datei, die Teil Ihrer endgültigen Spiel-Executable ist, steuern. Der Eingabewert muss eine Zweierpotenz sein und funktioniert so, dass:

  • Um die Texturseiten auf 1:1 zu setzen, verwenden Sie texture_global_scale(1)
  • Um die Texturen ständig zu halbieren, verwenden Sie texture_global_scale(2)
  • Um sie zu vierteln, verwenden Sie texture_global_scale(4), usw..

Auf diese Weise können Sie die Speichernutzung von Texturseiten auf Plattformen mit wenig Speicher besser kontrollieren. Beachten Sie, dass Textur-Seiten bei Bedarf aus dem WAD erstellt werden, so dass Sie dies entweder zu Beginn des Spiels aufrufen können, um alle Texturen skaliert zu laden, oder Sie Sie können sie zu bestimmten Zeiten aufrufen, um bestimmte Texturen zu laden; in diesem Fall sollten Sie wahrscheinlich draw_texture_flush() vor dem Aufruf dieser Funktion aufrufen und dann die Pre-Fetch-Funktionen verwenden, um die richtigen Textur-Seiten in den Speicher zu bekommen.

Syntax:

texture_global_scale(pow2integer);

Argument Beschreibung
pow2integer Der zu verwendende Skalierungsfaktor (1, keine Skalierung, 2, halbe Skalierung, 4, viertel Skalierung, usw...)

Rückgabe:

N/A

Beispiel:

draw_texture_flush();
texture_global_scale(2);
sprite_prefetch(spr_Trees);

Der obige Code spült alle Texturen aus dem Speicher, setzt dann die Textur-Skalierung auf 2 (so dass die Textur-Seiten halb so groß sind) und holt schließlich die Textur-Seite vor, in der das Sprite "spr_Trees" enthalten ist.
