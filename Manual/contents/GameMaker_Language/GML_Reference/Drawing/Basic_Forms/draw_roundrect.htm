draw_roundrect

Mit dieser Funktion können Sie entweder einen Umriss eines abgerundeten Rechtecks oder ein gefülltes abgerundetes Rechteck zeichnen, wobei die (x1,y1)-Position die obere linke Ecke und die (x2,y2)-Position die untere rechte Ecke ist. Sie können festlegen, wie genau das Zeichnen der Ecken ist, können Sie mit der Funktion draw_set_circle_precision()festlegen, aber diese Funktion verwendet einen festen Radius für sie (sollten Sie den Eckenradius ändern müssen, verwenden Sie die Funktion draw_roundrect_ext()). Bitte beachten Sie, dass das zu zeichnende Rechteck aufgrund von Unterschieden zwischen den verschiedenen unterstützten Plattformen möglicherweise andere Werte (+/-1 auf x, y oder Breite oder Höhe) benötigt, um mit den gewünschten Abmessungen gezeichnet zu werden.

HINWEIS: Wenn Sie eine Form mit einem Shader zeichnen wollen, sollten Sie wissen, dass die meisten Shader die folgenden Eingaben erwarten: Scheitelpunkt, Textur, Farbe. Wenn Sie diese Funktion verwenden, werden jedoch nur Scheitelpunkt- und Farbdaten übergeben, so dass der Shader möglicherweise nichts zeichnen (oder etwas zeichnen, aber nicht korrekt). Wenn Sie Formen auf diese Weise zeichnen müssen, sollte der Shader entsprechend angepasst werden.

Syntax:

draw_roundrect(x1, y1, x2, y2, outline);

Argument Beschreibung
x1 Die x-Koordinate der oberen linken Ecke des abgerundeten Rechtecks (als ob es ein normales Rechteck wäre).
y1 Die y-Koordinate der oberen linken Ecke des abgerundeten Rechtecks (als ob es ein normales Rechteck wäre).
x2 Die x-Koordinate der unteren rechten Ecke des abgerundeten Rechtecks (als ob es ein normales Rechteck wäre).
y2 Die y-Koordinate der unteren rechten Ecke des abgerundeten Rechtecks (als ob es ein normales Rechteck wäre).
Umriss Ob das abgerundete Rechteck ausgefüllt (false) oder als ein Pixel breite Kontur (true) gezeichnet wird.

Rückgabe:

N/A

Beispiel:

draw_set_colour(c_lime);
draw_roundrect(50, 100, 250, 200, 0);

Dadurch wird ein gefülltes, abgerundetes Rechteck gezeichnet, dessen theoretische (vor der Krümmung) obere linke Ecke bei (50,100) und dessen theoretische untere rechte Ecke bei (250,200) liegt.
