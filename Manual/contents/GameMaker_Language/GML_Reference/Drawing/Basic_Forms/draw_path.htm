Pfad_zeichnen

Mit dieser Funktion können Sie GameMaker Studio 2 dazu bringen, einen Pfad auf den Bildschirm zu zeichnen. Der Pfad wird als einfache Linie gezeichnet und kann entweder relativ zur aufrufenden Instanz oder an der absoluten Position sein, an der er im Pfad-Editor oder durch Code. Diese Funktion ist äußerst nützlich beim Debuggen von dynamischen Pfaden (z. B. solchen, die für Instanzen mit der mp_grid_path() Funktion).

HINWEIS: Wenn Sie eine Form mit einem Shader zeichnen wollen, sollten Sie wissen, dass die meisten Shader die folgenden Eingaben erwarten: Scheitelpunkt, Textur, Farbe. Wenn Sie diese Funktion verwenden, werden jedoch nur Scheitelpunkt- und Farbdaten übergeben, so dass der Shader möglicherweise nichts zeichnen (oder etwas zeichnen, aber nicht korrekt). Wenn Sie Formen auf diese Weise zeichnen müssen, sollte der Shader entsprechend angepasst werden.

Syntax:

draw_path(path, x, y, absolute);

Argument Beschreibung
Pfad Der zu zeichnende Pfad
x Die x-Koordinate, an der der Pfad gezeichnet wird
y Die y-Koordinate, an der der Pfad gezeichnet wird
absolut Ob der Pfad an der absoluten Position(true) oder an der relativen Position(false) gezeichnet wird

Rückgabe:

N/A

Beispiel:

if mp_grid_path(grid, path, x, y, obj_Player.x, obj_Player.y, 1)
{
draw_path(path, x, y, false);
}

Der obige Code verwendet die Funktion mp_grid_path, um einen Pfad zu erzeugen und ihn in der Variablen "path" zu speichern. Wenn der Pfad erfolgreich erstellt wurde, wird er dann auf dem Bildschirm an einer Position relativ zu der Instanz gezeichnet, die den Code ausführt.
