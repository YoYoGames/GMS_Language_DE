oberfläche_verkleinern

Diese Funktion ändert die Größe einer Oberfläche auf die angegebenen Abmessungen (in Pixel). Die "surface_id" ist die einer Oberfläche, die Sie zuvor erstellt haben, oder die application_sufaceund die Funktion ändert die Größe dieser Fläche. Beachten Sie, dass dadurch der Inhalt der Fläche weder beschnitten noch gestreckt wird, sondern dass die aktuelle Fläche zerstört und mit demselben Handle (surface_id) mit den neuen Abmessungen neu erzeugt wird, d. h. sie muss gelöscht und erneut gezeichnet werden muss (es sei denn, es handelt sich um die application_surface, in diesem Fall erledigt GameMaker Studio 2 dies automatisch).

HINWEIS: Wenn Sie die Größe der Anwendungsoberfläche ändern, werden diese Änderungen erst zu Beginn des nächsten Zeichnungsrahmens registriert, was bedeutet, dass der Aufruf der Funktion surface_get_width() oder surface_get_height() Funktionen in demselben Ereignis oder Schritt die vorherigen Werte zurückgeben.

Syntax:

surface_resize(surface_id, w, h);

Argument Beschreibung
oberfläche_id Die ID der zu ändernden Oberfläche.
w Die Breite der neuen Fläche.
h Die Höhe der neuen Oberfläche.

Rückgabe:

N/A

Beispiel:

if view_wport[0] != surface_get_width(application_surface) || view_hport[0] != surface_get_height(application_surface)
{
surface_resize(application_surface, view_wport[0],view_hport[0]);
}

Der obige Code prüft die Größe des Ansichtsports mit der der Oberfläche "aplication_surface" und wenn sie unterschiedlich ist, wird die Oberfläche neu dimensioniert.
