surface_exists

Diese Funktion ist bei der Arbeit mit Oberflächen aufgrund ihrer flüchtigen Natur unerlässlich. Oberflächen werden immer im Texturspeicher gehalten, was bedeutet, dass sie von einem Moment auf den anderen zerstört werden können (z. B. wenn ein Bildschirmschoner bei Fenstern gestartet wird, oder wenn sie auf einem Android-Gerät minimiert werden), daher sollten Sie immer prüfen, ob eine Oberfläche existiert, bevor Sie etwas mit ihr tun (dazu gehört auch das Zeichnen auf dem Bildschirm). Der Beispielcode unten zeigt eine typische Verwendung dieses Befehls im Draw-Ereignis einer Instanz um zu prüfen, ob eine Oberfläche vorhanden ist, und sie neu zu erstellen, wenn sie entfernt wurde (beachten Sie, dass die Oberfläche ursprünglich im Ereignis "create" des Objekts erstellt wurde).

Syntax:

surface_exists(surface_id);

Argument Beschreibung
oberfläche_id Die ID der zu prüfenden Oberfläche.

Rückgabe:

Boolesche

Beispiel:

if !surface_exists(surf)
{
surf = surface_create(1024, 1024);
}
sonst
{
if view_current = 1
{
draw_surface(oberfläche,0,0);
}
}

Der obige Code prüft, ob eine in der Variablen "surf" indizierte Oberfläche existiert, und wenn nicht, wird sie neu erstellt. Wenn sie vorhanden ist, wird geprüft, welche Ansicht gerade gezeichnet wird, und wenn es Ansicht[1] ist, wird die Oberfläche gezeichnet.
