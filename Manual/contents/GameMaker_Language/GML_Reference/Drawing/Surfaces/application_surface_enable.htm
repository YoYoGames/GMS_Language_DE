application_surface_enable

Mit dieser Funktion können Sie die Anwendungsoberfläche aktivieren oder deaktivieren. Standardmäßig ist die Anwendungsoberfläche aktiviert und alle Zeichnungen für die Draw-Ereignisse werden auf diese und die auf den Bildschirm gezeichnete Oberfläche ausgeführt. Auf einigen älteren Geräten und bestimmten Chipsätzen kann dies jedoch zu einer schlechteren Leistung führen. In diesen Fällen können Sie das Zeichnen in die Anwendungsoberfläche deaktivieren und alle Zeichenereignisse werden dann direkt in den Bildschirmpuffer gezeichnet.

Sie können diese Funktion verwenden, um die Anwendungsoberfläche jederzeit in Ihrem Spiel zu aktivieren oder zu deaktivieren, aber Sie sollten beachten, dass es mindestens einen Frame (Schritt) braucht, damit die Änderungen wirksam werden. Ausschaltenschaltet die gesamte Seitenverhältniskorrektur aus., wie in den Spieloptionen für die Zielplattform eingestellt. Das bedeutet, dass Sie eine Methode haben sollten, die das Spiel auf die gewünschten Geräte-, Fenster- oder Bildschirmabmessungen skaliert. Wenn Sie es wieder einschalten, werden die Spieloptionen im nächsten Schritt wieder wirksam.

Es wird empfohlen, diese Funktion sorgfältig zu verwenden, da sie sich darauf auswirkt, wie alles auf dem Bildschirm gezeichnet wird. Im Allgemeinen sollten Sie sie als Option in einem Menü Ihres Spiels haben oder sie nur einmal zu Beginn Ihres Spiels aufrufen und Ihren Code so schreiben, dass er anzeigt, ob die Funktion aktiviert oder deaktiviert ist.

Syntax:

application_surface_enable(enable);

Argument Beschreibung
aktivieren. Legt fest, ob die Anwendungsoberfläche aktiviert(true) oder deaktiviert(false) werden soll.

Rückgabe:

N/A

Beispiel:

if keyboard_check_pressed(vk_space)
{
if application_surface_is_enabled()
{
application_surface_enable(false);
}
sonst
{
application_surface_enable(true);
}
}

Der obige Code prüft auf einen Tastendruck und schaltet die Anwendungsoberfläche je nach Zustand ein oder aus (wie in einem Optionsmenü).
