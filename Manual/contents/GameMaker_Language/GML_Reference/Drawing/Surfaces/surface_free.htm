Oberfläche_frei

Wenn Sie mit Flächen arbeiten, sollten Sie diese Funktion immer dann verwenden, wenn Sie die Flächen nicht mehr benötigen. Flächen belegen Speicherplatz und müssen daher entfernt werden, normalerweise am Ende eines Raums, aber es kann jederzeit geschehen, je nachdem, wie Verwendung, der Sie sie unterzogen haben. Wenn Sie dies nicht tun, kann es zu Speicherlecks kommen, die schließlich Ihr Spiel verlangsamen und zum Absturz bringen.

HINWEIS: Bei der Arbeit mit Flächen besteht die Möglichkeit, dass diese jederzeit nicht mehr existieren, da sie im Texturspeicher abgelegt sind. Sie sollten daher IMMER prüfen, ob eine Fläche existiert, indem Sie surface_exists() überprüfen, bevor Sie sie direkt referenzieren.

Syntax:

surface_free(surface_id);

Argument Beschreibung
oberfläche_id Die ID der freizugebenden Fläche.

Rückgabe:

N/A

Beispiel:

if keyboard_check_pressed(vk_escape)
{
surface_free(surf);
room_goto(rm_Menu);
}

Der obige Code prüft auf einen Tastendruck und wenn er einen erkennt, gibt er den Speicher frei, der für die in der Variablen "surf" indizierte Oberfläche reserviert ist, und wechselt dann den Raum.
