physics_fixture_set_awake

Wenn Sie einen Raum mit Physik starten oder Fixtures an Instanzen binden, werden die Fixtures immer als "wach" betrachtet... das heißt, sie verarbeiten Events und interagieren mit den umgebenden Instanzen. Dies kann jedoch manchmal zu Problemen führen zu Problemen führen, besonders wenn Sie mehrere Instanzen mit Fixtures haben, die nebeneinander stehen, wenn ein Raum startet (denken Sie an einen Turm, der aus verschiedenen Instanzen mit Fixtures besteht), da sie sich, wenn sie "wach" sind, bewegen und möglicherweise ihre Position ändern werden aufgrund des plötzlichen Einsetzens der Schwerkraft und der Kollisionen, die auf sie einwirken. In diesen Fällen können Sie diese Funktion verwenden, um das Fixture als "schlafend" zu kennzeichnen, in diesem Fall wird es nichts verarbeiten, wenn es zum ersten Mal im Raum erstellt wird.

Syntax:

physics_fixture_set_awake(fixture, state)

Argument Beschreibung
Halterung der Index der Halterung
Flagge ob ein Gerät wach ist(true) oder nicht(false)

Rückgabe:

N/A

Beispiel:

physics_fixture_set_awake(fix_Cloud, true);

Der obige Code kennzeichnet das Fixture als "wach", wenn es an eine Instanz gebunden ist.
