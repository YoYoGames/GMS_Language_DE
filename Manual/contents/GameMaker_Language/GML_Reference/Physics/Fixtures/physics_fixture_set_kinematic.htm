physics_fixture_set_kinematic

In einem physikbasierten Spiel wird es bestimmte Situationen geben, in denen Sie sich bewegen möchten, aber nicht möchten, dass Kräfte wie die Schwerkraft oder Kräfte, die durch Kollisionen mit dynamischen Objekten entstehen, auf sie einwirken (denken Sie z. B. an sich bewegende Plattformen in einem Plattformspiel). Beispiel). Wenn Sie bei solchen Objekten einfach die Dichte eines Fixtures auf 0 setzen, geht die Physik davon aus, dass das Objekt statisch sein soll. Wenn Sie jedoch den Wert phy_Geschwindigkeit_x, phy_Geschwindigkeit_y oder phy_angular_velocity Variablen für eine Instanz setzen, an die ein solches Fixture gebunden wurde, wird es in ein in ein kinematisches Objekt umgewandelt und beginnt sich zu bewegen. Das Ändern der Instanz von statisch zu kinematisch mitten in der Simulation kann verarbeitungsintensiv sein (abhängig vom Zustand der Instanzen in der Physikwelt), daher wird empfohlen, die Fixture mit dieser Funktion kinematisch zu machen, bevor es an eine Instanz gebunden wird, so dass es von Kollisionen und Kräften (z. B. Schwerkraft) unbeeinflusst bleibt, sich aber mit konstanter linearer und/oder Winkelgeschwindigkeit bewegen kann.

Syntax:

physics_fixture_set_kinematic(fixture)

Argument Beschreibung
Halterung der Index der Halterung

Rückgabe:

N/A

Beispiel:

physics_fixture_set_kinematic(fix_Cloud);

Der obige Code zeigt an, dass das in "fix_Cloud" indizierte Fixture als kinematisch markiert werden soll.
