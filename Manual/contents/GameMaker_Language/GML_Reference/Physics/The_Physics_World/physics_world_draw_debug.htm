physics_world_draw_debug

Diese Funktion zeichnet eine Darstellung des Physik-Weltsystems des Raums, um bei der Fehlersuche zu helfen, wenn Sie Probleme haben, die Physikszene richtig zum Laufen zu bringen. Manchmal kann das Problem im Rendering-Teil Ihres Spiels liegen, zum Beispiel ein Sprite kann an der falschen Position oder mit der falschen Drehung gezeichnet werden, was den Anschein einer falschen Physik erweckt, oder es kann sein, dass Ihre Berechnungen nicht korrekt sind. In jedem Fall hilft Ihnen diese Funktion, das Problem zu finden und zu beheben, indem sie zeigt, was in Ihrem Physiksystem vor sich geht. in Ihrem Physiksystem vor sich geht. Dazu müssen Sie ihr mitteilen, welcher Teil des Systems gezeichnet werden soll, und dies geschieht durch die Übergabe eines Wertes, der aus den in verschiedenen "Flag"-Konstanten gespeicherten Bits erstellt wird:

Argument Beschreibung
phy_debug_render_aabb Dies zeigt die absolute Bounding Box jedes Geräts in Bezug auf die Raumachse
phy_debug_render_collision_pairs Dies zeigt alle Geräte an, die sich gerade in einer Kollision befinden
phy_debug_render_coms Dies markiert den Massenschwerpunkt jedes Geräts im Raum
phy_debug_render_core_shapes Zeigt die Grundformen, aus denen sich die Einbauten im Raum zusammensetzen
phy_debug_render_joints Dadurch werden die Verbindungen aller Einbauten im Raum gezeichnet
phy_debug_render_obb Dies zeigt den relativen Begrenzungsrahmen für die Einbauten im Raum
phy_debug_render_shapes Dies zeigt die tatsächlichen Formen, aus denen die Einbauten im Raum bestehen

Um also zwei dieser Optionen zu zeichnen, müssen wir ein bitweises "oder" (in GameMaker Studio 2 wird dies durch | dargestellt) mit den beiden Optionen durchführen und die resultierende Variable für die zu verwendende Funktion speichern. Hier ist ein Beispiel dafür, wie Sie eine Variable für die Funktion setzen, die nur die Massenschwerpunkte und die Gelenke aller Einbauten in einem Raum zeichnet:

flag = phy_debug_render_coms | phy_debug_render_shapes;

Wie Sie mit dem obigen Code sehen können, werden nur die Formen und Massenschwerpunkte auf dem Bildschirm gezeichnet (sowie Farben, die ihre Zustände repräsentieren - für weitere Informationen sehen Sie bitte in der Box 2D-Dokumentation nach). Auf dieselbe Weise können Sie mehr oder sogar alle Konstanten anzeigen, die Sie debuggen möchten... Verknüpfen Sie einfach alle mit "oder"! Eine weitere Sache sollte beachtet werden - diese Funktion sollte nur im Draw-Ereignis einer Instanz aufgerufen werden, die eine geringere Tiefe hat als die, die Sie debuggen möchten, sonst werden Sie die Debug-Informationen nicht sehen können.

Syntax:

physics_world_draw_debug(flag)

Argument Beschreibung
Flagge Der Flag-Wert zum Anzeigen verschiedener Debug-Funktionen

Rückgabe:

N/A

Beispiel:

flag = phy_debug_render_aabb | phy_debug_render_collision_pairs | phy_debug_render_obb;
physics_world_draw_debug(flag);

Der obige Code zeichnet alle absoluten Boundingboxen, die relativen Boundingboxen sowie die Fixtures in Kollision innerhalb des aktuellen Physiksystems.
