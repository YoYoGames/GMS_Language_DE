physik_welt_aktualisieren_Geschwindigkeit

GameMaker Studio 2 aktualisiert die Dinge basierend auf der Raumgeschwindigkeit, d. h. wenn Sie diese auf z. B. 30 einstellen, läuft GameMaker Studio 2 30 Schritte im Laufe einer Sekunde. Für die Physikfunktionen reicht das jedoch möglicherweise nicht aus, und Sie möchten vielleicht, dass die Dinge mit einer etwas schnelleren Geschwindigkeit aktualisiert werden, um die Stabilität oder Präzision zu erhöhen. Zu diesem Zweck verwenden wir die Funktion physics_world_update_speed(), die die Aktualisierungsgeschwindigkeit für das Physiksystem unabhängig von der Raumgeschwindigkeit einstellt. Das bedeutet dass Sie eine Raumgeschwindigkeit von 30 haben können, aber die Physik auf 60 setzen, wodurch sich die Geschwindigkeit, mit der das Physiksystem aktualisiert und seine Berechnungen durchführt, im Vergleich zur Geschwindigkeit, mit der die Schritte aktualisiert werden, effektiv verdoppelt.

HINWEIS: Sie können dies derzeit nicht auf eine geringere als die Raumgeschwindigkeit einstellen, aber zukünftige Updates könnten dies ändern.

Syntax:

physics_world_update_speed(Geschwindigkeit)

Argument Beschreibung
Geschwindigkeit die Anzahl der Aktualisierungen pro Sekunde, die das Physiksystem vornimmt

Rückgabe:

N/A

Beispiel:

physics_world_update_speed(room_speed * 2);

Der obige Code setzt die Aktualisierungsgeschwindigkeit des Physiksystems auf das Doppelte der Raumgeschwindigkeit.
