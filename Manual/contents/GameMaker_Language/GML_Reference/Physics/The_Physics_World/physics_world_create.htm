physik_welt_erschaffen

Selbst wenn Sie die Physik für Ihre Objekte definiert haben, werden sie ohne eine Welt, in der sie platziert werden, standardmäßig das traditionelle GameMaker Studio 2-Kollisionssystem verwenden, sodass Sie diese Funktion benötigen, um das Physiksystem für den angegebenen Raum entweder in einem Objektereignis oder im Erstellungscode des Raums selbst (es kann auch ohne Code im Raumeditor definiert werden). Diese Funktion verknüpft eine Physik-"Welt" mit dem Raum und alle Instanzen, die in dem Raum erzeugt werden, verhalten sich mit der Physik, die für sie und die Welt selbst definiert wurde.

Sie werden feststellen, dass wir ein Argument haben, das ein Verhältnis zwischen Pixelgröße und Metern definiert. Das liegt daran, dass die Physikfunktionen mit realen Maßen arbeiten und wir daher Pixelkoordinaten und Pixelgrößen mit Hilfe eines "Pixel-Meter-Maßstabs" in diese realen Maße übersetzen müssen. Sie sollten die Einstellung "Pixel-Meter-Skala" so lange anpassen, bis die durchschnittliche Pixelgröße der von Ihnen verwendeten Objekte grob in simulierte Physikobjekte einer angemessenen Größe übersetzt wird. Es sollte beachtet werden dass das Physiksystem sowohl mit größeren als auch mit kleineren Verhältnissen funktioniert, dass aber die beste Leistung erzielt wird, wenn die realen Abmessungen Ihrer Objekte nicht kleiner als 0,1 Meter und nicht größer als 50 Meter sind.

Syntax:

physics_world_create(pixeltometrescale);

Argument Beschreibung
pixeltometrische Skala definiert das Verhältnis von Pixeln auf dem Bildschirm zu Metern in der realen Welt. Ein Verhältnis von 32:1 wird als 1/32 (oder 0,03125) angegeben.

Rückgabe:

N/A

Beispiel:

physics_world_create(1/32);

Der obige Code erzeugt eine Physikwelt im aktuellen Raum mit einem Maßstabsverhältnis von 32 Pixeln, was 1 Meter entspricht.
