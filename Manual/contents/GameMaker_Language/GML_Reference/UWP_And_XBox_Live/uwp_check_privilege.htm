uwp_check_privilege

Mit dieser Funktion können Sie prüfen, ob ein bestimmter Benutzer ein gewähltes Privileg hat. Die Funktion gibt im asynchronen Systemereignis true oder false zurück, je nachdem, ob das Privileg aktiviert ist oder nicht, und wenn Sie das Argument attempt_resolution auf true setzen und das Privileg nicht aktiviert ist, wird außerdem ein Systemdialog geöffnet (das Spiel wird angehalten), um den Benutzer aufzufordern, das Konto zu aktualisieren oder was auch immer erforderlich ist, um das Privileg zu erhalten. Wenn der Benutzer dann die erforderliche Option aktiviert, gibt die Funktion wahr zurück.

Die Funktion erfordert, dass Sie eine der folgenden Konstanten für die Berechtigungsabfrage verwenden:

Konstant Beschreibung
uwp_privileg_internet_browsing Prüfen Sie, ob das Surfen im Internet erlaubt ist
uwp_privileg_social_network_sharing Prüfen Sie, ob die Weitergabe an soziale Netzwerke erlaubt ist
uwp_privilege_share_kinect_content Prüfen, ob die gemeinsame Nutzung des Kinect-Controllers erlaubt ist
uwp_privileg_video_kommunikation Prüfen Sie, ob Videokommunikation erlaubt ist
uwp_privileg_kommunikation Prüfen Sie, ob die Internetkommunikation erlaubt ist
uwp_privilege_user_created_content Prüfen, ob der Zugriff auf vom Benutzer erstellte Inhalte erlaubt ist
uwp_privileg_multiplayer_sessions Prüfen Sie, ob Online-Multiplayer-Sitzungen erlaubt sind
uwp_privilege_sessions Prüfen Sie, ob Online-Sitzungen erlaubt sind
uwp_privileg_fitness_upload Prüfen Sie, ob das Hochladen von Fitnessdaten erlaubt ist

Sobald die Funktion aufgerufen wurde, löst sie auf der XBox One ein asynchrones Systemereignis aus, wenn das Ergebnis des Rückrufs empfangen wurde, das die spezielle DS Karte async_load. Diese Map sollte dann nach dem folgenden Schlüssel geparst werden:

  • " event_type" - sollte den Wert "check_privilege_result" enthalten, wenn das Ereignis durch diese Funktion ausgelöst wurde

Wenn sich der Ereignistyp auf diese Funktion bezieht, dann gibt es beim Aufruf der Funktion an der XBox auch folgende zusätzliche Tasten:

  • "result" - Wird eine oder mehrere der folgenden Konstanten sein (kombinierte Werte):
    Konstante Beschreibung Wert
    uwp_privilege_result_no_issue Es gibt keine Berechtigungsprobleme mit dem Benutzer 0
    uwp_privilege_result_purchase_required Der Benutzer muss etwas Zusätzliches, meist ein Abonnement, für den Zugriff erwerben 1
    uwp_privilege_result_aborted Die Prüfung wurde abgebrochen 2
    uwp_privilege_result_banned Der Benutzer ist gesperrt 4
    uwp_privilege_result_restricted Der Benutzer ist vom Zugriff ausgeschlossen, normalerweise durch die Kindersicherung 8

  • "Privileg" - wird eine der oben aufgeführten Konstanten für die Verwendung in der Funktion sein, damit Sie bei Mehrfachprüfungen prüfen können, welches Privileg Sie angefordert haben.

Wenn das Projekt auf einer UWP-Plattform ausgeführt wird, bei der es sich nicht um die XBox One handelt, wird das Async-Ereignis nicht ausgelöst, und stattdessen gibt die Funktion sofort entweder -1, 0 oder 1 zurück:

  • -1 bedeutet -1 der Benutzer hat das Recht nicht
  • 0 bedeutet, dass die Funktion den Benutzer nicht abfragen konnte
  • 1 bedeutet, dass der Benutzer die Berechtigung hat.

Syntax:

uwp_check_privilege(user_id, privilege, attempt_resolution);

Argument Beschreibung
user_id Der zu prüfende Benutzer-ID-Zeiger.
Privileg Die zu prüfende Privilegkonstante (siehe Beschreibung, oben).
Versuch_Auflösung Aktivieren Sie den Auflösungsversuch, wenn auf true gesetzt, oder ignorieren Sie ihn, wenn auf false gesetzt.

Rückgabe:

Integer

Beispiel:

var user_one = xboxlive_get_user(0);
if (xboxlive_user_is_signed_in(user_one))
{
uwp_check_privilege(user_one, xboxlive_privilege_multiplayer_sessions, true);
}

Oben wird geprüft, ob der Benutzer angemeldet ist, und wenn ja, wird die Berechtigung für Multiplayer-Sitzungen geprüft.
