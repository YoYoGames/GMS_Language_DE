xboxlive_get_savedata_user

Diese Funktion gibt eine Konstante zurück (siehe unten), die Ihnen bei der Fehlersuche hilft, warum eine Dateifunktion fehlgeschlagen ist, so dass Sie entsprechende Maßnahmen in Ihrem Spielcode ergreifen können. Wenn eine der Dateifunktionen fehlschlägt, können Sie die Funktion aufrufen, um genau herauszufinden, was der Fehler war.

HINWEIS: Wenn Sie die asynchronen Funktionen zum Speichern/Laden verwenden, hat die async_load map einen zusätzlichen Schlüssel, " error", der auch eine der Konstanten enthält (siehe unten).

Syntax:

xboxlive_get_file_error();

Rückgabe:

Konstant

Konstant Beschreibung
xboxlive_fileerror_outoflocalstorage Wenn dies während des Speicherns auftritt, bedeutet dies, dass der gesamte verfügbare Speicherplatz des Spiels aufgebraucht wurde und der Benutzer das (vom Betriebssystem unterbreitete) Angebot, Speicherplatz freizugeben, abgelehnt hat. Das Spiel sollte dem Benutzer bestätigen, dass er beabsichtigt, weiterzuspielen ohne zu speichern. Wenn der Benutzer immer noch speichern möchte, sollte das Spiel den Speichervorgang erneut versuchen.
xboxlive_fileerror_quotaexceeded In diesem Fall sollte das Spiel den Benutzer darauf hinweisen, dass er keine weiteren Daten speichern kann, bis er etwas Speicherplatz freigibt, und ihm dann eine Möglichkeit anbieten, dies zu tun.
xboxlive_fileerror_noerror Die Funktion zum Speichern oder Laden von Dateien wurde ohne Probleme ausgeführt.

Beispiel:

buffer_save(buff, "SaveGame.bin");
if (xboxlive_get_file_error() != xboxlive_fileerror_noerror)
{
show_debug_message("Speichern fehlgeschlagen. Fehlercode = " + string(xboxlive_get_file_error()));
}

Der obige Code prüft, ob ein Fehler beim Speichern aufgetreten ist, und wenn ja, wird der Fehlercode auf der Konsole ausgegeben.
