xboxlive_get_stats_for_user

Diese Funktion kann verwendet werden, um Daten über bestimmte Statistiken von den Xbox Live-Servern abzurufen. Sie liefern die Benutzer-ID, wie sie von der Funktion xboxlive_get_user()zurückgegeben wird, dann Ihre Spiele Service Konfigurations-ID (wie auf der XDP-Konsole angezeigt) und schließlich die gewünschten Statistiken. Sie können bis zu maximal 14 Statistiken abfragen, aber das garantiert nicht, dass Sie 14 Statistik-Ergebnisse erhalten, da die Gesamtlänge der Anfrage begrenzt ist und daher hängt die maximale Anzahl der Statistiken von der Länge der Namen der Statistiken selbst ab.

Die Funktion gibt 0 zurück, wenn die Statistikanfrage gesendet wurde, oder -1, wenn ein Fehler aufgetreten ist, und der Callback löst ein asynchrones Systemereignis aus. Dieses Ereignis hat die spezielle DS-Map async_load die dann für den folgenden Schlüssel geparst werden sollte:

  • " event_type" - sollte die Zeichenkette " stat_result" enthalten, wenndas Ereignis durch diese Funktion ausgelöst wurde

Wenn sich der Ereignistyp auf diese Funktion bezieht, dann gibt es auch den folgenden zusätzlichen Schlüssel:

  • " user"- enthält die Benutzer-ID für die angeforderte Statistik.

Es kann auch einen zusätzlichen Satz von Schlüssel-Wert-Paaren geben, wobei der Schlüssel der angeforderte Statistikname ist und der Wert der Wert für diese Statistik auf dem Server. Beachten Sie, dass, wenn eine Statistik für diesen Benutzer nie erstellt wurde (weil er nicht die Ereignisse ausgelöst hat, die gesetzt wurde), kann kein Wert für diese Statistik zurückgegeben werden (ds_map_exists() kann verwendet werden, um nach den Statistiken in der async_load-Map zu suchen).

Wenn die Anfrage aufgrund einer zu großen Anfragelänge fehlschlägt, sollte in der Konsolenausgabe eine Meldung mit Angabe des Fehlercodes erscheinen:

xboxlive_get_stats_for_user - Ausnahme beim Erhalten der Ergebnisse aufgetreten - 0x80190190

Wenn dies geschieht, sollte die async-Ereignis-DS-Map einen "succeeded"-Schlüssel mit dem Wert "0" haben, und in diesem Fall sollten Sie versuchen, weniger Statistiken in einem einzigen Aufruf abzurufen.

Syntax:

xboxlive_get_stats_for_user(user_id, serviceconfig_id, statname1, ...);

Argument Beschreibung
user_id Der Zeiger für die Benutzer-ID.
serviceconfig_id Die ID der Dienstkonfigurationsdatei
statname1 (2, 3, usw...) Die Statistiknamen, für die die Informationen abgerufen werden sollen.

Rückgabe:

Real

Beispiel:

var _uid = xboxlive_get_user(0);
var _configid = "00000000-0000-0000-0000-00000000000000";
xboxlive_get_stats_for_user(xbuser, _configid, "GameProgress", "CurrentMode");

Der obige Code ruft die Benutzer-ID ab und verwendet diese dann, um Informationen über bestimmte Statistiken für diesen Benutzer anzufordern.
