xboxlive_sprite_add_from_gamerpicture

Mit dieser Funktion können Sie das Gamer-Bild für einen gegebenen Benutzer-ID-Zeiger abrufen. Die Funktion arbeitet asynchron und löst ein asynchrones Ereignis Image Loaded Ereignis aus, um Sie zu informieren, dass die Funktion beendet ist (wie bei HTML5/URL-Aufrufen an sprite_add()). Das Ereignis "Asynchronous Image Loaded" hat eine DS-Map, die in die eingebaute Variable async_load. Die Map enthält die folgenden Informationen:

  • "Dateiname": Der vollständige Pfad zu der angeforderten Datei.
  • " id": Die ID der Ressource, die Sie geladen haben. Dies ist die gleiche wie die Variable, der Sie die Ressource zugewiesen haben.
  • "Status": Gibt bei einem Fehler einen Wert kleiner als 0 zurück.

HINWEIS: Wenn Sie ein Sprite zur Laufzeit dynamisch in Ihr Spiel laden, müssen Sie daran denken, es wieder mit sprite_delete() zu entfernen, wenn es nicht mehr benötigt wird, sonst besteht die Gefahr eines Speicherlecks, das Ihr Spiel verlangsamt und schließlich zum Absturz bringt.

Syntax:

xboxlive_sprite_add_from_gamerpicture(user_id, imagesize, xorig, yorig);

Argument Beschreibung
user_id Die Benutzer-ID (ein Zeiger) zum Abrufen des Spielerbildes für
Bildgröße Größe in Pixeln des zurückzugebenden Sprites
xorig Geben Sie die x-Position des Ursprungs im Sprite an
yorig Gibt die y-Position des Ursprungs im Sprite an

Rückgabe:

Real

Beispiel:

var _a = 0;
var _num = xboxlive_get_user_count();
for (var i = 0; i < _num; ++i;)
{
var _uid = xboxlive_get_user(i);
if _uid != pointer_null
{
global.UserName[_a] = xboxlive_gamedisplayname_for_user(_uid);
global.UserScore[_a] = xboxlive_gamerscore_for_user(_uid);
global.UserRep[_a] = xboxlive_reputation_for_user(_uid);
global.UserAvatar[_a] = xboxlive_sprite_add_from_gamerpicture(_uid, 256, 0, 0);
++a;
}
}

Der obige Code durchläuft die angemeldeten Benutzer in einer Schleife und speichert deren Spieldaten in verschiedenen globalen Arrays.
