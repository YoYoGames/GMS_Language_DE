uwp_ist_suspendierend

Diese Funktion gibt bei einem Einzelbild true zurück (und sonst false ), ähnlich wie bei os_is_paused(). Das Spiel hat dann eine Sekunde Zeit, den Spielzustand zu speichern, bevor es bevor es möglicherweise vom System abgebrochen wird. Sobald das Spiel das Speichern der Daten beendet hat, sollte es die Funktion uwp_suspend() aufrufen, um das Spiel sauber zu beenden. Das Spiel sollte wahrscheinlich auch in einen Pausenzustand übergehen, damit es bei einer Wiederaufnahme durch das System wieder aufgenommen wird (wenn es nicht vollständig beendet wurde), der Spieler nicht direkt in eine Spielsituation zurückgeworfen wird.

Syntax:

uwp_is_suspending();

Rückgabe:

Boolesche

Beispiel:

if uwp_is_suspending()
{
scr_Save_Game_Data();
uwp_suspend();
}

Der obige Code prüft, ob die App in den Schwebezustand geht, und wenn ja, ruft er ein Skript auf, um die Spieldaten zu speichern, bevor das Spiel in den Schwebezustand versetzt wird.
