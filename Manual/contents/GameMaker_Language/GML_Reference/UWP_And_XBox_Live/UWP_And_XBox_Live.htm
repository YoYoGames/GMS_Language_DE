UWPAnd XBox Live

In diesem Abschnitt werden die Funktionen beschrieben, die speziell für die Verwendung auf der UWP-Zielplattform vorgesehen sind. Der erste Satz von Funktionen dient zur Steuerung des allgemeinen Spielverhaltens beim Erstellen einer UWP-App:

  • uwp_suspend
  • uwp_ist_suspendierend
  • uwp_is_constrained
  • uwp_wurde_beendet
  • uwp_wurde_vom_Benutzer_geschlossen
  • uwp_show_help
  • uwp_lizenz_trial_version
  • uwp_license_trial_user
  • uwp_license_trial_time_remaining
  • uwp_check_privilege

Mit UWP können Sie auch Spiele für die XBox One-Plattform erstellen, indem Sie die XBox Live-API mit den folgenden Funktionen verwenden. Diese Funktionen setzen voraus, dass Sie die Option " XBox Live aktivieren " in den UWP-Spieloptionen aktiviert haben. Einige dieser Funktionen erfordern, dass der Benutzer bei XBox Live angemeldet ist, was GameMaker Studio 2 versucht, automatisch zu tun, wenn Sie das Projekt ausführen. Diese "stille" Anmeldung erzeugt ein asynchrones Systemereignis, das vor dem Fortfahren überprüft werden kann, um zu sehen, ob sie erfolgreich war oder nicht. Beachten Sie, dass im Gegensatz zum Xbox One-Konsolenexport beim UWP-Export für XBox Live nur ein Benutzer gleichzeitig angemeldet werden kann.

WICHTIG! Das UWP-Ziel verfügt nur über ein Minimum an Unterstützung für XBox Live, wenn es als Teil des Creator-Programms eingerichtet wird, das auf einige grundlegende Benutzer- und Kontofunktionen beschränkt ist (weitere Informationen zum Einrichten finden Sie hier ). Um alle hier aufgeführten Funktionen nutzen zu können, müssen Sie ein ID@XBox-Konto besitzen.

Sie finden alle verschiedenen XBox Live-Funktionen zur Verwendung mit dem UWP-Exportziel in den unten aufgeführten Abschnitten:

  • Benutzer und Konten
  • Speichern von Daten
  • Statistiken und Ranglisten
  • Match Making
