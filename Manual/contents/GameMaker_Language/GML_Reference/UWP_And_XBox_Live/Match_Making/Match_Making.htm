Match Making

Matchmaking (Multiplayer) für Xbox Live und UWP erfordert ein wenig Konfiguration, teilweise über eine Konfigurationsdatei, die definiert, welche Netzwerkverbindungen Sie verwenden werden, und teilweise über die Einrichtung von Sitzungsvorlagen und Matchmaking-Hoppern im Xbox Entwickler-Portal. GameMaker Studio 2 unterstützt dies durch die Verwendung des UWP-Targets und erlaubt Ihnen Folgendes:

  • Erstellen einer Multiplayer-Sitzung
  • Finden Sie eine Multiplayer-Sitzung (und/oder treten Sie ihr bei)
  • Benachrichtigt werden, wenn relevante Spieler einer Sitzung beitreten oder diese verlassen, sowie deren Netzwerkverbindungsdetails erhalten
  • Benachrichtigt werden, wenn der Sitzungsleiter zu einem anderen Benutzer wechselt
  • Abrufen einer Liste von Benutzern in der Sitzung
  • Eine Sitzung verlassen

Die Xbox Live-Netzwerkkonnektivität beruht auf einer so genannten Secure Device Association, die im Grunde ein Vertrag ist, der genau definiert, wie sich das Gerät mit anderen Geräten verbindet und welche Art von Daten es über diese Verbindung kommunizieren kann dieser Verbindung kommunizieren kann. Diese werden über eine Datei konfiguriert, die Sie in Ihr Projekt einbinden müssen. Ein Beispiel für diese Datei finden Sie hier. Die wichtigsten Punkte in der Datei sind die Protokoll- und Portnummern für jede Socket-Beschreibung, da diese mit dem übereinstimmen müssen, was Sie tatsächlich im Code einrichten.

Sobald Sie diese Netzwerkmanifestdatei erstellt haben, sollte sie dem Projekt als eingeschlossene Datei hinzugefügt werden. Dann sollte sie in den UWP-Spieloptionen auf der Registerkarte Xbox One Allgemein im Kombinationsfeld "Enthaltene Datei für Netzwerkkonfigurationsmanifest" ausgewählt werden.

Um eine Multiplayer-Sitzung zu erstellen, müssen Sie angeben, welche Sitzungsvorlage verwendet werden soll. Diese Sitzungsvorlagen werden in der XDP definiert, aber ein Beispiel wird unten gezeigt:

Sitzungsname: MatchSession
Vertragsversion: 107
Sitzungsvorlage (JSON-Text):
{
"constants": {
"system": {
"version": 1,
"maxMembersCount": 12,
"capabilities": {
},
"memberInitialization": {
"externalEvaluation": false,
"membersNeededToStart": 1
}
},
"custom": {}
}
}

Eine Beschreibung der Felder finden Sie unter "Sitzungsübersicht" in der XDK-Hilfedatei. Beachten Sie, dass wir das dynamische Setzen von Sitzungsparametern aus GameMaker Studio 2 derzeit nicht unterstützen .

Zusammen mit der Sitzungsvorlage muss auch ein Matchmaking-Hopper eingerichtet werden, damit die Sitzung gefunden werden kann (dies geschieht ebenfalls in der XDP). Hopper werden verwendet, um Regeln zu definieren, die beim Matchmaking von Spielern verwendet werden. Wie bei den Sitzungsparametern unterstützen wir nicht Wie bei den Sitzungsparametern unterstützen wir derzeit nicht das dynamische Setzen von Hopper-Parametern von GameMaker: Studio. Weitere Informationen finden Sie im Abschnitt "SmartMatch Matchmaking-Konfiguration" in der XDK-Hilfedatei.

Für das XboxLive-Matchmaking stehen die folgenden Funktionen zur Verfügung (beachten Sie, dass Sie für die Verbindungen auch die GameMaker Studio 2 Netzwerkfunktionen verwenden müssen etc...):

  • xboxlive_matchmaking_start
  • xboxlive_matchmaking_stop
  • xboxlive_matchmaking_create
  • xboxlive_matchmaking_find
  • xboxlive_matchmaking_session_get_users
  • xboxlive_matchmaking_join_session
  • xboxlive_matchmaking_session_leave

GameMaker Studio 2 bietet einen zusätzlichen Satz von Matchmaking-Funktionen, mit denen Sie andere Spieler im Netzwerk einladen können, an einer Sitzung teilzunehmen, oder mit denen Sie eine Einladung annehmen können, an einer Sitzung eines anderen Spielers teilzunehmen. Die verfügbaren Funktionen sind:

  • xboxlive_matchmaking_send_invites
  • xboxlive_matchmaking_set_joinable_session
  • xboxlive_matchmaking_join_invite

Wenn der Spieler eine Einladung eines anderen Spielers annimmt, geschieht eines von zwei Dingen:

  • Wenn Ihr Spiel gerade läuft, erhalten Sie ein asynchrones Ereignis mit der DS-Map async_load, die die folgenden Schlüssel/Werte enthält:
    • " id" - Dieser Wert wird auf MATCHMAKING_INVITATION gesetzt.
    • "invitation_id"- Dies ist der eindeutige Sitzungs-ID-Wert.
    • "invitation_host"- Dies ist der eindeutige Sitzungs-ID-Wert für den Sitzungsleiter
    Sie sollten die ID- und Host-Werte speichern, dann alle Sitzungen verlassen, die Sie benötigen, und die Funktion xboxlive_matchmaking_join_invite() aufrufen, wobei Sie die Werte für die Einladungs-ID und den Einladungshost in einem Step-Event als Argumente übergeben.
  • Wenn Ihr Spiel derzeit nicht läuft, wird es vom Betriebssystem gestartet. Sie erhalten den gleichen Typ von Social Asynchronous Event mit den gleichen Schlüsseln, die oben aufgeführt sind. Speichern Sie sie in Variablen, verbinden Sie sich dann mit Xbox Live Matchmaking und rufen schließlich xboxlive_matchmaking_join_invite() auf.

Sobald Sie xboxlive_matchmaking_join_invite() aufgerufen haben, versucht der Benutzer, der Sitzung beizutreten, und Sie erhalten asynchrone soziale Ereignisse, als ob Sie xboxlive_matchmaking_find() für diese Sitzung aufgerufen hätten dieser Sitzung aufgerufen hätten.
