DS-Stapel

Ein DS-Stack ist eine sogenannte Last-in-First-out (LIFO)-Datenstruktur, ähnlich einer DS-Warteschlange. Sie können Werte auf einen Stapel schieben und dann wieder entfernen, indem Sie sie vom Stapel stoßen, wobei der Wert der zuletzt auf den Stapel geschoben wurde, als erster wieder vom Stapel gepoppt wird (denken Sie an einen Münzstapel, bei dem jede Münze, die Sie hinzufügen, zuerst wieder entfernt werden muss, bevor Sie an die restlichen Münzen darunter gelangen können). Stapel werden oft verwendet wenn Unterbrechungen zu handhaben sind, oder wenn Sie rekursive Funktionen haben, oder sogar wenn Sie eine rudimentäre KI für Ihre Spiele konstruieren.

HINWEIS: Wie alle dynamischen Ressourcen belegen auch Datenstrukturen Speicherplatz und sollten daher immer zerstört werden, wenn sie nicht mehr benötigt werden, um Speicherlecks zu vermeiden, die Ihr Spiel verlangsamen und schließlich zum Absturz bringen.

HINWEIS: Wenn Sie prüfen müssen, ob eine Datenstruktur existiert, können Sie die ds_exists() Funktion verwenden.

Es gibt die folgenden Funktionen, die sich mit DS-Stapeln befassen.

  • ds_stack_create
  • ds_stack_destroy
  • ds_stack_clear
  • ds_stack_empty
  • ds_stack_size
  • ds_stack_copy
  • ds_stack_top
  • ds_stack_pop
  • ds_stack_push
  • ds_stack_read
  • ds_stack_write
