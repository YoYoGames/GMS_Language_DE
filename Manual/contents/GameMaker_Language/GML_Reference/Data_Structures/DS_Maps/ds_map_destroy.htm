ds_map_destroy

DS-Maps belegen Platz im Speicher, der ihnen bei der Erstellung zugewiesen wird. Das bedeutet, dass Sie diesen Speicher auch wieder freigeben müssen, wenn die DS-Map nicht benötigt wird, um Fehler, Speicherlecks und Leistungseinbußen beim Ausführen Ihres Spiels zu vermeiden. Diese Funktion erledigt genau das. Beachten Sie, dass das Zerstören einer Map alle Datenstrukturen, die in der Map gespeichert sind, de-referenziert, was zu einem Speicherleck führt. Die Dies ist nur dann nicht erforderlich, wenn Sie Elemente in der Map als DS-Liste oder als eine andere DS-Map gekennzeichnet haben; in diesem Fall werden diese Elemente automatisch zerstört und ihr Speicher bereinigt. auch automatisch gelöscht.

WICHTIG! Wenn Sie eine Datenstruktur anlegen, ist der Indexwert zu ihrer Identifizierung ein Integer-Wert, der bei 0 beginnt. Das bedeutet, dass Datenstrukturen unterschiedlichen Typs denselben Indexwert haben können, daher sollten Sie im Zweifelsfall die ds_exists() Funktion verwenden, bevor Sie auf sie zugreifen. Beachten Sie auch, dass Indizes wiederverwendet werden, so dass ein zerstörter Datenstruktur-Indexwert anschließend von einer neu erstellten verwendet werden kann, daher empfehlen wir Wir empfehlen daher, die Variable, die den DS-Index enthielt, nach dem Zerstören immer auf -1 zu setzen.

Syntax:

ds_map_destroy(id);

Argument Beschreibung
id Die ID der zu zerstörenden Karte.

Rückgabe:

N/A

Beispiel:

ds_map_destroy(inventory);
inventory = -1;

Der obige Code zerstört die DS-Map mit der in der Variablen "inventory" indizierten ID.
