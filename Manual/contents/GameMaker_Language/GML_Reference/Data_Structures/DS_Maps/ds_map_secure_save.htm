ds_map_secure_save

Diese Funktion speichert den Inhalt der angegebenen DS-Map in einer Datei, die mit dem Gerät verknüpft ist, auf dem sie erstellt wurde (d. h. sie kann nicht gelesen werden, wenn sie auf ein anderes Gerät übertragen wird). Die Datei selbst kann fast jede beliebige Erweiterung haben (z. B. *.dat, *.json, *.bin, etc...) und wird verschleiert und im lokalen Speicher der Zielplattform abgelegt. Sie können dann die ds_map erneut laden, indem Sie die Funktion ds_map_secure_load(). Beachten Sie, wenn die zu speichernde DS-Map ein Array enthält, wird dieses Array beim Speichern stattdessen in eine DS-Liste umgewandelt.

WICHTIG! Eine der Eigenschaften einer sicher gespeicherten Datei ist, dass sie an das Gerät gebunden ist, auf dem sie erstellt wurde. Sie können also eine auf einem Gerät gespeicherte Datei nicht in ein Projekt laden, das auf einem anderen Gerät läuft.

Syntax:

ds_map_secure_save(map, filename);

Argument Beschreibung
Karte Die ID der zu verwendenden Datenstruktur
Dateiname Der Name der Datei, in der die Karte gespeichert werden soll

Rückgabe:

Boolesche

Beispiel:

ds_map_secure_save(purchase_map, "p_data.dat");

Der obige Code speichert die in der Variablen "p_data" indizierte DS-Map zum späteren Abruf in der angegebenen Datei.
