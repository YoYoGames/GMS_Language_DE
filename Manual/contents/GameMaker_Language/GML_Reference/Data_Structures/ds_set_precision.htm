ds_set_präzision

Beim Vergleich von Werten, z. B. beim Suchen in einer Map oder beim Sortieren einer Liste, muss GameMaker Studio 2 entscheiden, wann zwei Werte gleich sind. Bei Strings und Integer-Werten ist das klar, aber bei reellen Zahlen können aufgrund von Fließkomma Punkt-Rundungsfehlern können scheinbar gleiche Zahlen leicht ungleich werden. Zum Beispiel ist es möglich, dass (5 / 3) * 3 nicht gleich 5 ist! Um dies zu vermeiden, wird ein Präzisionswert für alle reellen Zahlenfunktionen verwendet, und wenn die Differenz zwischen zwei Zahlen kleiner als diese Genauigkeit ist, werden sie als gleich angesehen. Standardmäßig wird eine Genauigkeit von 0,0000001 für alle Datenstrukturfunktionen verwendet verwendet, sofern sie nicht durch diese Funktion geändert wird.

HINWEIS: Diese Genauigkeit wird in allen Datenstrukturen verwendet, aber nicht in anderen Vergleichen in GML!

Syntax:

ds_set_precision(prec);

Argument Beschreibung
prec Der Präzisionswert (Standardwert 0,0000001)

Rückgabe:

Rückgabe:N/A

Beispiel:

ds_set_precision(0.0001);

Mit dem obigen Code wird die Standard-Präzisionseinstellung für alle Datenstrukturfunktionen geändert.
