ds_liste_markieren_als_liste

Diese Funktion "markiert" (oder "kennzeichnet") eine bestimmte Position innerhalb einer zuvor erstellten DS-Liste als eine andere DS-Liste enthaltend. Diese Funktionalität wird beim Kodieren von JSON-Strings benötigt (die Sie mit json_encode()), kann aber auch bei der Verschachtelung von Datenstrukturen nützlich sein, da auf diese Weise markierte Elemente automatisch entsorgt (zerstört) werden, wenn die übergeordnete DS-Liste zerstört wird. Das bedeutet, dass Sie nicht manuell durch den Listeninhalt gehen und und die markierten Datenstrukturen einzeln zerstören, bevor Sie die "übergeordnete" Liste zerstören. Wenn Sie jedoch die Listenposition einzeln löschen, wird die darin enthaltene DS-Liste nicht gelöscht, und Sie sollten die entsprechende DS-Listenzerstörungsfunktion aufrufen, bevor Sie das Element aus der übergeordneten Listenposition löschen. Beachten Sie auch, dass, wenn Sie die Funktion ds_list_clear() für eine Liste aufrufen, werden alle Elemente, die als Listen gekennzeichnet sind, ebenfalls zerstört, wenn die Liste geleert wird.

Wenn Sie also z. B. die Position 3 in der Liste markiert haben und dann 2 weitere Elemente darunter einfügen, so dass sie auf die Position 5 verschiebt, wird sie immer noch als Liste markiert.

Syntax:

ds_list_mark_as_list(id, pos);

Argument Beschreibung
id Die ID der zu markierenden Liste.
pos Die zu markierende Position innerhalb der Liste.

Rückgabe:

Real (Ganzzahl - ID der DS-Liste bei Erfolg oder -1 bei Misserfolg)

Beispiel:

var j_list = ds_list_create();
var sub_list = ds_list_create();
ds_list_add(sub_list, health);
ds_list_add(sub_list, lives);
ds_list_add(sub_list, score);
ds_list_add(j_list, sub_list);
ds_list_mark_as_list(j_list, 0);

Der obige Code erstellt zwei DS-Listen und füllt dann eine mit verschiedenen Werten aus globalen Variablen. Diese Liste wird dann in die zweite Liste eingefügt und die Position als solche "markiert", damit sie später korrekt kodiert werden kann.
