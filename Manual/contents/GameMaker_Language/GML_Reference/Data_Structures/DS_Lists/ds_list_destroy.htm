ds_list_destroy

Diese Funktion entfernt die angegebene Listen-Datenstruktur aus dem Speicher, gibt die von ihr verwendeten Ressourcen frei und entfernt alle enthaltenen Werte. Diese Funktion sollte immer verwendet werden, wenn Sie die Verwendung von ds_list beendet haben, um Speicherlecks zu vermeiden die Ihr Spiel verlangsamen und zum Absturz bringen können. Beachten Sie, dass das Zerstören einer Liste alle darin gespeicherten Datenstrukturen de-referenziert, was zu einem Speicherleck führt, so dass Sie die Liste durchgehen und alle Datenstrukturelemente manuell zerstören müssen, bevor Sie sie zerstören um dies zu verhindern. Dies ist nur dann nicht erforderlich, wenn Sie Elemente in der Liste als eine andere DS-Liste oder als eine DS-Map gekennzeichnet haben. In diesem Fall werden diese Elemente automatisch zerstört und ihr Speicher wird ebenfalls bereinigt.

WICHTIG! Wenn Sie eine Datenstruktur anlegen, ist der Indexwert zu ihrer Identifizierung ein Integer-Wert, der bei 0 beginnt. Das bedeutet, dass Datenstrukturen unterschiedlichen Typs denselben Indexwert haben können, daher sollten Sie im Zweifelsfall die ds_exists() Funktion verwenden, bevor Sie auf sie zugreifen. Beachten Sie auch, dass Indizes wiederverwendet werden, d. h. ein zerstörter Indexwert einer Datenstruktur kann anschließend von einer neu erstellten verwendet werden, weshalb wir empfehlen Wir empfehlen daher, die Variable, die den DS-Index enthielt, nach dem Zerstören immer auf -1 zu setzen.

Syntax:

ds_list_destroy(id);

Argument Beschreibung
id Die ID der zu entfernenden Datenstruktur.

Rückgabe:

N/A

Beispiel:

if (Leben == 0)
{
ds_list_destroy(AI_list);
AI_list = -1;
room_goto(rm_Menu);
}

Der obige Code prüft den Wert der eingebauten globalen Variablen "lives" und wenn er 0 ist, zerstört er die in der Variablen "AI_list" indizierte DS-Liste und wechselt dann die Räume.
