ds_list_mark_as_map

Diese Funktion "markiert" (oder "kennzeichnet") eine bestimmte Position innerhalb einer zuvor erstellten DS-Liste als eine DS-Map enthaltend. Diese Funktionalität wird benötigt, wenn Sie JSON-Strings kodieren (die Sie mit json_encode()), kann aber auch bei der Verschachtelung von Datenstrukturen nützlich sein, da auf diese Weise markierte Elemente automatisch "garbage collected" (zerstört) werden, wenn die übergeordnete DS-Liste zerstört wird. Das bedeutet, dass Sie nicht manuell durch den Listeninhalt gehen und die markierten Datenstrukturen einzeln zerstören müssen, bevor Sie die "übergeordnete" Liste zerstören. Wenn Sie jedoch die Listenposition einzeln löschen, Wenn Sie jedoch die Listenposition einzeln löschen, wird die darin enthaltene Datenstruktur nicht garbage collected und Sie sollten die entsprechende DS-Map-Zerstörungsfunktion aufrufen, bevor Sie die übergeordnete Listenposition löschen. Beachten Sie auch, dass, wenn Sie die Funktion ds_list_clear() für eine Liste aufrufen, werden alle als Maps gekennzeichneten Positionen ebenfalls zerstört, wenn die Liste gelöscht wird.

Es ist auch erwähnenswert, dass sich die "Markierung" mitbewegt, wenn sich die markierte Position innerhalb der Liste ändert, d. h. wenn Sie z. B. die Position 3 in der Liste markiert haben und dann 2 weitere Elemente darunter einfügen, so dass sie bis zur Position 5 verschiebt, wird es immer noch als Karte markiert.

Syntax:

ds_list_mark_as_map(id, pos);

Argument Beschreibung
id Die ID der zu markierenden Liste.
pos Die zu markierende Position innerhalb der Liste.

Rückgabe:

Real (Integer - ID der DS-Map bei Erfolg oder -1, wenn sie fehlschlägt)

Beispiel:

var sub_map = ds_map_create();
ds_map_add(sub_map, "player", player_array);
ds_map_add(sub_map, "enemy", enemy_array);
ds_list_add(j_list, sub_map);
ds_list_mark_as_map(j_list, 0);

Der obige Code erstellt eine DS-Map und füllt sie dann mit zwei Schlüsseln, die jeweils ein Array von Werten enthalten. Diese Map wird dann in die angegebene DS-Liste eingefügt und die Position als solche "markiert", damit sie später korrekt kodiert werden kann.
