ds_prioritaet_lesen

Mit dieser Funktion können Sie eine gespeicherte DS-Priorität wiederherstellen (eine, die zuvor als String mit ds_priority_write()). Sie müssen zunächst eine neue DS-Priorität erstellen, in die Sie den String einlesen, und wenn die DS-Priorität bereits existiert und Informationen darin gespeichert sind, werden diese vor dem Lesen gelöscht. Diese Funktion ist von entscheidender Bedeutung, wenn Sie Speicher-/Lademechanismen für Ihr Spiel erstellen. Beachten Sie, dass wenn die Datenstruktur mit früheren Versionen von GameMaker erstellt wurde, sollten Sie das optionale Argument "legacy" hinzufügen, indem Sie es auf true setzen, da sich das String-Format mit dieser Version geändert hat.

Syntax:

ds_priority_read(id, str [, legacy]);

Argument Beschreibung
id Die ID der zu schreibenden Datenstruktur.
str Die Zeichenkette, ab der geschrieben werden soll.
Vermächtnis(optional) Kann entweder wahr oder falsch sein oder komplett weggelassen werden.

Rückgabe:

N/A

Beispiel:

p_queue = ds_priority_create();
ini_open("save.ini");
var str = ini_read_string("P_Queues", "0", "");
if (str != "")
{
ds_priority_read(p_queue, str);
}
ini_close();

Der obige Code erstellt eine Prioritätswarteschlange und speichert den Index in der Variablen "p_queue". Dann wird eine Ini-Datei geöffnet und eine Zeichenkette daraus gelesen, wobei zuerst geprüft wird, dass die Zeichenkette nicht als leer zurückgegeben wird. Diese Zeichenkette wird dann in die neu erstellte DS-Priorität eingelesen.
