ds_prioritaet_zerstoeren

Diese Funktion entfernt die angegebene Datenstruktur der Prioritätswarteschlange aus dem Speicher, wodurch die von ihr verwendeten Ressourcen freigegeben werden und alle darin enthaltenen Werte entfernt werden. Diese Funktion sollte immer verwendet werden, wenn Sie die DS-Prioritätswarteschlange nicht mehr verwenden, um um Speicherlecks zu vermeiden, die Ihr Spiel verlangsamen und zum Absturz bringen können.

WICHTIG! Wenn Sie eine Datenstruktur anlegen, ist der Indexwert zu ihrer Identifizierung ein Integer-Wert, der bei 0 beginnt. Das bedeutet, dass Datenstrukturen unterschiedlichen Typs denselben Indexwert haben können, daher sollten Sie im Zweifelsfall die ds_exists() Funktion verwenden, bevor Sie auf sie zugreifen. Beachten Sie auch, dass Indizes wiederverwendet werden, d. h. ein zerstörter Indexwert einer Datenstruktur kann anschließend von einer neu erstellten verwendet werden, daher empfehlen wir, immer die Variable, die den DS-Index enthielt, nach der Zerstörung auf -1 zu setzen.

Syntax:

ds_priority_destroy(id);

Argument Beschreibung
id Die ID der zu entfernenden Datenstruktur.

Rückgabe:

N/A

Beispiel:

if (lives == 0)
{
ds_priority_destroy(AI_queue);
AI_queue = -1;
room_goto(rm_Menu);
}

Der obige Code prüft den Wert der eingebauten globalen Variablen "lives" und wenn er 0 ist, zerstört er die in der Variablen "AI_queue" indizierte DS-Prioritätswarteschlange und wechselt dann die Räume.
