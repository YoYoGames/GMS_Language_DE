DS-Gitter

Ein DS-Gitter ist im Grunde eine Art Array mit 2 Dimensionen. Sie definieren das Gitter, indem Sie seine ganzzahlige Breite und Höhe festlegen, die dann die Gesamtzahl der "Zellen" innerhalb des Gitters festlegen (Breite und Höhe multipliziert) und diese Zellen werden dann zur Aufnahme verschiedener Datenwerte (reelle Zahlen oder Zeichenketten) verwendet.

Mit diesem Strukturtyp können Sie den Wert von Zellen im Raster setzen und abrufen, indem Sie den x- und y-Index angeben (der sowohl für die x- als auch für die y-Achse mit 0 beginnt und bis zur Breite - 1 und der Höhe -1 geht). Sie können aber auch den Wert in bestimmten Gitterregionen setzen, Einzelwerte hinzufügen und die Summe, das Maximum, das Minimum und den Mittelwert über Damit ist diese Datenstruktur die vielseitigste von allen, die Ihnen zur Verfügung stehen.

Wenn Sie auf DS-Grid-Datenstrukturen zugreifen, sollten Sie immer versuchen, ganzzahlige Werte für die Zellenposition zu verwenden; alle nicht-ganzzahligen Indizes werden von GameMaker Studio 2 sonst mit dem Bodensatz versehen. Wenn das nicht der Fall ist, müssen Sie die Rundung selbst vornehmen, bevor Sie den Index übergeben, den Sie prüfen möchten.

HINWEIS: Wie alle dynamischen Ressourcen belegen auch Datenstrukturen Speicherplatz und sollten daher immer zerstört werden, wenn sie nicht mehr benötigt werden, um Speicherlecks zu vermeiden, die Ihr Spiel verlangsamen und schließlich zum Absturz bringen.

HINWEIS: Wenn Sie prüfen müssen, ob eine Datenstruktur existiert, können Sie die ds_exists() Funktion.

Abgesehen von den unten aufgeführten spezifischen Funktionen können Sie alle einen Ausdruck (einen sogenannten Accessor) verwenden, um den Inhalt Ihres DS-Gitters hinzuzufügen oder zu ändern. Dieser Accessor sieht ähnlich aus wie ein 2D Array mit der folgenden Syntax:

grid_index[# xpos, ypos]

Weitere Informationen und Beispiele finden Sie auf der GML-Übersichtsseite über Accessors.
