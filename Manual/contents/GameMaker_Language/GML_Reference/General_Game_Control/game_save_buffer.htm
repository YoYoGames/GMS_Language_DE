game_save_buffer

Dies ist eine Variante der game_save() Funktion, also lesen Sie bitte zuerst deren Seite, da sie wichtige Informationen zu ihrer Verwendung und auch zu der dieser Funktion enthält.

Mit dieser Funktion können Sie den aktuellen Zustand des Spiels in einem zuvor angelegten "grow"-Puffer (siehe Puffer) speichern , der dann jederzeit mit game_load_buffer() Funktion geladen werden kann. Diese Funktion ist für die Verwendung in jeweils einem einzelnen Raum gedacht (d.h.: es ist nicht ratsam, einen Puffer in einem Raum zu speichern und dann zu versuchen, den Puffer aus einem anderen zu laden) und sollte idealerweise nur für Checkpoints oder Level-Neustarts verwendet werden.

HINWEIS: Diese Funktion ist sehr begrenzt und für den Anfänger gedacht, um ein Checkpoint-System schnell zum Laufen zu bringen, aber fortgeschrittene Benutzer ziehen es vielleicht vor, ihr eigenes System mit den Dateifunktionen zu programmieren, aufgrund der Tatsache, dass das Spiel keine dynamischen Ressourcen wie Datenstrukturen, Oberflächen, hinzugefügte Sprites usw. speichert.

Syntax:

game_save_buffer(buffer);

Argument Beschreibung
Puffer Die Puffer-ID, in der gespeichert werden soll.

Rückgabe:

N/A

Beispiel:

if keyboard_check_pressed(ord("S"))
{
global.Checkpoint = true;
game_save_buffer(save_buff);
}

Der obige Code setzt eine globale Variable auf true und speichert dann den aktuellen Spielzustand in den in der Variablen "save_buff" indizierten Puffer, wenn die Taste "S" gedrückt wird.
