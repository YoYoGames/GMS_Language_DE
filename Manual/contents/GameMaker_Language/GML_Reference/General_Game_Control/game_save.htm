Spiel_sichern

Dies ist eine Legacy-Funktion, mit der der aktuelle Zustand des Spiels gespeichert werden kann, und wird nicht mehr zur Verwendung empfohlen. Verwenden Sie stattdessen die Dateifunktionen, um ein benutzerdefiniertes Speichersystem zu erstellen, bei dem Sie nur und bestimmte Spieldaten laden.

Diese Funktion speichert den aktuellen Zustand des Spiels, speichert jedoch keine dynamischen Ressourcen, die zu diesem Zeitpunkt verwendet werden, wie z. B. Datenstrukturen, Oberflächen, zur Laufzeit hinzugefügte Elemente usw. Wenn eine solche Speicherdatei geladen wird, sind alle Spielaktualisierungen, die die nach dem Speichern vorgenommen wurden, nicht sichtbar, da die Version des Spiels wiederhergestellt wird, die zum Erstellen des Speichers verwendet wurde.

Beachten Sie, dass Speicherdateien, die mit dieser Funktion erstellt wurden, möglicherweise nicht von aktualisierten Laufzeit Versionen nicht unterstützt werden. Daher ist das Ersetzen dieser Funktion durch ein neues Speichersystem unerlässlich, um die Kompatibilität mit zukünftigen GameMaker-Updates und aus den oben genannten Gründen auch für Updates Ihres eigenen Spiels.

Syntax:

game_save(filename);

Argument Beschreibung
Dateiname Der Name der Datei, in der das Spiel gespeichert werden soll.

Rückgabe:

N/A

Beispiel:

if keyboard_check_pressed(ord("S"))
{
global.Saved = true;
game_save("Save.dat");
}

Der obige Code setzt eine globale Variable auf true und speichert dann das Spiel in der Datei "Save.dat", wenn die Taste "S" gedrückt wird.
