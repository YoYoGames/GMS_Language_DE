matrix_build_lookat

Diese Funktion baut eine "Look-at"-Matrix (Ansichtsmatrix) auf der Grundlage der unten aufgeführten angegebenen Parameter auf. Da diese Funktion die Ansichtsmatrix und nicht die Projektionsmatrix modifiziert, sollten Sie zuerst die Projektionsmatrix mit der anderen Matrix initialisieren Funktion matrix_build_projection_perspective()initialisieren und dann diese Funktion verwenden, um die Fachkamera innerhalb der Projektion zu bewegen.

Um die Ansicht einzustellen, müssen Sie zunächst die Position festlegen, von der aus Sie schauen, was durch die Parameter (xfrom, yfrom, zfrom) angegeben wird. Als nächstes müssen Sie die Richtung angeben, in die Sie blicken, und das geschieht, indem Sie einen zweiten Punkt angeben, auf den Sie blicken mit den Argumenten (xto, yto, zto). Schließlich können Sie die Kamera noch um die Linie vom Standpunkt zum Blickpunkt drehen, und um dies anzugeben, müssen wir einen "up"-Vektor angeben - die Richtung, die in der Kamera nach oben zeigt. Dieser wird durch die letzten drei Argumente (xup, yup, zup) gegeben.

Syntax:

matrix_build_lookat(xfrom, yfrom, zfrom, xto, yto, zto, xup, yup, zup);

Argument Beschreibung
xvom Die x-Koordinate der Position, von der aus gesucht werden soll.
yvom Die y-Koordinate der Position, von der aus gesucht werden soll.
zvom Die z-Koordinate der Position, von der aus geschaut werden soll.
xbis Die x-Koordinate der Position, auf die geschaut werden soll.
yzu Die y-Koordinate der Position, nach der gesucht werden soll.
zto Die z-Koordinate der Position, auf die geschaut werden soll.
xup Die x-Koordinate des "Aufwärts"-Vektors.
ja Die y-Koordinate des "Aufwärts"-Vektors.
zup Die z-Koordinate des "Aufwärts"-Vektors.

Rückgabe:

Matrix-Index

Beispiel:

viewmat = matrix_build_lookat(640, 240, -10, 640, 240, 0, 0, 1, 0);
projmat = matrix_build_projection_ortho(640, 480, 1.0, 32000.0);
camera_set_view_mat(view_camera[0], viewmat);
camera_set_proj_mat(view_camera[0], projmat);

Der obige Code erstellt eine neue Look-at-Matrix und orthografische Matrix, speichert ihre IDs in lokalen Variablen und verwendet sie dann, um die Ansichts- und Projektionsmatrizen für die Kamera zu setzen, die dem Viewport[0] zugewiesen ist.
