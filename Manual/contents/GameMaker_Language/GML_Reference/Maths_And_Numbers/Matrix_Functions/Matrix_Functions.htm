Matrix-Funktionen

Eine Matrix ist eine Sammlung von Zahlen, die in einer festen Anzahl von Zeilen und Spalten angeordnet sind. Normalerweise sind die Zahlen reelle Zahlen, aber das muss nicht immer der Fall sein. Das folgende Bild zeigt allgemein, wie eine Matrix mit vier Zeilen und vier Spalten aufgebaut ist und vier Spalten:

Die oberste Zeile ist Zeile 1, die ganz linke Spalte ist Spalte 1, und diese Matrix ist eine 4x4-Matrix weil sie vier Zeilen und vier Spalten hat (Matrizen anderer Größe können konstruiert werden). Bei der Beschreibung von Matrizen ist das Format immer Zeilen/Spalten, und jede Zahl, aus der sich eine Matrix zusammensetzt, wird als ein Element der Matrix bezeichnet. Die Elemente in einer Matrix haben bestimmte Positionen, die durch ihre Zeilen- und Spaltenposition beschrieben werden, wobei die obere linke Ecke der Matrix Zeile 1 Spalte 1 ist. In dem obigen Matrixbeispiel ist das Element in Zeile 1 Spalte 1 der Wert "1". Das Element in Zeile 2 Spalte 3 ist der Wert "4.8".

Matrizen sind für viele Dinge sehr wichtig, werden aber am häufigsten im Umgang mit dem 3D-Raum verwendet, z. B. zum Einstellen der Kameraansicht oder zum Verschieben oder Transformieren eines Modells. Für den Umgang mit Matrizen in GameMaker Studio 2 gibt es die folgenden Funktionen:

  • matrix_get
  • matrix_set
  • matrix_build
  • matrix_multiplizieren
  • matrix_build_identity
  • matrix_build_lookat
  • matrix_build_projection_ortho
  • matrix_build_projektion_perspektive
  • matrix_build_projection_perspective_fov
  • matrix_transform_vertex
  • is_matrix

Die folgenden Funktionen dienen zur Verwendung eines Matrix-Stacks, der einem DS-Stack ähnelt, aber nur für die Verwendung mit Matrizen vorgesehen ist. Dies ist eine praktische Methode, um mehrere Matrixoperationen (wie Transformationen) nacheinander anzuwenden, wenn Sie eine 3D-Szene erstellen.

  • matrix_stack_is_empty
  • matrix_stack_clear
  • matrix_stack_set
  • matrix_stack_push
  • matrix_stack_pop
  • matrix_stack_top
