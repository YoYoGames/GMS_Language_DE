Zahlenfunktionen

Reelle Zahlen gelten in GameMaker Studio 2 als doppelgenaue Fließkommazahlen - das heißt, sie haben einen Dezimalpunkt und keine feste Anzahl von Ziffern vor oder nach dem Punkt - oder Ganzzahlen - das heißt, sie sind ganze Zahlen ohne Dezimalpunktwert. 2 ist z. B. eine ganze Zahl, aber 2,01 ist eine Fließkommazahl.

HINWEIS: Auf dem HTML5-Ziel sind alle reellen Zahlen Doppelwerte.

Diese Unterscheidung zwischen Ganzzahlen und Fließkommazahlen ist sehr wichtig, wenn es um plattformübergreifende Entwicklung geht, da die Genauigkeit von Berechnungen, die auf einem Windows-PC durchgeführt werden, nicht dieselbe ist wie die Genauigkeit der gleichen Berechnungen wenn sie auf einem mobilen Gerät durchgeführt werden. Das bedeutet, dass Sie z. B. bei Vergleichen besondere Aufmerksamkeit walten lassen sollten:

if (image_index == 1)
{
tun Sie etwas...
}

Wenn wir im obigen Beispiel die image_speed beispielsweise auf 0,25 gesetzt haben, können Sie nach vier Schritten davon ausgehen, dass der image_index-Wert 1 sein würde, aber es könnte aufgrund der Funktionsweise der Fließkommamathematik ein Wert wie 1.0000002 sein und somit wird die Auswertung nicht stimmen. Diese Art von Fehlern kann ziemlich schwer zu beheben sein, daher ist es immer gut, sich dessen bewusst zu sein und im Voraus zu planen und Es ist daher immer gut, sich ihrer bewusst zu sein, vorausschauend zu planen und andere Mittel zur Überprüfung von Werten zu verwenden oder die entsprechenden Boden- oder Rundungsfunktionen (siehe unten) zu verwenden, um die zu prüfende Zahl in eine Ganzzahl umzuwandeln (weitere Informationen zur Fließkommamathematik und warum dies ein Problem ist, finden Sie hier). Zum Beispiel könnte der obige Code geschrieben werden als:

if (floor(image_index) == 1)
{
tun Sie etwas...
}

Es ist auch erwähnenswert, dass bei den YoYo-Compiler-Zielen alle Ausdrücke und Funktionen von links nach rechts ausgewertet werden, während sie auf allen anderen Zielplattformen von rechts nach links ausgewertet werden, was bedeutet, dass dies - zum Beispiel - je nach Plattform unterschiedliche Ergebnisse liefert:

val = max(num, ++num, num++);

HINWEIS: Weitere Informationen finden Sie im Abschnitt " Auswerteauftrag".

Sie können auch eine spezielle Funktion in GameMaker Studio 2 verwenden, um den Epsilon-Wert für die Fließkomma-Mathematik einzustellen. Wenn eine reelle Zahl auf die nächstliegende Fließkommazahl gerundet wird, Wenn eine reelle Zahl auf die nächste Fließkommazahl gerundet wird, bildet das Epsilon (auch "Maschinen-Epsilon" genannt) eine Obergrenze für den relativen Fehler, und Sie können den Epsilon-Wert mit den folgenden Funktionen abrufen und einstellen:

  • math_set_epsilon
  • math_get_epsilon

Diese Funktionen befassen sich alle mit der Verwendung von Zufallszahlen und -werten:

HINWEIS: Wenn Sie die Zufallsfunktionen verwenden, behält GameMaker Studio 2 bei jedem Start des Spiels denselben Zufallswert bei. Dies erleichtert die Fehlersuche erheblich, da Sie sicher sein können, dass die Zufallsfunktionen anfangs immer den gleichen Wert zurückgeben. Sollten Sie dies jedoch nicht wünschen, müssen Sie zu Beginn des Spiels einen neuen Zufallswert setzen, entweder mit randomise() oder random_set_seed().
