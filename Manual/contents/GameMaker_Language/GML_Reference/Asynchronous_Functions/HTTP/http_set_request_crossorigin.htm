http_set_request_crossorigin

Mit dieser Funktion können Sie den Cross-Origin-Typ festlegen, der beim Laden von Bildern aus einer Datei verwendet werden soll. Die Funktion ist ausschließlich für die HTML5-Plattform und ist nur nützlich, wenn Sie Sprites aus einer Datei mit der sprite_add() Funktion. Beachten Sie, dass Sie bei der Einstellung "use-credentials" keinen absoluten Pfad zum geladenen Sprite mehr angeben müssen, sondern stattdessen einen relativen Pfad (wie im Beispiel unten gezeigt). Standardmäßig ist der Cross-Origin-Typ auf "anonymous" eingestellt.

Syntax:

http_set_request_crossorigin(origin_type);

Argument Beschreibung
ursprungs_typ Der zu verwendende Cross-Origin-Typ (eine Zeichenkette)

Rückgabe:

N/A

Beispiel:

if string_lower(http_get_request_crossorigin()) != "use-credentials"
{
http_set_request_crossorigin("use-credentials");
}
sprite_add("sprites/player_outfit_1.png", 0, false, false, 0, 0);

Der obige Code prüft zunächst den aktuell eingestellten Cross-Origin-Typ und wenn dieser nicht "use-credentials" ist, dann wird er auf "use-credentials" gesetzt und dann wird ein Sprite aus einer Datei hinzugefügt.
