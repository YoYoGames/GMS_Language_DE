HTTP

Dieser Abschnitt listet alle verschiedenen Asynchronous HTTP-Funktionen auf, die in GameMaker Studio 2 verfügbar sind. Diese Funktionen erzeugen ein Asynchrones HTTP-Ereignis in allen Instanzen, die es haben:

  • http_Anfrage
  • http_get
  • http_get_file
  • http_post_string

Bitte beachten Sie, dass die oben genannten Funktionen von http_* aufgrund von domänenübergreifenden Sicherheitsproblemen möglicherweise nicht wie erwartet funktionieren. Dies bedeutet, dass Anfragen an Ihren Server oder Versuche, Ressourcen aus anderen Domänen zu laden, blockiert werden und möglicherweise leere Ergebnisse blockiert werden und anscheinend leere Ergebnisse oder 404-Fehler zurückgeben. Eine Möglichkeit, dies zu umgehen, besteht darin, serverseitig PHP zu verwenden, das bestimmten Domains den Zugriff auf Ihren Server erlaubt (dies ist auch eine Möglichkeit, Ihre Ressourcen zu schützen und Server zu blockieren die nicht in der PHP-Zulassungsliste enthalten sind).

Im Folgenden finden Sie ein Beispiel für den Code, den Sie dafür verwenden können:

$http_origin = $_SERVER['HTTP_ORIGIN'];
if ($http_origin == "http://127.0.0.1:51268")
{
header("Access-Control-Allow-Origin: *");
}

Für Bildladeanfragen, bei denen die Bestimmung oder das Setzen ihres Cross-Origin-Typs wichtig ist, gibt es die folgenden Funktionen:

  • http_get_request_crossorigin
  • http_set_request_crossorigin

In den meisten Fällen sollten diese Funktionen nicht verwendet werden müssen, aber wenn das Spiel auf einem gesicherten Server gespeichert ist - wo bestimmte Assets möglicherweise eine grundlegende Authentifizierung erfordern, um darauf zuzugreifen, und beim Laden Sicherheitsfehler erzeugen - kann es erforderlich sein, den Cross-Origin Typ für Bildanfragen auf "use-credentials" zu setzen (im Gegensatz zur Standardeinstellung "anonymous"), kann notwendig sein.
