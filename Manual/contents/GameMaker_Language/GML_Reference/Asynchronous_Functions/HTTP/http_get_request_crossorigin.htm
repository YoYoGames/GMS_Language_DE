http_get_request_crossorigin

Diese Funktion kann verwendet werden, um den ursprungsübergreifenden Typensatz für HTML5-Spiele zu erhalten und gibt einen String zurück (auf allen anderen Plattformen wird ein leerer String "" zurückgegeben).

Syntax:

http_get_request_crossorigin();

Rückgabe:

Zeichenfolge

Beispiel:

if string_lower(http_get_request_crossorigin()) != "use-credentials"
{
http_set_request_crossorigin("use-credentials");
}
sprite_add("sprites/player_outfit_1.png", 0, false, false, 0, 0);

Der obige Code prüft zunächst den aktuell eingestellten Cross-Origin-Typ und wenn dieser nicht "use-credentials" ist, dann wird er auf "use-credentials" gesetzt und dann wird ein Sprite aus einer Datei hinzugefügt.
