Dialog

Die meisten modernen Computer und Mobilgeräte mögen es nicht, wenn ein Spiel angehalten wird, während ein Dialogfenster angezeigt wird. Sie neigen dazu, dies so zu interpretieren, dass der Game Runner gestoppt wurde, und können dies als Fehler betrachten und die App vorzeitig schließen. Um dies zu verhindern, verfügt GameMaker Studio 2 asynchrone Dialogfunktionen, die es Ihnen erlauben, bestimmte Formen von Benutzerfeedback oder -eingaben anzufordern, ohne dass der Game Runner zu irgendeinem Zeitpunkt blockiert wird, d. h. das Spiel läuft - wenn es aufgerufen wird - ganz normal im Hintergrund während der Benutzer auf das angezeigte Dialogfenster antwortet. Sobald der Benutzer geantwortet hat, wird ein asynchrones Dialog-Ereignis aufgerufen, so dass Sie mit den Ergebnissen umgehen können.

HINWEIS: Mit Ausnahme von shop_leave_rating()sind diese Funktionen nur zum Debuggen und Testengedacht und sollten nicht in veröffentlichten Spielen verwendet werden. Zu diesem Zweck sollten Sie eine benutzerdefinierte Benutzeroberfläche erstellen, um Eingaben von Spielern zu erhalten, so dass Sie die vollständige Kontrolle über das Aussehen und Verhalten der Dialoge haben.

Im Folgenden finden Sie eine Liste aller verfügbaren asynchronen Dialogfunktionen:

  • get_login_async
  • get_string_async
  • get_integer_async
  • show_message_async
  • show_question_async
  • shop_leave_rating
