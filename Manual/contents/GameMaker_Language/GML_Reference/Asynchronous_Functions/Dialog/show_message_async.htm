show_message_async

Diese Funktion öffnet ein Fenster und zeigt dem Benutzer die Nachricht an, die Sie in der Funktion definieren. Dies ist eine asynchrone Funktion, und als solche blockiert GameMaker Studio 2 das Gerät, auf dem sie ausgeführt wird, nicht, während es auf eine Antwort wartet, sondern sondern führt die Ereignisse normal weiter aus. Sobald der Benutzer die "Okay"-Taste gedrückt hat, wird ein asynchrones Dialog Ereignis ausgelöst, das nur für die Dauer nur für die Dauer dieses Ereignisses eine ds_map in der Variablen async_load.

Diese Map enthält die beiden Schlüssel "id" und "status", wobei "id" der Wert ist, der beim Aufruf der Funktion zurückgegeben wurde, und "status" entweder true ist, wenn die Schaltfläche "Okay" gedrückt wurde, oder false, wenn die Nachricht abgebrochen wurde (sofern vorhanden, da nicht alle Zielplattformen den Abbruch von Dialogen erlauben). gedrückt wurde, oder false, wenn die Nachricht abgebrochen wurde (sofern vorhanden, da nicht alle Zielplattformen den Abbruch von Dialogen erlauben).

HINWEIS: Diese Funktion ist nur für Debugging- und Testzweckegedacht und sollte nicht in veröffentlichten Spielen verwendet werden. Zu diesem Zweck sollten Sie eine benutzerdefinierte Benutzeroberfläche erstellen, um Eingaben von Spielern zu erhalten, so dass Sie die vollständige Kontrolle darüber haben, wie die Dialoge aussehen und sich verhalten.

Syntax:

show_message_async(string);

Argument Beschreibung
Zeichenfolge Die Meldung, die dem Benutzer angezeigt werden soll.

Rückgabe:

Real

Erweitertes Beispiel:

Das linke Maustasten-Ereignis (z. B.) des Objekts, das die Nachricht anzeigt, würde den folgenden Code haben:

msg = show_message_async("Sie werden jetzt in die Filiale geführt");

Der obige Befehl zeigt eine Nachricht mit der angegebenen Zeichenfolge an. Die Nachrichten-ID wird in der Variablen "msg" gespeichert und im asynchronen Ereignis "Dialoge" wie unten gezeigt verwendet:

var i_d, stat;

Der obige Code prüft den "id"-Schlüssel der zurückgegebenen DS-Map gegen den in der Variablen "msg" gespeicherten Wert. Wenn sie übereinstimmen, wird geprüft, ob die Schaltfläche "Okay" gedrückt wurde (und nicht das Fenster geschlossen/abgebrochen), und wenn er true zurückgibt, wird eine Url geöffnet.
