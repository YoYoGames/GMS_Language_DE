show_question_async

Diese Funktion öffnet ein Fenster und zeigt dem Benutzer die Frage an, die Sie in der Funktion definieren. Dies ist eine asynchrone Funktion, und als solche blockiert GameMaker Studio 2 das Gerät, auf dem sie ausgeführt wird, nicht, während es auf eine Antwort wartet, sondern sondern führt die Ereignisse wie gewohnt weiter aus. Die Funktion hat zwei Schaltflächen, die "Ja" und "Nein" anzeigen, und sobald der Benutzer eine davon gedrückt hat, wird ein asynchrones Dialog-Ereignis ausgelöst, das nur für die Dauer dieses Ereignisses eine DS-Map hat, die in der Variablen async_load.

Diese Map enthält die beiden Schlüssel "id" und "status". "id"ist der Wert, der beim Aufruf der Funktion zurückgegeben wurde, während "status" entweder wahr oder falsch für "Ja" bzw. "Nein" ist .

HINWEIS: Diese Funktion ist nur für Debugging- und Testzweckegedacht und sollte nicht in veröffentlichten Spielen verwendet werden. Zu diesem Zweck sollten Sie eine benutzerdefinierte Benutzeroberfläche erstellen, um Eingaben von Spielern zu erhalten, so dass Sie die vollständige Kontrolle darüber haben, wie die Dialoge aussehen und sich verhalten.

Syntax:

show_question_async(string);

Argument Beschreibung
Zeichenfolge Die Frage, die Sie dem Benutzer stellen müssen.

Rückgabe:

Real

Erweitertes Beispiel:

Das linke Maustasten-Ereignis (z. B.) des Objekts, das die Nachricht anzeigt, würde den folgenden Code haben:

msg = show_question_async("Möchten Sie eine Rüstung für " + string(Rüstung[0, 5]) + "Münzen kaufen?");

Die obige Funktion zeigt eine Frage mit der angegebenen Zeichenfolge an und fordert den Benutzer auf, entweder "Ja" oder "Nein" zu drücken. Die Funktions-ID wird in der Variablen "msg" gespeichert und wird im Ereignis "Asynchroner Dialog" wie unten gezeigt verwendet:

var i_d, stat;
i_d = ds_map_find_value(async_load, "id");
if i_d == msg
{
if ds_map_find_value(async_load, "status")
{
coins -= panzer[0,5];
global.protection += panzer[0,0];
}
}

Der obige Code prüft den "id"-Schlüssel der zurückgegebenen DS Map gegen den in der Variablen "msg" gespeicherten Wert. Wenn sie gleich sind, prüft er dann, ob eine der beiden Tasten gedrückt wurde, und wenn er "true" zurückgibt, zieht er dann einen Wert von einer Variablen ab und fügt auch einen Wert zu einer globalen Variablen hinzu.
