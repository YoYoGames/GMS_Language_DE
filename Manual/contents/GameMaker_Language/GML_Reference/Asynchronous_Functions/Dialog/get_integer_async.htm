get_integer_async

Diese Funktion öffnet ein Fenster und zeigt eine Meldung sowie ein Feld für die Eingabe eines Wertes durch den Benutzer an (das den mitgelieferten Standardwert enthält, mit dem begonnen wird). Dies ist eine asynchrone Funktion, und als solche blockiert GameMaker Studio 2 nicht GameMaker Studio 2 blockiert das Gerät, auf dem sie ausgeführt wird, nicht, während es auf eine Antwort wartet, sondern führt die Ereignisse wie gewohnt weiter aus. Sobald der Benutzer seine Zeichenfolge eingegeben und die Schaltfläche "Okay" gedrückt hat, wird ein asynchrones Dialog Ereignis ausgelöst, das nur für die Dauer dieses Ereignisses eine DS-Map in der Variablen async_load.

Diese Map enthält die drei Schlüssel " id"," status" und "value". " id" ist derWert, der beim Aufrufder Funktion zurückgegeben wurde, der "status" ist entweder true, wenn die Schaltfläche "Okay" gedrückt wurde, oder false, wenn die Nachricht abgebrochen wurde (gegebenenfalls, da nicht alle Zielplattformen das Abbrechen von Nachrichten erlauben). Schließlich gibt " value" die Ganzzahl zurück, die der Benutzer eingegeben hat als Zeichenkette zurück (eine leere Zeichenkette "" wird zurückgegeben, wenn keine Eingabe gemacht wurde).

HINWEIS: Diese Funktion ist nur für Debugging- und Testzweckegedacht und sollte nicht in veröffentlichten Spielen verwendet werden. Zu diesem Zweck sollten Sie eine benutzerdefinierte Benutzeroberfläche erstellen, um Eingaben von Spielern zu erhalten, so dass Sie die vollständige Kontrolle darüber haben, wie die Dialoge aussehen und sich verhalten.

Syntax:

get_integer_async(string, default);

Argument Beschreibung
Zeichenfolge Die Meldung, die im Dialog angezeigt werden soll.
Standard Der Standardwert.

Rückgabe:

Real

Erweitertes Beispiel:

Das linke Maustasten-Ereignis (z. B.) des Objekts, das die Nachricht anzeigt, würde den folgenden Code haben:

msg = get_integer_async("Wie alt sind Sie?", 0);

Oben wird eine Meldung angezeigt, die den Benutzer auffordert, eine Zeichenfolge einzugeben und "Okay" zu drücken. Die Funktions-ID wird in der Variablen "msg" gespeichert und im asynchronen Ereignis "Dialoge" wie unten gezeigt verwendet:

var i_d = ds_map_find_value(async_load, "id");
if i_d == msg
{
if ds_map_find_value(async_load, "status")
{
global.Age = ds_map_find_value(async_load, "value");
}
}

Der obige Code prüft den " id"-Schlüssel der zurückgegebenen DS-Map gegen den in der Variablen "msg" gespeicherten Wert. Wenn sie übereinstimmen, wird geprüft, ob "Okay" gedrückt wurde (und nicht das Fenster Wenn dies der Fall ist, wird eine globale Variable mit der zurückgegebenen Ganzzahl gesetzt.
