Asynchrone Funktionen

Im Kontext von GameMaker Studio 2 ist eine asynchrone Funktion eine Funktion, die an einen Web-Server sendet oder eine Benutzereingabe abfragt oder sogar Daten streamt, während GameMaker Studio 2 weiterläuft, ohne das laufende Spiel zu blockieren. Wenn wird ein spezielles Async-Ereignis ausgelöst, das wiederum jeder Instanz, der ein Async-Ereignis zugewiesen ist, erlaubt, weitere Funktionen auszuführen (obwohl dies nicht zwangsläufig der Fall sein muss). Im Grunde ist es also eine Möglichkeit, mit einer externen Ressource (Server, Benutzer oder Gerät) zu kommunizieren, ohne dass das Spiel oder Projekt angehalten wird, während es auf eine Antwort wartet.

Warum ist das wichtig? Nun, weil es GameMaker Studio 2 erlaubt, weiter zu funktionieren, während es Daten sendet oder empfängt, und das bedeutet, dass Sie viele Dinge tun können, ohne die Spielschleife zu blockieren, wie:

  • Daten vom Gerät in Ihr Spiel streamen, ohne dass der Spieler warten muss, bis alles geladen ist
  • Callback-Ereignisse haben, um Dinge nur dann zu tun, wenn die richtigen Informationen empfangen werden
  • mit einem Webserver kommunizieren und Daten austauschen

Dadurch funktioniert alles viel flüssiger und unauffälliger. Es ist zu beachten, dass die Async-Ereignisse für alle Instanzen ausgelöst werden, die sie haben, so dass Sie eine asynchrone Funktion in einer Instanz verwenden können und das asynchrone Event, das sich mit der Antwort befasst, in einer anderen, oder sogar in verschiedenen anderen. Weitere Informationen über die Funktionsweise des asynchronen Ereignisses finden Sie hier - Asynchrone Ereignisse.
