Push-Benachrichtigungen

Auf mobilen Geräten kann normalerweise immer nur eine Anwendung im Vordergrund aktiv sein, aber viele Spiele und Anwendungen arbeiten in einer zeitbasierten oder vernetzten Umgebung, in der Ereignisse von Interesse für die Benutzer auftreten können, wenn die Anwendung nicht im Vordergrund ist. In diesen Fällen können diese Spiele mithilfe von lokalen und Remote-Push-Benachrichtigungen ihre Benutzer benachrichtigen, wenn Ereignisse eintreten.

Wenn das Betriebssystem eine lokale oder entfernte Push-Benachrichtigung liefert und die Zielanwendung nicht im Vordergrund läuft, präsentiert es dem Benutzer die Benachrichtigung in Form eines Banners. Wenn es eine Benachrichtigung gibt und der Benutzer darauf tippt darauf, dann wird das Spiel gestartet und ein asynchrones Push-Ereignis aufgerufen, bei dem Sie die angegebene Callback-Zeichenkette überprüfen und den Push basierend auf seinem Wert. Wenn die Anwendung im Vordergrund läuft, wenn die Benachrichtigung zugestellt wird, erhält die Anwendung trotzdem ein Asynchronous Push Event, aber der Benutzer bekommt keine Nachricht angezeigt, es sei denn, Sie erstellen sie selbst.

Es gibt ein paar Unterschiede zwischen den unterstützten Plattformen zu beachten:

  • Unter Android werden alle Benachrichtigungen, die empfangen werden, wenn die App nicht läuft/im Hintergrund läuft, gespeichert und die Daten werden durch das async-Ereignis geliefert, wenn die App das nächste Mal läuft, und Sie müssen Push-Benachrichtigungen innerhalb der Android-Spieloptionen aktivieren, damit sie funktionieren.
  • Unter iOS werden die Daten für eine bestimmte Benachrichtigung nur geliefert, wenn die Benachrichtigung ausgewählt ist (oder wenn die App im Vordergrund läuft, wenn die Benachrichtigung empfangen wird) und Sie müssen Push-Benachrichtigungen in den iOS-Spieloptionen aktivieren, damit sie funktionieren.

Lokale Benachrichtigungen

Eine lokale Push-Benachrichtigung ist lokal auf dem Gerät, auf dem das Spiel installiert ist, und erfordert keinen Back-End-Server. Sie legen einfach das Datum und die Uhrzeit für die Benachrichtigung fest und sie wird dem Benutzer angezeigt, wenn das Spiel gerade nicht läuft. Diese Art Diese Art von Benachrichtigung ist nützlich, um den Benutzer daran zu erinnern, das Spiel wieder zu spielen, oder um eine tägliche Belohnung für das Spielen anzubieten usw. Wenn der Benutzer auf die Benachrichtigung tippt, wird das Spiel gestartet und ein Datenstring übergeben, der dann vom Spiel vom Spiel geparst werden kann, um eine Belohnung oder was auch immer zu geben.

HINWEIS: Diese Funktionalität ist beschränkt auf die iOS und Android Zielmodule. Für Android müssen Sie die Erweiterung Google Push Notification installiert haben.

Die folgenden Funktionen sind für lokale Benachrichtigungen verfügbar:

  • push_local_notification
  • push_get_first_local_notification
  • push_get_next_local_notification
  • push_cancel_local_notification

Remote-Benachrichtigungen werden von einem Server an einen Dienst gesendet, der vom App-Store der Geräteplattform bereitgestellt wird, und dieser leitet diese Nachrichten dann an alle Geräte weiter, auf denen Ihre Anwendung installiert ist. Dies wird von GameMaker Studio 2 auf den iOS- und Android-Zielmodulen unterstützt.

Es gibt keine Funktionen in GameMaker Studio 2, um mit Remote-Benachrichtigungen umzugehen, da sie alle von Ihrem Server generiert und von den jeweiligen App-Stores verarbeitet werden müssen. Sobald sie jedoch korrekt eingerichtet sind, werden GameMaker Studio 2-Spiele diese Benachrichtigungen empfangen, die dann im Asynchronen Push-Ereignis behandelt werden können, wie Sie es mit einer lokalen Benachrichtigung tun würden.

Bitte beachten Sie, dass es keine Garantie dafür gibt, dass Remote-Push-Benachrichtigungen zugestellt werden, und dass die erlaubte Datennutzlast ziemlich klein ist. Dies variiert zwischen den Plattformen, aber iOS ist besonders eingeschränkt - der Apple-Dienst liefert nur die aktuellste Benachrichtigung, die vom Empfänger ausgewählt werden muss, damit die Nutzdaten an Ihr asynchrones Ereignis geliefert werden können, und diese Benachrichtigungen haben eine maximale Nutzdatengröße von 256 Byte. Normalerweise würde eine Remote-Push-Benachrichtigung nur anzeigen, dass neue Daten zum Beispiel von Ihrem Server verfügbar sind.
