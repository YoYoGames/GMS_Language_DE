push_cancel_local_notification

Diese Funktion bricht die angegebene Push-Benachrichtigung ab. Sie können die ID der abzubrechenden Benachrichtigung mit den Funktionen push_get_first_local_notification() und push_get_next_local_notification() und die Funktion gibt bei Erfolg true oder andernfalls false zurück (z. B. wenn die Benachrichtigung nicht existiert oder bereits ausgelöst wurde).

HINWEIS: Diese Funktion ist beschränkt auf die iOS und Android Zielmodule.

Syntax:

push_cancel_local_notification(ID);

Argument Beschreibung
ID Die ID der zu stornierenden Benachrichtigung.

Rückgabe:

Real

Beispiel:

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
{
var data = ds_map_find_value(map, "data");
if data == "Daily_Reward"
{
push_cancel_local_notification(ntf);
}
ntf = push_get_next_local_notification(map);
}
ds_map_destroy(map);

Der obige Code erstellt eine DS-Map und fordert dann Daten für die erste verfügbare Benachrichtigung in der Warteschlange an. Wenn eine Benachrichtigung vorhanden ist, wird die Map aufgefüllt und der Code prüft dann den Schlüssel "data" auf eine bestimmte Zeichenfolge. Wenn die Zeichenfolge gefunden wird, wird die Benachrichtigung abgebrochen und mit der Prüfung der nächsten Benachrichtigung fortgefahren, bis alle in der Warteschlange geprüft wurden.
