push_get_next_local_notification

Diese Funktion füllt eine vorgefertigte DS-Map mit einer Reihe von Schlüssel/Wertpaaren für die nächste lokale Push-Benachrichtigung in der Warteschlange, die angezeigt werden soll. Sie gibt -1 zurück, wenn es keine weiteren Benachrichtigungen in der Warteschlange sind, oder andernfalls einen realen Wert, der die ID für die Benachrichtigung darstellt. Diese ID kann dann verwendet werden, um die Benachrichtigung mit der Funktion push_cancel_local_notification(). Sie sollten rufen Sie die Funktion push_get_first_local_notification() aufrufen, um die erste Benachrichtigung in der Warteschlange zu erhalten, und dann diese Funktion verwenden, um sie weiter durchzugehen.

HINWEIS: Diese Funktion ist beschränkt auf die iOS und Android Zielmodule.

ie DS-Map enthält die folgenden Schlüssel/Wertpaare:

  • " title" - Der Titel der Benachrichtigung (dieser ist nur auf der Android Plattform, da iOS die Titelinformationen nicht speichert)
  • "message" - Der Text der Nachricht
  • "data"- Die Zeichenkette des Datenpakets

Syntax:

push_get_next_local_notification(map);

Argument Beschreibung
Karte Die zu verwendende DS Map ID.

Rückgabe:

Real

Beispiel:

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
{
var data = ds_map_find_value(map, "data");
if data == "Daily_Reward"
{
push_cancel_local_notification(ntf);
}
ntf = push_get_next_local_notification(map);
}
ds_map_destroy(map);

Der obige Code erstellt eine DS-Map und fordert dann Daten für die erste verfügbare Benachrichtigung in der Warteschlange an. Wenn eine Benachrichtigung vorhanden ist, wird die Map aufgefüllt und der Code prüft dann den Schlüssel "data" auf eine bestimmte Zeichenfolge. Wenn die Zeichenfolge gefunden wird, wird die Benachrichtigung abgebrochen und mit der Prüfung der nächsten Benachrichtigung fortgefahren, bis alle in der Warteschlange geprüft worden sind.
