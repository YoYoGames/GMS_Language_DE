cloud_file_save

Mit dieser Funktion wird eine Datei zur Speicherung an den gewählten Cloud-Dienst übergeben. Die Funktion gibt einen eindeutigen ID-Wert zurück, der dann im entsprechenden asynchronen Ereignis verwendet werden sollte, um die DS-Map zu identifizieren, die als "Rückruf" zurückgegeben wird vom Cloud-Dienst zurückgegeben wird. Die Datei sollte alle Informationen enthalten, die Sie für Ihr Spiel speichern müssen, da Sie nur einen einzigen "Daten-Blob" in der Cloud speichern können, und ein erneutes Ausführen dieser Funktion überschreibt alle zuvor gespeicherten Werte (ebenso wie die Verwendung der cloud_string_save() Funktion). Die Beschreibung sollte eine kurze Zeichenkette mit Informationen sein, die den Speicher beschreiben, z. B.: "Level2, Stage2".

Weitere Informationen zu den zurückgegebenen asynchronen Daten finden Sie in der Funktion cloud_synchronise().

Syntax:

cloud_file_save(file, description);

Argument Beschreibung
String Die Datei, die hochgeladen werden soll (als String).
Beschreibung Eine kurze Beschreibung der zu speichernden Daten.

Rückgabe:

Real

Beispiel:

var t_str = "";
for (var i = 0; i < 10; i++;)
{
t_str += string(global.Highscore[i]) + "|"
}
var file = file_text_open_write("Highscores.txt");
file_text_write_string(file, t_str);
file_text_close(file);
save_check = cloud_file_save("Highscores.txt", "Aktuelle Highscores");

Der obige Code erstellt eine Zeichenfolge aus den im globalen Array "Highscores" gespeicherten Werten und schreibt diese Zeichenfolge dann in eine Datei zur lokalen Speicherung. Die Datei wird dann zur Speicherung an den Cloud-Dienst gesendet.
