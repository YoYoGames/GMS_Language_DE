Wolkenspeicherung

Bei der Erstellung von Spielen ist es oft notwendig, Informationen über den Spielzustand in einer Art Datei zu speichern, aber das Speichern dieser Informationen auf dem Gerät ist nicht immer die beste Option, da diese Informationen, wenn der Spieler das Spiel löscht und dann neu installiert, verloren gehen verloren gehen können. Um dies zu verhindern, können Sie verschiedene Cloud-Dienste nutzen, die eine Datenspeicherung über das Internet anbieten, um sie jederzeit abrufen und ändern zu können. Dabei sollten Sie beachten, dass diese Funktion auf einen einzigen Daten-"Blob" pro Spiel beschränkt ist, d. h. jedes Mal, wenn Sie einen neuen Spielstand an den Cloud-Dienst senden, egal ob es sich dabei um eine Zeichenfolge oder eine Datei handelt, werden alle zuvor gespeicherten Daten überschrieben.

HINWEIS: Derzeit nutzt nur das Android Ziel diese Funktionen, und Sie müssen das Kontrollkästchen Google Cloud Saving aktivieren im Abschnitt Social in den Android-Spieloptionen aktivieren. Dadurch werden Sie aufgefordert, die Google Services-Erweiterung herunterzuladen, die erforderlich ist, damit Cloud Saving auf Android funktioniert. Weitere Informationen zum Cloud Saving finden Sie in dem folgenden Helpdesk-Artikel: AndroidCloud Saving

GameMaker Studio 2 unterstützt dies mit ein paar einfachen Funktionen, die in Verbindung mit dem Asynchronous Cloud Event arbeiten. Diese Funktionen werden auf den folgenden Seiten erklärt:

  • cloud_synchronisieren
  • cloud_string_save
  • cloud_file_save
