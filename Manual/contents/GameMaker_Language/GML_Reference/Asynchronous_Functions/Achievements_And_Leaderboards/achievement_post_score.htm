leistung_post_score

Mit dieser Funktion können Sie Ihren Punktestand an den gewählten Bestenlisten- und Achievement-Dienst senden. Unter Android senden Sie die ID der Punktetabelle als String (dies ist die eindeutige Leaderboard-ID, die beim Einrichten des Leaderboards zugewiesen wurde), während Sie auf allen anderen Plattformen liefern Sie den definierten Leaderboard-Namen als String, und dann geben Sie den aktuellen Score-Wert an. Die Funktion löst ein soziales asynchrones Ereignis aus, wobei die zurückgegebene async_load DS-Map die folgenden Schlüssel/Wertpaare enthält:

  • "Typ"- Dies ist der Typ des Ereignisses, das ausgelöst wurde, wasfür diese Funktion die Zeichenfolge "achievement_post_score_result" sein wird.
  • " id" - Dies ist die ID des Ereignisses, und für diese Funktion wird eine GML-Konstante GooglePlayServices_EVENT_ID_POST_SCORE zurückgegeben .
  • "leaderboard_id" - Dies ist die eindeutige ID des Leaderboards, das aktualisiert wurde.
  • "Status" - Dieser Wert ist 1, wenn die Werte erfolgreich aktualisiert wurden, oder 0, wenn die Anfrage fehlgeschlagen ist
  • "daily_new_best" - Dieser Wert ist 1, wenn die eingereichte Punktzahl die beste des Benutzers am letzten Tag war, ansonsten 0.
  • " daily_best" - Hier wird die höchste Punktzahl des Benutzers vom letzten Spieltag gespeichert.
  • "weekly_new_best" - Dieser Wert ist 1, wenn die eingereichte Punktzahl die beste des Benutzers in der letzten Woche war, ansonsten 0.
  • "weekly_best" - Hier wird die höchste Punktzahl des Benutzers aus der letzten Woche gespeichert.
  • "all_time_new_best" - Dieser Wert ist 1, wenn der eingereichte Wert der beste aller Zeiten des Benutzers war, ansonsten 0.
  • " all_time_best" - Hier wird die beste Punktzahl des Benutzers aller Zeiten gespeichert.

Syntax:

achievement_post_score(leaderboard, score)

Argument Beschreibung
Bestenliste Der Name/die ID der Bestenliste.
Ergebnis Die zu sendende Partitur.

Rückgabe:

N/A

Erweitertes Beispiel:

if achievement_available()
{
achievement_post_score(global.LeaderboardID, 1000);
}

Der obige Code prüft, ob der gewählte Leaderboard- und Achievement-Dienst verfügbar ist, und wenn ja, postet er einen Punktestand in die Punktetabelle "Leaderboard". Dies löst ein Social Async Event aus, das wie folgt geparst werden kann:

if async_load[? "id"] == GooglePlayServices_EVENT_ID_POST_SCORE
{
if async_load[? "type"] == "achievement_post_score_result"
{
if async_load[? "status"] == 1
{
global.Best_Day = async_load[? "daily_best"];
global.Best_Week = async_load[? "weekly_best"];
global.Best = async_load[? "all_time_best"];
}
}
}

Der obige Code prüft die DS-Map async_load auf die Schlüssel "id" und "type", die dieser Funktion entsprechen, und prüft dann den Status des Ereignisses. Wenn der Status 1 ist, dann werden einige Werte aus der DS-Map in globalen Variablen gespeichert.
