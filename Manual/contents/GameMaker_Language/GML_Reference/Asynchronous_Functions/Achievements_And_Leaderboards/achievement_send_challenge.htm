Leistung_senden_Herausforderung

Diese Funktion sendet eine Herausforderung über das Netzwerk an den ausgewählten Spieler. Sie können die Spielernummer mit der Funktion achievement_load_friends() oder der achievement_load_leaderboard() Die challengeid ist der eindeutige Wert, den Sie bei der Erstellung der Herausforderung in Ihrem iTunes Connect- oder Google Play-Dashboard angegeben haben. Sie müssen auch einen Punktestand und eine kurze Textnachricht angeben sowie den Aufgabentyp festlegen. Dies kann eine der folgenden Konstanten sein:

Konstant Beschreibung
leistungsart_punktzahl_challenge Eine Herausforderung auf der Grundlage des Punktwerts.
leistung_art_leistung_herausforderung Eine Herausforderung, die auf einer Leistung basiert.

Diese Funktion löst einen Callback Social Asynchronous Event für den Spieler aus, der die Herausforderung erhalten soll, und in diesem Ereignis haben Sie eine DS-Map, die referenziert wird in der Variablen async_load. Der id-Schlüssel dieser DS-Map wird verwendet, um den richtigen Callback zu identifizieren (es kann mehr als eine Auslösefunktion für ein bestimmtes asynchrones Ereignis geben), und wird mit der Konstante achievement_challenge_received sowie einer Reihe von anderen Schlüssel/Wert-Paaren für jeden Spieler gepaart sein. Der genaue Inhalt der Karte wird unten gezeigt:

HINWEIS: dass der Benutzer auch Toast-Benachrichtigungen für erhaltene Herausforderungen erhalten kann, aber dies hängt von den Werten ab, die Sie mit der Funktion achievement_show_challenge_notifications().

  • " id" - Für diese Funktion sollte es achievement_leaderboard_infosein
  • "playerid" - Die Spieler-ID für die Herausforderung.
  • "issuerid" - Die ID des Ausstellers für die Challenge.
  • "state" - Der Status der Challenge, der einen Wert von 0 - 3 (als String) für invalid, pending , completed oder declined hat.
  • "message" - Die Textnachricht für die Herausforderung.
  • "completeddate" - Das Fertigstellungsdatum für die Challenge.
  • "issueddate" - Das Ausgabedatum für die Challenge.
  • "type"- Der Typ der angegebenen Herausforderung. Kann eine der beiden oben aufgeführten Konstanten sein
  • "Bezeichner" - Die Identifizierungszeichenfolge für die Challenge, wie auf dem Dashboard des Anbieters eingestellt
  • " score" - Der mit der Herausforderung verbundene Punktestand (falls zutreffend).

Syntax:

achievement_send_challenge(playerid, challengeid, score, type, message)

Argument Beschreibung
playerid Die eindeutige ID des herauszufordernden Spielers.
challengeid Die eindeutige Challenge-ID (wie auf dem Provider-Dashboard zu finden).
Ergebnis Der zu schlagende Punktestand.
Typ der Typ der auszugebenden Challenge - eine der beiden unten angegebenen Konstanten.
Nachricht Die Nachricht, die an die Herausforderung angehängt werden soll.

Rückgabe:

N/A

Erweitertes Beispiel:

Um eine Challenge über das Netzwerk zu senden, würden Sie diesen Code verwenden:

achievement_send_challenge(global.playerid[0], global.challengeid[0], score, achievement_type_score_challenge, "Beat that sucker!");

Diese Aufforderung löst dann das soziale Ereignis in Ihrem Spiel für den Spieler aus, an den die Herausforderung gerichtet war, und dies kann wie folgt behandelt werden:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_challenge_received
{
player_id = ds_map_find_value(async_load, "playerid");
issuer_id = ds_map_find_value(async_load, "issuerid");
state = ds_map_find_value(async_load, "state");
message = ds_map_find_value(async_load, "message");
date_completed = ds_map_find_value(async_load, "completeddate");
date_issued = ds_map_find_value(async_load, "issueddate");
ach_type = ds_map_find_value(async_load, "type");
ach_ident = ds_map_find_value(async_load, "identifier");
ach_score = ds_map_find_value(async_load, "score");
}

Der obige Code prüft die zurückgegebene DS-Map im sozialen asynchronen Ereignis und wenn ihre "id" mit der zu prüfenden Konstante übereinstimmt, extrahiert er dann die relevanten Werte für jeden der Schlüssel in der Map und speichert sie in Variablen für die zukünftige Verwendung.
