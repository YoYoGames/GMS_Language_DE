leistung_get_pic

Diese Funktion sendet eine Anfrage an den Server für das Bild eines Spielers oder eines Freundes und löst einen Callback Social Asynchronous Event aus, der die async_load Map mit den relevanten Schlüssel/Wertpaaren befüllt. Der id-Schlüssel dieser DS-Map wird verwendet, um den richtigen Callback zu identifizieren (es kann mehr als eine Auslösefunktion für ein bestimmtes asynchrones Ereignis geben), und wird mit der Konstante achievement_pic_loaded sowie einer Reihe anderer Schlüssel/Wertpaare gepaart. Der genaue Inhalt der Map ist unten dargestellt:

HINWEIS: Diese Funktion ist für iOS nur.

  • " id" - Für diese Funktion sollte es achievement_pic_loaded sein
  • "playerid" - Die ID des Spielers oder Freundes, von dem wir das Bild empfangen.
  • "spriteid" - Die Sprite-ID für das Bild, die dann zum Zeichnen auf dem Bildschirm mit einer der Draw-Sprite-Funktionenverwendet werden kann

WARNUNG! Diese Funktion erzeugt Sprites und als solche sollten ihre IDs in Variablen gespeichert und dann mit der Funktion sprite_delete() Funktion aus dem Speicher entfernt werden. Andernfalls führt die wiederholte Verwendung zu einem Speicherleck, das schließlich Ihr Spiel verlangsamt oder zum Absturz bringt.

Syntax:

achievement_get_pic(char)

Argument Beschreibung
Zeichen Die ID der Person, von der das Bild abgerufen werden soll (entweder aus einer Freundesliste oder einem Leaderboard)

Rückgabe:

N/A

Erweitertes Beispiel:

Der folgende Code würde aufgerufen werden, nachdem Sie eine Freundesliste oder eine Ranglistenliste mit achievement_load_friends() oder achievement_load_leaderboard(). Diese Funktionen geben eindeutige "id"-Werte für jede Person zurück, die dann in der Funktion verwendet werden, um das Profilbild für sie abzurufen:

achievement_get_pic(global.player_id[0]);

Dadurch wird eine Anforderung für das Bild gesendet, das mit der im globalen Array gespeicherten Player-ID verknüpft ist, und ein asynchroner Callback mit der speziellen async_load DS-Map erzeugt, der die folgenden Daten enthält:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_pic_loaded
{
var picid = ds_map_find_value(async_load, "playerid");
var spriteid = ds_map_find_value(async_load, "spriteid");
if global.playerid[0] == picid)
{
global.playerpic[0] = spriteid;
}
}

Der obige Code prüft die zurückgegebene DS-Map im sozialen asynchronen Ereignis und wenn ihre "id" mit der zu prüfenden Konstante übereinstimmt, prüft er dann, ob die mit dem Bild verknüpfte id die gleiche ist wie die angeforderte und wenn ist, wird ein weiteres globales Array gesetzt, um die eindeutige ID für das Bild zu speichern (die verwendet werden kann, um das Bild als Sprite zu zeichnen).
