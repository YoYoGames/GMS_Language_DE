leistung_holen_herausforderungen

Diese Funktion sendet eine Anfrage an den Server, um Informationen über alle aktuellen Herausforderungen zu erhalten und löst einen Callback Social Asynchronous Event aus, der die async_load Map mit den relevanten Schlüssel/Wertpaaren befüllt. Der id-Schlüssel dieser DS-Map wird verwendet, um den richtigen Callback zu identifizieren (es kann Es kann mehr als eine Auslösefunktion für ein bestimmtes asynchrones Ereignis geben) und wird mit der Konstante achievement_challenge_list_received sowie einer Reihe anderer Schlüssel/Wertpaare für jede Herausforderung gepaart. Der genaue Inhalt der Map ist unten dargestellt:

HINWEIS: Diese Funktion ist für iOS nur.

  • " id" - Für diese Funktion sollte es achievement_challenge_list_receivedsein
  • "numchallenges" - Die Anzahl der aktuell verfügbaren Herausforderungen (lokal und remote).
  • "ChallengeNplayerid" - Die Player-ID für die Challenge, wobei "N" eine ganze Zahl ist, z. B.: "Challenge5playerid" ist die Player-ID für die fünfte Challenge in der Liste.
  • "ChallengeNissuerid" - Die ID der Person, die die Challenge ausgestellt hat, wobei "N" eine ganze Zahl ist, EG: "Challenge2issuerid" ist die Aussteller-ID für die zweite Challenge in der Liste.
  • "ChallengeNstate" - Der Status der Challenge "N", der einen Wert von 0 - 3 (als String) für ungültig, ausstehend, abgeschlossen oder abgelehnt hat .
  • "ChallengeNmessage" - Die Textnachricht für die Challenge "N".
  • "ChallengeNissueddate" - Das Ausstellungsdatum für Challenge "N".
  • "ChallengeNcompleteddate" - Das Abschlussdatum für Challenge "N".
  • "ChallengeNtype" - Der Typ der gegebenen Herausforderung. Kann eine von zwei Konstanten sein:
Konstant Beschreibung
achievement_filter_friends_only Erhalten Sie nur die Spielstände der befreundeten Spieler.
leistung_filter_alle_spieler Holen Sie sich alle Spielstände für alle Spieler.

  • "ChallengeNidentifier" - Die identifizierende Zeichenfolge für die Challenge.
  • "ChallengeNscore" - Der Punktestand, der mit der Challenge verknüpft ist.

Syntax:

leistung_holen_herausforderungen()

Rückgabe:

N/A

Erweitertes Beispiel:

Der folgende Code würde wahrscheinlich aufgerufen werden, nachdem sich der Spieler in sein Spielkonto eingeloggt hat mit achievement_login():

achievement_get_challenges();

Dies sendet eine Anfrage nach den aktuellen Challenge-Informationen und erzeugt einen asynchronen Callback mit der speziellen async_load ds_map, die die folgenden Daten enthält:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_challenge_list_received
{
var numentries = ds_map_find_value(async_load, "numchallenges");
for(var i = 0; i < numentries; i++;)
{
player_id[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"playerid");
issuer_id[i] = ds_map_find_value(async_load, "Challenge" + string(i) +"issuerid");
state[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"state");
message[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"message");
date_completed[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"completeddate");
date_issued[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"issueddate");
ach_type[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"type");
ach_ident[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"identifier");
ach_score[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"score");
}

Der obige Code prüft die zurückgegebene DS Map im Social Asynchronous Event und wenn deren "id" mit der zu prüfenden Konstante übereinstimmt, wird die Map in einer Schleife durchlaufen und alle verschiedenen Werte in einer Reihe von Arrays gespeichert. Diese Informationen können dann z. B. verwendet werden, um eigene, personalisierte Herausforderungsseiten im Spiel zu erstellen.
