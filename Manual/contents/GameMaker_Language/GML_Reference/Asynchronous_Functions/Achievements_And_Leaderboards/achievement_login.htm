leistung_anmeldung

Diese Funktion meldet den Benutzer beim entsprechenden Leaderboard- und Achievement-Dienst an. Wenn der Dienst nicht verfügbar ist, wird der Benutzer in einem "vorgetäuschten" Game Center angemeldet und alle Erfolge und Spielstände werden auf dem Gerät gespeichert, so dass, wenn der tatsächliche Dienst verfügbar ist, diese Details hochgeladen werden können. Die Funktion löst einen Callback Social Asynchronous Event aus, der die async_load Map mit den relevanten Schlüssel/Wertpaaren befüllt. Der id-Schlüssel dieser DS-Map wird verwendet, um den richtigen Callback zu identifizieren (es kann mehr als eine Auslösefunktion für ein bestimmtes asynchrones Ereignis geben), und wird mit der Konstante achievement_our_info sowie einer Reihe anderer Schlüssel/Wertpaare für jede Herausforderung gepaart. Der genaue Inhalt der Map ist unten dargestellt:

  • " id" - Für diese Funktion sollte es achievement_our_info sein
  • "Name"- Der Name des Benutzers, der gerade angemeldet ist.
  • "playerid" - Die eindeutige Player-ID für den Benutzer, der gerade angemeldet ist.

Syntax:

leistung_anmeldung()

Rückgabe:

N/A

Erweitertes Beispiel:

Der folgende Code würde wahrscheinlich gleich zu Beginn Ihres Spiels oder von einem speziellen Schaltflächenobjekt, das Sie in einem Raum platziert haben, aufgerufen werden und verlangt, dass sich der Spieler bei seinem Google Game Services- oder Apple Game Center-Konto anmeldet:

achievement_login();

Dadurch wird eine Anfrage für die aktuellen Player-Informationen gesendet und ein asynchroner Callback mit der speziellen async_load DS-Map erzeugt, der die folgenden Daten enthält:

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_our_info
{
var name = ds_map_find_value(async_load, "name" );
var playerid = ds_map_find_value(async_load, "playerid" );
global.OurName = name;
global.OurId = playerid;
if os_type == os_android
{
achievement_load_leaderboard("CgkIs9_51u0PEAIQBw", 1, 100, achievement_filter_friends_only);
}
else
{
achievement_load_leaderboard("leaderboard1id", 1, 100, achievement_filter_friends_only);
}
achievement_get_challenges();
achievement_load_friends();
achievement_load_progress();
}

Der obige Code prüft die zurückgegebene DS-Map im sozialen asynchronen Ereignis und wenn ihre "id" mit der zu prüfenden Konstante übereinstimmt, wird die Map in einer Schleife durchlaufen, wobei alle verschiedenen Werte in Variablen gespeichert werden, bevor der Rest der Funktionen aufruft, um die Bestenlisten, Herausforderungen und Freundeslisten vorzubereiten.
