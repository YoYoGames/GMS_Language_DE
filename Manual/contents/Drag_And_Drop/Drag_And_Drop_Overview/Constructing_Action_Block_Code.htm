Aktionsblock-Code konstruieren

Um Objekten Verhaltensweisen hinzuzufügen, können Sie Ihren Code mithilfe von Aktionen aus den verschiedenen Bibliotheken konstruieren, die Ihnen in der Action Toolbox zur Verfügung stehen. Zu Beginn müssen Sie ein neues Drag-and-Drop-Projekt erstellen und dann ein neues Objekt erstellen ( Sie können dem Objekt bei Bedarf auch ein Sprite zuweisen). In Ihrem neuen Objekt können Sie beginnen, Ereignisse hinzuzufügen, und in den Ereignissen Ihre DnD™-Code-Aktionenhinzufügen.

Beachten Sie, dass beim Hinzufügen eines neuen Ereignisses ein "Code"-Fenster mit einer Registerkarte für das gegebene Ereignis geöffnet wird (siehe das Bild oben), und Sie können nun eine beliebige Aktion aus der Toolbox auf der rechten Seite in den Bereich "Action Block" des Codefensters ziehen. Es stimmt zwar, dass Sie jede beliebige Aktion hinzufügen können, aber das bedeutet nicht, dass sie alle funktionieren oder dass das Projekt mit ihnen kompiliert. Einige Aktionen erfordern mindestens eine Variable, um zu funktionieren, während andere - wie die Draw-Aktionen - nur funktionieren, wenn sie in einem bestimmten Ereignis verwendet werden. funktionieren, wenn sie in einem bestimmten Ereignis verwendet werden. Woher wissen Sie, welche Sie verwenden müssen? Nun, im Allgemeinen ist es einfach eine Frage der Logik... wenn eine Aktion eine Variable benötigt und wir noch keine definiert haben, dann sollten wir sie nicht verwenden, bis wir eine Aktion hinzugefügt haben, um die Variable zu erzeugen.

Wenn Sie eine Aktion aus der Toolbox in den Hauptarbeitsbereich des Aktionsblocks ziehen, wird dieser erweitert, um die verfügbarenArgumente (Parameter) anzuzeigen, die Sie ausfüllen und ändern können, um das Verhalten ausfüllen und ändern können, um das Verhalten einzustellen. Ziel-Variablen

Es ist wichtig zu beachten, dass viele Aktionen eine "Ziel"-Variable anbieten, die z. B. als "Temporär" gekennzeichnet werden kann:

Das bedeutet, dass Sie eine Variable bereitstellen können, die das Ziel ist, um den Wert zu halten, der von der Aktion zurückgegeben wird. In der obigen Beispielaktion gibt die Aktion die Lautstärke der ausgewählten Audioressource zurück, also geben Sie eine Zielvariable an, die diesen Wert enthält, damit Sie ihn später referenzieren können.

Diese Zielvariable muss zuvor entweder mit " Variable zuweisen " (erzeugt eine Instanzvariable) oder "Temporäre Variable deklarieren " (erzeugt eine lokale temporäre Variable) deklariert worden sein. Wenn Sie die Option "Temp" aktivieren, können Sie einfach einen Variablennamen hinzufügen, und die Aktion erstellt diese Variable und legt sie so fest, dass sie den Rückgabewert für Sie enthält (wodurch eine temporäre lokale Variable). Alle Aktionen, die danach verwendet werden, können nun auf den Wert in der temporären Variable zugreifen, aber nur innerhalb desselben Ereignisses oder Skripts. Temporäre Variablen sind nur innerhalb des Bereichs verfügbar, in dem sie erstellt wurden. Ausführlichere Informationen über Variablen und Variablenbereiche finden Sie hier.

Auto komplett

Wenn Sie mit Drag and Drop arbeiten, müssen Sie Variablen und Ausdrücke in die verschiedenen Eingabefelder der Aktionen einfügen. Dabei wird Ihnen jedoch oft das Fenster " Auto-Vervollständigen " als Hilfe angezeigt:

Dieses Popup-Fenster listet alle eingebauten GML(GameMaker Language)-Variablen, -Konstanten und -Funktionen sowie alle Ressourcen auf, die den Anfangsbuchstaben des eingegebenen Wortes enthalten. sowie alle Ressourcen, die die Anfangsbuchstaben dessen enthalten, was Sie getippt haben. Es kann verwendet werden, um schnell die Ressource oder Variable zu finden, die Sie für die Aktion benötigen, ohne alles selbst eintippen zu müssen. Wenn zum Beispiel alle Ihre Räume mit dem Präfix "rm_" versehen sind, wird nach der Eingabe und einem Moment Wartezeit das Fenster "Auto-Vervollständigen" mit allen Ressourcen angezeigt, die mit "rm_" beginnen. Beachten Sie, dass alle eingebauten Variablen die im Autovervollständigen-Fenster angezeigt werden, überall dort verwendet werden können, wo eine Variable oder ein Ausdruck in einer Aktion erwartet wird, ebenso wie die meisten GML-Funktionen.

Aktion Erweiterung

Manchmal sehen Sie bei der Verwendung einer Aktion ein kleines Plus-Symbol an der Seite. Dies bedeutet, dass Sie die Aktion erweitern können, um zusätzliche Aufgaben auszuführen oder weitere Argumente. Wenn Sie sich zum Beispiel die Aktion "Temporäre Variablen deklarieren" ansehen, sehen Sie, dass sie dieses Symbol hat:

Wenn Sie auf das Symbol klicken, wird die Aktion erweitert und erlaubt Ihnen, weitere Variablen zu deklarieren, wodurch Es ist einfacher und schneller, mehrere Variablen gleichzeitig zu definieren.

Das Symbol kann auch für die Aktionen verwendet werden, die optionale Argumente erfordern, wie z. B. die Aktion " Auswählen", die es Ihnen erlaubt, verschiedene Werte hinzuzufügen, die zurückgegeben werden sollen:
