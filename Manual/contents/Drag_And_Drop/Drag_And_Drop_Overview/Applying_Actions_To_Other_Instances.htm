Anwenden von Aktionen auf andere Instanzen

Die meisten Aktionen in den DnD™-Bibliothekenverfügen über eine Option, um die Aktion auf verschiedene Arten anzuwenden. Dies wird als Einstellen des Aktionsbereichs bezeichnet und kann eine von mehreren Möglichkeiten sein:

Im Wesentlichen teilen Sie GameMaker Studio 2 mit, welche Instanz die Aktion ausführen soll. Der Standard-Aktionsbereich ist self, was bedeutet, dass, sobald das Objekt als Instanz im Raum angelegt ist, diese Instanz den Aktionscode ausführen wird. Dies ist jedoch nicht immer das, was Sie wollen, und Sie möchten vielleicht, dass einige Aktionen andere oder sogar alle Instanzen im Raum auswirken. An dieser Stelle kommt das Ändern des Aktionsbereichs ins Spiel.

Die verschiedenen Bereiche für die Durchführung von Aktionen sind unten aufgeführt, aber es ist zu beachten, dass das Ändern des Bereichs einer Aktion auf diese Weise nur den neuen Bereich auf diese Aktion und nicht auf nachfolgende Aktionen in der Kette anwendet. Wenn Sie Wenn Sie eine Änderung des Aktionsbereichs auf mehrere verkettete Aktionen anwenden möchten, verwenden Sie zuerst die Aktion Anwenden auf....

Selbst

Dies ist der Standardbereich für eine Aktion und besagt einfach, dass die Aktion nur von der Instanz aufgerufen werden soll, die den Code ausführt.

Andere

Der Bereich other hat zwei Hauptfunktionen, und der Wert, den er zurückgibt, hängt davon ab, wo und wie Sie ihn verwenden. Im Ereignis "Kollision" gibt other den eindeutigen Wert der Instanz-ID zurück (ein eindeutiger Wert, der verwendet wird, um einzelne Instanzen jedes Objekts zu unterscheiden), so dass Sie z. B. eine "bullet"-Instanz erstellen und ein Kollisionsereignis mit einer "player"-Instanz haben können und in diesem den Bereich other verwenden, um Trefferpunkte vom "player"-Objekt zu entfernen und dann zum Bereich self zurückzukehren, um die "bullet"-Instanz zu zerstören.

Außerhalb des Kollisionsereignisses verhält sich die Einstellung other so, als ob sie auf noone gesetzt wäre, es sei denn, sie wird innerhalb eines begrenzten Aktionsblocks aufgerufen. Das bedeutet, dass, wenn Sie den Bereich einer Gruppe von Aktionen auf ein bestimmtes Objekt ändern, dann gibt der Bereich other während des Aufrufs dieser Aktionen die Instanz-ID der Instanz zurück, die die Aktionsgruppe ursprünglich aufgerufen hat. Zum Beispiel könnten Sie eine Aktion " Anwenden auf..." ausführen und dann im nächsten Codeblock den Bereich auf other setzen, um eine Aktion für die Instanz auszuführen, die den gesamten Ereignisblock ausführt, und nicht für die Instanz auszuführen, die in den "Anwenden auf"-Codeblöcken erfasst wird. Alle

Wenn Sie eine Aktion für alle einrichten, weisen Sie GameMaker Studio 2 an, diesen Block für jede einzelne aktive Instanz im aktuellen Raum auszuführen. Wenn Sie zum Beispiel eine Aktion "Objektinstanz zerstören und setzen ihren Gültigkeitsbereich auf all, so wird jede Instanz im Raum verschwinden, unabhängig davon, aus welchem Objekt sie erzeugt wurde.

Objekt

Einer Aktion kann auch ein Objekt als Geltungsbereich zugewiesen werden. Das bedeutet, dass alle Instanzen des angegebenen Objekts diese Aktion zum gleichen Zeitpunkt ausführen, zu dem sie aufgerufen wird. Wenn Sie also z. B. 100 Feind-Instanzen im Raum haben und Sie wollen sie alle auf einen bestimmten Punkt ausrichten. Sie rufen die Aktion "Set Point Direction " auf und legen den Aktionsbereich auf das Objekt "obj_Enemy" fest, und wenn die Aktion aufgerufen wird, ändern alle Instanzen dieses Objekts ihre Richtung.

Ausdruck

In das Eingabefeld "Ausdruck" können Sie die ID einer bestimmten Instanz eingeben, auf die die Aktion angewendet werden soll. Es kann der eindeutige ID-Wert sein, der einer Instanz aus dem Raum-Editor zugewiesen wurde, oder es kann die ID einer Instanz sein, die Sie in einer Variablen gespeichert haben (wobei die Variable der Eingabewert wäre), oder es kann sogar ein Ausdruck mit Code sein, wie im Bild unten gezeigt:

In diesem Fall ruft eine Instanz das Set Sprite von der Instanz auf, die durch den für den Ausdruck angegebenen Code erstellt wurde. Beachten Sie, dass in diesem Fall die erstellte Instanz zuerst ihr Create-Event ausführt, bevor die Aktion auf sie angewendet wird.
