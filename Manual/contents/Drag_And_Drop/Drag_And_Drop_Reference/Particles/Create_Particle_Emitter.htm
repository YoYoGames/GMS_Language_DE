Partikelemitter erstellen

Um Ihre Partikel sehen zu können, müssen Sie zuerst ein Partikelsystem erstellt haben, dann einige Partikeltypen definiert haben und schließlich einen Partikelemitter mit dieser Aktion erstellt haben. Um den Emitter zu erstellen, rufen Sie diese Aktion auf und weisen den Emitter einem Partikelsystem zu, das Sie zuvor erstellt haben (siehe Partikelsystem), und geben ihm dann eine Zielvariable, um seinen eindeutigen ID-Wert für die Verwendung in weiteren Aktionen zu speichern. Sie können diese Variable als temporäre lokale Variable kennzeichnen, aber das bedeutet, dass sie am Ende des Aktionsskripts oder des Ereignisses verworfen wird, was bedeutet, dass der Emitter zwar noch existiert, Sie aber keinen Zugriff mehr darauf haben. Das bedeutet, dass der Emitter noch vorhanden ist, Sie aber nicht mehr auf ihn zugreifen können. Seien Sie also vorsichtig, wenn Sie dies verwenden (im Allgemeinen müssen Emitter nie in einer temporären lokalen Variablen gespeichert werden, aber für bestimmte Dinge, wie das Zeichnen von Partikeln auf einer Oberfläche, kann dies erforderlich sein).

WICHTIG! Wenn Ihr Partikelsystem persistent ist, müssen Sie entweder alle Emitter am Ende jedes Raums mit "Destroy Particle Emitter" zerstören und sie im nächsten Raum nach Bedarf wieder neu erstellen, oder ihre ID-Werte in globalen Variablen speichern, so dass sie jederzeit in jedem Raum aufgerufen werden können, oder sie von einem persistenten Objekt erstellen lassen, so dass das Objekt persistent ist und über Instanzvariablen auf sie zugreifen kann. Beachten Sie auch, dass wenn Sie die Aktion " Restart Game " aufrufen, Sie das Partikelsystem vorher zerstören sollten, damit das System und die zugewiesenen Emitter aus dem Speicher entfernt werden, bevor Sie erneut starten. Wenn Sie die Emitter nicht ordnungsgemäß aus dem Systemspeicher nicht wie oben beschrieben aus dem Systemspeicher entfernt werden, kommt es zu einem Speicherleck. Dies geschieht, wenn eine Ressource in den Speicher eingefügt und dann "de-referenziert" wird, d. h., dass GameMaker Studio 2 keine Möglichkeit mehr hat, mit dieser Ressource zu interagieren.
