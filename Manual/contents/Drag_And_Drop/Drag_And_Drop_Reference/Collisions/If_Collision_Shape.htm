Wenn Kollisionsform

Diese Aktion wird verwendet, um zu prüfen, ob es eine Kollision mit einer Instanz eines bestimmten Objekts innerhalb eines bestimmten geformten Bereichs oder entlang einer Linie gibt. Sie geben die Form (Rechteck, Ellipse oder Linie) und das zu prüfende Objekt an und liefern dann die Koordinaten für die obere linke und untere rechte Ecke der verwendeten Form bzw. im Falle einer Linie die Start- und Endposition. Dabei kann es sich um absolute Positionen innerhalb des Spielraums oder um Positionen relativ zu der Instanz handeln, die die Aktion ausführt. Sie können sehen, wie dies funktioniert, sehen Sie im folgenden Bild:

Wie Sie sehen können, hängen die Ergebnisse von der Kollisionsmaske der Instanz ab ab, d. h. die Instanz des gesuchten Objekts muss ein Sprite haben, das dem sprite_index zugeordnet ist, oder ein Sprite, das dem mask_index zugeordnet ist. Wenn dies nicht der Fall ist, wird es von diesen Funktionen nicht erkannt, unabhängig davon, ob sie ein Sprite zeichnet oder nicht. Es ist erwähnenswert, dass jede Instanz, die vollständig innerhalb der für die Ellipsen- oder Rechteckform festgelegten Grenzen liegt, ebenfalls eine Kollision zurückgibt. Die Aktion wird als true ausgewertet, wenn Die Aktion wird als true ausgewertet, wenn eine Kollision erkannt wird, oder andernfalls als false, obwohl Sie auch das Not-Flag überprüfen können, um zu prüfen, ob es keine Kollision mit dem angegebenen geformten Bereich gibt, und die Aktion wird dann nur als true ausgewertet, wenn keine Kollisionen gefunden werden, andernfalls wird sie als false ausgewertet. Sie können auch die Option Exclude Self setzen, um die aufrufende Instanz von der Kollisionsprüfung auszuschließen.

Als nächstes können Sie die Option " Liste zurückgeben" aktivieren/deaktivieren. Wenn Sie dies ankreuzen, erstellt und füllt die Aktion eine Datenstruktur mit allen Instanzen, die mit der Form kollidieren. mit der Form und setzt die Variable Target auf die DS-Listen-ID. Wenn das Kontrollkästchen nicht aktiviert ist, wird die Variable Target nur dann auf einen Wert kleiner als 0 gesetzt, wenn keine Kollision erkannt wird, oder auf den eindeutigen ID-Wert einer der Instanzen in der Kollision (beachten Sie, dass Sie bei einer Kollision mehrerer Instanzen nicht wissen können, welche Instanz-ID zurückgegeben wird, wenn Sie keine Liste verwenden).

Wenn Sie ausgewählt haben, dass eine Liste zurückgegeben werden soll - und die Zielvariable nicht auf Temp gesetzt ist -, dann können Sie auch die Option Free Target auswählen. Wenn diese Option auf true gesetzt ist, wird die in der Zielvariablen enthaltene Liste zerstört und eine neue Liste erstellt (und an die Zielvariable zurückgegeben), wenn Sie die Aktion aufrufen. Auf diese Weise können Sie dieselbe Variable wiederverwenden, ohne sich Gedanken über Speicherlecks zu machen, denn wenn Sie die Option "Free Target " nicht aktivieren, sind Sie dafür verantwortlich für das Löschen der Liste verantwortlich, bevor Sie die Variable erneut in der Aktion verwenden. Beachten Sie, dass unabhängig davon, ob diese Option aktiviert ist oder nicht, wenn Sie die Aktion auf "Liste zurückgeben" eingestellt haben, die Zielvariable immer eine Listen-ID enthält, und als solche muss sie freigegeben werden, wenn die Instanz zerstört wird oder der Raum endet (Sie können dafür z. B. das Ereignis Aufräumen Ereignis verwenden, z. B. zusammen mit der Aktion " Datenstruktur freigeben" ). Es ist erwähnenswert, dass Sie die Option "Free Target" so einstellen können, dass eine vordefinierte Variable oder ein Ausdruck verwendet wird und Sie so steuern können, wann die Listendatenstruktur freigegeben wird, indem Sie die Variable an anderer Stelle setzen (oder unterschiedliche Ergebnisse für den Ausdruck haben) solange sie zu true/false ausgewertet wird.

Schließlich geben Sie die Zielvariable an, die den zurückgegebenen Wert oder die Listen-ID enthalten soll. Wenn Sie die Zielvariable, die den zurückgegebenen Wert enthalten soll, als Temp-Variable (lokale Variable) kennzeichnen, wird die Aktion diese Variable erstellen, um den Rückgabewert nur bis zum Ende des Ereignisses zu halten. In diesem Fall bewirkt die Option "Free Target" nichts und Sie sind für das Löschen der DS-Liste verantwortlich, wenn diese Option aktiviert wurde. Wenn keine Zielvariable übergeben wird und die Option Rückgabeliste aktiviert ist, wird keine Liste erstellt.

WICHTIG! Kollisionen werden nur für die Instanzen registriert, die eine gültige Kollisionsmaske haben, d. h.: sie haben ein Sprite, das dem sprite_index zugeordnet ist, oder ein Sprite, das dem mask_index zugeordnet ist. Wenn die Instanzen des Objekts, das überprüft werden, keine Kollisionsmaske haben, wird die Kollision nicht erkannt, unabhängig davon, was die Instanz zu diesem Zeitpunkt zeichnet.

Beachten Sie, dass zum Hinzufügen von Aktionen in einen "if" Block, diese an der Seite der Aktion abgelegt werden sollten, wie im Bild unten gezeigt:

Aktion Syntax:

Argumente:

Argument Beschreibung
Form Die zu verwendende Form
Objekt Das zu prüfende Objekt für Instanzen von
x1 Die x-Position, von der aus gestartet werden soll
y1 Die y-Position, von der aus gestartet werden soll
x2 Die x-Position, an der beendet werden soll
y2 Die y-Position, an der beendet werden soll
Freies Ziel Kann auf eine Variable oder einen Ausdruck gesetzt werden oder auf true/false gesetzt werden, um das Freigeben des Listenspeichers bei der Wiederverwendung einer Listenvariablen zu steuern
Ziel Die Variable, die als Ziel für den Rückgabewert der Aktion dienen soll

Die Aktion erzeugt eine Liste aller Instanzen, bei denen eine Kollision festgestellt wurde, und speichert die ID der Liste in der Variablen Target. Wenn die Aktion den Wert true auswertet, wird die Liste in einer Schleife durchlaufen und für jede Instanz, deren ID in der Liste gespeichert ist, wird die Variable "hp" um 1 reduziert. Die DS-Liste wird dann freigegeben.
