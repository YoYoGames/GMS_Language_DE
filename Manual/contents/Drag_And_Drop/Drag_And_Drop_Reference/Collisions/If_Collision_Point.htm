Wenn Kollisionspunkt

Diese Aktion wird verwendet, um zu prüfen, ob es eine Kollision mit einer oder mehreren Instanzen eines bestimmten Objekts an einem bestimmten Punkt im Raum gibt, und wird als true ausgewertet, wenn eine Kollision erkannt wird, oder false andernfalls. Sie geben das Objekt an, auf das geprüft werden soll, und die Position, an der geprüft werden soll - das kann eine absolute Position im Raum oder eine Position relativ zur Instanz sein -, und Sie können auch das Flag "Not " setzen, um zu prüfen, ob an der angegebenen Position keine Kollision vorliegt, was bedeutet dass die Aktion nur dann als wahr ausgewertet wird, wenn keine Kollisionen gefunden werden, andernfalls wird sie als falsch ausgewertet. Sie können auch die Option Exclude Self setzen, um die aufrufende Instanz von der Kollisionsprüfung auszuschließen.

Als nächstes können Sie die Option " Liste zurückgeben" aktivieren/deaktivieren. Wenn Sie diese Option aktivieren, erstellt und füllt die Aktion eine Datenstruktur mit allen Instanzen, die mit dem Punkt kollidieren. mit dem Punkt und setzt die Variable Target auf die DS-Listen-ID. Wenn diese Option nicht aktiviert ist, wird die Variable Target nur dann auf einen Wert kleiner als 0 gesetzt, wenn keine Kollision erkannt wird, oder auf den eindeutigen ID-Wert einer der Instanzen in der Kollision (beachten Sie, dass Sie bei einer Kollision mehrerer Instanzen nicht wissen können, welche Instanz-ID zurückgegeben wird, wenn Sie keine Liste verwenden).

Wenn Sie ausgewählt haben, dass eine Liste zurückgegeben werden soll - und die Zielvariable nicht auf Temp gesetzt ist - dann können Sie auch die Option Free Target auswählen. Wenn diese Option auf true gesetzt ist, wird die in der Zielvariablen enthaltene Liste zerstört und eine neue Liste erstellt (und an die Zielvariable zurückgegeben), wenn Sie die Aktion aufrufen. Auf diese Weise können Sie dieselbe Variable wiederverwenden, ohne sich Gedanken über Speicherlecks zu machen, denn wenn Sie die Option "Free Target " nicht aktivieren, sind Sie dafür verantwortlich für das Löschen der Liste verantwortlich, bevor Sie die Variable erneut in der Aktion verwenden. Beachten Sie, dass unabhängig davon, ob diese Option aktiviert ist oder nicht, wenn Sie die Aktion auf "Liste zurückgeben" eingestellt haben, die Zielvariable immer eine Listen-ID enthält, und als solche muss sie freigegeben werden, wenn die Instanz zerstört wird oder der Raum endet (Sie können dafür z. B. das Ereignis " Clean Up " zusammen mit der Aktion " Free Data Structure " verwenden). Es ist erwähnenswert, dass Sie die Option "Free Target" so einstellen können, dass eine vordefinierte Variable oder ein Ausdruck verwendet wird, und so steuern können, wann die Listendatenstruktur freigegeben wird, indem Sie die Variable an anderer Stelle setzen (oder unterschiedliche Ergebnisse für den Ausdruck haben) solange sie zu true/false ausgewertet wird.

Schließlich geben Sie die Zielvariable an, die den zurückgegebenen Wert oder die Listen-ID enthalten soll. Wenn Sie die Zielvariable, die den zurückgegebenen Wert enthalten soll, als Temp-Variable (lokale Variable) kennzeichnen, wird die Aktion diese Variable erstellen, um den Rückgabewert nur bis zum Ende des Ereignisses zu halten. In diesem Fall bewirkt die Option "Free Target" nichts und Sie sind für das Löschen der DS-Liste verantwortlich, wenn diese Option aktiviert wurde. Wenn keine Zielvariable übergeben wird und die Option Rückgabeliste aktiviert ist, wird keine Liste erstellt.

WICHTIG! Kollisionen werden nur für die Instanzen registriert, die eine gültige Kollisionsmaske haben, d. h.: sie haben ein Sprite, das dem sprite_index zugeordnet ist, oder ein Sprite, das dem mask_index zugeordnet ist. Wenn die Instanzen des Objekts, das überprüft werden, keine Kollisionsmaske haben, wird die Kollision nicht erkannt, unabhängig davon, was die Instanz zu diesem Zeitpunkt zeichnet.
