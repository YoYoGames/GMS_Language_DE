Instanzvariable setzen

Mit dieser Aktion können Sie eine der eingebauten Variablen setzen. Alle Instanzen aller Objekte haben bereits eine Reihe von Variablen in sich angelegt, die wichtige Informationen wie die aktuelle x/y-Position, die Geschwindigkeit, die Richtung, die Sichtbarkeit und viele andere Dinge. Die Instanzvariablen, die Sie verwenden können, sind:

  • X-Koordinate- Die Position entlang der X-Achse im Raum.
  • Y-Koordinate - Die Position entlang der y-Achse im Raum.
  • Direction - Die Bewegungsrichtung der Instanz (Standard ist 0, wobei rechts 0°, oben 90°, links 180° und unten 270° ist).
  • Geschwindigkeit - Die Geschwindigkeit (in Pixel pro Spielframe), mit der sich die Instanz bewegt.
  • Horizontal Speed - Die horizontale Geschwindigkeit (in Pixeln pro Spielframe), mit der sich die Instanz bewegt. Eine negative Geschwindigkeit bedeutet, dass sie sich links, eine positive Geschwindigkeit bedeutet rechts und der Standardwert ist 0.
  • Vertikale Geschwindigkeit - Die vertikale Geschwindigkeit (in Pixel pro Spielframe), mit der sich die Instanz bewegt. Eine negative Geschwindigkeit bedeutet, dass sie sich nach oben bewegt, eine positive Geschwindigkeit bedeutet nach unten und der Standardwert ist 0.
  • Visible - Die Umschaltvariable für die Sichtbarkeit (Standard ist true, sofern im Objekteditor nicht anders eingestellt, auf false gesetzt, um unsichtbar zu machen). Beachten Sie, dass eine unsichtbare Instanz weiterhin alle Ereignisse außer dem Ereignis "Zeichnen" ausführt.
  • Persistent - Die Toggle-Variable für die Persistenz (Standard ist false, sofern im Objekteditor nicht anders eingestellt, auf true setzen, um persistent zu machen). Damit wird eine Instanz von Raum zu Raum "persistent" gemacht, d. h.: die Instanz wird am Raumende nicht aufgeräumt, sondern in den nächsten Raum übertragen. Beachten Sie, dass, wenn eine Instanz persistent gemacht wird nicht ihr Ereignis "Create" oder "Destroy" aufruft, sondern die Ereignisse "Clean Up", "Room End" und "Room Start".
  • Ebene - Die Ebene, der die Instanz zugewiesen wurde.
  • Depth - Der Tiefenwert, der der Ebene zugewiesen ist.
  • Sprite - Die Sprite-Ressource, die der Instanz zugewiesen ist.
  • Image Index- Der Bildindex des zugewiesenen Sprites. Dieser Wert legt das Bild des Sprites fest, das der Instanz zugewiesen ist, und wenn Wenn der Wert größer ist als die Anzahl der Frames in der Animation, wird eine "Schleife" gebildet, so dass ein Sprite mit 5 Frames, wenn es auf Frame 7 gesetzt ist, tatsächlich Frame 2 zeigt.
  • Bildwinkel - Der Winkel, auf den die Instanz eingestellt werden soll (Standard ist 0, wobei rechts 0°, oben 90°, links 180° und unten 270° ist).
  • Image Alpha- Der Alphawert für die Instanz (Standard ist 1) von 0 bis 1, wobei 0 vollständig transparent und 1 vollständig undurchsichtig ist.
  • Image Blend - Die Farbe, die mit dem Instance-Sprite überblendet werden soll (Standard ist weiß).
  • Bildgeschwindigkeit - Die Animationsgeschwindigkeit für die Instanz.
  • Horizontal Scale - Der horizontale Maßstab, mit dem das Instanz-Sprite gezeichnet werden soll (dies wirkt sich auch auf den Begrenzungsrahmen der Instanz Box und Kollisionen). Ein Maßstab von 1 bedeutet keine Skalierung (1:1), kleinere Werte skalieren nach unten (0,5 z. B. halbiert die Breite des Sprites), größere Werte skalieren nach oben und negative Werte spiegeln das Sprite und skalieren es, außer der verwendete Wert ist genau -1 (in diesem Fall wird das Sprite nur an der y-Achse gespiegelt und nicht skaliert).
  • Vertical Scale - Der vertikale Maßstab, mit dem das Instanz-Sprite gezeichnet werden soll (dies wirkt sich auch auf den Begrenzungsrahmen der Instanz Box und Kollisionen). Ein Maßstab von 1 bedeutet keine Skalierung (1:1), kleinere Werte skalieren nach unten (0,5 zum Beispiel halbiert die Höhe des Sprites), größere Werte skalieren nach oben und negative Werte spiegeln das Sprite und skalieren es, außer der verwendete Wert ist genau -1 (in diesem Fall wird das Sprite nur entlang der y-Achse gespiegelt und nicht skaliert).
  • Maske- Der Maskenindex für die Instanz. Der Standardwert ist -1 (d. h. die Instanz verwendet die Kollisionsmaske für den aktuell zugewiesenen Sprite-Index), aber wenn er auf eine andere Sprite-Ressource gesetzt wird, wird die Kollisionsmaske für diese Instanz anstelle des dem Sprite-Index zugewiesenen Sprites verwendet.
  • Reibung - Die Reibung, mit der die Instanz bei jedem Spielframe verlangsamt wird. Dieser Wert wird einfach von dem aktuellen Geschwindigkeitswert abgezogen (oder addiert, wenn der Geschwindigkeitswert negativ ist).
  • Schwerkraft - Die Schwerkraft übt eine konstante Kraft in der Schwerkraftrichtung (siehe unten) der Instanz aus, die ihre Geschwindigkeit beeinflusst (Standard ist 0). Beachten Sie, dass die Schwerkraft eine kumulative Kraft ist und das Objekt beschleunigt, wenn Sie die Endgeschwindigkeit nicht begrenzen wollen.
  • Gravity Direction - Hier können Sie die Bewegungsrichtung festlegen, wenn die Schwerkraft der Instanz größer als 0 ist (Standard ist 270). Beachten Sie, dass Richtungen in GameMaker Studio 2 so berechnet werden, dass 0° rechts, 90° oben, 180° links und 270° unten ist.
  • Score - Legen Sie die Score-Variable fest.
  • Leben - Legen Sie die Variable Leben fest.
  • Gesundheit - Legen Sie die Gesundheitsvariable fest.

Sobald Sie die zu verwendende eingebaute Variable ausgewählt haben, legen Sie den Wert für diese Variable fest. Der Wert, den Sie verwenden, hängt von der ausgewählten Variable ab (siehe die Liste oben für Standardwerte und Bereiche), und Sie können den Wert so kennzeichnen, dass er relativ zum aktuellen Wert für die Variable zu sein. Sie können auch weitere Variablen hinzufügen, die in derselben Aktion gesetzt werden sollen, indem Sie auf das Plus-Symbol neben der Aktion klicken und eine weitere Instanzvariable auswählen und ihr einen Wert geben.
