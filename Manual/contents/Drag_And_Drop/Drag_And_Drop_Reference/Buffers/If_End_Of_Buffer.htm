Wenn Ende des Puffers

Diese Aktion kann verwendet werden, um zu prüfen, ob die Lese-/Schreibposition das Ende des Puffers erreicht hat. Sie geben den eindeutigen Puffer-ID-Wert an, der von der Aktion " Puffer erstellen " zurückgegeben wird, und dann gibt die Aktion "true" zurück, wenn die Lese-/Schreibposition das Ende aller verfügbaren Daten im Puffer erreicht hat, oder "false", wenn dies nicht der Fall ist. Sie können "not" markieren, damit die Aktion prüft, ob die Lese-/Schreibposition nicht am Ende des Puffers liegt.

Beachten Sie, dass zum Hinzufügen von Aktionen in den "if"-Block diese an der Seite der Aktion abgelegt werden sollten, wie im Bild unten gezeigt:

Der obige Aktionsblockcode erstellt eine DS-Liste in der Instanz und führt dann eine Schleife durch die Daten innerhalb des angegebenen Puffers und liest die Daten aus und fügt sie der Liste hinzu. Dies wird so lange fortgesetzt, bis das Ende des Puffers erreicht ist, woraufhin die Schleife abgebrochen wird.
