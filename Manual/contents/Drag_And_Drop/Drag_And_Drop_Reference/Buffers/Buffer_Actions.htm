Puffer Aktionsbibliothek

Ein Puffer ist ein Bereich des physischen Speichers, der verwendet wird, um Daten vorübergehend zu halten, während sie von einem Ort zu einem anderen verschoben oder auf irgendeine Weise manipuliert werden. GameMaker Studio 2 verfügt über eine Reihe von spezifischen Aktionen im Zusammenhang mit dem Erstellen, Ändern und Laden/Speichern von Puffern, aber bevor Sie diese verwenden, beachten Sie bitte die folgenden Punkte:

  • Wenn ein Puffer angelegt wird, wird er automatisch gelöscht und auf 0 gefüllt.
  • Das Argument "Größe" der Pufferfunktionen bezieht sich immer auf die Größe des Puffers in Bytes.
  • DnD™-Puffersind immer 1 Byte ausgerichtet. Die "Ausrichtung" bezieht sich darauf, wie Daten innerhalb eines Puffers gespeichert werden und wird bei der Verwendung der GML-Pufferfunktionen verwendet , aber für Drag and Drop ist es nicht relevant.
  • Daten in Puffern werden sequentiell gelesen und geschrieben (d. h.: Sie rufen die Aktion "Puffer schreiben" dreimal auf, dann werden die ersten Daten geschrieben, vorwärts bewegt, die zweiten Daten richtig, vorwärts bewegt und dann geschrieben die letzten Daten), mit etwas, das "seek"-Wert genannt wird, um Ihnen zu sagen, wo im Puffer das nächste Lesen/Schreiben stattfinden wird.
  • Alle Daten werden in den Puffer geschrieben und aus dem Puffer als String zurückgegeben. Wenn Sie also echte Zahlen gespeichert haben, müssen Sie eine der Datentyp-Aktionen verwenden, um sie vom String in den Wert zu konvertieren.

Der Speicher, der für die Erstellung von Puffern verwendet wird, ist der Systemspeicher, so dass alle in einem Puffer gespeicherten Daten sicher sein sollten, auch wenn das Spiel nicht den Fokus hat; wenn die App jedoch geschlossen oder neu gestartet wird, gehen sie verloren. Es ist auch erwähnenswert, dass Puffer Puffer auch als Dateien gespeichert und dann mit den Dateiaktionen wieder in das Spiel geladen werden können.
