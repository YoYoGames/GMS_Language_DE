Lesepuffer

Diese Aktion kann verwendet werden, um Daten aus einem zuvor erstellten Puffer zu lesen. Sie geben den eindeutigen Puffer-ID-Wert (wie von der Aktion " Puffer erstellen" zurückgegeben) und eine Variable zum Speichern des zurückgegebenen Wertes an. Wenn Sie die Variable als temporäre lokale Variable kennzeichnen, wird sie für Sie erstellt und am Ende des Ereignis- oder Aktionsskripts wieder verworfen. Alle Daten werden als String zurückgegeben. Wenn Sie also echte Zahlen gespeichert haben, müssen Sie eine der Datentyp-Aktionen verwenden, um sie vom String in den Wert umzuwandeln. Denken Sie auch daran, dass Puffer sequentiell gelesen und beschrieben werden, so dass beim Lesen aus einem Puffer die "Suchposition" zum nächsten Eintrag verschoben wird, was bedeutet, dass Sie zum Lesen verschiedener Daten um verschiedene Daten zu lesen, rufen Sie diese Aktion einfach so oft wie nötig auf und speichern jedes zurückgegebene Datenstück in einer Variablen.

Aktion Syntax:

Argumente:

Argument Beschreibung
Puffer Der Pufferindex (gespeichert in einer Variablen)
Ziel Die Zielvariable zur Aufnahme der zurückgegebenen Daten

Der obige Aktionsblockcode setzt die Puffer-Lese-/Schreibposition an den Anfang des Puffers und liest dann zwei Daten zurück und speichert sie (als Strings) in temporären lokalen Variablen. Diese Variablen werden dann verwendet, um die Daten auf den Bildschirm zu zeichnen.
