Pufferposition holen

Mit dieser Aktion können Sie die aktuelle Position für das Lesen/Schreiben in den Puffer abrufen. Wenn Sie einen Puffer erstellen, wird die Lese-/Schreibposition auf 0 Bytes gesetzt, und wenn Sie dem Puffer Daten hinzufügen, bewegt sich diese Position sequentiell weiter, was bedeutet, dass Sie können diese Aktion verwenden, um die Anzahl der Bytes in den Puffer zu bekommen, auf die er zu jeder Zeit eingestellt ist. Sie geben den eindeutigen Puffer-ID-Wert an, der von der Aktion " Puffer erstellen " zurückgegeben wird, und geben dann eine Zielvariable an, an die den Positionswert. Die Zielvariable kann als temporäre lokale Variable gekennzeichnet werden. In diesem Fall wird sie für Sie erstellt und am Ende des aktuellen Aktionsskripts oder Ereignisses wieder aus dem Speicher entfernt.

Aktion Syntax:

Argumente:

Argument Beschreibung
Puffer Der Pufferindex (gespeichert in einer Variablen)
Position Die Lese-/Schreibposition (in Bytes) innerhalb des Puffers

Beispiel:

Der obige Aktionsblockcode ermittelt die Lese-/Schreibposition für den angegebenen Puffer und wenn sie nicht 0 ist (d. h.: am Anfang des Puffers), wird sie auf den Anfang des Puffers gesetzt.
