Wenn undefiniert

Diese Aktion wird verwendet, um zu prüfen, ob eine Variable oder ein Ausdruck den Wert "undefiniert" zurückgibt. Wenn dies der Fall ist, gibt diese Aktion true zurück, wenn nicht, wird false zurückgegeben. Ein undefinierter Wert kann einen Absturz oder einen schwerwiegenden Fehler in Ihrem Projekt verursachen, so dass Sie diese Aktion verwenden können, um dies zu überprüfen und abzufangen, bevor es passiert. Wenn Sie das Argument "Not" markieren, dann prüfen Sie, ob ein Wert nicht "undefiniert" ist.

Beachten Sie, dass zum Hinzufügen von Aktionen in den "if"-Block diese an der Seite der Aktion abgelegt werden sollten, wie im Bild unten gezeigt:

Diese Aktionen werden nun ausgeführt, wenn der "if"-Block als wahr ausgewertet wird, während alle an anderer Stelle abgelegten Aktionen nach dem "if"-Block ausgeführt werden. Beachten Sie, dass Sie undefinierte Prüfungen in derselben Aktion "stapeln" können, indem Sie auf das Plus-Symbol neben der Aktion klicken und eine weitere zu prüfende Variable angeben. Dadurch wird geprüft, ob alle angegebenen Variablen in einer Prüfung undefiniert sind, und der hinzugefügte Aktionscode wird nur ausgeführt, wenn dies der Fall ist.

Aktion Syntax:

Argumente:

Argument Beschreibung
Variabel Der Name der zu prüfenden Variablen.
