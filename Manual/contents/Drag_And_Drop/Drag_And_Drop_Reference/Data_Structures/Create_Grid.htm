Raster erstellen

Diese Aktion erstellt eine neue Grid-Datenstruktur und gibt den Indexwert zurück, damit Sie später über die anderen Datenstruktur-Aktionen auf das Grid zugreifen können. Der Gitterindex wird an die von Ihnen angegebene Zielvariable zurückgegeben, die zuvor mit Sie können auch das Kontrollkästchen " Temp" markieren, um eine temporäre lokale Variable zu benennen und zu erstellen, die den Wert bis zum Ende der Anwendung speichert. um den Wert bis zum Ende des Scripts oder Ereignisses zu speichern. Beachten Sie, dass Gitter im Gegensatz zu anderen Datenstrukturen nicht "leer" erstellt werden und jede Zelle eines neu erstellten Gitters den Wert 0 hat. Es liegt an Ihnen, die Zelldaten zu ändern oder das Raster selbst mit der Aktion Raster löschen auf einen anderen Wert zu setzen. Beachten Sie, dass Sie weitere DS-Gitter erstellen können, indem Sie auf das Plus-Symbol neben der Aktion klicken und eine weitere Variable auswählen, die die ID des Gitters enthält, und erneut die Werte für Breite und Höhe angeben.

WICHTIG! Das Erstellen von Datenstrukturen verbraucht Speicher auf der Zielplattform, und daher sollten alle Datenstrukturen mit der Aktion "Datenstruktur freigeben" freigegeben werden, wenn sie nicht mehr benötigt werden, sonst entsteht ein Speicherleck, das die Leistung Ihres Spiels beeinträchtigen oder sogar zum Absturz führen kann. Dies ist besonders relevant, wenn Sie temporäre lokale Variablen zum Speichern von Datenstrukturindizes verwenden, da diese Variablen am Ende des Codes oder Ereignisses entfernt werden, aber Das bedeutet aber nicht, dass auch die Datenstruktur entfernt wird! Die Datenstruktur existiert weiterhin, Sie haben nur keine Möglichkeit, sie zu referenzieren, also verwenden Sie entweder eine Instanzvariable und geben Sie die Struktur zu einem späteren Zeitpunkt frei, oder geben Sie die Struktur vor dem Ende von des Ereignisses oder der Funktion, wenn ihr Index in einer temporären Variablen gespeichert ist.
