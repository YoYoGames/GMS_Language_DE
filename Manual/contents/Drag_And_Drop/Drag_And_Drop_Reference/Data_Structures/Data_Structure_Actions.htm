Datenstruktur-Aktionsbibliothek

In Spielen müssen Sie oft Informationen in einer präzisen und geordneten Weise speichern. Sie müssen z. B. Listen von Gegenständen speichern, die eine Person bei sich trägt, oder Sie möchten ein Raster von Orten speichern, die noch besucht werden müssen. Nun, Sie können mehrere Variablen verwenden, aber wenn Sie kompliziertere Operationen durchführen wollen, wie z. B. das Sortieren der Daten oder die Suche nach einem bestimmten Gegenstand, müssen Sie große Teile des Codes von Aktionsblöcken schreiben, die nur langsam ausgeführt werden können, schwer zu debuggen sind und sehr mühsam zu schreiben sind!

Um hier Abhilfe zu schaffen, verfügt GameMaker Studio 2 über eine Reihe von eingebauten Datenstrukturen, auf die über spezielle Funktionen zugegriffen werden kann. Es gibt vier verschiedene Arten von Datenstrukturen, die über Drag and Drop verfügbar sind (es sind sechs, wenn Sie GML), Jede hat ihre eigenen Vor- und Nachteile, je nachdem, welche Art von Informationen Sie speichern wollen und wie Sie sie später manipulieren möchten - die vier Typen sind:

Listen Eine Liste ist eine Datenstruktur, die Informationen sequentiell speichert, während sie hinzugefügt werden. Listenindizes beginnen bei 0 und steigen mit jedem Element, das am Ende der Liste hinzugefügt wird, obwohl Sie auch Daten an einer anderen als der letzten Position einfügen können. Anders als bei der Datenstruktur "Stack" (siehe unten) können Sie jederzeit von jedem Index der Liste auf die Daten zugreifen.
Karten Die Map-Datenstruktur ist eine außerordentlich nützliche Struktur, da sie die Speicherung von Schlüssel und Wert Paare. Zum Beispiel kann ein Charakter in Ihrem Spiel eine Menge verschiedener Gegenstände haben (Schlüssel) und für jeden einzelnen Gegenstand kann er eine Anzahl davon haben (Werte), wie in einem RPG, wo man 10 Gesundheitstränke, 5 Manatränke und 100 Gold haben kann. Karten halten solche Paare alle zusammen an einem Ort, und Sie können der Karte Paare hinzufügen und mit einigen einfachen Funktionen auch nach dem Wert suchen, der bestimmten Schlüsseln entspricht. Karten sind nicht in irgendeiner (erkennbaren) Weise sortiert, was bedeutet, dass Sie, um einen bestimmten Schlüssel zu finden, möglicherweise durch das ganze Ding iterieren müssen (was sehr langsam ist), und es gibt auch keine Möglichkeit, zwei Schlüssel zu halten, die gleich sind, noch können Sie einem Schlüssel zwei Werte zuweisen.
Gitter Ein Raster ist eine zweidimensionale Anordnung von Werten, deren Breite und Höhe Sie festlegen. Dadurch wird die Gesamtzahl der Gitter-"Zellen" innerhalb des Gitters festgelegt (die Breite und Höhe multipliziert) und diese Zellen werden dann verwendet, um verschiedene Datenwerte zu halten, die jeder der zulässigen Datentypen sein können. Mit diesem Strukturtyp können Sie den Wert von Zellen im Raster setzen und abrufen, indem Sie die x- (Spalte) und y-Position (Zeile) davon angeben (sowohl Zeilen als auch Spalten beginnen bei 0 und gehen bis zur Rasterbreite - 1 und der Rasterhöhe -1). Wenn Sie auf Grid-Datenstrukturen zugreifen, sollten Sie immer versuchen, ganzzahlige Werte für die Zellenposition zu verwenden, und alle nicht ganzzahligen Indizes werden von GameMaker Studio 2 werden alle nicht-ganzzahligen Indizes unterdrückt, wenn Sie dies nicht tun. Wenn das nicht der Fall ist, müssen Sie die Rundung vorher selbst vornehmen, bevor Sie den Index übergeben, den Sie überprüfen möchten.
Stapel Eine Stack-Datenstruktur ist eine so genannte Last-In-First-Out (LIFO)-Struktur. Sie können Werte auf einen Stapel schieben und wieder entfernen, indem Sie sie aus dem Stapel herausnehmen, und der Wert, der zuletzt auf den Stapel geschoben wurde, ist der erste, der wieder herausgenommen wird (denken Sie einfach an einen Münzstapel, bei dem jede Münze, die Sie hinzufügen, zuerst wieder entfernt werden muss, bevor Sie an die restlichen Münzen darunter gelangen können). Stapel werden oft verwendet, wenn Unterbrechungen zu behandeln sind, oder wenn Sie rekursive Funktionen haben, oder sogar wenn Sie eine rudimentäre KI für Ihre Spiele konstruieren.

Im Wesentlichen funktionieren alle Datenstrukturen auf die gleiche Weise - Sie erstellen eine Datenstruktur und speichern ihren Indexwert in einer Variablen. Diesen Index verwenden Sie dann, um die Datenstruktur in allen weiteren Funktionsaufrufen zu referenzieren, die Operationen auf ihr durchführen können. Zum Schluss, wenn Sie fertig sind, zerstören Sie die Datenstruktur wieder, um sie aus dem Speicher zu entfernen. Sie können so viele der Strukturen gleichzeitig verwenden, wie Sie benötigen, und alle Strukturen können sowohl Zeichenketten als auch reale Werte speichern.

HINWEIS: Wie alle dynamischen Ressourcen belegen auch Datenstrukturen Speicherplatz und sollten daher immer zerstört werden, wenn sie nicht mehr benötigt werden, um Speicherlecks zu vermeiden, die Ihr Spiel verlangsamen und schließlich zum Absturz bringen.
