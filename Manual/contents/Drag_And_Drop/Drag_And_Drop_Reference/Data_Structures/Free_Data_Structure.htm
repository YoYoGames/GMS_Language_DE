Freie Datenstruktur

Diese Aktion gibt die Datenstruktur aus dem Speicher frei und sollte immer dann aufgerufen werden, wenn eine Datenstruktur nicht mehr benötigt wird, z. B. beim Destroy-Event einer Instanz oder dem Room-End-Event, etc. Das Freigeben einer Datenstruktur bedeutet, dass keine weiteren Aktionen referenziert werden kann, da sie nicht mehr existiert, obwohl Sie dies auch mit der Aktion If Data Structure Exists überprüfen können.

Beachten Sie, dass die Datenstruktur nicht von einem Raum in einen anderen übertragen wird, es sei denn, der Index der Datenstruktur ist einer Variablen des globalen Bereichs zugewiesen. Wenn Sie z. B. eine Player-Instanz erstellen und eine Listendatenstruktur einer Variablen in dieser Instanz zugewiesen ist, muss die Liste freigegeben werden, wenn der Player zerstört wird oder wenn sich der Raum mit der Player-Instanz darin ändert (es sei denn, die Player-Instanz ist als persistent gekennzeichnet), so dass die Player-Instanz zwei "Free Data Structure"-Aktionen für diese beiden Ereignisse enthält. diese beiden Ereignisse. Es ist wichtig, dass alle erstellten Datenstrukturen auf diese Weise zerstört werden, wenn sie nicht mehr benötigt werden, um Speicherlecks zu vermeiden, die zu Leistungsproblemen oder sogar zum Absturz Ihres Spiels führen können.

Aktion Syntax:

Argumente:

Argument Beschreibung
Typ Der Typ der Datenstruktur, die Sie freigeben (Liste, Map, Grid oder Stack)
Variabel Die Variable, die den Index der freizugebenden Datenstruktur enthält

Beispiel:

Der obige Aktionsblockcode ruft die Größe der Liste ab, auf die in der globalen referenziert wird, und verwendet dann eine for-Schleife, um sie durchzugehen und die Instanzen mit den in der Liste gespeicherten IDs zu zerstören. Nachdem die Schleife beendet ist, wird die Liste aus dem Speicher freigegeben.
