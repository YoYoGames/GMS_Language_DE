Wenn Taste losgelassen

Diese Aktion prüft, wann eine Taste der Tastatur losgelassen wurde. Sie gibt bei dem einzelnen Spielschritt (Frame), bei dem die Taste losgelassen wird, true zurück und zu allen anderen Zeiten false. Wenn Sie den Modifikator not aktivieren, wird diese Aktion Wenn Sie den Modifikator not aktivieren, wird diese Aktion umgekehrt und prüft, ob die Taste nicht losgelassen wurde, und gibt false bei dem Spielschritt (Frame) zurück, bei dem sie losgelassen wurde, und true zu allen anderen Zeitpunkten.

Beachten Sie, dass zum Hinzufügen von Aktionen in den "if"-Block diese an der Seite der Aktion abgelegt werden sollten, wie im Bild unten gezeigt:

Diese Aktionen werden nun ausgeführt, wenn der "if"-Block als wahr ausgewertet wird, während alle Aktionen, die an anderer Stelle ausgelassen werden, nach dem "if"-Block ausgeführt werden.

Aktion Syntax:

Argumente:

Argument Beschreibung
Taste Der Name des zu prüfenden Schlüssels.
Nicht Negieren der Prüfung (aus true wird false und umgekehrt)

Beispiel:

Der obige Code des Aktionsblocks fragt den Tastaturstatus bei jedem Schritt ab und wenn eine Taste gedrückt gehalten wird, prüft er, ob die Taste beim ersten Drücken gedrückt wird. Wenn die Prüfung auf den anfänglichen Tastendruck zutrifft, wird ein Ton abgespielt und dann, während die Taste gedrückt gehalten wird, wird die Sprite-Überblendung auf rot geändert. Wenn die Taste nicht gehalten wird wird eine Prüfung beim Loslassen der Taste durchgeführt, um die Überblendungsfarbe auf Weiß zurückzusetzen.
