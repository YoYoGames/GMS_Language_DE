Virtuelle Tastatur anzeigen

Diese Aktion kann verwendet werden, um die virtuelle Tastatur auf dem Gerät anzuzeigen, auf dem das Spiel läuft. Wenn Sie diese Aktion aufrufen, werden Sie aufgefordert, für jedes der ersten drei Argumente eine der folgenden Konstanten anzugeben:

  • Tastaturtyp: Mit dieser Konstante wird eingestellt, welcher Tastensatz auf der virtuellen Tastatur zur Verfügung steht. Die verfügbaren Tastaturtypen sind:
    Tastatur-Typ Beschreibung
    kbv_type_default Der Standard-Tastaturtyp für das aktuelle System.
    kbv_type_ascii Eine reine ASCII-Tastatur.
    kbv_type_url Eine normale Tastatur, die für die Eingabe von URLs optimiert ist. Verfügt in der Regel über eine ".com"- oder eine andere Domänen-Taste sowie über "/"- und "."-Tasten.
    kbv_type_email Eine normale Tastatur, die für die E-Mail-Eingabe optimiert ist. Verfügt in der Regel über "@"- und "."-Zeichen.
    kbv_type_numbers Eine reine Zifferntastatur, die in der Regel als Ziffernblock angezeigt wird.
    kbv_type_phone Eine Tastatur mit Telefonblock. Normalerweise nur Zahlen mit den Tasten "*" und "#".
    kbv_type_phone_name Eine Tastatur, die für die Eingabe einer Rufnummer und eines Namens optimiert ist. In der Regel ähnlich wie eine ASCII-Tastatur, jedoch mit eingeschränkter Sonderzeichenauswahl.
  • Rückgabetyp: Mit dieser Konstante wird eingestellt, was auf der Return-/Aktionstaste der virtuellen Tastatur angezeigt wird. Die verfügbaren Rückgabetypen sind:
    Rückgabetyp Beschreibung
    kbv_returnkey_default Der Standard-Rückkehrtastentitel für das aktuelle System.
    kbv_returnkey_go Setzt den Titel der Rückgabetaste auf "Go".
    kbv_returnkey_google Setzt den Titel der Rückgabetaste auf "Google" oder in manchen Fällen auf ein allgemeines Suchsymbol.
    kbv_returnkey_join Setzt den Titel der Rückgabetaste auf "Los".
    kbv_returnkey_next Setzt den Titel der Rückgabetaste auf "Weiter".
    kbv_returnkey_route Setzt den Titel der Rückgabetaste auf "Route".
    kbv_returnkey_search Setzt den Titel der Rückgabetaste auf "Suchen" oder in manchen Fällen auf ein generisches Suchsymbol.
    kbv_returnkey_send Setzt den Titel der Rückgabetaste auf "Senden".
    kbv_returnkey_yahoo Setzt den Titel der Rückgabetaste auf "Yahoo" oder in manchen Fällen auf ein allgemeines Suchsymbol.
    kbv_returnkey_done Setzt den Titel der Rückgabetaste auf "Fertig".
    kbv_returnkey_continue Setzt den Titel der Rückgabetaste auf "Weiter".
    kbv_returnkey_emergency Setzt den Titel der Rückgabetaste auf "Notruf".
  • Autokapitalisierungstyp: Mit dieser Konstante wird festgelegt, wie die über die virtuelle Tastatur getippten Wörter automatisch großgeschrieben werden sollen. Die verfügbaren Autokapitalisierungsarten sind:
    Autokapitalisierung Typ Beschreibung
    kbv_autocapitalize_none Autokapitalisierung ist deaktiviert.
    kbv_autocapitalize_words Wörter werden automatisch großgeschrieben.
    kbv_autocapitalize_sentences Sätze werden automatisch großgeschrieben.
    kbv_autocapitalize_characters Alle Zeichen werden großgeschrieben.

Das letzte Argument ist das Aktivieren/Deaktivieren von Textvorhersage, und dies würde auf true gesetzt werden, um es zuzulassen, und andernfalls auf false, aber beachten Sie, dass nur weil es erlaubt ist, nicht bedeutet, dass es verwendet wird, da dies von den Einstellungen des Benutzers auf dem Gerät abhängt. Im Modus "Textvorhersage" erzeugt die virtuelle Tastatur keine normalen GameMaker Studio 2-Tastendruckereignisse. Stattdessen werden nur das zuletzt gedrückte Zeichen und die Tastaturstring-Variablen aktualisiert. Dies ist darauf zurückzuführen, dass der Unfähigkeit zu erkennen, ob eine Änderung im internen Textfeld, das zur Erkennung von Tastendrücken verwendet wird, von einer tatsächlichen Taste der virtuellen Tastatur oder von einem Textvorschlag stammt. In diesen Fällen würden Sie die keyboard_string-Eingabe lesen wollen, im Gegensatz zu als jede Art von roher Tasteneingabe zu lesen.

Es ist auch wichtig zu beachten, dass der Benutzer je nach Plattform-Betriebssystem unterschiedliche Tastaturen mit unterschiedlichen Fähigkeiten erhält, mit den folgenden Vorbehalten für die Verwendung pro Ziel:

  • UWP: Die UWP-Implementierung unterstützt nicht return_key_type, autocapitalization_type oder predictive_text_enabled. Die Übergabe dieser Parameter in eine Funktion führt zu keinen Fehlern, sie haben nur keine Auswirkungen auf die Tastatur.
  • Android / Amazon Fire: Unter Android wird der Tastaturtyp kbv_type_phone_name nicht unterstützt und zeigt stattdessen eine ASCII-Tastatur an, und der Rückgabetastentyp kann nur kbv_returnkey_go, kbv_returnkey_search, kbv_returnkey_next sein, kbv_returnkey_send oder kbv_returnkey_done sein. Wenn auf dieser Plattform andere Rückgabetastentypen verwendet werden, werden standardmäßig die unten aufgeführten Ersatzrückgabetasten verwendet:
    Nicht verfügbarer Rückgabeschlüssel Ersatztaste
    kbv_returnkey_google
    kbv_returnkey_yahoo    		 kbv_returnkey_search
    kbv_returnkey_join
    kbv_returnkey_route
    kbv_returnkey_emergency    		 kbv_returnkey_go
    kbv_returnkey_continue kbv_returnkey_next

  • AndroidTV/ FireTV: Benutzerdefinierte Return-Tastentypen werden auf ASCII-Tastaturen nicht unterstützt - es wird immer die Standard-Return-Taste angezeigt.
  • tvOS: Prädiktiver Text und automatische Großschreibung werden unter tvOS nicht unterstützt, und die Höhe der Tastatur, die von Systemereignissen und der Funktion keyboard_virtual_height() zurückgegebene Höhe der Tastatur entspricht standardmäßig der Bildschirmhöhe, da sie sich über den gesamten Bildschirm erstreckt und die Funktionen zur Rektaldimensionierung der Tastatur unter tvOS deaktiviert sind. Beachten Sie auch, dass physische (Bluetooth-)Tastatur-Ereignisse nicht gesendet werden, solange die virtuelle Tastatur geöffnet wurde.

Der Aufruf dieser Funktion erzeugt ein asynchrones Systemereignis, bei dem die async_load DS-Map mit den folgenden Schlüssel/Wert-Paaren aufgefüllt wird:

  • " event_type" - dies wird " virtualkeyboard status" sein, wenn es durch Aktionen der virtuellen Tastatur ausgelöst wird.
  • "screen_height" - die Höhe der virtuellen Tastatur (in Pixel). Dieser Wert ist 0, wenn die Tastatur unsichtbar ist.
  • "keyboard_status" - der aktuelle Status der Tastatur, zurückgegeben als eine der folgenden Zeichenketten:
    • "versteckt"
    • "versteckt"
    • "zeigend"
    • "sichtbar"

Aktion Syntax:

Argumente:

Argument Beschreibung
keyboard_type Legt den auf der virtuellen Tastatur verfügbaren Tastensatz fest.
return_key_type Legt fest, was auf der Return-/Aktionstaste der virtuellen Tastatur angezeigt wird.
autokapitalisierung_type Legt fest, wie/ob die über die virtuelle Tastatur eingegebenen Wörter automatisch großgeschrieben werden.
predictive_text_enabled Setzen Sie diese Option auf true/false, um die Texterkennung zu aktivieren/deaktivieren.
