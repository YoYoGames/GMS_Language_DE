Ladepuffer

Mit dieser Aktion können Sie einen Puffer laden, der zuvor mit der Aktion Puffer speichern gespeichert wurde. Die Aktion erstellt einen neuen Puffer, lädt die Datendatei in diesen und gibt den eindeutigen ID-Wert zur Identifizierung des Puffers an die Variable zurück. Sie geben den Namen der zu ladenden Datei (als String) und eine Variable an, in der die zurückgegebene Puffer-ID gespeichert werden soll. Beachten Sie, dass Sie, wenn Sie die Variable als temporäre lokale Variable kennzeichnen, sicherstellen müssen, dass Sie den Puffer, den diese Funktion erstellt, löschen, bevor Sie müssen sicherstellen, dass Sie den von dieser Funktion erstellten Puffer vor dem Ende des Aktionsskripts oder Ereignisses löschen, da Sie sonst den Verweis darauf verlieren und ein Speicherleck entsteht, wenn die Variable verworfen wird.

Aktion Syntax:

Argumente:

Argument Beschreibung
Dateiname Der Name (als String) der zu öffnenden Datei
Ziel Die Zielvariable, die den eindeutigen ID-Wert für den beim Laden erstellten Puffer enthält

Beispiel:

Der obige Code des Aktionsblocks erstellt eine Variable zur Aufnahme von Pufferdaten und prüft dann, ob eine Pufferspeicherdatei zuvor erstellt wurde. Wenn die Datei vorhanden ist, wird sie in die erstellte Puffervariable geladen. Wenn sie jedoch nicht vorhanden ist, wird sie erstellt, beschrieben und dann gespeichert, damit sie beim nächsten Mal geladen werden kann.
