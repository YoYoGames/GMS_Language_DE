Kacheldaten in Zelle setzen

Mit dieser Aktion können Sie die Kacheldaten für eine Kachelzelle auf einer Kachelkartenebene festlegen. Die Kacheldaten sind einfach ein Wert, der den Index der Kachel zusammen mit den Werten für Spiegeln, Drehen und Spiegeln wiedergibt, sowie alle benutzerdefinierten Kachelmasken, die Sie verwendet haben. Bei dieser Aktion geben Sie zuerst die Ebene an, die Sie anvisieren möchten, indem Sie den Namen der Ebene (eine Zeichenkette) wie im Raumeditor definiert angeben, und dann die Spalte und Zeile auf der Kachelplanschicht, für die die Kacheldaten zu setzen. Die "Zelle" ist der Bereich auf dem Kachelplanraster, der die Kachel enthält, für die Sie die Daten festlegen möchten. Wenn Ihre Kacheln also z. B. 16x16 sind und der Raum 1024x768 groß ist, hat der Kachelplan 64 Spalten und 48 Zeilen.

HINWEIS: Die gewählte Ebene muss im Raum-Editor als Kachelkartenebene definiert worden sein, andernfalls können Fehler auftreten.

Das letzte Argument für die Aktion sind die Kacheldaten selbst. Normalerweise würden Sie die Kacheldaten für die Kachelkartenzelle mit der Aktion Kacheldaten in Zelle holen abrufen und dann mit der Aktion Kacheldatentransformation setzen manipulieren, bevor Sie sie mit dieser Aktion wieder setzen. Weitere Informationen zu Kacheldaten finden Sie im GML-Abschnitt über Kachelkartenfunktionen.

Aktion Syntax:

Argumente:

Argument Beschreibung
Ebene Die Ebene mit der anzuvisierenden Kachelkarte
Säulen Die Zellenspaltenposition entlang der horizontalen Achse, um die Kacheldaten für
Zeile Die Position der Zellreihe entlang der vertikalen Achse zum Setzen der Kacheldaten
Daten Die Kacheldaten, die für die Einstellung der Zelle verwendet werden sollen

Beispiel:

Der obige Code des Aktionsblocks durchläuft jede Kachelzelle im Raum, holt die Kacheldaten für die Zelle ab, spiegelt sie und setzt die Zelle dann wieder.
