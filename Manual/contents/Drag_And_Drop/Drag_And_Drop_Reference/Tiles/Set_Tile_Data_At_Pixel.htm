Kacheldaten an Pixel setzen

Mit dieser Aktion können Sie die Kacheldaten für eine Kachelzelle setzen, die sich an einer bestimmten Position auf einer Kachelkartenebene befindet. Die Kacheldaten sind einfach ein Wert, der den Index der Kachel zusammen mit den Die Kacheldaten sind einfach ein Wert, der den Index der Kachel zusammen mit den Werten für "Spiegeln", "Kippen" und "Drehen" sowie den von Ihnen verwendeten benutzerdefinierten Kachelmasken wiedergibt. Bei dieser Aktion geben Sie zunächst die Ebene an, die Sie anvisieren möchten, indem Sie den Namen der Ebene (eine Zeichenkette), wie im Raumeditor definiert, und dann die x- und y-Achse im Raum für die die Kacheldaten gesetzt werden sollen. Beachten Sie, dass die Kacheldaten für die "Zelle" der Kachelkarte gesetzt werden, in der die angegebene Position liegt. Wenn Ihre Kacheln zum Beispiel 16x16 groß sind, enthält die Kachelkartenebene Kacheln in Zellen von 16x16 Pixeln. Wenn Sie diese Aktion verwenden, um die Raumposition (8, 8) anzuvisieren, ändern Sie die Kachel in der Zelle (0, 0) der Kachelübersicht.

HINWEIS: Die gewählte Ebene muss im Raum-Editor als Kachelkartenebene definiert worden sein, andernfalls können Fehler auftreten.

Das letzte Argument für die Aktion sind die Kacheldaten selbst. Normalerweise würden Sie die Kacheldaten für die Kachel-Map-Zelle mit der Aktion Kacheldaten bei Pixel holen abrufen und dann mit der Aktion Kacheldaten transformieren setzen manipulieren, bevor Sie sie mit dieser Aktion wieder setzen. Weitere Informationen zu Kacheldaten finden Sie im GML-Abschnitt über Kachelkartenfunktionen.

Aktion Syntax:

Argumente:

Argument Beschreibung
Ebene Die Ebene mit der anzuvisierenden Kachelkarte
X Die Position entlang der horizontalen (X-)Achse, für die die Kacheldaten gesetzt werden sollen
Y Die Position entlang der vertikalen (Y-)Achse, um die Kacheldaten zu setzen
Daten Die Kacheldaten, die zum Setzen der Zelle an der Position verwendet werden sollen

Beispiel:

Der obige Code des Aktionsblocks testet auf einen Mausklick und wenn einer empfangen wird ruft er die Kacheldaten für die Zelle ab, die mit der Mausposition übereinstimmt. Diese Daten werden dann geändert und an die Kachelkartenebene zurückgegeben, wobei die Kachel in dieser Zelle gedreht wird.
