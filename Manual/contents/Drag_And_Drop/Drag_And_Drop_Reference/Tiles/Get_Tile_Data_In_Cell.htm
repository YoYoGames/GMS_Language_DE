Kacheldaten in Zelle holen

Mit dieser Aktion können Sie die Kacheldaten für eine Kachelzelle auf einer Kachelkartenebene abrufen. Die Kacheldaten sind einfach ein Wert, der den Index der Kachel zusammen mit den Werten für Spiegeln, Drehen und Spiegeln wiedergibt, sowie alle benutzerdefinierten Kachelmasken, die Sie verwendet haben. Mit dieser Aktion geben Sie zunächst die Ebene an, die Sie anvisieren möchten, indem Sie den Namen der Ebene (eine Zeichenkette) wie im Raumeditor definiert angeben, und dann die Spalte und Zeile der Kachelkartenebene, für die Sie die Kacheldaten für. Die "Zelle" ist der Bereich auf dem Tile-Map-Raster, der die Kachel enthält, von der Sie die Daten abrufen möchten. Wenn Ihre Kacheln also z. B. 16x16 groß sind und der Raum 1024x768 groß ist, wird die Tile-Map 64 Spalten und 48 Zeilen haben.

HINWEIS: Die gewählte Ebene muss im Raum-Editor als Kachelkartenebene definiert worden sein, andernfalls können Fehler auftreten.

Der zurückgegebene Wert wird dann in der Zielvariablen gespeichert, die zuvor erstellt worden sein kann oder eine neue temporäre sein kann (wenn Sie das Kontrollkästchen "Temp" aktivieren). Weitere Informationen zu Kacheldaten finden Sie im GML-Abschnitt über Kachelkartenfunktionen.

Aktion Syntax:

Argumente:

Argument Beschreibung
Ebene Die Ebene, aus der die Kachel geholt werden soll
Säule Die Spalte (von links nach rechts) der Zelle

Beispiel:

Der obige Code des Aktionsblocks durchläuft jede Kachelzelle im Raum, holt die Kacheldaten für die Zelle ab, spiegelt sie und setzt die Zelle dann wieder.
