Kacheldaten bei Pixel abrufen

Mit dieser Aktion können Sie die Kacheldaten für eine Kachelzelle auf einer Kachel-Kartenebene von einer Position abrufen. Die Kacheldaten sind einfach ein Wert, der den Index der Kachel zusammen mit den Werten für "Spiegeln", "Drehen" und "Spiegeln" sowie allen benutzerdefinierten Kachelmasken wiedergibt, die Sie die Sie verwendet haben. Bei dieser Aktion geben Sie zuerst die Ebene an, die Sie anvisieren möchten, indem Sie den Namen der Ebene (eine Zeichenkette) wie im Raumeditor definiert angeben, und dann die x- und y-Achse im Raum, für die Sie die Kacheldaten erhalten möchten. Der zurückgegebene Kacheldatenwert stimmt überein mit was auch immer die Zelle der Kachelkarte an diesem Punkt sein mag. Wenn Ihre Kacheln beispielsweise 16x16 groß sind, enthält die Kachelkartenebene Kacheln in Zellen von 16x16 Pixeln; wenn Sie also mit dieser Aktion die Raumposition (8, 8) ansteuern, erhalten Sie die Kacheldaten für die Zelle (0, 0) der Kachelabbildung.

HINWEIS: Die gewählte Ebene muss im Raum-Editor als Kachelkartenebene definiert worden sein, andernfalls können Fehler auftreten.

Der zurückgegebene Wert wird dann in der Zielvariablen gespeichert, die zuvor erstellt worden sein kann oder eine neue temporäre sein kann (wenn Sie das Kontrollkästchen "Temp" aktivieren). Weitere Informationen zu Kacheldaten finden Sie im GML-Abschnitt über Kachelkartenfunktionen.

Aktion Syntax:

Argumente:

Argument Beschreibung
Ebene Die Ebene mit der anzuvisierenden Kachel-Map
X Die Position entlang der horizontalen (X-)Achse, für die die Kacheldaten gesetzt werden sollen
Y Die Position entlang der vertikalen (Y-)Achse, um die Kacheldaten zu setzen

Beispiel:

Der obige Code des Aktionsblocks testet auf einen Mausklick und wenn einer empfangen wird ruft er die Kacheldaten für die Zelle ab, die mit der Mausposition übereinstimmt. Diese Daten werden dann geändert und an die Kachelkartenebene zurückgegeben, wobei die Kachel in dieser Zelle gedreht wird.
