Der Geräte-Manager

Der Geräte-Manager wird durch Klicken auf auf die Bleistift-Schaltfläche innerhalb der Zielliste aufgerufen:

Wenn Sie auf diese Schaltfläche klicken, wird der Geräte-Manager geöffnet, in dem Sie verschiedene Geräte für die Zielplattformen konfigurieren können, die Sie kompilieren und testen möchten. und testen möchten (beachten Sie, dass die verfügbaren Plattformen von Ihrer Lizenz abhängen und möglicherweise nicht alle sichtbar sind). Im Folgenden finden Sie Details zur Konfiguration von Geräten für jede der Zielplattformen:

Android

Hier können Sie auf die Schaltfläche Neues Gerät hinzufügen klicken, um ein neues Android-Gerät manuell zur Liste hinzuzufügen, oder Sie können auf die Schaltfläche Gerät erkennen klicken, damit GameMaker Studio 2 automatisch alle Geräte erkennt, die über eine Kabelverbindung angeschlossen sind. Sie können auch ein Android Virtual Device einrichten, indem Sie auf die Schaltfläche AVD ausführen. Dadurch wird der Android Virtual Device Manager geöffnet, in dem Sie ein neues AVD erstellen können (siehe hier für weitere Informationen). Sobald Sie ein AVD erstellt haben, müssen Sie es aus dem AVD-Manager starten und dann in GameMaker Studio 2 auf die Schaltfläche Gerät erkennen klicken und es sollte automatisch zur Geräteliste hinzugefügt werden:

Sobald Sie ein Gerät (physisch oder virtuell) hinzugefügt haben, können Sie dann die Verbindung über die Schaltfläche Schaltfläche Verbindung testen überprüfen oder es wieder aus der Liste entfernen, indem Sie die Schaltfläche Gerät entfernen.


HTML5

Der HTML5-Gerätemanager dient dazu, verschiedene Browser zur Geräteliste hinzuzufügen, um sie bei der Ausführung anzusprechen. Für den Anfang sollten Sie auf die Schaltfläche Browser erkennen klicken, wodurch Geräte für jeden der auf Ihrem Rechner installierten Browser erstellt werden sollten. Wenn Ihr Browser nicht aufgelistet ist, können Sie auch auf die Schaltfläche Neuen Browser hinzufügen klicken und dann dem Gerät einen Namen geben und GameMaker Studio 2 auf den Pfad für die ausführbare Datei des Browsers verweisen.

Die zusätzlich verfügbaren Felder können eingerichtet werden, um zu steuern, wie der Browser Ihre Projekte ausführt, wobei die folgenden Optionen zur Verfügung stehen:

  • Browser-Befehlszeilenargumente: Hier können Sie beliebige Kommandozeilenargumente eintragen, die an den Browserpfad angehängt werden sollen. Eine Liste dieser Argumente finden Sie unter den folgenden Links zu den wichtigsten gängigen Browsern:
    • Chromium (Chrom)
    • Firefox
    • Internet Explorer
  • Web-Server-Adresse: Die Adresse des Webservers Servers (Standard ist 127.0.0.1). Möglicherweise müssen Sie dies je nach den Spezifikationen der verschiedenen APIs ändern, z. B. muss dies für Facebook auf http://localhost gesetzt werden (und dann in der App-Konsole auf die Whitelist gesetzt werden).
  • Webserver-Port: Legen Sie die Portnummer für den Webserver ein (Standard ist 51264).
  • Erlaubte Webserver-IPs / CIDR-IP-Bereiche: Stellen Sie die erlaubten Webserver, Hostnamen und CIDR-IP-Bereiche nach Bedarf ein.

iOS

Bevor Sie Ihre iOS-Geräte einrichten können, müssen Sie müssen Sie Ihr Mac-Gerät einrichten und dessen Verbindung testen (siehe unten für Details). Sobald das geschehen ist, können Sie den Mac, den Sie ansteuern möchten, aus dem Dropdown-Menü oben im Fenster auswählen und dann auf die Schaltfläche Gerät erkennen klicken, damit GameMaker Studio 2 automatisch alle angeschlossenen iOS-Geräte, einschließlich Simulatoren, findet. Sobald eine Verbindung besteht, werden die Gerätedetails in der Liste in diesem Fenster angezeigt, und Sie können den Gerätenamen bearbeiten, um ihn benutzerfreundlicher zu gestalten (diese Änderung wirkt sich auf den Namen aus). tvOS

Bevor Sie Ihre tvOS-Geräte einrichten können, müssen Sie Ihr Mac-Gerät eingerichtet und seine Verbindung getestet haben (siehe unten für Details). Sobald das geschehen ist, können Sie den Mac, den Sie ansteuern möchten, aus dem Dropdown-Menü oben im Fenster auswählen und dann auf die Schaltfläche Gerät erkennen klicken, damit GameMaker Studio 2 automatisch alle angeschlossenen tvOS-Geräte, einschließlich Simulatoren, findet. Sobald die Verbindung hergestellt ist, werden die Gerätedetails in der Liste in diesem Fenster angezeigt, und Sie können den Gerätenamen bearbeiten, um ihn benutzerfreundlicher zu gestalten (diese Änderung wirkt sich auf die Sie können den Gerätenamen bearbeiten, um ihn benutzerfreundlicher zu gestalten (diese Änderung wirkt sich nur darauf aus, wie das Gerät im Target Manager dargestellt wird, und auf nichts anderes).


macOS

Um ein Mac-Gerät zum Manager hinzuzufügen, müssen Sie zunächst auf auf die Schaltfläche mit der Aufschrift Neues Gerät hinzufügen klicken, wodurch das Gerät hinzugefügt wird und einige leere Felder erstellt werden, die Sie ausfüllen müssen, um die Verbindung zu aktivieren.

Zuerst müssen Sie das Gerät benennen , damit Sie es in der Zielliste identifizieren können, und geben dann den Hostnamen (oder IP) des Macs angeben. Anschließend müssen Sie den Benutzernamen des Benutzers, mit dem Sie eine Verknüpfung herstellen möchten, und das Kennwort , das sie verwendet haben. Wenn Sie möchten, können Sie auch den Installationsordner auf dem Mac dorthin ändern, wo GameMaker Studio 2 die Build-Dateien für Ihre Projekte speichern soll.

Wenn das alles erledigt ist, können Sie auf die Schaltfläche Verbindung testen klicken, um sicherzustellen, dass GameMaker Studio 2 mit dem Mac kommunizieren kann.

HINWEIS: Wenn Sie für iOS entwickeln, müssen Sie dies zuerst getan haben und eine funktionierende Verbindung zu dem Mac herstellen, den Sie für die Entwicklung verwenden möchten.

HINWEIS: Sie müssen aktiviert haben Remote-Anmeldung auf dem Ziel-Mac aktiviert haben. Dies können Sie tun, indem Sie zu Voreinstellungen -> Freigabe und Aktivieren Sie Remote Login für den Benutzer, der sich anmelden können soll.

Ubuntu(Linux)

Um ein Ubuntu(Linux)-Gerät zum Manager hinzuzufügen, müssen Sie zunächst auf die Schaltfläche Neues Gerät hinzufügen klicken. Dadurch wird das Gerät hinzugefügt und es werden einige leere Felder erstellt, die Sie ausfüllen müssen, um die Verbindung zu aktivieren.

Zuerst müssen Sie das Gerät benennen , damit Sie es in der Zielliste identifizieren können, und dann den Hostnamen (oder die IP) des Ubuntu-Rechners (oder der VM) angeben. Dann müssen Sie den Benutzernamen des Benutzers angeben, den Sie verknüpfen möchten, und das Kennwort , das sie verwendet haben. Außerdem müssen Sie einen Installationsordner auf dem Ubuntu-Rechner angeben, den Sie für die Installation Ihrer Projekte verwenden möchten. Wenn das alles erledigt ist, können Sie auf auf die Schaltfläche Verbindung testen, um sicherzustellen, dass GameMaker Studio 2 mit dem Ubuntu-Rechner kommunizieren kann.

Beachten Sie, dass die endgültige ausführbare Datei, die von GameMaker erstellt wird, von der Architektur des angepeilten Geräts abhängt, die entweder x64 oder ARM sein kann. Wenn Sie zum Beispiel einen Raspberry Pi 3 oder 4 anschließen, wird die erstellte ausführbare Datei eine 32-Bit-ARM-Datei sein, und wenn Sie einen Linux-PC anschließen, wird es eine 64-Bit-x64-Datei sein.

Windows UWP

Bevor Sie ein Gerät für das Windows 10 UWP-Ziel hinzufügen können, müssen Sie den Entwicklermodus aktiviert haben und das Geräteportal auf dem Gerät einrichten. Wenn Sie dies nicht tun, kann GameMaker Studio 2 weder mit dem Gerät kommunizieren mit dem Gerät kommunizieren und Ihre UWP-Apps korrekt erstellen.

Der einfachste Weg, den Entwicklermodus zu aktivieren, ist, einfach " Entwicklermodus" in die Windows-Suche einzugeben und dann auf den angezeigten Eintrag " Einstellungen" zu klicken. Daraufhin öffnet sich das folgende Fenster:

Sie müssen die Option " Entwicklermodus" auswählen (was möglicherweise erfordert, dass Windows einige zusätzliche Pakete herunterlädt und einige zusätzliche Pakete herunterladen und installieren muss), und wählen Sie dann die Option darunter, Geräteportal aktivieren. Das Geräteportal ermöglicht es GameMaker Studio 2, mit dem Windows 10 PC zu kommunizieren, und wenn es aktiviert ist, müssen Sie einen Benutzernamen und ein Passwort eingeben:

Beachten Sie, dass der Benutzername und das Kennwort für das Portal eindeutig sein sollten und nicht denen entsprechen dürfen, die von Ihrem Microsoft-Konto oder anderen Windows-Anmeldeinformationen. Wenn Sie diese Angaben gemacht haben, klicken Sie auf die Schaltfläche Übernehmen , schließen Sie den Einstellungsmanager und kehren Sie zu GameMaker Studio 2 zurück.

Weitere Informationen über das Geräteportal finden Sie in diesem Microsoft-Artikel: Geräteportal für Desktop.

Sie können nun den Geräte-Manager in GameMaker Studio 2 öffnen und ein Gerät (oder mehrere Geräte) zur Verwendung einrichten:

Um ein Windows 10 UWP-kompatibles Gerät zum Manager hinzuzufügen, müssen Sie zunächst auf die Schaltfläche " Neues Gerät hinzufügen " klicken. Dadurch wird das Gerät hinzugefügt und es werden einige leere Felder erstellt, die Sie ausfüllen müssen, um die Verbindung zu aktivieren.

Zuerst müssen Sie das Gerät benennen , damit Sie es in der Zielliste identifizieren können, und dann den Host angeben - dies ist die IP-Adresse des Windows 10-Geräts im Netzwerk, das das Spiel hosten soll, und muss mit dem Suffix " :50443" versehen werden, um die Portnummer festzulegen. Danach fügen Sie den Benutzernamen und das Passwort hinzu, die Sie im Geräteportal eingerichtet haben (wie zuvor erwähnt). Sie können die PIN leer lassen, da sie für UWP Windows 10 Desktop nicht erforderlich ist.

Für spezifischere Einrichtungsdetails, die die verschiedenen UWP-Ausgaben (Desktop, XboxOne) abdecken, besuchen Sie bitte den YYG Helpdesk: Einrichten von UWP.

Alle verschiedenen Gerätefenster haben am unteren Rand eine Option zum Löschen von Geräten, die, wenn sie angeklickt wird, alle angeschlossenen Geräte aus der Liste entfernt.
