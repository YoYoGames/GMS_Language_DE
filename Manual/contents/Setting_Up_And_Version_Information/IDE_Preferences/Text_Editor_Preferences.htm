Texteditor Editor-Voreinstellungen

Mit den Voreinstellungen für die Texteditoren wird festgelegt, wie die verschiedenen Code-/Skripteditoren aussehen und sich anfühlen. Die Hauptseite hat die folgenden Optionen:

  • Verzögerung der Syntaxprüfung (ms): Die Verzögerungszeit (in Millisekunden), bevor der Syntax-Checker läuft. Der Standardwert ist 2000 ms.
  • Code Completion Delay (ms): Die Verzögerungszeit (in Millisekunden)
  • Skripte im Vollbild-Editor öffnen: Wenn Sie diese Option aktivieren, werden Skript-Ressourcen automatisch in einem neuen Arbeitsbereich geöffnet. Wenn diese Option nicht aktiviert ist, werden sie in einem Fenster innerhalb des aktuellen Arbeitsbereichs geöffnet. Diese Option ist standardmäßig deaktiviert. standardmäßig deaktiviert.
  • Shader im Vollbild-Editor öffnen: Wenn Sie diese Option aktivieren, werden die Shader-Ressourcen automatisch in einem neuen Arbeitsbereich geöffnet. Wenn diese Option nicht aktiviert ist, werden sie in einem Fenster innerhalb des aktuellen Arbeitsbereichs geöffnet. Diese Option ist standardmäßig nicht standardmäßig deaktiviert.
  • Objekt-Events im Vollbild-Editor öffnen: Wenn Sie diese Option aktivieren, werden Objekt-Event-Code-Editoren automatisch in einem neuen Arbeitsbereich geöffnet. Wenn diese Option nicht aktiviert ist, werden sie in einem Fenster geöffnet, das mit dem Objekt-Event-Editor verkettet ist. Diese Option ist standardmäßig nicht aktiviert.
  • Event-Skripte als Registerkarten im selben Fenster öffnen: Beim Hinzufügen von Code oder beim Ziehen und Ablegen zu einem Ereignis wird das Codefenster mit dem Ereignisfenster verkettet, und wenn diese Option aktiviert ist, werden alle weiteren Ereignisse, die zur Bearbeitung geöffnet werden demselben Fenster als Registerkarten am oberen Rand hinzugefügt. Wenn Sie diese Option deaktivieren, wird jedes Ereignis als ein eigenes verkettetes Fenster geöffnet. Diese Option ist standardmäßig aktiviert.
  • Verknüpfte Scripts als Registerkarten im selben Fenster öffnen: Wenn Sie mehrere verknüpfte Scripts haben und diese öffnen, werden sie standardmäßig im selben Fenster als Registerkarten gruppiert. Wenn Sie diese Option deaktivieren, werden sie jeweils in einem eigenen Fenster/Arbeitsbereich geöffnet. Diese Option ist standardmäßig aktiviert.
  • Texteditor bei Fokus in Ansicht schwenken: Wenn Sie einen Texteditor von einem Objekt-Event aus öffnen, wird bei Auswahl dieser Option der Arbeitsbereich automatisch geschwenkt, um den Editor innerhalb der Begrenzungen zu bringen. Diese Option ist standardmäßig deaktiviert.
  • Standardbreite des Codefensters: Legen Sie die Standardbreite (in Pixel) fest, die das Codefenster beim Öffnen haben soll. Der Standardwert ist 900.
  • Standardhöhe des Codefensters: Legen Sie die Standardhöhe (in Pixel) fest, die das Codefenster beim Öffnen haben soll. Der Standardwert ist 600.
  • Minimale Codefensterbreite: Legen Sie die minimale Breite (in Pixel) fest, die das Codefenster haben darf. Der Standardwert ist 300.
  • Minimale Höhe des Codefensters: Legen Sie die minimale Höhe (in Pixel) fest, die das Codefenster haben darf. Der Standardwert ist 200.

Neben den oben genannten Optionen gibt es auch eine Unterkategorie für die Einstellungen des Code-Editors:

Code-Editor

Die Farbeinstellungen werden verwendet, um zu ändern, wie der Code- und Skript-Editor verschiedene Texte anzeigt. Jeder Eintrag hat eine Reihe von Untereinträgen, mit denen Sie genau festlegen können, wie jeder einzelne Text aussehen soll, wie im Beispiel Bild oben gezeigt. Im Allgemeinen können Sie die Farbe, die Schriftart, die Größe und den Stil für jede der verschiedenen Optionen einstellen. Weiter unten können Sie auch ein allgemeineres Styling für Dinge wie die Zeilenauswahl von Codeklammern usw. einstellen.

Unter den Farbeinstellungen haben Sie die verschiedenen Styling-Optionen. Mit diesen Voreinstellungen wird das Verhalten des Code-/Skript-Editors festgelegt, wobei die folgenden Optionen zur Verfügung stehen:

  • Tabulatoren beibehalten: Wenn diese Option aktiviert ist, werden gedrückte Tabulatortasten als Tabulatorzeichen - ' \t ' - im Editor, während nicht markiert bedeutet, dass Tabulatoren in Leerzeichen umgewandelt werden. Die Voreinstellung ist aus.
  • Tabulatorgröße: Die Anzahl der Leerzeichen, um die der Code durch Drücken von "Tab" eingerückt werden soll. Standard ist 4.
  • Whitespace anzeigen: Anzeige " . ", um Whitespace anzuzeigen (siehe Abbildung unten). Die Voreinstellung ist aus.
  • Zeilennummern anzeigen: Ob die Zeilennummern unten links angezeigt werden sollen oder nicht. Die Voreinstellung ist an.
  • Tabulatorzeilen anzeigen: Anzeige " .... ", um Tabulatorzeilen anzuzeigen. Die Voreinstellung ist aus.
  • Aktivieren Sie die Farbcodierung: Aktivieren oder deaktivieren Sie die Farbcodierung für den Editor. Wenn die Option "Aus" aktiviert ist, wird der Code nicht gestylt, während er bei "Ein" mit der oben im Abschnitt "Farbe" angegebenen Formatierung gestylt wird. Die Voreinstellung ist ein.
  • Dokumentzeilenlimit: Legen Sie die maximale Anzahl von Zeilen für ein bestimmtes Editor-Dokument fest. Der Standardwert ist 0 (0 oder weniger ist praktisch unendlich) und jeder andere Wert begrenzt die zulässigen Zeilen.
  • Einrückung bei neuen Zeilen anpassen: Aktivieren oder deaktivieren Sie die automatische Einrückung für Ihren Code. Die Standardeinstellung ist "Ein", wodurch alle in einer Zeile gesetzten Tabulatoren beibehalten werden, wenn eine neue Zeile hinzugefügt wird.
  • Automatisches Einrücken nach Klammern: Aktivieren oder deaktivieren Sie die automatische Einrückung für Ihren Code, wenn Sie geschweifte Klammern hinzufügen {} . Standardmäßig ist diese Option aktiviert, wodurch die nächste Codezeile mit einem Tabulator versehen wird, wenn eine neue Klammer { und eine neue Zeile hinzugefügt werden.
  • Aktivieren Sie die Codevervollständigung: Aktivieren oder deaktivieren Sie das Codevervollständigungsfenster. Wenn Sie Code schreiben und dies aktiviert ist, erhalten Sie ein Popup-Fenster, das mögliche Funktionen anzeigt, die Sie basierend auf dem aktuellen Eingabetext verwenden können. Die Voreinstellung ist ein.
  • Funktionsklammern automatisch hinzufügen: Aktivieren oder deaktivieren Sie das automatische Hinzufügen von Klammern () wenn Sie die Autovervollständigung verwenden. Wenn Sie Code schreiben und dies aktiviert ist, fügt die IDE automatisch die beiden Klammern () ein, die eine Funktion benötigt, wobei der Cursor innerhalb der Klammern platziert wird, wenn die Funktion Argumente annimmt, oder hinter den Klammern, wenn sie keine Argumente annimmt. Diese Funktion funktioniert nur, wenn die Funktion aus der Autovervollständigung ausgewählt wurde. Standardmäßig ist sie eingeschaltet.
  • Undo / Redo Stack Limit: Hier können Sie das Undo/Redo Stack Limit einstellen. Alles unter 0 ist praktisch unendlich (begrenzt durch den verfügbaren Speicher), während positive Werte den Stapel auf diese Zahl begrenzen und jede Aktion am Ende des Stapels entfernen, wenn das Limit erreicht ist. Aktion am Ende des Stapels entfernt, wenn das Limit erreicht ist. Die Voreinstellung ist -1.
  • Auswahlübereinstimmungen hervorheben: Damit wird jeder Codeabschnitt hervorgehoben, der an anderer Stelle dupliziert wird. Wenn Sie also z. B. eine Funktion auswählen, werden alle Instanzen dieser Funktion im aktuellen Codefenster ebenfalls hervorgehoben auch. Standardmäßig ist diese Option eingeschaltet.
  • Farbcodierung/Codevervollständigung für veraltete Funktionen: Damit wird ein veralteter Codeabschnitt hervorgehoben. Standardmäßig ist dies eingeschaltet.
