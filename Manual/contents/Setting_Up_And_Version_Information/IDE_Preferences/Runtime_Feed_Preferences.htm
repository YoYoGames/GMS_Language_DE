Laufzeit-Feed-Voreinstellungen

Die Runtime Feeds Preferences dienen der Anzeige und Änderung der aktuell installierten Laufzeit als auch die aktuell verwendete Runtime. In GameMaker Studio 2 können mehrere verschiedene Runtime-Tools gleichzeitig installiert sein, und Sie können in dieser Voreinstellung festlegen, welches davon verwendet werden soll. Idealerweise sollten Sie die Idealerweise möchten Sie die aktuellste Runtime verwenden, aber manchmal kann es ein Problem geben oder Sie müssen aus Gründen der Vergleichbarkeit eine ältere Runtime verwenden usw... und deshalb können Sie hier die Runtime-Tools ändern, die GameMaker Studio 2 zum Erstellen Ihres Spiels verwendet.

Der erste Abschnitt dieser Voreinstellung bezieht sich darauf, woher Sie die Runtime-Tools beziehen, und erfordert die Eingabe einer URL zu einem RSS-Feed. Dieser Feed wird dann nach Updates für die Runtime-Tools durchsucht, die dann heruntergeladen und installiert werden können. In diesem Abschnitt wird standardmäßig die URL des Master-Laufzeit-Feed angezeigt, aber Sie können auf die Schaltfläche Hinzufügen klicken, um einen neuen Laufzeit-Feed zu erstellen, bei dem Sie den Namen und den URL für den RSS-Feed angeben und dann auf Anwenden klicken, um GameMaker Studio 2 zu sagen, dass es darauf zugreifen soll. Sie müssen das Fenster "Voreinstellungen" schließen und erneut öffnen, damit der neue Feed angezeigt wird.

Sobald ein Runtime-Feed erstellt wurde, wird er in diesem Abschnitt als Unterkategorie aufgeführt, und Sie sollten immer mindestens einen Feed aufgeführt haben (standardmäßig wäre dies der Master-Feed ). Die Runtime Feed-Voreinstellungen dienen zur Auswahl, Installation und Deinstallation der verschiedenen Runtime-Tools. Unter oben sehen Sie das aktuell verwendete Runtime-Tool, das Datum seiner Erstellung und den Speicherort auf der Festplatte. Unterhalb dieser Angaben finden Sie eine Liste aller verfügbaren Runtime-Tools, aus der Sie ein beliebiges auswählen können, um es als "laufendes Programm" zu verwenden. Sie können eine beliebige Laufzeit aus dieser Liste auswählen, um sie als "laufende" Laufzeit für die Kompilierung Ihrer Projekte zu verwenden.

Um eine andere Runtime zu verwenden, suchen Sie diese einfach in der Liste und doppelklicken Sie auf . Wenn die Laufzeitumgebung noch nicht heruntergeladen und installiert wurde, werden Sie dazu aufgefordert. Sobald sie installiert ist, werden Sie aufgefordert, die IDE zu schließen und neu zu starten, damit die neue Laufzeitumgebung initialisiert werden kann.

Diese Liste enthält alle verfügbaren Laufzeiten, ihre Versionsnummer, das Veröffentlichungsdatum und - wenn sie installiert sind - den Pfad, in dem sie installiert wurden. Auf der rechten Seite werden außerdem die verschiedenen Statussymbole angezeigt, die wie folgt lauten:

Dies markiert die aktuelle Laufzeit, die für die Verwendung beim Erstellen Ihrer Projekte ausgewählt wurde.
Wenn Sie darauf klicken, wird der Download und die Installation der ausgewählten Runtime gestartet (eine Statusleiste wird oben rechts in der IDE angezeigt).
Wenn Sie hierauf klicken, wird die ausgewählte Runtime von Ihrer Festplatte entfernt.

Sie können die Laufzeitliste jederzeit aus dem angegebenen RSS-Feed aktualisieren, indem Sie auf die Schaltfläche "Aktualisieren" klicken, und Sie können Ihre Festplatte von alten Laufzeiten bereinigen, indem Sie auf die Schaltfläche "Runtimes jetzt bereinigen "...
