Plugin-Voreinstellungen

Die Plugin-Voreinstellungen werden verwendet, um alle zusätzlichen Plugins zu steuern, die der GameMaker Studio 2 IDE hinzugefügt wurden. Standardmäßig gibt es hier einen Abschnitt für das Source Control Plugin zur Integration von GameMaker Studio 2-Projekte mit Git.

Versionskontrolle(Git)

Dieser Abschnitt befasst sich mit den Einstellungen, die Sie vornehmen müssen, um die Git SCM-Plugin mit GameMaker Studio 2 zu verwenden. Zunächst haben Sie den Satz von G eneral Optionen:

  • Quellsteuerungs-Ausgabefenster automatisch öffnen: Wenn dies aktiviert ist, wird das Quellsteuerungs-Ausgabefenster automatisch angezeigt, wenn sich etwas ändert. Dies ist standardmäßig eingeschaltet.
  • Ausführliche Protokollierung: Aktivieren Sie die ausführliche Protokollierung für das Git-Ausgabeprotokoll. Diese Option ist standardmäßig deaktiviert.

Als nächstes folgt der Abschnitt Benutzeridentität mit den folgenden Feldern, die ausgefüllt werden müssen:

  • Benutzername zum Übertragen als: Hier müssen Sie den Benutzernamen eingeben, mit dem Sie alle Übertragungen an Git vornehmen werden (dies muss derselbe sein wie der Benutzername, den Sie beim Einrichten Ihres Git-Repositorys angegeben haben).
  • E-Mail zum Übertragen als: Die E-Mail, die mit dieser Git-Identität verknüpft ist.

Nachdem Sie die Benutzeridentität eingerichtet haben, können Sie eine neue Benutzername/Kennwort-Kombination zur Authentifizierung hinzufügen, indem Sie auf die Schaltfläche mit der Aufschrift Neue Benutzer/Passwort-Authentifizierung hinzufügen klicken, wodurch sich das folgende Fenster mit verschiedenen Feldern öffnet, die Sie ausfüllen können:

Sie können auch ein neues SSH-Schlüsselpaar zur Authentifizierung hinzufügen, indem Sie auf die Schaltfläche mit dieser Bezeichnung klicken. Daraufhin wird das folgende Fenster geöffnet, das Sie ausfüllen können:

Unterhalb dieser Schaltflächen können Sie dann das Merge-Tool und das Diff-Werkzeug für SCM mit diesen Optionen:

  • Pfad zum Zusammenführungswerkzeug: Hier geben Sie den vollständigen Dateipfad zum Git-Zusammenführungswerkzeug an.
  • Optionen für das Verschmelzungswerkzeug: In diesem Feld können Sie zusätzliche Befehle hinzufügen, die ausgeführt werden sollen, wenn Sie das Zusammenführungswerkzeug verwenden.
  • Pfad zum Diff-Tool: Hier geben Sie den vollständigen Dateipfad zum Git-Diff-Tool an.
  • Optionen für das Diff-Tool: In diesem Feld können Sie zusätzliche Befehle hinzufügen, die bei Verwendung des Diff-Tools ausgeführt werden sollen.

Nachdem Sie diese eingerichtet haben, haben Sie noch eine letzte Möglichkeit:

  • Versionskontrolle automatisch aktivieren: Wenn dies aktiviert ist, wird die Versionskontrolle für alle neuen Projekte automatisch aktiviert. In der Voreinstellung ist dies deaktiviert.
