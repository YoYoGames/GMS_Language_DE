Dateiüberwachungs-Voreinstellungen

Der Datei-Überwacher ist die Bezeichnung für den Teil der IDE, der die Dateien eines Projekts auf Änderungen durch externe Software (z. B.: Bearbeiten eines Bildes in einem externen Editor) oder durch interne Änderungen wie das Bearbeiten eines Skripts Asset. Im Allgemeinen sollten diese Einstellungen nicht bearbeitet werden müssen, aber auf einigen Systemen - z. B. solchen, die eine SSD als Speicherlaufwerk für die Projektdateien verwenden - kann der Datei-Überwacher Sie über Änderungen informieren, wenn Sie Dinge tun wie das einfache Speichern des Projekt speichern oder wenn Sie ein Projekt-Asset bearbeiten. In diesem Fall müssen Sie diese Einstellungen möglicherweise ändern.

Die verfügbaren Einstellungen sind:

  • Datei-Überwacher aktivieren: Hier können Sie das Dateiüberwachungssystem aktivieren oder deaktivieren. Wenn dies deaktiviert ist, wird die IDE keine Dateien mehr erkennen, die von externen Quellen geändert wurden, daher ist Vorsicht geboten, wenn Sie diese Funktion deaktivieren. diese Funktion. Der Datei-Überwacher ist standardmäßig aktiviert.
  • Stellen Sie die Toleranz des Dateiüberwachers in MS nach dem Speichern in der IDE ein: Hier wird die Zeit zwischen dem Speichern einer Datei und der Überprüfung des Dateiüberwachers auf Änderungen eingestellt. Sie würden diesen Wert anpassen, wenn Sie Ihr Projekt speichern und der Dateiüberwacher Sie würden diesen Wert anpassen, wenn Sie Ihr Projekt speichern und der Dateiwächter fälschlicherweise anzeigt, dass sich etwas geändert hat; in diesem Fall würden Sie die Verzögerung erhöhen, bis die Meldung nicht mehr erscheint. Der Standardwert ist 2000ms, kann aber von 1ms bis zu 10.000ms reichen.
