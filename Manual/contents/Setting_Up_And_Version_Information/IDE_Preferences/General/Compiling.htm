Kompilieren von Voreinstellungen

Die Voreinstellungen für das Kompilieren befassen sich mit bestimmten Optionen, die sich auf die Ausgabe beziehen, wenn Sie Ihr Spiel zum Testen oder zum Erstellen eines Pakets kompilieren, sowie mit bestimmten Bereichen des Kompiliervorgangs. Die Optionen sind:

  • Automatisches Öffnen des Syntaxfehlerfensters bei Erkennung: Aktivieren oder deaktivieren Sie die Benachrichtigung über Syntaxfehler beim Kompilieren. Dies ist standardmäßig ausgeschaltet.
  • Zeigen Sie das Kompilierfenster während der Kompilierung an: Standardmäßig wird das Kompilierfenster angezeigt, wenn Sie Ihre Projekte kompilieren, auch wenn das Kompilierdokument geschlossen ist (es wird für die Kompilierung geöffnet). Wenn Sie dieses Kontrollkästchen deaktivieren, wird dieses Verhalten ausgeschaltet und das Kompilierfenster wird angezeigt, wenn es geöffnet ist.
  • Compiler-Befehle in der Compiler-Ausgabe anzeigen: Dies ist standardmäßig aktiviert und zwingt den Compiler, alle relevanten Befehle im Kompilierfenster anzuzeigen. Wenn Sie dieses Kontrollkästchen deaktivieren, wird die Ausgabe kleiner und enthält weniger Informationen.
  • Ausführliche Compiler-Ausgabe anzeigen: Wenn Sie diese Option aktivieren, wird der Compiler gezwungen, zusätzliche Befehle im Zusammenhang mit dem Compiliervorgang anzuzeigen. Im Allgemeinen ist dies nur beim Debuggen erforderlich und kann vom Support angefordert werden, wenn Sie ein Problem haben. Sie ist standardmäßig ausgeschaltet (und dies ist die empfohlene Einstellung).
  • Subst für Laufwerkspfade verwenden: Wenn diese Option aktiviert ist, erstellt GameMaker Studio 2 virtuelle Laufwerke, die als Ersatz für vollständige Dateipfade verwendet werden, und entfernt sie wieder, wenn sie nicht mehr benötigt werden. Diese virtuellen Laufwerke umgehen jedes mögliche Problem Diese virtuellen Laufwerke umgehen mögliche Probleme mit Dateipfaden, die für das Betriebssystem zu lang sind (insbesondere beim Kompilieren für Android oder bei der Verwendung von Erweiterungen), und es wird empfohlen, diese Option zu aktivieren. Wenn Sie jedoch Probleme damit haben, dass GameMaker Studio 2 virtuelle Laufwerke auf diese Weise erstellt, können Sie können Sie die Option deaktivieren. Standardmäßig ist diese Option aktiviert.
  • Paket bei Build-Abschluss im Explorer anzeigen: Wenn diese Option aktiviert ist, öffnet GameMaker Studio 2 das Fenster des Betriebssystem-Explorers an dem Ort, an dem das endgültige ausführbare Paket erstellt wurde. Sie ist standardmäßig aktiviert.
  • Ausgabefenster beim Start der Kompilierung löschen: Standardmäßig löscht das Compiler-Fenster zu Beginn jedes Builds alle vorherigen Meldungen aus dem Compiler-Ausgabefenster, wenn Sie dies jedoch deaktivieren, bleiben die vorherigen Compiler-Protokolle erhalten. Dies ist standardmäßig aktiviert und ist die empfohlene Konfiguration, da große Compilerprotokolle die Kompilierzeiten verlangsamen können.
  • Maximale parallele Kompilieraufträge: Dieser Wert legt die maximale Anzahl der CPU-Kerne fest, die für die Kompilierung eines Projekts verwendet werden sollen. Der Standardwert ist 8 und im Allgemeinen sollten Sie diesen Wert nicht ändern müssen, aber wenn Sie auf einem Laptop oder einem Wenn Sie jedoch einen Laptop oder einen niedrigeren Rechner verwenden, sollten Sie diesen Wert eventuell herabsetzen. Der Standardwert ist auf 8 eingestellt.
