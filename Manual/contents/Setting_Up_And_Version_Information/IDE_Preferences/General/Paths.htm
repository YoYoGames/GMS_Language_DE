Pfadvorgaben

Die Pfadvorgaben steuern die verschiedenen Dateipfade, die von Elementen in der GameMaker Studio 2-IDE sowie für externe Editoren benötigt werden, die Sie benötigen. Die folgenden Optionen sind für die IDE vorhanden:

  • Temp-Verzeichnis: Der Ort, an dem alle temporären Verzeichnisse gespeichert werden. Unter Windows ist dies standardmäßig

    \Benutzer\<Benutzername>\AppData\Local\Temp\GameMakerStudio2

    Und unter macOS ist es:

    /var/ordner/<Pfad_Hash>/GameMakerStudio2

  • IDE-Cache-Verzeichnis: Der Speicherort für den IDE-Cache. Standardmäßig ist dies unter Windows

    \Benutzer\<Benutzername>\AppData\Roaming\GameMakerStudio2\Cache

    Und unter macOS ist es:

    /Benutzer/<Benutzername>/.config/GameMakerStudio2/Cache

  • Asset-Cache-Verzeichnis: Der Speicherort für den Asset-Cache für jedes Projekt. Unter Windows ist dies standardmäßig

    \Benutzer\<Benutzername>\AppData\Roaming\GameMakerStudio2\Cache

    Und unter macOS ist es:

    /Benutzer/<Benutzername>/.config/GameMakerStudio2/Cache

  • Speicherort "Meine Projekte": Der Ort, an dem GameMaker Studio 2 anfänglich neue Projekte erstellt. Unter Windows ist dies standardmäßig

    \Benutzer\<Benutzername>\Dokumente\GameMakerStudio2

    Und unter macOS ist es

    /Benutzer/<Benutzername>/GameMakerStudio2

  • Temp-Verzeichnis beim Schließen automatisch löschen: Wenn Sie diese Option aktivieren, wird GameMaker Studio 2 gezwungen, den Temp-Ordner, den es pro Projekt zum Kompilieren usw. anlegt, automatisch zu löschen. Diese Einstellung ist standardmäßig aktiviert, und wenn Sie das Häkchen entfernen, wird schaltet sie aus (was bedeutet, dass Sie später alle Temp-Dateien manuell entfernen müssen).
  • Asset-Cache beim Beenden automatisch löschen: Wenn diese Option aktiviert ist, wird der Asset-Compiler-Cache-Ordner entfernt, wenn Sie GameMaker Studio 2 beenden. Diese Option ist standardmäßig deaktiviert, und wenn Sie sie aktivieren, bedeutet dies, dass der Cache jedes Mal neu aufgebaut werden muss, wenn Sie ein Projekt laden und ausführen. Wenn Sie diese Option aktivieren, bedeutet dies, dass der Cache bei jedem Laden und Ausführen eines Projekts neu erstellt werden muss (was je nach Größe des Spiels einige Zeit dauern kann).
  • IDE-Cache beim Beenden automatisch löschen: Wenn dies aktiviert ist, wird der IDE-Compiler-Cache-Ordner entfernt, wenn Sie GameMaker Studio 2 beenden. Diese Option ist standardmäßig deaktiviert, und wenn Sie sie aktivieren, bedeutet dies, dass bei jedem Start von GameMaker Studio 2 starten, muss der Cache neu erstellt werden.
  • Temp-Ordner löschen: Wenn Sie auf diese Schaltfläche klicken, wird der Temp-Ordner für das Projekt gelöscht.
  • Asset-Cache löschen: Wenn Sie auf diese Schaltfläche klicken, wird der Compiler-Asset-Cache für das Projekt gelöscht.
  • IDE-Cache löschen: Durch Klicken auf diese Schaltfläche wird der IDE-Cache gelöscht

Es gibt folgende Möglichkeiten, Pfade zu externen Editoren einzustellen:

  • Pfad zum externen Editor/Viewer für SWF-Dateien: Wenn Sie mit Sprites im SWF-Format arbeiten, können Sie hier den Pfad zu Ihrem bevorzugten Viewer/Editor angeben, und wenn Sie im Sprite-Editor auf die Schaltfläche Bild bearbeiten klicken, wird wird das angegebene Programm geöffnet. Der Standardwert ist hier, dass kein Pfad angegeben wird.
  • Pfad zum externen Editor/Betrachter für Spine-Dateien: Wenn Sie mit Sprites im Spine-Format arbeiten, können Sie hier den Pfad zu Ihrem bevorzugten Viewer/Editor angeben, und wenn Sie im Sprite-Editor auf die Schaltfläche Edit Image klicken, wird wird das angegebene Programm geöffnet. Der Standardwert hier ist, dass kein Pfad angegeben wird.
