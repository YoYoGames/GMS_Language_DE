Marktplatz-Einstellungen

Die Marktplatz-Voreinstellungen werden verwendet, um das Marktplatz-Zertifikat für das Hochladen oder den Kauf von Assets einzurichten. Das Zertifikatsformular besteht aus den folgenden Abschnitten, wenn Sie bereits ein Zertifikat erstellt haben (siehe weiter unten auf dieser Seite, um zu erfahren, wie Sie ein Zertifikat erstellen):

  • Zertifikatsdatei: In diesem Abschnitt geben Sie die Datei *.pem an, die der Marketplace für die Zertifizierung von Assets verwendet. Wenn Sie ein zuvor erstelltes Zertifikat haben, können Sie hier auf die Schaltfläche Dateibrowser Wenn Sie jedoch kein Zertifikat haben und eines erstellen möchten, sollten Sie hier keine Angaben machen und stattdessen auf die Schaltfläche Neues Zertifikat erstellen klicken (siehe unten).
  • Zertifikatspasswort: Hier müssen Sie das Zertifikatspasswort für die oben ausgewählte Datei eingeben. Das Passwort wird beim erstmaligen Erstellen der Datei *.pem erstellt (siehe Neues Zertifikat erstellen, unten) und wird unwiderruflich mit der erstellten Zertifikatsdatei verknüpft.
  • Prüfen: Nachdem Sie die obigen Felder ausgefüllt haben, sollten Sie auf diese Schaltfläche klicken, um sicherzustellen, dass das Passwort und die Dateiauswahl korrekt sind und zusammenpassen.

Dialoge bestätigen

Hier können Sie das Verhalten der Bestätigungsdialoge beim Arbeiten mit einer Ressource aus dem Asset-Browser auswählen. Die verfügbaren Optionen sind:

  • Automatische Reaktion auf das Löschen von Offline-Inhalten: Wenn Sie ein Marktplatz-Asset ohne Webverbindung aus Ihrer Bibliothek löschen, erhalten Sie normalerweise eine Warnung mit der Frage, ob Sie sicher sind, dass Sie fortfahren möchten. Wenn Sie diese Option auf "OK" setzen wird die Meldung unterdrückt, als ob Sie auf die Schaltfläche "OK" geklickt hätten, um fortzufahren. Der Standardwert ist "Meldung anzeigen".

Ein neues Zertifikat erstellen

Wenn Sie noch keine Zertifikatsdatei für die Nutzung des Marktplatzes erstellt haben, sollten Sie auf die Schaltfläche Neues Zertifikat erstellen am unteren Rand dieses Formulars klicken. Daraufhin öffnet sich ein neues Fenster, in dem Sie aufgefordert werden die folgenden Informationen einzugeben:

  • Name des Herausgebers: Dies ist der Name, der als Publisher-Name für das Asset verwendet werden soll. Innerhalb des Marketplace-Dashboards können Sie bei Bedarf verschiedene Verleger einrichten, und jeder sollte seine eigene entsprechende *.pem Zertifikatsdatei haben.
  • Certificate Password: Hier müssen Sie ein eindeutiges Passwort für die zu erstellende Zertifikatsdatei angeben. Achten Sie darauf, was Sie wählen, da Sie es später nicht mehr ändern können, und wenn Sie nicht das richtige Passwort für die Zertifikatsdatei in den obigen Abschnitten angeben, können Sie keine vorhandenen Marketplace-Assets aktualisieren (und auch keine neuen hochladen).

Mit diesen Angaben können Sie dann auf die Schaltfläche Zertifikat erstellen klicken, um um die Datei *.pem zu erzeugen. Sie werden aufgefordert, einen Speicherort und einen Namen für die Datei anzugeben, den Sie sich für spätere Zwecke notieren sollten. Sobald Sie das getan haben, werden die neue Datei und das Passwort automatisch und Sie können auf die Schaltfläche Prüfen klicken, um zu testen, ob das Zertifikat korrekt generiert wurde.

HINWEIS: Wir empfehlen Ihnen, einen Screenshot der aktuellen Einstellungen sowie eine Kopie der Datei *.pem zu erstellen. Beide sollten zusammen an einem sicheren Ort gespeichert werden, damit Sie später darauf zurückgreifen können, oder für den Fall, dass Sie GameMaker Studio 2 zu einem späteren Zeitpunkt neu installieren müssen. Sie können keine Assets aktualisieren, die bereits hochgeladen wurden, es sei denn, das aktualisierte Paket ist mit demselben Zertifikat signiert, mit dem das Original erstellt wurde.
