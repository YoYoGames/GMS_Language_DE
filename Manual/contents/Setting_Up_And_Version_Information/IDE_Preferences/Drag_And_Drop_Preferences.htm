Drag And Drop™-Voreinstellungen

Mit den Drag and Drop™-Voreinstellungen legen Sie fest, wie der Drag and Drop™-Skripteditoraussieht und funktioniert. Die verschiedenen verfügbaren Optionen sind unten aufgeführt, zusammen mit einer Beschreibung der jeweiligen Funktion.

  • DnD™-Werkzeugkasten Standardposition: Standardmäßig befindet sich die DnD™-Toolbox(in der alle verschiedenen Aktionsbibliotheken enthalten sind) auf der rechten Seite des Editors, aber Sie können diese Option verwenden, um sie standardmäßig auf der linken Seite zu platzieren.
  • DnD™-Knotenübersicht-Sichtbarkeitsmodus: Eine Knoten-"Übersicht" ist der kurze beschreibende Text, der neben bestimmten Aktionen im Aktionsarbeitsbereich angezeigt wird, z. B. diese Aktion "Instanz zerstören":

    Sie können wählen, ob diese immer aktiviert werden sollen, ob sie nur aktiviert werden sollen, wenn die Aktion zugeklappt ist (die Standardeinstellung), oder sie ganz deaktivieren.
  • Automatische Reaktion auf die DnD™-Konvertierungin GameMaker Language: Wenn Sie ein Drag-and-Drop-Ereignis oder ein Script in GML-Code konvertieren, erhalten Sie standardmäßig eine Meldung, die Sie vor dem Vorgang warnt und fragt, ob Sie fortfahren möchten. Wenn Sie diese Option auf "OK" setzen, wird die Änderung einfach genehmigt, ohne dass die Meldung angezeigt wird.
  • Automatische Reaktion auf das Löschen von DnD™-Knoten: Wenn Sie einen Drag-and-Drop-Ereignisknoten löschen, erhalten Sie standardmäßig eine Warnmeldung über das, was Sie gerade tun, und werden gefragt, ob Sie fortfahren möchten. Wenn Sie diese Option auf "OK" setzen wird der Löschvorgang einfach genehmigt, ohne dass die Meldung angezeigt wird.
  • Maximale Größe für DnD™-Toolbox-Aktionsminiaturen: Unten in der Aktions-Toolbox haben Sie einen Schieberegler zum Ändern der Größe der Drag-and-Drop-Symbole. Mit dieser Einstellung wird die maximal zulässige Größe festgelegt (Standard 64px).
  • Mindestgröße für DnD™-Toolbox-Aktionsminiaturen: Unten in der Aktions-Toolbox haben Sie einen Schieberegler zum Ändern der Größe der Drag-and-Drop-Symbole. Mit dieser Einstellung wird die minimal zulässige Größe festgelegt (Standard 16px).
  • Standardgröße für DnD™-Werkzeugkasten-Aktionsminiaturen: Mit dieser Einstellung wird die Standardgröße für alle Aktionssymbole in der Toolbox festgelegt (Standard 32px).
  • Maximale Anzahl der zuletzt verwendeten Aktionen, die angezeigt werden sollen: In der Aktions-Toolbox befindet sich oben eine Bibliothek mit den Aktionen, die Sie zuletzt verwendet haben. Mit dieser Option können Sie die maximale Anzahl der zuletzt verwendeten Aktionen ändern, die angezeigt werden (Standardwert 5).
  • Dicke der Kette, die DnD™-Aktionen miteinander verbindet: Hier wird die Dicke der Kettengrafik festgelegt, die die verschiedenen Aktionen in einem DnD-Codeblock verbindet (Standard 3px).
  • Mausabwärtszeit zum Starten des Ziehens von Knoten (ms): Wenn Sie auf einen Knoten klicken und ihn ziehen, positionieren Sie den Knoten innerhalb des Aktionscodes neu. Wenn Sie jedoch die Maus einen Moment lang gedrückt halten und dann ziehen, verschieben Sie die Knotenposition innerhalb des Arbeitsbereichs, aber nicht innerhalb der Reihenfolge des Codes. Diese Einstellung legt die Anzahl der Millisekunden fest, die bei gedrückter Maustaste gewartet werden muss, bevor der Knoten verschoben (nicht neu positioniert) werden kann. Die Voreinstellung ist 350.
  • DnD™-Knotenübersichtsfeld ausblenden: Hier können Sie wählen, ob das Übersichtsfeld des Aktionscode-Editors ausgeblendet werden soll. Standardmäßig ist diese Funktion deaktiviert. Wenn Sie sie aktivieren, wird die Klartextliste der Aktionsketten aus dem Editor entfernt.

Es gibt auch ein Untermenü für DnD™-Kommentare:

Die Voreinstellungen für Kommentare steuern die verschiedenen Farb- und Formatierungsoptionen für alle Kommentare, die die zu Ihren Drag and Drop™-Aktionsskriptenhinzugefügt wurden. Diese Optionen sind:

  • Farbe: Die Standardfarbe für den Text, der in das Fenster geschrieben wird ( Standard-ARGB #FF53E270).
  • Schriftart-Name: Der Name der zu verwendenden Schriftart (die verfügbaren Schriftarten hängen von den auf Ihrem System installierten ab).
  • Stil: Der zu verwendende Schriftstil, wie z. B. fett oder kursiv (beachten Sie, dass nicht alle Fenster die gleichen Stile zulassen und es auch von der verwendeten Schriftart abhängt).
  • Größe: Die Größe, auf die die Schrift für das angegebene Ausgabefenster eingestellt werden soll.
  • Anti-Aliasing: Hier können Sie den Anti-Aliasing-Modus für das Rendern der Schriftart des Ausgabefensters einstellen. Sie können die IDE entscheiden lassen, indem Sie Standard verwenden wählen, oder Sie können es ein- oder ausschalten , indem Sie eine der beiden Optionen wählen.
