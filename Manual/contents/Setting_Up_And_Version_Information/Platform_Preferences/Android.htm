Android-Voreinstellungen

Die Android Einstellungen sind in verschiedene Abschnitte unterteilt, die im Folgenden aufgeführt sind:

Oben in den Voreinstellungen können Sie die maximale Größe des JDK-Speicher-Heaps festlegen (Standard ist 2GB). Als Nächstes müssen Sie die verschiedenen Pfade einrichten, damit GameMaker Studio 2 weiß, wo es nach den verschiedenen Tools suchen muss, um das endgültige ausführbare Paket zu erstellen und Ihr Spiel zu testen. Dies ist in drei Abschnitte unterteilt für das Android SDK, Android NDK und Java JDK. Diese Pfade sollten automatisch für Sie ausgefüllt worden sein, aber wenn Sie Probleme haben, können Sie auf die Schaltfläche rechts klicken, um den Datei-Explorer zu öffnen öffnen und die Speicherorte für die einzelnen Elemente manuell auswählen. Details zu den erforderlichen Versionen für die einzelnen SDK-Elemente finden Sie hier. Pfade sollten validiert werden, wenn Sie sie anwenden und eine Meldung zeigt "Gefunden" anoder informiert Sie über eventuelle Fehler (in diesem Fall sollten Sie alle Informationen überarbeiten, um sicherzustellen, dass sie korrekt sind).

Darunter haben Sie die verschiedenen Verpackungsoptionen, die da wären:

  • APK immer vollständig installieren: Wenn Sie dies markieren, entfernt GameMaker Studio 2 alle früheren Versionen Ihres Spiels von Ihrem Gerät sowie alle zugehörigen Dateien (z. B. *.ini ) von Ihrem Testgerät entfernt, bevor eine neue Version des Spiels installiert wird.
  • Auf Paket installieren: Wenn diese Option markiert ist, wird beim Kompilieren eines endgültigen ausführbaren *.apk Pakets dieses auch automatisch auf dem aktuell ausgewählten Zielgerät installiert.

Schließlich kommen wir zum KeyStore. Dies ist eine Datei, die verwendet wird, um alle Ihre Android-Apps zu "signieren ". Füllen Sie also bitte alle Details korrekt aus und denken Sie daran, dass diese Datei benötigt wird, um alle Ihre Android-Apps zu erstellen und zu aktualisieren in der Zukunft benötigt wird, d. h. nach der Einrichtung von es wird empfohlen, einen Screenshot der verwendeten Einstellungen zu machen und eine Sicherung der endgültigen Datei zu erstellen. Wenn Sie diese Datei verlieren, werden Sie nicht in der Lage sein, alle bestehende Spiele, die in den Store hochgeladen wurden, zu aktualisieren.

Um eine neue Keystore-Datei zu erstellen, sind die folgenden Angaben erforderlich:

  • Dateiname: Dies ist der Name der Keystore-Datei.

HINWEIS: Dies ist nicht Ihr Name oder der Firmenname, sondern der Name der zu erzeugenden Datei! Wenn Sie bereits eine Keystore-Datei aus früheren Projekten haben, können Sie hier auf die Schaltfläche "Dateibrowser" klicken und GameMaker Studio 2 auf die zuvor erstellte Keystore-Datei verweisen (oder auf die Schaltfläche " Importieren" klicken). In diesem Fall müssen Sie immer noch den Rest der erforderlichen Informationen ausfüllen (genau wie beim ersten Erzeugen der Datei), aber Sie dürfen nicht auf die Schaltfläche Keystore generieren ! klicken. Dadurch wird eine neue Keystore-Datei erzeugt und die vorherige überschrieben.

  • Allgemeiner Name: Dies wäre normalerweise Ihr Name.
  • Passwort: Ihr Sicherheitspasswort für die Keystore-Datei, das mindestens sechs Zeichen lang sein muss.
  • Alias: Dies ist ein weiterer Name, der für die Keystore-Datei verwendet werden kann und der sich von dem oben eingegebenen Namen unterscheiden sollte.
  • Alias-Passwort: Ihr Alias-Sicherheitspasswort für die Keystore-Datei, das mindestens sechs Zeichen lang sein muss.
  • Organisatorische Einheit: Die Abteilung innerhalb des Unternehmens, zu der Sie gehören.
  • Organisation: Der Name Ihrer Firma.
  • Ort: Der Name der Stadt, in der Sie sich befinden.
  • Land: Der zweistellige Standardcode für das Land, in dem Sie ansässig sind.

Wenn diese Informationen ausgefüllt sind und Sie damit zufrieden sind, drücken Sie die Schaltfläche " Keystore für GameMaker Studio 2 generieren ", um die erforderliche Keystore-Datei zu erstellen. Dies kann einen Moment dauern, aber wenn es fertig ist, muss es normalerweise nicht mehr geändert werden. nicht mehr geändert werden müssen. Beachten Sie, dass Sie auf die Schaltfläche " Importieren" klicken können, um eine zuvor erstellte Keystore-Datei zu importieren (siehe Abschnitt " Dateiname" oben), und Sie können auch den SHA1-Hash des Keystore abrufen, indem Sie auf die Schaltfläche " Schlüsselhash generieren " klicken. Wenn Sie darauf klicken, werden zwei spezielle Sicherheits-Hash-Schlüssel generiert, die direkt mit Ihrer Keystore-Datei verknüpft sind und die einige Websites für die korrekte Funktionalität benötigen (z. B. Facebook oder Google Play für den Sha1-Hash). Sollten Sie diese benötigen, drücken Sie einfach die und kopieren Sie dann die generierten Hash-Schlüssel und fügen Sie sie ein.
